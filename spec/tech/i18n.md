# tech: i18n seam

> As-built rules for cosimosi's UI string seam (web + mobile). The architectural
> frame lives in [ARCHITECTURE.md](../ARCHITECTURE.md) §3.1 (shared/i18n) and §5
> (English code, i18n'd copy with Paraglide); this doc is the detailed rulebook the
> foundation (plan/08) installed. Canonical domain terms remain owned by
> [ubiquitous-language.md](../ubiquitous-language.md) and never become message keys.

## 1. Packages and the app seam

| Concern | Location | Depends on |
|---|---|---|
| Message sources (canonical) | `packages/i18n/messages/{en,ko}.json` | inlang message-format plugin |
| inlang project config | `packages/i18n/project.inlang/settings.json` | — |
| Compiled message functions + runtime | `packages/i18n/src/gen/**` (committed, generated) | none at runtime (self-contained) |
| Platform-pure facade (`m`, locale store, `resolveLocale`) | `packages/i18n/src/{index,locale}.ts` | the generated runtime only |
| App i18n seam (`m` re-export + `useActiveLocale` hook) | `apps/{web,mobile}/src/shared/i18n/index.ts` | `@cosimosi/i18n`, `react` |
| Locale provider (negotiation) | `apps/{web,mobile}/src/app/i18n-provider.tsx` | platform locale sources |

Both apps consume `@cosimosi/i18n` directly. The package stays free of DOM/native
deps so the same `m.*` calls and locale store run verbatim on web, React Native,
and in Node tests. The React seam lives in each app's `shared/i18n` (mirroring the
state-machine `shared/model` seam) because locale **negotiation** is
platform-specific; the package owns only the pure parts.

## 2. Messages

- Add or change copy in `packages/i18n/messages/en.json` and `ko.json`. English is
  the base locale; every key exists in both files.
- Keys are snake_case and name a UI role, not a product noun
  (`app_greeting`, `common_retry`), never a domain term.
- Run `pnpm gen:messages` (also part of `pnpm gen`) to compile. The output lands in
  `packages/i18n/src/gen/`, is committed, and is freshness-checked by
  `pnpm check:gen` (a second run is byte-identical). The compiler's auto-emitted
  `.gitignore`/`README.md` are stripped so the output stays tracked and
  machine-independent; the inlang plugin cache (`project.inlang/cache/`) is
  gitignored.
- Scope is the platform seam, not product copywriting: hello-world remnants,
  auth/session proofing, harness labels, and common primitive labels. No
  marketing/blog localization (the Astro blog keeps its own pipeline), no
  server-side or AI translation, no DB-stored localized content.

## 3. Locale resolution and the active-locale store

`packages/i18n/src/locale.ts` owns an in-memory active-locale store and drives
Paraglide through `overwriteGetLocale`/`overwriteSetLocale`, so message functions
read the locale each app set — with no cookie/URL/DOM coupling. The compile uses
the `globalVariable` strategy precisely so the generated runtime references no
browser globals.

- `defaultLocale` is the inlang `baseLocale` (**en**); `supportedLocales` is
  `["en", "ko"]`.
- `matchLocale(tag)` maps a BCP-47-ish tag to a supported locale or `undefined`:
  exact, then primary subtag, case-insensitive, across both `-` and `_`
  separators ("en-US" / "ko_KR" → "en" / "ko").
- `resolveLocale(candidates)` returns the first candidate that matches, else the
  default. Providers pass candidates in precedence order, which keeps resolution
  pure and unit-testable **without** browser globals (the suites run in Node).
- `setActiveLocale` / `getActiveLocale` / `subscribeLocale` are the store API;
  `subscribeLocale` pairs with `useSyncExternalStore`.

Provider precedence:

- **Web** (`apps/web`): stored explicit choice (`localStorage["cosimosi.locale"]`)
  → `navigator.languages` → default. Negotiation runs in an effect (never during
  render), keeping render free of `window`/`navigator`. The provider also syncs
  `<html lang>` to the resolved locale.
- **Mobile** (`apps/mobile`): device locale (RN `NativeModules`, best-effort, no
  extra native dependency) → default. Wired into the app shell in Phase 1; final
  composition order is owned by job 13.

## 4. The raw user-facing string rule

`scripts/lint-raw-strings.mjs` (wired into `pnpm lint` as `lint:raw-strings`)
parses every `.tsx`/`.jsx` under `apps/{web,mobile}/src` with the TypeScript AST
and fails on user-facing copy that bypasses message functions:

- JSX text nodes containing letters (`<h1>hello</h1>`);
- string literals — bare or `{"..."}` — on user-facing attributes (`title`,
  `placeholder`, `alt`, `label`, `aria-label`/`aria-*`, `accessibilityLabel`/
  `accessibilityHint`).

Everything else is intentionally out of scope, which is how documented non-UI
cases stay quiet: developer logs, thrown `Error` messages, route ids,
`className`/`id`/`testID` and other structural attributes, `{m.x()}` expressions,
and domain names in model/proto/SQL (those are not `.tsx` JSX literals). Tests,
generated code, and stories are skipped by path. A line carrying `i18n-ignore`
opts that node out for a documented exception. `pnpm lint:raw-strings:probe`
self-tests the rule: it must catch a deliberate UI string and ignore the
documented cases beside it.

## 5. Design-system primitives

Primitives accept copy as **props** — labels, aria-labels, descriptions,
validation messages — and never embed product strings. Callers pass the result of
a message function; the primitive stays domain- and copy-agnostic. Shared primitive
examples may use message calls or test-only placeholders. (The primitives
themselves arrive with plan/09; this rule binds them.)

## 6. Non-goals (held by plan/08)

No product copywriting for final flows, no marketing/blog localization, no user
preference settings screen, no server-side/AI/DB translation, and no changes to
canonical domain terms.
