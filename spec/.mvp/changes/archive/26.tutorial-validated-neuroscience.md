# 26. 변경: 튜토리얼에 검증한 뇌과학 반영 (배속·작성흐름·형태 변화)

> [plan/48.demo-guided-tour](../plan/48.demo-guided-tour.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 스포트라이트 튜토리얼이 이번에 바뀐 체험(배속 연속 시계·프리셋 일기 작성·고유 형태/추상화·거리 드리프트·감정 링크)을
> 짚어 주도록 단계 문구·동작을 갱신한다. "별을 누르면 별 모양이 바뀌어요" 같은 안내를 넣는다.

## 현재 (as-is)

튜토리얼(`widgets/demo-tour`)은 자유모드 셸 위 스포트라이트 투어로, 10단계를 짚는다([plan 48](../plan/48.demo-guided-tour.md) §단계 시나리오):
UI 숨김/보이기 → 테마 → 페르소나 → **시간** → 시점 전환(+항해 실습) → 망원경 → 일기 탭 → 별 탭 → **새 별 띄우기** → **별 클릭/회상** → 끝.

이 중 다음이 옛 체험·옛 모델 기준이다:

- **시간 단계**: `한 달 후로 이동` **이산 버튼**을 짚고 "한 달 뒤로 이동했어요 — 오래 못 본 별은 빛이 바래요"로 안내한다(change 24로
  이산 버튼이 사라지므로 target·문구가 깨진다).
- **새 별 띄우기 단계**: "오늘 날짜의 랜덤 감정 별이 바로 태어난다"로 안내한다(change 25로 프리셋 일기 작성 흐름이 되므로 어긋난다).
- **별 클릭/회상 단계**: "별을 누르면 기억 조각/일기를 읽고 회상할 수 있다"까지만. 회상이 별의 **형태·추상화를 다시 빚는다**(spec 23·53·54)는 안내가 없다.
- **누락**: 거리=강함(연결 많은 별은 가까이), 감정 유사도 링크, 야간 자동 공고화("밤이 오면 우주가 스스로 정리") 같은 이번 검증 사실이 투어에 없다.

## 목표 (to-be)

튜토리얼 단계를 change 24·25의 새 체험과 jobs 32–38 검증 사실에 맞춰 갱신한다. 투어 FSM 아키텍처·스포트라이트 레이어·접근성은 그대로 두고,
**단계 target·문구·자동진행 조건·소수 정보 단계**만 바꾼다.

- **시간 단계 → 배속**: 시간 버튼을 짚고 배속 셀렉터(`1초=1시간`…`4시간`)를 안내한다 → 사용자가 배속을 올리면(행동 phase) 시간이 흐르며
  "오래 못 본 별은 **점점 멀어지고 어두워져요**(거리가 곧 강함)"를 보여주고, simulated 밤을 지나면 "**밤이 오면 우주가 스스로 정리돼요**
  (약한 연결은 빛만 낮추고, 멀어진 별은 형태가 더 단순해져요)" 정보 phase로 마무리한다. 이산 `한 달 후로 이동` 짚기는 제거한다.
- **새 별 띄우기 단계 → 작성 흐름**: "여기서 일기를 써요" → 누르면 read-only 프리셋 일기 폼이 열리고("일기를 쓰면 별이 태어나요") →
  `별 나누기`를 짚어 "하루는 한 감정이 아니에요 — 사건 조각마다 별이 나뉘어 태어나요"로 조각화 fan-out을 보여준다(change 25 흐름 재사용).
- **별 클릭/회상 단계 → 형태 변화**: 별을 누르면 기억을 읽고 회상하는 것에 더해, **"별을 누르면 별 모양이 조금씩 바뀌어요"** — 회상이
  별을 다시 빚어(재공고화) 형태·밝기가 변하고, 오래 멀어진 별일수록 형태가 추상화된다는 걸 짧게 안내한다(spec 23·53; 가능하면 내용 변형 spec 54도 한 줄).
- **검증 사실 보강(정보 단계, 최소 추가)**: (a) "자주 함께 떠올린·연결 많은 별은 서로 **가까이** 머물러요"(거리=강함), (b) "비슷한 **감정**의
  기억은 더 굵게 이어져요"(감정 유사도 링크). 단계를 과하게 늘리지 않고 기존 단계의 마무리 문구로 녹이거나 1~2개 정보 phase로만 추가한다.

## 범위 / 비목표 (회귀 경계)

**범위**: `widgets/demo-tour`의 step 정의(`model/steps.ts`)·문구·target·자동진행 조건과, 시간/새별/별클릭 단계가 change 24·25의 새
UI(배속 컨트롤·프리셋 작성 폼)를 가리키도록 잇는 부분. 정보 phase 소수 추가.

**비목표 (회귀 경계)**:
- 투어 FSM(`tour.machine.ts`) 구조, 스포트라이트/딤/코치카드 레이어, target anchor 계약(`data-tour-id`), 접근성(키보드·SR·reduced-motion)을 **재설계하지 않는다**.
- 미션·점수·보상·컷신을 만들지 않는다(plan 48 비목표 유지).
- 시점 전환 항해 실습(change 12) 동작·임계·제스처 규칙을 바꾸지 않는다.
- backend/proto/RPC/sqlc/DB 변경 없음. 실계정 우주엔 튜토리얼이 안 뜬다(유지).
- change 24(배속 시계)·25(프리셋 작성 흐름)에 **의존**한다 — 그 UI가 있어야 해당 단계가 가리킬 target이 존재한다(구현 순서: 24·25 → 26).
- 헌법: 별·시냅스·원본 삭제·수정 없음.

## 수용 기준 (인수 조건)

1. 시간 단계가 이산 `한 달 후로 이동`이 아니라 **배속 컨트롤**을 짚고, 사용자가 배속을 올리면 별이 멀어지고 어두워지는 변화를 보여주며,
   simulated 밤을 지나면 "밤이 오면 우주가 스스로 정리돼요" 안내로 마무리한다.
2. 새 별 띄우기 단계가 "랜덤 별"이 아니라 **프리셋 일기 작성 → `별 나누기` 조각화 fan-out**을 짚고, "사건 조각마다 별이 나뉘어 태어나요"를 안내한다.
3. 별 클릭/회상 단계에 **"별을 누르면 모양이 조금씩 바뀌어요"**(회상=재공고화로 형태/밝기 변화, 멀수록 추상화)가 포함된다.
4. 투어 어딘가에서 **거리=강함**("연결 많은 별은 가까이")과 **감정 유사도 링크**("비슷한 감정의 기억이 더 굵게 이어져요")가 안내된다(단계 과증식 없이).
5. 투어 FSM·스포트라이트 레이어·target anchor·접근성·항해 실습이 회귀 없이 동작하고, 행동/정보 phase 게이팅 규칙(`다음` 노출)이 유지된다.
6. 실계정 우주엔 튜토리얼이 나타나지 않고, `건너뛰기`/`자유롭게 탐험하기`/`다시 보기`/step 지속성이 그대로 동작한다.
7. 백엔드·proto·RPC·sqlc 변경 없이 FE만으로 구현되고, build·lint·typecheck·steiger 통과. plan 48 기존 단계(테마·페르소나·망원경·탭 등) 회귀 없음.
