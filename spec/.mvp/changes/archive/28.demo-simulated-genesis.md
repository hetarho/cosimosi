# 28. 변경: 데모 우주 = 30일 시뮬레이션 genesis (정적 프리셋 별 폐기·페르소나별 일기 확률 생성)

> [plan/47.demo-free-mode-onboarding](../plan/47.demo-free-mode-onboarding.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 데모 우주를 *t=0에 한꺼번에 떠 있는 정적 프리셋 코퍼스*에서 **매 세션 새로 사는 30일**로 바꾼다 — 페르소나마다 매일
> 일기 쓸 확률을 굴리고, 쓴다면 무엇을 쓸지 골라, production 엔진으로 우주가 하루씩 자라난다. 30일을 보낸 뒤에야 자유 작성이 열린다.

## 현재 (as-is)

데모 세션은 진입 즉시 페르소나 **정적 코퍼스**([shared/lib/demo/personas.ts](../../frontend/src/shared/lib/demo/personas.ts) — student 30편/80별,
worker 27편/54별, homemaker 30편/61별)를 한 번에 시드한다([data.ts](../../frontend/src/shared/lib/demo/data.ts) `ensureSeeded()` →
`simulate(CORPORA[persona])`). 각 일기는 `entryDaysAgo`(3~185일 전)로 고정돼 있어, 사용자는 **이미 다 자란 성숙한 우주**에 떨어진다.

문제는 그 프리셋이 실제와 괴리가 크다는 것이다. 별·연결·거리·밝기가 한 시점에 통째로 결정돼 있어, "하루하루 일기를 쓰며 우주가 자라난다"는
제품의 핵심 경험을 데모가 보여주지 못한다. 코퍼스는 매 세션 **동일**하고(같은 별·같은 배치), genesis 과정 — 빈 우주에서 별이 하나씩
태어나고, 밤마다 공고화되고, 거리/밝기가 드리프트하는 — 을 사용자가 볼 수 없다.

또 plan 47 자유모드는 진입 직후부터 `새 별 띄우기`로 별을 추가할 수 있다(첫 30일 자동 생성 개념 없음).

## 목표 (to-be)

데모 진입 상태를 **정적 시드**에서 **30일 시뮬레이션 genesis**로 바꾼다. 페르소나를 고르고 자유모드를 선택하면, 우주는 **빈(또는 거의 빈)
상태에서 시작**해 30일을 하루씩 살아가며 스스로 채워진다.

- **빈 우주에서 출발**: 페르소나 정적 코퍼스 일괄 시드(`simulate(CORPORA[persona])`)를 폐기한다. genesis 시작 시 우주는 비어 있고,
  시뮬레이션 day 1부터 별이 태어난다.
- **하루 루프(페르소나별 확률)**: simulated day마다 페르소나별 **일기 작성 확률**(`demo_genesis.daily_write_prob`)로 난수를 굴려 그 날
  일기를 쓰는지 결정한다. 하루 최대 1편(일기 1편은 production 조각화로 여러 별이 됨 — 별 개수는 조각 수가 결정). 페르소나마다 확률이 달라
  학생은 자주, 직장인은 드물게, 주부는 가장 자주 쓰는 식의 **다른 생활 리듬**이 나온다.
- **무엇을 쓸지(페르소나별 분포)**: 일기를 쓰기로 한 날, **새 경량 per-persona 템플릿 풀**에서 그 페르소나의 토픽·감정 분포에 맞춰 한 편을
  뽑는다. 풀은 사전분절(조각 텍스트 + 조각별 mood/intensity/valence)된 콘텐츠로, change 25의 `새 별 띄우기` 프리셋 작성 흐름과 **같은
  콘텐츠 출처**를 공유한다(genesis와 자유모드 프리셋이 한 풀을 본다). 매 세션 난수가 달라 **데모마다 다른 우주**가 자란다.
- **production 엔진으로 자라남**: 뽑힌 일기는 데모 전용 휴리스틱이 아니라 change 25(조각화 fan-out)·change 27(서버 동치 엔진 — 일내 결속·
  의미 KNN·감정 유사도 링크·거리=강함·밝기=반경)로 우주에 들어간다. 밤(04:00 KST)을 지날 때마다 change 24 연속 시계가 야간 공고화를
  발화한다. genesis는 같은 엔진을 30번 돌리는 것일 뿐이다.
- **가속 시계로 관전**: genesis는 change 24 연속 시계를 **빠른 배속**(`demo_genesis.hours_per_second`)으로 자동 재생해, 사용자가 하루하루 별이
  태어나고 밤마다 정리되는 30일을 **지켜본다**. 카메라 이동/줌/별 클릭(읽기·회상)은 되지만, **`새 별 띄우기`는 잠긴다**(첫 30일 별 추가 금지).
- **30일 후 명시적 환영 → 자유 작성 개방**: 30 simulated days를 마치면 시계가 멈추고 **환영 안내**(예: "30일을 함께 보냈어요 — 이제 당신의
  별을 띄워보세요")를 띄운 뒤 `새 별 띄우기`(change 25 프리셋 작성 흐름)와 사용자 배속 컨트롤이 열린다. 자유모드는 그때부터 평소대로 동작한다.
- **(선택) 회상으로 살아 있는 우주**: 리듬을 더 자연스럽게 하려면 페르소나별 **일일 회상 확률**(`demo_genesis.daily_recall_prob`)로 과거 별을
  다시 떠올려 거리/밝기 드리프트가 의미를 갖게 한다(production 회상 경로 재사용). 0이면 일기 작성만으로 genesis가 돈다.

> AI 부재의 정직한 경계: 데모엔 LLM/임베딩/서버가 없으므로 일기 *콘텐츠*와 *조각화*는 프리셋(template pool)이고, 의미 KNN은 클라 근사다
> (change 25·27이 정한 경계). 그 위의 **확률 루프·엔진·연결/밝기/드리프트·공고화**는 production과 동일 로직으로 돈다 — genesis는 데모
> 전용 새 공식을 만들지 않고, 같은 엔진에 30일치 입력을 흘려보내는 오케스트레이션일 뿐이다.

## 범위 / 비목표 (회귀 경계)

**범위**: 데모 진입 시드 경로(`ensureSeeded`/`simulate(CORPORA)`)를 30일 genesis 오케스트레이터로 교체, 페르소나별 일기-작성(+선택 회상)
확률 루프, 새 경량 per-persona 사전분절 템플릿 풀(change 25 프리셋과 공유), genesis 동안 `새 별 띄우기` 잠금, 30일 후 환영 안내 →
자유 작성 개방, genesis용 정전 노브(`spec/values.yaml` `demo_genesis.*`). genesis는 change 24 시계·25 fan-out·27 엔진 위에 얹는다.

**비목표 (회귀 경계)**:
- 백엔드·proto·RPC·sqlc·DB schema를 바꾸지 않는다. 데모는 서버·AI·임베딩을 호출하지 않는다. FE + values 생성물만으로 구현.
- **genesis 엔진 로직을 새로 만들지 않는다** — change 25/27이 정의한 production-동치 생성·연결·공고화·드리프트 함수를 *그대로 호출*한다.
  난수는 genesis **입력**(작성 여부·템플릿 선택·회상 대상)에만 있고, 엔진 함수는 입력이 같으면 결정론이다 → change 27 골든 parity를 안 깬다.
- 온보딩 구조(페르소나 선택 → 모드 선택)와 change 24 배속 컨트롤·야간 공고화 발화·change 25 작성 폼 자체를 재설계하지 않는다(genesis는 그
  위의 진입 시퀀스만 추가).
- 실계정(인증) 우주의 동작을 바꾸지 않는다. 데모 별/시냅스/genesis 상태는 실서버에 기록되지 않고 demo exit/페르소나 전환 리셋 경계에서 정리된다.
- 헌법 불변식 유지: 원본 일기 불변, 별·시냅스 삭제 없음, 서버 비좌표, `A_MIN=0.05` 단일 바닥, model 순수성.

## 수용 기준 (인수 조건)
1. 페르소나를 고르고 자유모드로 들어가면 정적 프리셋 코퍼스가 한꺼번에 뜨지 않고, 우주가 **빈(또는 거의 빈) 상태에서 시작**해 30일 genesis가 재생된다(`simulate(CORPORA[persona])` 일괄 시드 폐기).
2. genesis는 simulated day마다 페르소나별 `demo_genesis.daily_write_prob`로 난수를 굴려 그 날 일기 작성 여부를 정하고(하루 최대 1편), 페르소나마다 작성 빈도가 눈에 띄게 다르다.
3. 일기를 쓰는 날엔 **새 per-persona 사전분절 템플릿 풀**에서 그 페르소나 분포로 한 편을 뽑아, change 25 조각화 fan-out + change 27 엔진(일내 결속·의미 KNN·감정 유사도 링크·거리=강함·밝기=반경)으로 별·연결이 태어난다. 데모마다 뽑히는 일기가 달라 서로 다른 우주가 자란다.
4. genesis는 change 24 연속 시계를 `demo_genesis.hours_per_second` 배속으로 자동 재생하고, 30일 안의 매 밤(04:00 KST)에 야간 공고화가 발화해 별이 태어나고·정리되고·드리프트하는 과정이 보인다.
5. genesis 동안 `새 별 띄우기`는 잠겨 사용자가 별을 추가할 수 없다(카메라 이동/줌/별 클릭=읽기·회상은 허용).
6. 30 simulated days를 마치면 시계가 멈추고 **명시적 환영 안내**가 뜬 뒤 `새 별 띄우기`(change 25 작성 흐름)와 배속 컨트롤이 열려 자유모드가 평소대로 동작한다.
7. genesis 입력(작성 여부·템플릿 선택)에만 난수가 있고 엔진 함수는 결정론을 유지해 change 27 골든 parity 대조가 깨지지 않는다.
8. 페르소나 전환·`처음으로`는 genesis를 처음부터 다시 돌린다(가상 시계 0, 추가 별 0, 새 난수). demo exit 시 genesis 상태가 정리되고 실계정 우주에 섞이지 않는다.
9. 별·시냅스·원본 삭제·수정 없음, 서버 미호출, `A_MIN=0.05` 단일 바닥 유지. 실계정 우주엔 genesis가 나타나지 않는다.
10. 백엔드·proto·RPC·sqlc 변경 없이 FE + values 생성물만으로 구현되고 build·lint·typecheck·steiger 통과. 기존 데모 체험(회상·잠든 별·일기 목록·페르소나 전환·시간 흐름·자유모드 작성)·실계정 우주 회귀 없음.
