# 12. 변경: 항해 튜토리얼 — 디바이스·모드별 항해 실습 단계

> [plan/48.demo-guided-tour](../plan/48.demo-guided-tour.md)의 이미 구현된 스포트라이트 투어에, **항해하는 법**을 직접 해보며 익히는 실습 단계를 더한다. 멀리서(원거리)·가까이서(근거리) 두 모드 각각을, 마우스(웹)·터치(모바일)에 맞는 다른 문구·다른 동작으로 안내하고, 사용자가 실제로 움직이면 자동으로 다음으로 넘어간다.

## 현재 (as-is)
<!-- plan/48.demo-guided-tour 중 이 변경이 건드리는 현재 동작. 출발점. -->

plan 48의 투어는 자유모드 HUD 버튼을 하나씩 짚는 행동 안내형 스포트라이트 투어다. 그중 **시점 전환(`view`) 단계**는 단일 정보 phase로만 되어 있다 — `view` 버튼을 하이라이트하고 "멀리서 우주를 조망하거나, 별들 가까이서 탐험하는 시점을 오갈 수 있어요"라고 **말로만** 알려줄 뿐, 실제로 **어떻게 회전·줌·비행하는지**는 가르치지 않는다. 투어 어디에도 항해(카메라 운동) 조작을 직접 해보는 단계가 없다.

투어의 기존 행동 신호(`TourAwait`)는 `ui-hidden`·`ui-shown`·`persona-open`·`time-open` 넷뿐이고, `DemoGuidedTour`가 React prop(`uiHidden`·`popover`)으로 관찰해 자동 진행한다([steps.ts](../../frontend/src/widgets/demo-tour/model/steps.ts), [DemoGuidedTour.tsx](../../frontend/src/widgets/demo-tour/ui/DemoGuidedTour.tsx)). 투어 문구·단계는 **디바이스(터치/마우스)로 분기하지 않는다** — 모든 사용자가 같은 문구를 본다.

항해 조작 자체는 이미 구현돼 있고([policy/domain/navigation.md](../policy/domain/navigation.md)), 이 변경은 그걸 **관찰만** 한다:

- **멀리서 보기(`nebula`, 기본)**: 한 손가락(마우스) 드래그 = 아크볼 회전, 휠/핀치 = 줌, 두 손가락 = pan/zoom.
- **가까이서 탐험하기(`recall`)**: 한 손가락(마우스) 드래그 = 시선 회전(yaw/pitch), 두 손가락 세로 쓸기 = 전후진. 웹(비터치)은 **D-pad + `W/S/A/D`·방향키**로 전진하고(마우스 드래그는 시선만), 터치 기기엔 D-pad를 렌더하지 않는다.
- 모드 전환은 `view` 토글(`navigation.machine` `resetNonce` 증가)로 카메라 fly 보간된다.
- 연속 입력은 React state를 거치지 않고 `navigation-input` 싱글턴(`lookDelta` 누적·`thrust` 레벨)과 `useFrame`로 흐른다(헌법4). 디바이스 분기 헬퍼 `useCoarsePointer()`가 이미 있다([use-coarse-pointer.ts](../../frontend/src/shared/ui/use-coarse-pointer.ts)).

## 목표 (to-be)
<!-- 변경 후 참이어야 하는 동작·계약. 구체적으로. -->

plan 48 투어의 얇은 `view` 단계를 **항해 실습 segment**로 확장한다. 흐름은 하나의 자연스러운 동선이다:

1. **시점 전환 안내** (정보 phase, 현행 유지): `view` 버튼을 하이라이트하고 두 시점을 오갈 수 있음을 짧게 설명한다.
2. **멀리서 보기 실습** (기본 모드 `nebula`):
   - **회전**: "우주를 돌려보세요" — 웹=마우스로 드래그, 모바일=한 손가락으로 쓸기. 충분히 회전하면 자동으로 다음.
   - **줌**: "가까이 당겨보세요" — 웹=마우스 휠, 모바일=두 손가락으로 오므렸다 펴기(핀치). 줌 반경이 충분히 바뀌면 자동으로 다음.
3. **가까이서 탐험하기로 전환**: 투어가 `view` 토글을 안내(또는 자동 전환)해 `recall` 모드로 들어간다.
4. **가까이서 탐험하기 실습** (`recall`):
   - **시선**: "둘러보세요" — 웹=마우스 드래그, 모바일=한 손가락 드래그. 시선이 충분히 돌면 자동으로 다음.
   - **전진**: "별 쪽으로 나아가 보세요" — 웹=화면의 **D-pad 전진 버튼을 마우스로 누르기**(보조로 `W`/방향키 안내), 모바일=두 손가락으로 위로 쓸기. 충분히 전진하면 자동으로 다음.
5. 실습이 끝나면 다음 투어 단계(망원경 등)에 맞는 모드 상태로 정리한다.

각 실습 phase의 **문구는 `useCoarsePointer()`로 마우스/터치를 갈라** 다르게 보여준다 — 웹은 "마우스로", 모바일은 "손가락으로". 각 실습 phase는 **해당 동작이 실제로 일어나면(임계값 도달) 자동으로 다음 phase로 진행**한다. 단 plan 48의 "강제하지 않는다" 불변과 접근성(키보드/스크린리더)을 위해 `다음`·`건너뛰기` 탈출구는 그대로 유지한다.

새 행동 신호(`TourAwait`)를 추가한다: `nebula-rotated`·`nebula-zoomed`·`recall-looked`·`recall-thrusted`. 이 신호는 `navigation-input` 싱글턴과 항해 FSM의 현재 모드를 **rAF로 샘플링**해 임계값을 넘는 순간에만 충족으로 판정한다 — 매 프레임 React rerender를 만들지 않는다(헌법4). 임계값(회전 각·줌 반경 변화·시선 각·전진 거리)은 코드 하드코딩이 아니라 `spec/values.yaml` `demo_tour` 그룹의 튜닝값에서 온다.

이 변경은 [job 22 / plan 49(embodied-recall-navigation)](../plan/49.embodied-recall-navigation.md)가 구현 완료된 상태를 전제한다 — `recall`에서 "나"가 어깨 너머 앵커로 함께 항해하므로, 가까이서 실습 문구는 "내가 빛을 들고 별 사이를 지난다"는 체화된 1인칭 항해를 안내한다.

## 범위 / 비목표 (회귀 경계)
<!-- 무엇을 바꾸고, 무엇은 안 바꾸나. -->

**범위 (FE 전용)**

- plan 48 투어의 `view` 단계를 멀리서/가까이서 항해 실습 segment로 확장한다(회전·줌·시선·전진 = 모드당 핵심 2동작).
- 실습 phase 문구를 `useCoarsePointer()`로 마우스(웹)/터치(모바일)로 분기한다.
- 새 `TourAwait` 신호(`nebula-rotated`·`nebula-zoomed`·`recall-looked`·`recall-thrusted`)와, 이를 `navigation-input`·항해 FSM 모드에서 rAF 샘플링으로 관찰하는 경로를 더한다.
- 모드 전환(멀리서→가까이서)을 투어가 안내/구동하고, segment 종료 시 후속 단계용 모드 상태를 정리한다.
- 실습 성공 임계값을 `spec/values.yaml` `demo_tour` 그룹에 추가한다(`pnpm gen:values`).

**비목표 (안 바꾼다 — 회귀 경계)**

- **항해 조작·물리·제스처 규칙 자체를 바꾸지 않는다.** 회전/줌/look/thrust 입력, 속도·관성·벽·shake, 제스처 문법(`touch-action`·deadzone·NavPad fallback)은 [navigation.md](../policy/domain/navigation.md)·plan 06·change 08 그대로다. 튜토리얼은 기존 입력을 **관찰만** 한다.
- **job 22 / plan 49의 임베디드 recall 렌더·자아 앵커를 바꾸지 않는다** — 구현 완료를 전제로만 삼는다.
- plan 47 자유모드 HUD 구조, plan 48의 다른 투어 단계(UI 토글·테마·페르소나·시간·메뉴·망원경·탭·새 별·별 클릭·끝)는 바꾸지 않는다. 이 변경은 `view` 단계만 확장한다.
- 실계정 우주·일반 온보딩·강제 튜토리얼을 만들지 않는다. 항해 실습은 데모 튜토리얼모드 안에서만 나타난다.
- 별/시냅스/원본 일기 데이터를 수정·삭제하지 않는다. 실습은 카메라만 움직인다.
- 서버·proto·RPC·sqlc·DB schema를 바꾸지 않는다.
- **매 프레임 React state 구동 금지**(헌법4) — 항해 신호 관찰은 rAF 샘플링 + 임계 도달 시에만 setState.
- **3D 씬 안 `<Html>` 금지**(헌법8) — 투어 overlay는 캔버스 밖 DOM 그대로.
- 순수 step 정의/임계값 model은 React/DOM/three를 import하지 않는다(헌법4·8).

## 수용 기준 (인수 조건)
<!-- 변경 완료의 판정 기준. job의 "인수 조건" 섹션으로 복사된다. -->
1. 투어의 시점 전환 단계가 정보 안내 + 멀리서/가까이서 항해 실습으로 확장돼, `view` 버튼 설명 뒤에 실제 항해를 직접 해보는 phase들이 이어진다.
2. 멀리서 보기 실습은 **회전**과 **줌** 두 동작을 차례로 안내한다. 가까이서 탐험하기 실습은 **시선**과 **전진** 두 동작을 차례로 안내한다.
3. 각 실습 phase의 문구가 디바이스에 따라 다르다 — 터치 기기(`pointer: coarse`)에선 손가락 조작(드래그/핀치/두 손가락 세로 쓸기)으로, 비터치(웹)에선 마우스 조작(드래그/휠/D-pad 전진 버튼)으로 안내한다.
4. 사용자가 안내된 동작을 실제로 수행해 임계값을 넘으면 그 phase가 자동으로 다음으로 진행한다(회전 각·줌 반경 변화·시선 각·전진 거리 기준).
5. 웹 가까이서 실습의 전진은 D-pad 전진 버튼을 마우스로 눌러(보조로 `W`/방향키) 충족되고, 모바일은 두 손가락 세로 쓸기로 충족된다. 마우스 드래그는 가까이서에서 시선만 충족한다.
6. 각 실습 phase에서 사용자가 동작을 하지 않아도 `다음`·`건너뛰기`로 진행할 수 있다(plan 48의 비강제·접근성 불변 유지).
7. 멀리서→가까이서 전환이 실습 segment 안에서 이뤄지고(투어 안내/구동), segment가 끝나면 다음 투어 단계에 맞는 모드 상태로 정리된다.
8. 항해 신호 관찰은 `navigation-input`·항해 FSM 모드의 rAF 샘플링으로 이뤄지며, 임계 도달 순간에만 상태가 바뀐다 — 매 프레임 React rerender를 만들지 않는다(헌법4).
9. 실습 성공 임계값은 코드 하드코딩이 아니라 `spec/values.yaml` `demo_tour` 그룹에서 생성된 상수를 import해 쓴다.
10. 항해 조작·물리·제스처 규칙(navigation.md·plan 06·change 08), job 22 임베디드 recall 렌더, plan 47 자유모드 HUD, plan 48의 다른 투어 단계에 회귀가 없다.
11. 실계정 우주에는 항해 실습 단계가 나타나지 않고, 별/시냅스/원본 일기를 수정·삭제하지 않으며, 서버·proto·RPC·sqlc 변경이 없다(FE + values 생성물만).
12. 투어 overlay는 캔버스 밖 DOM 그대로이고, 순수 step/임계값 model은 React/DOM/three를 import하지 않는다. `prefers-reduced-motion`에서 실습 안내 전환이 과한 motion 없이 동작한다.
13. build·lint·typecheck·steiger를 통과한다.
