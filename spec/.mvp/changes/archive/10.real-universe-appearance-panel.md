# 10. 변경: 실제 우주 편집 패널 — 좌측/하단 resize + 감정색 마이페이지 이동

> [plan/44.customization-system](../plan/44.customization-system.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 홈 꾸미기 표면이 실제 우주가 아니라 별도 샘플 씬을 보여 어색한 문제를 해결한다. 선택 결과의 권위 있는
> 미리보기는 항상 메인 `UniverseCanvas`가 맡고, 편집 패널은 그 우주를 덮지 않고 레이아웃을 밀어낸다.
> STEPS는 /create-change-job이 만드는 job에.

## 현재 (as-is)
- 홈 좌상단 별가루·테마 알약을 누르면 `pages/home/ui/AppearanceModal`이 **전면 집중 모달**로 열린다.
  모달이 열리는 동안 `HomePage`는 실제 `UniverseCanvas`를 **언마운트**한다.
- 모달의 `스킨` 탭은 상단 고정 프리뷰 `FixedPreview`를 띄운다. 이 프리뷰는 실제 사용자 우주가 아니라
  `CosmosScene`으로 만든 **샘플 우주**(나 + 별 3개 + 강·약 시냅스)다. 항목 칩도 축별로 작은
  `CosmosScene`을 띄워 별도 3D 샘플을 보여준다.
- 선택을 바꾸면 appearance store의 라이브 드래프트가 바뀌지만, 사용자는 메인 우주가 아니라 모달 내부
  샘플 씬에서 주로 변화를 본다. 실제 우주는 모달이 닫힌 뒤 다시 마운트되어 보인다.
- 같은 모달 안에 최상위 `스킨 / 감정 색` 2탭이 있다. `감정 색` 탭은 `EmotionColorEditor`를 호스팅하고,
  저장 후 커밋되지 않은 스킨 드래프트를 `revertSelection`으로 되돌린다.
- 현재 `ViewOffsetController`는 꾸미기 모달에는 쓰이지 않는다(꾸미기 중 캔버스가 언마운트되기 때문).
  다만 코드와 문서상으로는 모바일 작성 시트(`setSheetOpen`), 데모 실험실 시트, 별 회상 bottom sheet,
  일기 조망 `DiaryCard`가 하단을 가릴 때 `camera.setViewOffset()`/`zoom`으로 우주를 위로 미는 데 여전히
  연결되어 있다.
- 진행 중인 job 18(plan 47)은 자유모드에서 `DemoSimPanel`을 기본 화면에서 제거할 예정이라 데모 실험실
  경로의 `setSheetOpen` 사용은 사라질 수 있다. 그러나 현재 코드 기준으로는 작성 `Surface`, 별 회상
  bottom sheet, 일기 조망 `DiaryCard`가 남아 있어 `ViewOffsetController` 전역 제거 가능 여부는 구현 시점
  실제 tree를 다시 감사해야 한다.

## 목표 (to-be)
- 좌상단 별가루·테마 알약은 전면 모달이 아니라 **실제 우주 편집 패널**을 연다. 패널이 열려도
  `UniverseCanvas`는 언마운트되지 않고, 선택 결과는 메인 우주에 라이브 드래프트로 즉시 반영된다.
- 데스크톱/fine pointer에서는 **좌측 사이드바**가 생기고, 우주 캔버스 영역은 그만큼 **폭이 줄어든다**.
  사이드바가 우주 위를 덮지 않는다. 캔버스는 기존 `ResizeObserver` 경로로 새 컨테이너 폭에 맞춰
  renderer size·camera aspect를 갱신한다.
- 모바일/coarse pointer에서는 **하단 편집 영역**이 생기고, 우주 캔버스 영역은 그만큼 **높이가 줄어든다**.
  하단 패널이 우주를 덮는 방식이 아니라, 남은 위쪽 영역에 실제 우주가 다시 layout/resize되어 보인다.
  safe-area를 포함해 마지막 저장 버튼과 항목 리스트가 홈 인디케이터에 가리지 않는다.
- 홈 꾸미기 표면에서는 **상단 고정 프리뷰를 제거**한다. "내가 고른 것이 어떻게 보이는가"의 답은 별도
  샘플 씬이 아니라 항상 실제 `UniverseCanvas`다.
- 항목 표시는 이름만 나열하지 않는다. 각 항목은 이름·짧은 설명·잠금/가격 상태와 함께 **가벼운 시각 토큰**을
  가진다. 토큰은 축별 카탈로그에서 파생한 식별 보조 수단이다:
  - 배경: 실제 배경 카탈로그의 색/질감 tile.
  - 별: 해당 별 form의 간단한 실루엣 또는 정적 아이콘.
  - 광원(나): 해당 self form의 간단한 실루엣 또는 정적 아이콘.
  - 시냅스: 해당 style의 선/입자 stroke 샘플.
  텍스트만 있는 목록은 수용하지 않는다. 반대로, 항목마다 별도 라이브 `CosmosScene` 우주를 띄워 "가짜 우주"를
  다시 만드는 것도 수용하지 않는다.
- 홈 꾸미기 표면은 **스킨만** 다룬다(배경/별/광원/시냅스 4축). `감정 색` 편집은 `/my-page`로 이동한다.
  마이페이지는 기존 `EmotionColorEditor`를 호스팅해 13감정 색을 저장할 수 있게 한다. 최초 설정 게이트
  `/emotion-colors`는 그대로 유지한다.
- 꾸미기 패널은 `ViewOffsetController`에 의존하지 않는다. 꾸미기 중에는 projection trick으로 우주를 밀지 않고,
  DOM 레이아웃이 캔버스 컨테이너 크기 자체를 바꾼다.
- 구현 작업 중 `ViewOffsetController` 관련 UX 사용처를 반드시 감사한다. 감사 대상은
  `ViewOffsetController`, `useViewport.sheetOpen`/`setSheetOpen`, `camera.setViewOffset`, `camera.clearViewOffset`,
  `SHEET_VIEW_SHIFT`, `SHEET_ZOOM`, 그리고 관련 문서 참조다. 감사 결과를 사용자에게 보고하고, 남은 실제 UX
  사용처가 없으면 같은 job에서 view-offset 코드를 제거한다.

## 범위 / 비목표 (회귀 경계)
- **범위**
  - `AppearanceModal`의 전면 모달/상단 프리뷰/스킨+감정색 탭 구조를 폐기하고, 홈 전용
    `AppearancePanel`(이름은 구현 재량)로 재구성한다.
  - `HomePage`를 꾸미기 열림 상태에서 split layout으로 렌더한다. 데스크톱은 좌측 패널 + 우측 캔버스,
    모바일은 상단 캔버스 + 하단 패널이다.
  - `UniverseCanvas`는 꾸미기 패널이 열려도 계속 마운트된다. 선택 드래프트가 실제 별/나/시냅스/배경에
    즉시 반영된다.
  - 항목 리스트는 시각 토큰 + 이름/설명/가격/잠금/선택 상태를 제공한다. 토큰은 실제 선택을 이해시키는
    보조 표현이고, 권위 있는 프리뷰는 메인 우주다.
  - `EmotionColorEditor`를 마이페이지에 추가하고, 홈 꾸미기 표면에서는 감정색 탭을 제거한다. 관련 문서
    plan 45와 policy를 함께 갱신한다.
- **비목표**
  - 별가루 경제·소유권·구매 검증·서버 계약(proto/RPC/DB)·가격(values)·드래프트 저장 모델은 바꾸지 않는다.
    저장 시 미구매 유료 아이템 일괄 구매, 잔액 부족 가드, 체험 무상 저장은 그대로다.
  - 4축 색 규율은 바꾸지 않는다: 별=mood, 시냅스=양끝 mood 블렌드, 나=ambient 파생, 배경=배경 자체 색
    + 감정색 짜임. 감정색 자체 판매/잠금은 여전히 금지다.
  - 랜딩/사인인/초대의 미인증 플레이그라운드 FAB 동작은 이번 변경의 직접 대상이 아니다. 단, 홈용 컴포넌트
    재구성 때문에 공용 `AppearanceControls` export가 흔들리면 기존 플레이그라운드 회귀가 없어야 한다.
  - `ViewOffsetController`의 **전역 제거는 조건부**다. 구현 시작/마감 시점의 실제 코드 감사에서 작성·회상·일기
    조망·데모 등 어떤 제품 UX도 더 이상 view offset에 기대지 않는다면 같은 job에서 제거한다. 아직 남은 UX
    사용처가 있으면 제거하지 않고, 남은 사용처와 보류 이유를 사용자에게 보고한다.
  - 제거하더라도 `useViewport` 전체를 무조건 없애지 않는다. 현재 `quietSettleSeq`/`requestQuietSettle`은 데모
    시간 이동 재안정화 신호로 별도 의미가 있으므로, view-offset용 `sheetOpen`만 사라질 수 있다.

## 수용 기준 (인수 조건)
1. 좌상단 별가루·테마 알약을 누르면 홈 꾸미기 표면이 열리지만 `UniverseCanvas`가 언마운트되지 않는다.
   기존처럼 별들이 재초기화되거나 샘플 프리뷰로 대체되지 않는다.
2. 데스크톱/fine pointer에서 꾸미기 표면은 **좌측 사이드바**로 나타나고, 우주 캔버스는 사이드바 오른쪽
   남은 폭으로 줄어든다. 사이드바가 우주 위를 덮는 fixed overlay가 아니다.
3. 모바일/coarse pointer에서 꾸미기 표면은 **하단 영역**으로 나타나고, 우주 캔버스는 위쪽 남은 높이로
   줄어든다. 하단 패널이 우주를 가리지 않으며, 캔버스 resize 후 renderer size·camera aspect가 정상 갱신된다.
4. 홈 꾸미기 표면에는 상단 고정 `FixedPreview`/샘플 `CosmosScene`이 없다. 선택 결과는 실제 메인 우주에서
   라이브로 확인한다.
5. 항목 리스트는 이름만 보여주지 않는다. 각 항목은 축별 시각 토큰(배경 tile, 별/광원 silhouette, 시냅스
   stroke 등)과 이름/설명/잠금/가격/선택 상태를 함께 보여준다. 항목마다 독립 라이브 3D 우주 캔버스를
   마운트하지 않는다.
6. 배경/별/광원/시냅스 탭 또는 축 전환으로 항목을 고르면 실제 `UniverseCanvas`가 즉시 드래프트 선택을
   반영한다. 저장 전에는 서버에 영속하지 않고, 저장/나가기/되돌리기 규칙은 기존과 같다.
7. 저장은 기존 경제 규칙을 지킨다. 미구매 유료 아이템이 있으면 합계 별가루를 표시하고, 저장 시 구매 후
   4축 `UpdateSettings`를 수행한다. 잔액 부족이면 저장이 막힌다. 체험은 전부 잠금 해제·무상 로컬 확정이다.
8. 홈 꾸미기 표면에서 `감정 색` 탭은 사라진다. `/my-page`에서 `EmotionColorEditor`로 13감정 색을 편집하고
   저장할 수 있다. 최초 온보딩 게이트 `/emotion-colors`는 그대로 동작한다.
9. 꾸미기 표면은 `useViewport.sheetOpen`이나 `ViewOffsetController`를 사용하지 않는다. 꾸미기 중 우주 이동은
   projection offset이 아니라 캔버스 컨테이너 resize로만 일어난다.
10. 구현 작업 중 `ViewOffsetController` 관련 사용처 감사를 수행하고 사용자에게 결과를 보고한다. 실제 UX 사용처가
    더 이상 없으면 `ViewOffsetController`, `sheetOpen`/`setSheetOpen`, `SHEET_VIEW_SHIFT`/`SHEET_ZOOM`,
    `camera.setViewOffset`/`clearViewOffset` 경로와 관련 문서를 제거·갱신한다. 남은 사용처가 있으면 제거하지 않고
    목록과 보류 이유를 남긴다.
11. 기존 플레이그라운드 FAB(랜딩/사인인/초대), 공유 우주 렌더, 구매/소유권/서버 선택 검증, 감정색 온보딩
    게이트에 회귀가 없다. FE build/lint/steiger가 통과한다.
