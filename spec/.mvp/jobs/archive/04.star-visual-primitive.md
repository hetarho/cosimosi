**A. 별 프리미티브 통합**
- [x] A2 별 4형태(deepfield/aurora/liquid/ember)의 geometry와 TSL 셰이딩이 `entities/star`의 **단일 통합 빌더 한 곳에서만** 정의된다. `forms.ts`/`single.ts`로 갈렸던 form 셰이더 중복이 제거된다.
- [x] A3 통합 빌더는 입력을 노드 접근자(`StarShadeInputs`)로 받아, **attribute 바인딩(인스턴스)·uniform 바인딩(단일)** 양쪽에서 같은 셰이더를 생성한다.
- [x] A4 우주 `StarField`는 통합 빌더(attribute 바인딩)를 쓰고, 별 그림결과(형태·mood색·밝기·자전·birth/focus/resonance)가 이전과 동일하다 — 코어 루프 시각 회귀가 없다.
- [x] A5 사인인·초대·랜딩의 별(`BrandMark`/`ThemedStar`/`Star3D`)이 통합 빌더(uniform 바인딩)를 쓰고, 현재 룩을 유지한다(자가발광 통일에 따른 directional 하이라이트 미세 정규화만 허용).
- [x] A6 통합 빌더는 위치·움직임을 만들지 않는다 — instance matrix, force-sim 좌표, 표상 부동(38), wobble, mesh spin은 전부 소비처가 주입한다.
- [x] A7 time(및 단일의 brightness) uniform의 소유·매 프레임 갱신이 소비처(StarField/Star3D)에 있고, 빌더는 노드 그래프만 반환하는 순수 함수다(uniform .value를 돌리지 않는다).
- [x] A8 통합 빌더 모듈은 앱 store·테마·force-sim·FSM·전송 계층을 import하지 않는다(`three`/`three/tsl`/`three/webgpu`만 의존). FSD 단방향·라이브러리화 전제를 충족한다.
- [x] A9 셰이더는 TSL, 우주 별은 InstancedMesh를 유지한다(헌법8). build·lint·typecheck·steiger(FSD) 게이트를 통과하고, 우주 코어 루프(작성→별 등장·회상·focus·항행)에 회귀가 없다.
**B. 랜딩 히어로 한-캔버스 합성 (feasibility)**
- [x] A11 랜딩 히어로가 별 본체와 배경을 **하나의 R3F 씬/WebGPU 컨텍스트**에 렌더한다 — 히어로 영역에서 동시에 떠 있는 WebGPU 컨텍스트가 1개다(이전: 별 캔버스 + 배경 분리).
- [x] A12 그 씬에서 배경 레이어가 **별 뒤와 별 앞 양쪽**에 존재하고(깊이/`renderOrder`로 정렬), 별이 앞 레이어 사이로 비친다 — "배경 일부는 별 앞으로·일부는 뒤로 흐른다"가 실제로 보인다.
- [x] A13 히어로 별은 A의 `buildStarBody`를 소비하며(별도 셰이더를 새로 만들지 않음), 위치·자전은 히어로 씬이 주입한다(프리미티브가 만들지 않는다 — 기준 5와 일관).
- [x] A14 랜딩의 나머지(배경·스크롤 시스템, 히어로 외 섹션)는 변경되지 않는다. 합성이 히어로 경계를 넘어 배경 시스템 전체로 번지면 plan 43으로 분리한다(이번 인수에서 제외).

**A. 별 프리미티브 통합**
- [x] T001 통합 빌더 모듈 신설 `entities/star/ui/star-body.ts` — `StarShadeInputs`(mood·brightness·seed·hueShift·time, 전부 TSL 노드) + `buildStarBody(object, inputs): { geometry, material }`. forms.ts 4형태 셰이딩을 이식, 입력은 `inputs` 노드 사용. **순수 함수**(uniform/`.value` 변경 없음, time도 inputs).
- [x] T002 조명 자가발광 통일 — 4형태를 forms.ts의 emissive/fresnel(ndv) 기반으로 정전화. single.ts의 directional 전제 튜닝은 emissive 경로로 수렴(같은 모듈). + form별 mesh-spin은 `STAR_FORM_SPIN` 메타데이터로 노출(자전 *적용*은 소비처).
- [x] T003 `StarField.tsx` 이주 — `aMood/aBrightness/aSeed/aHueShift` attribute + StarField 소유 `uTime`(update 클로저)으로 `buildStarBody` 소비. brightness/seed attribute 값 계산은 StarField에 그대로 유지. 셰이더 노드는 forms.ts와 동일 → 출력 동일.
- [x] T004 `widgets/star3d/ui/Star3D.tsx` 이주 — `uniform(Color)`·`uniform(bright)`·`float(0)`(hueShift)·`float(seed)`·`uniform(0)`(time)으로 `buildStarBody` 소비. time·brightness uniform 소유·갱신을 Star3D `useFrame`으로(반환 shape `{geometry,material,spin,update}` 보존해 나머지 무변경).
- [x] T005 중복 제거 — `single.ts`·`forms.ts` 삭제. `entities/star/index.ts` 공개 API 정리(`buildStarBody`·`STAR_FORM_SPIN`·`StarShadeInputs`·`StarBodyBuild` 노출, `buildSingleStar`/`SingleStarBuild` 제거). `@x/*`는 타입만 노출이라 영향 없음.
- [x] T006 [P] 게이트 — `tsc -b`·eslint·steiger 통과. (우주 출력 동일·showcase 렌더는 시각 확인 권장 — 아래 검증 참조.)

- [x] T008 히어로 씬 신설 `pages/landing/ui/section/HeroScene.tsx` — `StarCanvas`(한 R3F 캔버스·fit 카메라·dispose 재사용) 안에 (1) **배경 뒤**(radial fluid, renderOrder −10), (2) **별**(`Star3D`=buildStarBody uniform 소비, 위치·자전 주입), (3) **배경 앞**(foreground additive fluid, renderOrder +10). 깊이는 renderOrder로 정렬.
- [x] T009 `HeroSection.tsx` 배선 — `<ThemedStar>` 대신 `<HeroScene>`. 테마 accent·object는 HeroSection이 읽어 prop 전달. reduced-motion이면 배경 시간 정지.
- [x] T010 [P] 게이트 — build·eslint·steiger 통과. 구조상 히어로 WebGPU 컨텍스트 1개(A11)·앞뒤 레이어(A12)·랜딩 나머지 무변경(A14). 레이어 가시성은 시각 확인 권장.
