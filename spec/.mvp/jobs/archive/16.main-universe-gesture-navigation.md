## 인수 조건 (acceptance — 기획 관점, changes/08.main-universe-gesture-navigation에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [x] A1 **용어 교체.** 메인 우주 화면에서 카메라 모드 이름, 버튼 title, aria-label, 안내 copy에 `성운`/`회상`이 카메라 모드명으로 노출되지 않는다. 토글은 **멀리서 내 우주 보기** 와 **별들 가까이서 탐험하기** 로 표시된다.
- [x] A2 **회상 도메인 보존.** 별 선택 후 2초 dwell로 `RecallMemory`가 발생하는 도메인 행위 이름과 동작은 그대로다. 카메라 모드 이름 변경이 회상 패널, 공동 회상, 잠든 별 재점화에 회귀를 만들지 않는다.
- [x] A3 **근접 광원 이동.** 별들 가까이서 탐험하기 모드에서 카메라/탐험자가 이동하면 `StarField.selfLightPos`가 같은 기준 위치로 갱신되어 별 표면 반사가 함께 이동한다. 원거리 모드에서는 중심 자아 광원 규칙이 유지된다.
- [x] A4 **광원 채널 경계.** 이동 광원은 반사 채널만 바꾼다. `selfGlow`, `activation`, `λ_eff`, 별 색, 별 좌표, `A_MIN` 밝기 바닥은 변하지 않는다.
- [x] A5 **근접 한 손가락 look.** 별들 가까이서 탐험하기에서 한 손가락/마우스 드래그가 고개 회전으로 동작한다. 좌우는 yaw, 상하는 pitch이며, 기존 화살표 버튼과 같은 방향 감각이다.
- [x] A6 **근접 두 손가락 전후진.** 별들 가까이서 탐험하기에서 두 손가락을 올린 뒤 위로 움직이면 전진, 아래로 움직이면 후진한다. 추력은 deadzone 이후 이동량에 비례해 ramp하고, 손을 떼면 기존 관성/제동으로 멈춘다.
- [x] A7 **근접 버튼 비주도화.** 터치 지원 모바일에서 전진/후진/화살표 `NavPad`가 기본 조작 표면으로 고정 노출되지 않는다. 데스크톱 키보드와 접근성 fallback은 유지된다.
- [x] A8 **원거리 한 손가락 회전 유지.** 멀리서 내 우주 보기에서 기존 한 손가락/마우스 arcball 회전, wheel zoom, pinch zoom은 회귀 없이 동작한다.
- [x] A9 **원거리 두 손가락 pan.** 멀리서 내 우주 보기에서 두 손가락을 같은 방향으로 움직이면 화면 평면 기준으로 camera와 target이 함께 이동해 조망 기준점이 중앙에 고정되지 않는다.
- [x] A10 **원거리 double-tap-hold zoom.** 멀리서 내 우주 보기에서 double-tap-hold 후 위로 드래그하면 zoom in, 아래로 드래그하면 zoom out이 된다. 이 gesture가 lock된 동안 pan은 발생하지 않는다.
- [x] A11 **제스처 충돌 방지.** 탭/드래그/별 선택/두 손가락/pinch/double-tap-hold가 lock 규칙으로 분리된다. 별을 탭하면 선택되고, 드래그 중 별이 잘못 선택되지 않으며, 두 손가락 gesture가 `onPointerMissed` dismiss를 유발하지 않는다.
- [x] A12 **브라우저 gesture 제어.** 우주 캔버스는 Pointer Events 기반으로 동작하고, 캔버스 표면에 `touch-action: none` 또는 동등한 처리를 적용해 브라우저 스크롤/핀치와 커스텀 gesture가 충돌하지 않는다. 차단 범위는 우주 캔버스에 한정된다.
- [x] A13 **전환/포커스 중 stand down.** `flyingToStar`, `framingDiary`, `modeTransition`, 별 focus 중에는 사용자 gesture가 유도 비행/포커스 컨트롤러와 싸우지 않는다. 전환이 끝나면 해당 모드의 gesture가 다시 활성화된다.
- [x] A14 **값 단일 출처.** gesture deadzone, sensitivity, double-tap 시간/거리, pan/zoom 속도는 `spec/values.yaml`에서 생성된 `VALUES`를 사용한다. 새 하드코딩 튜닝 숫자가 없다.
- [x] A15 **성능/아키텍처.** pointermove가 React state 리렌더를 유발하지 않는다. 연속 입력은 ref/useFrame 경로로 처리되고, `shared/lib`/model 순수 모듈은 three/React/DOM을 import하지 않는다.
- [x] A16 **모바일 검증.** Playwright 또는 수동 모바일 시뮬레이션으로 최소 390x844와 데스크톱에서 다음을 확인한다: 근접 look, 근접 두 손가락 전/후진, 원거리 pan, 원거리 zoom scrub, 별 탭 선택, 빈 우주 탭 dismiss.
- [x] A17 build·lint·steiger·관련 vitest가 통과하고, 우주 셸 첫 로드/빈 우주/렌더러 fallback/일기 조망/fly-to에 회귀가 없다.
