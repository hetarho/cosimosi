## 인수 조건 (acceptance — 기획 관점, plan/44.customization-system에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [x] A1 인증 사용자가 `GetInventory`를 처음 호출하면 별가루 잔액 `starting_stardust`(=100)가 시드되어 반환되고, 소유 아이템은 빈 목록이다(무료 종은 응답에 없음 — 묵시 소유).
- [x] A2 `PurchaseItem`은 한 트랜잭션에서 (a) 알 수 없거나 무료인 아이템 → `InvalidArgument`, (b) 이미 소유 → `FailedPrecondition`, (c) 잔액 < 가격 → `FailedPrecondition`로 거부하고 **어떤 행도 바꾸지 않는다**. 통과 시 잔액이 가격만큼 줄고 소유 집합에 해당 id가 추가된 새 상태를 반환한다.
- [x] A3 별가루 잔액은 구매로만 줄고 음수가 되지 않으며, 충전·재설정 경로는 존재하지 않는다(이번 plan 비목표 — 시작 100 외 증가 수단 없음).
- [x] A4 `UpdateSettings`로 **소유하지 않은(무료도 아닌) 유료 아이템**을 선택하려 하면 `FailedPrecondition`(ErrNotOwned)로 거부되고 선택이 바뀌지 않는다. 무료 종·소유한 유료 종은 정상 선택·영속된다.
- [x] A5 선택은 **4축 전부**(배경=`theme`·별=`star_object`·나=`self_object`·시냅스=`synapse_style`) 서버에 저장되어 새로고침·타 기기에서 같은 선택으로 로드된다(나 축이 더는 기기-로컬이 아니다).
- [x] A6 별 색은 mood(13감정) 팔레트로만 정해지고, 어떤 별 모양 아이템을 선택해도 색이 감정을 따른다(모양만 바뀐다 — "감정의 색" 보존).
- [x] A7 자아 "나" 별의 **몸체 색**은 요즘 감정(ambient mood)에서 파생되고 테마/배경에 귀속되지 않는다. 데이터 없음·미인증이면 중립/배경 accent로 폴백한다. 자아 별이 **다른 별에 던지는 빛은 중립색**으로 남는다(spec 03 보존 — mood 이중 주입 없음).
- [x] A8 시냅스 스타일을 아이템으로 선택할 수 있고, 색은 여전히 양끝 별 mood 블렌드이며 weight→밝기/alpha/펄스 시각과 삭제금지(헌법2) 불변식이 유지된다.
- [x] A9 배경 아이템은 색만이 아니라 텍스처/요소를 포함한 backdrop 번들로 적용되고, 배경 변경이 별의 mood 색을 바꾸지 않는다.
- [x] A10 스위처가 4축을 보여주고 잔액·잠금 배지·가격·구매 버튼을 노출한다. 무료/소유 아이템만 선택 가능하고, 잠긴 아이템은 구매 후 선택된다. 잔액 부족이면 구매가 막힌다.
- [x] A11 축마다 정확히 1종이 무료(묵시 소유)이고 나머지는 유료(잠금)다 — 무료 1종 매핑(배경 vast·별 deepfield·나 nebula-heart·시냅스 filament)이 values 생성 상수로 단일 출처화된다.
- [x] A12 랜딩/사인인 플레이그라운드에서 미인증자가 4축 전부(유료 포함)를 자유로이 바꿔 미니 코스모스(배경+별+나 앵커+시냅스 표본)에 즉시 반영되며, 아무것도 서버에 저장되지 않고 별가루도 차감되지 않는다.
- [x] A13 미인증·오프라인·체험·로딩 중에도 4축 클라 기본값으로 렌더가 깨지지 않고, 알 수 없는/잠긴 선택은 축 기본값으로 폴백한다.
- [x] A14 가격·무료여부·시작 잔액은 `spec/values.yaml`에서 생성된 상수로 FE/BE가 공유한다(FE/BE 하드코딩·이중 출처 없음). 아이템 id는 `"<axis>:<kind>"` 안정 식별자다.
- [x] A15 모든 신규 RPC(GetInventory·PurchaseItem)와 확장 RPC(UpdateSettings)는 미인증이면 `Unauthenticated`로 거부하고 unary다(헌법6). build·lint·vet·steiger 통과, 우주 셸·기존 화면 회귀 없음.
