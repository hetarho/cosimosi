- [x] A1 루트 `/` 우주 셸의 우측 상단 최상단에는 햄버거 버튼이 있고, 누르면 우측 사이드바가 열린다.
- [x] A2 사이드바에는 `로그아웃`, `마이페이지`, 구분선, `우주 공개`, `주고받은 별`, 구분선, `일기`가 이 순서로 표시된다.
- [x] A3 사이드바의 `로그아웃`은 기존 sign-out과 동일하게 세션·우주 데이터·일기 캐시 출처 경계를 리셋한다. 우주 셸에서는 별도 우상단 로그아웃 pin이 보이지 않는다.
- [x] A4 사이드바의 `우주 공개`와 `주고받은 별`은 기존 기능 본문을 열며, 서버 계약과 공개/선물 도메인 동작은 변하지 않는다.
- [x] A5 햄버거 버튼 아래에는 시점 전환 버튼, 그 아래에는 망원경 버튼이 세로로 배치된다. 시점 전환은 기존 카메라 모드 토글 동작을 유지한다.
- [x] A6 망원경 버튼을 누르면 모바일에서는 바텀시트, 데스크톱에서는 기존 overlay-shell 원칙에 맞는 비차단 탐색 표면이 열리고, `일기`와 `별` 탭을 제공한다.
- [x] A7 망원경 `일기` 탭은 날짜·본문 발췌 검색과 감정/날짜 필터를 제공하고, 일기 선택 시 기존처럼 그 일기의 별들을 frame-all 조망 및 강조한다.
- [x] A8 망원경 `별` 탭은 일반 별과 잠든 별을 한 목록에서 보여주며, 감정/날짜/잠든 상태 필터를 제공한다. 별 선택은 기존 `FLY_TO_STAR` 후 회상 패널로 수렴한다.
- [x] A9 제품 UI에서 별도 `잠든 별` 메뉴/패널 진입점은 제거되고, 잠든 별 탐색은 망원경 `별` 탭으로 통합된다.
- [x] A10 사이드바의 `일기`는 신규 보호 일기 페이지로 이동한다. 이 페이지는 일기 목록, 검색, 날짜/감정 필터, 읽기 전용 상세, `우주에서 보기` 동선을 제공한다.
- [x] A11 일기 페이지와 망원경 일기 탭 모두 원본 일기를 수정·삭제하지 않는다. 운영 코드에 `UPDATE records` 또는 `DELETE FROM records` 성격의 흐름을 새로 만들지 않는다.
- [x] A12 일기 상세의 `우주에서 보기`는 루트 우주 셸로 돌아와 해당 record의 별들을 조망한다. 우주 캔버스의 좌표 권위는 여전히 클라이언트 force-sim이다.
- [x] A13 상단 중앙 `UI 숨기기`를 누르면 그 버튼을 제외한 우주 HUD와 열린 표면이 숨겨지고, `UI 보이기`를 누르면 기본 HUD가 복귀한다.
- [x] A14 HUD 숨김 상태에서도 우주 캔버스 직접 조작과 별 클릭은 가능하다. 숨김/보이기 전환으로 WebGPU 캔버스가 재초기화되지 않는다.
- [x] A15 좌상단 테마 버튼은 유지되고, 열리는 꾸미기 표면에는 단독 헤더 텍스트 `외형`이 보이지 않는다.
- [x] A16 꾸미기 표면은 `스킨`과 `감정 색`으로 나뉜다. `스킨`은 기존 4축 커스터마이즈와 구매/저장 규칙을 유지하고, `감정 색`은 13개 mood 색을 기존 full-set 저장 규칙으로 편집한다.
- [x] A17 하단 중앙 floating `새 별 띄우기` 버튼이 현재 작성 폼을 열고, 햄버거 사이드바에는 작성 항목이 없다.
- [x] A18 어떤 진입/표면 전환에서도 별·시냅스 렌더링, 회상, 공유, 선물, 감정색, 외형 저장의 기존 도메인 불변식이 깨지지 않는다.
- [x] A19 FSD 임포트 방향과 캔버스 밖 DOM HUD 원칙이 유지된다. 3D 씬 안에 `<Html>`을 추가하지 않고, 순수 model은 React/DOM/three에 의존하지 않는다.
- [x] A20 build·lint·typecheck·steiger를 통과하고, 기존 `/?fly=`, 일기 frame-all, 별 회상, 우주 공개, 주고받은 별, 감정색 최초 게이트에 회귀가 없다.
