- [x] A1 좌상단 별가루·테마 알약을 누르면 홈 꾸미기 표면이 열리지만 `UniverseCanvas`가 언마운트되지 않는다. 기존처럼 별들이 재초기화되거나 샘플 프리뷰로 대체되지 않는다.
- [x] A2 데스크톱/fine pointer에서 꾸미기 표면은 **좌측 사이드바**로 나타나고, 우주 캔버스는 사이드바 오른쪽 남은 폭으로 줄어든다. 사이드바가 우주 위를 덮는 fixed overlay가 아니다.
- [x] A3 모바일/coarse pointer에서 꾸미기 표면은 **하단 영역**으로 나타나고, 우주 캔버스는 위쪽 남은 높이로 줄어든다. 하단 패널이 우주를 가리지 않으며, 캔버스 resize 후 renderer size·camera aspect가 정상 갱신된다.
- [x] A4 홈 꾸미기 표면에는 상단 고정 `FixedPreview`/샘플 `CosmosScene`이 없다. 선택 결과는 실제 메인 우주에서 라이브로 확인한다.
- [x] A5 항목 리스트는 이름만 보여주지 않는다. 각 항목은 축별 시각 토큰(배경 tile, 별/광원 silhouette, 시냅스 stroke 등)과 이름/설명/잠금/가격/선택 상태를 함께 보여준다. 항목마다 독립 라이브 3D 우주 캔버스를 마운트하지 않는다.
- [x] A6 배경/별/광원/시냅스 탭 또는 축 전환으로 항목을 고르면 실제 `UniverseCanvas`가 즉시 드래프트 선택을 반영한다. 저장 전에는 서버에 영속하지 않고, 저장/나가기/되돌리기 규칙은 기존과 같다.
- [x] A7 저장은 기존 경제 규칙을 지킨다. 미구매 유료 아이템이 있으면 합계 별가루를 표시하고, 저장 시 구매 후 4축 `UpdateSettings`를 수행한다. 잔액 부족이면 저장이 막힌다. 체험은 전부 잠금 해제·무상 로컬 확정이다.
- [x] A8 홈 꾸미기 표면에서 `감정 색` 탭은 사라진다. `/my-page`에서 `EmotionColorEditor`로 13감정 색을 편집하고 저장할 수 있다. 최초 온보딩 게이트 `/emotion-colors`는 그대로 동작한다.
- [x] A9 꾸미기 표면은 `useViewport.sheetOpen`이나 `ViewOffsetController`를 사용하지 않는다. 꾸미기 중 우주 이동은 projection offset이 아니라 캔버스 컨테이너 resize로만 일어난다.
- [x] A10 구현 작업 중 `ViewOffsetController` 관련 사용처 감사를 수행하고 사용자에게 결과를 보고한다. 실제 UX 사용처가 더 이상 없으면 `ViewOffsetController`, `sheetOpen`/`setSheetOpen`, `SHEET_VIEW_SHIFT`/`SHEET_ZOOM`, `camera.setViewOffset`/`clearViewOffset` 경로와 관련 문서를 제거·갱신한다. 남은 사용처가 있으면 제거하지 않고 목록과 보류 이유를 남긴다.
- [x] A11 기존 플레이그라운드 FAB(랜딩/사인인/초대), 공유 우주 렌더, 구매/소유권/서버 선택 검증, 감정색 온보딩 게이트에 회귀가 없다. FE build/lint/steiger가 통과한다.

## 구현 노트
- **view-offset 감사 결과(T001·T010·A10):** `ViewOffsetController`는 **제거하지 않고 유지**한다. 실제 UX 사용처 3곳이 남아 있다 — ① 작성 표면(`HomePage`의 `setSheetOpen(composeOpen && !uiHidden)`), ② 회상 포커스(`ViewOffsetController`가 `focusActor` 선택 별을 직접 구독), ③ 일기 조망 카드(`diaryFramed`). 따라서 `sheetOpen`/`setSheetOpen`/`SHEET_VIEW_SHIFT`/`SHEET_ZOOM`/`setViewOffset`/`clearViewOffset` 경로 모두 보존. 꾸미기 패널(`AppearancePanel`)만 view-offset을 쓰지 않고 캔버스 컨테이너 resize로 동작(A9). `DemoSimPanel`은 자유모드에 미마운트(plan 47/job 18)라 그 경로의 view-offset 사용은 이미 비활성. `quietSettleSeq`/`requestQuietSettle`(데모 시간 이동 재안정화)도 유지.
- **AppearanceControls 재사용:** 새 패널은 항목 토큰을 재구현하지 않고 공용 `features/switch-appearance`의 `AppearanceControls`(`draft` 모드)를 호스팅한다 — swatch 토큰 + 이름/설명/잠금/가격(A5). 플레이그라운드 FAB(`playground` 모드)와 같은 본문이라 회귀 위험이 낮다(T011).
- **저장 모델 무변(parity):** 부분 구매 후 실패 시 사용자 피드백 부재(`console.error`만)는 기존 `AppearanceModal`에서 그대로 이식한 동작이다 — change 10 비목표("드래프트 저장 모델은 바꾸지 않는다")라 의도적으로 유지(향후 별도 change 후보). 잔액 가드는 합계 `pendingCost` 기준으로 선검사(무변).
