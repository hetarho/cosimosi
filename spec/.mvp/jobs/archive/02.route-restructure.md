## 인수 조건 (acceptance — 기획 관점, changes/02.route-restructure에서)

- [x] A1 로그아웃 상태로 `/`에 진입하면 `/sign-in?redirect=/`로 리다이렉트되고 사인인 화면이 보인다. 인라인 사인인은 더 이상 어떤 보호 라우트 URL에서도 렌더되지 않는다.
- [x] A2 로그아웃 상태로 `/admin`·`/gift/$token`에 진입하면 `/sign-in?redirect=<해당 경로>`로 리다이렉트된다.
- [x] A3 `/sign-in`에서 이메일 OTP 검증 또는 Google OAuth 복귀로 인증되면 `redirect` 경로(없으면 `/`)로 이동하고 우주 셸이 렌더된다.
- [x] A4 이미 인증된 사용자가 `/sign-in`을 직접 열면 `/`(또는 `redirect`)로 곧장 이동한다.
- [x] A5 `/landing`은 세션 없이 열리고 기존 랜딩(배경·히어로·여정·스위처)이 그대로 렌더된다(spec 15 동작 무회귀).
- [x] A6 `/landing` 히어로 "우주 만들어보기"는 데모 모드로 `/`에 진입(로그인 없이 더미 우주), "로그인하기"는 `/sign-in`으로 이동한다.
- [x] A7 데모 모드 "체험 종료"는 데모 플래그를 비운 뒤 `/landing`으로 이동한다.
- [x] A8 옛 `/universe`로 진입하면 NotFound(404) 화면이 뜬다(리다이렉트하지 않는다). 내부 어디에도 `/universe`로 향하는 링크·OAuth `redirectTo`가 남지 않는다.
- [x] A9 `/dormant`로 진입하면 NotFound(404) 화면이 뜬다. dormant 오버레이는 `/?panel=dormant`로 열리며 시트·`FLY_TO_STAR`·회상 후 무효화는 그대로 동작한다.
- [x] A10 `redirect` search param은 내부 경로(`/`로 시작, `//`·외부 절대 URL 아님)만 허용한다 — 그 외는 무시하고 `/`로 폴백한다(오픈 리다이렉트 없음).
- [x] A11 인증 사용자의 `/` 진입·새로고침 시 우주 셸이 보이고, `sim`·`panel`·`fly` 딥링크 search param이 `/`에서 옛 `/universe`와 동일하게 동작한다.
- [x] A12 `prefers-reduced-motion`·계정 전환 캐시 격리·토큰 주입 등 01의 기존 수용 기준이 회귀 없이 유지된다.

## 구현 체크리스트 (implementation — 구현 관점)
<!-- 어떻게 만드나. 위→아래 순서. [P]=서로 다른 파일·의존 없는 병렬. (gen)/(migrate)/(gen:values) 표시.

> 설계 결정(범위 고정): **(a) `/sign-in`은 app 레이어에 둔다** — `SignInScreen`이 app의 `useAuthActions`/`SessionContext`에 의존하고 FSD상 `pages`는 `app`을 import할 수 없다(불변 4). `pages/sign-in` 신설 안 함. **(b) 게이트는 SessionGate(React/XState 컨텍스트)에 유지** — router `beforeLoad`로 옮기지 않는다(세션 상태가 router context에 없음; tech/state-machines 일관). 보호 라우트의 `anon` 처리만 인라인 렌더 → 리다이렉트로 교체. **values.yaml 변경 없음**(라우트 경로 = 문자열 식별자).

- [x] T001 `app/ui/SessionGate.tsx` — `anon` 분기를 `<SignInScreen/>` 인라인 렌더에서 `/sign-in?redirect=<현재 path+search>`로 **리다이렉트**(`<Navigate>` 또는 `useNavigate` 효과; 현재 위치는 `useLocation()`/router state로). `loading`(스플래시)·`demo`(통과+배지)·`authed`(children+로그아웃)는 그대로. 데모 "체험 종료" navigate `'/'` → **`'/landing'`**(이제 `/`는 보호 라우트라 anon이면 사인인으로 튕김).
- [x] T002 신규 `/sign-in` 라우트 콘텐츠(app 레이어 — 예: `app/ui/SignInRoute.tsx`): `redirect` search param(아래 T003 검증)을 읽어 `authed`면 `<Navigate to={redirect ?? '/'}>`, `loading`이면 스플래시, `anon`이면 기존 `SignInScreen`을 렌더. (`SessionContext.useSelector`로 상태 구독 — SessionGate와 같은 머신.)
- [x] T003 `app/router.tsx` 라우트 테이블 재구성:
  - index `path:'/'` → `component = SessionGate(HomePage)`. `validateSearch`(`sim`·`panel`·`fly`)를 옛 `universeRoute`에서 index로 이동.
  - 신규 `path:'/landing'` → `LandingPage`(공개, 게이트 없음).
  - 신규 `path:'/sign-in'` → T002 컴포넌트. `validateSearch`: `{ redirect?: string }` — **내부 경로만 허용**(`/`로 시작, `//`·`http(s):`·콜론 스킴 아님; 그 외는 `undefined`/`'/'`로 폴백 → A10·오픈 리다이렉트 방지).
  - 옛 `universeRoute`(`/universe`)·`dormantRedirectRoute`(`/dormant`) **삭제**. `redirect` import 미사용 시 제거. `routeTree.addChildren` 갱신.
  - `/admin`·`/u/$slug`·`/gift/$token`은 그대로(보호 정책 유지).
- [x] T004 `pages/home/ui/HomePage.tsx` — `useSearch({from:'/universe'})`·`useNavigate({from:'/universe'})` → `from:'/'`(T003과 함께 떨어져야 빌드됨).
- [x] T005 [P] `app/ui/session-context.tsx` — `universeRedirect()`(`origin+'/universe'`) → OAuth redirect를 `origin+'/'`로(헬퍼명·주석도 정리). Google OAuth 복귀는 `/`로 떨어진다(라운드트립에서 `redirect` param은 소실 — OAuth는 항상 `/`로, A3의 "없으면 `/`"와 일치).
- [x] T006 [P] 랜딩 진입 동선: `pages/landing/ui/section/HeroSection.tsx`(`tryDemo`→`'/'`, `goSignIn`→`exitDemoMode()`+`resetDemo()`+`'/sign-in'`), `CtaFooterSection.tsx`(동일 두 동선), `TryInUniverse.tsx`(`enterDemoMode()`+`{to:'/', search:{sim}}`).
- [x] T007 [P] `pages/gift/ui/GiftPage.tsx` — `navigate({to:'/universe'…})` 4곳 → `'/'`(수락 후 fly는 `{to:'/', search:{fly:res.memoryId}}`).
- [x] T008 [P] 폴백·CTA 경로: `shared/ui/NotFoundScreen.tsx`(primary "내 우주로" `/universe`→`'/'`; ghost "처음으로" `/`→**`'/landing'`**), `app/ui/ErrorScreens.tsx`(`RouteErrorScreen` "처음으로" `/`→**`'/landing'`**), `pages/visit/ui/VisitPage.tsx`(`CreateUniverseCTA` `<Link to="/">`→**`'/landing'`** — 비로그인 방문자에게 마케팅 랜딩 노출).
- [x] T009 [P] 주석 정리(`/universe` 표기 → `/`): `app/ui/SignInScreen.tsx` 헤더, `app/ui/SessionGate.tsx`, `features/switch-appearance/ui/AppearanceSwitcher.tsx`(L157), `widgets/demo-sim/ui/DemoSimPanel.tsx`(`/universe?sim=`).
- [x] T010 검증: `pnpm --filter ./frontend build` + `lint`. 수동 라우트 점검(로그아웃/데모/인증 3상태 × `/`·`/landing`·`/sign-in`·`/universe`(404)·`/dormant`(404)·`/admin`·`/gift/$token`)으로 A1–A12 확인.
