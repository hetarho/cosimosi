## 인수 조건 (acceptance — 기획 관점, plan/41.invite-membership에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [x] A1 `/sign-in`은 랜딩(15)과 같은 별·반짝임 배경 + "cosimosi" 워드마크를 렌더하고, 이메일 OTP·Google 로그인 수단은 전부 활성이며 동작은 01과 동일하다(reduced-motion이면 모션 폴백).
- [x] A2 멤버가 아닌 사용자가 로그인에 성공하면 `/invite?redirect=<원래경로>`로 리다이렉트된다. 이미 멤버인 사용자는 `/invite`를 거치지 않고 `redirect`(없으면 `/`)로 입장한다.
- [x] A3 `/invite`에서 유효한 코드를 confirm하면 `RedeemInviteCode`가 성공하고 환영 연출 후 `redirect`(없으면 `/`)로 입장한다. 무효 코드는 소비되지 않고 사유별 에러를 보인다.
- [x] A4 **서버 강제:** 멤버십이 없는 인증 토큰으로 `MemoryService`·`SettingsService`·`ShareService`·`GiftService`의 RPC를 호출하면 `PermissionDenied`를 받는다. 같은 토큰으로 `InviteService`(GetMembershipStatus·ValidateInviteCode·RedeemInviteCode)는 핸들러에 도달한다. 미인증은 `Unauthenticated`다.
- [x] A5 1회용 코드(`max_uses=1`)는 첫 redeem 후 두 번째 redeem이 `EXHAUSTED`로 거부된다. 같은 코드를 동시에 redeem해도(경쟁) 정확히 1회만 소비된다(`FOR UPDATE` 직렬화).
- [x] A6 시간지정 코드는 `expires_at` 이후 redeem이 `EXPIRED`로 거부되고, 그 전에는 (무제한이면) 여러 번 redeem된다.
- [x] A7 무제한 코드(`max_uses` NULL·`expires_at` NULL)는 횟수·시간 제한 없이 redeem된다.
- [x] A8 `RedeemInviteCode`는 멱등하다 — 이미 멤버인 사용자가 다시 호출하면 `used_count`를 늘리지 않고 `ok=true`를 반환한다.
- [x] A9 `RevokeInviteCode` 후 그 코드의 redeem은 `REVOKED`로 거부된다.
- [x] A10 발행된 코드 목록(`ListInviteCodes`)은 각 코드의 코드값·상태·`used_count`/`max_uses`·만료·**발행인(`created_by`)**·라벨을 포함한다.
- [x] A11 `InviteAdminService`의 모든 RPC는 `ADMIN_USER_IDS` allowlist 뒤에 있다 — 비관리자/미인증은 발행·목록·revoke에 접근할 수 없다(34와 동일 게이트, 멤버십 무관).
- [x] A12 `/admin`은 "LLM 관리"·"초대 코드" 두 탭으로 나뉘고, LLM 관리 탭의 기능(키·모델·활성 선택·대시보드)은 34와 동일하게 동작한다(회귀 0). `?tab=`으로 딥링크된다.
- [x] A13 `INVITE_GATE_ENABLED=false`이면 멤버십 게이트가 강제되지 않고(`GetMembershipStatus`가 인증 전원에 `is_member=true`), 코어 RPC가 멤버십 없이 통과하며 FE는 `/invite`로 보내지 않는다.
- [x] A14 관리자는 멤버십(redeem) 없이도 `/admin`과 `InviteAdminService`로 첫 코드를 발행할 수 있다(admin 체인에 멤버십 게이트 부재).
- [x] A15 도메인 타입(`internal/invite`)에는 db/json/proto 태그가 없다(헌법5). 좌표·원본 일기와 무관한 독립 테이블이라 헌법1/2를 건드리지 않는다.

> **설계 결정 (범위 고정):**
> - **(a) 게이트는 서버 인터셉터가 강제, FE는 신호만.** 35의 "노출 수위별 체인 분리"를 차용 — 공유 auth `opts` 슬라이스를 **복제 후 누적**(앨리어싱 금지, 34 admin_gate 선례). 코어 4서비스=auth+membership, `InviteService`=auth-only, `InviteAdminService`=auth+admin.
> - **(b) 멤버십 = `invite_redemptions`(user_id PK) 1행.** `used_count`는 `invite_codes`에 비정규화(캡 판정·표시), 사용 내역은 `invite_redemptions`. 멤버십 룩업은 인터셉터당 1 `EXISTS`.
> - **(c) 직교 저장(`max_uses`·`expires_at`), 프리셋은 UI/IssueRequest(`ttl_seconds`).** `Redeem`은 `FOR UPDATE` 트랜잭션으로 원자·멱등(이미 멤버면 무소비 ok).
> - **(d) 사인인은 app 레이어 유지 — 비주얼만 교체(로그인 로직·01 불변).** 재사용 자산은 `shared/ui`로 승격(`GrainOverlay` 이동·`CosmosBackdrop` 신설) — `app`이 `pages/landing` 내부를 직접 import하지 않게(불변4).
> - **(e) `MembershipGate`는 `SessionGate` 안쪽 React 컨텍스트 게이트**(router `beforeLoad` 아님 — 세션이 router context에 없음, job 02·tech/state-machines 일관). `/invite`·`/admin`은 멤버십 게이트 밖.
> - **(f) 출처 경계:** 코드 알파벳(모호문자 제외)은 콘텐츠 → `internal/invite` 상수. enum→proto, 테이블→sqlc, `INVITE_GATE_ENABLED`·crypto·timeout→env/config. `values.yaml`엔 수치 노브만.
> - **(g) 제거성:** `INVITE_GATE_ENABLED=false`면 멤버십 인터셉터를 체인에 **미장착** + `GetMembershipStatus`가 인증 전원에 `is_member=true`.
