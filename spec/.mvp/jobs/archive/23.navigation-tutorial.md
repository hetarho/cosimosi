- [x] A7 멀리서→가까이서 전환이 실습 segment 안에서 이뤄지고(투어 안내/구동), segment가 끝나면 다음 투어 단계에 맞는 모드 상태로 정리된다.

## 구현 체크리스트 (implementation — 구현 관점)

- [x] T001 (gen:values) `spec/values.yaml`에 `demo_tour` 그룹 신설 — 실습 성공 임계값: `rotate_threshold_rad`(멀리서 회전 누적 각, 초깃값 후보 `0.9`)·`zoom_ratio_threshold`(멀리서 줌 반경 변화 비율, 후보 `0.25`)·`look_threshold_rad`(가까이서 시선 누적 각, 후보 `0.9`)·`thrust_distance_threshold`(가까이서 전진 누적 거리 world 단위, 후보 `8`). `pnpm gen:values` → `frontend/src/shared/config/values.gen.ts`에 `VALUES.demoTour.*` 생성. (코드는 생성 상수만 import — 하드코딩 금지, A9.)
- [x] T002 항해 활동 **passive 탭** 추가(관찰 전용·입력 의미 불변, A8·A10): `consumeLookDelta()`는 매 프레임 `NavController`가 read-and-clear하므로 투어가 직접 폴링하면 입력을 가로챈다 → `navigation-input.ts`에 **단조 누적 카운터**(예: `lookTravelRad`·`thrustTravel`)를 더해 기존 consume 경로를 건드리지 않고 누적만 한다(`addLookDelta`/`setThrust` 시 가산). 멀리서 회전·줌은 `NebulaOrbitController`(`UniverseCanvas.tsx`)가 같은/형제 누적 카운터(`orbitTravelRad`·`zoomRatioTravel`)를 bump. **회귀 경계: 기존 입력·물리·consume 동작은 불변, 카운터는 읽기 전용 가산만.**
- [x] T003 `demo-tour/model/steps.ts` 순수 모델 확장(React/DOM/three import 없음, A12): (a) `TourAwait`에 `nebula-rotated`·`nebula-zoomed`·`recall-looked`·`recall-thrusted` 추가. (b) phase가 **디바이스 변형 문구**를 갖도록 모델 확장(예: `body`를 `{ mouse: string; touch: string }`로, 또는 `bodyMouse`/`bodyTouch`) + 그 phase가 기대하는 카메라 모드 힌트(`'nebula' | 'recall'`). (c) 현행 단일 정보 `view` 단계를 **항해 실습 segment**로 교체: ① 시점 전환 안내(정보, 현행 문구 유지) → ② 멀리서 회전(`nebula-rotated`) → ③ 멀리서 줌(`nebula-zoomed`) → ④ 가까이서로 전환(view 토글 안내/구동) → ⑤ 가까이서 시선(`recall-looked`) → ⑥ 가까이서 전진(`recall-thrusted`). 디바이스별 문구는 change §목표대로.
- [x] T004 `demo-tour/ui/DemoGuidedTour.tsx`: (a) `isAwaitMet`에 새 신호 4종 추가 — `useCameraMode`(현재 모드 prop/selector)와 T002 누적 카운터를 **rAF로 샘플링**해 임계(`VALUES.demoTour.*`) 도달 시 충족 판정(매 프레임 setState 금지, 임계 crossing에서만, A8). (b) phase 문구를 `useCoarsePointer()`로 mouse/touch 변형 중 골라 렌더(A3). (c) `다음`·`건너뛰기` 탈출구·reduced-motion 전환 유지(A6·A12).
- [x] T005 `pages/home/ui/HomePage.tsx`: `DemoGuidedTour`에 카메라 `mode`(`useSelector(navigationActor, selectHeadingMode)`) 전달. 가까이서-전환 phase 진입 시 `navigationActor.send({ type: 'TOGGLE_MODE' })` 구동(투어 콜백 또는 phase mode 힌트 기반), segment 종료/이탈 시 기본 모드(`nebula`)로 정리(A7). 실습 중 다른 표면(망원경·사이드바)과 충돌하지 않게 정리 — 기존 surface 정리 패턴 재사용.
- [x] T006 회귀 확인(A10·A11): 멀리서 회전/줌·두 손가락 pan/zoom scrub, 가까이서 시선/전진, 데스크톱 NavPad·키보드, plan 48 다른 투어 단계(UI 토글·페르소나·시간·망원경·탭·새 별·별 클릭·끝), 실계정 우주(투어 미노출), job 22 임베디드 recall 렌더 불변. 서버·proto·RPC·sqlc 무변.
