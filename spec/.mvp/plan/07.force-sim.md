# 07. force-sim - as-built

> 현재 force-sim은 웹 우주에서 모든 별을 strength shell 위에 두고 연결 그래프가 각도를 빚게 하는 순수 3D 물리 코어다. 과거 Web Worker 래퍼는 제품 호출 경로가 없어 제거됐고, `UniverseCanvas`의 production path는 main-thread `useFrame` pump를 사용한다.

## 현재 범위
- 순수 코어: `frontend/src/shared/lib/force-sim/types.ts`, `sim.ts`, `octree.ts`, `seed.ts`
- live canvas adapter: `frontend/src/widgets/universe-canvas/ui/controllers/layout.tsx#LiveLayoutController`
- radial layout 단일 출처: `frontend/src/widgets/universe-canvas/model/radial-layout.ts`
- shared math: `frontend/src/shared/lib/layout.ts`

## 공개 API
```ts
export interface SimNode {
  id: string
  pinned: boolean
  x: number
  y: number
  z: number
  radius?: number
}

export interface SimEdge {
  source: string
  target: string
  weight: number // 0..1
}

export interface SimParams {
  theta: number
  repulsion: number
  linkDistance: number
  centerGravity: number
  velocityDecay: number
  alphaMin: number
  radialStrength: number
}

export function createSim(graph: SimGraph, params?: Partial<SimParams>, opts?: CreateSimOptions): SimState
export function advance(state: SimState, steps?: number): void
export function tick(state: SimState, steps?: number): Float32Array
export function positions(state: SimState): Float32Array
export function alpha(state: SimState): number
export function isSettled(state: SimState): boolean
export function seedNearCluster(...)
```

## 순수 코어 동작
- `createSim()`은 입력 node 순서를 보존한다. `state.ids[i]`와 `state.px[i*3..i*3+2]`가 renderer instance slot과 1:1로 대응한다.
- unknown endpoint를 가진 edge, self edge는 조용히 버린다.
- `px`, `vx`, `radius`, `free`는 mutable typed array다. `advance()`는 live buffer를 제자리에서 갱신하고, `tick()`은 `advance()` 후 fresh `positions()` snapshot을 반환한다.
- alpha는 d3-style decay로 약 300 tick에 `alphaMin`까지 떨어진다. `isSettled()`는 `alpha <= alphaMin`이다.
- `octree.ts`는 3D Barnes-Hut octree다.
  - repulsion source는 모든 node다.
  - force를 받는 쪽은 `free[i] === 1`인 node뿐이다.
  - coincident/non-finite 방어를 위해 `MIN_DIST2`, `MIN_HALF`, `MAX_DEPTH` guard가 있다.
- spring은 edge별로 `weight`를 곱하고 `linkDistance`를 rest length로 사용한다.
- per-tick velocity는 `linkDistance * 2` 상한으로 clamp한다. dense/strong-spring graph가 Infinity/NaN으로 발산해 octree가 터지는 것을 막는 안전장치다.

## free set과 pinned 의미
- 라이브러리 기본 free set은 `!pinned` node와 그 1-hop neighbor다. 이 정책은 기존 단위 테스트와 legacy partial-placement API를 위해 유지된다.
- 현재 웹 우주의 live canvas는 이 정책을 그대로 노출하지 않는다. `LiveLayoutController`가 모든 memory star를 `pinned: false`로 넘기므로 현재 우주에서는 모든 별이 움직일 수 있다.
- Flutter 구현 기준:
  - 일반 우주 렌더는 웹과 같이 모든 memory star를 free로 둔다.
  - pinned/1-hop partial mode는 라이브러리 기능으로 보존하되 현재 `/universe`의 as-built 배치 계약으로 해석하지 않는다.

## radial shell 계약
- `SimNode.radius`가 없거나 0이면 classic center gravity가 작동한다.
- `radius > 0`이면 그 node는 self-centered shell target을 가진다.
- shelled node에서는 repulsion/link force의 radial component를 제거하고 tangential component만 velocity에 더한다.
- radial distance는 `radialStrength` spring이 소유한다. 즉 link와 repulsion은 별을 shell 위에서 "어느 방향"으로 둘지 결정하고, 중심으로부터의 거리는 strength가 결정한다.
- origin에 정확히 놓인 node는 radial direction이 없으므로 full external force와 +x nudge로 shell에 탈출시킨다.
- target radius는 tick 사이에 변경 가능하다. caller가 `state.radius[i]`를 바꾸고 `state.alpha`를 올리면 별이 새 shell로 glide한다.

## strength -> radius 단일 출처
- `frontend/src/widgets/universe-canvas/model/radial-layout.ts`
  - `radiusOf(mem, now)`
  - `atRadius(pos, r)`
  - `RADIAL_SIM_PARAMS`
- `radiusOf`는 다음 흐름이다.
  - `act = max(A_MIN, activation(mem.lastRecalledAt, now))`
  - `strength = W_ACT * act + W_INT * intensity`
  - `targetRadius(strength)`
- shared constants:
  - `A_MIN = 0.05`
  - `R_MIN = 6`
  - `R_MAX = 40`
  - `W_ACT = 0.7`
  - `W_INT = 0.3`
- canvas params:
  - `RADIAL_SIM_PARAMS = { repulsion: -18, linkDistance: 14, radialStrength: 0.1 }`
- 강하고 최근 활성화된 별일수록 중심 "나" 별에 가깝고, 흐려진 별일수록 바깥 shell로 이동한다.

## LiveLayoutController
- 위치 buffer의 단일 출처는 `positionsRef.current`다.
- graph topology signature는 `star ids + edge pairs`다. 같은 topology에서 stars/edges array ref만 바뀌면 sim을 rebuild하지 않는다.
- topology가 바뀔 때:
  - 이전 live positions를 id별로 저장해 새 sim이 같은 각도에서 이어진다.
  - 새 별은 `seedNearCluster()`로 최근 활성 cluster 각도에 심고, `atRadius()`로 strength radius에 맞춘다.
  - placed neighbor가 없으면 `scatterDirection(seed)`를 사용한다. index 기반 fibonacci spiral로 새 별이 줄줄이 늘어서는 것을 피한다.
  - `createSim(..., RADIAL_SIM_PARAMS, { seedNewNodes: false })`로 sim을 만든다.
- 매 프레임:
  - `virtualNowMs()` 기준으로 target radius를 다시 계산한다.
  - `max(abs(newRadius - state.radius[i])) > 0.5`이면 `state.radius.set(targets)` 후 `alpha`를 최소 `0.3`까지 re-kick한다.
  - settled가 아니면 `tick(sim, 2)`를 호출해 live buffer를 갱신한다.
  - settled가 되면 fresh snapshot을 publish해 synapse geometry가 relaxed 좌표로 다시 bake된다.
- heat seeding:
  - 최근성 heat는 `exp(-ageMs / HOT_TAU_MS)`다.
  - `HOT_TAU_MS = 6 * 60 * 60 * 1000`.
  - `seedNearCluster()`는 neighbor position을 `0.1 + heat`로 가중 평균하고 deterministic jitter를 더한다.
- demo mode에서만 representational drift를 적용한다.
  - `night = floor(virtualNow / DAY_MS)`가 증가할 때마다 `applyAngularDrift(pos, seed, nights)`를 적용한다.
  - radius는 보존하고 방향만 회전한다.
  - velocity는 0으로 지우고 `alpha`를 re-kick한다.

## Worker 계약
- `worker.ts`는 web platform wrapper일 뿐 순수 코어의 필수 런타임이 아니다.
- 메시지:
```ts
{ type: 'init'; graph: SimGraph; params?: Partial<SimParams> }
{ type: 'tick'; steps?: number }
{ type: 'dispose' }

{ type: 'positions'; buffer: ArrayBuffer; alpha: number; settled: boolean }
```
- worker는 자체 rAF loop를 돌리지 않는다. 호출자가 tick message로 pump한다.
- 반환 buffer는 `postMessage(..., [buf.buffer])`로 transfer된다.
- 현재 `UniverseCanvas`는 worker를 쓰지 않고 main thread에서 `tick(sim, 2)`를 호출한다. Flutter는 isolate/worker를 선택할 수 있지만 결과와 scheduling semantics는 위 live controller와 맞춘다.

## 테스트로 고정된 동작
- `sim.test.ts`
  - strong edge pair가 weak edge pair보다 가까워진다.
  - pinned node는 움직이지 않는다.
  - library default에서는 new node + 1-hop만 움직인다.
  - `positions()` 길이와 순서가 입력 node 순서와 같다.
  - unknown endpoint edge는 무시한다.
  - N=3000 synthetic graph에서 한 tick이 finite하게 완료된다.
- `sim.radial.test.ts`
  - radius 없음은 center gravity로 origin 쪽으로 간다.
  - radius set node는 shell로 수렴한다.
  - origin stuck node가 shell로 탈출한다.
  - 같은 radius node들이 shell 위에서 angular spread를 가진다.
  - shelled node는 먼 neighbor link에 radial로 끌려가지 않고 자기 shell을 유지한다.
  - `state.radius` 변경 후 re-kick하면 새 shell로 이동한다.
- `seed.test.ts`
  - hot neighbor cluster 쪽으로 deterministic seed한다.
  - placed neighbor가 없으면 fallback shell seed를 사용한다.
  - `seedNewNodes: false`면 caller-provided coordinate를 보존한다.

## Flutter 동기화 포인트
- force-sim core는 DOM/React/three 없이 이식 가능한 수학이어야 한다.
- live universe는 모든 별 free + radial shell이다.
- node order는 rendering instance order와 같아야 한다.
- links/repulsion은 shell 위 각도를 빚고, radius는 strength가 빚는다.
- synapse renderer는 settled layout snapshot을 bake하되 live buffer drift를 따라갈 수 있어야 한다.
- worker/isolate 사용 여부는 플랫폼 선택이지만, worker가 자체 loop를 갖지 않는다는 scheduling 계약은 유지한다.
