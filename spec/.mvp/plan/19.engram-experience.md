# 19. engram-experience - 기억 실험실(데모 가상 시계·시뮬) + 랜딩 재정비 (as-built)

> 현재 구현 기준 문서. 체험 우주(demo 데이터 우주)에 가상 시계와 기억 실험실 컨트롤을 넣어 이미 구현된 엔그램 이론을 사용자가 직접 트리거해 눈앞에서 보게 하고, 랜딩이 그 이론을 정확히 표현하도록 정합한다. | 범위: FE | 상태: 구현 완료, 반영: 21·22·23·24·25·26·27·37(데모 프레임 위 확장)·39(회상 flush 머신)

## 목적

MVP(01–13)가 엔그램 이론 여러 개를 이미 구현했지만, 시간 감쇠는 며칠을 기다려야 보이고 잠든 별은 오래 지나야 생긴다. 새 이론 로직을 더하지 않고, 이미 동작하는 이론을 데모 우주 안에서 시뮬레이션 가능(simulatable)하게 만드는 것이 목적이다.

핵심 메커니즘은 데모 가상 시계다. 밝기·잠듦 계산은 전부 `now`를 인자로 받는 순수 함수(`entities/memory/model/activation.ts`)이므로, 데모에서 `now`를 `Date.now() + offset`으로 바꾸고 "하루/한 달 지나기"로 offset을 키우면 별이 어두워지고, 임계를 넘은 별이 잠들고, 회상하면 다시 밝아진다. 시간이 실제 흐른 것과 동일한 코드 경로다(시뮬레이션 ≠ 연출).

이 가상 시계와 데모 데이터 모듈은 20–27의 확장 프레임이기도 하다. 조각화(21)·경쟁 할당(22)·재공고화(23)·변천사(24)·요즘 상태(25)·관련성 감쇠(26)·야간 공고화(27)·겹쳐보기(37)는 모두 같은 데모 데이터 모듈(`shared/lib/demo/data.ts`)에 자기 근사를 얹고, 같은 시간 보내기 버튼으로 시연된다. 본 스펙은 그 프레임(가상 시계·재점화·연결 생성·헵 로컬·기억 실험실·이론 레지스트리)을 소유한다.

## 현재 구현

### 가상 시계 (`shared/lib/demo/clock.ts`)

- 시간 시뮬레이션의 단일 주입 지점. 모듈 변수 `offsetMs`(누적 가상 시간)와 `speed`(배속)가 전부다(새로고침 시 소멸 — 데모 수명 규칙).
- `virtualNowMs()`는 `Date.now() + (isDemoMode() ? offsetMs : 0)`. 비데모면 정확히 `Date.now()` — 일반 모드 무영향.
- **배속 연속 시계(change 24):** 시계는 이산 점프가 아니라 배속으로 *계속 흐른다*. `advanceDemoClock(realElapsedMs)`가 경과 실시간을 배속만큼 가상 시간으로 환산해 누적하고(`Δoffset = realElapsedMs·speed·3600`, `speed`=실제 1초당 흐르는 가상 *시간*), 이 틱이 지난 simulated 04:00 KST 경계 수를 돌려준다. rAF 드라이버(pages)는 그 횟수만큼 야간 공고화를 발화한다.
- 배속 옵션 `getDemoClockSpeed`/`setDemoClockSpeed` — `1`·`2`·`4`(values `demo_clock.hours_per_second`, 첫 값이 기본) 또는 `'paused'`(누적 멈춤). 타입 `DemoClockSpeed = number | 'paused'`.
- `consolidationBoundariesCrossed(prevMs, nextMs)` — (prev, next] 안의 04:00 KST(=`consolidation.hour_utc`=19 UTC) 경계 수. 24h 간격이라 day-bucket 전이마다 정확히 1회(빠른 배속이 한 틱에 여러 날을 건너도 누락·중복 없음).
- `demoOffsetDays()`는 표시·트리거용 흘러간 가상 일수(내림).
- `resetDemoClock()`은 offset을 0으로, 배속을 기본으로 되돌린다 — `resetDemo()`가 호출한다.
- three/React/DOM 미의존(헌법4 — rAF 루프는 호출자(pages 드라이버)가 굴린다).

렌더·파생 계층은 `Date.now()`가 아니라 이 함수를 읽는다. 실제 호출부: `entities/star/ui/StarField.tsx`(rebuild·spotlight 2곳), `entities/memory/api/map-star.ts`, `entities/memory/api/universe-query.ts`(`applyUniverse`·`refreshActivation`), `entities/synapse/model/store.ts`, `features/recall/ui/MemoryPanel.tsx`, `features/recall/model/recall-flush.machine.ts`, `widgets/universe-canvas/ui/{UniverseCanvas,overlay/OverlayUniverse}.tsx`, `pages/visit/ui/VisitPage.tsx`(비데모라 `=Date.now()`), `pages/home/ui/HomePage.tsx`(겹쳐보기 재시뮬 deps). 시계 주입이 한 군데로 모인 덕에 이후 스펙은 별도 시간 인자 없이 시간 의존 이론을 데모에 노출한다.

### 데모 데이터 모듈 (`shared/lib/demo/`)

데모 우주는 더 이상 손으로 박은 별·엣지 목록(옛 `DEMO_ENTRIES`/`DEMO_EDGES`)이 아니라 페르소나 일기 코퍼스를 시뮬레이션해서 만든다.

- `personas.ts` — 세 페르소나 코퍼스 `CORPORA`(`student`/`worker`/`homemaker`). 각 코퍼스는 일기(조각 N개)·회상 이력의 순수 데이터. `demoPersonaList()`가 `{id, label, tagline}`을 낸다(예: '20대 대학생' · '사소한 것에 크게 흔들리는 밤').
- `simulate.ts` — 코퍼스를 별·시냅스 그래프로 빚는 순수 시뮬레이터. 실서버 worker의 연결 규칙을 조각 단위로 근사한다:
  - 일내 결속 `intra_entry` 고정 weight `INTRA_WEIGHT=0.8`(같은 일기 조각 모든 쌍).
  - 의미 링크: 시간순으로 각 조각이 이전 조각 중 topic-cosine 유사도 top-`KNN_K=5`에 잇는다(자격 바닥 `SIM_TAU=0.4`). weight `= 0.42 + sim·0.4 + temporal_bonus`, 상한 `SEM_CAP=0.79`(< intra). 사람·개체 주제를 공유하면 `entity`, 아니면 `semantic`.
  - temporal 보너스: 같은 주(`TEMPORAL_DAYS=7`) 안에서 최대 `TEMPORAL_MAX=0.3`.
  - 회상 세션: 그 일기 조각을 재점화(밝아짐)하고 함께 읽은 다른 주제 일기끼리 `co_recall` 다리(`CO_RECALL_BASE=0.5`, 기존 선엔 `CO_RECALL_BUMP=0.1`).
  - 연결성 보장: 다른 일기와 잇는 선이 없는 일기는 시간순 이웃에 약한 다리(외톨이 방지 안전망).
  - `crossResonances(a, b, max=4)`: 두 페르소나 코퍼스 사이 공명 쌍(겹쳐보기 37 데모 — 1:1 매칭, 결정론적).
- `data.ts` — proto 어댑터 + 데모 변이의 집. 코퍼스를 `simulate`로 빚어 proto `Star`/`Record`/`Synapse`로 파생하고(`GetUniverse`·`RecallMemory`·`ListDormant`·`ListRecords`와 같은 id 공간 공유), 모듈 변수에 런타임 상태를 둔다. 본 스펙이 소유하는 함수:
  - `demoStars()` / `demoSynapses()` — base + 체험 중 추가분.
  - `demoMarkRecalled(id)` — 그 별의 `lastRecalledAt`을 가상 now로 전진(재점화). `renewStar`로 별을 **불변 교체**해 protobuf structural sharing이 변경을 감지하게 한다(제자리 변이는 refetch가 no-op이 됨).
  - `createDemoDiary(body, date, frags)` — 검토를 마친 조각들을 별 fan-out으로 추가하고 production 규칙으로 연결한다(아래 §데모 연결 생성). 내부 공용 primitive.
  - `beginDemoCompose()` / `demoComposeSegments()` / `demoRecordMemory(input)` — 자유모드 `새 별 띄우기` 프리셋 작성 흐름(plan 47·change 25): 프리셋 일기 선택·본문/날짜 주입 → 사전분절 조각 반환(SegmentMemory 대체) → 조각 별 생성(RecordMemory 대체). compose 머신이 데모 분기에서 호출한다.
  - `demoAddStar(mood, date)` / `demoAddMultiSceneStar(date)` — 테스트·내부 시뮬 경로: 미리 쓴 일기(`QUICK_ENTRIES`/`MULTI_SCENE_ENTRIES`)를 조각으로 `createDemoDiary`에 넘긴다.
  - `demoToday()` — 별 띄우기 날짜 기본값(가상 시계 기준 YYYY-MM-DD).
  - `resetDemo()` — base·추가분·재성형 상태를 비우고 `resetDemoClock()`까지 호출.
  - 이후 스펙이 같은 모듈에 얹은 함수(본 스펙 소유 아님, 데모 프레임 위 확장): `demoReshape`(23)·`demoEvolution`(24)·`demoConsolidate`(27)·`demoOverlayData`/`buildPersonaUniverse`(37)·`demoListRecords`/`demoFragmentText`(28)·valence 근사(25).
- `flag.ts` — `isDemoMode`/`enterDemoMode`/`exitDemoMode`·페르소나 게터/세터·`setDemoModeListener`(app이 캐시·스토어 리셋 주입). sessionStorage 보관.
- `observe.ts` — 관찰 셀렉터 `brightestStarId`/`thickestEdge`/`sameDayPair`/`dormantStarId`(코퍼스 그래프에서 id만 파생, 순수). index에서 export되나 현재 소비처는 없다(패널이 fly-to 앵커 방식에서 이론 캐러셀로 바뀌며 호출이 사라짐 — 죽은 export).

### 재점화 — 데모에서 완결 (`features/recall`)

실서버는 `RecallMemory`가 `last_recalled_at=now`를 쓰고 refetch 병합으로 별이 밝아진다. 데모는 그 대칭으로:

- `api/recall.ts`의 `recallMemory(memoryId)`는 데모 분기에서 `demoRecall(id)`(원본 일기) 후 `demoMarkRecalled(id)`(가상 now로 전진)와 `demoReshape(id)`(23 PE 게이트 재성형)를 호출한다. 쓰기 게이트(겹쳐보기 37의 `isWriteBlocked()`)가 켜져 있으면 undefined.
- `ui/MemoryPanel.tsx`는 ≥2초 active dwell 후 `recallMemory`를 발화하고, 데모·프로덕션 모두 회상 성공 시 `universeInvalidateKey()`로 universe 쿼리를 즉시 무효화한다(전진한 타임스탬프·재성형 결과가 화면에 반영).
- 잠재우기 → fly-to → 회상 → 재점화 루프가 데모에서 완결된다.

병합 단조성: `entities/memory/model/merge.ts`의 `mergeStars`는 `max(lastRecalledAt)`. 그래서 시간 시뮬레이션은 데이터를 과거로 미는 게 아니라 시계를 미래로 보낸다 — 전진한 타임스탬프는 병합을 그대로 통과한다.

### 헵 강화 — 데모 로컬 (`entities/synapse`, `features/recall`)

- `entities/synapse/model/store.ts`의 `bumpEdgeWeight(aId, bId, delta)`: 정규화 페어(a<b)로 엣지를 찾아 `weight = min(1, w+delta)`·`brightness = 1`·`coActivationCount++`·`lastActivatedAt = virtualNowMs()`로 갱신한다. 페어가 없으면 데모 한정 `co_recall` 엣지를 로컬 생성(`weight = min(1, delta)`·`coActivationCount = 1`). 서버 쓰기 없음.
- co-recall 누적·증분은 `features/recall/model/co-recall.ts`(`CO_RECALL_DELTA=0.05`·`DWELL_MS=2000`·간격 효과 `spacingBoost`)가 순수하게 한다.
- 데모 즉시 굵어짐 훅 지점은 **`recall-flush.machine.ts`의 `accumulate` 액션**이다(옛 zustand `recordActiveView`가 아니라 XState 머신). `RECORD_VIEW`마다 `onActiveView(session, id, virtualNowMs())`로 누적하고, 직전 active view와 다른 별이며 데모면 `useSynapseStore.getState().bumpEdgeWeight(prev, id, CO_RECALL_DELTA)`를 직접 부른다. 비데모 영속 경로(`reinforceLinks` flush)와 11의 강화 의미론은 불변. `<2초` 스침은 dwell 타이머가 끝나기 전에 취소돼 `RECORD_VIEW`가 안 가므로 어떤 연결도 굵어지지 않는다(능동 인출 게이팅).

### 데모 연결 생성 (`shared/lib/demo/data.ts`의 `createDemoDiary`/`linkNewDiary`)

새 일기는 검토를 마친 조각들(프리셋 사전분절 또는 실 입력)을 별로 빚고, production 규칙·정전 값으로 연결한다(change 25·27 — 데모 전용 가중치·키워드 추정·`DEMO_MAX_FRAGMENTS` 상한 폐기):

- 일내 결속: 모든 조각 쌍을 `intra_entry` weight `connection.intraEntryWeight`(0.8)로.
- 교차 일기(`linkNewDiary`): 대표(첫) 조각을 기존 일기와 production 식으로 잇는다 — 후보는 같은 감정(임베딩 없는 데모의 의미 KNN 근사) 또는 같은 날, weight = `clamp(weightAlpha·sem + temporalBonus + emoAlpha·emotionSimilarity, 0, semanticWeightCap)`(서버 worker.go·excitability.go 식). 한 기존 일기엔 가장 센 조각 하나로만(record 단위 중복 방지), 상위 `excitability.biasedK`(5)개만 남긴다. 감정 유사도는 공유 순수 함수 `emotionSimilarity`(job 37).

추가 엣지는 `addedEdges`(가상 now `lastActivatedAt`)에 쌓이고 `resetDemo`가 비운다. 임베딩/LLM 의존부(의미 KNN·조각화·감정 추출)는 데모에 없어 명시적 근사·프리셋으로 남고(change 27 A6 환원 불가 경계), 그 외 결정론 로직(일내 결속·weight 식·거리·밝기·재공고화·야간 공고화)은 서버와 동치다. 패널 문구가 이 경계를 밝힌다.

### 데모 자유모드와 시간 이동 (`widgets/demo-sim`)

> 체험 진입은 plan 47에서 "페르소나 선택 → 모드 선택 → 자유모드"로 재구성됐다. 자유모드 기본 HUD는
> 일반 우주 셸과 같고(좌상단 테마 + 데모 페르소나/시간 버튼, 하단 중앙 `새 별 띄우기` → 프리셋 작성 폼,
> 우상단 햄버거에 `체험 종료`), 좌하단 `기억 실험실`/`뇌과학 이론` 칩과 큰 컨트롤러는 제거됐다.
> 기능 안내(이론 포함)는 plan 48 튜토리얼 스포트라이트 투어와 랜딩 카드가 맡는다. 본 스펙이 소유하는 데모
> **데이터 프레임**(가상 시계·데이터 모듈·재점화·헵 로컬·연결 생성·이론 레지스트리)은 자유모드에서도
> 그대로 쓰인다.

자유모드 컨트롤은 `pages/home/ui/DemoFreeModeControls.tsx`가 맡는다. 좌상단 테마 pill 아래 페르소나·시간 버튼을 두고, 시간 버튼 팝오버는 **배속 셀렉터**(`24시간`·`48시간`·`72시간`·`정지` — `demo_clock.hours_per_second`)와 `처음으로`(`resetDemoExperience`)를 둔다 — 이산 "하루/한 달 후로 이동" 버튼은 배속 컨트롤로 대체됐다(change 24). 휴지/기본 배속은 셀렉터 옵션과 분리된 `demo_clock.default_hours_per_second`(1 — 잔잔한 흐름)라 첫 옵션이 곧 기본은 아니다. 우주 키우기는 하단 중앙 `새 별 띄우기` 버튼이 production 작성 폼을 read-only 프리셋 일기로 여는 단일 경로다(change 25 — `beginDemoCompose`로 본문/날짜 주입 후 `openCompose`). 이론 시각화는 `entities/theory`의 개별 데모 컴포넌트를 랜딩 카드가 직접 소비한다.

배속 흐름 드라이버는 `pages/home/model/use-demo-flow.ts`의 rAF 루프다(데모 자유/튜토리얼·비겹쳐보기·비정지에서만): 매 프레임 `tickDemoClock`을 호출하고, `refreshActivation()`은 `demo_clock.refresh_throttle_ms`(200ms)로 throttle하며, 정수 가상 일이 바뀔 때만 React 상태로 끌어올린다(매 프레임 setStars·머신 이벤트 금지 — 헌법8). rAF라 탭이 가려지면 자동으로 멎어 누적 폭주가 없다.

시간 흐름 오케스트레이션은 `model/time-travel.ts`:

- `tickDemoClock(queryClient, realElapsedMs)` — `advanceDemoClock`으로 가상 시계를 누적하고, 지나친 04:00 경계마다 `demoConsolidate()`를 1회씩 발화한다(production change 20 패스의 충실한 데모 포트 — §데모 야간 공고화). 데이터가 바뀐 밤에만 universe/dormant 무효화. 지난 경계 수를 돌려준다.
- `resetDemoExperience()` — `exitDemoMode`→`resetDemo`→`enterDemoMode`(랜딩 tryDemo와 같은 검증된 리셋 경로).
- `switchDemoPersona(id)` — 페르소나 영속 후 같은 리셋 경로(다음 GetUniverse가 새 코퍼스로 재시드).

`tickDemoClock`/`refreshActivation`/`demoConsolidate`(데모 야간 공고화)는 본 스펙이 둔 자리를 27이 채운 것이다 — 시간이 흘러야 보이는 이론이 배속 시계로 실시간 트리거됨을 보여주는 사례.

### 즉시 재파생 (`entities/memory/api/universe-query.ts`의 `refreshActivation`)

시계가 흐르는 동안엔 refetch가 아니라 재파생이 맞다(데이터는 그대로, "지금"만 움직였다). `refreshActivation()`은 별 배열 identity를 갱신(`setStars([...stars])`)해 StarField가 새 now로 밝기를 다시 굽게 하고, 엣지 brightness를 `starBrightness(lastActivatedAt, virtualNow)`로 직접 재파생하며, ambient도 새 now로 다시 종합한다(요즘 상태가 식음). 배속 드라이버는 이 재파생을 throttle해 매 프레임 setStars를 피한다(헌법8). 야간 경계가 지난 밤엔 공고화로 데이터가 실제로 바뀌므로 universe/dormant를 무효화해 refetch로 반영한다.

### 데모 야간 공고화 (`shared/lib/demo/data.ts`의 `demoConsolidate`)

배속 시계가 simulated 04:00 KST를 지날 때마다 1회 발화하는 멱등 패스로, 서버 change 20 RunConsolidation의 결정론 부분을 충실히 포팅한다(좌표 재안정화·재분배·spread는 라이브 force-sim이 refetch에서 도맡으므로 데이터엔 안 싣는다). 모두 서버 `consolidation.*` 정전 값을 쓴다(데모 전용 노브 폐기 — change 27):

- **추상화 단계(요지):** 별의 자아 거리 반지름(`starRadius`, 공유 순수 함수)에서 단계를 파생하고(`abstractionStageForRadius` — 서버 `stageForRadius`, 임계 `gist_stage_radii=[40,55,68,78]`), `GREATEST(현재, 파생)`으로 단조 승급한다. 멀어진 별일수록 단계가 올라 형태가 한 단계씩 단순해진다(렌더 `aShape.w=abstractionStage`). `toStar`가 이 단계를 항상 싣는다 — 데모에 영영 0이던 버그 해소.
- **링크 재가중:** 시간·일내 결속은 ×`temporal_link_decay`(0.97) 약화, 의미·개체·공동회상은 +`semantic_link_gain`(0.01) 강화(캡 `semantic_weight_cap`=0.79).
- **가지치기:** 약하고(weight<`weak_edge_threshold`) 안 쓰인(idle>`weak_edge_idle_days`) 선의 weight를 `weak_edge_floor`(0.05)로 — 빛만 낮추고 삭제 0(헌법2). 각 별의 가장 센 연결은 보호(고립 방지).
- **re-KNN:** 연결이 하나도 없고 오래된(>`reknn_min_age_days`) 별을 같은 감정 근사로 재연결(시드 우주는 연결돼 있어 거의 무발화 — 안전망). 생성만, 삭제 0.

별·선 개수는 절대 줄지 않는다(헌법2). parity는 공유 순수 함수(`shared/lib/memory-physics`: 데모·실렌더 동일 import)와 골든 대조 테스트(`memory-physics.test.ts`: Go 식 출력 고정)로 지킨다.

## Flutter 동등성 기준

- 데모/시뮬은 순전히 웹 클라 로컬 기능이다(서버·proto·RPC 변경 없음). Flutter는 데모 모드 자체를 가질 필요는 없다.
- 포팅이 의미 있는 것은 가상 시계 패턴뿐이다: 밝기·잠듦 파생 함수(`activation`/`starBrightness`/`isDormant`)는 `now`를 주입받는 순수 함수여야 하고, 시간 시뮬을 붙인다면 "데이터를 과거로 밀지 말고 현재 시각을 미래로 보내라"는 단조성 규칙(병합이 `max(lastRecalledAt)`)을 따라야 한다.
- 시뮬 패널·이론 캐러셀·페르소나·랜딩 SVG 인터랙션은 웹 렌더 표면이며 동등성 대상이 아니다.

## 수용 기준

1. 체험 진입은 plan 47 온보딩(페르소나 → 모드 선택)을 거쳐 자유모드로 들어간다. 자유모드 기본 화면에는 좌하단 `기억 실험실`/`뇌과학 이론` 칩과 컨트롤러가 뜨지 않고(plan 47), 데모 데이터 프레임(가상 시계·재점화·연결 생성)은 그대로 동작한다. 비demo(로그인) 메인 우주에는 데모 UI가 나타나지 않는다.
2. 시간 컨트롤은 배속 셀렉터(`24시간`·`48시간`·`72시간`·`정지`)다(change 24). 휴지/기본 배속은 셀렉터와 분리된 `demo_clock.default_hours_per_second`(1)이고, 셀렉터 첫 옵션이 곧 기본은 아니다. 정지가 아니면 가상 시계가 선택 배속으로 계속 흐르고, 별 반지름·밝기·드리프트가 갱신돼 별이 서서히 멀어지고 어두워진다(점프 아님, 실제 감쇠 수식 `LAMBDA=ln2/30` 그대로). 밝기 재파생(`refreshActivation`)은 04:00 경계 없는 step에서만 `demo_clock.refresh_ms`로 throttle하고, 경계 step은 공고화 invalidate→refetch가 새 now로 다시 굽는다(이중 재구성 방지 — 배속을 올려도 렉이 늘지 않는다). 별 위치 드리프트는 layout이 매 프레임 따로 굴린다. 시계가 simulated 04:00 KST를 지날 때마다 `demoConsolidate()`가 1회씩 발화한다(빠른 배속의 다중 경계도 각 1회).
3. 가상 시간이 충분히 지나 raw activation이 임계(`2·A_MIN`) 이하가 된 별은 잠든 별로 분류돼 잠든 별 목록에 나타나되, 밝기는 `A_MIN=0.05` 바닥 밑으로 내려가지 않는다.
4. 별을 클릭해 ≥2초 머무르면 불변 원본 일기가 열리고 그 별의 `lastRecalledAt`이 가상 now로 전진해(재점화) 별이 다시 밝아진다. universe 무효화로 화면에 반영된다.
5. 두 별을 각각 ≥2초씩 번갈아 회상하면 사이 연결 weight가 +0.05씩(상한 1.0) 즉시 굵어지고, 연결이 없던 페어면 `co_recall` 엣지를 로컬 생성한다(서버 쓰기 없음).
6. 별을 <2초만 스치면 어떤 연결도 굵어지지 않는다(능동 인출 게이팅 — dwell 타이머 취소).
7. 체험 우주에서 새 일기/별을 띄우면 같은 날(temporal)·같은 mood(semantic)·최근 회상 별(흥분성 편향) 연결이 생성된다. 기억 실험실은 별도 설명 카드 없이 짧은 피드백만 띄운다.
8. "다감정 하루"는 한 일기를 색이 다른 N개 조각 별로 갈라 강한 일내 선(0.8)으로 묶어 태운다.
9. 랜딩 카드의 "체험 우주에서 해보기"는 `startDemoSession()` 후 `/?sim=<id>`로 이동한다(plan 47: 온보딩부터 시작). 자유모드 셸에서는 `?sim`으로 이론 모달을 자동으로 열지 않는다(이론 표면은 plan 48 튜토리얼 소관).
10. 비데모 모드에서 `virtualNowMs() === Date.now()`이고 패널·재점화·로컬 weight 분기가 일절 개입하지 않는다.
11. "처음으로"/페르소나 전환은 가상 시계 0 + demo 리셋으로 초기 우주 복귀(자유모드 유지 — 온보딩으로 돌아가지 않는다). "체험 종료"는 진입 흐름까지 비우고, 새로고침 시 offset과 추가분이 소멸한다.
12. 새 이론은 `THEORIES`에 항목 하나를 더하면 이론 모달에 나타난다(시간 의존 이론은 기존 시간 보내기 버튼이 트리거).
13. `shared/lib/demo/{clock,observe,simulate}.ts`·`entities/theory/model/registry.ts`·`widgets/demo-sim/model/time-travel.ts`는 three/React/DOM을 import하지 않는다.
14. 백엔드·proto·RPC·sqlc 변경 없음(FE 전용).
15. 랜딩 카드는 대응 이론 상태 배지(✅/🚧 + plan NN)를 달고, JourneyAct 챕터 순서가 19–27 이론 흐름(엔그램→분할→연결·강화→재공고화→망각→요즘 상태→야간→공명)과 정합한다.
