# 48. demo-guided-tour - 첫 별 튜토리얼 스포트라이트 투어 (as-built)

> 빈 우주에서 **첫 별을 띄우고, 그 별을 눌러 기억을 읽은 뒤, 조작법을 배우는** 첫 별 흐름을 스포트라이트로 안내한다(change 34). 같은 투어 머신(`widgets/demo-tour/tour.machine`)이 **데모 우주**(plan 47 `기능 하나하나 알아보기` — 고정 fixture 별)와 **실계정 최초 빈 우주 진입**(감정색 게이트 이후 별이 없고 미완료면 자동 시작 — 실제 작성 결과 별)을 모두 구동하고, `tourContext`(demo/account)로 단계를 거른다. 특별한 게임 미션이 아니라 cosimosi의 어두운 우주·별빛 톤에 맞춘 가벼운 dim + spotlight + coach card 레이어다. | 범위: FE | 상태: 구현 완료

## 목적

처음 온 사용자가 가장 먼저 해야 할 일은 "일기를 써서 첫 별을 띄우는 것"이다. 그래서 튜토리얼은 기능 버튼을 둘러보는 순서가 아니라 **첫 별을 만들고, 그 별을 눌러 기억을 읽고, 그다음 조작법을 배우는** 사용자의 실제 첫 경험 순서로 흐른다(change 34). 데모 우주(`기능 하나하나 알아보기`)와 실계정 최초 빈 우주 진입이 같은 튜토리얼 언어를 공유하되, 데모는 고정 fixture 별을, 실계정은 실제 작성 결과 별을 대상으로 한다.

사용자가 긴 설명서를 읽는 대신, 현재 화면의 버튼·별·패널이 빛으로 강조되고 작은 안내문이 따라붙는다. 첫 별을 만들고 읽기 전까지는 카메라 조작이 잠겨 길을 잃지 않게 하고, 회상 설명이 끝나면 풀려 시점 전환·항해를 실습한다. 행동 단계는 직접 눌러 진행하고, 정보 단계만 `다음`으로 넘긴다.

튜토리얼은 과한 미션이나 보상 시스템이 아니다. 디자인은 cosimosi 컨셉과 맞게, 어두운 우주 위에 별빛 테두리와 은은한 dim을 얹는 정도로 충분하다.

## 범위 / 비목표

**범위**

- 데모 모드 선택 `기능 하나하나 알아보기`(빈 fixture 우주)와 실계정 최초 빈 우주 진입(per-user 미완료 시 자동 시작)이 같은 투어 머신을 공유하고, `tourContext`로 단계를 거른다(데모 페르소나/시간 단계는 실계정에 안 나온다).
- 첫 별 중심 순서로 하이라이트한다: ① 빈 우주 안내 + `새 별 띄우기` → ② 작성 폼·`별 나누기`·조각 확인·`별 띄우기` → ③ 생성된 별 클릭 → ④ 회상 패널 설명(job 47 포트레이트·메타·조각/원본) → ⑤ 망원경(일기 패널·`별` 탭·별 패널) → ⑥ 시점 전환·항해 실습(change 12) → ⑦ 테마·페르소나/시간(데모만)·UI 숨김·완료.
- 행동(action) phase는 그 행동(버튼/탭/별 클릭)으로만 진행하고 `다음`이 없다. 정보(info) phase는 이미 열린 표면/패널 전체를 하이라이트하고 `다음`으로 넘긴다. 이 구분은 phase의 `kind`로 머신 상태가 된다.
- 데모는 고정 fixture 별/id, 실계정은 실제 첫 별 id(`compose` `submitted` payload)를 추적해 카메라가 그 별을 frame-all 프레이밍하고 spotlight가 하이라이트한다. 캔버스 안 별은 live 좌표를 화면 rect로 투영해 짚는다(씬 안 `<Html>` 없이 — 헌법8).
- 튜토리얼 시작부터 첫 별 클릭/회상 설명 전까지 카메라(마우스·터치·키보드) 조작을 잠그고, 그 뒤 해제한다(별 클릭·폼 버튼·HUD는 불간섭).
- `이전`, `다음`, `건너뛰기`, `자유롭게 탐험하기`, `다시 보기` 동선을 제공한다. 데모 진행 상태는 데모 세션에, 실계정 완료 상태는 per-user localStorage에 저장한다.

**비목표**

- 복잡한 미션, 점수, 보상, 업적, 애니메이션 컷신을 만들지 않는다.
- 실계정 튜토리얼 완료 상태를 서버/proto/DB에 저장하지 않는다(FE per-user localStorage만).
- 실계정 첫 튜토리얼을 위해 가짜 서버 별이나 고정 서버 좌표를 만들지 않는다(실제 작성 결과 + force-sim 결과 위 투영, 헌법3).
- 별/시냅스/원본 일기 데이터를 튜토리얼 때문에 삭제·수정하지 않는다.
- 튜토리얼 전용 backend/proto/RPC/DB schema를 만들지 않는다.
- `MemoryForm`·`SegmentMemory`·`RecordMemory`·데모 프리셋 compose의 production 동치 규칙을 바꾸지 않는다(튜토리얼은 그 흐름을 앞에서 안내할 뿐).

## 설계 요점

### 데모 flow 상태

plan 47의 `tutorial_tbd` 상태를 실제 튜토리얼 상태로 확장한다.

- `tutorial`: 튜토리얼 투어가 진행 중인 상태.
- `free`: 자유모드. 튜토리얼 완료 또는 건너뛰기 후 도착하는 상태.

튜토리얼의 현재 단계 index는 데모 세션 전용 sessionStorage에 저장한다. 새로고침해도 같은 데모 세션이면 같은 단계에서 이어갈 수 있고, `exitDemoMode()`는 이 상태를 지운다. 사용자가 `자유롭게 탐험하기`로 끝내면 flow는 `free`가 되며, 이후 같은 세션에서는 온보딩/튜토리얼이 다시 자동으로 뜨지 않는다. `다시 보기`는 튜토리얼 완료 후 사용자가 명시적으로 누르는 재진입점이다.

### 스포트라이트 레이어

튜토리얼은 우주 캔버스 밖의 DOM 레이어다. 3D 씬 안에 `<Html>`을 넣지 않는다.

구성은 단순하다.

- 딤 + 둥근 구멍: box-shadow 큰 spread로 현재 target 영역만 남기고 화면을 어둡게 덮는다(구멍 모서리는 glow와 같은 둥근 라운드). 투명 패널 4개가 구멍 밖 클릭을 막아, 하이라이트된 버튼만 누를 수 있고 나머지는 비활성이다.
- glow 하이라이트: 같은 둥근 구멍 둘레에 바짝 붙는 흰 테두리 + 번지는 별빛 그림자(box-shadow). target이 빛나듯 떠오른다. reduced-motion이 아니면 은은히 맥동한다.
- coach card: 현재 단계의 제목, 짧은 문장, `이전`/`다음`/`건너뛰기` 버튼을 가진 작은 안내 카드.
- progress dots: 현재 단계 위치만 보여주는 작은 점들.

캔버스는 사라지거나 언마운트되지 않는다. 디자인 톤은 검은 우주 + 따뜻한 별빛 + 현재 mood/스킨과 어긋나지 않는 얇은 색 포인트다. 큰 마케팅 카드나 장식용 orb를 만들지 않는다. target을 찾지 못하면(예: 캔버스 안 별, 시작/끝 인사) 딤이 클릭을 막지 않아 별을 직접 눌러볼 수 있게 하고 화면 중앙 안내 카드만 띄운다(crash 없음).

### target anchor 계약

튜토리얼이 버튼 위치를 안정적으로 찾도록, plan 47에서 추가된 주요 HUD 요소에 `data-tour-id` 또는 동등한 ref registry를 부여한다.

target id(change 34 확장):

- `ui-toggle`·`theme`·`view`·`telescope`: HUD 버튼(상단 중앙·좌상단·우상단).
- `persona`·`persona-popover`·`time`·`time-speed`: 데모 전용 페르소나/시간 버튼·팝오버(account 단계엔 없음, A16).
- `new-star`: 하단 중앙 `새 별 띄우기`(첫 행동 phase에서만 하이라이트).
- `compose-form`·`compose-body`·`segment`·`review-panel`·`submit-stars`: 작성 폼 root·본문·`별 나누기`/`별로 분해` 버튼·조각 검토 패널·`별 띄우기` 버튼. 행동 phase는 버튼을, 정보 phase는 본문/검토 패널 전체를 짚는다(A5·A6).
- `explorer-diary-tab`·`explorer-star-tab`: 망원경 탭 버튼(행동 — `별` 탭은 눌러 전환, A14). `explorer-diary-panel`·`explorer-star-panel`: 탭 콘텐츠 영역 전체(정보 — A13·A14).
- `recall-panel`: 회상 패널 전체(정보 — job 47 포트레이트·메타·조각/원본 동선 설명, A11).
- `canvas-star`: 3D 캔버스 안 생성/fixture 별. DOM rect가 없으므로(헌법8) universe-canvas가 그 별의 live force-sim 좌표를 화면 rect로 투영해 shared 레지스트리(`shared/lib/tour-target`)에 싣고, 투어 overlay가 그 rect로 spotlight를 친다(A7·A8).

DOM rect는 `getBoundingClientRect`를 rAF로 따라가고, canvas-star는 카메라 투영 rect를 구독한다. 이 hook은 DOM/캔버스를 직접 참조하므로 ui 인접에 둔다. 순수 step 정의(`steps.ts`)는 React/DOM/three를 import하지 않는다(헌법4).

### 단계 시나리오

튜토리얼은 사용자의 실제 첫 경험 순서로 흐른다(change 34) — 빈 우주에서 첫 별을 만들고, 그 별을 눌러 읽고, 그다음 조작법을 배운다. 1–4단계 동안 카메라 조작은 잠긴다.

1. **빈 우주 + 새 별 띄우기**(행동): "지금 우주가 비어 있어요. 일기를 써서 첫 별을 띄워볼까요?" 하단 `새 별 띄우기`를 하이라이트한다 → 사용자가 눌러 작성 폼을 열면 진행.
2. **작성 폼**(정보·행동·정보·행동): ① 본문 영역을 정보 하이라이트("여기에 오늘 하루를 적어요") → ② `별 나누기`(데모)/`별로 분해`(실계정) 버튼을 행동 하이라이트 → 누르면 ③ 조각 검토 패널을 정보 하이라이트 → ④ `별 띄우기` 버튼을 행동 하이라이트 → 눌러 첫 별을 만든다. 데모는 고정 fixture, 실계정은 실제 작성.
3. **생성된 별**(행동): 카메라가 그 별을 frame-all 프레이밍하고 캔버스 별 투영 rect로 spotlight를 친다. "별을 눌러 그 안의 기억을 펼쳐볼까요?" → 별을 눌러 회상 패널을 열면 진행.
4. **회상 패널**(정보 2): job 47 완료 기준의 상단 별 3D 포트레이트, 날짜·기분·강도·추상화 메타, 조각/원본 동선을 설명하고, 재공고화로 형태·빛이 달라짐을 덧붙인다. — 여기서 카메라 lock이 풀린다.
5. **망원경**(행동·정보·행동·정보): 망원경 버튼(행동) → `일기` 패널 전체 정보 하이라이트(이미 일기 탭이 열려 있으므로 버튼이 아니라 패널을, A13) → `별` 탭 버튼 행동 하이라이트(눌러 전환, A14) → `별` 패널 전체 정보 하이라이트(잠든 별 안내).
6. **시점 전환 + 항해 실습**(change 12): 멀리서 회전·줌, 가까이서 시선·전진을 디바이스별 문구로 안내하고, 임계값(`spec/values.yaml` `demo_tour`)을 넘으면 자동 진행한다. 실습 구간은 딤·카드를 거의 투명하게.
7. **부수 기능**: 테마(정보), 데모 전용 페르소나·시간(account엔 없음, A16), UI 숨김/보이기(행동), 완료.

`자유롭게 탐험하기`/`건너뛰기` → 데모는 `free`, 실계정은 per-user 완료 저장. `다시 보기`로 재진입.

각 단계의 문장은 짧게, 현재 target과 직접 관련된 한두 문장만 보여준다.

### 사용자 액션 처리

튜토리얼 진행은 XState 머신(`widgets/demo-tour/model/tour.machine.ts`)이 소유한다 — step·phase 전이·게이팅·항해 실습 자동진행을 한 순수 FSM으로 모델링하고, 컴포넌트는 스냅샷에서 파생해 렌더하며 입력을 이벤트로 보낸다. 단계는 phase로 나뉜다. **행동 phase는 그 행동(버튼 누름·UI 토글·팝오버 열림·페르소나 전환·항해 동작)으로만 진행한다 — `다음` 버튼이 없다.** 머신이 그 상태에서 `NEXT`를 받지 않아 `snapshot.can('NEXT')=false`이고, 그것으로 UI가 버튼 노출을 끈다(명령형 분기 없음). **정보 phase**(결과 안내·인사)만 `다음`으로 넘긴다. 모든 단계의 마지막 phase는 정보 phase라 단계 전환이 막히지 않는다. `이전`은 phase/step을 거슬러 가고, `건너뛰기`는 투어 전체를 종료한다. 망원경 탭처럼 표면을 열어야 target이 있는 단계는 튜토리얼이 그 시트를 열어준다.

- 페르소나/시간 단계: 먼저 버튼을 짚고("눌러보세요"), 사용자가 누르면 팝오버가 열리며 하이라이트가 팝오버 위로 옮겨간다 — coach card가 팝오버를 가리지 않게 버튼을 먼저, 팝오버를 나중에 짚는다. 팝오버를 연 행동·페르소나 전환·시간 이동은 페이지가 머신에 이산 이벤트로 알리고, 머신의 가드가 충족을 판정한다(페르소나 전환·시간 이동은 phase 진입 baseline 대비 변화로).
- 망원경/탭 단계: 망원경 시트를 열고 해당 탭을 하이라이트한다. 다음 단계로 넘어갈 때 필요하면 시트를 유지하거나 닫는다.
- 새 별 단계: 사용자가 `새 별 띄우기`를 누르면 plan 47의 프리셋 작성 폼이 열리고, 안내가 `별 나누기` 조각화 fan-out을 짚는다(change 25). 정보 phase(`다음`)라 누르지 않아도 진행에 문제 없다.
- 별 클릭 단계: 별 선택/회상이 가능하면 해 보게 하고, 대상 별을 못 찾으면 "별을 눌러 기억을 읽어요" fallback 안내만 보여준다.
- 시점 전환 실습 단계(change 12): 항해 입력을 **관찰만** 한다(조작·물리·제스처 규칙 불변). 머신이 invoke한 `navSampler` 액터(ui/pages가 provide)가 `navigation-input` 싱글턴의 단조 누적 카운터(시선·전진)와 회전·줌 카운터를 **rAF로 샘플링**해 phase 진입 baseline 대비 임계 도달 순간에 `PRACTICE_MET`를 한 번 보낸다 — 매 프레임 React state·머신 이벤트를 만들지 않는다(헌법4). 가까이서로의 전환은 투어가 `TOGGLE_MODE`로 구동하고, segment를 벗어나면 기본 멀리서로 되돌린다. 웹 전진은 D-pad 전진 버튼(보조 `W`·↑), 모바일 전진은 두 손가락 세로 쓸기로 충족된다. 실습 phase는 그 동작으로만 진행하고(`다음` 없음), `건너뛰기`로 투어를 종료할 수 있다(접근성 불변).

### 기존 UI와 충돌 방지

튜토리얼 overlay는 다른 강한 표면과 한 번에 하나만 공존한다. 단계 전환 시 사이드바, 팝오버, 망원경 시트, 회상 패널 중 현재 단계에 필요 없는 표면은 닫는다. 단, 해당 단계에서 설명하는 표면은 열어 둔다.

`UI 숨기기` 단계가 특수하다. 사용자가 UI를 숨기면 다음 target들이 사라질 수 있으므로, 튜토리얼은 다음 단계로 넘어갈 때 UI를 자동으로 보이게 복구한다. 튜토리얼 overlay 자체는 UI 숨김 대상이 아니다.

## 수용 기준 (인수 조건)

1. 데모 모드 선택 `기능 하나하나 알아보기`는 자유모드 30일 genesis가 아니라 튜토리얼 전용 고정 fixture 우주(빈 우주에서 출발)로 진입한다.
2. 실계정 사용자가 인증·멤버십·감정색 게이트를 통과해 `/`에 처음 들어왔고, 개인 우주에 별이 없으며, per-user 튜토리얼 완료 상태가 없으면 첫 별 튜토리얼이 자동 시작된다(`universe.isPending` 동안은 시작 안 함).
3. 이미 별이 있는 실계정 사용자 또는 완료/건너뛰기 상태가 있는 사용자는 첫 별 튜토리얼이 자동 재시작되지 않는다.
4. 첫 단계는 빈 우주 안내 + `새 별 띄우기` 행동 하이라이트다(행동 phase — `다음` 없음, 버튼을 눌러야 진행).
5. 작성 폼이 열리면 정보 phase는 폼 본문 영역을, 행동 phase는 `별 나누기`/`별 띄우기` 버튼을 짚는다(이미 열린 표면의 버튼을 오해시키지 않는다).
6. 데모는 `별 띄우기` 후 fixture 고정 별이, 실계정은 실제 생성된 첫 별이 태어나고, 카메라가 그 별을 frame-all 프레이밍하고 spotlight가 하이라이트한다(서버 좌표 영속 없음 — force-sim 결과 위 투영).
7. 튜토리얼 시작부터 첫 별 클릭/회상 설명 전까지 마우스·터치·키보드 카메라 조작이 잠긴다(별 클릭·폼 버튼은 잠기지 않음). 회상 설명 후 풀리고 항해 실습이 그다음에 온다.
8. 생성된 별 단계는 행동 phase로, 별을 눌러 회상 패널을 열어야 진행된다. 회상 패널 설명은 job 47 기준 포트레이트·메타·조각/원본 동선이다.
9. 망원경 `일기`가 이미 선택된 상태에선 탭 버튼이 아니라 일기 패널 전체를 정보 하이라이트하고, `별` 탭은 행동 하이라이트(눌러 전환)이며 전환 후 별 패널을 정보 하이라이트한다.
10. 행동 phase에는 `다음`이 없고 정보 phase에만 있다 — phase의 `kind`(머신 `can('NEXT')`)로 보장된다.
11. 컨텍스트(`tourContext`)에 없는 단계는 자동 제외된다 — 실계정엔 데모 페르소나/시간 단계가 나타나지 않는다.
12. 완료/건너뛰기 → 데모는 `free`로 수렴, 실계정은 per-user localStorage(`cosimosi:first-star-tour:{userId}`)에 완료 저장해 자동 재시작 안 함. `다시 보기`는 데모=고정 fixture 재준비, 실계정=현재 우주 비파괴 둘러보기.
13. 튜토리얼 overlay는 캔버스 밖 DOM이며 3D 씬 안 `<Html>`을 추가하지 않는다(캔버스 별은 좌표 투영 rect로 짚는다). 순수 step 정의/머신은 React/DOM/three를 import하지 않는다(헌법4).
14. 시점 전환 항해 실습(change 12)은 디바이스별 문구로 멀리서 회전·줌·가까이서 시선·전진을 안내하고, 임계값(`spec/values.yaml` `demo_tour`)을 rAF 샘플링으로만 관찰해 자동 진행한다(매 프레임 React state 없음, 항해 물리 불변).
15. 백엔드·proto·DB schema 변경 없이 FE 상태와 데모 fixture만으로 구현된다.
16. build·lint·typecheck·steiger를 통과하고, 기존 데모 자유모드 genesis·데모 프리셋 작성·실계정 `새 별 띄우기`·별 클릭 회상·망원경 일기/별 탭·카메라 조작에 회귀가 없다.
