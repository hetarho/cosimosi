# 51. 배경 스킨 — 셰이더 아트 툴킷 적용 (background-skins)

> [plan/50 셰이더 아트 툴킷](50.shader-art-toolkit.md)의 **이펙트 패밀리**를 배경 축에 적용한다. 배경 효과의 **조립을 entity(`entities/appearance/ui`)로 옮기고**(별 `star-body`·나 `self-form`과 같은 층), **위젯은 스킨이 몇 개든 N-제네릭하게** 분기/배치만 한다. 매직넘버가 박힌 기존 7효과를 제거하고, 툴킷으로 빚은 새 종(`galaxy` 무료 · `vortex` · `crystal` · `mandala` + 확장 가능)으로 교체한다. | 범위: FE

> **3층 아키텍처의 2층(배경).** 선행: plan 50(툴킷). plan 52(형태 스킨)와는 독립(다른 축). 토대 도메인 모델: [plan/44 커스터마이즈](44.customization-system.md).

## 목적
<!-- 왜 이 기능인가. 사용자·제품 관점. -->

배경은 우주의 첫인상이고 사용자가 가장 먼저 바꿔보는 스킨이다. 그런데 지금 배경만 3층 패턴에서 어긋나 있고, 만들기·튜닝이 가장 어렵다.

- **배경 조립이 위젯에 있다.** 별·나·시냅스는 entity의 `ui`가 형태를 조립([star-body](../../frontend/src/entities/star/ui/star-body.ts)·[self-form](../../frontend/src/entities/appearance/ui/self-form.ts)·[SynapseFilaments](../../frontend/src/entities/synapse/ui/SynapseFilaments.tsx))하는데, **배경만** [widgets/universe-canvas/ui/UniverseNebula.tsx](../../frontend/src/widgets/universe-canvas/ui/UniverseNebula.tsx)의 거대 `if/else`가 7효과를 직접 조립한다.
- **튜닝·격리가 안 된다.** 7효과가 한 `useMemo`에 매직넘버로 묶여, 한 효과의 수치를 바꾸면 옆 효과에 영향이 갈 수 있고 "이 숫자가 뭘 바꾸나"를 알 수 없다.
- **새 배경이 비싸다.** 효과 추가 = 거대 분기에 가지 + 계수 박기.

해법은 plan 50 툴킷 위에서 **배경을 나머지 축과 같은 패턴**으로 맞추는 것이다: 조립을 `entities/appearance/ui`로 옮기고, 위젯은 선택된 스킨을 **스킨 수와 무관하게** 그리기만 하며, 각 스킨은 툴킷 기법 조립 + 주석 달린 params로 정의한다. 그러면 사용자는 카탈로그 한 곳에서 수치를 직접 만지며 미감을 잡고(스킨 간 격리), 새 배경은 카탈로그 항목 1개 + 조립 함수 1개로 끝난다.

## 범위 / 비목표
<!-- 무엇을 포함하고, 무엇은 명시적으로 안 하나. -->

**범위**
- **조립을 entity/ui로 이전:** `entities/appearance/ui/`에 배경 스킨별 조립(예: `background-form.ts`) — plan 50 이펙트 기법으로 색/마스크를 만들고 mood(감정색)를 합성한다. `self-form.ts`의 형제.
- **위젯 N-제네릭:** [widgets/universe-canvas](../../frontend/src/widgets/universe-canvas)는 선택 id로 entity 조립을 골라 **머티리얼/메시 lifecycle·`uTime`·uniform만 운반**한다. 효과별 분기 없이 registry/`for`로 N개를 처리 — 스킨이 몇 개든 위젯 코드는 그대로.
- **카탈로그 교체:** `entities/appearance/model`의 `Background`·`BackgroundEffect`를 const 카탈로그에서 파생(plan 44 패턴), 스킨별 **주석 달린 `params`**, `DEFAULT=galaxy`. 기존 7종(`vast`·`lively`·`calm`·`aurora-veil`·`signal-noise`·`abyssal-sea`·`cosmic-cliffs`)·7effect 제거.
- **새 종:** `galaxy`(무료) · `vortex` · `crystal` · `mandala`. **확장 가능**(추가 = 카탈로그 항목 + entity 조립 함수, 위젯·BE 무수정).
- **검증·경제 정합:** 유효 배경 id를 카탈로그에서 파생(미지 id→`galaxy` 폴백). `values.customization.free.background=galaxy` + 유료 3종 가격(기존 배경 가격 제거).
- **소비처 정합:** `AppearanceSwitcher`·랜딩/사인인/초대/방문의 `backgroundMeta`/`paletteForBackground`를 새 id로.

**비목표**
- **별·나·시냅스 형태는 plan 52.** 이 plan은 배경 축만.
- **별가루/구매/소유권/proto/DB 재설계는 안 한다(plan 44 유지).** `theme` 와이어 id는 문자열 그대로, item-id 소유권 모델 그대로. 유효-id를 카탈로그에서 파생하는 작은 정리만(스킨 추가가 BE 코드 수정을 요구하지 않게).
- **최종 미감 계수를 plan이 못 박지 않는다(육안 튜닝·job).**
- **멀티패스 인프라를 강제하지 않는다.** 기본 단일 패스·단일 구 유지(스킨이 필요로 하면 toolkit/job이 확장).

## 설계 요점
<!-- 어떻게 구현하나. -->

### A. 조립을 entity/ui로 (3층 정합)
- `entities/appearance/ui/background-form.ts`(또는 유사) — 스킨별 조립 함수가 plan 50 이펙트 기법(`polar`·`logSpiral`·`worley`·`kaleido`·`fresnel`·`fbm`·`domainWarp` …)을 조립해 색/마스크 노드를 만든다.
- **mood(감정색) 합성은 여기서**(도메인 규칙): 검은 우주 받침 위에 `emotionSlots`개의 요즘 감정색(presence·R-비중)을 입힌다. 툴킷은 "형태", entity는 "그 형태에 감정색을 입힌 스킨".
- `self-form.ts`와 같은 층·같은 패턴(entity가 자기 시각을 소유, widget이 소비).

### B. 위젯 N-제네릭 (스킨 수 무관)
- [UniverseNebula.tsx](../../frontend/src/widgets/universe-canvas/ui/UniverseNebula.tsx)(또는 정명한 `BackgroundLayer`)는 **공통 셸만**: geometry/material 생성·dispose, `uTime`/`uBright`/`uMotion`/`uPresence`/감정 uniform, reduced-motion·arousal 게인, BackSide·`depthWrite/Test=false`·`renderOrder=-11`·raycast none·draw call 1.
- 선택 id로 **entity 조립을 lookup**해 `colorNode`(또는 머티리얼)를 주입한다. 효과별 `if/else`/`switch` 없음 — registry/map. **스킨이 N개여도 위젯 코드는 동일**, 새 스킨이 위젯 수정을 요구하지 않는다.

### C. 카탈로그·params (plan 44 + 50 패턴)
- `Background`·`BackgroundEffect`를 const 카탈로그 key에서 파생(수동 union 제거). `BackgroundMeta.params` = 효과별 **주석 달린 수치**(plan 50에서 정한 "튜닝값=entity params"). 예: `galaxy { arms, spinSpeed, coreGlow, armSharpness, scatter }`.
- `DEFAULT_BACKGROUND='galaxy'`. zod/normalize 경계에서 미지 id→`galaxy`.

### D. 4종 레시피 (핵심 기법 — 정확 수치는 job·육안)
| kind | 무료 | 기법 조합 | presence=0(빈 우주) |
|---|---|---|---|
| `galaxy` | **무료/기본** | `polar`+`logSpiral` 나선팔 · `fbm` 흩뿌림 · `fresnel` 코어 · 회전 | 중립 코어 글로우 + 희미한 나선(차분) |
| `vortex` | 유료 | `polar` 중심 어둠 · `logSpiral`+`domainWarp` 강착원반 · `fresnel` 링 | 어두운 중심(안전) + 희미한 링 |
| `crystal` | 유료 | `worley`→`cellEdge` 경계선 · 셀 내부 미세 색차 · 엣지 글로우 | 어두운 격자 + 희미한 중립 엣지 |
| `mandala` | 유료 | `kaleido` 대칭 fold · `fbm`/`worley` · `contourSteps` 층 | 희미한 대칭 무늬(중립) |

### E. 검증·경제 정합 (스킨 추가가 BE 무수정)
- 유효 배경 id를 카탈로그(또는 values 가격키 + 무료 id)에서 파생 — plan 30/44의 `validThemes` 화이트리스트를 카탈로그 파생으로. 새 배경 추가 = FE 카탈로그 + values 가격만(BE 코드·proto·DB 무수정).
- `theme` 문자열 id·item-id 소유권·구매 흐름(plan 44)은 그대로.

### F. 보존 불변 (회귀 경계)
검은 우주 받침 + mood 합성, 별색·깊이 불간섭(BackSide·depth off·renderOrder·raycast none), draw call 1, frozen-time `uTime`, reduced-motion 정지, arousal 게인. 전부 유지.

## 수용 기준 (인수 조건)
<!-- 평서문. -->
1. 배경 조립이 `entities/appearance/ui`(별 `star-body`·나 `self-form`과 같은 층)에 있고, 위젯(`universe-canvas`)은 조립을 소유하지 않는다 — 선택 id로 entity 조립을 lookup해 렌더 lifecycle(머티리얼/uniform/`uTime`)만 운반한다.
2. 위젯은 스킨 수와 무관하게 동작한다 — 효과별 `if/else`/`switch` 없이 registry/`for`로 N개를 처리하고, **새 배경 스킨 추가가 위젯 코드 수정 없이** 된다.
3. 배경 스킨은 plan 50 툴킷의 이펙트 기법을 조립해 만들어진다(자체 매직넘버 재구현이 아님). 튜닝 수치는 entity 카탈로그의 **주석 달린 `params`** 에 있고, 한 스킨의 `params`를 바꿔도 다른 스킨에 영향이 없다.
4. 배경 카탈로그는 `galaxy`(무료·`DEFAULT_BACKGROUND`)·`vortex`·`crystal`·`mandala`이고 확장 가능하다(추가 = 카탈로그 항목 + entity 조립 함수). 기존 7종·7effect는 제거됐다.
5. 4종 모두 검은 우주 받침 위에 mood(`emotionSlots`·presence·감정색)를 입히고, 별의 mood 색·깊이를 침범하지 않으며, presence=0이면 안전한 중립 딥스페이스로 보인다. reduced-motion 정지·arousal 게인·draw call 1이 유지된다.
6. 미지/레거시 배경 id는 `galaxy`로 폴백하고 렌더·편집 UI가 크래시하지 않는다.
7. 유효 배경 id와 무료/가격이 카탈로그·values에서 파생돼 **새 배경 추가가 BE 코드 수정 없이** 된다(`theme` 문자열 id·proto·DB·item-id 소유권은 plan 44 그대로, 회귀 없음).
8. 튜닝 `params`는 코드(entity)에 있고 `values.yaml`엔 셰이더 계수가 없다 — values는 `customization.free.background=galaxy` + 유료 3종 가격만(기존 배경 가격 제거).
9. FE `build`·`lint`·typecheck·`steiger` 통과, `AppearanceSwitcher`·랜딩/사인인/초대/방문·공유 우주 회귀 없음. 테스트 추가(카탈로그/스키마 파생·미지 폴백·위젯 N-제네릭·스킨 조립).
