# 01. 사인인 - Supabase Auth + 세션 머신 (as-built)

> 현재 구현 기준 문서. 웹과 Flutter가 같은 인증 경계, 라우트 보호, 토큰 주입 방식을 공유하기 위한 SSOT다. | 범위: FE/Auth | 상태: 구현 완료(세션 상태는 XState 머신 — tech/state-machines.md)

## 목적

cosimosi의 개인 우주는 Supabase 계정의 `sub`로 스코프된다. 미인증 사용자는 보호된 우주/관리/선물 수락 화면에 들어갈 수 없고, 인증된 세션의 access token은 모든 보호 RPC의 `Authorization: Bearer <token>` 헤더로 전달된다. 세션 수명주기는 현재 XState v5 머신이 소유한다.

## 현재 구현

- Supabase 브라우저 클라이언트는 `frontend/src/shared/api/supabase.ts`에 있다.
  - `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`가 없으면 앱 부팅 시 명확한 에러로 실패한다.
  - `persistSession`, `autoRefreshToken`, `detectSessionInUrl`가 켜져 있다.
  - `getAccessToken(): Promise<string | null>`을 공개하고, Connect transport가 이를 await해서 Bearer 헤더를 붙인다.
- 세션 상태는 `frontend/src/app/model/session.machine.ts`가 소유한다.
  - 상태는 `loading | authed | anon`.
  - 부팅 시 `supabase.auth.getSession()`을 호출하고, 상시 `onAuthStateChange`를 구독한다.
  - 사용자 id가 바뀌면 `resetUniverseData()`와 analytics identity 동기화를 수행해 이전 계정의 우주/일기 캐시가 다음 계정에 섞이지 않게 한다.
  - sign-out 실패로 Supabase 이벤트가 오지 않는 경우를 위해 `FORCE_ANON` 이벤트를 지원한다.
- React 바인딩은 `frontend/src/app/ui/session-context.tsx`가 소유한다.
  - `SessionContext = createActorContext(sessionMachine)`.
  - `useAuthActions()`가 `signInWithOtp`, `verifyOtp`, `signInWithGoogle`, `signOut`을 제공한다.
  - `App.tsx`는 `SessionContext.Provider`로 `RouterProvider`를 감싼다.
- 사인인 UI는 `frontend/src/app/ui/SignInScreen.tsx`(폼 본체)이며, `/sign-in` 라우트가 `frontend/src/app/ui/SignInRoute.tsx`로 마운트한다. 보호 라우트의 `SessionGate`는 미인증을 이 라우트로 리다이렉트하고(인라인 렌더 아님), 인증 후 `?redirect`로 복귀시킨다.
  - 이메일은 OTP 코드 전용이다. `signInWithOtp({ email })`로 코드를 보내고, 같은 탭에서 `verifyOtp({ email, token, type: 'email' })`로 세션을 수립한다. 검증 성공 시 `SignInRoute`가 `redirect`(없으면 `/`)로 보낸다.
  - Google OAuth는 `signInWithOAuth({ provider: 'google', options: { redirectTo: origin + '/' } })`를 호출한다. `/` 복귀 시 `detectSessionInUrl`이 세션을 파싱한다(풀페이지 라운드트립이라 `?redirect`는 보존되지 않고 항상 `/`로 복귀).
  - 이메일 템플릿은 Supabase 대시보드에서 `{{ .Token }}`을 노출해야 한다. Google provider와 redirect allowlist도 대시보드 설정이 필요하다.

## 라우트 보호

- 공개 라우트:
  - `/landing`: 마케팅 랜딩 페이지. 인증 게이트 없음(루트가 우주로 옮겨가며 직접 진입 전용이 됐다).
  - `/sign-in`: 사인인 페이지(공개). `SignInScreen`을 라우트 콘텐츠로 렌더한다. `?redirect=<내부 경로>`로 인증 후 복귀 대상을 싣되, 내부 경로만 통과시킨다(`/`로 시작, `//`·`/\` 아님 — 오픈 리다이렉트 방지). 이미 인증된 사용자는 `redirect`(없으면 `/`)로 즉시 보낸다.
  - `/u/$slug`: 공개 우주 방문 페이지. `VisitService`만 호출하며 무인증 공개 표면이다.
- `SessionGate` 보호 라우트:
  - `/`: 개인 우주 셸.
  - `/my-page`: 최소 마이페이지(인증 + 멤버십 게이트). 세션 이메일 + 로그아웃 버튼만 보인다(로그아웃은 셸과 같은 `signOut` 경로 재사용).
  - `/admin`: 관리자 콘솔. 인증 뒤 서버의 AdminService allowlist가 추가로 판정한다.
  - `/gift/$token`: 받은 별 수락/거절 페이지. 양쪽 모두 cosimosi 사용자여야 하므로 인증이 필요하다.
- 옛 `/universe`·`/dormant` 라우트는 제거됐다(매칭 없음 → NotFound). 우주는 `/`이고, 잠든 별 탐색은 별도 경로 없이 우주 셸의 망원경 탐색기 `별` 탭에 흡수됐다(change 09 — 옛 `?panel=dormant`는 legacy redirect).
- `SessionGate` 상태 분기:
  - `loading`: 스플래시를 보여 사인인 화면 깜빡임을 막는다.
  - `anon`: `/sign-in?redirect=<현재 경로>`로 리다이렉트(`<Navigate replace>`, 인라인 렌더 아님).
  - `authed`: children 렌더 (+ `showChrome`일 때만 우상단 로그아웃 핀).
- `showChrome` prop(change 09): `SessionGate`는 자체 로그아웃 핀(+데모 배지/종료 버튼)을 그릴지를 `showChrome`으로 받는다. 우주 셸·일기·마이페이지(`/`·`/diary`·`/my-page`)는 `showChrome=false` — 독립 우상단 로그아웃 핀을 띄우지 않고, 로그아웃을 **우주 셸의 햄버거 사이드바**로 수렴시킨다(데모의 "체험 종료"도 사이드바로 함께 수렴). 나머지 보호 라우트(`/admin`·`/gift`)는 기본값(`showChrome=true`)이라 SessionGate 로그아웃 핀을 유지한다.
- 데모 모드:
  - `isDemoMode()`가 참이면 로그인 없이 children을 통과시킨다(anon 리다이렉트보다 먼저 판정 — 데모는 사인인으로 튕기지 않는다).
  - `showChrome`일 때 상단에 "체험 모드" 배지와 "체험 종료" 버튼을 표시한다. 우주 셸(`showChrome=false`)에서는 "체험 종료"가 햄버거 사이드바로 수렴한다.
  - 종료 시 `exitDemoMode()`, `resetDemo()`, `/landing` 이동을 수행한다(루트는 이제 보호 우주라 마케팅 랜딩으로 보낸다).

## Flutter 동등성 기준

- Flutter도 Supabase 세션을 앱 전역 lifecycle로 관리하고, 보호 화면 진입 전 `loading | authed | anon`과 동일한 세 가지 상태를 표현해야 한다.
- 이메일 OTP는 매직링크가 아니라 "코드 입력 후 verify" 흐름이어야 한다.
- Google OAuth 복귀 후 개인 우주 화면으로 들어가는 결과가 같아야 한다.
- 보호 RPC 호출 전 access token을 읽어 `Authorization: Bearer <token>`을 붙여야 한다.
- 계정 전환/로그아웃 시 우주 그래프, 일기 목록, 로컬 demo/cache 성격의 데이터가 이전 사용자에서 다음 사용자로 새지 않아야 한다.
- 우주 셸에서는 독립 로그아웃 핀 대신 셸 내비게이션(햄버거 사이드바 등)으로 로그아웃을 수렴시키고, 마이페이지 같은 보조 화면에서 같은 `signOut` 경로를 제공해야 한다.

## 수용 기준

1. 미인증 사용자가 `/`, `/my-page`, `/admin`, `/gift/$token`에 진입하면 `/sign-in?redirect=<원래 경로>`로 리다이렉트되어 사인인 화면을 본다(인라인 렌더 없음).
2. `/landing`, `/sign-in`, `/u/$slug`는 세션 없이 열린다.
3. 이메일 OTP 검증 후 `/sign-in`은 `redirect`(없으면 `/`)로 보내고, Google OAuth 복귀(`/`) 후 `SessionGate`가 우주 셸을 렌더한다. `?redirect`는 내부 경로만 허용한다(오픈 리다이렉트 방지).
4. 새로고침 후 저장된 세션이 복원되면 `authed` 상태가 유지된다.
5. `getAccessToken()`은 유효 세션에서 비어 있지 않은 access token을 반환한다.
6. sign-out 또는 사용자 변경 후 이전 계정의 우주 데이터가 남지 않는다.
7. Supabase env가 비어 있으면 무한 로딩이 아니라 부팅 오류로 드러난다.
8. 우주 셸·일기·마이페이지(`/`·`/diary`·`/my-page`)에서는 `SessionGate`가 독립 우상단 로그아웃 핀(데모의 "체험 종료" 포함)을 띄우지 않고, 로그아웃이 우주 셸 햄버거 사이드바로 수렴한다. `/admin`·`/gift`는 기본 로그아웃 핀을 유지한다.
9. `/my-page`(인증·멤버 게이트)는 세션 이메일과 로그아웃 버튼을 보여주고, 로그아웃은 셸과 같은 `signOut` 경로로 세션 + 우주/일기 캐시 출처 경계를 리셋한다.
