# 04. memory-api - SegmentMemory, RecordMemory, GetUniverse (as-built)

> 현재 구현 기준 문서. 초기 "일기 1개 -> 별 1개" 흐름은 이후 조각 fan-out 구현으로 대체되었다. | 범위: BE MemoryService core | 상태: 구현 완료

## 목적

Memory API는 원본 일기를 불변 `records`에 저장하고, 그 일기를 조각 별 `memories`로 fan-out해 우주 그래프에 나타나게 한다. 저장은 unary RPC이며, 별/시냅스 도착은 동기 fan-out 또는 비동기 worker 경로를 통해 이루어진다. `GetUniverse`는 클라이언트가 렌더할 전체 개인 그래프와 부가 시각 입력을 반환한다.

## 현재 RPC 표면

이 문서는 `MemoryService` 안에서 04의 core loop와 직접 관련된 RPC를 중심으로 한다. 같은 서비스에는 후속 기능 RPC도 함께 구현되어 있다.

- `SegmentMemory`: 저장 없는 preview. body를 AI extractor로 조각 후보로 나눈다.
- `RecordMemory`: record 저장과 조각 별 fan-out 시작.
- `GetUniverse`: 모든 별/시냅스와 ambient/relevance 등 렌더 입력 조회.
- 함께 구현된 후속 RPC:
  - `RecallMemory`
  - `ReinforceLinks`
  - `ListDormant`
  - `GetEvolutionHistory`
  - `ListRecords`

## 도메인 모델

- `RecordInput`
  - `UserID`, `Body`, `EntryDate`, whole-diary hints인 `Mood/Intensity/Valence`, `IdempotencyKey`, user-confirmed `Segments`.
- `SegmentInput`
  - 조각 별 하나의 `Text`, `Mood`, `Intensity`, `Valence`.
- `Memory`
  - 별 렌더 입력. `ID`, fragment mood/intensity/valence, `LastRecalledAt`, `RecordID`, `FragmentIndex`, reshaping state, `Relevance`, `Resonant`.
  - body/entry_date는 별 렌더 입력에 직접 들어가지 않는다.
- `Universe`
  - `Memories`, `Synapses`, `Ambient`.
- 도메인 타입에는 db/json/proto 태그를 두지 않는다.

현재 mood 값은 13개다. 빈 값은 unspecified로 취급한다.

## `SegmentMemory`

- 인증 필요.
- `body`만 입력받고 DB에 아무 것도 쓰지 않는다.
- service가 body 공백/길이 제한을 검증한다. 현재 body cap은 4000 runes다.
- handler는 25초 timeout을 둔다. response stream deadline보다 긴 LLM 호출로 비용을 태우지 않기 위해서다.
- 성공 시 `SegmentDraft[]`를 반환한다.
- 실패해도 저장된 행은 없으므로 재시도 안전하다.
- extractor가 keyless/mock으로 동작하면 whole-body fallback segment를 반환할 수 있다.

## `RecordMemory`

### Request / Response

- request:
  - `body`
  - `entry_date` (`YYYY-MM-DD`, 비면 서버가 UTC today로 기본값)
  - optional whole-diary hints: `mood`, `intensity`, `valence`
  - `idempotency_key`
  - `segments[]`: 사용자가 preview를 검토/수정해 확정한 조각 목록
- response:
  - `record_id`: 불변 원본 id
  - `memory_ids[]`: 조각 별 id. sync fan-out이면 채워지고, async extract path이면 보통 비어 있다.

### 저장 경로

- 공통:
  - 인증 context에서 `userID`를 읽는다.
  - body, intensity, valence, segment count/text/range를 트랜잭션 전에 검증한다.
  - `idempotency_key`가 있고 기존 `(user_id, key)` record가 있으면 새 행을 쓰지 않는다.
  - idempotent hit에서는 기존 `record_id`와 이미 생성된 fragment memory ids를 반환한다.
- `segments[]`가 비어 있지 않은 경우:
  - 하나의 트랜잭션에서 `InsertRecord` 후 `fragment.FanOutTx`를 실행한다.
  - `FanOutTx`는 N개의 `memories`, N개의 embed job, 같은 일기 조각 사이의 intra-entry links를 만든다.
  - response의 `memory_ids[]`는 최종 조각 별 id 목록이다.
- `segments[]`가 비어 있는 경우:
  - 하나의 트랜잭션에서 `InsertRecord` 후 `EnqueueExtractJob`을 실행한다.
  - 별은 아직 없고, worker의 extract 단계가 이후 조각 별을 만든다.
  - response의 `memory_ids[]`는 비어 있을 수 있다. 클라이언트는 "조각내는 중" 상태 후 `GetUniverse` refetch로 별 도착을 확인한다.

### 불변성과 힌트

- `records.body`는 원본 전체 일기이며 수정하지 않는다.
- `records.mood/intensity/valence`는 전체 일기 단위의 fallback hint다.
- 조각 별의 실제 감정/강도/정동가는 `memories.mood/intensity/valence`에 저장된다.
- `visual_spec`은 이 경로에서 쓰지 않는다. 별 형태는 클라이언트가 id 기반 seed와 렌더 입력으로 결정한다.

## `GetUniverse`

- 인증 필요.
- 현재 사용자의 모든 별과 모든 시냅스를 반환한다. dormant 별/약한 선도 필터링하지 않는다.
- 별 응답:
  - `memory_id`
  - fragment mood/intensity/valence
  - `last_recalled_at`
  - reshaping state: `brightness_offset`, `hue_shift`, `form_seed_delta`, `version`
  - wayfinding: `record_id`, `fragment_index`
  - `resonant`
  - `relevance`
- 시냅스 응답:
  - `a_id`, `b_id`, `weight`, `link_type`, `last_activated_at`, `co_activation_count`
- ambient 응답:
  - 최근 fragment emotions를 시간 가중으로 집계한 `hue/sat/arousal/valence`.
- service는 ambient/relevance 읽기가 실패해도 core graph는 실패시키지 않고 neutral fallback으로 degrade한다.
- 서버는 좌표, 최종 밝기, 선 두께를 계산하지 않는다. 이 값은 클라이언트 렌더 모델의 책임이다.

## Error Mapping

- 인증 context가 없으면 `Unauthenticated`.
- entry date parse 실패나 validation sentinel은 `InvalidArgument`.
- 일반 저장/조회 실패는 `Internal`.
- `SegmentMemory` extractor 실패는 저장 없는 retryable preview 실패로 `Unavailable`.

## Flutter 동등성 기준

- Flutter 작성 flow는 `SegmentMemory -> review/edit -> RecordMemory(segments)`를 1급 경로로 구현해야 한다.
- legacy/backup으로 `RecordMemory`에 segments 없이 저장할 수 있지만, 이때 별은 즉시 오지 않고 refetch로 도착한다.
- 저장 응답의 `record_id`와 `memory_ids[]`를 구분해야 한다. `record_id`는 원본 일기, `memory_id`는 조각 별이다.
- `GetUniverse`의 별은 좌표 없는 렌더 입력이며, Flutter가 자체 force-sim/레이아웃을 수행해야 한다.

## 수용 기준

1. `SegmentMemory`는 저장 없이 조각 후보를 반환한다.
2. `RecordMemory`는 유효 입력을 하나의 트랜잭션으로 기록하고 partial rows를 남기지 않는다.
3. user-confirmed segments가 있으면 response에 조각 `memory_ids[]`가 채워진다.
4. segments가 없으면 extract job이 적재되고 response의 `memory_ids[]`는 비어 있을 수 있다.
5. 같은 idempotency key 재호출은 새 record를 만들지 않는다.
6. `GetUniverse`는 dormant 포함 전체 별/시냅스를 user scope로 반환한다.
7. `records`에 대한 수정/삭제 API나 쿼리가 없다.
8. 도메인 타입은 transport/db 태그 없이 순수하다.
