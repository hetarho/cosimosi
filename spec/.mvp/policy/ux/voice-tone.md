# 보이스·톤 — 카피와 시각의 절제 (voice-tone)

> cosimosi가 화면에서 **어떻게 말하고 어떻게 비치는가**의 정전(SSOT). 카피·라벨·마이크로카피·UI 장식이 지켜야 하는 규칙이다. 모든 문구·버튼은 여기 맞춘다 — 어기는 변경은 리뷰에서 막는다.

## 정의

cosimosi는 "기억의 우주"라는 정서적 제품이다. 목소리는 **문학적·절제·구체적**이고, 화면은 **우주에 집중하도록 비운다.** 텍스트는 *사람이 직접 쓴 한 문장*처럼 읽혀야 하며, 자동 생성물 특유의 티(번역투·명사화·사무체·이모지 남발)를 내지 않는다. 절제가 기본값이다 — 더 넣는 쪽이 아니라 덜어내는 쪽으로 판단한다.

## 규칙

| # | 규칙 | 무엇을 / 왜 |
|---|---|---|
| V1 | **카피는 문학적·구체·간결.** | 번역투("~을 위한", "~에 대한", 수동태 남발)·추상 명사화("진행/수행/처리")·사무체를 피한다. 짧고 담백하게, 사람이 쓴 문장처럼. 느낌표·"~해보세요" 남발 금지. |
| V2 | **장식용 이모지 금지.** | 라벨·버튼·상태·안내 문구에 장식 이모지(✦ 📖 🌙 🤝 🎁 🔭 🚀 ✨ 등)를 넣지 않는다 — 의미는 단어로 전한다. 이모지 반복은 가장 흔한 AI 티다. 아이콘이 꼭 필요하면 일관된 아이콘 세트(`lucide-react`)를 절제해서 쓰고(예: 메뉴 런처), 텍스트 라벨에 이모지를 덧붙이지 않는다. |
| V3 | **화면을 비운다 — 상시 노출 최소화.** | 우주 몰입이 우선. 상시 버튼은 꼭 필요한 것만 두고(현재 `/universe`는 카메라 시점 토글 + "메뉴" 둘), 나머지 기능은 단일 메뉴 뒤로 접는다. 결과·기능은 코너에 박힌 칩이 아니라 비차단 표면(`Surface`)으로 띄운다. |
| V4 | **마이크로카피도 같은 보이스.** | 빈 상태·로딩·에러·확인 문구까지 한 목소리. 담백하고 구체적으로("우주를 불러오지 못했어요." / "아직 별이 없어요."). 기능을 설명하지 말고 그 순간을 말한다. |
| V5 | **정서는 비유로, 과장 없이.** | 별·빛·우주의 은유는 제품의 결이지만 장식이 아니라 의미일 때만. 미사여구·감탄을 쌓지 않는다(concept.md의 시적 톤을 따르되 UI에선 더 절제). |

## 불변식

- 새 UI 텍스트·버튼·상태 문구는 **V1·V2를 항상 지킨다.** 장식 이모지가 라벨에 들어오면 회귀로 본다.
- `/universe` 메인은 **상시 버튼 2개**(카메라 시점 토글 + 메뉴)를 넘기지 않는다 — 새 기능은 버튼을 늘리지 말고 메뉴/표면에 얹는다(V3).
- 직접 쓴 카피는 **적대적 리뷰로 재검증**한다(AI 티가 남았는지 스스로 의심하고 고친다).
