# 우주 공개 (sharing) 도메인 정책 (policy/domain/sharing)

> 현재 구현된 우주 공개 표면(공개 URL `/u/:slug` + `ShareService`/`VisitService`)의 공개 단위·
> 표면 격리·슬러그·프라이버시 규칙의 사실 정의. (spec 35 — `backend/internal/share`,
> `proto/cosimosi/v1/share.proto`, `backend/internal/platform/rpcserver/server.go`,
> `frontend/src/pages/visit`, `frontend/src/features/share-universe`)
> 함께한 기억 — 공명(별 1개 1:1 보내기·수락=재작성·공명 연결, spec 36)도 여기에 담는다
> (`backend/internal/gift`, `proto/cosimosi/v1/gift.proto`, `frontend/src/features/send-star`,
> `frontend/src/pages/gift`).

## 공개 단위 — 풍경만, 내용 제로

- 공개되는 것은 **우주의 풍경뿐**: 별의 감정(색)·강도(크기)·날짜, 시냅스(인덱스 쌍+weight),
  소유자의 시각 설정(배경 테마·오브제·감정색). **일기 본문·조각 텍스트는 어떤 경로로도 나가지
  않는다.**
- 전용 DTO로 **타입 수준 보장**: `SharedStar`/`SharedSynapse`엔 `body`·`fragment_text`·
  `record_id`·`memory_id`·정확 타임스탬프 **필드가 아예 없다**(누락이 아니라 부재). `GetUniverse`의
  `Star`(필드가 계속 자람)를 재사용하지 않는다 — 새 필드가 공개로 새는 것을 구조적으로 차단.
- **별 식별자 없음:** 응답 배열의 인덱스가 곧 별 id다. 시냅스는 그 인덱스 쌍으로만 잇는다.
- **타임스탬프는 일 단위 양자화**(`last_recalled_day`·`created_day` = epoch days). 정확 시각을
  주지 않아 행동 핑거프린팅(언제 일기를 쓰고 회상하는지)을 무디게 한다.
- **좌표는 보내지 않는다**(헌법3). 방문자 브라우저의 force-sim이 재창발하므로 방문마다 배치가
  미세하게 다른 건 의도된 자연스러움이다.
- **요즘 하늘색(ambient, spec 25)은 서버가 계산하지 않는다** — 클라가 공개된 별들에서 파생한다
  (`deriveAmbient`, 데모/폴백과 같은 경로). proto `Ambient` 필드는 향후 서버 계산용으로 비어 둔다.

## 공개 표면 격리 — 무인증은 `VisitService` 하나뿐 (fail-closed)

- 무인증으로 도달 가능한 RPC는 **`VisitService.GetSharedUniverse` 단 하나**다. 별도 service로
  분리해 **그 핸들러에만 auth 없는 인터셉터 체인**(logging→recover, 256KB read cap 유지)을
  장착한다. 나머지 모든 service(`Memory`·`Settings`·`Admin`·`Share`)는 기존 auth 체인을 그대로
  유지한다 — allowlist 구멍이 아니라 service 분리라, 실수로 다른 RPC가 공개로 새는 구조가 없다.
- `ShareService`(소유자 설정)는 **인증 체인 그대로**다. `user_id = JWT sub`로 자기 행만 읽고
  쓴다(교차 사용자 접근 경로 없음).
- 17의 요청 한도(256KB read cap·서버 타임아웃)는 무인증 표면에도 동일 적용된다.

## 슬러그 규칙

- **추측 불가가 1차 방어선:** `crypto/rand` 16바이트 → base64url 22자(128bit).
- **최초 켜기에 생성**된다. 끄고 다시 켜면 같은 슬러그를 재사용한다(회전이 아닌 한).
- **회전(rotate):** 새 슬러그를 발급하고 **이전 URL은 즉시 무효**가 된다.
- **균일 NotFound:** 존재하지 않는 슬러그·꺼진 공유(`enabled=false`)·빈 우주를 **구분 없이 동일한
  NotFound**로 응답한다(존재 비노출). 슬러그→소유자 조회는 `enabled=true`일 때만 행을 돌려준다.

## 방문자 — 순수 읽기

- 방문 페이지(`/u/:slug`)는 **SessionGate 밖**의 공개 라우트. 우주 캔버스를 읽기 전용으로
  재사용한다 — 기록 폼·회상 패널·시뮬 패널 없음, 별을 눌러도 **어떤 일기·조각 텍스트도 뜨지
  않는다**(바라보기·fly-to만).
- 방문은 **쓰기 RPC를 한 건도 호출하지 않는다**(회상 강화·기록은 소유자 전용). 공개 전송은
  인증 인터셉터가 없는 **전용 무인증 transport**라 토큰을 절대 싣지 않는다.
- 방문 페이지는 소유자의 시각 설정·풍경을 적용하되, **방문자 자신의 시각 설정은 보존**한다
  (진입 시 스냅샷→떠날 때 복원; 소유자 풍경이 방문자 기기에 눌러앉지 않게).
- 비로그인 방문자에겐 하단에 "나의 우주 만들기" CTA를 띄운다(유입 경로).

## 함께한 기억 — 공명 (별 1개 보내기·수락=재작성, spec 36)

- **보내기 = 그 조각 1개의 1:1 opt-in 공개.** 내 별(조각) 상세에서 "이 별 보내기" → 토큰 링크
  (`/gift/:token`)를 발급해 사용자가 직접 전달한다. **발급 전 명시 고지 필수**("받는 사람이 이 조각
  글을 읽게 돼요"). 보내는 건 **그 조각 하나의 텍스트**(`memories.fragment_text`)뿐 — 보낸이의 다른
  조각·다른 일기·원본 일기 전문은 어떤 경로로도 나가지 않는다. (구 단일조각 데이터로 `fragment_text`가
  NULL이면 record가 조각 1개일 때만 본문=그 조각으로 폴백하고, 다조각 record에선 절대 본문을 안 보낸다.)
- **남의 별은 못 보낸다:** 보내기는 보낸이 소유 별에 한한다(소유 가드).
- **토큰:** `crypto/rand` 16바이트 → base64url 22자(128bit, 35 슬러그와 동일 엔트로피). **생성 +30일
  만료**(미수락 링크가 영원히 떠돌지 않게). 만료는 지연 판정(스위퍼 없음 — 읽을 때 effective 상태로 환산).
- **수락 = 재작성.** 수신자가 사인인 후 *자기 관점으로* 그 사건을 다시 쓴다(13감정·강도). 한 트랜잭션으로
  수신자의 **불변 record + 단일 조각 별**(extract 생략·`fragment_index=0`·21 fan-out 코어 재사용) + **공명
  쌍**(`resonances`: memory↔memory) + embed job을 원자 생성한다(부분 실패 전부 롤백). 거절은 상태만, 보낸
  쪽 취소는 유효-pending에 한함(만료된 건 취소 대상 아님).
- **상태 머신:** `pending → accepted | declined`(수신자) | `canceled`(보낸이) | `expired`(지연). 자기 자신이
  보낸 토큰엔 응답(수락/거절) 불가(`InvalidArgument`). 중복 수락·만료·취소·이미 응답됨은 상태 검사로 **거부**
  (멱등 거절 — 재수락이 새 별·공명을 또 만들지 않는다). 미수락(pending) 링크 열기만 조각 텍스트·감정·보낸이
  표시명·메시지를 보여주고, **종료/만료 상태는 status만**(내용·메시지·보낸이 표시명 일절 비노출 — 취소·만료된
  링크가 관계/메시지 메타데이터로 남지 않게).
- **공명은 링크일 뿐 — 상태 비전파.** 공명으로 이어진 두 별은 이후 **각자의 우주에서 따로 회상되고 따로
  변한다**(각자의 변천사). 한쪽 별을 회상·재성형해도 상대 우주의 별엔 아무 변경도 전파되지 않는다.
- **공명은 삭제하지 않는다**(헌법2의 정신 — 별이 잠들어도 공명은 남는다).
- **보낸이/상대 표시명은 35의 `universe_shares.display_name` 재사용**(빈 값 = "어느 우주"). 상대가 35로
  우주를 공개 중일 때만 별 상세에 방문 링크(`/u/:slug`)를 보인다.
- **공개 우주(35)엔 공명을 절대 싣지 않는다.** `SharedStar`엔 `resonant` 필드가 없고, 무인증 `VisitService`
  경로는 `resonant`를 false로 고정한다. `GetResonanceInfo`는 인증 `GiftService`에만 있다 — 제3자에게 관계
  비노출(공명의 시각화는 각자 자기 우주 + 37 겹침 뷰에서만).

## 두 우주 겹쳐보기 — 공명 겹침 (spec 37)

공명(36)의 보상 장면 — 친구의 공개 우주(35) 위에 내 우주를 한 씬에 이격 배치하고, 공명으로 이어진 별
쌍 사이에 두 우주를 가로지르는 **빛의 다리**를 놓는다. 서버가 합성한 우주 상태가 아니라 **순간의 읽기
뷰**다(`pages/visit`·데모 home의 겹침 토글, `widgets/universe-canvas/ui/overlay/*`).

- **로그인 전용 진입.** 친구의 `/u/:slug`(방문)와 데모 home에서만 겹쳐보기 토글이 뜬다 — 비로그인
  방문자에겐 노출하지 않는다(35 화면 그대로).
- **두 우주 = 두 force-sim.** 한 씬에 세로로 이격 배치하되(`OVERLAY_OFFSET`) 각 우주 좌표는 각자의 클라
  force-sim에서 창발한다(헌법3 — 서버 좌표 없음). 친구 별무리는 틴트·atmosphere로 "남의 하늘"로 구분된다.
- **다리는 공명 쌍만 잇는다.** `ShareService.GetResonanceBridges(slug)`(**인증**·`NO_SIDE_EFFECTS`)가
  호출자↔주인 공명 쌍을 `{my_memory_id, their_star_index}`로 반환한다. 친구 우주 데이터 자체는 35의 무인증
  `VisitService.GetSharedUniverse`를 그대로 쓴다(겹침이 공개 표면을 넓히지 않는다).
- **상대 memory id는 서버를 떠나지 않는다.** 응답은 내 memory id + 공개 스냅샷 인덱스(`their_star_index`)만
  싣고, `GetResonanceBridges`와 `GetSharedUniverse`가 같은 `ORDER BY m.id`를 공유해 인덱스가 정합한다(범위 밖
  인덱스는 끝점 없는 다리로 버린다).
- **비당사자·공유 off 비노출.** 호출자가 공명 당사자가 아니면 빈 배열(다리·공명 존재를 제3자에게 안 알림).
  주인이 공유를 끄거나 슬러그를 회전하면 겹침도 균일 NotFound로 막힌다(35와 동일).
- **순수 읽기 — 쓰기 0.** 겹침 뷰가 떠 있는 동안 회상 강화를 포함해 **어떤 쓰기 RPC도 호출하지 않는다**.
  navigation `overlay` 상태가 쓰기 게이트(`shared/lib/write-gate.ts`)의 단일 출처다.
- **비대칭 비교 패널.** 다리/별 선택 시 내 별은 감정·강도 + 조각 텍스트를, 친구 별은 **감정·강도 시각
  정보만**(텍스트 절대 없음 — 콘텐츠 제로, 35) 보여준다.
- **공명은 비전파(36 재확인).** 겹쳐 봐도 두 별은 각자의 우주에서 따로 회상·재성형된다 — 한쪽 변경이 상대
  우주로 전파되지 않는다.

## 비목표 (현재 미구현)

- 별 단위 공개 수위(제목만/일부) — 35는 전역 한 단계(모든 별 내용 비공개·풍경 전체 공개).
- 친구 계정 그래프·검색·팔로우·피드 — 없음(공유는 링크 전달, 공명은 1:1 토큰으로만).
- 알림(푸시·이메일) — 없음. 수락 사실은 보낸 목록의 상태로 확인.
- 그룹 전송(1:N)·재전송 체인 — 1:1 단건만(같은 별을 여럿에게 보내는 건 gift 여러 건으로 자연 지원).
