# 52. 형태×표면 스킨 — form/surface 2축 (form-skins)

> 별·나·시냅스의 스킨을 **형태(form, =shape · 지오메트리/실루엣)** 와 **표면(surface · 셰이딩/질감)** 두 개의 **독립 선택 축**으로 나눈다. [plan/50 셰이더 아트 툴킷](50.shader-art-toolkit.md)의 순수(도메인 무관) 기법을 **entity가 조립**해 스킨을 만들고, **feature/widget/page가 소비**하는 3층 구조를 형태 축에 완성한다. 형태·표면은 **둘 다 따로 판매**한다. | 범위: FE(+BE 검증 로직)

> **3층 아키텍처(이 plan의 핵심 의도).** ① **shared** = 도메인 무관 순수 기법(plan 50 toolkit: geometry 생성기 + surface 셰이더). ② **entity** = 그 기법을 조합해 컨셉별 스킨(form·surface 카탈로그·registry). ③ **feature/widget/page** = 스킨을 소비(선택 store·N-제네릭 렌더·스위처 UI). 선행: plan 50. plan 51(배경)과 같은 패턴의 형태 축 버전. 도메인 토대: [plan/44 커스터마이즈](44.customization-system.md).

## 목적
<!-- 왜 이 기능인가. 사용자·제품 관점. -->

별·나·시냅스는 형태(지오메트리)와 표면(셰이딩)이 **한 id에 묶여** 있어(예: `ember` = 8면체 + 용암), 조합 자유가 없고 새 스킨이 비싸다. 이를 **형태 × 표면** 두 축으로 분리하면:

- **적은 부품으로 조합 폭발** — 형태 5종 × 표면 5종 = 25 룩(8면체+용암, 변위구+facet…). 이게 plan 50 "toolkit 조립" 철학의 정수다.
- **사용자가 형태와 표면을 따로 고른다** — "이 모양에 저 질감"을 직접 조합.
- **3층이 또렷해진다** — shared(순수 기법)·entity(조립)·feature/widget/page(소비)가 형태 축에서도 분리되어, 새 형태/표면 추가가 카탈로그 항목 + 조립 함수로 끝나고 위젯·소비처는 무수정.

## 범위 / 비목표
<!-- 무엇을 포함하고, 무엇은 명시적으로 안 하나. -->

**범위**
- **2축 분리:** 별·나·시냅스 각각 **form 축 + surface 축**. 사용자가 형태와 표면을 독립 선택.
- **shared 순수 기법(plan 50):** form = geometry/SDF 생성기(저폴리·8면체·매끈·변위구 등), surface = 셰이딩 레시피(facet·glossy·lava·cloud·pulse·mirror·prism·neuron·flow 등). 머티리얼/uniform/store/도메인 미의존.
- **entity 조립:** 각 entity `ui`가 form×surface를 `buildX(form, surface)`로 조립. `StarForm`·`StarSurface`(+self·synapse) 카탈로그 const 파생 + 형태별 주석 `params`. 위젯은 N-제네릭(registry, per-form/surface 분기 없음).
- **합성 선택 — proto/DB 마이그레이션 회피:** 선택은 *기존 wire 필드*(`star_object`·`self_object`·`synapse_style`)에 **합성 id `"<form>+<surface>"`** 로 직렬화. 레거시 단일 id는 (form, surface) **프리셋**으로 정규화 폴백.
- **경제 — 둘 다 따로 판매:** form·surface 각각 축당 무료 1 + 유료 N, 가격·무료는 `spec/values.yaml` 단일 출처. 아이템 id `"<axis>:form:<id>"`·`"<axis>:surface:<id>"`. 합성 선택의 소유 = **양쪽 sub-item** 소유(또는 무료); BE가 합성을 디코드해 검증.
- **스위처 UI:** 객체 축마다 form·surface 2 피커(잠금·가격·소유 기존 모델 재사용).
- **보존:** crystal(저폴리 결정)·liquid·ember는 (form×surface) **프리셋으로 시각 보존**.

**비목표**
- **배경은 plan 51**(이펙트 축, geometry 없음). 이 plan은 형태 있는 3축만.
- **proto/DB 스키마/마이그레이션 안 한다** — 합성 wire id로 회피(`user_owned_items`는 임의 문자열이라 sub-item 소유도 스키마 무변경).
- **색·mood 규칙 변경 안 한다(plan 44):** 별=mood(13감정)·시냅스=양끝 mood 블렌드·나=ambient 파생(던지는 빛 중립). form·surface만.
- **구체 신규 form/surface의 최종 미감을 plan이 못 박지 않는다** — 육안 튜닝은 job + 카탈로그 `params`.
- **헌법8 변경 없음:** 별 InstancedMesh·시냅스 Line2·수천 노드·시냅스 전역 스칼라 한계 유지.

## 설계 요점
<!-- 어떻게 구현하나. -->

### A. 3층 + form/surface 모델
- **shared(plan 50)**: 순수 기법. form 패밀리(geometry/SDF 생성기), surface 패밀리(셰이딩 헬퍼). 노드/geometry in-out, 부수효과·도메인 없음.
- **entity**: `buildStarBody(form, surface)`·`buildSelfForm(.)`·synapse 조립 — shared 기법을 조합. `FORMS`·`SURFACES` registry(`satisfies Record<…>` 총괄성). 형태별 `params`(주석)는 entity 카탈로그.
- **feature/widget/page**: appearance store가 합성 선택 운반, 위젯이 N-제네릭 렌더, 스위처가 2 피커. 새 form/surface 추가 시 이 층은 무수정.

### B. id·인코딩 스킴
- 선택 모델: 축별 `{ form, surface }`. wire 직렬화 `${form}+${surface}`; 디코드는 `+` 분리(레거시: `+` 없으면 단일 id → 프리셋 디컴포지션 테이블).
- 아이템 id: `"<axis>:form:<id>"`·`"<axis>:surface:<id>"`. 소유권은 sub-item 단위(`user_owned_items` 스키마 무변경). 합성 소유 = 두 sub-item 모두 `isOwned`.
- values 무료/가격: `customization.free`에 슬롯 키(`"star:form"`·`"star:surface"`…), `price` 키 = sub-item id. FE 생성 상수 + BE가 같은 출처에서 파생.

### C. 디컴포지션 (보존 프리셋 + 조각)
| 축 | 레거시 프리셋 | = form | × surface |
|---|---|---|---|
| 별 | crystal(deepfield·무료) | `lowpoly` | `facet`(면 음영+엣지+반사) |
| 별 | liquid | `liquid`(변위 구) | `glossy`(림+스페큘러) |
| 별 | ember | `octa` | `lava`(용암 fbm+flicker+반사) |
| 별 | (aurora→조각) | `cloudy` | `cloud`(도메인워프 발광) |
| 별 | (pulsar→조각) | `smooth` | `pulse`(코어+맥동) |
| 나 | mirrorball(무료) | `orb` | `mirror`(케이지+글린트) |
| 나 | prism-cube | `cube` | `prism`(프레임+색분산) |
| 나 | neuron-bloom | `bloom`(변위 구) | `neuron`(돌기 셸+핵광) |
| 시냅스 | filament(무료) | `strands`(꼬인 다발) | `flow`(빛 패킷 흐름) |
| 시냅스 | dendrite | `branched`(가지 다발) | `flow` |
| 시냅스 | particle | `dotted`(가는 한 줄) | `beads`(점점이 비드) |

무료 프리셋: 별 lowpoly+facet · 나 orb+mirror · 시냅스 strands+flow. crystal·liquid·ember·mirrorball·prism-cube·
neuron-bloom·filament·particle·dendrite는 위 조합으로 시각 보존, aurora/pulsar 프리셋(별)은 폐기하되 조각
(cloudy·smooth form, cloud·pulse surface)은 재조합 가능. 시냅스 추가 surface `steady`(흐름 없는 잔잔 발광)는
조합용 신규. (정확 분해는 `entities/{star,synapse}/model/forms.ts`·`entities/appearance/model/self-forms.ts`가 출처.)

### D. 색·불변식 보존 (plan 44)
- 별=mood(13감정 회전)·시냅스=양끝 mood 블렌드·나=ambient 파생(던지는 self-light 중립). form/surface는 형태/질감만 바꾼다.
- 헌법8(InstancedMesh·Line2), 시냅스 불변식(per-edge 두께 변조 불가·삭제금지 바닥 불투명·weight→밝기/alpha/펄스) 유지.

## 수용 기준 (인수 조건)
<!-- 평서문. -->
1. 별·나·시냅스 각각 **form·surface 2개 선택 축**이 있고, 같은 surface를 다른 form에·같은 form을 다른 surface에 적용할 수 있다.
2. shared(plan 50)에 form=geometry 생성기·surface=셰이딩 레시피가 순수 함수로 있고, entity가 `buildX(form, surface)`로 조립한다(축마다 bespoke 중복 재구현이 아니다).
3. 위젯·소비처는 form/surface 수와 무관하게 동작한다(registry, per-form/surface 분기 없음). 새 form/surface 추가가 feature/widget/page 코드 수정 없이 된다.
4. 선택은 기존 wire 필드에 합성 id `"<form>+<surface>"`로 직렬화되어 **proto/DB 스키마 변경이 없다**. 레거시 단일 id는 (form, surface) 프리셋으로 폴백한다.
5. **둘 다 따로 판매:** form·surface 각각 무료 1 + 유료 N, 가격·무료는 `spec/values.yaml` 단일 출처. 합성 선택은 양쪽 sub-item을 소유(또는 무료)해야 서버 저장되고, BE가 합성을 디코드해 검증한다.
6. crystal(저폴리 결정)·liquid·ember가 form×surface 조합으로 **시각 보존**된다(프리셋).
7. 색 규칙 보존(별=mood·시냅스=양끝·나=ambient), 헌법8·시냅스 불변식 유지. form/surface만 바뀐다.
8. 스위처가 객체 축마다 form·surface 2 피커를 보인다.
9. 미지/레거시 합성·sub-id는 축 기본 form/surface로 폴백(크래시 없음). FE `build`·`lint`·`steiger`·test + BE `build`·test 통과, 공유 우주(35/37)·근접 탐험(49) 회귀 없음.
