# 30. 개인 설정 - 서버 per-user 시각 설정 (as-built)

> 현재 구현 기준 문서. 우주 배경 테마·별 오브제 형태·감정색을 계정 종속 per-user 오버라이드로 서버에 저장하고, 클라가 자기 기본값 위에 머지하기 위한 SSOT다. | 범위: FS | 상태: 구현 완료, 반영: 29 감정 13종, 35 공유 우주 appearance, 38 자아 별

## 목적

cosimosi의 시각 언어("감정의 색", "우주의 분위기")는 사용자의 것이고, 계정을 따라 모든 기기에 와야 한다. 두 가지를 서버화한다: (1) 배경 테마·오브제 형태는 원래 `entities/appearance` zustand store의 localStorage에만 살아 기기-로컬이었다. (2) 감정색(`MOOD_PALETTE`)은 코드 고정이라 바꿀 수 없었다. 세 축(우주 배경 테마·별 오브제·감정색)을 서버 per-user 오버라이드로 통일하되, 기본값은 클라가 소유한다 — 미인증·오프라인·체험 경로에서도 렌더가 깨지면 안 되므로(헌법 견고성). 서버는 사용자가 바꾼 오버라이드만 저장하고, 감정색은 정규화 행으로 두어 미래의 `GROUP BY (mood, color)` 집계("xx%가 골랐어요")가 가능하게 자리만 만든다. 온보딩 색 피커와 추천/집계 UI는 이 데이터 모델 위에 미래가 붙인다.

## 현재 구현

- 시각 설정은 별 그래프(`MemoryService`)와 분리된 별도 바운디드 컨텍스트다.
  - BE: `backend/internal/settings/` (memory 패턴 미러: settings/service/handler/repository_pg).
  - proto: `SettingsService`는 `proto/cosimosi/v1/memory.proto`에 함께 선언되지만 Connect 서비스로는 독립 등록된다(별도 path).
  - FE: `entities/appearance` store(서버 백킹) + `features/switch-appearance` 스위처.
- 서버는 오버라이드만 저장한다. 기본값(테마·오브제·`MOOD_PALETTE`)은 BE에 상수로 두지 않는다 — 팔레트가 BE/FE 이중 출처로 갈라져 드리프트하는 것을 막기 위해 클라가 단일 출처로 소유한다.
- per-user 행은 사용자가 바꿀 때만 생긴다. 행을 미리 시드하지 않는다(`users`는 Supabase auth 소관). 온보딩 피커가 아직 없으므로 현재 DB는 비어 있고 전원 기본값으로 렌더한다.

## 세 축

- **Theme** `'vast' | 'lively' | 'calm'` — 우주의 배경색·분위기. `entities/appearance/model/types.ts`의 `Theme`. 카탈로그는 `themes.ts`의 `THEMES`(노출 순서이자 단일 출처), 기본값 `DEFAULT_THEME = 'vast'`.
  - 각 테마는 `bg`(THREE 캔버스 깊은 배경색), `accent`(mood hex), `swatch`(스위처 미리보기)를 번들한다. bg 값은 `vast #070b1e` · `lively #120617` · `calm #04140f`(SSOT는 `policy/domain/universe.md`).
- **StarObject** `'deepfield' | 'aurora' | 'liquid' | 'ember'` — 별의 생성형 형태(form). `entities/star/model/types.ts`의 `StarObject`. 카탈로그는 `star/model/kinds.ts`의 `STAR_OBJECTS`(4종), 기본값 `DEFAULT_OBJECT = 'deepfield'`. 표시 이름은 Crystal/Nebula/Liquid/Ember.
- **emotionColors** — mood(소문자) → `#RRGGBB` per-user 오버라이드. 29의 13 mood 키 중 사용자가 바꾼 것만. 없는 mood는 `MOOD_PALETTE` 기본 폴백.
- **SelfObject** `'nebula-heart' | 'core' | 'well'`(spec 38, 자아 별 형태, 3종)는 같은 store(`entities/appearance/model/self-objects.ts`의 `SELF_OBJECTS`)에 있지만 **서버 동기 대상이 아니다 — 기기-로컬 선호로 localStorage에만 산다**(proto/BE에 `self_object` 필드 없음).

## 현재 RPC 표면

`SettingsService`(`memory.proto`에 선언, 독립 Connect 서비스로 등록). 둘 다 인증 필수(02 auth 인터셉터 — `MemoryService`와 같은 인터셉터 스택).

- `GetSettings` (`NO_SIDE_EFFECTS`, HTTP GET 노출): 호출자의 저장된 오버라이드만 반환. 행이 없으면 빈 `Settings`(theme=""·star_object=""·emotion_colors 없음). 클라가 기본값 위에 머지한다.
- `UpdateSettings`: 부분 갱신. 보낸 필드만 upsert하고 나머지는 보존. `emotion_colors`를 보내면 그 subset만 upsert하며 다른 mood를 지우지 않는다. 응답은 갱신 후 저장된 오버라이드(merged stored overrides).

### proto 메시지 (`proto/cosimosi/v1/memory.proto`)

- `EmotionColor { Mood mood = 1; string color = 2; }` — color는 `"#RRGGBB"`.
- `Settings { string theme = 1; string star_object = 2; repeated EmotionColor emotion_colors = 3; }` — 저장된 오버라이드(완전한 config 아님). `theme=""` = 클라 기본(vast), `star_object=""` = 클라 기본(deepfield).
- `GetSettingsRequest {}` / `GetSettingsResponse { Settings settings = 1; }`.
- `UpdateSettingsRequest { optional string theme = 1; optional string star_object = 2; repeated EmotionColor emotion_colors = 3; }` — `optional`이라 미전송 필드는 보존된다.
- `UpdateSettingsResponse { Settings settings = 1; }`.
- `Mood` enum(13종 + UNSPECIFIED): `JOY=1 CALM=2 SAD=3 ANGER=4 FEAR=5 LOVE=6 NEUTRAL=7 EXCITEMENT=8 GRATITUDE=9 RELIEF=10 STRESS=11 TIRED=12 EMPTINESS=13`(1–7 frozen, 8–13 appended; spec 29와 단일 출처).

## 도메인 모델 (BE)

`backend/internal/settings/settings.go`. db/proto 태그 없는 순수 타입(헌법5).

- `EmotionColor { Mood string; Color string }` — Mood는 소문자 enum 이름("joy".."emptiness"), Color는 `#RRGGBB`.
- `Settings { Theme string; StarObject string; EmotionColors []EmotionColor }` — 저장된 오버라이드. 빈 Theme/StarObject = "오버라이드 안 됨".
- `Patch { Theme *string; StarObject *string; EmotionColors []EmotionColor }` — 부분 갱신. nil 포인터 = "필드 미전송 — 보존", `EmotionColors`는 upsert할 mood subset(삭제 없음).
- `Repository` 포트: `Get(ctx, userID) (Settings, error)` · `Update(ctx, userID, Patch) error`. delete 없음(오버라이드는 추가·변경만).

## 저장/검증 경로 (BE)

- `service.go` `Service.Update`는 **쓰기 전에 전체 patch를 검증**한다(부분 적용 금지 — AC 3).
  - `validThemes = {vast, lively, calm}`, `validObjects = {deepfield, aurora, liquid, ember}` — 클라 카탈로그의 최소 미러(명백한 미지값 거부용; 카탈로그가 커지면 동기 필요).
  - `hexColor = ^#[0-9a-fA-F]{6}$` 정규식 — 색 형식만 검사하고 미적 판단은 안 한다.
  - 검증 sentinel: `ErrInvalidColor` · `ErrInvalidMood` · `ErrInvalidTheme` · `ErrInvalidObject`.
  - 검증 통과 시 `repo.Update`(부분 upsert) 후 `repo.Get`(merged overrides) 반환.
- `handler.go`는 thin: 인증 + proto↔도메인 매핑 + mood enum 검증.
  - `moodKey(Mood) (string, bool)`: proto enum → 소문자 저장 키. `MOOD_UNSPECIFIED`와 범위 밖은 거부(13 named mood만 유효).
  - `toProtoSettings(Settings)`: 도메인 → proto. 소문자 mood → enum 역매핑(미지 mood는 방어적으로 skip).
- `repository_pg.go` `Update`는 한 트랜잭션:
  - 단일값 행(`user_settings`)은 `theme` 또는 `star_object`가 있을 때만 upsert(감정색만 바꾸는 갱신이 빈 행을 만들지 않게).
  - 그 다음 각 감정색을 upsert. 실패 시 partial rows 없음.
  - `Get`은 `GetUserSettings`(없으면 `pgx.ErrNoRows` → 빈 값) + `ListUserEmotionColors`(없으면 빈 슬라이스).

## 주요 쿼리/저장소

- `backend/internal/db/migrations/00002_user_settings.sql` (up/down):
  - `user_settings (user_id TEXT PK, theme TEXT, star_object TEXT, created_at, updated_at)` — theme/star_object 둘 다 nullable(NULL → 클라 기본).
  - `user_emotion_colors (user_id TEXT, mood TEXT, color TEXT, PRIMARY KEY (user_id, mood))` — 정규화 행. `(user_id, mood)` 유일 → mood당 한 색.
  - 00001은 append-only 수정 금지. 30이 00002를 점유했고, 그 결과 21 memory-fragmentation은 00004로 밀렸다(34 admin이 00003 점유 — 단일 기준은 00.overview 공유 설계 결정).
- `backend/internal/db/queries/settings.sql`:
  - `GetUserSettings :one` — `SELECT theme, star_object`.
  - `ListUserEmotionColors :many` — `SELECT mood, color ... ORDER BY mood`.
  - `UpsertUserSettings :exec` — `ON CONFLICT (user_id) DO UPDATE SET theme = COALESCE(EXCLUDED.theme, ...)` → NULL 들어온 필드는 기존 값 보존(부분 갱신을 SQL 레벨에서 보장).
  - `UpsertUserEmotionColor :exec` — `ON CONFLICT (user_id, mood) DO UPDATE SET color = EXCLUDED.color`.

## FE — store 서버화

- `entities/appearance/model/store.ts`의 `useAppearance`(zustand + persist):
  - 상태: `theme` · `object`(StarObject) · `selfObject`(SelfObject) · `emotionColors: Record<string,string>`.
  - 액션: `setTheme` · `setObject` · `setSelfObject` · `setEmotionColor(mood, color)` · `applyServerSettings(ServerAppearance)` · `resetServerSettings()`.
  - **localStorage 영속은 기기 선호(theme·object·selfObject)만**(`partialize`). `emotionColors`는 per-user라 메모리 전용 — 공용 PC에 개인 데이터를 영속하지 않는다(`policy/domain/data-sync.md` 정신).
  - `merge`가 알 수 없는/손상된 저장값을 각 축 기본값으로 폴백한다(헌법 견고성).
  - 레거시 1회 승계: 구 키 `cosimosi.landing.theme`가 색+형태를 함께 담았으므로, 그 값이 StarObject 4-값이면 `object`로 승계하고 `theme`은 `vast`로 폴백(`legacyInitial`). 새 키 `cosimosi.appearance`가 이미 있으면 안 건드린다.
  - `applyServerSettings`는 감정색 내용이 그대로면 참조를 유지해(`sameColors`) 테마·오브제만 바뀐 재시드에서 별·시냅스 색 전체 재베이킹을 피한다.
- `entities/appearance/api/settings-query.ts` 데이터 계층(헌법4 — three/React/DOM 미의존, 옵션 빌더 + 순수 매핑):
  - `settingsQueryOptions()` — connect-query `GetSettings` 옵션. `staleTime 5분` · `gcTime 30분` · `refetchOnWindowFocus: false`(설정은 드물게 바뀌고 포커스 refetch는 쓰기와 경합). 체험 모드(`isDemoMode()`)는 서버 없이 빈 응답을 반환해 같은 쿼리 경로로 태운다.
  - `applySettings(res)` — `GetSettings` 응답을 store에 머지.
  - `pushSettings(patch)` — 낙관적 store 갱신 후 호출하는 fire-and-forget 쓰기. 체험 모드이거나 access token이 없으면(`getAccessToken()`) 서버 쓰기를 건너뛴다(401 잡음·체험 오염 방지, 로컬 변경만). 실패는 삼키고 다음 시드가 재동기화.
  - `toServerAppearance(ProtoSettings)` — proto → store 형태(mood enum → 소문자 키, UNSPECIFIED skip).
- 시드 지점: `pages/home/ui/HomePage.tsx`가 인증된 우주에서 `useQuery(settingsQueryOptions())` → `applySettings`. 미인증 랜딩은 이 페이지를 안 그린다.
- 출처 경계 리셋: `app/model/reset-universe-data.ts`와 로그아웃·계정 전환·체험 enter/exit이 `resetServerSettings()`로 감정색 오버라이드를 비운다(테마·오브제는 기기 선호라 유지). 다음 사용자가 `GetSettings`로 재시드.
- 공유 우주 방문(35): `pages/visit/ui/VisitPage.tsx`가 소유자의 `Settings`(공유 스냅샷 `share.proto`의 `Universe.appearance` = field 4, `Settings` 재사용)를 `resetServerSettings()` 후 `applySettings`로 적용해 방문자 화면에 소유자 색을 입힌다.

## 감정색 소비 (렌더)

- `shared/config/mood.ts`:
  - `MOOD_PALETTE: Record<Mood, RGB>` — 13 mood 기본 팔레트(linear RGB 0..1). `NEUTRAL_RGB = [0.6,0.6,0.6]`.
  - `moodRgb(mood)` — 기본 팔레트 + neutral 폴백(throw 금지).
  - `hexToRgb(hex)` — `#RRGGBB` → linear RGB 튜플(8-bit 직접 매핑, 미지/손상 → null).
  - `resolveMoodRgb(mood, overrides?)` — 오버라이드가 있고 유효하면 그 색, 아니면 `moodRgb`. 별 색의 단일 진입점.
- `resolveMoodRgb(mood, emotionColors)`를 쓰는 곳: `entities/star/ui/StarField.tsx`(per-instance aMood 베이킹), `widgets/universe-canvas/ui/UniverseCanvas.tsx`(시냅스 colById), 오버레이 우주(`overlay/OverlayUniverse.tsx`·`OverlayComparePanel.tsx`), `features/record-memory/ui/MemoryForm.tsx`, `features/evolution/ui/EvolutionPanel.tsx`, `pages/gift/ui/GiftPage.tsx`, `widgets/demo-sim/ui/DemoSimPanel.tsx`.

## Error Mapping

- 인증 context가 없으면 `Unauthenticated`(두 RPC 모두).
- 미지 theme/object, 비-`#RRGGBB` color, UNSPECIFIED/범위 밖 mood → `InvalidArgument`(쓰기 전 거부, 부분 적용 금지).
- 일반 저장/조회 실패는 `Internal`.

## Flutter 동등성 기준

- `SettingsService.GetSettings`/`UpdateSettings`는 공유 BE 계약이다. Flutter도 인증 헤더로 호출하고, 빈 응답을 자기 기본값(vast·deepfield·`MOOD_PALETTE`) 위에 머지해야 한다.
- 서버는 오버라이드만 저장한다 — Flutter가 BE에 13색 기본 상수를 기대하면 안 된다(기본은 클라 소유).
- `UpdateSettings`는 부분 갱신이다. 보낸 필드만 upsert되고 나머지는 보존된다는 의미를 클라가 지켜야 한다(전체 config를 매번 보내지 않는다).
- 감정색 키는 mood enum의 소문자 이름이고 색은 `#RRGGBB`다. UNSPECIFIED는 유효 mood가 아니다.
- 미인증·오프라인·로딩 중에도 기본값으로 렌더가 깨지지 않아야 한다(점진적 향상).
- 별 색은 `resolveMoodRgb`처럼 "오버라이드 있으면 그 hex, 없으면 기본 팔레트, 미지값은 neutral 폴백"으로 해석해야 한다.
- `SelfObject`(자아 별 형태)는 현재 서버 동기 대상이 아니다 — Flutter도 기기-로컬로 둔다(서버 동기는 후속).

## 수용 기준

1. 인증 사용자가 `GetSettings`를 호출하고 오버라이드가 없으면 빈 `Settings`(theme=""·star_object=""·emotion_colors 없음)를 반환하고, 클라가 자기 기본값 위에 머지해 완전한 설정을 만든다.
2. 일부 필드만 `UpdateSettings`로 보내면 그 필드만 upsert되고 나머지는 보존되며, 이후 `GetSettings`가 (덮어쓴 값 ⊕ 기본값)을 반영한다.
3. 비-`#RRGGBB` color, 13 화이트리스트 밖 mood, 미지 theme/star_object는 `InvalidArgument`로 거부되고 어떤 행도 쓰이지 않는다(부분 적용 금지).
4. 감정색은 `(user_id, mood)` 유일 정규화 행으로 저장돼 `GROUP BY (mood, color)` 집계가 가능하다.
5. 감정색만 바꾸는 `UpdateSettings`는 빈 단일값 행(`user_settings`)을 만들지 않는다.
6. FE가 미인증·오프라인·체험·로딩 중이면 기본 테마·오브제·`MOOD_PALETTE`로 렌더하고 깨지지 않는다.
7. 인증 사용자가 `AppearanceSwitcher`로 테마/오브제를 바꾸면 `UpdateSettings`로 영속되고 새로고침·타 기기에서 같은 설정이 로드된다.
8. 별·시냅스 렌더가 감정색 오버라이드를 반영하고, 오버라이드 없는 mood는 기본 팔레트로 그린다.
9. 로그아웃·계정 전환·체험 전환 시 이전 사용자의 감정색 오버라이드가 다음 출처로 새지 않는다(테마·오브제는 기기 선호라 유지).
10. 미인증·체험 세션에서는 `pushSettings`가 서버 쓰기를 건너뛰고 로컬 변경만 반영한다.
11. 두 RPC 모두 미인증이면 `Unauthenticated`로 거부한다.
12. `selfObject`는 서버에 전송·저장되지 않고 localStorage에만 영속한다.
