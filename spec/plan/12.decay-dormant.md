# 12. 감쇠·최소 밝기·잠든 별 탐색 (decay-dormant)

> 활성도 감쇠를 렌더 시 계산하고 최소 밝기 바닥을 둬 별·시냅스가 어두워질 뿐 사라지지 않게 하며, 잠든 별을 탐색·재점화한다. | 범위: MVP | 선행: 08, 09, 11 | 차단: —

## 목적 (Why)
원칙 2(별·시냅스는 우주에서 삭제하지 않음, 감쇠는 밝기만)를 실제로 구현한다. 오래 회상하지 않은 별·연결은 `exp(-λΔt)`로 어두워지되 `a_min=0.05` 바닥 아래로는 내려가지 않고 "잠든(dormant) 상태"로 계속 렌더링된다. 어두워 눈에 안 띄는 별을 시간순·검색으로 다시 찾아 회상(재점화)하는 회고 동선을 만든다.

## 범위
- **포함:**
  - FE: `entities/memory/model/activation.ts`(08이 `exp(-λΔt)`·`a_min` floor·`activation`·`starBrightness` 정의)에 **`synapseBrightness`(=`weight·max(a_min, activation)`)·`isDormant`만 추가**(08 심볼은 재사용·재정의 금지). 별 유효 밝기 `max(a_min, activation)`는 08의 `starBrightness` 그대로 사용.
  - FE: `universe-canvas`에서 잠든 별을 **우주에서 제거하지 않고** 어둡게 잔존 렌더(밝기 0이 아닌 은은한 잔광).
  - FE: `pages/dormant` — 잠든(어두운) 별 목록·검색 → 클릭 시 해당 별로 카메라 이동 → 회상으로 재점화.
  - BE: `ListDormant` RPC — 밝기가 `a_min` 부근(임계 이하)인 별을 탐색 보조용으로 반환(제거 아님). `db/queries/memory.sql`에 쿼리 추가.
- **비목표(Out of scope):**
  - 관련성 가중 감쇠(연결 수·요즘 관련성·감정 강도 가중) — v1 #23(`competitive-allocation + relevance-decay`)이 담당. MVP는 **시간 기반 감쇠만**.
  - 요지 추상화(잠든 별 형태 단순화) — v1 #23/별 형태 고도화.
  - 야간 배치(전체 재안정화·가지치기) — v1 #22(`nightly-consolidation`).
  - 회상 강화·`last_recalled_at` 갱신 자체 — 11(`recall-reinforce`)이 담당. 본 스펙은 그 값을 **읽어 감쇠를 계산**하고 재점화 동선만 연결.
  - `GetUniverse` 신규 구현 — 04(`memory-api`)가 담당(여기선 전체 그래프 반환 유지를 **검증**만).

## 설계 요점
**FE 활성도 (순수 model — `three`/React/DOM 미의존, 원칙 4):**

> ⚠️ **08이 권위.** `LAMBDA`(이름 `LAMBDA`, `DECAY_LAMBDA` 아님)·`A_MIN`·`HALF_LIFE_DAYS`·`activation`·`starBrightness`는 08이 이미 `entities/memory/model/activation.ts`에 정의·export한다(StarField가 사용 중). 12는 **재정의하지 않고 import해 재사용**하며, 같은 파일에 **`synapseBrightness`·`isDormant`만 추가**한다. 기존 심볼의 시그니처·식을 다시 쓰지 않는다.

```ts
// entities/memory/model/activation.ts — 08 정의(재사용, 재정의 금지):
//   export const HALF_LIFE_DAYS = 30;
//   export const LAMBDA = Math.LN2 / HALF_LIFE_DAYS;            // ≈ 0.0231 /day
//   export const A_MIN = 0.05;
//   export function activation(lastRecalledAt: number, now: number): number;   // exp(-LAMBDA·Δt_days), (0,1]
//   export function starBrightness(lastRecalledAt: number, now: number): number; // max(A_MIN, activation)
//
// 12가 같은 파일에 추가하는 것만(08 심볼은 import해 사용):
/** 시냅스 유효 밝기 = weight · max(A_MIN, activation). `now`는 외부 주입. */
export function synapseBrightness(weight: number, lastActivatedAt: number, now: number): number {
  return weight * Math.max(A_MIN, activation(lastActivatedAt, now));
}
/** 잠든 판정: 활성도가 임계 이하(기본 a_min 부근). 임계는 RAW activation 기준(밝기 바닥 max 적용 전). */
export function isDormant(lastRecalledAt: number, now: number, threshold = 2 * A_MIN): boolean {
  return activation(lastRecalledAt, now) <= threshold;
}
```
- 렌더는 이 값을 받아 별 밝기 / 시냅스 두께·emissive로 매핑(셰이더 식은 08·09 소관). 잠든 별도 `A_MIN` 잔광으로 그려져 클릭 가능.
- `now`는 외부 주입(테스트 가능). 활성도는 **로드 시·회상 직후에만** uniform/attribute에 반영하고, 매 프레임 React state 재계산은 하지 않는다(ref/uniform 갱신, §3.2). 별 추가·회상 같은 이벤트에서 `aBrightness`/시냅스 밝기 attribute를 다시 채운다.

**brightness 배선(없으면 dimming이 렌더 안 됨):**
- 별: `GetUniverse`(04) `Star{ memory_id, mood, intensity, last_recalled_at }` → `StarNode`(08 store) 매핑 후, `starBrightness(lastRecalledAt, now)`로 per-instance **`aBrightness`** attribute(08 StarField가 소비)를 채운다. `last_recalled_at`(RFC3339 문자열)→epoch ms 파싱은 매퍼에서.
- 시냅스: `GetUniverse` `Synapse{ a_id, b_id, weight, link_type, last_activated_at }` → 11이 신설한 **`SynapseEdge`(entities/synapse 또는 entities/memory 엣지 스토어)** 매핑 후, `synapseBrightness(weight, lastActivatedAt, now)`로 09 시냅스 머티리얼의 emissive/alpha/펄스 입력을 채운다(09는 per-edge 밝기·alpha·펄스로 강도 표현, 전역 두께 + 선택적 두께 버킷). 12는 이 매퍼 산출 밝기를 **소비**만 하고 엣지 스토어/매퍼 자체는 11이 만든다.

**카메라 fly-to(06은 카메라 모드만, 08 store는 selectedId만; 부족분 정의):**
- `focusStarId: string | null` 상태와 `focusStar(id: string | null)` 액션을 둔다(우주 셸/카메라 스토어). `/dormant`에서 항목 클릭 시 `/universe`로 이동하며 `focusStarId`를 해당 별로 설정.
- 카메라 컨트롤러(06 카메라 모드 위)는 `focusStarId`가 바뀌면 그 별의 force-sim 좌표를 타깃으로 **부드럽게 fly-to**(lerp/damp, `useFrame`에서 카메라 위치·lookAt 보간)하고, 도달 후 `select(id)`로 회상 패널(11) 트리거.

**BE `ListDormant`(탐색 보조, 제거 아님):**
```sql
-- db/queries/memory.sql
-- name: ListDormant :many
-- 오래 회상하지 않은(잠든) 별만 탐색 보조용으로 반환. 우주에서 제거하지 않는다.
-- ⚠️ WHERE에 exp()/감쇠식 금지 — sargable 유지 위해 last_recalled_at 임계만 사용(밝기는 클라 계산).
-- ⚠️ memories만으로 충분치 않음 — 본문/날짜/감정은 records에 있으므로 JOIN(원본=records 불변 / 별=memories 가변).
SELECT m.id, r.entry_date, r.body, r.mood, r.intensity, m.last_recalled_at
FROM memories m
JOIN records r ON r.id = m.record_id
WHERE m.user_id = $1
  AND m.last_recalled_at < sqlc.arg(cutoff)::timestamptz
ORDER BY m.last_recalled_at ASC;
```
- 서비스가 `threshold`(기본 `2·a_min`)를 `cutoff = now - HALF_LIFE_DAYS·log2(1/threshold)일` 같은 시각 임계로 환산해 인자로 넘긴다(감쇠 역산은 서버 비즈니스 로직; WHERE는 시각 비교만). 잠든 판정 임계 식의 단일 출처는 FE `isDormant`이며, 서버는 동등한 cutoff를 계산한다.
- 핸들러는 row→도메인→proto 매핑(원칙 5). proto는 **`ListDormantRequest{}`(빈 메시지) / `ListDormantResponse{ repeated Star stars }`로 02의 `Star`를 재사용**(별도 `DormantStar` 메시지 금지). `Star{ memory_id, mood, intensity, last_recalled_at }`에 채우고 body는 전송하지 않는다(별 시각 입력엔 body 없음). 회상 시 본문은 11 `RecallMemory`로 별도 fetch.
- 모든 쿼리는 `WHERE m.user_id = $1`(인터셉터 컨텍스트 user_id)로 격리.
- ⚠️ `GetUniverse`는 변경 없음 — **여전히 전체 그래프 반환**(잠든 별 포함). `ListDormant`는 별도 보조 조회일 뿐 삭제·필터 영속화가 아니다.

**FE `pages/dormant`:** `ListDormant` unary 호출(원칙 6) → 목록 + 텍스트 검색 → 항목 클릭 → `focusStar(memory_id)`로 `focusStarId` 설정 후 `/universe`로 이동 → 카메라 fly-to(위 정의)로 해당 별로 이동 → 11의 회상 액션(`select`/`RecallMemory`)으로 재점화. 응답이 `Star`(body 없음)이므로 목록의 본문 검색은 `Star` 필드(mood 등) 또는 별도 메모 표시 범위로 한정한다(원본 본문은 회상 시 11이 fetch).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 별의 `last_recalled_at`이 현재보다 며칠~수십 일 과거일 때 THE SYSTEM SHALL 그 별의 표시 밝기를 `exp(-λ·Δt_days)`(λ=ln2/30)로 감소시킨다.
1.2 WHEN 감쇠 계산 결과가 `a_min=0.05`보다 작아질 만큼 오래되었을 때 THE SYSTEM SHALL 별 유효 밝기를 `max(a_min, activation)`로 바닥 처리하여 0(소멸)으로 내려가지 않게 한다. (원칙 2)
1.3 WHEN 시냅스 밝기를 계산할 때 THE SYSTEM SHALL `weight · max(a_min, activation)`을 적용한다.
2.1 GIVEN 활성도가 `a_min` 부근으로 떨어진 잠든 별 WHEN 우주를 렌더링할 때 THE SYSTEM SHALL 그 별을 우주에서 제거하지 않고 어두운 잔광으로 계속 렌더링하며 클릭 가능 상태로 둔다.
2.2 WHEN 사용자가 잠든 별을 회상(11)하여 `last_recalled_at`이 now로 갱신될 때 THE SYSTEM SHALL 다음 렌더에서 그 별을 다시 밝게(activation→1 근처) 표시한다.
3.1 WHEN 클라이언트가 `ListDormant`(빈 요청)를 호출할 때 THE SYSTEM SHALL 호출한 user의 별 중 `last_recalled_at`이 서버 cutoff(기본 임계 `2·a_min` 환산) 이전인 별만 `last_recalled_at` 오름차순으로 `Star`(02 재사용)로 반환한다. (WHERE에 exp() 없음 — 밝기는 클라가 `starBrightness`로 계산.)
3.2 WHEN `GetUniverse`를 호출할 때 THE SYSTEM SHALL 잠든 별을 포함한 **전체 그래프**를 변함없이 반환한다(ListDormant는 삭제·영속 필터가 아니다).
4.1 WHEN 사용자가 `/dormant`에서 항목을 클릭할 때 THE SYSTEM SHALL 우주 화면으로 전환하고 카메라를 해당 별로 이동시킨다.
4.2 IF `ListDormant`가 빈 목록을 반환하면 THEN THE SYSTEM SHALL "잠든 별이 아직 없습니다" 안내를 표시하고 빈 화면 오류를 내지 않는다.
5.1 WHEN `activation.ts`를 작성할 때 THE SYSTEM SHALL `three`/React/DOM을 import하지 않는다(순수 model, 원칙 4·모바일 재사용).

## 완료 정의 (DoD)
- `frontend/src/entities/memory/model/activation.ts`가 `three`/React/DOM import 없이 컴파일되고, 단위 테스트에서 Δt=30일→`activation≈0.5`, 충분히 큰 Δt→유효 밝기가 정확히 `a_min=0.05`에서 멈춤(0 아님)을 검증. (`pnpm --filter frontend test`)
- 어떤 별의 `last_recalled_at`을 과거로 조작하면(예: DB에서 수개월 전으로 갱신) 우주에서 그 별이 **어두워지지만 사라지지 않고** 여전히 클릭 가능. 11의 회상 동작 후 다시 밝아짐.
- `MemoryService.ListDormant`(빈 요청)가 cutoff 이전 별만 `Star`(02 재사용)로 반환하고, 동일 상태에서 `GetUniverse`는 잠든 별 포함 전체 그래프를 반환(`buf curl` 또는 클라 호출로 양쪽 응답 비교; ListDormant ⊆ GetUniverse stars 확인).
- `/dormant` 페이지에서 잠든 별이 목록·검색으로 발견되고, 클릭 시 `focusStarId` 설정→`/universe`로 이동→카메라 fly-to→회상 재점화됨(수동 E2E). 잠든 별이 없으면 빈 상태 안내 표시.
- `pnpm build`(프론트) 성공, 백엔드 `go build ./...`(Docker/WSL)·`sqlc generate`·`buf generate --template backend/buf.gen.yaml proto` 무오류.
