# 12. 감쇠·최소 밝기·잠든 별 탐색 - 망각 모델 + ListDormant (as-built)

> 현재 구현 기준 문서. 별·시냅스는 삭제되지 않고 시간 감쇠로 어두워질 뿐이며(A_MIN 바닥), 잊혀진(잠든) 별을 다시 찾아 회상하는 동선을 정의한다. | 범위: FS(FE 렌더 모델 + BE ListDormant) | 상태: 구현 완료, 반영: 26 변조 감쇠, overlay 셸(tech/overlay-shell.md), 항행 XState 머신(tech/state-machines.md)
>
> **(spec 03 — changes/03, job 08):** `activation`(=최근성)은 그대로이되, 이제 별 빛 3채널 중 **반사(reflection)** 채널의 per-instance 입력(`aRecency`)이기도 하다(가까운=최근 별이 더 밝게 비춰짐). **dormant 판정·ListDormant는 무변** — 여전히 raw `activation ≤ 2·A_MIN`(연결성/self-glow 무관, "외로운 별 ≠ 잠든 별"). A_MIN 밝기 바닥은 self-glow 채널이 보증. 상세는 [policy/domain/star.md](../policy/domain/star.md) §밝기·감쇠.

## 목적

헌법 2(별·시냅스는 우주에서 삭제하지 않고, 감쇠는 밝기만)를 실제로 구현한다. 오래 회상하지 않은 별·연결은 `exp(-λ·Δt)`로 어두워지되 `A_MIN` 바닥 아래로 내려가지 않고 "잠든(dormant) 상태"로 계속 렌더된다. 밝기·활성도는 전부 클라이언트가 렌더 시 결정론적으로 계산하고(서버엔 밝기 컬럼이 없다), 서버는 `last_recalled_at`만 권위로 들고 있는다. 어두워 눈에 안 띄는 별을 다시 찾도록 `ListDormant`로 잠든 별 목록을 보조 제공하고, 고르면 그 별로 날아가 회상으로 재점화한다.

감쇠/잠듦 판정의 수치 SSOT는 `spec/policy/domain/star.md`(밝기·감쇠 표)다. 이 문서는 그 값을 구현이 어디서·어떻게 쓰는지를 기술한다.

## 현재 구현

### FE 활성도 모델 (순수 model)

`frontend/src/entities/memory/model/activation.ts`는 `three`/React/DOM에 의존하지 않는 순수 함수 모음이다(헌법 4 — 모바일 재사용 가능). 한 파일에 세 스펙의 함수가 누적돼 있고 소유권이 분리된다:

- **08 소유 (시간 감쇠 기반)** — 12·26이 재사용하며 재정의하지 않는다:
  - `HALF_LIFE_DAYS`, `LAMBDA = Math.LN2 / HALF_LIFE_DAYS`, `A_MIN`.
  - `activation(lastRecalledAt, now)` = `exp(-LAMBDA·Δt_days)` ∈ (0,1]. `Δt`는 `max(0, …)`로 미래 타임스탬프를 0으로 막는다(밝기 > 1 방지).
  - `starBrightness(lastRecalledAt, now)` = `max(A_MIN, activation(...))`. 별 유효 밝기 바닥.
- **12 소유 (이 스펙이 추가)**:
  - `synapseBrightness(weight, lastActivatedAt, now)` = `weight · max(A_MIN, activation(...))`. 시냅스 유효 밝기를 한 함수로 표현한 순수 헬퍼. 단위테스트로 검증돼 있으나 **라이브 렌더 경로는 호출하지 않는다**(아래 시냅스 배선의 2단계 분해형이 동치값을 만든다). 프로덕션 소비자 없는 사양상 헬퍼.
  - `isDormant(lastRecalledAt, now, threshold = 2·A_MIN)` = `activation(...) ≤ threshold`. 판정 기준은 **바닥 적용 전 RAW activation**이라 A_MIN 아래에서도 의미가 유지된다. 데모 모드의 클라 잠듦 필터와 서버 `dormantCutoff`가 이 식을 공유한다.
- **26 소유 (변조 감쇠)** — `LAMBDA`/`A_MIN`/`activation`/`starBrightness`를 그대로 재사용해 추가:
  - 계수 `ALPHA_CONN`/`BETA_RECENT`/`GAMMA_EMO`/`DELTA_VAL`.
  - `lambdaEff(degreeNorm, relevance, intensityNorm, valence)` = `LAMBDA · R_conn · R_recent · R_emo`. 각 `R ∈ (0,1]`이라 변조는 감쇠를 늦추기만 한다(λ_eff ≤ LAMBDA). 모든 입력 clamp.
  - `modulatedBrightness(lastRecalledAt, now, degreeNorm, relevance, intensityNorm, valence)` = `A_MIN + (1-A_MIN)·exp(-λ_eff·Δt_days)` ∈ [A_MIN, 1]. `starBrightness`의 drop-in 대체이며 **현재 라이브 별 경로가 쓰는 함수**(26 소관).

`now`는 항상 외부 주입이라 테스트 가능하다. 데모 모드에서는 `virtualNowMs()`(가상 시계, 19)가 `now`를 공급해 시간 머신이 흐른 만큼 별이 잠든다.

### 별 밝기 배선

별 밝기 attribute(`aBrightness`)는 `frontend/src/entities/star/ui/StarField.tsx`가 채운다(InstancedMesh per-instance attribute는 08 StarField 소유, 헌법 8). 현재 식은 변조 감쇠 위에 재성형 오프셋을 합성한 것이다:

```
aBrightness[i] = reshapedBrightness(
  modulatedBrightness(lastRecalledAt, now, degreeNorm, relevance, intensity, valence),  // 26
  brightnessOffset,  // 23
)
```

- `degreeNorm`은 `degreeNormById(edges)`(synapse 엔티티)로 별의 연결 수를 정규화한 값이다.
- 같은 식이 두 곳에서 attribute를 다시 굽는다: 별 세트/형태가 바뀌는 레이아웃 effect, 그리고 선택(focus) 변화 시 재가중 effect(선택 별 `FOCUS_BOOST`·나머지 `FOCUS_DIM`). 매 프레임 React state 재계산은 하지 않는다 — 좌표만 `useFrame`에서 갱신한다(헌법 3).
- 12 도입 당시의 출발점은 `starBrightness(lastRecalledAt, now)`였고, 26이 `modulatedBrightness`로 확장하며 라이브 경로를 대체했다. `starBrightness`는 여전히 시냅스·dormant 뷰모델 경로에서 쓰인다.
- 좌표는 force-sim 버퍼(07/22)에서, 버퍼 미준비 시엔 결정론적 fibonacci 더미(`fibonacciStarPosition`)에서 온다. 같은 더미 좌표를 fly-to 카메라 타깃이 공유한다.

### 시냅스 밝기 배선

시냅스 유효 밝기는 `synapseBrightness` 한 함수가 아니라 **시간 감쇠 인자 굽기 → 가중 곱**의 2단계로 분해돼 있다(동치값):

1. `frontend/src/entities/memory/api/universe-query.ts`의 `applyUniverse`가 `GetUniverse` `Synapse`를 `SynapseEdge`로 매핑하며 `edge.brightness = starBrightness(lastActivatedAt, now)`로 시간 감쇠 인자를 굽는다(weight를 접지 않은 순수 감쇠 인자). `last_activated_at`(RFC3339)→epoch ms 파싱은 `parseEpochMs`.
2. `frontend/src/entities/synapse/model/mapping.ts`의 `visualIntensity(e)` = `clamp(0..1, weight · max(A_MIN, brightness))`가 가중을 곱한다. 09 머티리얼의 `emissive`/`alpha`/`pulseAmp`/`strandStyle` 입력이 모두 이 값에서 나온다.

결과적으로 잠든 링크도 `A_MIN` 잔광으로 그려지고 `ALPHA_MIN` 바닥으로 보인다 — 어두워질 뿐 사라지지 않는다(헌법 2). `refreshActivation`은 데모 시간 머신 직후 데이터는 그대로 두고 `now`만 옮겨 별 배열 identity 갱신 + 엣지 `brightness` 재파생을 한다.

### BE `ListDormant`

인증 필요. 호출자의 오래 회상하지 않은 별만 탐색 보조용으로 반환한다 — 삭제·필터 영속화가 아니다(헌법 2).

- 쿼리 `backend/internal/db/queries/memory.sql`의 `-- name: ListDormant :many`:
  - `memories`만 읽고 `records`를 JOIN하지 않는다. mood/intensity/valence는 조각 별 자신의 것이다(memories, 21).
  - `WHERE m.user_id = $1 AND m.last_recalled_at < sqlc.arg(cutoff)::timestamptz`, `ORDER BY m.last_recalled_at ASC`. WHERE는 시각 비교만 — `exp()`/감쇠 수학을 SQL에 넣지 않는다(sargable; 밝기는 클라 계산).
  - SELECT 컬럼은 `m.id AS memory_id, m.mood, m.intensity, m.valence, m.last_recalled_at` + reshape 4컬럼(`brightness_offset, hue_shift, form_seed_delta, version`, 23). `ListMemoriesByUser`와 같은 컬럼 모양이라 같은 도메인 `Memory`로 매핑된다.
  - `body`는 SELECT하지 않는다 — 별 시각 입력엔 본문이 없다(원본은 회상 시 11 `RecallMemory`로 fetch).
- 서비스 `backend/internal/memory/service.go`:
  - `ListDormant(ctx, userID)`가 `dormantCutoff(time.Now().UTC())`를 repo에 시각으로 넘긴다.
  - `dormantCutoff(now)`는 잠듦 임계(RAW activation, `dormancyThreshold = 2·A_MIN`)를 시각 cutoff로 환산한다: `activation = exp(-λ·Δt) ≤ threshold ⟺ Δt ≥ HALF_LIFE_DAYS·log2(1/threshold)일`. 감쇠 역산은 서버 비즈니스 로직이고 WHERE는 시각 비교만 한다. 순수 함수(now 주입)라 테스트 가능. 잠듦 임계 식의 단일 출처는 FE `isDormant`이며 서버는 동등한 cutoff를 계산한다.
- 리포지토리 `backend/internal/memory/repository.go`/`repository_pg.go`:
  - 포트 `ListDormant(ctx, userID string, cutoff time.Time) ([]Memory, error)`.
  - sqlc gen `ListDormant(ctx, gen.ListDormantParams{UserID, Cutoff})` 호출 후 row→태그 없는 도메인 `Memory` 매핑(헌법 5).
- 핸들러 `backend/internal/memory/handler.go`:
  - `ListDormant(ListDormantRequest{})` → service → `toStar(m)`로 도메인→proto 매핑 → `ListDormantResponse{Stars}`.
  - `toStar`는 GetUniverse와 공유한다(두 렌더 경로가 노출 필드에서 드리프트하지 않게). dormant 응답에서 `record_id`=""·`relevance`=0·`resonant`=false(그룹/스코어/공명 조인 없음), body 미전송.
  - 인증 컨텍스트 없으면 `Unauthenticated`, 조회 실패는 `Internal`.
- 모든 쿼리는 인터셉터 컨텍스트 `user_id`로 격리된다.
- `GetUniverse`는 변경 없음 — 여전히 잠든 별 포함 전체 그래프를 반환한다(04 소관). `ListDormant`는 별도 보조 조회일 뿐이다.

## Flutter 동등성 기준

- BE 계약은 그대로 따른다: `ListDormant(ListDormantRequest{})` unary → `Star[]`(body 없음)을 `last_recalled_at` 오름차순으로 받고, `GetUniverse`는 잠든 별 포함 전체 그래프를 반환한다. Flutter는 ListDormant 결과가 GetUniverse stars의 부분집합임을 가정할 수 있다.
- 밝기·activation은 클라가 계산해야 한다 — 서버는 `last_recalled_at`/`last_activated_at`만 보낸다. 별 유효 밝기는 `max(A_MIN, activation)` 바닥을, 시냅스는 `weight · max(A_MIN, activation)`을 적용해 어떤 별/링크도 0으로 사라지지 않게 한다(헌법 2). 라이브 별 밝기는 변조 감쇠(26)를 쓴다.
- 잠듦 판정은 RAW activation `≤ 2·A_MIN`(바닥 적용 전)으로 하고, 서버 cutoff와 같은 결과를 내야 한다.
- 잠든 별을 고르면 그 별로 카메라를 이동하고 회상으로 재점화하는 동선을 제공한다. 회상 후 그 별을 잠든 목록에서 제외한다.
- 수치(HALF_LIFE_DAYS·A_MIN·dormancy threshold·계수)는 `spec/policy/domain/star.md`를 단일 출처로 따른다.

## 수용 기준

1. 별의 `last_recalled_at`이 과거일수록 표시 밝기가 `exp(-λ·Δt)` 기반으로 감소한다(라이브 별은 별마다 변조된 λ_eff).
2. 감쇠가 충분히 진행돼도 별 유효 밝기는 `A_MIN` 바닥에서 멈추고 0(소멸)으로 내려가지 않는다.
3. 시냅스 밝기는 `weight · max(A_MIN, activation)`을 적용하며 잠든 링크도 잔광으로 보인다.
4. 잠든 별은 우주에서 제거되지 않고 어두운 잔광으로 계속 렌더되며 클릭 가능하다.
5. 잠든 별을 회상해 `last_recalled_at`이 now로 갱신되면 다음 렌더에서 다시 밝아진다.
6. `ListDormant`(빈 요청)는 호출자 별 중 `last_recalled_at`이 서버 cutoff(기본 임계 `2·A_MIN` 환산) 이전인 별만 `last_recalled_at` 오름차순으로 `Star`(body 없음)로 반환한다.
7. `ListDormant`의 SQL WHERE에는 `exp()`/감쇠 수학이 없고 `last_recalled_at` 시각 비교만 있다.
8. 같은 상태에서 `GetUniverse`는 잠든 별 포함 전체 그래프를 변함없이 반환한다(ListDormant ⊆ GetUniverse stars).
9. 잠든 별 시트는 우주(`/`)의 `/?panel=dormant` 오버레이로 뜬다(별도 `/dormant` 경로는 제거 — 진입 시 NotFound).
10. 시트에서 잠든 별을 고르면 카메라가 그 별로 날아가고(FLY_TO_STAR) 도착 후 회상 패널이 열린다 — 우주를 떠나지 않는다.
11. 잠든 별이 없으면 빈 상태 안내를 표시하고 빈 화면 오류를 내지 않는다.
12. `activation.ts`는 `three`/React/DOM을 import하지 않는다(순수 model).
13. 모든 dormant 쿼리는 인터셉터 컨텍스트의 `user_id`로 격리된다.
