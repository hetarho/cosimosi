# 01. platform foundation

> Monorepo skeleton — three clean app roots (`api` · `web` · `mobile`) each emit a verifiable "hello world"; the MVP
> apps move aside to `*-mvp` (reference-only). | Scope: Infra | Status: built

## Purpose

Everything in the rebuild stands on the monorepo wiring. Before any transport, data, or domain work, this unit proves
the toolchain end-to-end on the **clean** structure: install → build → run, for backend, frontend, and mobile, each
showing a trivial "hello world". It also sets the MVP apps aside (`apps/{api,web}` → `apps/{api,web}-mvp`) so the
clean names are free and the old code survives as a read-only reference that **cannot break the new build**. No
business logic — this is the empty stage the rest of the roadmap is built on.

## Scope / Non-goals

**In scope**
- **Set the MVP aside.** `apps/api` → `apps/api-mvp`, `apps/web` → `apps/web-mvp` (preserve git history). `apps/blog`
  is new and untouched.
- **Exclude `apps/*-mvp` from the active build** — out of the pnpm workspace, CI, lint, and typecheck. They stay on
  disk for reading only.
- **Three fresh roots, each "hello world":**
  - `apps/api` — Go module; `cmd/api/main.go` serves `GET /` → `hello world` and `GET /health` → 200 (plain
    `net/http`).
  - `apps/web` — React 19 + Vite + TypeScript; renders "hello world".
  - `apps/mobile` — React Native + TypeScript (Metro, native `ios/`+`android/`); a screen renders "hello world".
- **pnpm workspace** wiring so a clean `pnpm install` + per-app build/dev works from the repo root (JS apps +
  `packages/*`); Go app built via the Go toolchain.
- **Repoint root tooling** that referenced the old `apps/api` / `apps/web` paths (docker-compose, `wrangler.jsonc`,
  CI workflows, root `scripts/`) at the clean roots; drop `*-mvp` from them.
- **A documented local run path** — how to start each app and see "hello world".

**Non-goals** (later units / phases)
- No proto / Connect / RPC (that is the transport unit), no DB / persistence, **no domain code, no backend context
  packages, no FSD entities/features** beyond an empty app shell.
- No three.js / WebGPU / TSL rendering in web or mobile (rendering phase). Mobile is a bare hello-world screen; the
  `react-native-webgpu` renderer is integrated when rendering lands.
- No i18n, design system, auth/session, observability, or feature flags (later Phase 0 units).
- **No real deployment / hosting** (Cloudflare / Lightsail / Supabase). Local build + run only.
- **No code, import, or dependency carried over from `apps/*-mvp` or `spec/.mvp/`.**

## Design

### Repository layout after this unit
```
cosimosi/
├── proto/                  exists; unused until the transport unit
├── packages/               workspace member; empty until a second consumer promotes shared code in
├── apps/
│   ├── api/        NEW      Go — cmd/api/main.go hello world (net/http)
│   ├── web/        NEW      React 19 + Vite + TS — hello world
│   ├── mobile/     NEW      React Native + TS (Metro, ios/ + android/) — hello world
│   ├── api-mvp/    renamed  was apps/api — reference-only, excluded from workspace/CI
│   ├── web-mvp/    renamed  was apps/web — reference-only, excluded from workspace/CI
│   └── blog/                unchanged (Astro)
├── pnpm-workspace.yaml      members: apps/web, apps/mobile, apps/blog, packages/*  (NOT apps/*-mvp)
└── package.json            root scripts target the clean roots
```

### Setting the MVP aside
- `git mv apps/api apps/api-mvp` and `git mv apps/web apps/web-mvp` (history preserved; `apps/blog` left as-is).
- `pnpm-workspace.yaml` package globs select `apps/web`, `apps/mobile`, `apps/blog`, `packages/*` and **must not**
  match `apps/*-mvp` (enumerate the members, or add a `!apps/*-mvp` exclusion) — so the MVP apps are never installed,
  built, linted, or typechecked.
- Remove `*-mvp` from CI job paths, lint/tsc globs, `docker-compose*.yml` services, and `wrangler.jsonc`. The
  directories remain readable but inert.
- `apps/api-mvp` keeps its own `go.mod`; it is outside any Go build/workspace for the clean roots, so its module path
  is irrelevant going forward.

### `apps/api` (Go)
- Fresh `go.mod`, module path `github.com/cosimosi/api`, Go 1.26 (the `*-mvp` module is not reused).
- `cmd/api/main.go`: a `net/http` server on a port from env (default e.g. `8080`); `GET /` → `hello world`,
  `GET /health` → `200`. No Connect, no router framework, no `internal/` contexts yet (a context is created by the
  first feature that needs it — ARCHITECTURE §2.3).
- Runnable with `go run ./cmd/api`; optionally a dev compose service builds the same.

### `apps/web` (React + Vite)
- Vite 8 + React 19 + TypeScript 6 (scaffolded with `pnpm create vite`, `react-ts`). `index.html` → `src/main.tsx` →
  an `src/app/` entry that renders "hello world". Lint is **oxlint** (the scaffolder's default); package name
  `@cosimosi/web`.
- FSD is honored by *layout*, not by pre-built slices: only the `app/` layer exists now (entry/providers shell). No
  `pages/widgets/features/entities` yet — those are added per feature (promote-on-reuse). `dev` (Vite server, :5173)
  and `build` (static bundle → `apps/web/dist`) both work.

### `apps/mobile` (React Native)
- Scaffolded from the official React Native **TypeScript** template (bare, RN 0.86): Metro bundler, committed `ios/`
  and `android/` projects, `src/app/App.tsx` rendering a "hello world" `<Text>`. Same FSD layout intent as web (`app/`
  shell only for now). The `react-native-webgpu` + three renderer is **not** wired here. Package name `@cosimosi/mobile`;
  the native registration name (`app.json`) stays `CosimosiMobile`. The Vite demo / RN sample-screen scaffolding (and
  the template's jest sample, whose preset isn't installed) are removed — test/lint infra is the quality-gates unit.

### pnpm workspace & scripts
- `pnpm-workspace.yaml` members: `apps/web`, `apps/mobile`, `apps/blog`, `packages/*`. The Go `apps/api` is not a pnpm
  member; it is driven by the Go toolchain.
- **`.npmrc` sets `node-linker=hoisted`** — React Native's Metro bundler resolves from a flat `node_modules`, so the
  workspace uses a hoisted layout (not pnpm's default symlinked one). `apps/mobile/metro.config.js` also adds the repo
  root to `watchFolders` + `nodeModulesPaths` so Metro finds the hoisted deps.
- Root `package.json` scripts run each app: `dev:web` / `dev:api` (Docker) / `dev:mobile` (Metro) / `ios` / `android`,
  plus `build:web`, `build:api` (`go build`), and `dev` (web + api concurrently). `pnpm run setup` is `.env` + `pnpm
  install` only — Postgres/migrations/codegen return with their units (`pnpm setup` is pnpm's own built-in, unrelated).
- **Library versions are pinned at implementation** via current official scaffolders, not memory: React 19.2, Vite 8,
  TypeScript 6, React Native 0.86, Go 1.26. Toolchain baseline: **Node ≥ 20**, pnpm via corepack, Go 1.26 (run via the
  `golang:1.26` Docker image when not on the host).

### Root config updates
- `docker-compose.yml` is reduced to a single `api` dev service that builds the fresh Go hello-world
  (`apps/api/Dockerfile.dev`); a `apps/api/Dockerfile` (multistage → distroless) is added for image builds. Postgres
  and the MVP runtime env (Supabase/AI/admin/invite) are dropped — they return with their own units.
  `docker-compose.prod.yml` is image-based (no app path) and left dormant.
- `wrangler.jsonc`: the static-asset path (`./apps/web/dist`) already points at the new `apps/web` build output.
- `.github/workflows/ci.yml`: web lint+build, blog build, api vet+build target the clean roots; the Go cache keys off
  `go.mod` (no `go.sum` until a dependency lands). `deploy-backend.yml` is **manual-only** (`workflow_dispatch`) during
  the rebuild so foundation pushes never auto-deploy; the deployment unit re-arms the push trigger when hosting returns.
- The MVP proto contract is moved aside to `proto-mvp/` (reference-only, mirroring `apps/*-mvp`); `proto/` keeps only
  `buf.gen.yaml`. The codegen sentinels skip cleanly when there's nothing to build — `pnpm gen` runs `gen:values`
  (skips: no `spec/values.yaml`), buf (skips: no `.proto` under `proto/`), and sqlc (skips: no `schema.sql`), so it is
  safe to run and generates nothing into the fresh apps. The transport unit authors the real `proto/**/*.proto` (buf
  lights up then); the data unit adds `schema.sql` (sqlc) and the migrate path.
- `apps/blog` owns its content: the blog source markdown lives at `apps/blog/src/blog.md` (the spec rebuild had
  orphaned it under `spec/.mvp/`), so the blog build no longer reaches into an archived spec dir.

## Acceptance Criteria
1. `apps/api`, `apps/web`, `apps/mobile` exist as fresh roots; `apps/api-mvp` and `apps/web-mvp` are the renamed MVP
   apps with git history preserved; `apps/blog` is unchanged.
2. `apps/*-mvp` are excluded from the pnpm workspace and CI — a clean `pnpm install` and full build/lint/typecheck
   never touch them, and their contents cannot break the clean build.
3. Backend: from `apps/api`, `go run ./cmd/api` serves `GET /` → `hello world` and `GET /health` → 200.
4. Web: `pnpm --filter web dev` serves a page showing "hello world"; `pnpm --filter web build` produces a bundle.
5. Mobile: the React Native app builds and runs (emulator/simulator or Metro) showing a "hello world" screen.
6. No file under `apps/{api,web,mobile}` or `packages/` imports from `apps/*-mvp` or `spec/.mvp/`.
7. No proto/RPC, database, domain logic, three.js/WebGPU, auth, or deployment is introduced.
8. Root tooling (docker-compose, `wrangler.jsonc`, CI, root scripts) references the clean roots, not the renamed MVP
   paths.

## Policy / Values Impact
- **policy / ux:** none. This unit defines no domain or UX behavior, so it lands no rules in `policy/`.
- **spec/values.yaml:** no entries. The only configurable here (e.g. the API port) is **environment/runtime config**
  (`.env`), not a science/tuning constant — `values.yaml` is untouched.
- **Dependencies:** none (this is the root unit; every other plan depends on it).
