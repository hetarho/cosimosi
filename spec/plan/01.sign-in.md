# 01. 사인인 - Supabase Auth + 세션 머신 (as-built)

> 현재 구현 기준 문서. 웹과 Flutter가 같은 인증 경계, 라우트 보호, 토큰 주입 방식을 공유하기 위한 SSOT다. | 범위: FE/Auth | 상태: 구현 완료(세션 상태는 XState 머신 — tech/state-machines.md)

## 목적

cosimosi의 개인 우주는 Supabase 계정의 `sub`로 스코프된다. 미인증 사용자는 보호된 우주/관리/선물 수락 화면에 들어갈 수 없고, 인증된 세션의 access token은 모든 보호 RPC의 `Authorization: Bearer <token>` 헤더로 전달된다. 세션 수명주기는 현재 XState v5 머신이 소유한다.

## 현재 구현

- Supabase 브라우저 클라이언트는 `frontend/src/shared/api/supabase.ts`에 있다.
  - `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`가 없으면 앱 부팅 시 명확한 에러로 실패한다.
  - `persistSession`, `autoRefreshToken`, `detectSessionInUrl`가 켜져 있다.
  - `getAccessToken(): Promise<string | null>`을 공개하고, Connect transport가 이를 await해서 Bearer 헤더를 붙인다.
- 세션 상태는 `frontend/src/app/model/session.machine.ts`가 소유한다.
  - 상태는 `loading | authed | anon`.
  - 부팅 시 `supabase.auth.getSession()`을 호출하고, 상시 `onAuthStateChange`를 구독한다.
  - 사용자 id가 바뀌면 `resetUniverseData()`와 analytics identity 동기화를 수행해 이전 계정의 우주/일기 캐시가 다음 계정에 섞이지 않게 한다.
  - sign-out 실패로 Supabase 이벤트가 오지 않는 경우를 위해 `FORCE_ANON` 이벤트를 지원한다.
- React 바인딩은 `frontend/src/app/ui/session-context.tsx`가 소유한다.
  - `SessionContext = createActorContext(sessionMachine)`.
  - `useAuthActions()`가 `signInWithOtp`, `verifyOtp`, `signInWithGoogle`, `signOut`을 제공한다.
  - `App.tsx`는 `SessionContext.Provider`로 `RouterProvider`를 감싼다.
- 사인인 UI는 `frontend/src/app/ui/SignInScreen.tsx`.
  - 이메일은 OTP 코드 전용이다. `signInWithOtp({ email })`로 코드를 보내고, 같은 탭에서 `verifyOtp({ email, token, type: 'email' })`로 세션을 수립한다.
  - Google OAuth는 `signInWithOAuth({ provider: 'google', options: { redirectTo: origin + '/universe' } })`를 호출한다. `/universe` 복귀 시 `detectSessionInUrl`이 세션을 파싱한다.
  - 이메일 템플릿은 Supabase 대시보드에서 `{{ .Token }}`을 노출해야 한다. Google provider와 redirect allowlist도 대시보드 설정이 필요하다.

## 라우트 보호

- 공개 라우트:
  - `/`: 랜딩 페이지. 인증 게이트 없음.
  - `/u/$slug`: 공개 우주 방문 페이지. `VisitService`만 호출하며 무인증 공개 표면이다.
- `SessionGate` 보호 라우트:
  - `/universe`: 개인 우주 셸.
  - `/admin`: 관리자 콘솔. 인증 뒤 서버의 AdminService allowlist가 추가로 판정한다.
  - `/gift/$token`: 받은 별 수락/거절 페이지. 양쪽 모두 cosimosi 사용자여야 하므로 인증이 필요하다.
- `/dormant`는 별도 페이지가 아니라 `/universe?panel=dormant`로 리다이렉트한다. 최종 보호는 목적지 `/universe`에서 적용된다.
- `SessionGate` 상태 분기:
  - `loading`: 스플래시를 보여 사인인 화면 깜빡임을 막는다.
  - `anon`: `SignInScreen`.
  - `authed`: children 렌더 + 로그아웃 버튼.
- 데모 모드:
  - `isDemoMode()`가 참이면 로그인 없이 children을 통과시킨다.
  - 상단에 "체험 모드" 배지와 "체험 종료" 버튼을 표시한다.
  - 종료 시 `exitDemoMode()`, `resetDemo()`, `/` 이동을 수행한다.

## Flutter 동등성 기준

- Flutter도 Supabase 세션을 앱 전역 lifecycle로 관리하고, 보호 화면 진입 전 `loading | authed | anon`과 동일한 세 가지 상태를 표현해야 한다.
- 이메일 OTP는 매직링크가 아니라 "코드 입력 후 verify" 흐름이어야 한다.
- Google OAuth 복귀 후 개인 우주 화면으로 들어가는 결과가 같아야 한다.
- 보호 RPC 호출 전 access token을 읽어 `Authorization: Bearer <token>`을 붙여야 한다.
- 계정 전환/로그아웃 시 우주 그래프, 일기 목록, 로컬 demo/cache 성격의 데이터가 이전 사용자에서 다음 사용자로 새지 않아야 한다.

## 수용 기준

1. 미인증 사용자가 `/universe`, `/admin`, `/gift/$token`에 진입하면 사인인 화면을 본다.
2. `/`와 `/u/$slug`는 세션 없이 열린다.
3. 이메일 OTP 검증 또는 Google OAuth 복귀 후 `SessionGate`는 children을 렌더한다.
4. 새로고침 후 저장된 세션이 복원되면 `authed` 상태가 유지된다.
5. `getAccessToken()`은 유효 세션에서 비어 있지 않은 access token을 반환한다.
6. sign-out 또는 사용자 변경 후 이전 계정의 우주 데이터가 남지 않는다.
7. Supabase env가 비어 있으면 무한 로딩이 아니라 부팅 오류로 드러난다.
