# 37. 두 우주 겹쳐보기 (universe-overlay)

> 친구의 공개 우주(35) 위에 **내 우주를 겹쳐 띄운다** — 두 별무리가 한 화면에 떠 있고, 공명(36)으로 이어진 별 쌍 사이에 **두 우주를 가로지르는 빛의 다리**가 놓인다. 같은 사건이 두 우주에서 어떻게 *다르게 빛나고 다르게 변해 왔는지*를 한 장면에서 본다. concept §함께한 기억의 구현 형태 (b)("두 사람이 공유하는 겹침 공간")의 실현. | 범위: FE(+BE read 1 rpc) | 선행: 35, 36, 39(P2 navigation·P1 focus — 제어흐름 선행) | 차단: 없음 (소셜 트랙 캡스톤)

## 목적 (Why)

- **공명의 보상 장면이다.** 36까지는 공명이 "내 별의 마커 + 상대 표시명"으로만 보인다. 두 우주를 실제로 겹쳐 다리가 놓인 장면이, 별을 보내고 재작성하는 수고의 감정적 결실이다 — concept이 (a) 각자 우주의 연결과 (b) 겹침 공간 **두 방향 모두 비전에 포함**한다고 못 박은 그 (b).
- **차이를 보여주는 것이 핵심이다.** 공명 쌍은 같은 사건의 두 기억 — 색(감정)·밝기(회상 빈도)·형태(재성형)가 다르게 변해 왔다. 겹침 뷰는 그 *갈라짐*을 나란히 보여준다(회상의 재구성성 — 비전의 심장).

## 범위

- **포함:**
  - 방문 페이지(35 `/u/$slug`)에 **로그인 사용자용 "내 우주와 겹쳐보기" 토글**.
  - 겹침 렌더(`widgets/universe-overlay` 또는 `features/overlay` — FSD 배치는 구현 시 확정): 내 우주 + 친구 우주를 한 씬에 이중 배치, force-sim 2개(그래프별 독립), 친구 별무리는 구분 처리(틴트/오프셋).
  - **공명 다리**: `ShareService.GetResonanceBridges(slug)`(인증) — 호출자와 우주 주인 사이의 공명 쌍(내 memory_id ↔ 상대 SharedStar 인덱스)을 반환, FE가 Line2 빛줄기로 잇는다.
  - 다리 클릭/근접 → 두 별을 나란히 보는 카메라 + 비교 패널(내 쪽만 텍스트, 상대 쪽은 색·밝기·형태만).
- **비목표(Out of scope):**
  - **공유 편집·합성 우주 저장** — 겹침은 순간의 뷰일 뿐, 서버에 합성 상태를 만들지 않는다.
  - **상대 별 내용 열람** — 겹침 뷰에서도 상대 우주의 정보는 35의 `SharedStar`(콘텐츠 제로) 그대로다. 공명 상대 별이라도 *상대가 재작성한 텍스트는 보이지 않는다*(각자의 기억은 각자의 것 — 보고 싶으면 본인에게 듣는 것이 컨셉).
  - **3인 이상 겹침** — 두 우주만. N-way는 시각·성능 모두 폭발.
  - 실시간 동시 접속 표시("친구가 지금 보고 있어요") — 없음(unary 전용, 헌법6).
- ⚠️ 전제: 상대가 35 공유를 켜 둔 상태여야 한다(끄면 겹침도 불가 — NotFound 동일).

## 설계 요점

**공개 표면은 넓히지 않는다 — 다리 정보는 인증 rpc로.** 친구 우주 데이터는 35의 무인증 `GetSharedUniverse` 그대로 쓴다(콘텐츠 제로 유지). 공명 쌍 매칭에는 양쪽 식별이 필요한데 `SharedStar`엔 의도적으로 id가 없으므로, **인증 `ShareService.GetResonanceBridges(slug)`** 를 신설한다: 서버가 (호출자 user_id × slug 주인 user_id)의 `resonances`를 조회해 `{my_memory_id, their_star_index}` 쌍을 돌려준다 — `their_star_index`는 같은 시점의 `GetSharedUniverse` 응답 배열 인덱스(서버가 동일 정렬 기준으로 계산; 정렬 키를 share 스냅샷 조립과 공유해 인덱스 표류 방지). 공명이 없으면 빈 배열 — **공명 당사자가 아닌 방문자는 다리를 영영 볼 수 없다**(36의 비노출 원칙 유지).

```proto
// share.proto에 추가(37 소유)
message ResonanceBridge { string my_memory_id = 1; int32 their_star_index = 2; }
message GetResonanceBridgesResponse { repeated ResonanceBridge bridges = 1; }
// rpc GetResonanceBridges(GetResonanceBridgesRequest{slug}) — 인증 ShareService, NO_SIDE_EFFECTS
```

**씬 구성 — 두 하늘이 맞닿는다.** 내 우주와 친구 우주를 **세로 축 오프셋으로 이격 배치**(예: 내 우주 아래, 친구 우주 위 — 수평선 너머 또 하나의 하늘). 각 우주는 자기 force-sim으로 좌표 창발(헌법3 — Worker 인스턴스 2개 또는 1 Worker 2 그래프, 구현 시 프로파일로 결정). 친구 별무리는 시각 구분: 친구의 30 설정 색을 유지하되 약한 공통 틴트/채도 보정으로 "남의 하늘" 느낌 — 정확한 값은 구현 중 튜닝(policy로 확정).

**공명 다리 — 우주 사이를 가로지르는 유일한 선.** 다리 = `my star 좌표 ↔ their star 좌표`를 잇는 Line2(09 재사용·전용 TSL — 느린 빛 흐름은 수동 uniform). 두 sim이 각자 틱마다 움직이므로 다리 끝점은 매 프레임 양쪽 좌표를 다시 읽는다. 다리 근접/클릭 → **navigation 머신**에 `FRAME_PAIR` 이벤트를 보내 두 별을 한 프레임에 담는 위치로 fly-to(=`framingPair` 상태, tag `transitioning`) + **비교 패널**: 내 별(조각 텍스트·감정·밝기) vs 상대 별(감정 색·밝기·형태 *시각 정보만* — 텍스트 없음). 쌍 선택은 **focus 머신의 이중 선택 상태 `pairFocus{ myId, theirIndex }`** 로 표현한다(단일 `starFocus`와 배타 — 한 번에 하나).

**겹침은 뷰 상태일 뿐 — navigation 머신의 `overlay` 상태(전제: spec 39 done).** 겹쳐보기 on = navigation 머신을 `overlay`로 진입(기존 `nebula`/`recall`과 배타), 토글 off → 친구 우주만(35 원래 화면, view-only 복귀). **`overlay` 상태에서는 쓰기 RPC가 게이트로 막힌다**(회상 강화 비활성 — boolean 플래그가 아니라 *상태로* 단순화). 모든 데이터는 읽기.

**39 머신에 가산적으로 얹는다.** overlay·다리 fly-to는 navigation 머신에 상태/이벤트를 *추가*하고(골격 재설계 아님 — [state-machines §4](../state-machines.md)의 비행 상태와 동형), 공명쌍 선택은 focus 머신에 이중 선택 상태를 추가한다. 따라서 37의 **제어흐름은 39 P2(navigation)·P1(focus) 확정 이후** 작성한다 — 단 렌더·force-sim·다리 시각 레이어(InstancedMesh·Line2·2그래프)는 머신과 무관하므로 **P2와 병행 가능**(손실 0). ⚠️ 39 P2/P1 설계 시 overlay 모드(`mode` 3-way vs 직교 플래그)·focus 이중 선택을 *입력으로* 반영해 둔다(나중 재방문 방지).

**성능.** 별 2배·시냅스 2배 + 다리 소수 — InstancedMesh 2개·Line2 배치로 헌법8 전제 내. 베타 규모(별 수백 개×2)에선 여유; sim 2개의 Worker 비용만 프로파일.

## 수용 기준 (EARS / Given-When-Then)

1.1 WHEN 로그인 사용자가 친구의 `/u/:slug`에서 겹쳐보기를 켜면 THE SYSTEM SHALL 내 우주와 친구 우주를 한 씬에 이격 배치해 렌더하고, 각 우주의 좌표는 각자의 force-sim에서 창발한다(서버 좌표 없음 — 헌법3).
1.2 WHEN 비로그인 방문자가 같은 페이지를 보면 THE SYSTEM SHALL 겹쳐보기 토글을 노출하지 않는다(35 화면 그대로).
2.1 WHEN 호출자와 우주 주인 사이에 공명(36)이 있으면 THE SYSTEM SHALL `GetResonanceBridges`가 그 쌍을 반환하고, FE가 두 별을 잇는 다리를 렌더한다.
2.2 WHEN 호출자가 공명 당사자가 아니면 THE SYSTEM SHALL 빈 배열을 반환한다 — 다리·공명 존재가 제3자에게 노출되지 않는다.
2.3 WHEN 다리를 클릭/근접하면 THE SYSTEM SHALL 두 별을 한 프레임에 담는 카메라로 이동하고, 비교 패널에 내 별의 텍스트·감정과 **상대 별의 시각 정보만**(텍스트 없음)을 표시한다.
3.1 WHEN 겹침 뷰가 떠 있는 동안 THE SYSTEM SHALL 어떤 쓰기 RPC도 호출하지 않는다(회상 강화 포함 — 순수 뷰).
3.2 WHEN 상대가 공유(35)를 끄면 THE SYSTEM SHALL 방문·겹침 모두 균일 NotFound로 막힌다.
3.3 WHEN 두 sim이 틱마다 움직여도 THE SYSTEM SHALL 다리 끝점이 매 프레임 양쪽 별 좌표를 따라간다(떨어진 다리 없음).
