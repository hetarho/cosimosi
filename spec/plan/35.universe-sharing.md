# 35. 우주 공개 - ShareService / VisitService 공개 풍경 (as-built)

> 현재 구현 기준 문서. 내 우주를 URL 하나로 공개하되 공유되는 건 일기 텍스트가 아니라 풍경(별 배치·색·시냅스·하늘색)뿐인 공개 표면의 SSOT다. | 범위: FS | 상태: 구현 완료, 반영: 37(겹쳐보기 — ShareService에 GetResonanceBridges·repo ListStarIDs 추가)·36(display_name 재사용)

## 목적

일기는 사적이지만 우주는 보여주고 싶다. 공유 단위를 *내용*이 아니라 *풍경*으로 못 박으면 프라이버시를 깨지 않고 cosimosi의 핵심 시각 경험을 나눌 수 있다. 35는 가장 단순한 공개 프리미티브 하나만 만든다: 소유자가 켜면 추측 불가능한 슬러그(`/u/:slug`)가 생기고, 누구나 그 링크로 우주를 읽기 전용으로 거닌다. 일기 본문·조각 텍스트는 어떤 경로로도 나가지 않는다. 친구 계정 그래프 없이도 링크 전달 자체가 사실상의 1:1 공유다(36 공명·37 겹쳐보기가 이 위에 얹힌다). 공개 우주는 비로그인 방문자가 처음 보는 cosimosi이므로 방문 화면의 "나의 우주 만들기" CTA가 유입 경로가 된다.

값 SSOT는 `spec/policy/domain/sharing.md`다.

## 현재 RPC 표면

공개 계약은 `proto/cosimosi/v1/share.proto`에 있고, **노출 수위가 의도적으로 다른 두 service**로 나뉜다.

- `VisitService` — **무인증** 공개 읽기. RPC 단 하나.
  - `GetSharedUniverse(slug)`: 공개 우주 풍경 스냅샷. `NO_SIDE_EFFECTS`.
- `ShareService` — **인증** 소유자 설정. `user_id = JWT sub`로 자기 행만 읽고 쓴다.
  - `GetShareSettings()`: 소유자 공유 설정 조회. `NO_SIDE_EFFECTS`.
  - `UpdateShareSettings(enabled, display_name)`: on/off + 표시명. 최초 켜기에 슬러그 생성.
  - `RotateShareSlug()`: 새 슬러그 발급, 옛 URL 즉시 무효.
  - `GetResonanceBridges(slug)`: 겹쳐보기(spec 37 소유) 공명 다리. `NO_SIDE_EFFECTS`.

`GetResonanceBridges`는 37이 소유하지만 service 위치는 35의 `ShareService`다 — 호출자 신원이 있어야 공명을 매칭하므로 무인증 `VisitService`에 둘 수 없다(공개 표면은 콘텐츠 제로 유지). 공명 정보 자체는 36 소관이다.

## 공개 표면 격리 — fail-closed

기존 auth 인터셉터는 전 RPC 공통이다. 절차 allowlist로 구멍을 내는 대신 **공개 RPC를 별도 service로 분리**해, 그 핸들러에만 auth 없는 인터셉터 체인을 장착한다(`backend/internal/platform/rpcserver/server.go`).

- 공통 인증 체인(`opts`): `WithReadMaxBytes(256KiB)` + `WithInterceptors(Logging, Auth)` + `WithRecover`. `MemoryService`·`SettingsService`·`ShareService`·`GiftService`가 이 체인을 그대로 쓴다. `AdminService`는 여기에 admin allowlist 게이트(spec 34)를 한 겹 더 얹는다.
- 무인증 체인(`visitOpts`): `WithReadMaxBytes(256KiB)` + `WithInterceptors(Logging)` + `WithRecover` — **Auth 인터셉터만 빠진 별도 슬라이스**. `VisitService`만 이 체인으로 마운트한다.
- 257번 줄 `opts`(auth 포함)는 공유하고, 88번 줄 `visitOpts`는 새 슬라이스로 만든다 — auth가 실린 슬라이스를 alias하지 않는다. allowlist 오타로 다른 RPC가 새는 구조적 가능성이 없다(공개 표면을 넓히려면 service 자체를 새로 마운트해야 한다).
- 17의 요청 한도(256KiB read cap)·서버 타임아웃(read/write 30s, idle 120s)은 무인증 표면에도 동일 적용된다.

`backend/internal/platform/rpcserver/server_test.go`의 `TestUnauthBoundary`가 이를 회귀 가드한다: 토큰 없이 Memory·Settings·Admin·Share·Gift는 `Unauthenticated`, `VisitService.GetSharedUniverse`만 핸들러에 도달한다.

핸들러 한 구조체(`backend/internal/share/handler.go`의 `Handler`)가 `ShareServiceHandler`·`VisitServiceHandler` 둘 다 구현한다 — auth 경계는 핸들러가 아니라 마운트 체인이 강제한다. `cmd/api/main.go`는 같은 `shareHandler`를 owner·public 두 인자로 모두 넘긴다. `VisitService` 핸들러는 의도적으로 `UserIDFromContext`를 호출하지 않는다(방문 체인엔 auth 인터셉터가 없다).

## 공개 스냅샷 — 콘텐츠 제로를 타입으로 보장

`GetUniverse`의 `Star`를 재사용하지 않는다 — `Star`는 앞으로도 필드가 자란다(record_id·fragment_text 등). 공개판은 전용 메시지로 풍경 렌더에 필요한 최소만 담는다.

- `SharedStar`: `mood`(13감정 → 색)·`intensity`(크기)·`last_recalled_day`·`created_day`. **식별자 없음** — 응답 배열의 인덱스가 곧 별 id다. **body·fragment_text·record_id·memory_id·정확 타임스탬프 필드가 아예 없다**(누락이 아니라 타입 수준 부재).
- `SharedSynapse`: `a`·`b`(SharedStar 배열 인덱스 쌍)·`weight`. 활성 시각·co_activation 같은 행동 신호는 없다.
- 좌표는 보내지 않는다(헌법3) — 방문자 브라우저의 force-sim이 재창발하므로 방문마다 배치가 미세하게 다른 건 의도된 자연스러움이다.

**타임스탬프 일 단위 양자화.** `share.go`의 `toEpochDay(t)`가 instant를 UTC epoch-day(`t.UTC().Unix() / 86400`)로 환원한다. 방문자는 별을 마지막으로 만진 *날*만 알 뿐 정확한 시각은 모른다(행동 핑거프린팅 둔화). zero time → 0. 같은 UTC 날의 00:01과 23:59는 같은 day로 무너지고 다음 날은 +1(`service_test.go` `TestSnapshot_QuantizesTimestampsToDay`).

**배열 순서는 `m.id` 정렬 — 시간순 아님.** `ListSharedStars`/`ListSharedStarIDs`는 `ORDER BY m.id`다(`backend/internal/db/queries/share.sql`). `created_at` 순으로 정렬하면 시각 필드를 일 단위로 양자화해 놓고도 응답 *배열 순서*가 일내 시간순·조각 묶임이라는 2차 채널로 샌다. `memories.id`는 `crypto/rand` 16바이트 → base64url(`backend/internal/memory/repository_pg.go`의 ID 생성과 동일 규약)이라 비시간성·결정적이다 — 순서는 호출마다 안정적이지만 어떤 연대기도 드러내지 않는다.

**시냅스 인덱스 매핑.** `service.go`의 `Snapshot`이 별 배열을 만들며 `id → 인덱스` 맵을 쌓고, 시냅스 끝점 id를 그 인덱스로 환원한다. 끝점이 별 집합에 없으면 그 시냅스를 버린다(id를 절대 새지 않게).

**요즘 하늘색(ambient, spec 25)은 서버가 계산하지 않는다.** proto `GetSharedUniverseResponse.ambient`(`AmbientMood`) 필드는 향후 서버 계산용으로 비워 둔다(zero value). 클라가 공개된 별들에서 `deriveAmbient`로 파생한다(데모/폴백과 같은 경로). `Snapshot` 도메인 구조체에는 ambient 필드 자체가 없다.

**소유자 시각 설정(spec 30)은 풍경의 일부.** `GetSharedUniverseResponse.appearance`는 메모리 proto의 `Settings`(테마·오브젝트·감정색)다. `Snapshot`은 `SettingsReader` 포트로 소유자 appearance를 best-effort 병합한다 — 설정 읽기가 실패하면 코어 그래프를 막지 않고 클라 기본값으로 degrade한다(`TestSnapshot_AppearanceDegradesOnError`).

## 슬러그 규칙

- **추측 불가가 1차 방어선:** `share.go`의 `newSlug()`가 `crypto/rand` 16바이트(128bit) → `base64.RawURLEncoding`으로 22자(패딩 없음). `slugBytes = 16`.
- **최초 켜기에 생성:** `UpdateSettings`에서 행이 없거나 slug가 ""일 때만 새로 발급. 기존 행은 slug를 보존하므로 끄고 다시 켜면 같은 URL 재사용(회전이 아닌 한).
- **회전:** `RotateSlug`가 새 slug + `rotated_at`을 UPDATE해 옛 URL을 즉시 무효화한다. 공개한 적 없는 user는 회전 대상이 없어 `ErrNotShared`(→ `FailedPrecondition`).
- **균일 NotFound:** `GetShareUserBySlug`는 `enabled = true`인 행만 돌려준다. 존재하지 않는 슬러그·꺼진 공유(`enabled=false`)·회전된 옛 슬러그를 **구분 없이 동일한 `ErrNotFound`(→ `NotFound`)**로 응답한다(존재 비노출). 빈 우주도 구분하지 않는다(`TestSnapshot_DisabledAndUnknownAreIndistinguishable`·`TestRotateSlug_InvalidatesOldURL`).
- slug `UNIQUE` 제약 충돌은 128bit 엔트로피상 사실상 불가능 — 발생 시 INSERT가 23505로 실패해 `CodeInternal`로 표면화된다(재시도 루프 없음).
- 표시명은 `strings.TrimSpace` 후 60 runes 상한(`maxDisplayNameRunes`). 초과는 `ErrDisplayNameTooLong`(→ `InvalidArgument`). 빈 값 = 익명("어느 우주").

## 도메인 모델 (internal/share)

- `Settings{Enabled, Slug, DisplayName}` — 소유자 설정. zero = "공개한 적 없음".
- `SharedStar{Mood, Intensity, LastRecalledDay, CreatedDay}` — 공개 별 투영. id/text/정확시각 없음(타입 보장).
- `SharedSynapse{A, B, Weight}` — 인덱스 쌍.
- `Snapshot{DisplayName, Stars, Synapses, Appearance}` — 조립된 공개 풍경(ambient 없음).
- `Appearance{Theme, StarObject, EmotionColors}` / `EmotionColor{Mood, Color}` — settings 컨텍스트를 import하지 않고 30 시각 설정을 미러(composition root가 `settings.Service`를 `SettingsReader`로 어댑트).
- `StarLandscape{ID, Mood, Intensity, LastRecalledAt, CreatedAt}` / `SynapseLandscape{AID, BID, Weight}` — repo 읽기 모델. ID는 서버에만 남고 DTO에 안 나간다.
- `ResonancePair{MyMemoryID, TheirMemoryID}` / `ResonanceBridge{MyMemoryID, TheirStarIndex}` — 37 겹쳐보기 다리. `ResonanceReader` 포트로 gift 컨텍스트의 공명을 받아(composition root 어댑트), 상대 memory id를 공개 스냅샷 인덱스(`ListStarIDs` 순서)로 환원한다. 비당사자 호출자에겐 빈 목록(관계 비노출).
- 도메인 타입은 db/json/proto 태그 없는 순수 값이다(헌법5).

## 데이터 스키마

`backend/internal/db/migrations/00007_universe_shares.sql`(spec 35):

```sql
CREATE TABLE universe_shares (
    user_id      TEXT PRIMARY KEY,
    slug         TEXT NOT NULL UNIQUE,        -- base64url 22자(128bit)
    enabled      BOOLEAN NOT NULL DEFAULT false,
    display_name TEXT NOT NULL DEFAULT '',    -- '' = 익명
    created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
    rotated_at   TIMESTAMPTZ
);
```

사용자당 1행 overrides-only. 행이 없으면 "공개한 적 없음"(`GetShareSettings`가 enabled=false·slug="" 폴백). 원본 일기·조각 텍스트는 어떤 컬럼으로도 담지 않는다(콘텐츠 제로, 헌법1). 마이그레이션 번호는 00007(00001 engram·00002 settings·00003 admin·00004 fragmentation·00005 reconsolidation·00006 nightly 다음). 36의 `star_gifts`/`resonances`가 00008이다.

## 주요 쿼리/저장소

- `backend/internal/db/queries/share.sql`(풍경 컬럼만 SELECT — 원본/조각 텍스트·record_id·정확 시각은 쿼리에 등장하지 않는다):
  - `GetShareByUser :one` — 소유자 설정(slug·enabled·display_name). 0행 → `pgx.ErrNoRows` → 폴백.
  - `UpsertShareSettings :one` — `ON CONFLICT (user_id)`로 enabled/display_name만 갱신(slug 보존).
  - `RotateShareSlug :one` — slug 교체 + rotated_at. 0행 → `ErrNotShared`.
  - `GetShareUserBySlug :one` — `WHERE slug = $1 AND enabled = true`. 균일 NotFound의 핵심.
  - `ListSharedStars :many` — `m.id, m.mood, m.intensity, m.last_recalled_at, m.created_at FROM memories WHERE user_id ORDER BY m.id`.
  - `ListSharedStarIDs :many` — `m.id` only, 같은 WHERE·`ORDER BY m.id`(37 다리 인덱스 정합 필수).
  - `ListSharedSynapses :many` — `ml.a_id, ml.b_id, ml.weight FROM memory_links WHERE user_id`(활성 시각·co_activation 제외).
- `backend/internal/share/repository_pg.go` — sqlc/pgx 구현. `pgx.ErrNoRows`를 ok=false로 변환(never-shared / nothing-to-rotate / unknown-slug).
- composition root(`backend/cmd/api/main.go`):
  - `shareSettingsAdapter`: `settings.Service` → `share.SettingsReader`(share가 settings를 import하지 않게).
  - `shareResonanceAdapter`: `gift.Service` → `share.ResonanceReader`. **포인터 어댑터** — share↔gift가 배선 순환(share는 gift 공명이, gift는 share display_name이 필요)이라 share를 먼저 만들고 gift를 만든 뒤 `inner`를 바인딩한다.
  - `giftShareAdapter`: `share.Service` → `gift.ShareReader`(36이 display_name·slug 재사용).

## 라우트 / FE

- `frontend/src/app/router.tsx`: `/u/$slug`는 `rootRoute` 직속 공개 라우트 — **SessionGate 밖**. `lazyRouteComponent(() => import('@/pages/visit'), 'VisitPage')`로 코드 스플릿(방문 전용 화면이 메인 번들에 안 실리게).
- `frontend/src/pages/visit/ui/VisitPage.tsx`: 우주 캔버스를 읽기 전용으로 재사용. 기록 폼·회상 패널·시뮬 패널 없음. 별을 눌러도 어떤 일기·조각 텍스트도 뜨지 않는다(바라보기·fly-to만 — `SharedStar`엔 텍스트가 없다). 상단 헤더는 `display_name`("○○의 우주" / 익명이면 "어느 우주"), 하단에 비로그인 CTA "나의 우주 만들기"(→ `/`). NotFound는 균일 안내 카드("링크가 만료됐거나 공개가 꺼졌을 수 있어요") + CTA. 렌더러 불가/로드 실패는 17 폴백 재사용(`CanvasErrorFallback`·`RendererUnavailableError`).
  - **방문자 시각 설정 보존:** 첫 렌더에 lazy init으로 방문자 appearance를 1회 캡처하고, 스냅샷 도착 시 소유자 풍경을 적용했다가, 언마운트 시 방문자 값을 복원한다(소유자 색이 방문자 기기 localStorage에 눌러앉지 않게).
  - slug가 바뀌면 렌더 store(별·시냅스·focus·loadedEmpty)를 비운다(직전 소유자 잔상 제거).
  - 겹쳐보기 UI(토글·`UniverseOverlay`·`OverlayComparePanel`·`GetResonanceBridges`·내 우주/내 일기 로드)는 **spec 37 소유**다 — 로그인 + 토글 on에서만 동작하고 비로그인엔 토글 자체를 노출하지 않는다.
- `frontend/src/pages/visit/api/visit-queries.ts`:
  - **전용 무인증 transport:** `createConnectTransport({ baseUrl })`(인증 인터셉터 없음) — 방문 표면이 토큰을 절대 싣지 않는다.
  - **POST(unary) 사용, GET 금지:** `useHttpGet`을 쓰지 않는다. GET이면 브라우저/CDN HTTP 캐시가 응답을 보관해 소유자가 끄거나 회전한 뒤에도 옛 슬러그가 캐시에서 그려질 수 있다(즉시 무효화 위반). POST는 캐시되지 않아 매번 서버가 권위.
  - `sharedUniverseQueryOptions(slug)`: `staleTime 0 + gcTime 0 + refetchOnMount 'always' + retry false` — 캐시하지 않는다(즉시 무효화가 프라이버시 계약). NotFound는 재시도 안 함.
  - `mapSharedUniverse(res)`: PURE 매핑(store 미경유, 헌법4). `SharedStar`엔 id가 없으므로 인덱스로 합성 id `shared-N` 부여. valence/relevance/조각·재성형 메타는 중립값. `resonant: false` 고정(공개 우주엔 공명 비노출). day → ms(`DAY_MS = 86_400_000`) 환원해 밝기/반지름 파생. 시냅스는 범위 밖/self-loop 방어 필터 후 `a<b` 정규화.

## Flutter 동등성 기준

- 무인증 공개 표면은 `VisitService.GetSharedUniverse(slug)` 하나뿐이다. Flutter도 토큰을 싣지 않는 별도 공개 transport로 호출하고, 인증 클라이언트와 섞지 않아야 한다.
- 공개 응답은 콘텐츠 제로다: `SharedStar`엔 id·텍스트·정확 시각이 없고 타임스탬프는 일 단위(epoch days), 별 식별은 배열 인덱스, 시냅스는 인덱스 쌍이다. Flutter도 day → 시간 환원으로 밝기/형태를 파생하고 좌표는 자체 force-sim으로 재창발해야 한다.
- 끄기·회전 시 옛 URL은 즉시 NotFound여야 하므로 공개 응답을 HTTP 캐시(GET)로 보관하면 안 된다 — POST(unary)로 매번 서버에 재검증한다.
- 미존재/꺼짐/회전 슬러그는 구분 불가능한 동일 NotFound다.
- 방문은 순수 읽기 — 회상 강화·기록 등 쓰기 RPC를 한 건도 호출하지 않는다. 별을 눌러도 텍스트를 보이지 않는다.
- 소유자 설정(`ShareService`)은 인증 전용이고 `user_id = JWT sub`로 자기 행만 다룬다. 최초 켜기에 슬러그 생성, 회전은 즉시 무효, 표시명 60자 상한.
- 요즘 하늘색은 서버가 안 주므로 공개 별들에서 클라가 파생한다.

## 수용 기준

1. `GetSharedUniverse` 응답은 일기 본문·조각 텍스트·memory_id·record_id·일 단위보다 정밀한 타임스탬프를 포함하지 않는다(proto에 필드 부재).
2. 공유가 꺼져 있거나 슬러그가 존재하지 않거나 회전돼 무효이면 구분 불가능한 동일 `NotFound`를 반환한다.
3. `RotateShareSlug`는 새 슬러그를 발급하고 이전 슬러그는 즉시 `NotFound`가 된다.
4. 공유를 처음 켜면 `crypto/rand` 기반 128bit(base64url 22자) 슬러그가 생성되고, 끄고 다시 켜면 같은 슬러그를 재사용한다.
5. 공개 스냅샷의 별 배열은 `m.id` 순(비시간성·결정적)으로 정렬돼 생성 순서가 2차 채널로 새지 않는다.
6. 같은 UTC 날의 다른 시각은 동일한 `created_day`/`last_recalled_day`로 양자화되고 다음 날은 +1이다.
7. 비로그인 사용자가 `/u/:slug`(enabled)에 접속하면 소유자의 시각 설정과 클라 파생 하늘색이 적용된 우주가 force-sim으로 배치돼 렌더된다(응답에 좌표 없음).
8. 방문자가 별에 다가가거나 클릭해도 어떤 일기·조각 텍스트도 표시하지 않고, 쓰기 RPC를 호출하지 않는다.
9. 방문 페이지는 비로그인 상태면 "나의 우주 만들기" CTA를 표시한다.
10. 방문 응답은 HTTP 캐시 대상이 아니라(POST), 끄기·회전 직후 옛 슬러그가 캐시에서 그려지지 않는다.
11. `VisitService` 외 모든 RPC를 무인증으로 호출하면 `Unauthenticated`를 반환한다(공개 표면 = `GetSharedUniverse` 하나).
12. `ShareService`는 호출자 JWT sub의 행만 읽고 쓴다(교차 사용자 접근 경로 없음).
13. 방문 페이지를 떠나면 방문자 자신의 시각 설정이 복원돼 소유자 풍경이 방문자 기기에 남지 않는다.
14. 공개 응답을 조립할 때 소유자 시각 설정 읽기가 실패하면 코어 그래프를 막지 않고 클라 기본값으로 degrade한다.
