# 35. 우주 공개 (universe-sharing)

> 내 우주를 **URL 하나로 공개**한다 — 공유되는 것은 일기 텍스트가 아니라 **우주의 풍경**(별 배치·색·성단·시냅스·하늘색)뿐이다. 소유자가 켜면 추측 불가능한 슬러그(`/u/:slug`)가 생기고, 누구나 그 링크로 우주를 **읽기 전용으로 거닐 수** 있다. 일기 본문·조각 텍스트는 어떤 경로로도 나가지 않는다. | 범위: FS | 선행: 16 (권장: 21·25 이후 — 조각 별·하늘색까지 풍경이 완성된 뒤) | 차단: 37

## 목적 (Why)

- **일기는 사적이지만, 우주는 보여주고 싶다.** concept §우주 공유 — "남에게 글을 들키지 않고도 '내 마음의 모양'을 보여줄 수 있다." 공유 단위를 *내용*이 아니라 *풍경*으로 못 박으면, 프라이버시를 깨지 않고 cosimosi의 핵심 시각 경험을 나눌 수 있다.
- **URL 공개가 친구 시스템보다 먼저다.** concept은 "친구 간 1:1·소규모 공유"를 말하지만, 계정 간 친구 그래프 없이도 **링크 공유 = 사실상의 친구 공유**다(링크를 준 사람에게만 보임). 친구·수락 개념은 36(공명)이 소유하고, 35는 가장 단순한 공개 프리미티브(슬러그 URL)만 만든다 — 36·37이 이 위에 얹힌다.
- **성장 루프.** 공개 우주는 비로그인 방문자가 처음 보는 cosimosi다 — 방문 화면 하단의 "나의 우주 만들기" CTA가 자연스러운 유입 경로가 된다.

## 범위

- **포함:**
  - proto 신설 `share.proto` — 인증 `ShareService`(공유 설정 3 rpc) + **무인증 `VisitService`(`GetSharedUniverse` 1 rpc — 공개 표면을 service 단위로 격리)**.
  - DB 마이그레이션 `universe_shares`(user당 1행: slug·enabled·display_name).
  - `internal/share` 바운디드 컨텍스트 — 설정 CRUD·슬러그 생성/회전·**공개 스냅샷 조립(콘텐츠 제로 보장)**.
  - FE 공개 방문 페이지 `/u/$slug`(SessionGate 밖·읽기 전용 우주) + 내 우주의 공유 설정 UI(`features/share-universe`).
- **비목표(Out of scope):**
  - **별 단위 공개 수위(제목만/일부/비공개)** — concept §우주 공유의 per-star disclosure는 후속. 35는 전역 한 단계: *모든 별의 내용 비공개, 풍경 전체 공개*.
  - **친구 계정 그래프·팔로우·피드** — 없음(concept 비-목표 "공개 소셜 네트워크 아님"). 공유는 링크 전달로만.
  - **방문자의 회상·강화·기록** — 방문은 순수 읽기. 회상 강화(11)·기록(10)은 소유자 전용.
  - **공명 마커의 공개 노출** — 제3자 방문자에게 공명 존재를 광고하지 않는다(36·37 소관 — 공명은 각자 자기 우주와 37 겹침 뷰에서만).
  - 방문 통계·조회수 — PostHog 이벤트 1개로 충분(대시보드는 34 후속).

## 설계 요점

**공개 표면 격리 — 무인증은 `VisitService` 하나뿐.** 기존 auth 인터셉터는 전 RPC 공통이므로, 절차 allowlist로 구멍을 내는 대신 **공개 rpc를 별도 service로 분리**해 그 핸들러에만 auth 없는 체인(recover→logging)을 장착한다 — allowlist 오타로 다른 rpc가 새는 구조적 가능성을 차단(fail-closed). `ShareService`(설정)는 기존 인증 체인 그대로.

**슬러그 — 추측 불가가 1차 방어선.** `crypto/rand` 16바이트 → base64url 22자(128bit). 끄면(`enabled=false`) 즉시 NotFound, **회전(rotate)하면 옛 URL 즉시 무효**(슬러그 교체). 존재하지 않는 슬러그·꺼진 공유·빈 우주 구분 없이 동일한 NotFound를 반환한다(존재 비노출). 17의 요청 한도가 무인증 표면에도 적용되는지 확인한다.

**공개 스냅샷 — 전용 DTO로 콘텐츠 제로를 타입으로 보장.** `GetUniverse`의 `Star`를 재사용하지 않는다 — `Star`는 앞으로도 필드가 자란다(`record_id`·`fragment_text` 등). 공개판은 **`SharedStar`/`SharedSynapse` 전용 메시지**로, 풍경 렌더에 필요한 최소만 담는다:

```proto
message SharedStar {            // 식별자 없음 — 응답 내 인덱스가 곧 id
  string mood = 1;              // 13감정(색)
  float intensity = 2;
  int64 last_recalled_day = 3;  // ⚠️ 일 단위 양자화(행동 핑거프린팅 방지)
  int64 created_day = 4;        //   감쇠·형태 계산용 — 시각이 아니라 날짜만
}
message SharedSynapse { int32 a = 1; int32 b = 2; float weight = 3; }  // SharedStar 인덱스 참조
message GetSharedUniverseResponse {
  string display_name = 1;            // 소유자가 정한 표시명('' = 익명)
  repeated SharedStar stars = 2;
  repeated SharedSynapse synapses = 3;
  Appearance appearance = 4;          // 30의 시각 설정(배경·감정색·오브젝트) — 풍경의 일부
  Ambient ambient = 5;                // 25 완료 시(요즘 하늘색) — 미완이면 미설정
}
service VisitService {                // ⚠️ 무인증 — 이 service에 rpc를 더할 땐 공개 가능성만
  rpc GetSharedUniverse(GetSharedUniverseRequest) returns (GetSharedUniverseResponse) { option idempotency_level = NO_SIDE_EFFECTS; }
}
service ShareService {                // 인증 — 소유자 설정
  rpc GetShareSettings(GetShareSettingsRequest) returns (GetShareSettingsResponse) { option idempotency_level = NO_SIDE_EFFECTS; }
  rpc UpdateShareSettings(UpdateShareSettingsRequest) returns (UpdateShareSettingsResponse); // enabled·display_name(최초 켜기에 슬러그 생성)
  rpc RotateShareSlug(RotateShareSlugRequest) returns (RotateShareSlugResponse);
}
```

`memory_id`·`record_id`·조각 텍스트·정확한 타임스탬프는 **메시지에 필드 자체가 없다** — 누락이 아니라 타입 수준 보장. 시냅스는 인덱스 쌍으로만 잇는다. 좌표는 여전히 안 보낸다(헌법3 — 방문자 브라우저의 force-sim이 창발; 방문마다 미세하게 다른 배치는 의도된 자연스러움).

**DB — `universe_shares`(user당 1행, overrides-only).**

```sql
CREATE TABLE universe_shares (
  user_id      UUID PRIMARY KEY,
  slug         TEXT NOT NULL UNIQUE,        -- base64url 22자
  enabled      BOOLEAN NOT NULL DEFAULT false,
  display_name TEXT NOT NULL DEFAULT '',    -- '' = 익명("어느 우주")
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  rotated_at   TIMESTAMPTZ
);
```

⚠️ 마이그레이션 번호는 **착수 시점의 다음 번호**(잠정 `00007` — 00.overview §공유 설계 결정의 할당이 단일 기준, 27의 `00006`이 생략되면 한 칸 당겨 재확정).

**FE — 방문 페이지는 우주 캔버스의 읽기 전용 조립.** `/u/$slug`는 SessionGate **밖**의 공개 라우트. `pages/visit`이 home과 같은 entities(star·synapse·appearance)·force-sim을 재사용하되: 기록 폼·회상 패널·시뮬 패널 없음, 별 근접 시 **어떤 텍스트도 뜨지 않음**(fly-to·바라보기만), 상단에 `display_name`("○○의 우주" / 익명이면 "어느 우주"), 하단에 비로그인 CTA "나의 우주 만들기"(→ 랜딩/사인인). 렌더러 불가 폴백은 17의 기존 안내 재사용.

**FE — 공유 설정.** `features/share-universe` — 토글(켜기/끄기)·표시명 입력·URL 복사·회전 버튼("이전 링크는 끊겨요" 경고). 진입점은 31의 오버레이 셸 패턴(우주 위 모달 — 31 미완이면 임시로 home 내 간단한 패널, 31 완료 시 셸로 흡수).

## 수용 기준 (EARS / Given-When-Then)

1.1 WHEN `GetSharedUniverse` 응답을 검사하면 THE SYSTEM SHALL 일기 본문·조각 텍스트·`memory_id`·`record_id`·일 단위보다 정밀한 타임스탬프를 포함하지 않는다(proto에 필드 부재 — grep·메시지 정의로 검증).
1.2 WHEN 공유가 꺼져 있거나(enabled=false) 슬러그가 존재하지 않으면 THE SYSTEM SHALL 구분 불가능한 동일 `NotFound`를 반환한다.
1.3 WHEN `RotateShareSlug`를 호출하면 THE SYSTEM SHALL 새 슬러그를 발급하고 이전 슬러그는 즉시 `NotFound`가 된다.
1.4 WHEN 공유를 처음 켜면 THE SYSTEM SHALL `crypto/rand` 기반 128bit(base64url 22자) 슬러그를 생성한다.
2.1 WHEN 비로그인 사용자가 `/u/:slug`(enabled)에 접속하면 THE SYSTEM SHALL 소유자의 시각 설정·하늘색이 적용된 우주를 렌더하고, 별·시냅스가 force-sim으로 배치된다(좌표는 응답에 없음 — 헌법3).
2.2 WHEN 방문자가 별에 다가가거나 클릭해도 THE SYSTEM SHALL 어떤 일기·조각 텍스트도 표시하지 않으며, 회상 강화·기록 등 쓰기 RPC를 호출하지 않는다.
2.3 WHEN 방문 페이지가 비로그인 상태면 THE SYSTEM SHALL "나의 우주 만들기" CTA를 표시한다.
3.1 WHEN `VisitService` 외의 모든 기존 RPC를 무인증으로 호출하면 THE SYSTEM SHALL 기존대로 `Unauthenticated`를 반환한다(공개 표면 = `GetSharedUniverse` 단 하나).
3.2 WHEN `ShareService`를 다른 사용자의 설정에 대해 호출하면 THE SYSTEM SHALL 자신의 행만 읽고 쓴다(user_id = JWT sub — 교차 접근 경로 없음).
