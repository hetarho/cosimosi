# 26. 관련성·감정 가중 망각 - 변조 감쇠 λ_eff (as-built)

> 현재 구현 기준 문서. 단일 전역 λ(순수 시간 감쇠) 대신 별마다 변조되는 `λ_eff`로 별 밝기를 감쇠시키는 모델의 SSOT다 — 연결 많고·요즘과 닿고·감정 강한 별은 천천히, 고립된 저강도·요즘 무관 별은 빨리 어두워지되 바닥 아래로는 내려가지 않는다. | 범위: FS (FE 순수 model + 서버 relevance 계산 + proto Synapse 노출) | 상태: 구현 완료

## 목적

concept.md §망각 모델의 명제 — "망각은 시간만이 아니라 관련성의 함수다" — 를 실제 밝기 계산으로 구현한다. 12(decay-dormant)는 의도적으로 단일 λ(반감기 30일)만 썼다. 26은 그 단일 λ 자리에 **별마다 변조되는 `λ_eff`** 를 끼운다: 연결 많은 별(`R_conn`), 요즘 토픽과 가까운 별(`R_recent`), 감정이 강렬한 별(`R_emo`)은 천천히 어두워지고, 고립된 저강도·저정서 일회성 기억은 빨리 잠든다. 12의 불변 — `exp(-λΔt)` 결정론적 렌더 계산·밝기 바닥·"별·시냅스 삭제 금지(헌법2)" — 는 그대로다. 26은 λ를 변조할 뿐 **삭제를 구동하지 않으며**, 바닥은 12와 동일한 `A_MIN=0.05` 단일 바닥이다(랜딩 데모 카드가 쓰던 0.12는 시연용 값이었을 뿐 별도 바닥이 아니다).

`R_conn`·`R_emo`의 입력(degree·intensity·valence)은 클라가 이미 가진 값으로 클라가 계산한다. `R_recent`의 입력 `relevance`만은 별 임베딩이 서버 권위라 **서버가 `GetUniverse`에서 cos로 계산해** `Star.relevance`로 내려보낸다 — weight와 동급의 의미 그래프 파생값이므로 헌법3(밝기·좌표는 클라 계산)을 위반하지 않는다. 밝기 자체(`exp(-λ_eff·Δt)`)는 여전히 클라가 렌더 시 계산한다.

## 현재 구현

### FE 변조 감쇠 model (순수 — `three`/React/DOM 미의존, 헌법4)

`frontend/src/entities/memory/model/activation.ts`. 12가 정의한 `LAMBDA`(=ln2/30 ≈ 0.0231/day)·`A_MIN`(0.05)·`HALF_LIFE_DAYS`·`activation`·`starBrightness`·`synapseBrightness`·`isDormant`는 그대로 두고(재정의 없음), 26이 같은 파일 하단에 변조 변형과 계수만 추가한다.

- 계수 상수(튜닝 SSOT는 policy, 아래 §정책 참조):
  - `ALPHA_CONN = 0.6` — `R_conn` 계수(연결).
  - `BETA_RECENT = 0.5` — `R_recent` 계수(요즘 관련성).
  - `GAMMA_EMO = 0.7` — `R_emo`의 각성(intensity) 계수.
  - `DELTA_VAL = 0.4` — `R_emo`의 음 정서가 계수(강한 부정일수록 저항↑, Kensinger & Corkin 2004).
- `lambdaEff(degreeNorm, relevance, intensityNorm, valence)`: `λ_eff = LAMBDA · R_conn · R_recent · R_emo`.
  - `R_conn = 1/(1 + ALPHA_CONN·max(0, degreeNorm))`.
  - `R_recent = 1/(1 + BETA_RECENT·clamp01(relevance))`.
  - `R_emo = 1/(1 + GAMMA_EMO·clamp01(intensityNorm) + DELTA_VAL·max(0, -valence))`. valence는 [-1,1]로 clamp 후 음수부만 가산항으로 쓴다.
  - 각 `R ∈ (0,1]`이라 변조는 감쇠를 **늦추기만** 한다(`λ_eff ≤ LAMBDA`, 가속 없음). 모든 입력 clamp로 음수 degree·intensity>1·|valence|>1에도 `λ_eff>LAMBDA`나 밝기>1이 나오지 않는다.
- `modulatedBrightness(lastRecalledAt, now, degreeNorm, relevance, intensityNorm, valence)`: `A_MIN + (1-A_MIN)·exp(-λ_eff·Δt_days)` ∈ `[A_MIN, 1]`. Δt=0이면 1, Δt→∞이면 정확히 `A_MIN`에서 멈춘다(0 아님·삭제 없음, 헌법2). `starBrightness`의 drop-in 대체이며 23 재성형 오프셋은 호출처(StarField)가 `reshapedBrightness`로 위에 합성한다.
- `clamp01`은 내부 헬퍼다(비export).

### FE degree 정규화 (synapse store)

`frontend/src/entities/synapse/model/store.ts`.

- `degreeNormById(edges): Map<string, number>` — `R_conn` 입력. 별 id → degree(인접 엣지 수), 우주 전체 degree의 **중앙값**으로 정규화(`degree/median`). 중앙값 0(엣지 없음) 폴백 1로 비율을 유한하게 유지한다. 엣지 없는 별은 맵에 없어 호출처가 0으로 읽는다(연결 보너스 없음). 순수·three/DOM 미의존.
- `neighborsOf(edges, memoryId)`는 12/항행에서 쓰던 셀렉터로, degree 계산은 `degreeNormById`가 한 패스로 자체 집계한다.

### FE 링크 활력 (synapse mapping/types)

`frontend/src/entities/synapse/model/types.ts` — `SynapseEdge.coActivationCount: number`(서버 `memory_links.co_activation_count`를 그대로 운반).

`frontend/src/entities/synapse/model/mapping.ts`:

- `toSynapseEdge`는 `UniverseSynapse.coActivationCount`를 운반한다(서버 미노출·데모·구버전 응답은 `?? 0`).
- `vitality(e) = 0.12·min(1, log2(1 + coActivationCount)/4)` ∈ `[0, 0.12]` — 누적 공동 회상이 많을수록 "살아있는" 연결. log 압축이라 처음 몇 번이 가장 크게 기여한다. 0이면 기존 시각과 동일.
- `pulseAmp(e) = min(1, reinforcedRecency + vitality(e))` — 자주 함께 떠올린 링크는 갓 강화되지 않아도 은은한 펄스 베이스라인을 유지한다.

### FE 렌더 배선 (StarField)

`frontend/src/entities/star/ui/StarField.tsx`. per-instance `aBrightness` attribute를 두 지점에서 변조 밝기로 채운다(매 프레임 재계산 금지 — 로드/회상 직후/우주 갱신 시에만 attribute 재기록, 헌법3·08):

- 빌드 `useLayoutEffect`(약 315행): `reshapedBrightness(modulatedBrightness(m.lastRecalledAt, now, degreeNorm.get(id) ?? 0, m.relevance, m.intensity, m.valence), m.brightnessOffset)`.
- 포커스 `useEffect`(약 364행): 같은 식으로 base를 재산출 후 focus 가중 곱을 얹는다.
- `degreeNorm`은 `degreeNormById(edges)`를 토폴로지 시그니처(`aId~bId` 조인)로 메모해 받는다 — 시간 머신이 밝기 재파생만을 위해 엣지 배열을 갈아끼울 때 빌드가 헛돌지 않게 한다.
- import는 `@/entities/memory/@x/star`(modulatedBrightness 등)와 `@/entities/synapse/@x/star`(degreeNormById)로, entities 교차는 `@x`만(헌법4).

### 서버 relevance 계산 (`R_recent` 입력)

`backend/internal/memory/memory.go` + `service.go` + `handler.go`. `R_recent`의 `relevance`만 서버 권위다.

- `RelevanceByStar(vectors []StarVector, now) map[string]float64`(memory.go): 별 임베딩을 **시간가중 중심 벡터("요즘 토픽")** 로 접고, 별마다 `relevance = clamp(cos(star_emb, centroid), 0, 1)`을 매긴다.
  - 가중치는 `AggregateAmbient`(25)와 같은 envelope `w = intensity·exp(-Δt/TauMoodDays)`(`TauMoodDays=7`). 미래 타임스탬프(시계 skew)는 Δt=0으로, 음수 intensity는 0으로 방어.
  - 중심 벡터는 채점 대상 별을 **포함**한다(요즘 별은 자기 토픽이라 on-topic — 이미 밝고, 항은 주로 오래된 별을 지배). leave-one-out은 27의 가능 정련.
  - 임베딩 미생성(embed 잡 대기) 별은 0점. 빈 우주·zero-norm 중심은 전부 0 → 클라 `R_recent`가 무변 1.0으로 폴드. cos는 스케일 불변이라 중심 벡터는 비정규화로 둔다.
- `cosineSim(a, b)`(memory.go): 길이 다르거나 한쪽 zero-norm이면 0.
- `StarVector{ID, Embedding []float64, Intensity, LastRecalledAt}` — 도메인 타입(태그 없음). repo가 sqlc 행을 매핑.
- `service.go GetUniverse`: ambient와 동일한 best-effort 패턴. `ListStarVectorsByUser` 읽기 실패해도 core graph를 실패시키지 않고 relevance를 전부 중립(0)으로 둔다. 성공 시 `memories[i].Relevance = rel[memories[i].ID]`로 별에 접고, handler가 `Star.relevance`로 매핑한다.
- `RelevanceByStar`는 `GetUniverse` 전용 — `ListDormant`는 relevance를 매기지 않아 거기선 0이다.

## Flutter 동등성 기준

- 변조 감쇠 model(`lambdaEff`·`modulatedBrightness`·4 계수)은 BE 무의존 순수 함수다 — Flutter가 동일 식·동일 계수·동일 `A_MIN` 바닥으로 포팅해야 한다.
- `relevance`는 서버가 `Star.relevance`로 내려주는 값이다 — Flutter는 이를 **소비만** 하고 재계산하지 않는다. 임베딩 미생성·서버 부재 시 0=중립으로 폴드해야 한다.
- `R_conn` 입력 degree는 클라가 자체 synapse 그래프에서 중앙값 정규화로 산출해야 한다(서버가 degree를 내려주지 않는다).
- `R_emo` 입력 intensity·valence는 `Star`(proto)에서 읽는다.
- `Synapse.co_activation_count`를 받아 링크 활력 시각(펄스 베이스라인)에 반영해야 한다.
- 어떤 입력에서도 별/엣지를 삭제하지 않고 밝기만 변경해야 하며, 유효 밝기는 `A_MIN` 아래로 내려가지 않아야 한다(헌법2).

## 수용 기준

1. 별 표시 밝기는 단일 λ가 아니라 별마다 변조된 `λ_eff = λ_base · R_conn · R_recent · R_emo`(λ_base=ln2/30)로 감쇠한다.
2. degree(연결)가 클수록 `R_conn = 1/(1+0.6·degree_norm)`으로 감쇠가 늦어 같은 Δt에서 더 밝다. degree_norm은 우주 degree 중앙값 정규화(중앙값 0 폴백 1)다.
3. intensity가 클수록, 그리고 valence가 강한 음수일수록 `R_emo = 1/(1+0.7·intensity+0.4·max(0,-valence))`로 감쇠가 늦어진다.
4. 별이 요즘 토픽과 가까울수록(relevance↑) `R_recent = 1/(1+0.5·relevance)`로 감쇠가 늦어진다. relevance 미배선(데모·임베딩 미생성·읽기 실패) 시 0 폴백으로 `R_recent`가 무변 1.0이 된다.
5. relevance는 서버가 `GetUniverse`에서 별 임베딩과 요즘 토픽 시간가중 중심 벡터(`intensity·exp(-Δt/7d)` envelope)의 cos로 계산해 `Star.relevance`로 노출한다. `ListDormant`는 relevance를 매기지 않는다(0).
6. 고립·저강도·요즘 무관 별은 연결·고강도·요즘 관련 별보다 약 2~3배 빠르게 어두워진다.
7. Δt가 충분히 크면 어느 별도 정확히 `A_MIN=0.05`에서 멈추고 0이 되지 않으며, 어떤 입력에서도 별·시냅스를 삭제하지 않는다(헌법2).
8. 입력 범위를 벗어난 값(음수 degree, intensity>1, |valence|>1, relevance>1)에서도 clamp 방어로 `λ_eff ≤ λ_base`(감쇠 가속 없음)·밝기 ≤ 1을 보장한다.
9. `GetUniverse` 응답의 `Synapse.co_activation_count`(이미 영속·SELECT됨)가 DTO로 노출돼 `SynapseEdge.coActivationCount`로 운반되고, 링크 활력 시각(`vitality`→펄스)에 반영된다.
10. 변조 감쇠의 `relevance` 매핑·`co_activation_count` 노출로 좌표 필드는 추가되지 않는다(위치는 클라 force-sim, 헌법3).
11. `activation.ts`는 `three`/React/DOM을 import하지 않는다(순수 model, 모바일 재사용, 헌법4).
12. 랜딩 `SilentEngramDemo`와 데모 모달이 실제 `modulatedBrightness`로 두 별을 시간 슬라이더로 비교하고, 둘 다 `A_MIN`에서 멈추며 재점화로 다시 밝아진다.

## 정책

튜닝 값의 SSOT는 spec/policy다(이 문서는 가리키기만 한다):
- [domain/star.md](../policy/domain/star.md) — 변조 감쇠 `λ_eff`·`R_conn`/`R_recent`/`R_emo` 식·계수(0.6/0.5/0.7/0.4)·`A_MIN=0.05` 단일 바닥·`modulatedBrightness`·relevance 서버 계산·계산 위치(헌법3).
- [domain/synapse.md](../policy/domain/synapse.md) — `co_activation_count` proto 노출·링크 활력 `vitality = 0.12·min(1, log2(1+co_activation_count)/4)`·degree 정규화(`degreeNormById`)가 별 `R_conn` 입력임.
