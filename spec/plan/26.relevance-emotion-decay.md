# 26. 별 밝기 = 자기-거리 + 링크 활력 (as-built)

> 현재 구현 기준 문서. **별 밝기는 더 이상 독립 감쇠 모델을 갖지 않는다 — "나로부터의 거리(반지름)"를 빛으로 읽은 단조 함수다**(spec 38 change 19). 옛 변조 감쇠 `λ_eff = λ_base·R_conn·R_recent·R_emo`와 서버 `relevance`(요즘 토픽 정합도)는 폐기됐다. 이 문서는 그 폐기 후 남는 것 — 연결성(degree·Σweight) 정규화가 *반지름*을 늦추는 입력이라는 점, 그리고 시냅스 링크 활력(`co_activation_count`→`vitality`) — 의 SSOT다. | 범위: FS | 상태: 구현 완료(밝기=거리로 통합)

## 목적

concept.md §망각 모델의 명제 — "망각은 시간만이 아니라 관련성의 함수다" — 는 이제 **하나의 변수(반지름)** 로 통합돼 표현된다. 12(decay-dormant)의 단일 λ는 시냅스 밝기·휴면 판정에만 남고, 별의 "얼마나 잊혔나"는 자기-거리가 단독으로 답한다. 연결 많고·자주 떠올리고·감정이 강한 별은 천천히 멀어지고(가깝고 밝게 머묾), 고립된 저강도·일회성 기억은 빨리 멀어진다(어두워진다) — 연결·회상·감정이 *반지름*을 늦추므로(change 18) 그 효과가 거리를 통해 그대로 밝기에 전달된다(이중 계산 없음). 12의 불변 — 결정론적 렌더 계산·`A_MIN=0.05` 단일 바닥·"별·시냅스 삭제 금지(헌법2)" — 는 그대로다.

## 현재 구현

### 별 밝기 = 자기-거리 (spec 38 change 19)

밝기 모델의 SSOT는 [plan/38.self-anchored-layout](38.self-anchored-layout.md)·[policy/domain/star.md](../policy/domain/star.md)다. 요점:

- `brightnessFromRadius(radius)`(`frontend/src/entities/memory/model/activation.ts`): `R_MIN`→1(밝음)·`R_MAX`→`A_MIN`(휴면), 반지름에 단조 감소·`A_MIN` 바닥. 별 밝기는 시간·연결·감정·관련성의 *독립* 감쇠를 갖지 않는다.
- `starGlow(recallCount, intensity, lastRecalledAt, now, degreeNorm, weightedDegreeNorm)`: 레이아웃이 별을 놓는 **그 반지름**(Bjork 인출 강도 + 연결성 → `targetRadius`, change 18)을 그대로 다시 빛으로 읽는다 — `brightnessFromRadius(targetRadius(memoryRadiusR(…)))`. 따라서 밝기 = `A_MIN + (1-A_MIN)·R_radius^γ` 꼴로, 거리·밝기가 한 변수로 함께 움직인다.
- 회상으로 반지름↓ → 밝기↑(중앙 복귀), 시간으로 반지름↑ → 밝기↓. 연결 강한 별은 반지름이 천천히 늘어 더 오래 밝다 — 별도 `R_conn`/`R_recent` 항 없이 거리를 통해 성립.

### 폐기된 것 (옛 변조 감쇠 λ_eff·relevance)

- `lambdaEff`·`modulatedBrightness`·계수 `ALPHA_CONN`/`BETA_RECENT`/`GAMMA_EMO`/`DELTA_VAL`(values `decay.*`)·연결성 구동 `selfGlow`(values `self_glow.*`) — 전부 제거. 별 밝기 감쇠 모델은 더는 없다.
- 서버 `relevance`(요즘 토픽 cos 정합도) — 제거: `Star.relevance`(proto field 13 → `reserved`), 도메인 `Memory.Relevance`, `RelevanceByStar`·`StarVector`·`ListStarVectorsByUser`(쿼리·repo)·`TauMoodDays`(values `ambient.tau_mood_days`). `GetUniverse`의 relevance 폴드도 제거.
- self-light 3채널(spec 03)은 "거리 밝기 + 색"으로 단순화: 자가발광(`aGlow`)·반사 변조(`aRecency`) 두 렌더 채널을 **하나의 거리 밝기**가 함께 구동하고(연결성 채널 흡수), 색은 mood(hue_shift) 그대로.

### FE degree·Σweight 정규화 (synapse store) — 이제 *반지름* 입력

`frontend/src/entities/synapse/model/store.ts`. (옛 `R_conn`/self-glow 입력이었으나 change 18·19로 *반지름→밝기* 입력으로 이주.)

- `degreeNormById(edges)`·`weightedDegreeById(edges)` — 별 id → degree(인접 엣지 수)·Σweight(인접 링크 가중 합)를 우주 전체의 **중앙값**으로 정규화. 공유 헬퍼 `normalizedNodeMap(edges, edgeValue)`로 통합(degree는 `()=>1`, weighted는 `(e)=>e.weight`). 중앙값 0 폴백 1, 엣지 없는 별은 맵에 없어 호출처가 0으로 읽는다. 순수·three/DOM 미의존.
- `entities/synapse` 공개 API + `@x/star`로 노출 — `radiusOf`(라이브·오버레이 레이아웃)·`starGlow`(StarField)가 연결성 입력으로 소비한다.

### FE 링크 활력 (synapse mapping/types) — 유지

`frontend/src/entities/synapse/model/types.ts` — `SynapseEdge.coActivationCount: number`(서버 `memory_links.co_activation_count` 운반).

`frontend/src/entities/synapse/model/mapping.ts`:

- `toSynapseEdge`는 `UniverseSynapse.coActivationCount`를 운반(서버 미노출·데모·구버전 `?? 0`).
- `vitality(e) = 0.12·min(1, log2(1 + coActivationCount)/4)` ∈ `[0, 0.12]` — 누적 공동 회상이 많을수록 "살아있는" 연결(log 압축이라 처음 몇 번이 가장 크게 기여). 0이면 기존 시각과 동일.
- `pulseAmp(e) = min(1, reinforcedRecency + vitality(e))` — 자주 함께 떠올린 링크는 갓 강화되지 않아도 은은한 펄스 베이스라인을 유지한다.

### FE 렌더 배선 (StarField)

`frontend/src/entities/star/ui/StarField.tsx`. per-instance 밝기를 세 attribute로 채운다(매 프레임 재계산 금지 — 로드/회상 직후/우주 갱신 시에만, 헌법3·08):

- 빌드 효과: `b = starGlow(recallCount, intensity, lastRecalledAt, now, degreeNorm, weightedDegreeNorm)`; `aGlow = reshapedBrightness(b, brightnessOffset)`(23 재성형 오프셋 합성·`A_MIN` 바닥 OUTERMOST); `aRecency = b`(거리 밝기로 반사 변조 — 가까운=강한 별일수록 자아광 반사 밝음); `aFocus = 1`.
- 포커스 효과: `aFocus`에 배율(FOCUS_DIM/BOOST)만 기록. 셰이더가 `(자가발광 + 반사)·aFocus`로 합성해 디밍이 두 채널 공통.
- `degreeNorm`/`weightedDegreeNorm`은 토폴로지/weight 시그니처로 메모해 받는다 — 시간 머신이 밝기 재파생만을 위해 엣지 배열을 갈아끼울 때 빌드가 헛돌지 않게.
- import는 `@/entities/memory/@x/star`(`starGlow`)·`@/entities/synapse/@x/star`(`degreeNormById`)로 entities 교차는 `@x`만(헌법4).

## Flutter 동등성 기준

- 별 밝기는 자기-거리(반지름)의 단조 함수다 — Flutter는 `brightnessFromRadius`(R_MIN→1, R_MAX→A_MIN, A_MIN 바닥)를 동일하게 포팅하고, 레이아웃이 별을 놓는 그 반지름을 빛으로 읽어야 한다. 별도 시간·연결·감정 밝기 감쇠를 두지 않는다.
- 연결성(degree·Σweight)은 클라가 자체 synapse 그래프에서 중앙값 정규화로 산출해 *반지름*(change 18)에 먹인다 — 서버가 degree·relevance를 내려주지 않는다.
- `Synapse.co_activation_count`를 받아 링크 활력 시각(펄스 베이스라인)에 반영한다.
- 어떤 입력에서도 별/엣지를 삭제하지 않고 밝기만 변경하며, 유효 밝기는 `A_MIN` 아래로 내려가지 않는다(헌법2).

## 수용 기준

1. 별 표시 밝기는 독립 감쇠 모델이 아니라 자기-거리(반지름)의 단조 함수다 — 가까운(작은 반지름) 별이 더 밝고, 가장 먼(R_MAX) 별은 정확히 `A_MIN`에서 멈춘다.
2. 밝기 계산에 `relevance`·`λ_eff`·연결성 변조가 들어가지 않는다(코드에서 `lambdaEff`/`modulatedBrightness`/`relevance` 경로 제거).
3. 회상으로 반지름이 줄면 밝기가 따라 오르고, 시간으로 반지름이 늘면 밝기가 내린다 — 거리·밝기가 한 변수로 함께 움직인다.
4. 연결 많고 자주 떠올린 별이 (거리를 통해) 더 밝다 — 별도 연결성/최근성 밝기 항 없이 성립.
5. 색(감정·hue_shift)·크기(intensity)는 변경 전과 동일하게 렌더된다(회귀 없음).
6. `Star.relevance`가 proto·서버에서 제거되고(`reserved 13`), 구버전·데모(relevance 없음) 응답에서도 밝기가 정상 동작한다.
7. Δt가 충분히 크면 어느 별도 정확히 `A_MIN=0.05`에서 멈추고 0이 되지 않으며, 어떤 입력에서도 별·시냅스를 삭제하지 않는다(헌법2).
8. `Synapse.co_activation_count`가 DTO로 노출돼 `SynapseEdge.coActivationCount`로 운반되고 링크 활력 시각(`vitality`→펄스)에 반영된다. 좌표 필드는 추가되지 않는다(위치는 클라 force-sim, 헌법3).
9. `activation.ts`는 `three`/React/DOM을 import하지 않는다(순수 model, 모바일 재사용, 헌법4).
10. 랜딩 `SilentEngramDemo`와 데모 모달이 실제 `starGlow`로 두 별을 시간 슬라이더로 비교하고, 둘 다 `A_MIN`에서 멈추며 재점화로 다시 밝아진다.
