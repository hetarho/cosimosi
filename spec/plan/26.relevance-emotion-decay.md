# 26. 관련성·감정 가중 망각 (relevance-emotion-decay)

> 단일 전역 λ(순수 시간 감쇠)를 별마다 변조되는 감쇠율로 바꿔, 연결 많고·요즘 관련 있고·감정 강렬한 별은 천천히, 고립된 일회성은 빠르게 어두워지되 바닥 아래로는 절대 내려가지 않게 한다. | 범위: FE (+proto Synapse 필드) | 선행: 21, 25 | 차단: 27

## 목적 (Why)
concept.md §망각 모델의 핵심 명제 — **"망각은 시간만이 아니라 관련성의 함수다"** — 를 실제로 구현한다. 12(decay-dormant)는 의도적으로 단일 λ(반감기 30일)만 썼다. 26은 그 단일 λ를 **별마다 변조되는 `λ_eff`** 로 바꾼다: 연결이 많은 별(`R_conn`), "요즘 상태"·반복 주제와 맞닿은 별(`R_recent`, 25의 ambient에서), 감정이 강렬한 별(`R_emo`, 21의 per-fragment **intensity(각성) + valence(부호 있는 정서가)** 에서 — 강한 각성·강한 부정 정서일수록 더 잘 남는다; Kensinger & Corkin 2004)은 천천히 어두워지고, 고립된 저강도·저정서 일회성 기억은 빨리 잠든다. valence는 20이 추출·21이 영속한 값으로, "물리까지 v1" 결정에 따라 12의 순수 시간 감쇠가 아니라 여기 λ_eff에서 실제로 감쇠저항을 구동한다. **우주의 밝기가 최신성이 아니라 의미를 비추게** 만든다. 12의 불변 — `exp(-λΔt)`·밝기 바닥·"별·시냅스 삭제 금지(헌법2)" — 는 그대로 지킨다: 26은 λ를 변조할 뿐 **삭제를 구동하지 않으며**, 바닥은 12의 `A_MIN`보다 살짝 높은 `FLOOR=0.12`(SilentEngramCard 기준)를 변조 모델의 바닥으로 둔다.

## 설계 요점

**FE 변조 감쇠 (순수 model — `three`/React/DOM 미의존, 헌법4):**

> ⚠️ **12가 권위.** `LAMBDA`(=ln2/30)·`A_MIN`·`HALF_LIFE_DAYS`·`activation`·`starBrightness`·`synapseBrightness`·`isDormant`는 12가 이미 정의·export했다. 26은 **재정의하지 않고 import해 재사용**하며, 같은 파일에 **변조 변형(`lambdaEff`·`modulatedBrightness`)과 그 계수만** 추가한다. 기존 식·시그니처를 다시 쓰지 않는다.

```ts
// entities/memory/model/activation.ts — 12 정의(재사용, 재정의 금지):
//   export const LAMBDA = Math.LN2 / HALF_LIFE_DAYS;   // λ_base ≈ 0.0231 /day
//   export const A_MIN = 0.05;
//   export function activation(lastRecalledAt, now): number; // exp(-LAMBDA·Δt)
//
// 26이 같은 파일에 추가하는 것만:

/** 변조 모델 바닥. 12의 A_MIN(0.05)보다 살짝 높다 — 변조 밝기는 의미 신호를 더 또렷이 비춘다. */
export const FLOOR = 0.12
/** R_conn 계수: 연결(degree)이 많을수록 감쇠 저항↑. */
export const ALPHA_CONN = 0.6
/** R_recent 계수: 요즘 관심사(ambient)와 가까울수록 감쇠 저항↑. */
export const BETA_RECENT = 0.5
/** R_emo 계수: 감정 강렬할수록 감쇠 저항↑ (편도체 매개 — concept.md). */
export const GAMMA_EMO = 0.7

/** 입력은 모두 0..1로 정규화된 신호. relevance=cos(star_emb, ambient_topic) ∈ [-1,1]→[0,1] clamp.
 *  λ_eff = λ_base · R_conn · R_recent · R_emo, 각 R∈(0,1] 이므로 변조는 항상 감쇠를 늦춘다(가속 아님). */
export function lambdaEff(degreeNorm: number, relevance: number, intensityNorm: number): number {
  const rConn = 1 / (1 + ALPHA_CONN * Math.max(0, degreeNorm))
  const rRecent = 1 / (1 + BETA_RECENT * Math.max(0, Math.min(1, relevance)))
  const rEmo = 1 / (1 + GAMMA_EMO * Math.max(0, Math.min(1, intensityNorm)))
  return LAMBDA * rConn * rRecent * rEmo
}

/** 변조 밝기 = FLOOR + (1-FLOOR)·exp(-λ_eff·Δt_days). FLOOR 아래로 내려가지 않는다(헌법2). */
export function modulatedBrightness(
  lastRecalledAt: number, now: number,
  degreeNorm: number, relevance: number, intensityNorm: number,
): number {
  const dtDays = Math.max(0, (now - lastRecalledAt) / 86_400_000)
  const le = lambdaEff(degreeNorm, relevance, intensityNorm)
  return FLOOR + (1 - FLOOR) * Math.exp(-le * dtDays)
}
```
- **degree_norm = degree / median(degree)**: synapse store `neighborsOf(edges, memoryId).length`로 별별 degree를, 우주 전체 degree의 중앙값으로 정규화(중앙값 0 방지 폴백 1). 정규화 헬퍼도 model에 둔다(순수). 입력 신호는 ui가 계산해 model 함수에 주입(매퍼가 도메인 조립).
- **relevance = cos(star_emb, ambient_topic_vec)**: ambient_topic_vec는 **25**가 산출(GetUniverse 응답 ambient). 별 임베딩은 서버 권위라 클라가 직접 갖지 않으므로, 26은 **별 mood/주제와 ambient의 정합도**를 0..1 신호로 받는 형태로 두고(ambient 미배선 시 0=중립으로 폴백), 25 배선 후 실제 cos를 채운다. 비목표: ambient 계산 자체.
- **intensity_norm = clamp(intensity,0,1)**: 21이 memories에 per-fragment intensity를 실으면 Star→Memory.intensity로 들어온다. 21 전(단일값)에도 record intensity로 동작.
- **보정 앵커(SilentEngramCard 기준):** 연결·고강도 별 ≈ `connectedDecay 0.8`, 고립·저강도 별 ≈ `isolatedDecay 2.6` 의 감쇠 인상을 재현하도록 계수를 둔다 — 고립이 연결 대비 **2~3배 빠른 감쇠**. 계수는 상수로 노출해 튜닝 가능(주석으로 문서화). **NEVER 삭제를 구동하지 않는다**(밝기만).

**brightness 배선(StarField):**
- 12는 `aBrightness`를 `starBrightness(lastRecalledAt, now)`로 두 지점(`useLayoutEffect` 빌드 + focus `useEffect`)에서 채운다. 26은 그 두 지점을 **`modulatedBrightness(lastRecalledAt, now, degreeNorm, relevance, intensityNorm)`** 로 교체한다(focus 가중 곱·rebuild 패턴은 유지). degree는 synapse store에서, relevance는 ambient(25)에서, intensity는 `m.intensity`에서. 매 프레임 재계산 금지 — 로드 시·회상 직후·우주 갱신 시에만 attribute 재기록(헌법, §3.2).
- StarField가 synapse degree와 ambient를 알아야 하므로, 입력은 **별 id→degreeNorm 맵**과 **relevance 산출 함수**를 props/상위에서 주입(StarField는 ui이므로 store 구독 가능, 하지만 model 식 계산은 순수 함수에 위임).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 별의 표시 밝기를 계산할 때 THE SYSTEM SHALL `λ_eff = λ_base · R_conn · R_recent · R_emo`(`λ_base=ln2/30`)로 별마다 변조된 감쇠율을 사용한다. _(헌법3 — 클라 렌더 시 결정론적 계산)_
1.2 WHEN degree(연결 수)가 클수록 THE SYSTEM SHALL `R_conn=1/(1+0.6·degree_norm)`로 감쇠를 늦춰 같은 Δt에서 더 밝게 유지한다.
1.3 WHEN intensity(감정 강도)가 클수록 THE SYSTEM SHALL `R_emo=1/(1+0.7·intensity_norm)`로 감쇠를 늦춘다(감정=감쇠 저항).
1.4 WHEN 별이 요즘 관심사(ambient_topic)와 가까울수록(`relevance`↑) THE SYSTEM SHALL `R_recent=1/(1+0.5·relevance)`로 감쇠를 늦춘다. (ambient 미배선 시 `relevance=0` 폴백으로 안전.)
2.1 WHEN 고립된 저강도 별과 연결된 고강도 별을 같은 Δt로 비교할 때 THE SYSTEM SHALL 고립 별이 연결 별보다 **약 2~3배 빠르게** 어두워지게 한다(SilentEngramCard 앵커).
2.2 WHEN Δt가 충분히 클 때 THE SYSTEM SHALL 어느 별도 `FLOOR=0.12` 아래로 내려가지 않게 바닥 처리하고, 별·시냅스를 **삭제하지 않는다**. _(헌법2)_
2.3 WHEN 변조 감쇠를 적용할 때 THE SYSTEM SHALL 어떤 입력에서도 별/엣지 제거를 구동하지 않는다(밝기 상태만 변경). _(헌법2)_
3.1 WHEN `GetUniverse` 응답의 `Synapse`를 매핑할 때 THE SYSTEM SHALL `co_activation_count`(이미 영속·SELECT됨)를 DTO로 노출해 `SynapseEdge`로 운반한다. _(헌법5 — DTO 노출)_
4.1 WHEN `activation.ts`를 작성할 때 THE SYSTEM SHALL `three`/React/DOM을 import하지 않는다(순수 model, 모바일 재사용). _(헌법4)_
4.2 WHEN 각 R 항이 입력 범위를 벗어난 값(음수 degree, intensity>1 등)을 받을 때 THE SYSTEM SHALL clamp으로 방어해 `λ_eff>λ_base`(감쇠 가속)나 밝기>1이 나오지 않게 한다.
5.1 WHEN proto `Synapse`에 `co_activation_count`를 추가할 때 THE SYSTEM SHALL 좌표 필드를 추가하지 않는다(위치는 클라 force-sim). _(헌법3)_

## 과학적 근거
망각을 수동적 시간 감쇠가 아니라 **관련성 의존적·가역적 적응 과정**으로 보는 최신 엔그램 연구에 근거한다. 별마다 변조되는 `λ_eff`는 "지금의 삶에 덜 관련된 기억일수록 접근 불가 상태로 더 잘 넘어간다"는 명제의 직접 구현이다.

- 망각 = 접근 가능→불가 전환의 *적응적*·관련성 의존·가역 가소성: Ryan TJ, Frankland PW, *Nature Reviews Neuroscience* (2022). [DOI](https://doi.org/10.1038/s41583-021-00548-3)
- 자연적 망각이 가역적으로 엔그램 발현을 조절(관련성 낮은 기억이 더 잘 잊힘): O'Leary T, Bruckner R, Autore L, Ryan TJ, *eLife* (2024). [DOI](https://doi.org/10.7554/eLife.92860)
- 연결성·도식이 기억을 보호(통합된 기억일수록 보존, 고립된 기억일수록 소실): Audrain S, McAndrews MP, *Nature Communications* (2022). [DOI](https://doi.org/10.1038/s41467-022-33517-0)
- 감정 강도 = 기억의 무게(편도체 매개로 더 강하게 부호화·공고화·인출, 감쇠 저항): concept.md §핵심 인터랙션 "감정·강도의 역할" 및 기억 연구 정설.
