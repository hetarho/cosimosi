# 31. 우주 셸 + 오버레이 네비게이션 (universe-shell-overlays)

> 잠든 별 탐색·원본 일기 목록·변천사 등 리스트/탐색 화면을 별도 페이지가 아니라 **영속 우주 캔버스 위의 모달/바텀시트 오버레이**로 띄워, 사용자가 항상 우주 안에 머무는 느낌을 준다. | 범위: FE | 선행: 12 | 차단: —

## 목적 (Why)

지금은 잠든 별 찾기(`/dormant`) 같은 탐색이 **별도 라우트**라, 진입하면 `/universe`의 WebGPU 우주 캔버스가 통째로 언마운트되고 "우주를 떠나는" 단절감이 생긴다(`app/router.tsx`의 `/dormant` = `SessionGate > DormantPage`, 캔버스 없음). concept §우주 탐험의 정신 — *"전환은 별개 화면이 아니라 같은 우주의 다른 시점"* — 을 **리스트/탐색까지 확장**한다: 우주 캔버스를 **영속 셸**로 두고, 목록은 그 위에 **오버레이**(데스크톱=사이드 패널/모달, 모바일=바텀시트)로 떠오르며, 항목을 고르면 시트가 핸들만 남기고 잦아들며 **뒤의 우주에서 카메라가 그 별로 fly-to**한다. 19의 이론 렌즈·11의 회상 패널처럼 이미 우주 위 오버레이인 HUD와 한 셸에서 공존한다. WebGPU 캔버스를 재초기화하지 않아 성능·연속감도 함께 얻는다.

## 범위

- **포함:**
  - FE `shared/ui` 오버레이 프리미티브(신규): 반응형 `BottomSheet`(coarse pointer/모바일 — 스냅 closed/peek/half/full, 드래그 핸들) + `SidePanel`/`Modal`(fine pointer/데스크톱). 공통 API(open/peek/close, 스냅, **뒤 캔버스 비차단**). 캔버스 밖 DOM(헌법8·Architecture §3.1 — 3D 씬 안 `<Html>` 금지).
  - FE 우주 셸: `UniverseCanvas`가 `/universe`에서 **한 번 마운트되어 영속**(탐색을 열어도 언마운트·재init 안 됨). 오버레이는 셸 위 호스트에 렌더.
  - FE 잠든 별 탐색을 **오버레이로 전환**: 12의 `DormantPage` 목록·검색·`focusStar` 동선을 `DormantSheet`로 재구성(별도 라우트 진입 대신 우주 안 컨트롤로 토글). 딥링크는 `/universe?panel=dormant`로 **오버레이를 연 채** 렌더(캔버스 영속). 풀페이지 `/dormant`는 제거하거나 `/universe?panel=dormant`로 redirect.
  - FE 리스트→우주 동선: 항목 클릭 시 시트가 **peek(핸들/축소)로 내려가고** 뒤 우주에서 `focusStar`(12)/`frameStars`(28) fly-to → 회상(11). 시트가 열린 채로도 뒤 캔버스 회전·이동 가능(비차단; 차단형 dim은 확인 모달에만).
  - FE 셸 패널 상태 스토어: `panel: 'dormant'|'diary'|'evolution'|null` + `?panel=` search param 동기화(딥링크·뒤로가기). 라우트를 늘리지 않는다.
  - **패턴 정의**: 24(변천사)·28(원본 일기 목록)이 별도 풀페이지 대신 **이 오버레이 셸을 따른다**(각 스펙이 `…Sheet`로 콘텐츠만 제공).
  - 접근성: `prefers-reduced-motion`(시트 슬라이드 축소/생략)·coarse pointer(바텀시트·큰 터치 타깃)·모달 포커스 트랩/Esc — ux/motion-accessibility 정책.
- **비목표(Out of scope):**
  - 탐색 데이터/쿼리·카메라 fly-to·`frameStars` 로직 자체 → **12/24/28**(본 스펙은 오버레이 셸·전환·동선만; 콘텐츠는 각 스펙 재사용).
  - 랜딩(`/` 별도 페이지가 맞음) → **15**.
  - 새 이론·렌더 → 19–28.

## 설계 요점

**셸 구조(캔버스 영속, 라우트 비증가).** `/universe`(HomePage 또는 universe 셸)가 `UniverseCanvas` + HUD 레이어(MemoryForm·MemoryPanel·NavPad·AppearanceSwitcher·theory-lens) + **오버레이 호스트**를 함께 들고, 패널 상태를 셸 스토어가 관리한다. 탐색은 라우트가 아니라 **패널 상태**다 — `?panel=dormant` search param으로만 딥링크/뒤로가기를 동기화하고, 캔버스는 절대 언마운트하지 않는다.

```ts
// features/universe(또는 app)/model/shell-store.ts — 패널 상태(순수 zustand)
type Panel = 'dormant' | 'diary' | 'evolution' | null;
interface ShellStore { panel: Panel; openPanel(p: Panel): void; closePanel(): void; peek: boolean; setPeek(b: boolean): void; }
// 항목 선택 시: setPeek(true) + focusStar/frameStars(우주 fly-to) — 시트는 핸들만 남김
```

**오버레이 프리미티브(`shared/ui`, 캔버스 밖 DOM).** 반응형으로 한 곳에서 분기(플랫폼 분기는 ui — 헌법4).

```tsx
// shared/ui/OverlayHost.tsx — coarse pointer면 BottomSheet, fine이면 SidePanel
<OverlayHost open peek onClose>{children}</OverlayHost>
// BottomSheet: 스냅(closed/peek/half/full), 드래그 핸들, backdrop 없음(or 약한 dim) → 뒤 캔버스 보임·상호작용
// SidePanel: 데스크톱 우측 비차단 패널. Modal: 차단형(확인 등 한정).
```

- **캔버스는 한 번만 init.** WebGPU 캔버스 생성·`renderer.init()`은 비용이 크므로(06), 오버레이 열고 닫음이 캔버스를 재init하지 않는다 — 현재 `/dormant`가 캔버스를 떠나는 문제의 핵심 해결.
- **동선.** 시트 항목 클릭 → 셸이 `setPeek(true)` → `focusStar`(12)/`frameStars`(28) → 카메라가 우주에서 그 별로 이동(시트는 핸들로 남아 다시 끌어올릴 수 있음) → 회상(11). near/far 가드(28)와 정합.
- **잠든 별 시트.** `DormantSheet`는 12의 `list-dormant` 호출·목록·검색·`focusStar`를 그대로 재사용하고, 셸(컨테이너)만 페이지→시트로 바꾼다. `DormantPage`의 콘텐츠를 시트 본문으로 추출.

## 수용 기준 (EARS / Given-When-Then)

1.1 WHEN 사용자가 잠든 별 탐색을 열면 THE SYSTEM SHALL 우주 캔버스를 언마운트하지 않고 그 위에 오버레이(모바일=바텀시트 / 데스크톱=사이드 패널)로 목록을 띄운다.
1.2 WHEN 목록 항목을 고르면 THE SYSTEM SHALL 시트를 peek(핸들)로 낮추고 뒤 우주에서 카메라가 그 별로 fly-to하여 **우주를 떠나지 않는다**(11 회상으로 이어짐).
1.3 WHEN 오버레이가 열려 있어도 THE SYSTEM SHALL 뒤 우주 캔버스가 보이고(비차단 시트) 회전·이동 등 상호작용이 가능하다(차단형 dim은 확인 모달에만).
1.4 WHEN 화면이 coarse pointer(모바일)면 THE SYSTEM SHALL 바텀시트를, fine pointer(데스크톱)면 사이드 패널/모달을 쓴다(반응형, 분기는 ui 레이어).
1.5 WHEN 사용자가 탐색을 여러 번 열고 닫으면 THE SYSTEM SHALL WebGPU 우주 캔버스를 재초기화하지 않는다(영속 셸).
1.6 WHEN `/universe?panel=dormant`로 진입하면 THE SYSTEM SHALL 캔버스 위에 그 오버레이를 연 채 렌더하고, 뒤로가기로 닫힌다(라우트 증가 없이 search param 동기화).
1.7 WHEN `prefers-reduced-motion`이면 THE SYSTEM SHALL 시트 슬라이드 애니메이션을 축소·생략한다(즉시 표시).
1.8 THE SYSTEM SHALL 24(변천사)·28(원본 일기 목록)이 별도 풀페이지 라우트를 쓰지 않고 이 오버레이 셸을 따르도록 패턴을 제공한다.
1.9 WHEN `shared/ui` 오버레이·`features/universe/model`이 빌드되면 THE SYSTEM SHALL 3D 씬 안에 DOM(`<Html>`)을 만들지 않고, model은 `three`/React/DOM(model 한정)을 import하지 않는다(헌법4·8).

## 완료 정의 (DoD)

- 잠든 별 탐색이 **별도 페이지가 아니라 우주 위 오버레이**로 열린다: 열어도 우주 캔버스가 뒤에 그대로 보이고(비차단), 모바일은 바텀시트·데스크톱은 사이드 패널이다.
- 목록 항목을 고르면 **시트가 핸들로 잦아들고 뒤 우주에서 카메라가 그 별로 fly-to**한다(우주를 떠나지 않음). 다시 핸들을 끌어올리면 목록이 돌아온다.
- 탐색을 여러 번 열고 닫아도 WebGPU 캔버스가 **재초기화되지 않는다**(콘솔에 renderer 재생성 로그 없음 — 영속 셸).
- `/universe?panel=dormant` 딥링크로 진입하면 캔버스 위에 오버레이가 열린 채 렌더되고, 뒤로가기로 닫힌다(`/dormant` 풀페이지 라우트 제거 또는 redirect).
- `prefers-reduced-motion`에서 시트 슬라이드가 축소/생략된다.
- 24·28은 별도 풀페이지가 아니라 이 오버레이 셸(`…Sheet`)을 쓴다(각 스펙에서 채택).
- `pnpm --filter frontend build`·`pnpm lint` 통과. `shared/ui` 오버레이·셸이 3D 씬 안 DOM을 만들지 않고 `features/universe/model`이 순수하다(헌법4·8, grep 확인).

## 비고 — 신경과학 근거

본 스펙은 **UX 셸** 작업이라 신경과학 이론을 직접 구현하지 않는다. 다만 "탐색해도 늘 우주에 머문다"는 경험은 concept §우주 탐험의 *"전환은 별개 화면이 아니라 같은 우주의 다른 시점"* 원칙의 직접 실현이며, 별·연결을 떠나지 않는 연속적 항해 감각(공간 기억의 연속성)을 지지한다.
