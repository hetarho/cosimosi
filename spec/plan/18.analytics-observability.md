# 18. analytics-observability — Sentry 본격 배선 + 제품 지표(PostHog) 수집

> 베타 50명이 무엇을 하고 어디서 막히는지 측정할 수 있게 한다 — 에러는 Sentry, 행동 지표는 PostHog. | 범위: FS (베타 하드닝) | 선행: 17 | 차단: —

## 목적 (Why)

베타는 "출시"가 아니라 **측정 장치**다. 지금은:

- **Sentry가 반쯤 죽어 있다.** `main.tsx`의 `Sentry.init`은 DSN·environment만 넘긴다 — integration 0개, release 태깅 없음, 성능 추적 없음. 17이 에러 바운더리로 *캡처 경로*는 만들지만, 트레이싱·릴리즈 매핑·소스맵 없이는 "어떤 배포에서 어떤 사용자 흐름이 깨졌나"를 답할 수 없다.
- **제품 지표가 0이다.** 50명 중 몇 명이 일기를 썼는지, 별을 회상했는지, 잠든 별 페이지에 가봤는지, WebGPU와 WebGL2 폴백 비율이 얼마인지 — 아무것도 모른다. 개선 우선순위를 정할 데이터가 없다.

## 범위

**포함**
- Sentry FE 강화: `browserTracingIntegration`(낮은 샘플레이트), release/environment 태깅, (선택) 마스킹 기본 Session Replay
- 제품 분석 라이브러리 도입 — **PostHog 권장**(아래 결정 근거) + 이벤트 래퍼(`shared/lib/analytics`)
- 핵심 이벤트 계측(이벤트 설계표 포함) + 환경변수·배포 문서 갱신
- BE: 워커/큐 상태 주기 요약 로그(pending/failed/최고령 잡) — 백로그가 침묵 속에 쌓이는 걸 가시화
- **프라이버시 가드**: 일기 내용은 어떤 이벤트에도 싣지 않는다

**비목표 (Out of scope)**
- Prometheus/Grafana 등 풀 메트릭 스택(베타 규모 과잉 — 요약 로그 + Sentry로 충분)
- A/B 테스트·feature flag 운영(PostHog 기능이지만 베타 후)
- §4.7 비용 상한·서킷브레이커(별도 후속 — 검토 백로그 B4)
- 소스맵 업로드 CI 자동화는 **선택 태스크**(수동 업로드로 시작 가능)

## 설계 요점

**라이브러리 결정 — PostHog (구현 시 재검증):**

| 후보 | 판단 |
|---|---|
| **PostHog** ✅ | 이벤트+퍼널+리텐션+세션리플레이 일체, 무료 티어가 베타 50명에 충분, EU/셀프호스팅 옵션(프라이버시), autocapture로 계측 누락 보완 |
| Plausible | 가볍지만 페이지뷰 중심 — 행동 퍼널(작성→회상→재방문)에 부족 |
| Amplitude | 강력하나 베타 규모에 과잉 + 무료 티어 제약 |

**프라이버시 헌법(이 스펙의 불변 원칙):**
1. **일기 body·임베딩·기억 내용은 절대 이벤트에 싣지 않는다.** 길이 버킷(`short/medium/long`)·mood까지만.
2. Session Replay를 켠다면 **마스킹 기본**(입력·텍스트 전부) — 일기 작성 화면이 녹화되면 안 된다.
3. autocapture가 폼 내용을 줍지 않도록 설정 확인(`MemoryForm`의 textarea).
4. PII 최소화: 식별자는 Supabase uid만, 이메일 등은 보내지 않는다.

**이벤트 설계(초안 — 구현 완료, 정전은 [policy/ux/analytics-privacy.md](../policy/ux/analytics-privacy.md)로 이동. 구현은 표 + 공통 속성 `demo`(체험 모드 여부 — 베타 퍼널에서 체험 트래픽 필터링)):**

| 이벤트 | 속성 | 답하는 질문 |
|---|---|---|
| `sign_in` | — | 가입/재방문 |
| `universe_loaded` | star_count, synapse_count, load_ms, renderer(webgpu\|webgl2) | 초기 경험 성능, **폴백 비율** |
| `record_memory` | mood, body_length_bucket, success | 코어 루프 발화율 |
| `recall_open` | is_dormant | 회상이 일어나는가 |
| `reinforce_flush` | pair_count | 공동 회상(헵)이 실제로 도는가 |
| `dormant_visit` | dormant_count | 잠든 별 기능 발견율 |
| `appearance_switch` | theme | 외형 기능 사용률 |

**구조(FSD):**
```
shared/lib/analytics/        ← 래퍼: init + capture(name, props). key 없으면 전부 no-op.
  index.ts                      react/three import 금지(pure 레이어 — ESLint가 강제).
                                이벤트 이름은 상수로 export(오타 계측 방지).
main.tsx                     ← PostHog init + Sentry init 강화(integrations, release)
features/·pages/·widgets/    ← 각 행동 지점에서 capture 호출 (ui/model 레이어)
```
- Sentry init: `browserTracingIntegration()` + `tracesSampleRate` 낮게(0.1~0.2), `release`는 빌드 시 주입(`VITE_APP_VERSION` 또는 커밋 SHA — CF Pages 빌드 환경변수). Replay는 `maskAllText/blockAllMedia` 기본으로 선택 도입.
- BE 요약 로그: `cmd/api`(또는 worker 루프)에서 N분 주기로 `jobs` 상태 카운트·최고령 pending age를 slog Info 1줄 — 그라파나 없이도 `docker logs`로 백로그를 본다.

## 수용 기준 (EARS/GWT)

- 3.1 WHEN `VITE_POSTHOG_KEY`가 없으면 THE SYSTEM SHALL 모든 capture를 no-op으로 처리한다(로컬/체험 모드 무영향).
- 3.2 WHEN 사용자가 일기를 작성하면 THE SYSTEM SHALL `record_memory` 이벤트를 보내되 **body 원문·일부를 포함하지 않는다**(길이 버킷·mood만).
- 3.3 WHEN 우주가 처음 렌더되면 THE SYSTEM SHALL `universe_loaded`에 실제 선택된 렌더러 백엔드(webgpu/webgl2)와 로드 시간을 기록한다.
- 3.4 WHEN 프로덕션 빌드에서 에러가 캡처되면 THE Sentry 이벤트 SHALL release·environment 태그를 갖는다.
- 3.5 WHEN Session Replay를 켜는 경우 THE SYSTEM SHALL 텍스트·입력 마스킹을 기본으로 한다.
- 3.6 WHEN 워커가 동작 중이면 THE SYSTEM SHALL 주기적으로 큐 상태(pending/failed/최고령) 요약을 구조화 로그로 남긴다.
- 3.7 ESLint pure-레이어 규칙: `shared/lib/analytics`는 react/three를 import하지 않는다(기존 boundaries 규칙 통과).
