# 09. 시냅스 렌더링 — entities/synapse (Line2 · TSL)

> 가중 엣지를 fat-line(Line2)으로, 두께·밝기·펄스를 TSL로 매핑한다. | 범위: MVP | 선행: 06, 08 | 차단: —

## 목적 (Why)
별과 별을 잇는 시냅스(가중치 엣지)를 우주에 그린다. 강한 연결일수록 굵고 밝게, 최근 강화된 연결일수록 펄스로 살아 숨쉬게 보여 "시냅스가 공간을 빚는다"는 컨셉을 직접 시각화한다. 수천 엣지를 전제로 Line2 배칭과 TSL 매핑으로 성능을 확보한다.

## 범위
- **포함:** `entities/synapse` 슬라이스 신설 — `model/types.ts`(순수 `SynapseEdge` 타입), `ui/SynapseLines.tsx`(Line2 배칭 렌더 + TSL 머티리얼: 두께·emissive·펄스 매핑), `model/mapping.ts`(weight·brightness→시각 파라미터 순수 변환), `index.ts`(Public API).
- **비목표:**
  - 강화 로직(weight 증가·co_activation·last_activated_at 갱신) → **11 recall-reinforce**.
  - 감쇠 계산의 출처(`activation = exp(-λ·Δt)` 산출·잠든 상태 판정) → **12 decay-dormant**. 단 `brightness = max(a_min, activation)` 값을 **받아 시각에 매핑하는 것은 여기**서 담당.
  - 엣지 데이터를 서버에서 받아오는 RPC·매퍼 → **04 memory-api**(GetUniverse). 여기선 props로 받는 더미/주입 그래프로 검증.
  - 별 노드 렌더·인스턴싱 → **08 star-rendering**. 여기선 별 좌표 lookup만 사용.
  - 캔버스·WebGPU 렌더러·Bloom·카메라 셋업 → **06 universe-canvas**.

## 설계 요점
**model은 순수**(three/React/DOM 미의존, 원칙4·§2.11). 좌표는 받기만 한다(좌표 권위는 클라 force-sim — 원칙3).

```ts
// entities/synapse/model/types.ts — 순수 도메인 시각 모델
export type LinkType = "semantic" | "temporal" | "entity" | "co_recall";

export interface SynapseEdge {
  aId: string;            // 정규화 a_id < b_id (무방향 1행)
  bId: string;
  weight: number;         // 0~1 시냅스 강도(서버 권위 그래프)
  brightness: number;     // = max(a_min, activation) — 출처는 12, 여기선 입력
  reinforcedRecency: number; // 0~1 최근 강화도(펄스 amp 구동, 출처는 11/12)
  linkType: LinkType;
}

// entities/synapse/model/mapping.ts — 순수 변환(테스트 가능, three 미의존)
export const A_MIN = 0.05;
export const WIDTH_MIN_PX = 0.5;
export const WIDTH_MAX_PX = 4;
export const lerp = (a: number, b: number, t: number) => a + (b - a) * t;
// 유효 시각 강도 = weight · brightness (Architecture §5)
export const visualIntensity = (e: SynapseEdge) =>
  e.weight * Math.max(A_MIN, e.brightness);
export const lineWidthPx = (e: SynapseEdge) =>
  lerp(WIDTH_MIN_PX, WIDTH_MAX_PX, visualIntensity(e)); // clamp 0..1 가정
export const emissive = (e: SynapseEdge) => visualIntensity(e);
export const pulseAmp = (e: SynapseEdge) => e.reinforcedRecency; // sin(time·f)·amp
```

```tsx
// entities/synapse/ui/SynapseLines.tsx — Line2 배칭 + TSL (플랫폼 레이어)
// three의 LineSegments2/LineSegmentsGeometry + Line2NodeMaterial(TSL).
// - 모든 엣지를 단일 지오메트리에 배칭(positions = 별 좌표 lookup으로 매 tick 갱신).
// - per-edge instanceColor/attribute: emissive, width, pulseAmp 를 인스턴스 속성으로.
// - TSL 머티리얼: emissiveNode = attr·(1 + sin(time·f)·pulseAmp);
//   linewidth는 LineMaterial worldUnits=false(px), 두께는 인스턴스 attr로 변조.
// - useFrame: React state 미사용(원칙: 60fps 리렌더 금지) — geometry attribute/uniform만 갱신.
props: { edges: SynapseEdge[]; positionOf: (id: string) => [number,number,number] }
```

매핑 규칙(Architecture §5 인용):
- 선 두께 `lerp(0.5px, 4px, weight·brightness)`
- emissive `weight·brightness`
- 펄스 `sin(time·f)·amp` (amp = `reinforcedRecency`, 최근 강화일수록 ↑)
- brightness `max(a_min, activation)`, `a_min = 0.05`

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN `edges`가 주어지면 THE SYSTEM SHALL 모든 엣지를 단일 Line2/LineSegments2 지오메트리에 배칭하여 draw call 수가 엣지 수에 비례하지 않게 렌더한다.
1.2 WHEN 두 엣지의 `weight·brightness`가 다르면 THE SYSTEM SHALL `lerp(0.5px,4px,·)`에 따라 굵기를, `weight·brightness`에 따라 emissive 밝기를 서로 다르게 그린다.
1.3 WHEN `reinforcedRecency > 0`인 엣지가 있으면 THE SYSTEM SHALL `sin(time·f)·amp`로 emissive를 펄스시키고, `reinforcedRecency = 0`이면 펄스 없이 정적으로 그린다.
1.4 GIVEN `activation`이 매우 낮은(잠든) 연결 WHEN brightness가 계산되면 THE SYSTEM SHALL `brightness = max(a_min, activation)` 바닥을 적용해 0으로 사라지지 않고 은은히 잔존시킨다(원칙2).
1.5 WHEN 별 좌표가 force-sim tick으로 갱신되면 THE SYSTEM SHALL `useFrame`에서 geometry position attribute만 갱신하고 React state 리렌더를 발생시키지 않는다.
1.6 IF `positionOf(id)`가 좌표를 못 찾으면 THEN THE SYSTEM SHALL 해당 엣지를 건너뛰고(예외 없이) 나머지를 정상 렌더한다.
2.1 THE `model/*`(types·mapping) SHALL `three`/React/DOM을 import하지 않는다(순수 — 원칙4, 모바일 재사용).
