# 09. 시냅스 렌더링 - as-built

> 현재 우주에서 보이는 시냅스는 단순 screen-space 선이 아니라, 별 좌표 사이를 잇는 braided filament bundle과 dust mist다. weight, time brightness, recent reinforcement, co-activation vitality가 strand 수/밝기/opacity/pulse에 반영된다.

## 현재 범위
- Domain/store:
  - `frontend/src/entities/synapse/model/types.ts`
  - `frontend/src/entities/synapse/model/store.ts`
  - `frontend/src/entities/synapse/model/mapping.ts`
- Render:
  - `frontend/src/entities/synapse/ui/SynapseFilaments.tsx`
  - `frontend/src/entities/synapse/ui/SynapseDust.tsx`
- Canvas integration:
  - `frontend/src/widgets/universe-canvas/ui/UniverseCanvas.tsx#UniverseSynapses`
- `SynapseLines.tsx`는 여전히 export되어 있지만 현재 `UniverseCanvas`의 주 렌더 경로는 `SynapseFilaments` + `SynapseDust`다.

## SynapseEdge domain
```ts
export type LinkType = 'semantic' | 'temporal' | 'entity' | 'co_recall'

export interface SynapseEdge {
  aId: string
  bId: string
  weight: number
  brightness: number
  reinforcedRecency: number
  linkType: LinkType
  lastActivatedAt?: number
  coActivationCount: number
}
```

- 좌표는 edge에 저장하지 않는다. 별 좌표는 force-sim buffer 또는 settled layout map에서 조회한다.
- server `Synapse`는 `toSynapseEdge()`에서 domain edge로 매핑한다.
- unknown `linkType`은 `semantic`으로 fallback한다.
- `lastActivatedAt`은 `applyUniverse()`에서 proto string을 epoch ms로 바꾼 뒤, `starBrightness(lastActivatedAt, now)`로 brightness를 재파생한다.
- `coActivationCount`는 공동 회상 횟수이며 link vitality와 pruning/analytics 입력이다.

## Store와 병합
- `useSynapseStore` state:
  - `edges`
  - `setEdges(edges)`
  - `bumpEdgeWeight(aId, bId, delta)`
- `bumpEdgeWeight()`는 demo/local co-recall preview용이다.
  - pair를 정규화한다.
  - 없으면 `co_recall` edge를 만든다.
  - 있으면 `weight`를 1.0까지 올리고 `brightness=1`, `reinforcedRecency=1`, `coActivationCount += 1`, `lastActivatedAt=virtualNowMs()`로 갱신한다.
- `mergeEdges(local, incoming, now)`는 pair key로 병합한다.
  - `weight`, `reinforcedRecency`, `coActivationCount`, `lastActivatedAt`은 max/forward-safe 방식으로 병합한다.
  - `brightness`는 `lastActivatedAt`을 알면 `starBrightness(lastActivatedAt, now)`로 다시 계산한다.
  - local-only edge는 유지하고 server-only edge는 append한다.

## Mapping 함수
- `visualIntensity(e) = clamp01(weight) * max(A_MIN, brightness)`, 결과는 0..1 clamp.
- `A_MIN = 0.05`
- `alpha(e) = lerp(0.15, 1, visualIntensity(e))`
- `emissive(e) = visualIntensity(e)`
- `vitality(e) = 0.12 * min(1, log2(1 + coActivationCount) / 4)`
- `pulseAmp(e) = min(1, reinforcedRecency + vitality(e))`
- `strandStyle(e)`는 `visualIntensity` tier에 따라 strand bundle style을 고른다.

Current `STRAND_TIERS`:
```ts
[
  [0.35, { strands: 2, radius: 0.01, bright: 0.2, opacity: 0.5 }],
  [0.6,  { strands: 4, radius: 0.018, bright: 0.32, opacity: 0.62 }],
  [0.85, { strands: 6, radius: 0.028, bright: 0.45, opacity: 0.72 }],
  [Infinity, { strands: 9, radius: 0.04, bright: 0.6, opacity: 0.82 }],
]
```

- 오래되거나 약한 edge도 0으로 사라지지 않고 흐릿하게 남는다.
- 방금 강화된 edge는 `pulseAmp` 때문에 strand 수와 base radius가 일시적으로 증가하고 shader pulse가 커진다.

## UniverseSynapses integration
- `UniverseSynapses`는 memory store의 `stars`, synapse store의 `edges`, appearance store의 `emotionColors`, focus actor의 selected/highlight state를 읽는다.
- `positionOf(id)`는 settled `layout: Map<string, [x,y,z]>`에서 좌표를 찾는다.
- `colorOf(id)`는 별의 mood와 user emotion override를 반영한 endpoint color를 반환한다.
- `seedOf(id)`는 StarField wobble seed와 같은 seed를 반환한다.
- `idIndex`는 `stars` array slot 기준이다. live force-sim buffer row와 같은 순서여야 한다.
- star focus 또는 diary highlight 중에는 전체 web을 dim한다.
  - selected star focus: `dim = 0.1`
  - diary highlight: `dim = 0.12`
  - normal: `dim = 1`
- diary highlight가 활성이고 single star focus가 없으면 `edgesWithin(edges, highlightedIds)`를 별도 filament layer로 한 번 더 그려 해당 일기의 내부 연결만 또렷하게 올린다.
- `edgesWithin()`은 link type string이 아니라 endpoint membership으로 판단한다. server의 `intra_entry` 같은 값이 domain mapper에서 `semantic`으로 fallback될 수 있기 때문이다.

## SynapseFilaments
- 각 edge는 bowed center curve 위에 여러 개의 thin tube strand로 변환된다.
- edge가 300개를 넘으면 `visualIntensity`가 강한 300개만 렌더한다.
- edge별 deterministic hash와 `mulberry32`를 사용해 bow, helix radius, brightness/width/opacity jitter를 만든다.
- endpoint mood color가 strand 길이를 따라 A->B gradient로 bake된다.
- StarField와 같은 wobble seed를 endpoint마다 attribute로 패킹한다. shader가 StarField wobble 수식을 재현해 별이 미세 부유해도 strand endpoint가 별 중심을 따라간다.
- `MeshBasicNodeMaterial`의 `colorNode`가 emitted HDR color 역할을 한다. bloom이 이 값을 잡는다.
- built-in TSL time node를 쓰지 않고 manual `uTime` uniform을 `useFrame`에서 갱신한다.
- `uDim` uniform으로 focus/dim을 적용한다.
- geometry는 edge set/layout/color/seed lookup이 바뀔 때 한 번 bake한다.
- live-follow:
  - geometry는 settled layout snapshot으로 bake된다.
  - 매 프레임 `positionsRef.current`와 `idIndex`가 있으면 endpoint drift를 계산한다.
  - 각 edge vertex span을 `baked + lerp(driftA, driftB, along)`로 이동시켜 sim relaxation 중에도 synapse가 별을 따라간다.
  - 움직임이 `DRIFT_EPS` 이하이면 position upload를 건너뛴다.

## SynapseDust
- filament 주변에 작은 additive mote를 뿌려 연결이 은하수 먼지길처럼 읽히게 한다.
- edge cap은 filament와 같은 300개다.
- total mote cap은 `MAX_DUST = 14000`이다.
- edge 길이와 intensity에 따라 mote 수를 계산하고, bowed center curve 주변으로 deterministic scatter한다.
- endpoint color gradient, per-mote seed/brightness, additive material을 사용한다.
- manual `uTime`으로 twinkle을 구동하고 `uDim`으로 focus dim을 적용한다.
- live-follow도 filament와 같은 방식이다.
  - mote가 속한 edge index와 along t를 저장한다.
  - 매 프레임 endpoint drift를 계산하고 mote matrix를 갱신한다.

## 누락 좌표와 edge filtering
- `positionOf(id)`가 null이면 해당 edge는 geometry build에서 건너뛴다.
- 두 endpoint가 거의 같은 좌표이면 direction을 만들 수 없으므로 건너뛴다.
- 렌더러는 누락 좌표를 throw하지 않는다.
- edge/stars가 비면 `UniverseSynapses`는 null을 반환한다.

## Flutter 동기화 포인트
- Flutter 렌더러도 endpoint 좌표를 edge에 캐시하지 말고 star live buffer/index map에서 읽는다.
- 시냅스 강도는 `weight`만이 아니라 `weight * brightness`, `reinforcedRecency`, `coActivationCount`를 함께 반영한다.
- endpoint color는 두 별의 mood color gradient다. 사용자 emotion color override를 양쪽 렌더러가 같이 적용해야 한다.
- layout settle snapshot으로 geometry를 bake하되, sim relaxation 중 live buffer drift를 endpoint interpolation으로 따라가야 한다.
- diary highlight의 내부 연결 강조는 link type이 아니라 "두 endpoint가 강조 star set에 포함되는가"로 판단한다.
