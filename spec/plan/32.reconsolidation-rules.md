# 32. reconsolidation rules

> Reconsolidation rules — the pure domain rules that turn a recall-time rewrite into engram change: the prediction-error
> gate (an **LLM semantic compare** of the rewrite vs. the current memory text), the `Reshape` seed change on error
> ([V5]), the `EffectiveStrength` recall bump ([R3]), and the neighbor forgetting ± nudge ([R5]) — plus the two schema
> additions those rules write into (`forgetting_offset_days` and the append-only `memory_provenance` table). The `Diary`
> is never touched ([I2][R7]).
> Scope: BE | Status: planning
>
> Layer D (domain) — the first unit of Epic C (Recall & Reconsolidation). Home: the core-domain `internal/memory`
> context (be-architecture: recall/reconsolidation is the memory aggregate's behavior). It provides the pure predicates
>
> - numeric functions and the migration; the **orchestration** (the `Recall`/`Reconsolidate`/`Reinforce` use-cases, the
>   transaction, provenance appends on rewrite) is [33.recall-usecase](33.recall-usecase.md), and the recall UI is
>   [36.recall-flow-ui](36.recall-flow-ui.md). It fills the `current_text` / `seed` / `semantic_stages` / `decay_stages`
>   reconsolidation seams that [16.memory-aggregate-schema](16.memory-aggregate-schema.md) reserved.

## Purpose

In cosimosi a recall is not a replay — it is a _rewrite_. The user re-reads their own faded memory text and writes it
again from memory; the objective record stays in the `Diary`, but the _engram_ (the star's text, shape, and strength)
changes ([R1][R6], Nader 2000; Sinclair & Barense 2019). This unit owns the neuroscience of _how_ that change is decided
and shaped — pure, testable domain rules, distinct from _when_ recall runs (the use-case) and _how_ it looks (the UI):

- **The prediction-error gate ([R6]).** Reconsolidation fires **only** when the rewrite is _semantically different_ from
  the current memory text (a content change, not wording/spacing/word-order). That judgment is an **LLM semantic
  compare**, not string math — so it is a _port_, not a golden-parity pure function.
- **`Reshape` on error ([V5]).** On prediction error the star gets a **new `seed`**, so its form visibly shifts —
  reconstruction made visual. No error → shape unchanged ([R4]).
- **The recall strength bump ([R3]).** Every recall grows the star's `EffectiveStrength` (accumulated recall → bigger
  star [V3], larger universe-color contribution [M4]).
- **Neighbor forgetting ± ([R5]).** Recalling a star nudges the _forgetting progress of its neighbors_ by how many
  **semantic** neurons they share: 1 → forgetting slows (spreading activation); ≥2 → forgetting speeds (retrieval-induced
  forgetting; Anderson 1994). The recalled star itself recovers wholly ([F5]); the ± is for neighbors only.

The last two need durable state the schema lacks, so this unit — the first thing that _overwrites_ `current_text` and
_nudges neighbors_ — introduces both: a signed **`forgetting_offset_days`** column (the neighbor nudge, read by Epic D's
decay) and the append-only **`memory_provenance`** table (the 변천사 history, [R8a][D1]). The `Diary` is never in either
as mutable state; the original is the `Diary`/export alone ([I2][R7]).

## Scope / Non-goals

**In scope**

- **The prediction-error gate as a consumer-owned LLM port** ([R6]): a `PredictionError` interface (canonical name;
  representation may differ per language) with a single semantic-compare call — _given the current memory text and the
  rewrite, is the content meaningfully different?_ → boolean (with a keyless-mock fallback, matching the AI seam of
  [22.ai-worker-pipeline](22.ai-worker-pipeline.md) / [28.ai-provider-abstraction](28.ai-provider-abstraction.md)). This
  unit **declares** the port and its contract; binding a real provider is [28.ai-provider-abstraction](28.ai-provider-abstraction.md)'s
  adapter and wiring it is [33.recall-usecase](33.recall-usecase.md).
- **The pure numeric rules** in `internal/memory` (Go) with a `packages/memory-logic` (TS) mirror where a client reads
  them (see golden-parity stance below):
  - `Reshape(seed) → seed'` — a new visual-form seed on prediction error ([V5]); deterministic given a supplied source
    of new-seed entropy (the caller passes it, keeping the fn pure).
  - the `EffectiveStrength` **recall-accumulation term** ([R3]) — this unit _fills in_ the recall-bump that
    [18.synapse-plasticity](18.synapse-plasticity.md) reserved as the Epic-A stub
    (`EffectiveStrength(base_strength, recall_count)`), so `recall_count` now grows the star's size.
  - `NeighborForgettingDelta(sharedSemanticCount) → deltaDays` — the signed forgetting nudge per neighbor ([R5]): a
    negative (slow) delta at 1 shared semantic neuron, a positive (speed) delta at ≥2, `0` at 0 (a `Depress`-style ±,
    reusing the [18.synapse-plasticity](18.synapse-plasticity.md) `Depress`/LTD _seam_ the neighbor-inhibition case
    named).
- **Schema addition A — `forgetting_offset_days REAL NOT NULL DEFAULT 0`** on `episodic_memories` (goose migration owned
  by this unit, per [05.data-persistence](05.data-persistence.md)): the signed, accumulated neighbor nudge, written on
  _neighbors_ at recall; read at decay time by [37.forgetting-decay](37.forgetting-decay.md) as
  `effectiveElapsed = max(0, (now − last_recalled_universe_time) + forgetting_offset_days)`.
- **Schema addition B — the append-only `memory_provenance` table** (goose migration owned by this unit, per CC5): the
  time-ordered 변천사 history (`kind` × `source` × `text` × `universe_time`). Reconsolidation is the first event that
  appends to it (`kind=reconsolidated`, `source=user`); the "created" baseline is **synthesized at read**, not
  backfilled.
- **The reconsolidation _rule_ shape** ([R6][C7]): on prediction error, `current_text` is replaced, `seed` is
  `Reshape`d, and only the **not-yet-created remaining** semantic/decay stage texts are regenerated — already-created
  gist stars/stages are kept (z-axis one-way). No error → reinforce only, shape/text/seed unchanged ([R4][I8]). This
  unit states the rule; executing it (the transaction, the LLM regen calls) is [33.recall-usecase](33.recall-usecase.md).
- Enumerating the `reconsolidation.*` tuning keys these rules read (the neighbor ± magnitudes, the shared-count
  threshold, the recall strength-bump coefficient).

**Non-goals**

- **No use-case / orchestration.** `Recall`, `Reinforce` (idempotent batch LTP), and `Reconsolidate` — the transaction
  that resets brightness ([R2]), potentiates co-activated synapses ([R3] via `Potentiate`), resets the semanticize timer
  ([C6a]), applies the neighbor deltas, calls the LLM regen, and appends provenance — are
  [33.recall-usecase](33.recall-usecase.md). This unit is pure rules + schema + the port contract. The
  `Recall`/`Reconsolidate`/`GetProvenance` RPCs are the use-case's / panel plans' too.
- **No forgetting or consolidation math.** This unit _writes_ `forgetting_offset_days`; the read-time decay that
  consumes it (`EffectiveBrightness`, decay-stage word removal, arousal/connection modulation) is
  [37.forgetting-decay](37.forgetting-decay.md). The gist-timer, z-axis rise, and `Downscale` (SHY) are Epic E
  ([40.semanticization](40.semanticization.md), [41.consolidation-usecase](41.consolidation-usecase.md)).
- **No economy.** Recall/gist-view spend Twinkle ([G1]), but the spend gate is a consumer-owned no-op-until-Epic-F seam
  (CC2, [44.earn-spend-usecase](44.earn-spend-usecase.md)); no price constant appears here (cost values are

- **No `Diary` mutation by reconsolidation** ([I2][R7]). No `UPDATE diaries`, no diary row in `memory_provenance` as mutable state;

- **No provenance export.** CSV/MD export ([W6][R8a]) is [46.provenance-export](46.provenance-export.md), the export

## Design

### Placement (domain-first Go)

Recall and reconsolidation are the **memory aggregate's own behavior** — a star's text/shape/strength change, its
neighbors' forgetting change, a history row appended — so everything here lives in the core-domain `internal/memory`
context (ARCHITECTURE §2.2, be-architecture: keep tightly-coupled memory behavior in one context, do **not** spin up a
new one). The pure numeric rules (`Reshape`, the recall strength term, `NeighborForgettingDelta`) and the
`PredictionError` **port declaration** sit in the context package; the two schema additions + their queries live in
`memory/pg`, the sole sqlc/pgx package (§2.6). The port is **consumer-owned** (§2.4): this unit declares the interface
and its contract, but the concrete LLM adapter is [28.ai-provider-abstraction](28.ai-provider-abstraction.md)'s and the
binding is [33.recall-usecase](33.recall-usecase.md)'s composition root — the domain depends on the _interface_, never
on an SDK.

### The prediction-error gate — an LLM port ([R6])

The gate answers one question: _is the user's rewrite semantically different in content from the current memory text,
ignoring wording, spacing, and word order?_ This is a judgment about meaning, so it is an LLM call, not string math:

```
PredictionError.Differs(currentText string, rewrite string) → bool   // consumer-owned port; canonical name
```

- **True** (content changed) → the recall is a **reconsolidation** ([R6]): `current_text` is replaced with the rewrite,
  `seed` is `Reshape`d ([V5]), remaining stage texts are regenerated ([C7]), and a `reconsolidated/user` provenance row
  is appended.
- **False** (same content, only re-worded) → **reinforce only** ([R4][I8]): brightness reset ([R2]), recall LTP + the
  `EffectiveStrength` bump ([R3]), gist-timer reset ([C6a]); `current_text`, `seed`, and the stage texts are
  **unchanged**, and **no** `reconsolidated` provenance row is appended.

**Decided (2026-07):** the gate is a **boolean semantic-compare port**, not a similarity score with a `values.yaml`
threshold. The "content vs. wording" boundary is a _semantic_ judgment ([R6] explicitly defers spacing/order/phrasing to
the model), so exposing a numeric cutoff would be a false knob — the decision lives in the prompt/model, mirroring how
[20.encode-usecase](20.encode-usecase.md)'s split boundaries are the extractor's judgment, not a tuned score. The port
reuses the [22.ai-worker-pipeline](22.ai-worker-pipeline.md) keyless-mock fallback (mock returns a deterministic default
so the loop is testable offline) and its per-call/day cost metering. Because this is an LLM port, it is **not**
golden-parity (see below).

### `Reshape` — new visual-form seed on prediction error ([V5])

A star's `seed` is a meaningless visual value that gives it a unique form ([V5]); it changes **only** on reconsolidation,
so the shape visibly shifts when the content actually changes (reconstruction made visual). Plain recall leaves the seed
alone ([R4][V5][I8]).

```
Reshape(currentSeed int64, newSeed int64) → int64      // returns the new form seed (≠ current, by construction)
```

**Decided (2026-07):** `Reshape` takes the **new seed as an argument** (the caller supplies fresh entropy — the backend
mints it, clients never mint IDs/seeds, ARCHITECTURE §5), keeping the function pure and deterministic; the rule it
encodes is "a reconsolidation produces a _different_ seed" (the function guarantees the returned seed differs from the
current one). The randomness source is the use-case's, not the domain's — the domain stays IO-free. `seed` is stored on
`episodic_memories` (nullable BIGINT, already in the schema); no schema change for `Reshape`.

### `EffectiveStrength` recall accumulation ([R3])

[18.synapse-plasticity](18.synapse-plasticity.md) seated `EffectiveStrength(base_strength, recall_count)` as an Epic-A
**stub** (`recall_count = 0` → `= base_strength`) and named the recall-accumulation term as an Epic-C seam. This unit
**fills that seam**: `recall_count` now grows the effective strength (the star's size [V3], and its universe-color
weight [M4]), monotonically and with diminishing returns, saturating toward the same single strength cap so recall can
never produce a runaway star.

```
EffectiveStrength(baseStrength float, recallCount int) → strength   // now accumulates recall, no longer a stub
```

- **Monotone non-decreasing** in `recallCount`: more recalls → not smaller.
- **Diminishing returns**, asymptotic to the strength cap (`synapse.strength_cap = 1.0`, the single cap [L9]) — the
  first few recalls matter most; a heavily-recalled star does not blow past the cap.
- `EffectiveStrength(base, 0) = base` — Epic-A behavior is preserved at `recall_count = 0` (no regression to launched,
  never-recalled stars).

**Decided (2026-07):** the accumulation shares the **saturating shape** of `Potentiate` (headroom-proportional gain) so
size growth and synapse growth read consistently, governed by one new coefficient
`reconsolidation.recall_strength_gain` (the per-recall headroom fraction). The _shape_ stays in code (a formula, excluded
from values); only the coefficient is a value. Both call sites — client render + server gating — read the same generated
constant, so this specific function is golden-parity (see stance).

### `NeighborForgettingDelta` — the neighbor ± nudge ([R5], CC4)

Recalling a star nudges the **forgetting progress of its neighbors** (the star itself recovers wholly, [F5] — the offset
is for neighbors only). A neighbor is another episodic memory that **shares one or more _semantic_ neurons** with the
recalled star (spatial/entity neurons excluded, emotion never counted — [I3]). The direction and magnitude depend on the
shared _semantic_ count:

```
NeighborForgettingDelta(sharedSemanticCount int) → deltaDays float   // signed; added to the neighbor's offset
```

| shared semantic neurons | effect                | sign               | mechanism                                                                       |
| ----------------------- | --------------------- | ------------------ | ------------------------------------------------------------------------------- |
| 0                       | not a neighbor        | `0`                | —                                                                               |
| exactly 1               | forgetting **slows**  | **negative** (`−`) | spreading activation — the weak link is co-recalled                             |
| ≥ 2                     | forgetting **speeds** | **positive** (`+`) | retrieval-induced forgetting — a strong competitor is inhibited (Anderson 1994) |

The returned `deltaDays` is a signed number of universe-days added to the neighbor's `forgetting_offset_days`. It reuses
the [18.synapse-plasticity](18.synapse-plasticity.md) **`Depress` (LTD) seam** the neighbor-inhibition case named there:
the ≥2 branch is retrieval-induced weakening (LTD, associative/local — **not** `Downscale`/SHY, which is Epic E [I9]),
and the 1 branch is its spreading-activation counterpart. `NeighborForgettingDelta` is the pure sign+magnitude rule; the
use-case computes each neighbor's shared-semantic count and applies the delta.

**Decided (2026-07):** the ± magnitudes are **two separate values** (`reconsolidation.neighbor_slow_days` for the 1-share
slow-down, `reconsolidation.neighbor_speed_days` for the ≥2-share speed-up) rather than one symmetric magnitude — spreading
activation and retrieval-induced forgetting are different mechanisms and need independent tuning ([R5] treats them as a
unified _feature_ but distinct _effects_). The shared-count boundary is fixed by the PRD at 1 vs ≥2, so the threshold is
a **constant `reconsolidation.neighbor_speed_threshold = 2`** (the count at/above which the sign flips to speed) — a
value so the boundary is not hardcoded, even though the PRD pins its number.

### Schema addition A — `forgetting_offset_days` (CC4)

A signed offset column on `episodic_memories`, added by this unit (the owner of [R5]) and **read** by
[37.forgetting-decay](37.forgetting-decay.md):

```sql
-- Added to episodic_memories by this unit's goose migration ([R5], CC4). Signed, accumulated neighbor forgetting
-- nudge in universe-days. Written on RECALL to a star's NEIGHBORS (never to the recalled star itself — the star
-- recovers wholly [F5]). Epic D reads it: effectiveElapsed = max(0, (now − last_recalled_universe_time) + offset).
-- Negative slows forgetting (spreading activation), positive speeds it (retrieval-induced forgetting). Never deletes
-- a row [I1] — forgetting is a read-time offset, not a delete.
ALTER TABLE episodic_memories ADD COLUMN forgetting_offset_days REAL NOT NULL DEFAULT 0;
```

- **`NOT NULL DEFAULT 0`** — every existing memory reads a zero offset (no backfill needed; a never-recalled-as-neighbor
  star is unaffected). Additive: each recall of a shared-neighbor `+=` the signed delta.
- **Consumed, not read here.** This unit _defines_ the column and _writes_ it (via the use-case); the read-time decay
  that folds it into `effectiveElapsed` is [37.forgetting-decay](37.forgetting-decay.md) — this plan names that seam and
  fixes the read formula so Epic D has one contract to implement.
- The `max(0, …)` clamp lives in Epic D's read (a formula, code), so a large negative offset can slow forgetting to a
  standstill but never runs the clock backward past "now" ([I10] spirit — forgetting elapsed never goes negative).

### Schema addition B — `memory_provenance` (CC5)

The append-only 변천사 history ([R8a][D1]) — a time-ordered list of a memory's representational events with a kind label
and a source. Introduced here because reconsolidation is the **first event that overwrites `current_text`**, so its
prior→new transition must be logged:

```sql
-- memory_provenance — append-only 변천사 (variant history) for a memory's representation ([R8a][D1]). One row per
-- representational event. NEVER updated while retained (append-only, [I1]); user full-delete retention sweep may delete
-- it with the parent memory. The Diary is NOT represented here as mutable — the objective original is the Diary/export
-- while retained ([I2][R7]).
CREATE TABLE memory_provenance (
    id                 TEXT PRIMARY KEY,                 -- nanoid, backend-minted (ARCHITECTURE §5)
    user_id            TEXT NOT NULL,                    -- per-user isolation [U1]
    episodic_memory_id TEXT NOT NULL REFERENCES episodic_memories(id) ON DELETE CASCADE,
    kind               TEXT NOT NULL,                    -- 'created' | 'semanticized' | 'reconsolidated'
    source             TEXT NOT NULL,                    -- 'original' | 'system' | 'user'
    text               TEXT NOT NULL,                    -- the representation text at this event
    universe_time      DATE NOT NULL,                    -- when (in universe-time [T]) the event happened
    created_at         TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

- **Append-only while retained, never mutated** ([I1]): the panel reads it in time order; nothing updates a row.
  Letting-go/full-delete soft-delete excludes the _memory_ from dynamics but does not rewrite its history; the post-window
  full-delete sweep may delete provenance with the parent memory as the user-origin [X] exception.
- **This unit appends `reconsolidated`/`source=user`** rows (on prediction-error rewrite — logged by the use-case,
  [33.recall-usecase](33.recall-usecase.md)). [40.semanticization](40.semanticization.md) appends `semanticized`/`system`
  rows as gist stages rise ([C7]). No `kind=created` row is written by any event.
- **The `created`/`original` baseline is synthesized at READ** (CC5): the first entry is derived from the memory's
  creation facts (`created_universe_time` + the initial account text), **not** backfilled into the table. So a memory
  with zero reconsolidations still shows a one-entry 변천사 (its creation), and no Epic-A data migration is required.
  [35.star-detail-panel](35.star-detail-panel.md) reads baseline + rows; [46.provenance-export](46.provenance-export.md)
  is the export owner-of-record.
- **`kind`/`source` are small closed enums** (code content, not values — the values exclusion rule): kept as `TEXT`
  columns validated by the domain, not a Postgres enum type, to stay migration-light and match the schema's other
  string-typed enums (`neuron_type`, `jobs.kind`).
- **Per-user isolation** (`user_id`, `pnpm lint:persistence`) — every provenance read/write scoped to the authenticated
  user (§4).

### The reconsolidation rule shape ([R6][C7]) — stated, not orchestrated

This unit fixes _what must be true_ after a recall; the transaction that makes it true is [33.recall-usecase](33.recall-usecase.md):

- **Prediction error (Differs = true):** `current_text ← rewrite`; `seed ← Reshape(seed, newSeed)` ([V5]); the
  **not-yet-created remaining** `semantic_stages`/`decay_stages` texts are regenerated from the new `current_text`
  ([C7], an LLM call the use-case makes) while **already-created** gist stars/stages are kept unchanged (z-axis one-way);
  a `reconsolidated/user` provenance row is appended. Plus every no-error effect below (a reconsolidation is also a
  recall).
- **No prediction error (Differs = false):** reinforce only ([R4][I8]) — brightness reset ([R2]), recall LTP on
  co-activated synapses ([R3] via `Potentiate`), the `EffectiveStrength` recall bump ([R3]), gist-timer reset ([C6a]) —
  and `current_text`, `seed`, and stage texts are **left unchanged**; no `reconsolidated` provenance row.
- **In both cases:** `recall_count += 1`, `last_recalled_universe_time = now`, and each neighbor's
  `forgetting_offset_days += NeighborForgettingDelta(sharedSemanticCount)`; the **recalled star itself** recovers wholly
  ([F5]) and gets **no** self-offset. The `Diary` is untouched ([I2][R7]).

### Golden-parity stance (explicit, per plan 18's precedent)

This unit is a **mix**, and the boundary is deliberate:

- **The numeric pure rules are golden-parity** where a client reads them. `EffectiveStrength` (with the recall term now
  live) is rendered client-side every frame for star size ([V3]) and computed server-side for gating, so it stays in the
  `internal/memory` (Go) ↔ `packages/memory-logic` (TS) golden-fixture set of [18.synapse-plasticity](18.synapse-plasticity.md).
  `Reshape` and `NeighborForgettingDelta` are pure and deterministic and are pinned by the same harness (the client does
  not currently _drive_ recall, but keeping them parity-pinned costs nothing and prevents drift when the client
  simulation grows).
- **The prediction-error gate is NOT golden-parity.** It is an **LLM port** (consumer-owned, reusing the AI seam),
  server-side only, non-deterministic by nature — exactly the opposite of a pure mirror. Contrast
  [18.synapse-plasticity](18.synapse-plasticity.md), where the whole surface is pure and byte-parity-pinned. The gate is
  covered by _contract_ tests (the mock returns a deterministic default; the interface is honored) rather than golden
  fixtures. **Decided (2026-07):** the parity line is "numeric domain rules = parity; the semantic-compare gate = port,
  not parity", stated here so [33.recall-usecase](33.recall-usecase.md) wires the two differently without re-litigating.

### Purity & placement (ARCHITECTURE §2.4, §2.5)

- The numeric rules live in the pure domain of `internal/memory` (Go) + `packages/memory-logic` (TS) — no `pg`, `rpc`,
  SDK, clock, or randomness import; the new-seed entropy and the shared-semantic counts are passed in as arguments.
- The `PredictionError` port is an interface in the context package (consumer-owned); the concrete adapter is outside
  the domain (§2.4). No proto DTO or sqlc row crosses into the domain rules (§2.4 anti-corruption boundary).
- The two schema additions + their queries live only in `memory/pg` (§2.6); no `dbgen` row escapes into the domain.
- No formula is a value: only the coefficients/thresholds below go to `spec/values.yaml`; the saturating shape, the
  `Reshape` guarantee, the `max(0, …)` clamp, and the kind/source enums are code/content.

## Acceptance Criteria

1. The prediction-error gate is a **consumer-owned LLM port** (`PredictionError.Differs(currentText, rewrite) → bool`)
   declared in `internal/memory`, reusing the [22.ai-worker-pipeline](22.ai-worker-pipeline.md) keyless-mock fallback +
   cost metering; the domain depends on the interface, and no LLM SDK is imported into the domain ([R6], ARCHITECTURE §2.4).
2. On **prediction error (Differs = true)**, the reconsolidation rule requires `current_text ← rewrite`, `seed ←
Reshape(...)` ([V5]), regeneration of **only the not-yet-created remaining** stage texts with already-created gist
   stages kept ([C7]), and a `reconsolidated/user` provenance row ([R6][R8a]). (Executing it is [33.recall-usecase](33.recall-usecase.md).)
3. On **no prediction error (Differs = false)**, the rule requires **reinforce only** — brightness reset ([R2]), recall
   LTP + strength bump ([R3]), gist-timer reset ([C6a]) — with `current_text`, `seed`, and stage texts **unchanged** and
   **no** `reconsolidated` provenance row ([R4][I8]).
4. `Reshape(currentSeed, newSeed)` returns a seed that **differs** from `currentSeed`, is pure/deterministic given the
   supplied `newSeed`, and is invoked **only** on reconsolidation — plain recall never changes `seed` ([V5][R4][I8]).
5. `EffectiveStrength(base, recallCount)` is **monotone non-decreasing** in `recallCount`, exhibits **diminishing
   returns** asymptotic to `synapse.strength_cap` (never exceeding it), and satisfies `EffectiveStrength(base, 0) = base`
   (no regression to the Epic-A stub) ([R3][V3], [18.synapse-plasticity](18.synapse-plasticity.md)).
6. `NeighborForgettingDelta(sharedSemanticCount)` returns `0` at `0`, a **negative** delta at exactly `1` (slow
   forgetting), and a **positive** delta at `≥ reconsolidation.neighbor_speed_threshold (= 2)` (speed forgetting); the
   shared count is over **semantic neurons only** (spatial/entity excluded, emotion never counted — [I3]) ([R5], CC4).
7. A goose migration adds `episodic_memories.forgetting_offset_days REAL NOT NULL DEFAULT 0`; `pnpm db:migrate` applies
   cleanly and `pnpm db:down` reverses cleanly; existing rows read `0`; the read-time contract
   `effectiveElapsed = max(0, (now − last_recalled_universe_time) + forgetting_offset_days)` is fixed for
   [37.forgetting-decay](37.forgetting-decay.md) to consume (CC4).
8. A goose migration creates append-only-while-retained `memory_provenance (id, user_id, episodic_memory_id, kind, source,
text, universe_time, created_at)` with `episodic_memory_id → episodic_memories(id) ON DELETE CASCADE`; it is **never**
   UPDATE-d, and is DELETE-d only by Epic H's post-window user full-delete sweep; the **created/original baseline is
   synthesized at read**, not backfilled (a memory with zero reconsolidations still yields a one-entry 변천사) (CC5,
   [R8a][D1]).
9. **The `Diary` is never mutated** by any rule here: no `UPDATE diaries`, and `memory_provenance` carries no
   diary-as-mutable row — the objective original stays the `Diary`/export ([I2][R7]).
10. On recall, the ± offset applies to **neighbors only** (`forgetting_offset_days += delta`); the **recalled star
    itself** takes no self-offset and recovers wholly ([F5][R5]).
11. The neighbor nudge reuses `Depress`/LTD (associative/local) as its ≥2 mechanism and is **distinct** from `Downscale`
    (SHY, homeostatic/sleep, Epic E) — no `Downscale` is implemented here ([I9]).
12. **Golden-parity stance holds:** the numeric rules (`EffectiveStrength` recall term, `Reshape`,
    `NeighborForgettingDelta`) are pure and pinned by the [18.synapse-plasticity](18.synapse-plasticity.md) TS↔Go
    golden-fixture harness reading the same generated constants; the **prediction-error gate is a port, not golden-parity**,
    covered by contract tests instead.
13. Every `memory_provenance` and `forgetting_offset_days` read/write is **scoped to the authenticated user** and `pnpm
lint:persistence` passes ([U1], §4).
14. No formula, curve shape, clamp, or enum content is stored in `spec/values.yaml`; only the coefficients/threshold
    listed below are values.
