# 48. demo-guided-tour - 데모 튜토리얼모드 스포트라이트 투어

> plan 47의 데모 자유모드가 완료됐다는 전제에서, `기능 하나하나 알아보기`가 우주 HUD 버튼을 하나씩 하이라이트하며 안내하는 튜토리얼모드로 진입하게 한다. 특별한 게임 미션이 아니라 cosimosi의 어두운 우주·별빛 톤에 맞춘 가벼운 스포트라이트 투어다. | 범위: FE | 상태: 기획

## 목적

plan 47은 최초 체험 진입을 `페르소나 선택 → 튜토리얼/자유모드 선택`으로 나눴다. 자유모드는 바로 만져 보는 길이고, 튜토리얼모드는 처음 온 사용자가 우주 안의 버튼을 하나씩 알아보는 길이다.

이 plan의 목적은 `기능 하나하나 알아보기`를 더 이상 TBD로 두지 않고, 실제 우주 화면 위에서 버튼을 순서대로 짚어 주는 얇은 안내 레이어로 만드는 것이다. 사용자가 긴 설명서를 읽는 대신, 현재 화면에 있는 버튼이 빛으로 강조되고 작은 안내문이 따라붙는다. 다음/이전/건너뛰기로 진행하고, 끝나면 같은 페르소나의 자유모드 우주로 자연스럽게 이어진다.

튜토리얼은 과한 미션이나 보상 시스템이 아니다. 디자인은 cosimosi 컨셉과 맞게, 어두운 우주 위에 별빛 테두리와 은은한 dim을 얹는 정도로 충분하다. 핵심은 사용자가 "아, 이 버튼이 이거구나"를 한 번씩 보고 들어가게 하는 것이다.

## 범위 / 비목표

**범위**

- plan 47의 모드 선택 화면에서 `기능 하나하나 알아보기`를 누르면 튜토리얼모드로 진입한다.
- 튜토리얼모드는 plan 47의 자유모드 우주 셸과 같은 HUD를 사용하되, 그 위에 스포트라이트/코치 카드 레이어를 얹는다.
- 버튼·표면을 순서대로 하이라이트한다: UI 숨김/보이기, 테마, 페르소나, 시간, 햄버거 사이드바, 시점 전환, 망원경, 망원경의 `일기`/`별` 탭, 하단 `새 별 띄우기`, 별 클릭/회상.
- 각 단계는 사용자가 직접 버튼을 눌러 볼 수 있게 하되, 진행을 반드시 액션 성공에 묶지 않는다. `다음`으로 넘길 수 있고, 필요한 표면은 튜토리얼이 열고 닫을 수 있다.
- `이전`, `다음`, `건너뛰기`, `자유롭게 탐험하기`, `다시 보기` 동선을 제공한다.
- 튜토리얼 진행 상태는 데모 세션에만 저장하고, 체험 종료 시 함께 사라진다.

**비목표**

- 복잡한 미션, 점수, 보상, 업적, 애니메이션 컷신을 만들지 않는다.
- 실제 계정 우주의 온보딩이나 강제 튜토리얼을 만들지 않는다.
- 튜토리얼 진행을 서버에 저장하지 않는다.
- 별/시냅스/원본 일기 데이터를 튜토리얼 때문에 삭제·수정하지 않는다.
- 튜토리얼 전용 backend/proto/RPC/DB schema를 만들지 않는다.
- plan 47의 자유모드 HUD 구조 자체를 다시 바꾸지 않는다. 이 plan은 완료된 자유모드 UI 위에 안내 레이어만 얹는다.

## 설계 요점

### 데모 flow 상태

plan 47의 `tutorial_tbd` 상태를 실제 튜토리얼 상태로 확장한다.

- `tutorial`: 튜토리얼 투어가 진행 중인 상태.
- `free`: 자유모드. 튜토리얼 완료 또는 건너뛰기 후 도착하는 상태.

튜토리얼의 현재 단계 index는 데모 세션 전용 sessionStorage에 저장한다. 새로고침해도 같은 데모 세션이면 같은 단계에서 이어갈 수 있고, `exitDemoMode()`는 이 상태를 지운다. 사용자가 `자유롭게 탐험하기`로 끝내면 flow는 `free`가 되며, 이후 같은 세션에서는 온보딩/튜토리얼이 다시 자동으로 뜨지 않는다. `다시 보기`는 튜토리얼 완료 후 사용자가 명시적으로 누르는 재진입점이다.

### 스포트라이트 레이어

튜토리얼은 우주 캔버스 밖의 DOM 레이어다. 3D 씬 안에 `<Html>`을 넣지 않는다.

구성은 단순하다.

- 전체 화면 dim: 우주를 어둡게 눌러 주되 캔버스가 사라지거나 언마운트되지 않는다.
- spotlight ring: 현재 target 버튼의 DOM rect를 읽어 그 주변만 밝게 남긴다.
- coach card: 현재 단계의 제목, 짧은 문장, `이전`/`다음`/`건너뛰기` 버튼을 가진 작은 안내 카드.
- progress dots: 현재 단계 위치만 보여주는 작은 점들.

디자인 톤은 검은 우주 + 따뜻한 별빛 + 현재 mood/스킨과 어긋나지 않는 얇은 색 포인트다. 큰 마케팅 카드나 장식용 orb를 만들지 않는다. spotlight가 target을 찾지 못하면 화면 중앙 안내 카드만 띄우고 crash하지 않는다.

### target anchor 계약

튜토리얼이 버튼 위치를 안정적으로 찾도록, plan 47에서 추가된 주요 HUD 요소에 `data-tour-id` 또는 동등한 ref registry를 부여한다.

권장 target id:

- `ui-toggle`: 상단 중앙 `UI 숨기기`/`UI 보이기`.
- `theme`: 좌상단 테마 버튼.
- `persona`: 좌상단 페르소나 버튼.
- `time`: 좌상단 시간 버튼.
- `menu`: 우상단 햄버거 버튼.
- `view`: 우상단 시점 전환 버튼.
- `telescope`: 우상단 망원경 버튼.
- `explorer-diary-tab`: 망원경 시트의 `일기` 탭.
- `explorer-star-tab`: 망원경 시트의 `별` 탭.
- `new-star`: 하단 중앙 `새 별 띄우기`.
- `canvas-star`: 튜토리얼이 새 별을 만든 뒤 선택할 수 있는 데모 별 또는 별 클릭 안내용 canvas region fallback.

DOM rect는 `ResizeObserver`/`getBoundingClientRect`로 계산하고, scroll/resize/overlay open에 맞춰 갱신한다. 이 model은 DOM을 직접 참조하므로 UI 컴포넌트에 둔다. 순수 step 정의는 React/DOM/three를 import하지 않는 별도 model로 둘 수 있다.

### 단계 시나리오

튜토리얼은 자유모드의 버튼을 하나씩 보는 순서로 진행한다.

1. **시작**: "이 우주를 같이 둘러볼게요." 선택한 페르소나의 우주가 뒤에 보이고, 튜토리얼이 몇 가지 버튼만 짚어 준다고 알린다.
2. **UI 숨김/보이기**: 상단 중앙 버튼을 하이라이트한다. 우주만 보고 싶을 때 HUD를 접을 수 있음을 보여준다. 사용자가 실제로 누르면 튜토리얼 overlay는 남고, 다음 단계로 갈 때 UI를 다시 보이게 복구한다.
3. **테마**: 좌상단 테마 버튼을 하이라이트한다. `스킨`과 `감정 색`으로 우주 분위기를 바꾸는 곳임을 짧게 설명한다.
4. **페르소나**: 페르소나 버튼을 하이라이트하고, 팝오버를 열어 다른 사람의 우주로 바꿀 수 있음을 보여준다.
5. **시간**: 시간 버튼을 하이라이트하고, `하루 후`, `한 달 후`, `처음으로`가 데모 가상 시계를 움직인다는 점을 보여준다.
6. **햄버거**: 우상단 햄버거를 하이라이트하고, 체험 종료/일기/공유 같은 큰 메뉴가 여기에 모인다는 점을 보여준다.
7. **시점 전환**: 시점 전환 버튼을 하이라이트한다. 멀리서 조망하거나 가까이서 탐험하는 관점 전환이다.
8. **망원경**: 망원경 버튼을 하이라이트하고, 우주 안의 일기와 별을 찾는 입구임을 보여준다.
9. **일기 탭**: 망원경 시트를 연 뒤 `일기` 탭을 하이라이트한다. 원본 일기를 검색하고 그 일기의 별들을 조망하는 길이다.
10. **별 탭**: 같은 시트의 `별` 탭을 하이라이트한다. 일반 별과 잠든 별을 함께 찾는 곳이다.
11. **새 별 띄우기**: 하단 중앙 버튼을 하이라이트한다. 데모에서는 오늘 날짜의 랜덤 감정 별이 바로 태어난다. 사용자가 눌러 보거나, 튜토리얼이 예시 별을 하나 만든다.
12. **별 클릭/회상**: 새로 생긴 별 또는 화면의 기존 별을 가리켜, 별을 누르면 기억 조각/일기를 읽고 회상할 수 있음을 보여준다.
13. **끝**: "이제 자유롭게 탐험해볼까요?" 카드. `자유롭게 탐험하기`를 누르면 flow가 `free`가 되고 overlay가 사라진다. `다시 보기`는 첫 단계로 되돌린다.

각 단계의 문장은 짧아야 한다. 기능 설명을 화면 전체에 늘어놓지 않고, 현재 target과 직접 관련된 한두 문장만 보여준다.

### 사용자 액션 처리

튜토리얼은 "강제 클릭형"이 아니다. target을 직접 눌러 볼 수 있지만, `다음`을 누르면 그냥 넘어간다. 단, 표면을 열어야 target이 있는 단계는 튜토리얼이 필요한 표면을 열어준다.

- 페르소나/시간 단계: target 버튼을 누르면 팝오버가 열리고, coach card는 팝오버를 가리지 않는 위치로 이동한다.
- 햄버거 단계: 사이드바를 열어도 다음 단계로 넘어갈 수 있다. 다음 단계로 이동할 때 사이드바는 닫는다.
- 망원경/탭 단계: 망원경 시트를 열고 해당 탭을 하이라이트한다. 다음 단계로 넘어갈 때 필요하면 시트를 유지하거나 닫는다.
- 새 별 단계: 사용자가 `새 별 띄우기`를 누르면 기존 plan 47의 데모 랜덤 별 생성 경로를 재사용한다. 사용자가 누르지 않아도 튜토리얼이 끝나는 데 문제 없다.
- 별 클릭 단계: 별 선택/회상이 가능하면 해 보게 하고, 대상 별을 못 찾으면 "별을 눌러 기억을 읽어요" fallback 안내만 보여준다.

### 기존 UI와 충돌 방지

튜토리얼 overlay는 다른 강한 표면과 한 번에 하나만 공존한다. 단계 전환 시 사이드바, 팝오버, 망원경 시트, 회상 패널 중 현재 단계에 필요 없는 표면은 닫는다. 단, 해당 단계에서 설명하는 표면은 열어 둔다.

`UI 숨기기` 단계가 특수하다. 사용자가 UI를 숨기면 다음 target들이 사라질 수 있으므로, 튜토리얼은 다음 단계로 넘어갈 때 UI를 자동으로 보이게 복구한다. 튜토리얼 overlay 자체는 UI 숨김 대상이 아니다.

7. 페르소나/시간/햄버거/망원경 단계에서 필요한 팝오버·사이드바·시트는 튜토리얼이 열거나 닫을 수 있으며, 다음 단계에 불필요한 표면은 정리된다.
8. `UI 숨김/보이기` 단계에서 사용자가 UI를 숨겨도 튜토리얼 overlay는 남고, 다음 target 단계로 넘어가기 전 UI가 다시 보이게 복구된다.
9. `새 별 띄우기` 단계에서 사용자가 버튼을 누르면 plan 47의 데모 랜덤 별 생성 경로가 그대로 실행되고, 서버 쓰기 없이 새 별이 데모 우주에 나타난다.
10. 별 클릭/회상 단계는 기존 회상 경로를 사용한다. 튜토리얼은 별/시냅스/원본 일기를 수정·삭제하지 않는다.
11. `건너뛰기` 또는 마지막 단계의 `자유롭게 탐험하기`를 누르면 flow가 `free`가 되고 튜토리얼 overlay가 사라진다.
12. 튜토리얼 완료 뒤 같은 데모 세션에서 자유모드로 남으며, 새로고침해도 튜토리얼이 자동 재시작되지 않는다.
13. `다시 보기` 진입점으로 사용자가 명시적으로 요청하면 첫 단계부터 다시 볼 수 있다.
14. 체험 종료(`exitDemoMode`)는 튜토리얼 flow와 step 상태를 함께 지운다.
15. 실계정 우주에서는 튜토리얼모드, 스포트라이트 overlay, demo tour 상태가 나타나지 않는다.
16. 튜토리얼 overlay는 캔버스 밖 DOM으로 구현되며, 3D 씬 안 `<Html>`을 추가하지 않는다.
17. 순수 step 정의/model은 React/DOM/three를 import하지 않는다.
18. 키보드 사용자는 `Tab`/`Enter`/`Escape`로 coach card와 target을 사용할 수 있고, screen reader에는 현재 단계와 target 이름이 전달된다.
19. `prefers-reduced-motion`에서는 spotlight 이동/coach card 전환이 과한 motion 없이 즉시 또는 매우 절제된 전환으로 동작한다.
20. 백엔드·proto·RPC·sqlc 변경 없이 FE만으로 구현된다.
21. build·lint·typecheck·steiger를 통과하고, plan 47 자유모드 진입/페르소나 전환/시간 이동/망원경/새 별 생성/일반 실계정 우주에 회귀가 없다.

## 정책 영향 (policy / values)

- `policy/ux/interaction.md`: 구현 완료 후 체험(demo) 규칙에 `튜토리얼모드`를 추가한다. 튜토리얼은 실제 우주 UI 위의 spotlight tour이며, 완료/건너뛰기 후 자유모드로 수렴한다.
- `tech/overlay-shell.md`: 구현 완료 후 스포트라이트 overlay의 z-layer와 기존 사이드바/팝오버/바텀시트와의 공존 규칙을 현재 truth로 반영한다.
- `spec/values.yaml`: 새 스칼라 튜닝 수치를 요구하지 않는다. 단계 순서와 문구는 코드의 콘텐츠이며, spotlight 위치는 DOM rect에서 파생한다. 구현 중 padding/duration/threshold 같은 조절값을 새로 도입하면 `demo_tour.*` 아래 values 항목으로 추가하고 `pnpm gen:values`를 실행한다.
