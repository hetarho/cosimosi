# 48. demo-guided-tour - 데모 튜토리얼모드 스포트라이트 투어 (as-built)

> plan 47의 데모 자유모드 위에서, `기능 하나하나 알아보기`가 우주 HUD 버튼을 하나씩 하이라이트하며 안내하는 튜토리얼모드로 진입한다. 특별한 게임 미션이 아니라 cosimosi의 어두운 우주·별빛 톤에 맞춘 가벼운 스포트라이트 투어다 — 자유모드 셸 위에 dim + spotlight + coach card 레이어(`widgets/demo-tour`)만 얹는다. | 범위: FE | 상태: 구현 완료

## 목적

plan 47은 최초 체험 진입을 `페르소나 선택 → 튜토리얼/자유모드 선택`으로 나눴다. 자유모드는 바로 만져 보는 길이고, 튜토리얼모드는 처음 온 사용자가 우주 안의 버튼을 하나씩 알아보는 길이다.

이 plan의 목적은 `기능 하나하나 알아보기`를 더 이상 TBD로 두지 않고, 실제 우주 화면 위에서 버튼을 순서대로 짚어 주는 얇은 안내 레이어로 만드는 것이다. 사용자가 긴 설명서를 읽는 대신, 현재 화면에 있는 버튼이 빛으로 강조되고 작은 안내문이 따라붙는다. 다음/이전/건너뛰기로 진행하고, 끝나면 같은 페르소나의 자유모드 우주로 자연스럽게 이어진다.

튜토리얼은 과한 미션이나 보상 시스템이 아니다. 디자인은 cosimosi 컨셉과 맞게, 어두운 우주 위에 별빛 테두리와 은은한 dim을 얹는 정도로 충분하다. 핵심은 사용자가 "아, 이 버튼이 이거구나"를 한 번씩 보고 들어가게 하는 것이다.

## 범위 / 비목표

**범위**

- plan 47의 모드 선택 화면에서 `기능 하나하나 알아보기`를 누르면 튜토리얼모드로 진입한다.
- 튜토리얼모드는 plan 47의 자유모드 우주 셸과 같은 HUD를 사용하되, 그 위에 스포트라이트/코치 카드 레이어를 얹는다.
- 버튼·표면을 순서대로 하이라이트한다: UI 숨김/보이기, 테마, 페르소나, 시간, 시점 전환(+멀리서/가까이서 항해 실습, change 12), 망원경, 망원경의 `일기`/`별` 탭, 하단 `새 별 띄우기`, 별 클릭/회상.
- 행동 phase(특정 버튼·동작을 짚는 단계)는 그 행동으로만 진행하고 `다음` 버튼이 없다. 정보 phase(결과 안내·인사)만 `다음`으로 넘긴다. 필요한 표면은 튜토리얼이 열고 닫을 수 있다.
- `이전`, `다음`, `건너뛰기`, `자유롭게 탐험하기`, `다시 보기` 동선을 제공한다.
- 튜토리얼 진행 상태는 데모 세션에만 저장하고, 체험 종료 시 함께 사라진다.

**비목표**

- 복잡한 미션, 점수, 보상, 업적, 애니메이션 컷신을 만들지 않는다.
- 실제 계정 우주의 온보딩이나 강제 튜토리얼을 만들지 않는다.
- 튜토리얼 진행을 서버에 저장하지 않는다.
- 별/시냅스/원본 일기 데이터를 튜토리얼 때문에 삭제·수정하지 않는다.
- 튜토리얼 전용 backend/proto/RPC/DB schema를 만들지 않는다.
- plan 47의 자유모드 HUD 구조 자체를 다시 바꾸지 않는다. 이 plan은 완료된 자유모드 UI 위에 안내 레이어만 얹는다.

## 설계 요점

### 데모 flow 상태

plan 47의 `tutorial_tbd` 상태를 실제 튜토리얼 상태로 확장한다.

- `tutorial`: 튜토리얼 투어가 진행 중인 상태.
- `free`: 자유모드. 튜토리얼 완료 또는 건너뛰기 후 도착하는 상태.

튜토리얼의 현재 단계 index는 데모 세션 전용 sessionStorage에 저장한다. 새로고침해도 같은 데모 세션이면 같은 단계에서 이어갈 수 있고, `exitDemoMode()`는 이 상태를 지운다. 사용자가 `자유롭게 탐험하기`로 끝내면 flow는 `free`가 되며, 이후 같은 세션에서는 온보딩/튜토리얼이 다시 자동으로 뜨지 않는다. `다시 보기`는 튜토리얼 완료 후 사용자가 명시적으로 누르는 재진입점이다.

### 스포트라이트 레이어

튜토리얼은 우주 캔버스 밖의 DOM 레이어다. 3D 씬 안에 `<Html>`을 넣지 않는다.

구성은 단순하다.

- 딤 + 둥근 구멍: box-shadow 큰 spread로 현재 target 영역만 남기고 화면을 어둡게 덮는다(구멍 모서리는 glow와 같은 둥근 라운드). 투명 패널 4개가 구멍 밖 클릭을 막아, 하이라이트된 버튼만 누를 수 있고 나머지는 비활성이다.
- glow 하이라이트: 같은 둥근 구멍 둘레에 바짝 붙는 흰 테두리 + 번지는 별빛 그림자(box-shadow). target이 빛나듯 떠오른다. reduced-motion이 아니면 은은히 맥동한다.
- coach card: 현재 단계의 제목, 짧은 문장, `이전`/`다음`/`건너뛰기` 버튼을 가진 작은 안내 카드.
- progress dots: 현재 단계 위치만 보여주는 작은 점들.

캔버스는 사라지거나 언마운트되지 않는다. 디자인 톤은 검은 우주 + 따뜻한 별빛 + 현재 mood/스킨과 어긋나지 않는 얇은 색 포인트다. 큰 마케팅 카드나 장식용 orb를 만들지 않는다. target을 찾지 못하면(예: 캔버스 안 별, 시작/끝 인사) 딤이 클릭을 막지 않아 별을 직접 눌러볼 수 있게 하고 화면 중앙 안내 카드만 띄운다(crash 없음).

### target anchor 계약

튜토리얼이 버튼 위치를 안정적으로 찾도록, plan 47에서 추가된 주요 HUD 요소에 `data-tour-id` 또는 동등한 ref registry를 부여한다.

권장 target id:

- `ui-toggle`: 상단 중앙 `UI 숨기기`/`UI 보이기`.
- `theme`: 좌상단 테마 버튼.
- `persona`: 좌상단 페르소나 버튼.
- `time`: 좌상단 시간 버튼. `time-speed`: 시간 팝오버의 배속 셀렉터(change 24).
- `menu`: 우상단 햄버거 버튼.
- `view`: 우상단 시점 전환 버튼.
- `telescope`: 우상단 망원경 버튼.
- `explorer-diary-tab`: 망원경 시트의 `일기` 탭.
- `explorer-star-tab`: 망원경 시트의 `별` 탭.
- `new-star`: 하단 중앙 `새 별 띄우기`. `segment`: 작성 폼의 `별 나누기` 버튼(change 25 — 폼이 열렸을 때만 존재, 아니면 중앙 fallback).
- `canvas-star`: 별 클릭 안내용 canvas region fallback(중앙 카드).

DOM rect는 `ResizeObserver`/`getBoundingClientRect`로 계산하고, scroll/resize/overlay open에 맞춰 갱신한다. 이 model은 DOM을 직접 참조하므로 UI 컴포넌트에 둔다. 순수 step 정의는 React/DOM/three를 import하지 않는 별도 model로 둘 수 있다.

### 단계 시나리오

튜토리얼은 자유모드의 버튼을 하나씩, 직접 눌러보며 익히는 순서로 진행한다. 첫 화면부터 바로 첫 버튼(UI 숨기기)을 짚는다 — 별도 인사 카드는 두지 않는다.

1. **UI 숨김/보이기**: 상단 중앙 버튼을 하이라이트하고 "눌러보세요"라고 안내한다 → 사용자가 누르면 UI가 사라지고 "다시 눌러 켜보세요" → 다시 누르면 "잘했어요!"로 마무리한다(행동 안내형 phase). 다음 단계로 갈 때 UI는 다시 보이게 복구한다.
2. **테마**: 좌상단 테마 버튼을 하이라이트한다. `스킨`과 `감정 색`으로 우주 분위기를 바꾸는 곳임을 짧게 설명한다.
3. **페르소나**: 페르소나 버튼을 "눌러볼까요?"로 안내 → 누르면 팝오버가 열리고 하이라이트가 팝오버 위로 옮겨가 "하나 골라보세요" → 사용자가 다른 페르소나를 고르면 "저마다 뇌가 다르듯 우리의 기억도 다르게 빚어져요" 안내로 마무리한다.
4. **시간**(change 24·26): 시간 버튼을 "눌러볼까요?"로 안내 → 누르면 하이라이트가 **배속 셀렉터**로 옮겨가 "배속을 올려 시간을 흘려보세요 — 오래 못 본 별은 점점 멀어지고 어두워져요(거리가 곧 강함)" → "밤이 오면 우주가 스스로 정리돼요(약한 연결은 빛을 낮추고, 멀어진 별은 형태가 단순해져요)"로 마무리한다(이산 `한 달 후로 이동` 짚기 제거).
5. **시점 전환 + 항해 실습**(change 12): 시점 전환 버튼을 하이라이트해 두 시점을 오갈 수 있음을 짧게 알린 뒤, **직접 항해를 해보는 실습 phase**로 이어진다 — ① 멀리서 **회전**, ② 멀리서 **줌**, ③ 투어가 시점을 가까이서로 구동, ④ 가까이서 **시선**, ⑤ 가까이서 **전진**, ⑥ 마무리. 각 실습 문구는 디바이스에 따라 다르다(웹=마우스 드래그/휠/전진 버튼·`W`·↑, 모바일=한 손가락 드래그/핀치/두 손가락 세로 쓸기). 사용자가 안내된 동작으로 임계값(회전 각·줌 반경 변화·시선 각·전진 거리, `spec/values.yaml` `demo_tour`)을 넘으면 그 phase가 자동으로 다음으로 넘어간다. 실습 phase에서는 딤을 거의 투명하게, 코치 카드를 반투명하게 상단 중앙에 띄워 — 사용자가 움직일 우주가 그대로 보이게 한다. 실습 segment를 벗어나면 카메라는 기본 멀리서로 정리된다.
6. **망원경**: 망원경 버튼을 "눌러볼까요?"로 안내 → 누르면 탐색 시트가 열리고 하이라이트가 `일기` 탭으로 옮겨가 원본 일기를 검색·조망하는 길임을 보여준다.
7. **별 탭**: 시트의 `별` 탭을 하이라이트한다. 깨어 있는 별과 잠든 별을 함께 찾는 곳이다.
8. **새 별 띄우기**(change 25): 하단 중앙 버튼을 "여기서 일기를 써요"로 안내한다 → 누르면 read-only 프리셋 일기 폼이 열리고, 하이라이트가 `별 나누기` 버튼으로 옮겨가 "사건 조각마다 별이 나뉘어 태어나고, 비슷한 감정의 기억끼리는 더 굵게 이어져요"를 안내한다(조각화 fan-out).
9. **별 클릭/회상**(change 26): 화면의 별을 가리켜, 별을 누르면 기억 조각/일기를 읽고 회상할 수 있음을 보여준 뒤, "별을 누르면 모양이 조금씩 바뀌어요 — 회상이 별을 다시 빚어(재공고화) 형태·빛이 변하고, 오래 멀어진 별일수록 더 단순해져요"를 덧붙인다(딤이 클릭을 막지 않아 별을 직접 눌러볼 수 있다).
10. **끝**: "이제 자유롭게 탐험해볼까요?" 카드. `자유롭게 탐험하기`를 누르면 flow가 `free`가 되고 overlay가 사라진다. `다시 보기`는 첫 단계로 되돌린다.

메뉴(햄버거) 단계는 두지 않는다 — 메뉴 버튼은 별도 설명 없이도 자명하다.

각 단계의 문장은 짧아야 한다. 기능 설명을 화면 전체에 늘어놓지 않고, 현재 target과 직접 관련된 한두 문장만 보여준다.

### 사용자 액션 처리

튜토리얼 진행은 XState 머신(`widgets/demo-tour/model/tour.machine.ts`)이 소유한다 — step·phase 전이·게이팅·항해 실습 자동진행을 한 순수 FSM으로 모델링하고, 컴포넌트는 스냅샷에서 파생해 렌더하며 입력을 이벤트로 보낸다. 단계는 phase로 나뉜다. **행동 phase는 그 행동(버튼 누름·UI 토글·팝오버 열림·페르소나 전환·항해 동작)으로만 진행한다 — `다음` 버튼이 없다.** 머신이 그 상태에서 `NEXT`를 받지 않아 `snapshot.can('NEXT')=false`이고, 그것으로 UI가 버튼 노출을 끈다(명령형 분기 없음). **정보 phase**(결과 안내·인사)만 `다음`으로 넘긴다. 모든 단계의 마지막 phase는 정보 phase라 단계 전환이 막히지 않는다. `이전`은 phase/step을 거슬러 가고, `건너뛰기`는 투어 전체를 종료한다. 망원경 탭처럼 표면을 열어야 target이 있는 단계는 튜토리얼이 그 시트를 열어준다.

- 페르소나/시간 단계: 먼저 버튼을 짚고("눌러보세요"), 사용자가 누르면 팝오버가 열리며 하이라이트가 팝오버 위로 옮겨간다 — coach card가 팝오버를 가리지 않게 버튼을 먼저, 팝오버를 나중에 짚는다. 팝오버를 연 행동·페르소나 전환·시간 이동은 페이지가 머신에 이산 이벤트로 알리고, 머신의 가드가 충족을 판정한다(페르소나 전환·시간 이동은 phase 진입 baseline 대비 변화로).
- 망원경/탭 단계: 망원경 시트를 열고 해당 탭을 하이라이트한다. 다음 단계로 넘어갈 때 필요하면 시트를 유지하거나 닫는다.
- 새 별 단계: 사용자가 `새 별 띄우기`를 누르면 plan 47의 프리셋 작성 폼이 열리고, 안내가 `별 나누기` 조각화 fan-out을 짚는다(change 25). 정보 phase(`다음`)라 누르지 않아도 진행에 문제 없다.
- 별 클릭 단계: 별 선택/회상이 가능하면 해 보게 하고, 대상 별을 못 찾으면 "별을 눌러 기억을 읽어요" fallback 안내만 보여준다.
- 시점 전환 실습 단계(change 12): 항해 입력을 **관찰만** 한다(조작·물리·제스처 규칙 불변). 머신이 invoke한 `navSampler` 액터(ui/pages가 provide)가 `navigation-input` 싱글턴의 단조 누적 카운터(시선·전진)와 회전·줌 카운터를 **rAF로 샘플링**해 phase 진입 baseline 대비 임계 도달 순간에 `PRACTICE_MET`를 한 번 보낸다 — 매 프레임 React state·머신 이벤트를 만들지 않는다(헌법4). 가까이서로의 전환은 투어가 `TOGGLE_MODE`로 구동하고, segment를 벗어나면 기본 멀리서로 되돌린다. 웹 전진은 D-pad 전진 버튼(보조 `W`·↑), 모바일 전진은 두 손가락 세로 쓸기로 충족된다. 실습 phase는 그 동작으로만 진행하고(`다음` 없음), `건너뛰기`로 투어를 종료할 수 있다(접근성 불변).

### 기존 UI와 충돌 방지

튜토리얼 overlay는 다른 강한 표면과 한 번에 하나만 공존한다. 단계 전환 시 사이드바, 팝오버, 망원경 시트, 회상 패널 중 현재 단계에 필요 없는 표면은 닫는다. 단, 해당 단계에서 설명하는 표면은 열어 둔다.

`UI 숨기기` 단계가 특수하다. 사용자가 UI를 숨기면 다음 target들이 사라질 수 있으므로, 튜토리얼은 다음 단계로 넘어갈 때 UI를 자동으로 보이게 복구한다. 튜토리얼 overlay 자체는 UI 숨김 대상이 아니다.

## 수용 기준 (인수 조건)

1. plan 47의 모드 선택에서 `기능 하나하나 알아보기`를 누르면 `tutorial` flow로 진입하고, 선택한 페르소나의 데모 우주가 뒤에 보인다.
2. 튜토리얼 시작 시 우주 캔버스는 언마운트되지 않고, 그 위에 dim + spotlight + coach card가 표시된다.
3. 튜토리얼은 `UI 숨김/보이기`, 테마, 페르소나, 시간, 시점 전환(+멀리서 회전·줌 / 가까이서 시선·전진 항해 실습, change 12), 망원경, `일기` 탭, `별` 탭, `새 별 띄우기`, 별 클릭/회상을 순서대로 안내한다(메뉴 단계 없음).
4. 각 단계는 현재 대상 버튼 또는 표면을 시각적으로 하이라이트한다. 대상 DOM을 찾지 못해도 튜토리얼은 crash하지 않고 중앙 안내 fallback을 보여준다.
5. coach card에는 현재 단계 제목/짧은 설명/진행 표시/`이전`/`건너뛰기`가 있고, 정보 phase에는 `다음`이 함께 있다.
6. 행동 phase는 하이라이트된 target을 직접 누르거나 안내된 동작으로만 진행하며 `다음` 버튼이 없다(머신이 `NEXT`를 받지 않아 `can('NEXT')=false`). 정보 phase에서만 `다음`이 보이고 그것으로 진행한다.
7. 페르소나/시간/망원경 단계에서 필요한 팝오버·시트는 튜토리얼이 열거나 닫을 수 있으며, 다음 단계에 불필요한 표면은 정리된다.
8. `UI 숨김/보이기` 단계에서 사용자가 UI를 숨겨도 튜토리얼 overlay는 남고, 다음 target 단계로 넘어가기 전 UI가 다시 보이게 복구된다.
9. `새 별 띄우기` 단계에서 사용자가 버튼을 누르면 plan 47의 프리셋 작성 폼이 열리고 `별 나누기` 조각화 안내로 이어진다(change 25). 서버 쓰기 없이 새 별이 데모 우주에 나타난다.
10. 별 클릭/회상 단계는 기존 회상 경로를 사용한다. 튜토리얼은 별/시냅스/원본 일기를 수정·삭제하지 않는다.
11. `건너뛰기` 또는 마지막 단계의 `자유롭게 탐험하기`를 누르면 flow가 `free`가 되고 튜토리얼 overlay가 사라진다.
12. 튜토리얼 완료 뒤 같은 데모 세션에서 자유모드로 남으며, 새로고침해도 튜토리얼이 자동 재시작되지 않는다.
13. `다시 보기` 진입점으로 사용자가 명시적으로 요청하면 첫 단계부터 다시 볼 수 있다.
14. 체험 종료(`exitDemoMode`)는 튜토리얼 flow와 step 상태를 함께 지운다.
15. 실계정 우주에서는 튜토리얼모드, 스포트라이트 overlay, demo tour 상태가 나타나지 않는다.
16. 튜토리얼 overlay는 캔버스 밖 DOM으로 구현되며, 3D 씬 안 `<Html>`을 추가하지 않는다.
17. 순수 step 정의/model은 React/DOM/three를 import하지 않는다.
18. 키보드 사용자는 `Tab`/`Enter`/`Escape`로 coach card와 target을 사용할 수 있고, screen reader에는 현재 단계와 target 이름이 전달된다.
19. `prefers-reduced-motion`에서는 spotlight 이동/coach card 전환이 과한 motion 없이 즉시 또는 매우 절제된 전환으로 동작한다.
20. 백엔드·proto·RPC·sqlc 변경 없이 FE만으로 구현된다.
21. 시점 전환 항해 실습(change 12): 멀리서는 회전·줌, 가까이서는 시선·전진 두 동작을 차례로 안내하고, 실습 문구는 디바이스(마우스/터치)별로 다르다. 안내된 동작으로 임계값(`spec/values.yaml` `demo_tour`)을 넘으면 자동으로 다음 phase로 진행한다(실습 phase는 `다음` 없음; `건너뛰기`로 종료 가능). 신호 관찰은 머신이 invoke한 `navSampler` 액터의 rAF 샘플링으로만 이뤄져 매 프레임 React state·머신 이벤트를 만들지 않고(헌법4), 항해 조작·물리·제스처 규칙은 불변이다. 실습 구간 딤·코치 카드는 거의 투명해 우주가 보인다.
21. build·lint·typecheck·steiger를 통과하고, plan 47 자유모드 진입/페르소나 전환/시간 이동/망원경/새 별 생성/일반 실계정 우주에 회귀가 없다.
