# 40. 표상 부동 — 거리=강함 강제 + 위치 드리프트 (representational drift)

> 38이 "거리=강함, 각도=연결"을 도입했지만 **연결 스프링이 거리축을 압도해** 새 별이 중앙이 아니라 이웃(주로 바깥의 오래된 별) 곁으로 끌려가고, 이웃 없는 새 별은 황금각 **나선**으로 떨어졌다. 본 스펙은 (A) force-sim에서 **연결·척력을 접선 성분만** 남겨 거리축을 반지름 셸 힘이 독점하게 해 "거리=강함"을 강제하고(새 별=중앙·나선 제거), (B) **표상 부동**을 위치로 옮긴다 — 별의 *방향*이 **밤마다 한 스텝씩 천천히 표류**한다(실시간 변화 없음·야간 공고화 cadence). 거리는 회상으로 안, 시간으로 밖(38 그대로), 방향은 시간에 따라 흐른다. | 범위: FE | 선행: 38 | 차단: —

## 목적 (Why)

38은 자아 중심 + 반지름=강함을 코드에 넣었지만, **두 가지가 의도대로 동작하지 않는다**(2026-06-15 진단·코드 검증 + 동역학 수치 확인):

1. **새 별이 중앙이 아니라 바깥/사이드에 생긴다.** `radiusOf`는 갓 만든 별(activation≈1)의 목표 반지름을 올바르게 중앙 근처(`R_MIN=6`~16)로 잡고 시드도 `atRadius`로 중앙에 놓는다. 하지만 `sim.step()`의 **링크 스프링 계수(`weight` 0.6~0.8)가 반지름 셸 힘 계수(`radialStrength=0.1`)보다 6~8배 강해서**, 새 별이 자기가 이어진 이웃 쪽으로 *거리까지* 끌려간다. 데모에서 새 별은 거의 항상 같은 mood의 **오래된**(=바깥 반지름 ~40) 별과 semantic 링크가 생기므로, 이웃에서 링크길이(14)만큼 안쪽인 **~24 반지름**에 정착한다(콜리니어 평형 `x=(Rn·w−L·w+R·ks)/(w+ks)≈23~24`). "거리=강함"이 연결 스프링에 짓눌린다.

2. **이웃 없는 새 별은 나선으로 떨어진다.** `seedNearCluster`의 fallback이 `fibonacciStarPosition(i,…)`(θ=`GOLDEN·i`, 황금각)이라, 별을 연달아 추가하면 인덱스 `i`가 늘며 각도가 황금각으로 행진해 **나선**을 그린다(`atRadius`가 반지름은 중앙으로 재스케일하나 *각도*는 나선 그대로).

또한 38은 **위치를 settle 후 동결**한다(`alpha→alphaMin`에서 tick 정지, 각도는 `prevPos`로 영구 재개, 접선력 없음). 반지름만 시간에 따라 바깥으로 미끄러질 뿐, 방향은 절대 안 변한다 — concept §"표상 부동을 위치에도 흘려보낸다"(NEUROSCIENCE.md §6.1)가 ⏳ 미구현 상태다.

본 스펙은 둘을 함께 푼다 — 같은 메커니즘(접선/radial 분리)을 토대로 (A) 거리축을 셸 힘이 독점하게 하고, (B) 그 위에 **밤마다 한 스텝씩 방향을 흘리는 표상 부동**을 얹는다. 사용자 의도: *실시간으로는 움직이지 않고, 시간이 지나면(야간 공고화 cadence) 자연스레 바뀐다.*

## 범위

- **포함:**
  - **접선/radial 분리(거리=강함 강제).** `shared/lib/force-sim/sim.ts`의 `step()`에서, **목표 반지름이 있는(`radius>0`) free 노드**는 누적된 연결·척력 속도의 **반지름(방사) 성분을 제거**하고 **접선 성분만** 남긴다 — 연결·척력은 별을 셸 *위에서 회전(각도)* 시키기만 하고, 거리(반지름)는 **반지름 셸 스프링이 독점**한다. `radius=0`(미지정) 노드는 기존 `centerGravity` 거동 그대로(07/38 계약 보존).
  - **새 별 시드 나선 제거.** 이웃이 아직 배치 안 된 새 별의 fallback 방향을 황금각 인덱스 나선(`fibonacciStarPosition(i,…)`)에서 **id 시드 기반 분산 방향**(`scatterDirection(seed)`)으로 교체한다. `atRadius`가 강함 반지름으로 스케일하므로 *방향*만 바뀐다 — 연달아 추가해도 나선 호가 안 생긴다. (`fibonacciStarPosition`은 버퍼 미준비 시 렌더 폴백(`readBufferPosition`)으로는 유지.)
  - **표상 부동 — 야간 각도 드리프트(데모 쇼케이스).** `shared/lib/layout.ts`에 순수 헬퍼 `applyAngularDrift(pos, seed, nights)`(**고정 per-seed 축**(`scatterDirection`) 둘레 완전 Rodrigues 회전 → **반지름 보존·방향만 회전**, 회전군이라 `drift(N) = N×drift(1)` — 시계가 점프하든 트윈하든 같은 결과) + `DRIFT_STEP_RAD`. 컨트롤러가 **데모 모드에서** 가상 시계의 밤 경계 교차(`floor(now/DAY)`)를 감지해 넘은 밤 수만큼 각 free 별의 `sim.px`를 회전(+ 그 노드 `vx` 0)·re-kick. 데모 "하루/한 달 지나기"가 트리거 — **시간이 눈에 보이게 흐르는 유일한 곳**이자 표상 부동의 쇼케이스. **실시간(밤 경계 미교차)엔 정적.**
  - **차등 드리프트(창발 — 무비용).** 드리프트는 **별마다** 접선 회전인데, 연결 스프링이 잘 연결된 성단은 부분 복원하고 고립 별은 자유로이 표류한다 → **고립 기억일수록 더 많이 흐르고, 잘 연결된(도식) 기억은 제자리에 더 머문다**(4.2 적응적 망각·2.4 도식과 정합 — 별도 코드 없이 동역학에서 창발).
- **비목표(Out of scope):**
  - **실시간 연속 드리프트**(settle 후에도 끊임없이 떠다님). 본 스펙은 밤 경계에만 한 스텝 — 평소엔 정적.
  - **서버 변경**(좌표 캐시·야간 redistribute 로직). 좌표는 클라 창발(헌법3)이라 표상 부동도 **FE에서** 야간 cadence에 적용한다. 서버 `consolidate.go`의 stable_x/y/z 캐시·redistribute는 그대로(클라가 안 읽음). 드리프트 좌표는 영속하지 않는다(리마운트 시 재시드 — 좌표는 비권위).
  - **반지름(거리) 거동 변경.** 회상→중앙·시간→바깥은 38 그대로. 본 스펙은 *각도* 축을 살아 움직이게 하고, 거리축이 연결에 안 짓눌리게 *강제*만 한다.
  - **자아 별·appearance·카메라·시냅스 렌더** 변경 없음.
  - 재공고화 재성형(밝기·색·형태, 23)·요지화(27)·λ_eff(26)는 그대로 — 본 스펙은 *위치*만 다룬다.

## 설계 요점

### (A) 접선/radial 분리 — `sim.ts step()` (순수)
`step()`의 위치 힘 + 적분 루프에서, free 노드의 누적 속도 `v`(이 틱의 척력+링크 + 직전 틱 잔여)를 `radius>0`이면 **접선화**한 뒤 셸 스프링만 방사로 더한다:
```ts
// 현재: radius>0 → v += p·((r−len)/len)·radialStrength·alpha  (셸 스프링)
//       radius=0 → v += −p·centerGravity·alpha                 (원점 중력, 07)
// 변경: radius>0 일 때
//   len = |p|
//   if len > RADIAL_MIN_DIST:
//     r̂ = p/len
//     v -= (v·r̂)·r̂                                   // 연결·척력의 방사 성분 제거 → 접선만(각도)
//     v += r̂·(radius − len)·radialStrength·alpha       // 셸 스프링이 거리(반지름) 독점
//   else: v += +x 방향 nudge (기존 — 원점 탈출)
//   radius=0 분기·damp·maxSpeed 클램프·적분은 불변
```
- 효과: **연결은 별을 강함 셸 *위에서* 이웃 각도로 회전**시키기만 하고 거리는 못 바꾼다 → 새 별(강함↑)은 `R_MIN` 근처 셸에 머문다(이웃이 바깥이어도). 38의 "사슬형 바깥 행진"이 완전히 사라진다.
- 방사 성분 제거는 `(v·r̂)r̂`만 빼므로 **접선 운동(각도 분산·클러스터링)은 보존**된다. `radius=0` 경로 불변 → 07 단위테스트·`centerGravity` 폴백 그대로.

### (B) 새 별 시드 — 분산 방향(`layout.ts`, 순수)
```ts
/** id 시드(별 고유)에서 단위 구면 위 결정론적 방향. 황금각 인덱스 나선과 달리
 *  연속 추가해도 각도가 행진하지 않아 나선 호가 안 생긴다. atRadius가 거리를 정하므로 방향만. */
export function scatterDirection(seed: number): [number, number, number]
```
- 컨트롤러의 새 별 fallback: `fibonacciStarPosition(i,count,seed)` → `scatterDirection(s.memory.seed)`. `seedNearCluster(...)` → `atRadius(seeded, r)`는 그대로(이웃 있으면 이웃 각도, 없으면 분산 방향, 둘 다 강함 반지름).
- `readBufferPosition`(버퍼 미준비 렌더 폴백)은 `fibonacciStarPosition` 유지 — 버퍼는 `createSim` 직후 즉시 채워지므로 폴백은 일시적이라 무영향.

### (C) 표상 부동 — 야간 각도 드리프트(`layout.ts` 순수 + 컨트롤러, 데모)
```ts
export const DRIFT_STEP_RAD = 0.08   // 밤 1회당 회전(라디안, ~4.6°) — 손튜닝
/** pos를 **고정 per-seed 축**(scatterDirection(seed)) 둘레 nights·DRIFT_STEP_RAD 만큼
 *  완전 Rodrigues 회전 → |pos|(반지름) 보존·방향만 회전. 축이 pos-무관·고정이라 회전군:
 *  applyAngularDrift(pos,seed,N) === N회 단일 호출(가환·합산) → skip==wait. */
export function applyAngularDrift(
  pos: readonly [number, number, number], seed: number, nights: number,
): [number, number, number]
```
- **고정축이 핵심.** 축을 `cross(pos, k)`로 매번 pos에서 구하면, pos가 누적 회전해 축과 가까워질 때 **부호 반전 → 드리프트가 정체/진동**하고 "N일 스킵 ≠ N일 실시간"(데모 불변식 위반)이 된다. 축을 pos-무관 고정(`scatterDirection(seed)`)으로 두면 회전이 가환군이라 누적이 깔끔하고(스킵=대기), 완전 Rodrigues(축·pos 항 포함)가 `|p|`를 정확히 보존한다.
- 컨트롤러 `useFrame`(**`isDemoMode()` 게이트**): `night = floor(virtualNowMs()/DAY_MS)`. `nightRef===null`이거나 `night < nightRef`(시계 역행)면 baseline만 재설정(드리프트 0). `night > nightRef`면 `dn`만큼 각 free 별 `sim.px` 회전 + 그 노드 `vx` 0(이산 재배향 — 옛 운동량 폐기)·`nightRef=night`·re-kick. **밤 미교차엔 정적.**
- **데모 전용 이유.** 시간이 눈에 보이게 흐르는 곳은 데모 타임머신뿐이다. 프로덕션은 좌표가 세션마다 새로 창발(헌법3 — 비영속)하므로 세션 중 드리프트가 무의미하고, 사용자가 실시간 변화를 원치 않는다(2026-06-15 결정). 프로덕션 표상 부동은 나이 든 기억의 반지름·세션 재창발로 자연히 드러나며, 각도 드리프트를 morning-diff에 태우는 건 후속.
- 회전은 `|p|` 보존 → 거리=강함 유지(38). 회전 후 접선력이 재이완하며 잘 연결된 성단은 부분 복원(차등 드리프트 — 위 §범위). `nightRef`는 `useRef`로 리빌드 넘어 유지(드리프트 누적), 빈 우주(소스 리셋) 분기에서 리셋. 드리프트된 `sim.px`는 리빌드 시 `prevPos`로 재개돼 각도 연속성 유지(38).

## 수용 기준 (EARS / Given-When-Then)
1.1 GIVEN 바깥(큰 반지름)의 오래된 별과 semantic 링크로 이어진 새 별 WHEN 레이아웃이 이완되면 THE SYSTEM SHALL 새 별을 그 이웃 셸이 아니라 **자기 강함 반지름(중앙 근처)** 에 둔다(연결은 *각도*만 정함). _(거리=강함 강제)_
1.2 WHEN `radius>0`인 free 노드에 연결·척력이 작용하면 THE SYSTEM SHALL 그 힘의 **방사 성분을 제거**해 거리는 셸 스프링만 정하고 연결·척력은 셸 위 각도만 바꾼다. _(접선/radial 분리)_
1.3 WHEN `SimNode.radius`가 0/미지정이면 THE SYSTEM SHALL 기존 `centerGravity` 원점 거동을 그대로 유지한다(07/38 계약 불변). _(하위호환)_
1.4 WHEN 이웃이 아직 배치 안 된 새 별을 연달아 추가하면 THE SYSTEM SHALL `scatterDirection`으로 분산 방향에 시드해 **나선 호를 만들지 않는다**(황금각 인덱스 행진 제거). _(나선 제거)_
1.5 WHEN 데모 모드에서 가상 시계가 밤 경계를 넘으면(`floor(now/DAY)` 증가) THE SYSTEM SHALL 각 별의 *방향* 을 넘은 밤 수만큼 고정 per-seed 축으로 `DRIFT_STEP_RAD` 회전시키고(반지름 보존·`drift(N)=N×drift(1)`) 부드럽게 활강시킨다. 프로덕션은 좌표 비영속이라 세션 중 드리프트하지 않는다. _(표상 부동 — 야간 드리프트, 데모)_
1.6 WHEN 밤 경계를 넘지 않는 동안 THE SYSTEM SHALL 위치를 변경하지 않는다(settle 후 정적 — 실시간 연속 운동 없음). _(실시간 변화 없음)_
1.7 WHEN 각도가 드리프트해도 THE SYSTEM SHALL `|p|`(반지름=강함)를 보존하고 어떤 별·시냅스도 삭제하지 않는다(밝기·거리·*방향* 만 변함). _(헌법2·38 정합)_
1.8 WHEN 잘 연결된 성단과 고립 별이 함께 드리프트하면 THE SYSTEM SHALL 연결 스프링 복원으로 성단은 덜, 고립 별은 더 흐르게 한다(차등 드리프트 — 동역학 창발). _(도식 보존·고립 표류)_
1.9 WHEN `shared/lib/force-sim`·`shared/lib/layout.ts`를 빌드하면 THE SYSTEM SHALL `three`/React/DOM을 import하지 않는다. _(헌법4)_

## 과학적 근거
- 표상 부동(representational drift) — 기억은 안정적이어도 그것을 담당하는 신경 표상이 며칠~몇 주에 걸쳐 서서히 흐른다(본 스펙의 *각도* 표류): Zaki Y, Cai DJ, et al., representational drift 개관 (2024); 시간 창 기반 앙상블 접근성 Cai DJ, et al., *Nature* (2016). [DOI](https://doi.org/10.1038/nature17955)
- 연결성/도식 통합이 기억을 보존하고 고립 기억은 더 잘 흐른다(차등 드리프트의 근거): Audrain S, McAndrews MP, *Nat Commun* (2022). [DOI](https://doi.org/10.1038/s41467-022-32614-4)
- 회상이 기억을 더 견고하게(거리=강함의 회상→중앙, 38 계승): Roediger HL, Karpicke JD, *Psychological Science* (2006). [DOI](https://doi.org/10.1111/j.1467-9280.2006.01693.x)
