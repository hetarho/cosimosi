# 25. 요즘 상태 — 우주의 배경 (ambient-mood)

> 최근 조각들의 감정(mood·강도·정서가)을 시간가중 종합해 우주의 배경을 "지금의 나"로 물들이되, **단일 색조가 아니라 군데군데 넓은 광원(다중 nebula 풀)이 번지는 은은하고 불규칙한 그라디언트**로 표현하고, 종합 각성도를 할당 흥분성 게인으로 흘려보낸다. | 범위: FS | 선행: 21 | 차단: —

## 목적 (Why)
개별 별이 *과거의 한 순간*이라면 **우주 전체의 배경·앰비언트 색·분위기는 "지금의 나"** 다(concept §우주의 배경). 21이 일기 1건을 여러 **조각(fragment) 별**로 펼치면 하루는 여러 감정을 품게 되는데, 그 최근 감정들을 **7일 시간가중**으로 종합해 우주의 배경을 빚는다 — 요즘이 평온하면 잔잔한 색, 격동하면 강렬한 색. 같은 별들이라도 **오늘의 하늘색이 다르면 우주의 인상이 달라진다.** (우주의 *deep base 배경색*은 사용자가 고른 개인 설정 — plan 30 — 이고, 25의 ambient는 그 위에 번지는 감정 nebula overlay다. ambient는 base를 대체하지 않는다.)

**배경은 단일 색조가 아니라 "군데군데 넓은 광원"의 불규칙 그라디언트다(이 스펙의 비주얼 결정).** 요즘의 감정은 보통 한 가지가 아니라 *여러 감정의 분포*다 — 그래서 배경 한 겹을 평균색 하나로 칠하지 않고, **최근 우세한 감정 상위 몇 개(top-K)를 각각 하나의 넓고 은은한 광원(nebula 풀)으로** 흩뿌려 서로 screen-가산으로 번지게 한다(랜딩 `CalmBackground`/`VastBackground`의 흩뿌린 blur-orb 기법을 우주 3D로 옮긴 것). 각 광원의 **색 = 그 mood의 의미색(moodRgb)**, **밝기·또렷함 = 종합 각성(arousal)**, **온도(따뜻함↔차가움)·채도 = 종합 정서가(valence)** 가 정한다(20·21이 추출·영속한 valence를 25가 *색*으로 소비 — 감쇠 물리는 26). 평온한 단색 요즘은 광원 1~2개의 잔잔한 한 색으로, 격동·혼재된 요즘은 여러 색 광원이 불규칙하게 겹친 강렬한 하늘로 보인다.

요즘 상태는 단순 배경이 아니라 **기제**이기도 하다(concept §결정2 경쟁적 할당): 최근 활성화된 주제일수록 흥분성이 높아 새 조각을 더 잘 포섭한다. 그래서 본 스펙은 종합한 각성도(arousal)를 **흥분성 게인** `g = 1 + 0.3·arousal`로 노출해 22의 할당 바이어스 `W_EXC`를 스케일한다(22가 있으면 배선, 없으면 seam만 문서화).

## 범위
- **포함:**
  - BE 집계 쿼리(`memory.sql`): 최근 조각의 `mood·intensity·valence·last_recalled_at`을 7일 지수 감쇠로 가중해 종합. 종합 결과는 (a) 대표 요약 `ambient{hue,sat,arousal,valence}`와 (b) 각 mood의 시간가중 비중(클라가 광원 풀 색으로 쓸 분포)이다. `TAU_MOOD = 7d`.
  - proto: `GetUniverseResponse`에 `AmbientMood ambient { hue; sat; arousal; valence }` 추가(좌표 필드 없음 — 원칙3/5). 다중 광원의 *위치*는 proto에 싣지 않고(좌표 비전송 — 원칙3) 클라가 ambient 요약 + 시드로 결정론 생성.
  - BE `service.GetUniverse`가 집계 결과를 `Universe.Ambient`로 채워 핸들러가 proto에 매핑.
  - 흥분성 게인 `g = 1 + 0.3·arousal`를 도메인 헬퍼로 정의해 22의 `buildLinks` `W_EXC` 스케일에 배선(22 부재 시 seam 주석만). **게인은 arousal만**(valence는 색에만 작용; 감쇠 가중은 26).
  - FE: `UniverseCanvas`에 **다중 광원 앰비언트 레이어** — 최근 우세 감정 상위 K개(`ambientLights`)를 각각 넓고 은은한 가산(additive) nebula 풀로 흩뿌린다(카메라 뒤편 대형 빌보드/구 안쪽 면, 시드 기반 불규칙 배치, 별 구름 밖 큰 반경). 각 풀 색 = `moodRgb(mood)`를 valence로 온도·채도 보정, 밝기·난류 = arousal. 슬로우 드리프트 + ambient 변경 시 ~0.8s 색 크로스페이드. 애니메이션은 **수동 uniform**(frozen-time idiom). 단일 톤 백드롭/씬 외 DOM radial-gradient는 *대체*된다(3D 항행에 화면 고정 orb는 시차가 없어 공간감이 깨진다 — 광원은 씬 안 3D로).
  - FE `entities/memory/model`: 서버 `ambient` 미수신(체험·구버전) 시 로드된 별의 mood/intensity/valence/lastRecalledAt에서 파생하는 클라 폴백 셀렉터(`deriveAmbient` + `ambientLights`).
  - 색 기준 = 기존 `moodRgb` 팔레트(테마와 무관 — 색이 의미를 유지), valence는 그 색을 따뜻/차가움·채도로 미세 보정.
  - 랜딩 전용 카드 `AmbientMoodCard` 신설 + `LandingPage` 합성, 체험(demo) 배경 반영.
- **비목표(Out of scope):**
  - 별 단위 감쇠(`λ_eff`)·valence 감쇠저항 — **26**이 `ambient`를 `R_recent`로, valence를 `R_emo`로 소비한다. 25는 valence를 *배경 색*으로만 쓰고, 전역 배경/게인까지만.
  - 경쟁적 할당 코어(흥분성 클러스터 계산·소프트 억제·시드 위치) — **22**. 25는 게인 스칼라만 공급한다.

  - 야간 배치 흥분성 리셋·아침 diff — **27**.

## 설계 요점

**도메인(순수, 태그 금지 — 원칙5).** `internal/memory/memory.go`에 mood-벡터 종합 결과 + 게인 헬퍼:
```go
// AmbientMood: 최근 감정의 시간가중 종합("지금의 나"). 좌표/태그 없음(원칙3·5).
type AmbientMood struct {
    Hue     float64 // 0..360 대표 색조(HSV) — moodRgb 팔레트 가중합에서 환산
    Sat     float64 // 0..1 채도(감정의 또렷함 = 강도 가중치 합의 함수)
    Arousal float64 // 0..1 각성도(최근성·강도가 높을수록 ↑) → 흥분성 게인 입력
    Valence float64 // -1..1 시간가중 평균 정서가(음=차가운·무거운 하늘, 양=따뜻한 하늘) → 배경 색 보정
}
type Universe struct { Memories []Memory; Synapses []Synapse; Ambient AmbientMood } // 04에 Ambient 추가

const TauMoodDays = 7.0          // 기분의 느린 봉투(6h 흥분성보다 훨씬 느리다 — 22 TAU_EXC와 별개)
const arousalGainCoef = 0.3      // g = 1 + 0.3·arousal
// ExcitabilityGain: 요즘 상태가 새 조각의 할당을 끌어당기는 전역 게인(22의 W_EXC 스케일).
// arousal만 입력 — valence는 게인이 아니라 배경 색에 작용한다(감쇠 가중은 26).
func ExcitabilityGain(a AmbientMood) float64 { return 1 + arousalGainCoef*a.Arousal }
```
> ⚠️ 정규화는 도메인(순수)에서. 빈 우주(별 0개)면 `AmbientMood{}`(중립·Valence 0) → `ExcitabilityGain`=1.0(게인 무효). 색조 환산은 `moodRgb`와 동치인 BE 팔레트 단일 출처를 쓴다(BE는 mood→RGB 가중합 후 HSV 변환; FE 폴백과 같은 7색 매핑). `Valence`는 7일 시간가중 평균(`Σ w·valence / Σ w`, `w = intensity·exp(-Δt/τ)`). **다중 광원의 색 분포는 서버가 보내지 않는다** — 클라가 로드된 별(또는 ambient 요약)에서 top-K mood 비중을 파생해 광원 풀을 만든다(좌표·렌더 입력은 클라 권위 — 원칙3).

**집계 쿼리(`db/queries/memory.sql`, sqlc).** 시간 감쇠 가중치는 SQL에서 직접 계산하지 않고(원칙: sargable·결정론), **원자료(mood·intensity·last_recalled_at)** 만 최근 윈도로 추려 반환하고 가중 종합은 도메인(순수·테스트 가능)에서 한다 — `dormantCutoff`(12)와 같은 패턴.
```sql
-- name: ListRecentForAmbient :many
-- 요즘 상태(ambient) 종합 입력: 7일 윈도 안에서 회상/생성된 조각 별의 감정만.
-- 가중 종합(exp 감쇠·정규화)은 도메인에서 — SQL엔 감쇠 수식을 넣지 않는다(12 패턴).
-- 21 이후 mood/intensity/valence 출처는 memories(가변 별 레이어); fragment_index 무관(조각 전부 종합).
SELECT m.mood, m.intensity, m.valence, m.last_recalled_at
FROM memories m
WHERE m.user_id = $1
  AND m.last_recalled_at >= sqlc.arg(since)::timestamptz; -- since = now - TAU_MOOD·k
```
> `since`는 service가 `now - TauMoodDays·windowFactor`(예: 3·τ)로 환산해 주입 — exp 감쇠가 무시 가능한 만큼만 잘라 비용을 줄인다. `last_recalled_at`은 회상 시 21/11이 갱신하므로 "최근 떠올린 감정"이 자연히 무게를 받는다. (21 이전엔 `r.mood/r.intensity`로 JOIN — 21이 컬럼을 옮기면 본 쿼리도 함께 이동.)

**proto(`GetUniverseResponse.ambient`).** DTO에 좌표 없음(원칙3·5), unary 유지(원칙6):
```proto
// AmbientMood = "지금의 나"의 색조 요약. 별 좌표가 아닌 전역 배경(서버 권위는 가중 그래프뿐).
// 다중 광원의 위치/풀 색은 proto에 없다 — 클라가 이 요약 + 로드된 별 분포 + 시드로 생성.
message AmbientMood {
  double hue = 1;      // 0..360
  double sat = 2;      // 0..1
  double arousal = 3;  // 0..1 (흥분성 게인 입력)
  double valence = 4;  // -1..1 시간가중 평균 정서가(배경 색 온도·채도 보정; 20·21이 추출·영속)
}
message GetUniverseResponse {
  repeated Star stars = 1;
  repeated Synapse synapses = 2;
  AmbientMood ambient = 3; // 25 추가 — 비면 클라가 별에서 폴백 파생
}
```

**FE 앰비언트 레이어(`UniverseCanvas` — 플랫폼 레이어, three/TSL 여기서만).** 목표 = 랜딩 배경(`CalmBackground`/`VastBackground`의 흩뿌린 blur-orb + screen 합성)을 우주 3D로 옮긴 **다중 광원 nebula**. 단일 톤이 아니다.
- **다중 광원 풀(scattered nebula).** `ambientLights(stars|ambient, now)`가 만든 K개(기본 4~6, 우세 감정 수에 따라 가변) 광원을 각각 **넓고 부드러운 가산(additive) 빌보드**(soft radial falloff, `blur` 등가의 큰 페더)로 흩뿌린다. 위치는 **시드 기반 불규칙 배치**(별 구름 밖 큰 반경 ~120~200의 구면 위 jitter, `mulberry32`로 결정론)라 매번 같은 자리에 뜨고, 카메라가 움직이면 *3D 시차*로 깊이가 산다. 각 풀 색 = `moodRgb(그 mood)`를 valence로 보정. 풀끼리 screen-가산으로 겹쳐 **은은하고 불규칙한 그라디언트**가 된다.
- **색 보정.** 풀 밝기·또렷함(반경·알파) = `arousal`(격동할수록 크고 진하게). 색 온도·채도 = `valence`(음→차갑고 탁하게(채도↓·청록/보라 쪽), 양→따뜻하고 또렷하게(채도↑·금/장미 쪽)). `<color attach="background">`(테마색 `themeBg`)는 **깊은 베이스로 유지**하고 그 위에 광원을 가산한다(테마=깊이, mood=요즘색).
- **느린 드리프트 + 크로스페이드.** 각 풀은 랜딩 orb처럼 아주 느리게 위치·스케일이 일렁이고(시드별 위상), ambient(요즘 감정 분포)가 바뀌면 풀의 색·구성이 **~0.8s 크로스페이드**로 옮겨간다. 풀 개수가 변하면 사라지는 풀은 알파 0으로, 생기는 풀은 0→목표로 페이드.
- **frozen-time idiom(필수).** BloomPass `PostProcessing.render()`가 three nodeFrame을 진전시키지 않아 내장 `time` 노드가 멈춘다. 드리프트·크로스페이드·일렁임 uniform은 **반드시 `useFrame`에서 수동 bump**(`uMix += dt/0.8` 클램프, `uColorFrom/uColorTo`·`uPhase` 보간) — `StarField.update(elapsed)`·`Synapse* uTime`과 같은 관용. `prefers-reduced-motion`이면 드리프트·일렁임을 멈추고 색만 유지(정책: motion-accessibility).
- **구현 형태.** K개 빌보드 인스턴스(또는 큰 inside-out 스피어 1개 + TSL 프래그먼트에서 K개 가우시안 블롭 합성) 중 택1 — 둘 다 씬 안 3D, 씬 외 DOM radial-gradient는 채택하지 않는다(3D 항행에 화면 고정이라 시차가 없다). 별·시냅스보다 뒤(깊은 반경·`depthWrite=false`·낮은 renderOrder)에 둬 별빛을 가리지 않는다.
```ts
// entities/memory/model/ambient.ts (순수 — three/DOM 미의존, 원칙4)
export interface Ambient { hue: number; sat: number; arousal: number; valence: number }
// 광원 풀 하나: 의미색(moodRgb를 valence로 보정한 RGB) + 가중치(상대 밝기/크기).
export interface AmbientLight { rgb: readonly [number, number, number]; weight: number; mood: string }
export const TAU_MOOD_DAYS = 7
export const AROUSAL_GAIN = 0.3
export const AMBIENT_LIGHTS_K = 6 // 광원 상한(상위 우세 감정만; 비중 임계 미만은 버림)
export const excitabilityGain = (a: Ambient) => 1 + AROUSAL_GAIN * a.arousal
// 폴백: 서버 ambient 미수신 시 로드된 별의 mood/intensity/valence/lastRecalledAt에서 7일 가중 종합.
// moodRgb로 색 가중합 → HSV(hue/sat), arousal = Σw(=intensity·exp(-Δt/τ)) 정규화, valence = Σw·v/Σw.
export function deriveAmbient(stars: { mood: string; intensity: number; valence: number; lastRecalledAt: number }[], now: number): Ambient
export function ambientToRgb(a: Ambient): readonly [number, number, number] // 대표색(moodRgb 공간 + valence 보정)
// top-K 우세 감정 → 광원 풀 목록. mood별 시간가중 비중 상위 K개만, 비중을 weight로.
// 색은 moodRgb(mood)를 valence로 온도·채도 보정. 빈 우주 → [](배경=테마색만).
export function ambientLights(stars: { mood: string; intensity: number; valence: number; lastRecalledAt: number }[], now: number): AmbientLight[]
```

**흥분성 게인 배선(22 seam).** `ExcitabilityGain(ambient)`은 새 조각이 어느 성단에 붙을지의 바이어스를 키운다. 22가 `buildLinks`에 `score = semantic_cos + W_EXC·norm_e`를 도입하면 `W_EXC ← W_EXC · g`로 스케일한다. **22 미구현 시 25는 게인 헬퍼·테스트·proto·배경만 제공하고, `worker.go` `buildLinks` 위에 "22 배선 지점" 주석을 남긴다**(헬퍼는 호출되지 않아도 빌드·테스트 통과).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 인증 사용자가 `GetUniverse`를 호출하면 THE SYSTEM SHALL 최근 7일(`TAU_MOOD`) 윈도의 조각 감정을 `intensity·exp(-(now-t)/TAU_MOOD)`로 가중 종합한 `ambient{hue,sat,arousal,valence}`를 응답에 포함한다(valence는 시간가중 평균 `Σw·v/Σw`). _(원칙5·6)_
1.2 GIVEN 최근 격동(강한 anger/fear 다수)과 최근 평온(약한 calm 다수)의 두 우주 WHEN 각각 종합하면 THE SYSTEM SHALL 전자는 더 높은 `arousal`·음(-) 쪽 `valence`·다른 `hue`를, 후자는 낮은 `arousal`·양(+) 쪽 `valence`를 산출한다(배경이 요즘 감정을 반영). _(concept §우주의 배경)_
1.2b WHEN FE가 ambient/별 분포로 배경을 그리면 THE SYSTEM SHALL 우세 감정 상위 K개를 **각각 하나의 가산 광원(nebula 풀)** 으로 흩뿌려 단일 톤이 아닌 불규칙 그라디언트를 만들고, 각 풀 색=`moodRgb`(valence로 온도·채도 보정)·밝기=arousal로 한다. 우세 감정이 하나뿐이면 광원 1~2개의 잔잔한 단색으로 수렴한다. _(비주얼 결정)_
1.3 WHEN 같은 사건이 더 오래 전이면(Δt 증가) THE SYSTEM SHALL 그 조각의 종합 기여가 `exp` 감쇠로 단조 감소하고, 7일을 충분히 지나면 기여가 무시 가능해진다(느린 봉투). _(원칙5)_
1.4 GIVEN 별 0개(빈 우주) WHEN 종합하면 THE SYSTEM SHALL 중립 `AmbientMood{}`를 반환하고 `excitabilityGain`=1.0이 되어 할당 바이어스를 바꾸지 않는다(크래시 없음). _(원칙2)_
1.5 WHEN `excitabilityGain(ambient)`을 계산하면 THE SYSTEM SHALL `1 + 0.3·arousal`(arousal∈[0,1] → 게인∈[1,1.3])을 반환하고, 22가 존재하면 `buildLinks`의 `W_EXC`를 이 게인으로 스케일한다(부재 시 배선 지점 주석만, 빌드 무영향). _(concept §결정2)_
1.6 WHEN `GetUniverseResponse`가 도착하면 THE SYSTEM SHALL `ambient`를 별 좌표가 아닌 **전역 배경 한 겹**으로만 사용하고, 좌표 필드를 proto에 추가하지 않는다(좌표는 클라 force-sim). _(원칙3·5)_
1.7 WHEN ambient가 직전 값에서 바뀌면 THE SYSTEM SHALL 광원 풀의 색·구성을 ~0.8s 동안 크로스페이드하고(사라지는 풀 알파→0, 생기는 풀 0→목표) 풀의 느린 드리프트·일렁임을 함께 구동하되, 그 보간을 내장 TSL `time` 노드가 아니라 `useFrame`에서 bump하는 **수동 uniform**으로 한다(BloomPass frozen-time). reduced-motion이면 드리프트·일렁임을 멈추고 색만 유지한다. _(§3.1, motion-accessibility)_
1.8 THE SYSTEM SHALL 광원 풀 색을 `moodRgb`(7색 의미 팔레트)를 valence로 온도·채도 보정한 값으로 환산하여 **선택 테마와 무관**하게 색의 의미를 유지한다(테마는 배경 깊이, mood는 요즘색). _(concept §색)_
1.9 IF 서버 `ambient`가 비었거나(데모·구버전) 별만 있을 때 THEN THE SYSTEM SHALL 로드된 별의 mood/intensity/valence/lastRecalledAt에서 `deriveAmbient`·`ambientLights`로 같은 7일 가중 종합·광원 분포를 클라에서 파생해 배경을 물들인다. _(폴백)_
1.10 THE SYSTEM SHALL `entities/memory/model/ambient.ts`가 `three`/React/DOM을 import하지 않는다(순수·모바일 재사용). _(원칙4)_
1.11 WHEN 데모(체험)에서 최근 감정 분포가 바뀌면(별 추가/회상) THE SYSTEM SHALL 우주 배경이 크로스페이드로 그쪽 색으로 이동하고, 격동한 요즘일수록 새 조각의 끌림(게인)이 커지는 것을 보여준다. _(체험)_

## 과학적 근거
요즘 상태의 두 기둥 — (a) 감정이 *기억의 무게*로 작용한다(편도체 매개 부호화·공고화 강화), (b) 최근 활성 주제가 새 기억을 더 잘 포섭한다(흥분성 기반 경쟁적 할당, 6시간 척도의 시간 창) — 은 다음 문헌에 근거한다. mood-congruent 부호화/인출(현재 정서가 기억의 부호화·인출을 물들인다)은 기억 과학의 정설이며 concept §우주의 배경·§감정의 역할에 매핑된다. 배경을 단일 평균색이 아니라 **여러 광원의 분포**로 두는 것은 일상 정서가 보통 *혼합*이라는 점(감정 입자도·혼합 감정의 동시성 — 20 §과학적 근거의 Erbas 2018·Schneider & Stone 2015)과, 배경 색의 온도·채도를 **arousal(밝기)·valence(따뜻함)** 두 차원으로 가르는 정동 차원 모델(20의 mood/intensity/valence 삼분; Russell circumplex의 valence×arousal)에 부합한다.

- 신경 경쟁/흥분성 기반 할당이 부호화·연결을 조직(경쟁적 할당의 일반화): Wang et al., *Current Biology* (2026). [DOI](https://doi.org/10.1016/j.cub.2026.03.058)
- 내재 흥분성의 6시간 시간 창이 엔그램 할당·기억 연결을 좌우(흥분성 게인의 근거): Lau, Bukalo, et al., *Neurobiology of Learning and Memory* (2020). [DOI](https://doi.org/10.1016/j.nlm.2020.107284)
- CREB·흥분성 매개 엔그램 할당과 기억 연결의 시간 의존성: Cho & Han, *Frontiers in Behavioral Neuroscience* (2022). [DOI](https://doi.org/10.3389/fnbeh.2022.860027)
- 엔그램 종합(부호화·공고화·인출, 내재 흥분성·시냅스 가소성): Josselyn & Tonegawa, *Science* (2020). [DOI](https://doi.org/10.1126/science.aaw4325)

> 기분은 6시간 척도의 흥분성보다 **훨씬 느린 봉투**다 — 그래서 `TAU_MOOD = 7d`로 잡아 "요즘"을 며칠 단위로 천천히 물들이고, 흥분성 게인(빠른 6h 창, 22)에는 그 느린 봉투의 각성도만 스칼라로 흘려보낸다.
