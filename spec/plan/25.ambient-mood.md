# 25. 요즘 상태 — 우주의 배경 (ambient-mood) (as-built)

> 현재 구현 기준 문서. spec 07로 재구성됨: 떠 있던 무드 오브를 없애고, **하나의 과학적 기억 가중치**(Bjork 인출 강도 R)가 자기근접 반지름(38)과 감정 순위를 함께 구동하며, 그 순위를 **배경 스킨이 직접 짜 넣는다**. | 범위: FS | 상태: 구현 완료, 반영: 07(기억 가중치 배경)·21(조각 fan-out)·29(13감정)·30·44(배경 축)·26(relevance 동거)·38(반지름)

## 목적

개별 별이 *과거의 한 순간*이라면 우주의 배경은 "지금의 나"다(concept §우주의 배경). 같은 별들이라도 요즘의 결이 다르면 우주의 인상이 달라진다. spec 07은 이 "요즘"을 **별개의 떠 있는 오브가 아니라 배경 자체의 결**로 녹였고, "강한 기억"과 "요즘 감정"을 따로 세던 세 갈래 가중치를 **단일 Bjork 인출 강도 R**로 통합했다.

요즘 상태는 단순 배경이 아니라 기제이기도 하다(concept §결정2 경쟁적 할당): 최근 활성화된 주제일수록 흥분성이 높아 새 조각을 더 잘 포섭한다. 종합 각성도를 흥분성 게인 `g = 1 + 0.3·arousal`로 노출하고, 22의 할당 바이어스는 `wExc·g`로 라이브 스케일된다.

## 단일 기억 가중치 — Bjork 저장/인출 강도 (spec 07)

근거: Bjork & Bjork *New Theory of Disuse*(저장강도·인출강도 분리, 분산효과, 정서 공고화). **순수 클라 모듈** `frontend/src/entities/memory/model/weight.ts`(three/React/DOM 미의존, 헌법4)가 계산한다. 서버는 원자료 `recall_count`만 영속·전달한다(헌법3).

- **저장강도 S** — 누적·단조 비감소. `S = (storage_base + recall_count) · (1 + emo_consolidation · intensity)`. 회상할 때마다 `recall_count`가 늘고, 정서 강도가 회상당 공고화를 키운다(정서적 기억일수록 깊이 새겨짐). S는 줄지 않는다.
- **인출 강도 R** — 현재 접근성, 시간 감쇠. `R = exp(-Δt / τ(S))`, `τ(S) = tau0_days · (1 + tau_storage_gain · ln(1+S))`. S가 클수록 τ가 길어져 **잘 공고화된 기억은 천천히 잊힌다**(분산효과). 막 회상하면 R≈1, 오래 휴면하면 R→0.
- 계수는 `spec/values.yaml` `memory_weight`(`storage_base 1.0`·`emo_consolidation 0.6`·`tau0_days 14`·`tau_storage_gain 0.8`)에서 생성된 상수를 FE/BE가 공유한다(하드코딩 금지).

### R이 곧 가중치

- **자기근접(38 재배선)**: `radial-layout.radiusOf`가 `targetRadius(memoryR(recall_count, intensity, lastRecalledAt, now))`. `targetRadius(R) = R_MIN + (R_MAX−R_MIN)·(1−R)` — 거리=강함 보존(강할수록 중앙), **각도(방향)·force-sim 파라미터·표상 부동은 불변**. 회상→중앙, 망각→외곽, 자주 떠올린 기억은 더 오래 중앙에 머문다.
- **감정 순위(배경)**: `ambient.rankedEmotions(stars, emotionColors, now)`가 mood별 `Σ R`를 내림차순으로 매기고, 각 항목 색 = `resolveMoodRgb(mood, emotionColors)`(사용자 감정색, 45), weight = 그 mood의 Σ R 상대 비중. 배경 스킨이 상위 N개를 짜 넣는다.
- **arousal**: `ambient.arousalOf(stars, now) = 1 − exp(−Σ R)` ∈ [0,1) — 배경 스킨의 전역 생동(밝기·움직임)을 정한다.

## 배경이 감정색을 짜 넣는다 (spec 07)

- **무드 오브 제거.** 떠 있던 가산광 풀(`AmbientNebula`)을 삭제했다. 요즘 감정은 배경 스킨(`UniverseNebula`) 텍스처/톤에 녹는다.
- **배경 스킨 = 받침색 + 무늬/질감 + 감정색 슬롯 N**(44 Background 축, `entities/appearance/model/backgrounds.ts`):
  - **받침색**: 팔레트 base/c1 등 절제된 깊은 톤(원색 아님).
  - **무늬/질감**: 스킨마다 다른 `BackgroundPattern{warp, freq, detail}` — 같은 fbm·색만 다른 게 아니라 패턴 자체가 다르다(A6). 현재 vast=잔잔한 큰 결·lively=휘몰아치는 거친 결·calm=부드러운 느슨한 결·aurora-veil=흐르는 줄무늬.
  - **`emotionSlots: number`**: 이 스킨이 짜 넣는 상위 감정색 수. `0`=감정 무관 순수 텍스처·`1`(vast·calm)=주요 감정 1색·`3`(lively)=상위 3색·`13`(aurora-veil)=비중대로 전부. 시각 정의라 코드 카탈로그에 둔다(values 아님, A11).
- **렌더(`UniverseNebula`)**: 큰 안쪽 구(BackSide) 한 겹에 도메인워프 fbm을 입히고(draw call 1), 스킨 패턴으로 무늬를 조율한 뒤, 상위 N개 감정색을 노이즈 밴드 위로 R-비중 합성한다(dominant=넓은 결, 차순위=밝은 결 액센트). `arousal`이 밝기(`1+bg_brightness_gain·arousal`)·흐름 속도(`1+bg_motion_gain·arousal`)를 키운다. `prefers-reduced-motion`이면 모션 정지·색/밝기 유지.
- 감정색·arousal은 유니폼으로 갱신(셰이더 재컴파일 없음), 받침색·무늬는 스킨이 바뀔 때만 재빌드. 별 mood 색은 불간섭(이 구는 모든 것 뒤·depthWrite/Test 없음).

## 자아 몸체 색 (spec 44 J3 — R-인지)

- `ambient.deriveAmbient(stars, now)`가 R-가중 mood 블렌드로 `{hue, sat, arousal, valence}`를 내고, `ambientToRgb`가 valence 보정 단색을 만든다 — `SelfStar`의 몸체 색(요즘 감정 파생, 테마 무관). 자아 별이 던지는 빛(반사 채널)은 중립 유지(spec 03 — mood 이중 주입 금지). 자아 색은 고정 의미 팔레트(`moodRgb`), 배경 짜임은 사용자 감정색 — 역할 분리.

## 계약 정리 (spec 07)

- **proto `Star` += `recall_count` (필드 14)** — 서버 권위 원자료. `RecallMemoryTouch`가 회상마다 +1.
- **`GetUniverseResponse.ambient`(옛 필드 3) reserved** — 서버 요즘-감정 종합(`AggregateAmbient`/`ListRecentForAmbient`/`AmbientMood`/`moodRGB`)을 모두 제거했다. 클라가 로드된 별(+`recall_count`)에서 순위·arousal을 파생한다. `share.proto`의 `SharedUniverse.ambient`(옛 필드 5)도 reserved — 방문자도 SharedStar에서 직접 파생.
- **`ExcitabilityGain(arousal float64)`** 헬퍼는 22 경쟁 할당에 라이브 배선된다(시그니처가 옛 `AmbientMood` 인자 → `float64`로 바뀜). 서버 ambient summary는 없고, worker가 모든 별의 `recall_count`·`intensity`·`last_recalled_at`에서 같은 Bjork R envelope의 arousal을 재도출한다.

## 흥분성 게인 배선 (22)

- `GraphStore.LoadExcitabilityInputs`가 후보 성단 timestamp 입력과 함께 사용자 전체 별의 arousal 입력을 읽는다(`ListArousalInputs`: `intensity`·`recall_count`·`last_recalled_at`).
- `memory.ArousalFromSamples`가 클라 `arousalOf`와 같은 `1-exp(-ΣR)` envelope을 계산하고, `biasedLinks`는 `score = cos_sim + (wExc·ExcitabilityGain(arousal))·norm_e`로 후보를 재랭킹한다. arousal=0이면 기존 22 점수식과 동일하고, arousal>0이면 흥분성 항만 커진다.

## values (spec/values.yaml)

- `memory_weight`: `storage_base`·`emo_consolidation`·`tau0_days`·`tau_storage_gain`(신규, FE 전용 — R 계산).
- `ambient`: `arousal_gain`(22 경쟁 할당 게인, FE·BE)·`bg_brightness_gain`·`bg_motion_gain`(배경 생동, FE)·`valence_warm_gain`·`valence_sat_gain`(자아 색 보정, FE). (`tau_mood_days`는 26 relevance 폐기로 제거 — change 19.)
- 은퇴: `radial_layout.w_activation`/`w_intensity`(R로 대체)·`ambient.lights_k`/`light_min_share`/`window_tau_factor`·`ambient_nebula` 그룹 전체.
- `emotionSlots`(스킨별 N)·`BackgroundPattern`은 코드 카탈로그(시각 정의).

## 데모·랜딩

- 데모(`shared/lib/demo/data.ts`): 별에 `recall_count` 부여(정서적인 별일수록 높게, 회상 재점화 시 +1) — 같은 R-순위·배경 짜임 경로를 그대로 탄다. 서버 ambient 없음.
- 랜딩 `AmbientMoodCard`(`<TheoryBadge plan="25"/>`): 옛 떠 있는 오브 묘사를 "배경 결에 짜인 감정색"으로 재조정. R로 감정 순위→배경 텍스처 weave, arousal→배경 생동·흥분성 게인을 설명.

## Flutter 동등성 기준

- `recall_count`는 서버 권위 원자료. Flutter도 `Star.recall_count`를 받아 같은 `weight.ts` 수식(S·R)으로 자기근접 반지름과 감정 순위를 파생해야 한다(서버는 ambient 종합을 주지 않는다).
- 배경 스킨이 상위 `emotionSlots`개 감정색(`resolveMoodRgb`)을 R-비중으로 텍스처에 합성하고, arousal이 전역 생동을 정한다. `emotionSlots`·패턴은 코드 카탈로그 시각 정의다.
- 자아 몸체 색은 R-가중 `deriveAmbient`(고정 의미 팔레트), 반사 채널은 중립. 흥분성 게인 `g = 1 + 0.3·arousal`은 공유 모델이고 서버 worker 경쟁 할당에 라이브 배선된다.

## 수용 기준

1. `memories.recall_count`가 존재하고 회상(`RecallMemoryTouch`)마다 +1, `Star`로 클라에 전달된다(기존 별 1 백필).
2. 막 회상한 별은 R≈1(중앙), 오래 휴면한 별은 R→낮음(외곽). 회상 횟수가 많을수록 같은 Δt에서 천천히 잊힌다(τ가 S로 길어짐). R·S는 순수 클라 모듈에서 계산된다.
3. `targetRadius`가 R을 받고 거리=강함을 보존한다. 각도·force-sim·표상 부동 불변.
4. 배경 스킨이 상위 N(`emotionSlots`)개 감정색을 사용자 감정색·R-비중으로 텍스처에 합성한다(0이면 순수 텍스처).
5. `AmbientNebula`(떠 있는 가산광 풀)가 제거됐다.
6. 배경 아이템이 색만이 아니라 받침색+무늬/질감으로 갈린다(절제된 톤).
7. arousal이 Σ R에서 도출돼 배경 전역 생동을 정하고, 같은 arousal envelope이 worker의 경쟁 할당 `wExc`를 `ExcitabilityGain(arousal)`로 스케일한다.
8. 서버가 `ambient`를 더는 싣지 않고(필드 reserved) 클라가 별에서 파생한다. ambient 종합 코드 제거 후에도 코어 그래프 조회 회귀 없음.
9. 별 색은 여전히 감정색(`resolveMoodRgb`), 26 밝기 감쇠·self-glow·spec 03 반사 중립 불변.
10. 데모·랜딩이 새 모델을 미러한다. 38/44/45 회귀 없음.
11. 신규 계수·배경 생동 계수가 values.yaml 생성 상수로 FE/BE 공유. `emotionSlots`는 코드 카탈로그.
12. build·lint·vet·typecheck·steiger 통과, reduced-motion에서 배경 모션 정지(색 유지), 우주 셸·기존 화면 회귀 없음.
