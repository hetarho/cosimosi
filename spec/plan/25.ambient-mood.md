# 25. 요즘 상태 — 우주의 배경 (ambient-mood) (as-built)

> 현재 구현 기준 문서. 최근 조각 감정을 7일 시간가중으로 종합한 ambient 요약을 서버가 주고, 클라가 그 분포로 다중 광원 nebula 배경을 그리며, 종합 각성도를 흥분성 게인으로 노출하는 방식의 SSOT다. | 범위: FS | 상태: 구현 완료, 반영: 21(조각 fan-out)·29(13감정)·30(테마)·26(relevance 동거)

## 목적

개별 별이 *과거의 한 순간*이라면 우주의 배경·앰비언트 색은 "지금의 나"다(concept §우주의 배경). 21이 일기 1건을 여러 조각 별로 펼친 뒤, 그 최근 감정들을 7일 시간가중으로 종합해 배경을 빚는다 — 요즘이 평온하면 잔잔한 색, 격동하면 강렬한 색. 같은 별들이라도 오늘의 하늘색이 다르면 우주의 인상이 달라진다.

배경은 단일 색조가 아니라 "군데군데 넓은 광원"의 불규칙 그라디언트다. 요즘의 감정은 보통 여러 감정의 분포이므로, 우세 감정 상위 몇 개(top-K)를 각각 하나의 넓은 가산 광원(nebula 풀)으로 흩뿌려 screen-가산으로 번지게 한다. 각 풀 색은 그 mood의 의미색(moodRgb)을 valence로 온도·채도 보정한 값이고, 밝기는 종합 각성(arousal)이 정한다. 우주의 deep base 배경색은 사용자가 고른 개인 설정(30)이고, ambient는 그 위에 번지는 감정 nebula overlay다 — base를 대체하지 않는다.

요즘 상태는 단순 배경이 아니라 기제이기도 하다(concept §결정2 경쟁적 할당): 최근 활성화된 주제일수록 흥분성이 높아 새 조각을 더 잘 포섭한다. 그래서 종합 각성도를 흥분성 게인 `g = 1 + 0.3·arousal`로 노출해 22의 할당 바이어스 `W_EXC`를 스케일할 수 있게 한다. 단 라이브 배선은 현재 코드에 들어 있지 않고 seam 주석으로만 남아 있다(아래 "흥분성 게인" 참고).

## 현재 구현

- 서버는 `GetUniverse`에서 최근 7일 조각 감정을 시간가중 종합해 `AmbientMood{hue,sat,arousal,valence}` 한 묶음을 응답에 싣는다. 다중 광원의 위치와 풀 색 분포는 서버가 보내지 않는다 — 클라가 로드된 별에서 직접 파생한다(좌표·렌더 입력은 클라 권위, 헌법3).
- 가중 종합은 SQL이 아니라 순수 도메인 함수 `AggregateAmbient`에서 한다. 쿼리는 7일 윈도의 원자료(`mood/intensity/valence/last_recalled_at`)만 추리고 감쇠 수식을 넣지 않는다(`ListDormant`/12와 같은 패턴).
- 클라는 같은 7일 종합 수식을 미러한 `deriveAmbient`/`ambientLights`를 갖는다. 서버 `ambient`를 받으면 그걸 요약으로 쓰고, 미수신(데모·구버전)이면 로드된 별에서 폴백 파생한다. 광원 색 분포는 어느 경우든 항상 클라가 별에서 만든다.
- 배경 렌더는 `AmbientNebula`가 K개 가산 빌보드(InstancedMesh)로 그린다. 위치는 시드 기반 불규칙 배치라 매번 같은 자리에 뜨고 카메라가 움직이면 3D 시차로 깊이가 산다. 드리프트·일렁임·색 크로스페이드는 모두 `useFrame`에서 수동 uniform으로 구동한다(BloomPass가 내장 `time` 노드를 진전시키지 않으므로).
- 흥분성 게인 헬퍼는 BE/FE 양쪽에 정의돼 있다(`ExcitabilityGain` / `excitabilityGain`). 22의 `wExc`에 곱하는 라이브 스케일은 아직 배선되지 않았고, `worker.go`에 배선 지점 주석만 있다.

## 도메인 모델 (`backend/internal/memory/memory.go`)

- `AmbientMood{Hue, Sat, Arousal, Valence}` (모두 `float64`)
  - `Hue` 0..360 대표 색조(HSV), `Sat` 0..1 채도, `Arousal` 0..1 각성도(흥분성 게인 입력), `Valence` -1..1 시간가중 평균 정서가(배경 색 온도·채도 보정).
  - 전부 0인 zero value는 중립/빈 우주를 뜻한다. 태그 없음(헌법5).
- `EmotionSample{Mood, Intensity, Valence, LastRecalledAt}` — `AggregateAmbient`의 원자료 1건. repository가 sqlc row를 이 타입으로 매핑한다. `LastRecalledAt`은 생성 또는 회상 시각으로, 7일 봉투가 가중하는 활동 시간이다.
- `Universe{Memories, Synapses, Ambient}` — 04의 `Universe`에 `Ambient AmbientMood`가 더해진 형태.
- 상수: `TauMoodDays = 7.0`, `arousalGainCoef = 0.3`(둘 다 `memory.go`). `arousalGainCoef`는 unexported.

## `AggregateAmbient` (순수 종합)

`func AggregateAmbient(samples []EmotionSample, now time.Time) AmbientMood` — 7일 시간가중 종합. `now`를 주입받아 감쇠가 테스트 가능하다.

- 각 샘플 가중치 `w = intensity · exp(-Δt/τ)` (`Δt`는 일 단위, `τ = TauMoodDays`). 미래 타임스탬프(시계 어긋남)는 `Δt=0`으로, 음수 intensity는 0으로 클램프한다. `w ≤ 0`이면 건너뛴다.
- 출력:
  - `hue/sat` = 강도가중 평균 mood-RGB 블렌드(`Σw·rgb / Σw`)의 HSV. 혼재 감정 → 회색 → 낮은 채도, 단일 우세 감정 → 또렷한 색.
  - `arousal = 1 - exp(-Σw)` ∈ [0,1). 최근·강한 감정이 많을수록 높다.
  - `valence = clamp(Σ(w·valence)/Σw, -1, 1)` — 시간가중 평균 부호 정동.
- `Σw ≤ 0`(샘플 없음 또는 전부 0가중)이면 중립 `AmbientMood{}`를 반환한다(arousal 0 → 게인 1.0).
- HSV 변환은 헬퍼 `rgbToHueSat(r,g,b)`가 한다. value(명도)는 버린다 — 배경 밝기는 luminance가 아니라 arousal에서 온다. 회색(delta 0)은 hue 0으로 보고한다.
- 색 팔레트는 `moodRGB` 맵(`moodRgbOf`로 조회, 미지 mood는 중립 회색 `{0.6,0.6,0.6}`). 13개 mood(29)를 담고 FE `MOOD_PALETTE`와 같은 값을 미러한다. 이 맵·HSV 환산이 BE 요약과 FE 폴백이 같은 "요즘" 색에 합의하게 만드는 단일 출처다.

## `ExcitabilityGain`

`func ExcitabilityGain(a AmbientMood) float64 { return 1 + arousalGainCoef·a.Arousal }` — arousal∈[0,1] → 게인∈[1,1.3]. arousal만 입력한다(valence는 색에만, 감쇠 가중은 26). 빈 우주 → arousal 0 → 게인 1.0(no-op).

## 집계 쿼리 (`backend/internal/db/queries/memory.sql`)

```sql
-- name: ListRecentForAmbient :many
SELECT m.mood, m.intensity, m.valence, m.last_recalled_at
FROM memories m
WHERE m.user_id = $1
  AND m.last_recalled_at >= sqlc.arg(since)::timestamptz;
```

- 7일 윈도의 원자료만 추린다. 감쇠 수식·정규화는 SQL에 없다(도메인에서 — sargable·결정론, 12 패턴).
- mood/intensity/valence 출처는 `memories`(가변 별 레이어, 21 이후). `records` JOIN 없음. `fragment_index` 무관(조각 전부 종합).
- `last_recalled_at`은 생성 시 `now()` 기본값이라 막 적은 조각도, 회상으로 끌어올린 조각도 자연히 무게를 받는다.

## 서버 합성 (`backend/internal/memory/service.go`, `handler.go`, `repository.go`)

- `Service.GetUniverse(ctx, userID)`:
  - `ambientWindowFactor = 3.0` 상수로 `since = now - TauMoodDays·3·24h`(≈21일)를 환산해 `ListRecentForAmbient`를 호출한다. exp 감쇠가 무시 가능한 지점(exp(-3)≈0.05)에서 잘라 비용을 줄인다.
  - 조회 성공 시 `AggregateAmbient(samples, now)`로 종합, 실패하면 중립 `AmbientMood{}`로 degrade한다 — ambient는 장식·가산 tint이고 클라 폴백이 있으므로 읽기 실패가 코어 그래프 조회를 막아선 안 된다.
  - 같은 핸들러에서 relevance(26)도 best-effort로 별에 접고, `Universe{Memories, Synapses, Ambient}`를 반환한다.
- `handler.go`는 `uni.Ambient`를 proto `AmbientMood{Hue,Sat,Arousal,Valence}`로 1:1 매핑한다(빈 종합이면 0값).
- `repository.go`의 소비자 인터페이스에 `ListRecentForAmbient(ctx, userID, since) ([]EmotionSample, error)`가 있고, `repository_pg.go`가 sqlc row를 `EmotionSample`로 래핑한다.

## 흥분성 게인 배선 (22/27 seam)

- `worker.go`의 경쟁적 할당(22)은 `score = cos_sim + wExc·norm_e`로 KNN 후보를 재랭킹한다. 상수: `wExc = 0.25`, `tauExc = 6h`, `candidateK = knnK·2`, `biasedK = 5`(재랭킹 후 유지하는 최종 링크 수), `inhibitDecay = 0.5`.
- 25의 게인 라이브 스케일(`wExc ← wExc·ExcitabilityGain(ambient)`)은 **현재 코드에 들어 있지 않다.** `wExc` 상수 위에 배선 지점 주석만 있다. ambient는 6h 흥분성 창과 달리 느린 7일 봉투라, 매 embed 잡마다 재종합하기보다 27(야간 공고화)의 주기 재계산 seam으로 흘려보내는 게 자연스럽다고 명시돼 있다.
- 따라서 25는 게인 헬퍼·proto·배경·단위 테스트만 제공하고, `ExcitabilityGain`은 프로덕션 호출 없이(테스트에서만 호출) 빌드·테스트를 통과한다.

## FE 모델 (`frontend/src/entities/memory/model/ambient.ts` — 순수)

`@/shared/config`의 `moodRgb`/`RGB`만 import한다. three/React/DOM 미의존(헌법4).

- `interface Ambient { hue; sat; arousal; valence }` — proto `AmbientMood`와 동치.
- `interface AmbientLight { rgb: RGB; weight: number; mood: string }` — 광원 풀 1개.
- `interface AmbientStar { mood; intensity; valence; lastRecalledAt }` — 파생 입력(`lastRecalledAt`는 epoch ms).
- 상수: `TAU_MOOD_DAYS = 7`, `AROUSAL_GAIN = 0.3`, `AMBIENT_LIGHTS_K = 6`(광원 상한), `LIGHT_MIN_SHARE = 0.04`(이 미만 비중은 풀로 만들지 않음, unexported).
- `excitabilityGain(a) = 1 + 0.3·a.arousal` — BE `ExcitabilityGain` 미러.
- `deriveAmbient(stars, now): Ambient` — 서버 `ambient` 미수신 시 폴백. `weightOf = intensity·exp(-Δt/τ)`로 종합. 빈/0가중 입력 → 중립 0값. arousal = `1-exp(-Σw)`, valence = `Σwv/Σw` — BE와 같은 수식.
- `ambientLights(stars, now): AmbientLight[]` — mood별 시간가중 비중을 모아 `LIGHT_MIN_SHARE` 미만을 버리고 내림차순 정렬해 top-K로 자른다. 각 풀 색 = `moodRgb(mood)`를 valence로 보정. 단일 우세 감정 → 1풀, 빈 우주 → `[]`.
- `ambientToRgb(a): RGB` — 요약의 대표 단색(`hsvToRgb(hue,sat,1)`을 valence 보정). 테마 무관(mood 의미색 유지).
- valence 보정(`correctByValence`, unexported): 양→따뜻(R+0.12v, B-0.12v)·채도↑(satK=1+0.25v), 음→차갑·탁. 미세 보정이다.

## FE 렌더 (`frontend/src/widgets/universe-canvas/ui/AmbientNebula.tsx`)

three/TSL는 이 플랫폼 레이어에서만 쓴다.

- `AmbientNebula()` 컴포넌트가 `THREE.InstancedMesh`(`PlaneGeometry(1,1)` + `MeshBasicMaterial`)로 항상 `AMBIENT_LIGHTS_K`(6)개 인스턴스를 그린다. 머티리얼: `AdditiveBlending`, `transparent`, `depthWrite=false`, `depthTest=false`, soft radial glow `CanvasTexture`(하드 링 없음). `renderOrder = -10`(별·시냅스 뒤), `frustumCulled=false`, raycast no-op(별 클릭 통과).
- 슬롯 시드: `buildSlots()`가 `mulberry32(0xa3b1e)`로 6개 고정 슬롯을 만든다 — 반경 `RADIUS_MIN..MAX = 120..200` 구면 위 위치(별 셸 ~46 밖), 슬롯별 드리프트 주파수·위상·shimmer 위상. mood/색은 슬롯을 바꿔 채우지만 자리는 고정이다.
- 광원 = `useMemo(() => ambientLights(ambientStars(stars), virtualNowMs()), [stars])`. store의 `ambient.arousal`이 전체 밝기를 정한다. 슬롯 i = i번째 우세 풀(내림차순), 빈 슬롯은 검정(가산 하에서 비가시).
- 색·크기 타깃(`targetsFor`): 풀 밝기 `vivid = BASE_BRIGHT 0.06 + AROUSAL_BRIGHT 0.2·arousal`, 상대 비중 `wn`으로 `mag = vivid·(0.55+0.45·wn)`, `scale = SCALE_MIN 60 + (SCALE_MAX 120 - 60)·wn`.
- 크로스페이드(수동 uniform): from/to 슬롯별 색·스케일 배열을 `mix`로 보간. 풀/arousal 변경 시 현재 표시값을 새 `from`으로 스냅샷하고 새 타깃을 `to`로, `mix=0`에서 재시작한다(`CROSSFADE_S = 0.8`s). 타깃이 같으면 재시작 안 함(StrictMode 더블 인보크/한 로드 두 커밋 방어). 사라지는 풀 → 검정/min-scale 타깃으로 fade-out, 생기는 풀 → 검정에서 fade-in.
- `useFrame`에서 `mix += dt/0.8` 클램프, `phase += dt·DRIFT_SPEED 0.05`. 드리프트 진폭 `DRIFT_AMP 16` world-unit, `SHIMMER ±6%`. 빌보드는 매 프레임 카메라 quaternion으로 정렬한다. 내장 `time` 노드는 쓰지 않는다(BloomPass frozen-time).
- `prefers-reduced-motion`이면 드리프트·shimmer를 멈추고(`wob=0`, shimmer 1) 색 변경은 즉시 스냅(`mix=1`)한다 — 색은 유지, 모션만 정지.
- `UniverseCanvas.tsx`는 `<color attach="background" args={[bg]}/>`(테마 베이스 `themeBg(theme)`) 바로 뒤, `StarField`/`BloomPass` 앞에 `<AmbientNebula/>`를 합성한다. 별 셸 밖 깊은 반경에 있어 별빛을 가리지 않는다.

## Flutter 동등성 기준

- ambient 종합은 서버 권위다. Flutter도 `GetUniverse` 응답의 `AmbientMood{hue,sat,arousal,valence}`를 그대로 받아 배경 요약으로 쓰고, 미수신 시 로드된 별에서 같은 7일 종합(`w = intensity·exp(-Δt/τ)`, arousal `1-exp(-Σw)`, valence `Σwv/Σw`)으로 폴백 파생해야 한다.
- 다중 광원의 위치·색 분포는 proto에 없다 — Flutter도 `ambientLights`와 같은 로직(top-K ≤6 우세 mood, `LIGHT_MIN_SHARE 0.04` 미만 버림, `moodRgb`를 valence로 보정)을 클라에서 재현해야 한다.
- 광원 색은 테마와 무관한 13색 의미 팔레트(`moodRgb`)에서 와야 하며, 테마는 배경 깊이 베이스색만 정한다.
- 흥분성 게인 `g = 1 + 0.3·arousal`은 공유 모델이다. 단 현재 서버 할당 파이프라인에 라이브 배선돼 있지 않으므로, Flutter도 게인 값을 직접 할당에 곱이는 게 아니라 동일하게 seam(헬퍼·표시)으로만 둔다.
- 광원 렌더(가산 빌보드·시드 배치·수동 uniform 크로스페이드·드리프트)는 three 기반 웹 구현이다. Flutter는 같은 분포·색·~0.8s 크로스페이드·reduced-motion 정지 결과를 자기 렌더 스택으로 재현하면 된다.

## 수용 기준

1. 인증 사용자가 `GetUniverse`를 호출하면 응답에 최근 7일 윈도 조각 감정을 `w = intensity·exp(-(now-t)/τ)`로 가중 종합한 `ambient{hue,sat,arousal,valence}`가 포함된다(valence는 시간가중 평균 `Σwv/Σw`).
2. 최근 격동(강한 부정 감정 다수)인 우주는 평온(약한 calm 다수)인 우주보다 높은 arousal과 음(-) 쪽 valence를 산출한다.
3. 같은 사건이 더 오래 전일수록(Δt 증가) 그 조각의 종합 기여가 exp 감쇠로 단조 감소하고, 7일을 충분히 지나면 무시 가능해진다.
4. 빈 우주(별 0개, Σw=0)는 중립 `AmbientMood{}`(arousal 0)를 반환하고 `ExcitabilityGain`은 1.0이 된다(크래시 없음).
5. `ExcitabilityGain(ambient)` = `1 + 0.3·arousal`로 arousal∈[0,1]을 게인∈[1,1.3]에 매핑한다. 이 값은 현재 `worker.go`의 `wExc`에 곱해지지 않고 배선 지점 주석으로만 남아 있다.
6. `ambient`는 별 좌표가 아니라 전역 배경 요약으로만 쓰이고, proto에 좌표·광원 위치 필드가 없다.
7. FE는 우세 감정 상위 K개(≤6, 상대 비중 0.04 미만 버림)를 각각 하나의 가산 광원 풀로 흩뿌려 단일 톤이 아닌 불규칙 그라디언트를 만든다. 각 풀 색 = `moodRgb`(valence 보정), 밝기 = arousal. 우세 감정이 하나뿐이면 1풀로 수렴하고 빈 우주는 0풀(테마 베이스색만)이다.
8. ambient가 직전 값에서 바뀌면 광원 풀의 색·구성이 ~0.8s 크로스페이드된다(사라지는 풀 알파→0, 생기는 풀 0→목표). 드리프트·일렁임·크로스페이드는 내장 TSL `time` 노드가 아니라 `useFrame`에서 bump하는 수동 uniform으로 구동한다(BloomPass frozen-time).
9. `prefers-reduced-motion`이면 드리프트·일렁임을 멈추고 색만 유지한다(색 변경은 즉시 스냅).
10. 광원 풀 색은 선택 테마와 무관하게 의미 색을 유지한다(테마는 배경 깊이, mood는 요즘색).
11. 서버 `ambient`가 비었거나(데모·구버전) 별만 있을 때 클라가 로드된 별의 mood/intensity/valence/lastRecalledAt에서 같은 7일 종합과 광원 분포를 파생해 배경을 물들인다.
12. `entities/memory/model/ambient.ts`는 three/React/DOM을 import하지 않는다(순수·모바일 재사용).
13. ambient 읽기가 실패해도 `GetUniverse`는 코어 그래프를 실패시키지 않고 중립 ambient로 degrade한다.
