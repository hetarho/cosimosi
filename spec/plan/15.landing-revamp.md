# 15. landing-revamp — 랜딩(/) 환상적 우주 + 여정 (as-built)

> 현재 구현 기준 문서. 공개 랜딩(`/`)이 무엇을 그리고, 그 시각 시스템의 어느 조각을 어느 스펙이 소유하는지를 기술하는 웹 전용 SSOT다. | 범위: FE(랜딩 전용, 웹) | 상태: 구현 완료, 반영: 19 theory-demo, 21 fragmentation, 23 reconsolidation, 25 ambient-mood, 30 appearance-refactor, 38 self-anchor

## 목적

랜딩은 로그인·DB 없이 "기억의 우주"를 한눈에 느끼게 하는 공개 진입면이다. 히어로에서 시작해 장(章, JourneyAct)을 따라 내려가며 엔그램 이론을 차례로 만진다 — 탄생(엔그램) → 분할(사건 경계) → 연결·강화(헵·시간 창) → 재공고화 → 망각(침묵) → 요즘의 나 → 야간 공고화 → 공명. 배경·히어로·글래스 크롬은 앱 전역 시각 설정(테마)을 따라 통째로 바뀌고, 신경과학 시각화는 별·시냅스 메타포로 일관되게 그려진다. 시냅스는 항상 곡선이다(직선 금지).

이 문서가 처음 가정한 "랜딩 로컬 4테마(`aurora|deepfield|liquid|nebula`) + 페이지 전용 스위처/뷰즈" 설계는 대부분 후속 스펙으로 대체되어 더 큰 시스템으로 흡수되었다. 현재 랜딩은 그 시스템의 **소비자**다.

## 소유 경계 (어느 스펙이 무엇을 소유하는가)

랜딩 고유 자산은 `pages/landing/`에만 있다. 시각 시스템의 나머지는 entities/features가 소유하고 랜딩은 구독·소비한다.

- 시각 설정(테마·오브제) — `entities/appearance` + `features/switch-appearance` (30). 옛 `pages/landing/model/theme.ts` zustand 스토어는 없다.
- 별·시냅스 2D 프리미티브 — `entities/star`(VizStar), `entities/synapse`(VizSynapse, synapseCurve) (30 appearance refactor). 옛 `pages/landing/ui/viz/`는 없다.
- 이론 카드 본체(엔그램·헵·시간 창·침묵) — `entities/theory` 데모 (19). 랜딩 카드는 얇은 래퍼다.
- 3D 별(히어로 엠블럼) — `pages/landing/ui/star3d/`(ThemedStar)가 보유하되 `entities/star`의 `StarObject` 타입을 따른다. 소비처는 `HeroSection` 하나뿐이고, 신경과학 카드는 모두 2D `VizStar`를 쓴다.

랜딩이 직접 소유하는 것: 배경 atmosphere, 그레인, JourneyAct 레이아웃, 스크롤 훅, `--ld-*` 글래스/히어로 크롬 토큰, 그리고 후속 스펙 개념을 담은 신규 섹션들(분할·요즘의 나·야간 공고화 등).

## 현재 구현

### 진입과 셸

- 라우트 `/`는 `frontend/src/app/router.tsx`의 index route가 `LandingPage`(`frontend/src/pages/landing/ui/LandingPage.tsx`)를 마운트한다. 인증 게이트 없는 공개 표면이다(01).
- `LandingPage`는 루트 `<div>`에 `data-landing-theme={theme}`를 박는다. 이 속성은 `--ld-*` 글래스/히어로 크롬 토큰을 랜딩 안으로만 캐스케이드시킨다 — 다른 라우트 영향 0.
- `theme`은 `useAppearance((s) => s.theme)`로 읽는다(`entities/appearance`). 앱 전역 색은 `RootLayout`이 `<html>`에 `data-theme`로 별도로 박으며, 두 속성은 같은 값(`vast|lively|calm`)을 쓴다.
- 셸 구성: `<LandingBackground theme>` + `<AppearanceSwitcher />` + `<main>` 안의 히어로 → JourneyAct들 → `CtaFooterSection`.
- 관성 스크롤은 앱 루트의 `frontend/src/app/SmoothScroll.tsx`(Lenis `ReactLenis root`)가 감싼다. `prefers-reduced-motion`이면 Lenis를 끼우지 않고 네이티브 스크롤로 폴백한다.

### 시각 설정 — 두 축 (30 소유)

랜딩은 색/형태를 직접 갖지 않고 `entities/appearance` store를 구독한다. 두 축은 서로 독립이다.

- **Theme**(색·분위기) `vast | lively | calm` — `entities/appearance/model/types.ts`. 메타(이름·태그라인·칩 swatch·`accent` mood hex·`bg` THREE 배경색)는 `entities/appearance/model/themes.ts`의 `THEMES`. `DEFAULT_THEME = 'vast'`.
- **StarObject**(별의 형태) `deepfield | aurora | liquid | ember` — `entities/star/model/types.ts`, 목록은 `entities/star/model/kinds.ts`의 `STAR_OBJECTS`. `DEFAULT_OBJECT = 'deepfield'`. (이름 표기: Crystal/Nebula/Liquid/Ember.)
- 색(mood hex)은 두 축과 무관하게 보존된다(감정 의미색). 테마를 바꿔도 별·시냅스의 의미색은 그대로고, 형태·재질만 오브제 축을 따른다.
- 지속: store는 `cosimosi.appearance` 키로 localStorage에 persist한다. 단 기기 선호(theme·object·selfObject)만 영속하고, per-user `emotionColors`는 메모리 전용이다. 구 키 `cosimosi.landing.theme`(이 스펙이 만든 키)에서 1회 승계한다 — 구 저장본의 4-값 `theme`(deepfield/…)을 `object`로 옮기고 `theme`은 `vast`로 폴백한다.
- 스위처는 `features/switch-appearance/ui/AppearanceSwitcher.tsx`. 우하단 고정 FAB(접힘) → 펼치면 세 라디오그룹(테마·오브제·자아 별)을 WAI-ARIA radiogroup으로 고른다. 인증 세션이면 `pushSettings`로 서버 영속(30), 미인증·체험은 로컬만. 자아 별(`selfObject`, 38)은 기기 로컬 전용이다.

### 배경 atmosphere (랜딩 소유)

`frontend/src/pages/landing/ui/background/`. `LandingBackground`가 `theme`로 atmosphere를 선택한다(오브제 형태와 독립).

- `VastBackground` — 깊은 인디고 심연. 2겹 별 필드(고밀도 시차 + 성긴 근경) + 먼 violet 성운 글로우 + 강한 그레인.
- `LivelyBackground` — 채도 높은 자홍/앰버/핑크/바이올렛 블롭이 `screen` 합성으로 도는 메시 그라디언트 + 광택 시트 + 별 필드 + 그레인.
- `CalmBackground` — 청록 계열 색점이 아주 느리게 번지는 그레이니 오로라 메시 + 별 필드 + 그레인.
- `GrainOverlay` — 자산 없는 필름 그레인. 두 겹: feTurbulence 디밴딩 디더(`overlay`, `--ld-deband-opacity`) + 필름 그레인(`screen`/`soft-light`, `--ld-grain-opacity`). 같은 인자면 같은 텍스처(data-URI 캐시). 모션 허용 시 미세 지터, reduced-motion이면 정적.
- 모든 배경 컴포넌트는 `motion/react`의 `useReducedMotion()`을 읽어 무한 애니메이션을 정지한다. `fixed inset-0 -z-10 pointer-events-none`.
- 별 필드는 공용 `frontend/src/shared/ui/StarFieldCanvas.tsx`(Canvas 2D, three 미사용). 옵션 `count`·`color`·`sizeScale`·`maxAlpha`·`parallax`. parallax>0이면 깊이(z)에 비례해 마우스 시차로 떠 입체감을 준다. reduced-motion이면 1회만 정적으로 그리고 시차도 끈다. 탭이 백그라운드면 RAF를 멈춘다.

### 여정 레이아웃 (랜딩 소유)

- `frontend/src/pages/landing/ui/JourneyAct.tsx` — 장(章) 단위. 왼쪽 스파인(빛의 실)에 챕터 마커(별 글로우) → eyebrow(과학 앵커) → 문학적 제목 → 리드 → 인터랙티브 viz. props: `id`·`chapter`·`eyebrow`·`heading`·`lead`·`accent`·`layout('stacked'|'split')`·`flip`. `whileInView`로 스크롤 진입 시 위에서부터 떠오르고 reduced-motion이면 정지.
- `LandingPage`의 장 순서(각 `id`가 스크롤 앵커): `concept`(I) → `engram`(II) → `fragmentation`(III) → `connect`(IV: 헵+시간 창) → `reconsolidation`(V) → `silent`(VI) → `present`(VII: ambient+요즘의 나) → `nightly`(VIII) → `resonance`(IX), 그리고 `CtaFooterSection`. 이 순서는 19–27 이론 흐름과 같다.

### 섹션 = 이론 데모 위임 또는 직접 viz

- 이론 카드(`entities/theory` 위임, 19): `EngramCard`→`EngramDemo`, `HebbianCard`→`HebbianDemo`, `TimeWindowCard`→`TimeWindowDemo`, `SilentEngramCard`→`SilentEngramDemo`. 카드는 `GlassCard` + `TheoryBadge` + `TryInUniverse`로 감싼 얇은 래퍼다. 옛 TimeWindow 꺾은선 그래프 재설계·슬라이더 수식은 이제 theory 데모가 소유한다.
- 직접 viz 섹션(랜딩이 VizStar/VizSynapse로 직접 그림): `ConceptSection`(성단), `FragmentationCard`(하루→N조각 별, 21), `AmbientMoodCard`(요즘의 하늘색, 25), `PresentSelfCard`(경쟁적 할당, 22), `ReconsolidationCard`(회상마다 재성형, 23), `NightlyConsolidationCard`, `ResonanceSection`(같은 밤 두 별).
- 모든 직접 viz 섹션은 `useAppearance`로 현재 `object`(StarObject)를 읽어 `concept`로 내려보낸다 → 오브제를 바꾸면 카드 별·시냅스가 그 형태로 다시 그려진다. 의미색(`MOOD.*` hex)은 테마와 무관하게 인자로 고정한다.

### 시각화 프리미티브 (entities 소유, 30)

- `entities/star/ui/VizStar.tsx` — `concept`(StarObject)로 dispatch하는 SVG 별: `deepfield→CrystalStar`(회절 스파이크+패싯), `aurora→NebulaStar`(빛구름), `liquid→LiquidStar`(굴절 구슬), `ember→EmberStar`(용암 균열+백열 코어). 어떤 viewBox에든 `(cx,cy,r)`로 박힌다. 같은 `seed`면 같은 형태. 구현은 `entities/star/ui/svg/`.
- `entities/synapse/ui/VizSynapse.tsx` — 두 점을 잇는 "빛의 실": 넓고 옅은 글로우 + 가느다란 발광 코어. `strength`(0~1)가 굵기·밝기를, `arc`가 곡률을 정한다. `concept==='ember'`면 코어가 mood색, `deepfield`면 점선 별자리 선. `active`면 부스트.
- `entities/synapse/lib/curve.ts`의 `synapseCurve(x1,y1,x2,y2,arc)` — 중점을 수직으로 띄운 2차 베지어 path. 모든 시냅스는 이 곡선을 쓴다(직선 없음).
- 이론 데모용 cross-import: `entities/star/@x/theory.ts`(VizStar), `entities/synapse/@x/theory.ts`(VizSynapse)로 같은 시각 언어를 theory entity에 노출한다.

### 히어로

- `frontend/src/pages/landing/ui/section/HeroSection.tsx`. 중심 엠블럼은 3D 별 `ThemedStar`(`pages/landing/ui/star3d/`, Star3D + StarCanvas, three 기반)로, 형태는 현재 `object`, 색은 `themeAccent(theme)`를 따른다.
- 1차 CTA "우주 만들어보기" → `enterDemoMode()` 후 `/universe`로 이동(로그인 없이 더미 우주).

### 스크롤 (랜딩 소유)

- `frontend/src/pages/landing/lib/scroll.ts`의 `useScrollToSection()` — `useLenis()` 인스턴스가 있으면 `lenis.scrollTo('#id', { offset, duration: 1.1 })`, 없으면(=reduced-motion 폴백 트리) `getElementById(id).scrollIntoView({ behavior:'smooth', block:'start' })`. 네이티브 `scrollIntoView`가 Lenis 내부 위치와 어긋나 목적지를 넘기던 버그를 막는다.
- `frontend/src/pages/landing/lib/use-in-view.ts`의 `useInView()` — 무거운 3D 캔버스를 화면 근처에서 지연 장착(`mounted` 래치)하고 보이는 동안만 렌더 루프를 켠다(`visible`). IntersectionObserver 부재(SSR 등)면 처음부터 둘 다 true.

## CSS 토큰 (`--ld-*`, 랜딩 스코프)

`frontend/src/app/styles/index.css`. `[data-landing-theme]` 블록 안에서만 유효하다.

- 공통 블록 `[data-landing-theme]`: `--ld-base`, `--ld-accent`, `--ld-accent-soft`, `--ld-hero-grad`, `--ld-glass-bg`, `--ld-glass-border`, `--ld-glass-blur`, `--ld-glass-shadow`, `--ld-grain-opacity`, `--ld-deband-opacity`. accent류는 전역 `--theme-accent*`(RootLayout `data-theme`)에서 끌어온다.
- 테마별 override 블록: `[data-landing-theme='vast'|'lively'|'calm']`이 히어로 그라디언트·글래스 틴트/블러/섀도·그레인 강도를 다르게 둔다.
- `[data-landing-theme] .glass`만 토큰으로 재테마(전역 `.glass`는 그대로). 유틸 `.ld-hero-grad`(히어로 텍스트 그라디언트), `.ld-grain`/`.ld-grain--deband`/`.ld-grain--animated`.

## Flutter 동등성 기준

랜딩은 의도된 마케팅/온보딩 표면으로 웹 전용이다. Flutter 클라이언트가 그대로 재현해야 할 백엔드 계약이나 공유 도메인 모델은 없다. 다만 두 가지가 cross-platform 개념으로 유효하다.

- 시각 설정 두 축(Theme `vast|lively|calm` × StarObject `deepfield|aurora|liquid|ember`)과 색-보존 규칙은 30의 per-user 서버 설정이 SSOT이므로, Flutter도 같은 enum과 "색은 의미·형태는 선호" 분리를 따라야 한다(30 문서 소유).
- 별·시냅스 시각 메타포(별=기억, 곡선 시냅스=연결, 형태는 seed에서 결정·곡선만 사용)는 우주 캔버스 스펙(06–09)이 cross-platform 기준을 소유한다.

## 수용 기준

1. `/`는 인증 없이 열리고, `data-landing-theme`가 현재 테마로 박힌 채 배경·히어로·섹션이 렌더된다.
2. 우하단 스위처에서 테마를 바꾸면 배경 atmosphere·히어로 그라디언트·글래스 크롬이 즉시 그 테마로 전환된다.
3. 오브제(별 형태)를 바꾸면 히어로 엠블럼과 직접 viz 섹션의 별·시냅스가 그 형태로 다시 그려진다.
4. 새로고침 후 마지막으로 고른 테마·오브제가 복원된다(localStorage `cosimosi.appearance`). 구 키 `cosimosi.landing.theme` 저장본은 적절한 축으로 1회 승계된다.
5. 히어로 "천천히 둘러보기"와 스크롤 힌트는 `concept` 장 상단에 안착하며 목적지를 넘기지 않는다(Lenis 우선, 없으면 네이티브).
