# 54. 재공고화 — 추상화 단계에 따른 AI 내용 변형

> 추상화가 진행된 별은 다시 떠올릴 때 AI가 내용을 조금씩 바꾼다(요지화·기억 왜곡) | 범위: FS | 상태: 기획
> 기억은 떠올릴 때마다 다시 쓰인다(재공고화). 형태(plan 53)에 더해, *내용*도 추상화 단계가 높을수록 변형되게 한다 — 단 원본 일기는 영원히 그대로(헌법1).

## 목적

재공고화 이론: 회상은 기억을 단순 재생이 아니라 **다시 쓰기**다 — 떠올릴 때마다 조금씩 변형·요지화된다. cosimosi는 지금 재공고화를 *시각 재성형*(밝기/색/형태 seed)으로만 표현하고 *내용*은 안 바꾼다(서버 PE=0). 이 plan은 별이 충분히 추상화됐을 때(요지화 단계 2 이상) **다시 열람하면 AI가 그 별의 표시 내용을 조금씩 변형**하게 해, "기억이 흐려지고 요점만·때로 왜곡되어 남는" 과정을 실제 텍스트로 드러낸다. 단계가 높을수록(4로 갈수록) 더 많이 바뀐다.

## 범위 / 비목표

**포함:**
- 별의 **표시 내용(파생 텍스트)** 을, `abstraction_stage ≥ 2`인 별을 다시 열람할 때 AI가 미세 변형(요지화/추상화). 단계 2<3<4로 변형 폭 증가.
- 변형은 **append-only 변천사(`evolution_history`)** 에 스냅샷으로 쌓여, 변천사 타임랩스(plan 24)로 되돌아볼 수 있다.
- 원본 일기(record)와 병치 가능 — "내 기억은 흐려졌지만 그날 쓴 말은 그대로".

**비목표:**
- **원본 record는 한 글자도 안 바뀐다(헌법1).** AI 변형은 별의 파생 레이어에만.
- 단계 0~1(아직 또렷한) 별은 내용 변형 없음 — 형태(plan 53)만.
- 형태(지오메트리) 변형은 plan 53 소관. 여기선 *내용*.
- 임의·무한 변형 아님 — 단계와 변형 폭 상한이 있고, 변천사로 추적 가능.

## 설계 요점

- **트리거.** 별 열람(회상) 시 `abstraction_stage ≥ 2`면 변형 잡을 큐잉(동기 아님 — AI 호출 비용·지연 고려, 결과는 다음 로드나 비동기 반영). plan 23의 재성형 경로(PE 게이트)와 같은 진입점에 AI 내용 변형을 잇는다.
- **변형 폭 = 단계 함수.** stage 2=가벼운 요지화, 3=중간, 4=강한 추상/왜곡. 프롬프트가 "원본의 요점을 유지하되 단계만큼 흐리게/추상적으로" 변형.
- **저장.** 변형된 파생 텍스트 + 메타를 `evolution_history`에 append(version++). 별의 현재 표시 텍스트는 최신 변천사. 원본 record는 불변 참조로 병치.
- **비용·안전.** AI 호출은 사용자별·별별 빈도 제한(같은 별 재변형 디바운스), 실패 시 기존 텍스트 유지(graceful). 데모/AI 없음 모드는 변형 skip(형태만).
- **proto/DB.** `evolution_history`에 내용 텍스트 컬럼(또는 기존 스냅샷 확장). 별 payload에 현재 파생 텍스트.

## 수용 기준 (인수 조건)
1. `abstraction_stage ≤ 1` 별을 열람하면 내용은 안 바뀐다.
2. `abstraction_stage ≥ 2` 별을 다시 열람하면 표시 내용이 AI로 미세 변형되고, 단계가 높을수록 변형 폭이 크다.
3. 모든 변형이 `evolution_history`에 append되고, 원본 record는 변경·삭제되지 않는다(헌법1·2).
4. 변천사 타임랩스(plan 24)에서 내용 변형 이력을 되짚을 수 있고 원본과 병치된다.
5. AI 호출 실패/부재(데모)에서도 별은 기존 내용으로 정상 렌더된다(graceful).
6. 같은 별이 짧은 시간에 과도하게 재변형되지 않는다(빈도 제한).
