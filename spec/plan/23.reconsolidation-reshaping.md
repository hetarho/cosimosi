# 23. 재공고화 재성형 + 변천사 데이터 (reconsolidation-reshaping)

> 회상이 예측 오차(PE) 게이트로 말랑한 창을 열어 별을 양방향으로(밝게/흐리게·색조·형태) 제한적으로 다시 빚고, 모든 변화를 append-only 변천사에 쌓는다. | 범위: FS | 선행: 21 | 차단: 24

## 목적 (Why)
설계 결정 3("별은 살아있다")과 §비전이 구현에 주는 함의의 **재공고화**를 실제 데이터·렌더로 구현한다. 회상은 단순 재점화(11의 `last_recalled_at=now`)에 그치지 않고, 그 회상이 **새로운 맥락(예측 오차)** 을 담을 때만 별을 말랑하게 만들어 **양방향으로 다시 빚는다** — 밝아지거나 흐려지고, 색조가 좁게 흔들리고, 형태 시드가 미세하게 jitter한다. 두 생물학적 경계를 지킨다: (1) **강하고 오래된 별일수록 덜 변한다**(강도 의존), (2) 변화는 **회상한 별 + 직접 이웃에만 한정**(내용 한정적). 모든 변형은 **append-only `evolution_history`** 로그에 쌓이고(24가 읽음), **불변 원본(records)은 한 글자도 건드리지 않는다**(헌법1). 공동 회상 강화엔 **간격 효과**를 얹어, 마지막 함께 본 이후 시간이 길수록 강화 부스트를 키운다.

## 범위
- **포함:**
  - **DB(신규 마이그레이션 `00005`):** `evolution_history` 테이블(append-only, INSERT 전용 — 헌법1/2) + `memories`에 재성형 상태 컬럼(`brightness_offset`/`hue_shift`/`form_seed_delta`/`version`) 추가.
  - **BE `memory`:** `RecallMemory`(11)를 확장 — touch 후 **PE 계산**(회상 맥락 임베딩 vs 마지막 공고화 임베딩 코사인) → `pe≥PE_THRESHOLD`면 **회상 별 + 직접 이웃**에 대해 경계 안의 재성형 적용 + `evolution_history` append. embeddings(03/05)에서 회상 별 벡터를 읽어 PE 산출.
  - **BE `db/queries`:** `memory.sql`에 재성형 상태 UPDATE·`evolution_history` INSERT·이웃/임베딩 조회 쿼리, `link.sql`의 `ReinforceLinks`에 간격 가중치(클라가 산출한 delta에 BE는 손대지 않으므로 **간격 부스트는 FE `co-recall` 모델**에서 적용).
  - **proto:** `Star`에 `brightness_offset`/`hue_shift`/`form_seed_delta`/`version` 추가(렌더 입력) + `EvolutionSnapshot` 메시지·`GetEvolutionHistory` rpc 정의(데이터 계약 — UI는 24).
  - **FE:** `entities/memory` 도메인/매퍼에 재성형 필드 반영 → `StarField`가 `aBrightness`/`aSeed`/색조에 합성(frozen-time idiom). `features/recall`의 co-recall 간격 부스트. 데모(체험)에서 반복 회상 시 별이 경계 안에서 변하고 변천사가 쌓이는 것 + novelty 없는 회상은 무변.
  - **랜딩:** `ReconsolidationCard` 업데이트(PE 게이트 발화·강화될수록 작아지는 변화폭·형태 jitter 추가).
- **비목표(Out of scope):**
  - 변천사 **타임랩스 UI**·불변 원본 나란히 보기 → **24(evolution-history-ui)**. 본 스펙은 **데이터(append-only 로그) + 우주 내 최소 재성형 반영**까지만.
  - 야간 요지화(`trigger='nightly_gist'` append) → **27(nightly-consolidation)**.
  - 우주 배경 "요즘 상태"·앰비언트 → **25(ambient-mood)**.
  - 관련성/감정 가중 감쇠(λ_eff) → **26(relevance-emotion-decay)**.
  - 일기 → N 조각 별 fan-out·per-fragment 감정·`extract` job → **21(memory-fragmentation, 선행)**. 본 스펙은 21이 만든 조각 별(`memories.mood/intensity`, `fragment_text`)을 전제로 회상 재성형만 얹는다.

## 설계 요점

**재공고화 모델(파라미터).** 회상 시 다음을 계산한다(서버 비즈니스 로직 — 헌법3, 좌표 무관).
- **예측 오차(PE) 게이트:** `pe = clamp(1 - cos(recall_ctx_emb, last_consolidated_emb), 0, 1)`. MVP는 회상 별 자신의 임베딩을 `recall_ctx_emb`로, 마지막 재성형 시점의 임베딩 스냅샷(없으면 별 임베딩 자체 = pe 0)을 `last_consolidated_emb`로 둔다. `pe < PE_THRESHOLD=0.15`면 **재성형 없음**(novelty 없는 회상 = 단순 재점화만, 11과 동일).
- **강도 의존 강도(strength):** `strength = f(recall_count, age) = clamp(0.15·log2(1+co_recall_total) + age_term, 0, 1)`. 별이 자주·오래 공고화될수록 1에 가까워진다.
- **재성형 크기:** `magnitude = BASE_STEP · pe · (1 - strength)`. → `strength↑ ⇒ magnitude↓`(공고화될수록 덜 흔들림).
- **양방향 적용(경계 안):** 방향은 회상 별의 시드+version에서 결정론적으로 정한다(난수 재생성이 아닌 제약된 드리프트, `ReconsolidationCard.nextMemory`와 동치).
  - `brightness_offset += dir · clamp(magnitude, 0.10, 0.22)` (±0.10~0.22 한 발).
  - `hue_shift = clamp(hue_shift + jitter, -28, +28)` (감정 기준 색 ±28° 이내).
  - `form_seed_delta = clamp(form_seed_delta + small_jitter, -FORM_DELTA_MAX, +FORM_DELTA_MAX)` (형태 시드 미세 jitter).
  - `version++`.
- **범위(내용 한정적 — 헌법):** 회상 별 + **직접 이웃(memory_links로 1-홉)** 만 재성형. 간접 이웃·나머지 우주는 불변. 이웃 적용은 더 작은 크기(`NEIGHBOR_FACTOR=0.4·magnitude`).
- **간격 효과(Katz 2021):** 공동 회상 강화 delta를 마지막 함께 본 이후 경과에 비례해 키운다 — FE `co-recall`에서 `delta = CO_RECALL_DELTA · (1 + SPACING_GAIN · clamp(gapDays/SPACING_REF, 0, 1))`(한 세션 몰아보기 < 간격 두기). BE `ReinforceLinks`(11)는 클라 산출 delta를 그대로 멱등 업서트한다(변경 없음).

**proto 계약(누적 — 21이 `RecordMemoryResponse{record_id, memory_ids}` 추가; 23이 아래를 추가).** 모두 unary(헌법6). `Star`에 재성형 상태(렌더 입력) 추가 — 좌표 필드는 절대 없다(헌법3).
```proto
// Star: 재성형 상태(가변 별 레이어)를 노출. brightness_offset/hue_shift는 클라 셰이더 입력에 합성.
message Star {
  string memory_id = 1;
  Mood mood = 2;
  double intensity = 3;
  string last_recalled_at = 4;
  double brightness_offset = 5;   // 23: 누적 ±밝기 오프셋(재공고화)
  double hue_shift = 6;           // 23: 감정 기준 색 ±28° 색조
  double form_seed_delta = 7;     // 23: 형태 시드 미세 jitter
  int32  version = 8;             // 23: 재성형 횟수(변천사 길이)
}
// 24가 읽는 append-only 스냅샷(여기서 메시지·rpc 계약만 정의, UI는 24).
message EvolutionSnapshot {
  int32  version = 1;
  double brightness = 2;
  double hue_shift = 3;
  double form_seed_delta = 4;
  string trigger = 5;             // 'recall' | 'new_neighbor' | 'nightly_gist'(27)
  double pe = 6;
  int32  dir = 7;                 // +1 강화 / -1 약화
  string created_at = 8;
}
message GetEvolutionHistoryRequest { string memory_id = 1; }
message GetEvolutionHistoryResponse { repeated EvolutionSnapshot snapshots = 1; }
// service MemoryService 에 추가:
//   rpc GetEvolutionHistory(GetEvolutionHistoryRequest) returns (GetEvolutionHistoryResponse);
```

**DB — `evolution_history`(append-only, 헌법1/2) + `memories` 재성형 컬럼(신규 마이그레이션 `00005`).**
```sql
-- +goose Up
-- 변천사: 추가만 되는 로그(UPDATE/DELETE 쿼리 절대 금지 — 헌법1/2). 24가 읽고 27이 gist로 append.
CREATE TABLE evolution_history (
    id              TEXT PRIMARY KEY,
    memory_id       TEXT NOT NULL REFERENCES memories(id),
    user_id         TEXT NOT NULL,                 -- 격리
    version         INT  NOT NULL,
    brightness      REAL NOT NULL,                 -- 그 시점 brightness_offset 스냅샷
    hue_shift       REAL NOT NULL,
    form_seed_delta REAL NOT NULL,
    trigger         TEXT NOT NULL,                 -- 'recall' | 'new_neighbor' | 'nightly_gist'
    pe              REAL NOT NULL,
    dir             INT  NOT NULL,                 -- +1 / -1
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
-- +goose StatementBegin
CREATE INDEX evolution_history_memory_idx ON evolution_history (memory_id, version);
-- +goose StatementEnd
-- 가변 별 레이어에만 재성형 상태(헌법1: 불변 records엔 절대 두지 않음).
ALTER TABLE memories ADD COLUMN brightness_offset REAL NOT NULL DEFAULT 0;
ALTER TABLE memories ADD COLUMN hue_shift         REAL NOT NULL DEFAULT 0;
ALTER TABLE memories ADD COLUMN form_seed_delta   REAL NOT NULL DEFAULT 0;
ALTER TABLE memories ADD COLUMN version           INT  NOT NULL DEFAULT 0;
-- +goose Down
ALTER TABLE memories DROP COLUMN brightness_offset, DROP COLUMN hue_shift,
    DROP COLUMN form_seed_delta, DROP COLUMN version;
DROP TABLE IF EXISTS evolution_history;
```

**BE 쿼리(`memory.sql`, sqlc).** `RecallMemory`(11) 경로에 재성형을 얹는다. 재성형은 별만 UPDATE하고 원본 records는 절대 건드리지 않는다(grep으로 `UPDATE records` 0건 보장).
```sql
-- name: GetReshapeContext :one  (memory.sql) — PE/strength 입력(회상 별 1행)
-- version/누적 상태 + co_activation 합(강도) + 임베딩(PE용). embeddings는 03/05가 채움.
SELECT m.version, m.brightness_offset, m.hue_shift, m.form_seed_delta,
       e.embedding,
       COALESCE((SELECT SUM(ml.co_activation_count) FROM memory_links ml
                 WHERE (ml.a_id = m.id OR ml.b_id = m.id) AND ml.user_id = m.user_id), 0)::int AS co_recall_total,
       m.created_at
FROM memories m JOIN embeddings e ON e.memory_id = m.id
WHERE m.id = @id AND m.user_id = @user_id;
-- name: ListDirectNeighbors :many  (memory.sql) — 1-홉 직접 이웃만(내용 한정적 범위)
SELECT CASE WHEN ml.a_id = @id THEN ml.b_id ELSE ml.a_id END AS neighbor_id
FROM memory_links ml
WHERE ml.user_id = @user_id AND (ml.a_id = @id OR ml.b_id = @id);
-- name: ApplyReshape :exec  (memory.sql) — 가변 별만 UPDATE(원본 records 불변 — 헌법1)
UPDATE memories
SET brightness_offset = @brightness_offset, hue_shift = @hue_shift,
    form_seed_delta = @form_seed_delta, version = version + 1
WHERE id = @id AND user_id = @user_id;
-- name: AppendEvolution :exec  (memory.sql) — INSERT 전용(append-only — 헌법1/2)
INSERT INTO evolution_history (id, memory_id, user_id, version, brightness, hue_shift, form_seed_delta, trigger, pe, dir)
VALUES (@id, @memory_id, @user_id, @version, @brightness, @hue_shift, @form_seed_delta, @trigger, @pe, @dir);
-- name: GetEvolutionHistory :many  (memory.sql) — 24가 읽음(여기선 계약만)
SELECT version, brightness, hue_shift, form_seed_delta, trigger, pe, dir, created_at
FROM evolution_history WHERE memory_id = @memory_id AND user_id = @user_id ORDER BY version ASC;
```
`ListMemoriesByUser`/`ListDormant`는 `m.brightness_offset, m.hue_shift, m.form_seed_delta, m.version`을 추가 select(매퍼·`Star` 채움).

**BE 서비스(`service.go`).** `RecallMemory`를 touch → PE 계산 → 게이트 → (통과 시) 회상 별 + 직접 이웃 재성형 + append → `GetRecord` 반환으로 확장. PE 미달이면 **재성형 0건**(11과 동치). 재성형 수치 함수(`reshapeStep`/`strengthOf`/`directionFor`)는 순수 헬퍼로 단위 테스트한다(`buildLinks`/`temporalBonus`와 같은 패턴, worker.go).

**FE 도메인·매퍼(`entities/memory`, 순수 — three/React/DOM 미의존, 헌법4).** `Memory`에 `brightnessOffset`/`hueShift`/`formSeedDelta`/`version`을 더하고 `mapStar`가 채운다.
```ts
// entities/memory/model/types.ts — Memory 확장(순수)
export interface Memory {
  id: string
  mood: Mood
  intensity: number
  lastRecalledAt: number
  seed: number
  brightnessOffset: number   // 23: 재공고화 누적 ±밝기
  hueShift: number           // 23: 감정 기준 색 ±28°
  formSeedDelta: number      // 23: 형태 시드 미세 jitter
  version: number            // 23: 재성형 횟수
}
// entities/memory/model/reshape.ts — 렌더 합성식(순수, 클라/모바일 공용)
/** 재성형 반영 별 유효 밝기 = clamp(starBrightness + offset, A_MIN, 1). */
export function reshapedBrightness(base: number, offset: number): number
```

**FE 렌더(`entities/star/ui/StarField.tsx`).** `aBrightness`는 `reshapedBrightness(starBrightness(...), m.brightnessOffset)`로, `aSeed`는 `reshapedSeed(m.seed, m.formSeedDelta)`로 채운다(useLayoutEffect rebuild + Focus 재가중 동일 식). 색조 hueShift는 per-instance attribute(`aHueShift`)로 머티리얼에 합성. **회상 직후 갱신은 GetUniverse refetch로 반영**(낙관 갱신 아님 — §4.6 "연결은 다음 refetch"와 같은 결). 어떤 애니메이션(재성형 펄스 등)도 **manual uniform**을 `update(elapsed)`에서 bump한다 — three 내장 `time` 노드는 BloomPass에서 frozen이라 금지(MEMORY: TSL time-node frozen).

**FE 간격 부스트(`features/recall/model/co-recall.ts`).** 페어별 마지막 능동 열람 시각을 들고, 새 co-recall 이벤트의 delta를 위 간격 식으로 키운다(상한 후 BE가 1.0 cap). 순수 유지(setTimeout/Date만, three/DOM 금지 — 헌법4).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 사용자가 별을 회상하고 `pe < PE_THRESHOLD=0.15`이면(novelty 없음) THE SYSTEM SHALL `last_recalled_at`만 갱신하고(11과 동일) **재성형하지 않으며** `evolution_history`에 행을 추가하지 않는다. _(헌법1, concept §결정 3 "변형은 강제가 아님")_
1.2 WHEN 회상의 `pe ≥ 0.15`이면 THE SYSTEM SHALL 회상 별의 `brightness_offset`을 `dir·clamp(magnitude,0.10,0.22)`만큼, `hue_shift`를 ±28° 이내로, `form_seed_delta`를 미세 jitter로 **양방향** 갱신하고 `version++` 한다. _(concept §결정 3 "양방향")_
1.3 GIVEN 두 별의 `pe`가 같을 때 WHEN 더 자주·오래 공고화된(강도↑) 별을 회상하면 THE SYSTEM SHALL 더 작은 `magnitude`를 적용한다(`magnitude = BASE_STEP·pe·(1-strength)`, `strength↑ ⇒ magnitude↓`). _(과학적 근거 — 강도 의존 PE)_
1.4 WHEN 한 회상이 별을 재성형할 때마다 THE SYSTEM SHALL `evolution_history`에 `{version, brightness, hue_shift, form_seed_delta, trigger='recall', pe, dir}` 1행을 **INSERT(추가만)** 하고, 기존 변천사 행을 UPDATE/DELETE하지 않는다. _(헌법1·2, concept §변천사 append-only)_
1.5 WHEN 회상으로 재성형이 발생할 때 THE SYSTEM SHALL 회상한 별과 **직접 이웃(1-홉 memory_links)** 만 변경하고, 간접 이웃·나머지 우주는 그대로 둔다(이웃은 `NEIGHBOR_FACTOR` 축소 크기). _(concept §결정 3 "내용 한정적", 헌법)_
1.6 WHEN 어떤 회상·재성형이 일어나도 THE SYSTEM SHALL 원본 `records`(body·entry_date·mood·intensity)를 한 글자도 변경하지 않는다(`UPDATE/DELETE records` 쿼리 0건). _(헌법1)_
1.7 WHEN 재성형이 누적되어도 THE SYSTEM SHALL 별·시냅스를 삭제하지 않고 유효 밝기를 `a_min=0.05` 바닥 위로 유지한다(`reshapedBrightness = clamp(starBrightness+offset, A_MIN, 1)`). _(헌법2, §6)_
1.8 WHEN `GetUniverse`/`ListDormant`/`RecallMemory`가 `Star`를 반환할 때 THE SYSTEM SHALL `brightness_offset`/`hue_shift`/`form_seed_delta`/`version`을 포함하되 좌표 필드는 포함하지 않는다. _(헌법3, 헌법5)_
1.9 WHEN 페이지를 새로고침(refetch)하면 THE SYSTEM SHALL 누적된 재성형 상태가 영속되어 `GetUniverse`가 변형된 별을 반환하고, FE가 `aBrightness`/`aSeed`/`aHueShift`에 합성해 렌더한다. _(영속, 헌법8)_
1.10 WHEN `entities/memory/model/reshape.ts`·`features/recall/model/co-recall.ts`가 빌드되면 THE SYSTEM SHALL `three`/React/DOM을 import하지 않는다(순수·모바일 재사용). _(헌법4)_
2.1 GIVEN 한 페어를 직전에 함께 본 직후 다시(짧은 간격) 회상 WHEN co-recall delta를 산출하면 THE SYSTEM SHALL 간격을 두고 다시 본 경우보다 **작은** 부스트를 적용한다(간격 효과). _(과학적 근거 — Katz 2021)_
2.2 WHEN FE가 재성형 펄스 등 노드 머티리얼을 애니메이션할 때 THE SYSTEM SHALL 내장 TSL `time` 노드 대신 `update(elapsed)`로 bump하는 **manual uniform**을 쓴다(BloomPass에서 내장 time이 frozen). _(헌법8, 렌더 정합)_
3.1 WHEN 클라이언트가 `GetEvolutionHistory(memory_id)`를 호출하면 THE SYSTEM SHALL 그 별의 스냅샷을 `version` 오름차순으로 반환한다(UI 렌더는 24). _(데이터 계약)_
