# 36. recall flow ui

> Recall flow UI — the 회고하기 flow launched from the star-detail panel: _회고하기 → sync-consent (if the clock
> is behind) → rewrite the faded memory → the reconsolidation result lands on the star (reshape on prediction error,
> reinforce-only otherwise)_. One XState machine owns the flow's control-state; the rewrite text + result live in
> Zustand/Query. It **composes** the sync-consent modal built in Epic B and the Recall/Reconsolidate contract built in
> Epic C's use-case plans.
> Scope: FE | Status: built — web + mobile (UX policy: [policy/ux/recall-flow.md](../policy/ux/recall-flow.md))
>
> **As-built note:** the flow calls the `Recall` RPC via the shared `requestRecall`/`applyRecallResult`/
> `recallAdvanceAnnouncement` helpers in `@cosimosi/universe` (mirroring the launch helpers), and applies only the
> read-model-held anchors on success — `seed`, `recall_count`, `last_recalled` (the mirror carries no `current_text`,
> so the reshape rides the new seed; the faded-text prompt reuses `current-memory-text`, whose text source is still
> deferred). The panel→flow open is a shared one-slot `recall-target` store. `RecallResponse`/`RecallRequest` are now
> re-exported from `@cosimosi/api-client`.
>
> Layer P (presentation) — the closing unit of Epic C. Home: `widgets/writing-flow`'s sibling `widgets/recall-flow`
> (composing `features/recall-star` + the rewrite input) launched from [35.star-detail-panel](35.star-detail-panel.md)'s
> **회고하기** button, over the running `widgets/universe-canvas` ([23.universe-canvas](23.universe-canvas.md)). It fills
> the recall _UI + control-state_ seam; it leaves the recall/reconsolidation domain to [33.recall-usecase](33.recall-usecase.md)
> / [32.reconsolidation-rules](32.reconsolidation-rules.md), the sync modal to [31.time-acceleration-ui](31.time-acceleration-ui.md),
> and the Twinkle price to Epic F.

## Purpose

Epic C's read side lets a diarist open a past star and see its **faded current text** for free ([R1], via
[35.star-detail-panel](35.star-detail-panel.md)). But the product's central act — the thing the whole memory model is
built around — is _recall as reconsolidation_: there is no "show me the original" recall. To recall is to **summon a
blurred memory and write it again**, and in that writing the trace is reinforced and, when it drifts, reshaped ([R1],
PRD [§5.3.4](../PRD.md)). This unit is the **one flow** through which that happens.

It exists so the four moments of a recall — pressing 회고하기, consenting to sync the universe clock to today, rewriting
the faded memory, and seeing the result land on the star — are composed **once** as a single guided flow, rather than
scattered across the detail panel, the time overlay, and the canvas. It composes the sync-consent modal built by
[31.time-acceleration-ui](31.time-acceleration-ui.md) ([T2] case 2 / [R1a]) and calls the Recall/Reconsolidate contract
owned by [33.recall-usecase](33.recall-usecase.md) (whose rules are [32.reconsolidation-rules](32.reconsolidation-rules.md)).
It builds the **flow UI and its control-state only** — the clock/sync, the recall/reconsolidation domain, the visible
brightness/decay recovery (Epic D), and the Twinkle cost (Epic F) are their own plans.

## Scope / Non-goals

**In scope**

- **`widgets/recall-flow`** — a modal / bottom-sheet launched from the star-detail panel's **회고하기** button
  ([R1][D1]); it mounts over the running `widgets/universe-canvas` (it does not remount the renderer, [14]) and is the
  peer of [27.writing-flow-ui](27.writing-flow-ui.md)'s `widgets/writing-flow`. Scoped to an **episodic** star; the panel
  does not offer 회고하기 for a gist star (gist = read-only viewing, [34.view-semantic-usecase](34.view-semantic-usecase.md)/[R8]).
- **Compose the sync-consent modal** ([T2] case 2 / [R1a]) — when the universe clock ≠ today, the flow opens the reusable
  **`features/confirm-time-sync`** modal built by [31.time-acceleration-ui](31.time-acceleration-ui.md) through its
  public API. **예** ⇒ proceed to the `Recall` server call; `Recall` composes `SyncToToday`
  ([30.time-advance-usecase](30.time-advance-usecase.md)) and the recall write in one transaction, then returns the sync
  interval for [31]'s acceleration/reveal. **아니오** ⇒ cancel the whole flow with the clock **unmoved** ([R1a]). When the
  clock is already today the modal is skipped.
- **`features/recall-star`** — the rewrite input + the recall call. Renders the star's **faded current text** as the
  prompt-to-remember (read from the detail panel's already-loaded memory), a **rewrite** text area, and a **회고 confirm**
  affordance. On confirm it calls the **`Recall`** RPC ([33.recall-usecase](33.recall-usecase.md)) with the rewrite; the
  server runs the prediction-error compare + Recall/Reconsolidate and returns the outcome. Natural, restrained copy — the
  distortion is never announced ([R8a]).
- **Render the reconsolidation result on screen** — the response says which branch fired:
  - **Prediction error ([R6])** ⇒ _reconsolidated_: the star's **seed changed → shape changes** ([V5][R6]) and its
    **current text** is the rewrite; brightness resets to max ([R2]) and the decay stage recovers ([F5]). The flow
    surfaces the reshape on the canvas (new seed) and shows the new current text; provenance gains a `reconsolidated/user`
    entry ([R8a], via [33]).
  - **No prediction error ([R4])** ⇒ _reinforced only_: **shape unchanged**, current text unchanged; brightness/strength/
    gist-timer recovery only ([R3][R4][C6a]). The flow surfaces a restrained "reinforced, unchanged" outcome.
- **The flow control-state as one XState machine** — `model/recall-flow.machine.ts` (§3.2): the exclusive phases
  `idle → confirmingSync → rewriting → reconsolidating → result` (+ error/cancel paths). `reconsolidating` is the
  **loading** state covering the synchronous LLM compare + recall latency. The rewrite text, the loaded memory ref, and
  the result live in **Zustand/Query**; the machine holds ids/refs + the current phase only (§3.2).
- **Optimistic-but-server-authoritative result** — the _branch_ (reconsolidated vs reinforced), the _new seed_, the _new
  current text_, and the recovered brightness/decay/timer anchors are **server decisions** ([R6] LLM compare, [33]/[32]);
  the flow does **not** guess the branch client-side. It applies the result to the `entities/episodic-memory` store on
  success and the canvas reflects it; a failed recall rolls back to a retriable rewrite.
- **UI copy via i18n** ([08.i18n-seam](08.i18n-seam.md)) — no hardcoded strings; a **literary, restrained Korean** voice
  (the rewrite prompt especially — an invitation to remember, not a form). **web + mobile parity** — the same
  widget/feature; `*.native.tsx` siblings **only** for the sheet / text-input primitives that genuinely differ (§3.5).

**Non-goals**

- **No recall/reconsolidation domain.** The prediction-error compare (LLM), `Recall`, `Reinforce`, `Reconsolidate`, the
  seed change, the brightness/decay/timer/LTP anchor writes, the neighbor forgetting ± ([R5]), and provenance appends are
  owned by [33.recall-usecase](33.recall-usecase.md) / [32.reconsolidation-rules](32.reconsolidation-rules.md). This unit
  only _calls_ the contract and renders the result.
- **No clock / sync / acceleration.** `SyncToToday`, the clock value, the sync-consent modal component, and the
  acceleration choreography are [30.time-advance-usecase](30.time-advance-usecase.md) / [31.time-acceleration-ui](31.time-acceleration-ui.md).
  This unit _opens_ the modal and _sequences_ sync-then-recall; it neither owns the modal nor calls the clock directly.
- **No star-detail panel.** The meta view, the free faded current text, the 변천사 (provenance) view, and 원본 일기 보기
  are [35.star-detail-panel](35.star-detail-panel.md); this flow is _launched from_ the panel's 회고하기 button.
- **No gist viewing.** Reading a gist-star's text is [34.view-semantic-usecase](34.view-semantic-usecase.md)/[R8] — a
  separate, read-only, non-reshaping path; 회고하기 is episodic-only ([R1]).
- **No visible forgetting recovery visuals.** The brightness dimming / word-loss rendering that makes the reset _visible_
  is [39.forgetting-visuals](39.forgetting-visuals.md) (Epic D); this flow triggers the anchor reset, the felt recovery
  lands when D ships (**CC1** seam).
- **No economy gating.** Whether a recall costs 별가루 ([G1]) is Epic F ([44.earn-spend-usecase](44.earn-spend-usecase.md)/
  [45.stardust-ui](45.stardust-ui.md)); Epics C–E bind a **no-op allow-all** spend gate (**CC2**), so the flow assumes it
  may proceed and hardcodes no price.
- **No whole-diary recall.** "이 일기로 태어난 별 보기" ([D3]) — recalling every star of one diary at once — is
  [47.diary-reader-page](47.diary-reader-page.md). This flow recalls **one** star.
- No new tuning scalars hardcoded in FE (they come from generated config, §Policy).

## Design

### Placement (FSD, §3.1)

```
apps/web/src/widgets/recall-flow/
├── ui/            RecallFlowSheet (modal/bottom-sheet host: rewrite prompt + result view) + *.native.tsx sibling for sheet/input
├── model/         recall-flow.machine.ts (XState) · the rewrite/result Zustand store · pure result-branch helpers
└── index.ts       public API (the "open recall" affordance the star-detail panel's 회고하기 button calls)

apps/web/src/features/
└── recall-star/   ui (faded-text prompt + rewrite textarea + 회고 confirm + result surface) · api (Recall call + result→domain map + optimistic store apply)
```

- `widgets/recall-flow` composes **one** feature (`features/recall-star`) plus the **composed** `features/confirm-time-sync`
  (imported from [31]'s `widgets/universe-time` public API — a widget composing another widget's public affordance,
  mirroring how [27]'s launch triggers [25]'s awaken). It owns the **flow machine** and the rewrite/result store. It
  imports the `entities/episodic-memory` mirror from [23] to read the target memory and apply the result; it never
  imports `three` or a visual entity (§3.4) — the reshape (new seed) is the read model's projection, not this widget's
  code.
- **Single-feature widget rationale (§3.1).** Unlike [27]'s four-feature writing flow, recall is _one_ user action
  (rewrite-and-recall) plus a _composed_ consent modal. The widget still earns its existence: it owns the multi-phase
  machine + the sync-then-recall sequencing + the result branch, which are more than one feature's concern. **Decided
  (2026-07): recall stays a widget** (not a bare feature) because it sequences a composed cross-widget modal, a synchronous
  LLM call, and a two-branch result — flow orchestration that belongs at the widget layer.
- **web + mobile parity (§3.5):** the same widget + feature on both apps; `model` and `api` segments shared verbatim
  (pure, no DOM/native). Only `ui` forks where a primitive differs — the **sheet host** and the **rewrite text input** —
  via `*.native.tsx` siblings. web = open-beta scope, mobile = MVP.

### The recall contract (owned by [33], consumed here — §2.7 unary)

- **`Recall(RecallRequest{ episodic_memory_id, rewrite_text }) → RecallResponse{ outcome, ... }`** — the single call this
  flow makes. The server ([33]/[32]) runs the **prediction-error compare** (LLM semantic compare of `rewrite_text` vs the
  memory's current text — content change, not wording, **CC6**) and then:
  - on **error** → _reconsolidated_: updates `current_text`, changes `seed`, regenerates the **remaining** stage texts
    ([C7]), resets brightness ([R2]) + recovers decay stage ([F5]) + resets the gist-timer ([C6a]) + LTP ([R3]) + size
    bump ([R3]) + neighbor forgetting ± ([R5]), appends a `reconsolidated/user` provenance row ([R8a]).
  - on **no error** → _reinforced_: brightness/strength/gist-timer recovery + LTP only; **`current_text` and `seed`
    unchanged** ([R4]).
  - `RecallResponse` returns at minimum `{ outcome: RECONSOLIDATED | REINFORCED, current_text, seed }` (the fields the FE
    needs to reflect the branch); the exact response shape is owned by [33] (proto/DB schema is code, not this plan). **No
    persistence happens until confirm** — the rewrite text is session-only until `Recall` succeeds.
- **Schema-forced boundary ([I3][I11]):** the request carries **only** `{ episodic_memory_id, rewrite_text }` — no
  position / color / strength / seed / time fields. There is structurally nothing for the user (or a prompt-injected
  model) to set that could violate [I3]: the user rewrites _text_; the seed change is a **server** decision on error, not
  a user input.
- **The sync seam ([R1a]):** the flow never calls `SyncToToday` directly. It collects consent through [31], then calls
  `Recall`; [33] owns the server transaction that syncs the clock and applies the recall atomically, and [31] owns the
  acceleration over the returned interval. Order is fixed: **consent → `Recall`(sync + recall atomically) → acceleration
  / result reveal**, so recall always runs with `universe_time = today` and there is no "clock moved, recall failed"
  partial state (**[T5]**: one clock, always).

### Flow machine (`model/recall-flow.machine.ts`, §3.2)

Control-state only (the "one of N phases" of the recall session); **all data** (the loaded memory ref, the rewrite text,
the result) lives in the Zustand/Query store, referenced by id from the machine.

| State             | Meaning                                                 | Exits                                                                              |
| ----------------- | ------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| `idle`            | sheet closed                                            | `OPEN(memoryId)` → `confirmingSync` (clock ≠ today) or `rewriting` (clock = today) |
| `confirmingSync`  | sync-consent modal open (composed from [31])            | `ACCEPT` → `rewriting`; `REJECT` → `idle` (clock unmoved)                          |
| `rewriting`       | faded text shown + rewrite being entered ([R1])         | `RECALL` → `reconsolidating`; `CLOSE` → `idle`                                     |
| `reconsolidating` | **loading** — `Recall` in flight (LLM compare + recall) | `done` → `result`; `error` → `rewriting` (retriable)                               |
| `result`          | the outcome shown; the star updated on the canvas       | `CLOSE` / `RESET` → `idle`                                                         |

- `confirmingSync` is entered **only** when the clock is behind today; a clock already at today transitions straight to
  `rewriting`. `REJECT` returns to `idle` with the clock **unmoved** ([R1a]) — the whole flow is abandoned, no recall.
- `reconsolidating` is an explicit loading state so the UI can show a restrained "떠올리는 중" affordance during the
  synchronous LLM compare + recall, and so a failed/timed-out call returns to a retriable `rewriting` (retry/token caps
  are the backend's, [33]/[28.ai-provider-abstraction](28.ai-provider-abstraction.md) — the FE only surfaces the ceiling).
- Per §3.2, any per-frame scene effect (the reshape reveal on the canvas) reads the machine via `getSnapshot()` in
  `useFrame`, never via React state; in practice the reshape is a discrete event handed to the read model on `result`.

### The result branch — reshape vs reinforce (server-decided, [R4][R6])

On `Recall` success the response's `outcome` drives which surface renders (pure branch helper in `model`):

- **RECONSOLIDATED ([R6][V5]):** apply the returned `current_text` + `seed` to the `entities/episodic-memory` store; the
  canvas re-renders the star with the **new seed** (shape changes — [24.star-neuron-rendering](24.star-neuron-rendering.md)
  reads seed) and its brightness reads full again (anchors reset; the _visible_ dimming/recovery lands with
  [39.forgetting-visuals](39.forgetting-visuals.md), **CC1**). The result view shows the new current text in restrained
  copy — the distortion is **not** announced ([R8a]); the 변천사 in the detail panel is where the change is discoverable.
- **REINFORCED ([R4]):** the star's **shape and current text are unchanged**; only brightness/strength/timer recovered.
  The result view states, plainly, that the memory was recalled and reinforced without change. No seed reveal.
- The FE **never** decides the branch — it reflects the server's `outcome` ([R6] compare is server-side, **CC6**). The
  neighbor forgetting ± ([R5]) and provenance append are server side-effects the FE does not render here (they surface via
  the next `GetUniverse` read and the detail panel's 변천사, respectively).
- **Failure / rollback:** if `Recall` fails, nothing is applied to the store (persistence is server-side, per-call) and
  the machine returns to `rewriting` with the rewrite text intact and retriable. The `Diary` is never touched under any
  branch ([I2][R7]).

### The rewrite prompt — remember, then write ([R1][I8])

- `features/recall-star` shows the star's **faded current text** (the word-erased current memory, read from the
  already-loaded panel memory — the same free text [35] shows) as the _prompt to remember_, and a rewrite text area. The
  copy frames it as summoning a blurred memory to write again, not as an edit form — it never shows or restores the
  original `Diary` ([I8]: recall does not restore original content).
- The rewrite is **session-only** until `Recall` succeeds; there is no draft persistence and no per-keystroke server
  call. The single synchronous `Recall` call on confirm carries the whole rewrite.

### Copy (i18n) — literary, restrained Korean

All user-visible strings resolve through the i18n seam ([08.i18n-seam](08.i18n-seam.md)); none hardcoded
(`lint:raw-strings`). The rewrite prompt, the "떠올리는 중" loading affordance, the reconsolidated/reinforced result
copy, and the composed sync-consent copy (owned by [31]) use the restrained literary voice (PRD [§3.1](../PRD.md)) — an
invitation to remember, plain and honest, no decorative emoji, no translation-ese, no announced distortion ([R8a]). Web
and mobile draw from the **same** message keys (parity).

## Acceptance Criteria

1. A **회고하기** button on the star-detail panel opens `widgets/recall-flow` as a modal / bottom-sheet **over** the
   running universe canvas (the renderer is not remounted); the flow is scoped to an **episodic** star and is not offered
   for gist stars ([R1][D1][14]).
2. When the universe clock ≠ today, the flow opens the **reusable sync-consent modal** built in
   [31.time-acceleration-ui](31.time-acceleration-ui.md) ([T2] case 2 / [R1a]); **예** ⇒ the later `Recall` call
   server-side syncs via `SyncToToday` and applies the recall atomically, then [31] plays the returned interval while the
   result is revealed; **아니오** ⇒ the whole flow is cancelled with the clock **unmoved**.
3. When the clock is already today, the sync-consent modal is **skipped** and the flow goes straight to rewriting.
4. `features/recall-star` shows the star's **faded current text** as the prompt-to-remember (the same free text the panel
   shows, [35]) and a **rewrite** text area; the original `Diary` is never shown or restored ([R1][I8]).
5. On **회고 confirm** the flow calls the **`Recall`** RPC ([33]) with `{ episodic_memory_id, rewrite_text }` as a
   **synchronous** call (a `reconsolidating` loading state covers the LLM compare + recall); the request carries **no**
   position / color / strength / seed / time fields ([I3][I11]).
6. On **prediction error** ([R6]) the result is **reconsolidated**: the star's **seed changes → shape changes** ([V5])
   and its **current text** becomes the rewrite; brightness/decay/gist-timer/strength recover (anchors reset, the visible
   recovery lands with [39.forgetting-visuals](39.forgetting-visuals.md)); the distortion is **not** announced ([R8a]).
7. On **no prediction error** ([R4]) the result is **reinforced only**: the star's **shape and current text are
   unchanged**; only brightness/strength/gist-timer recovery occurs; the result view states this plainly.
8. The FE **never** decides the branch — it reflects the server's `outcome` ([R6] compare is server-side); a failed
   `Recall` applies nothing and returns to a retriable `rewriting` with the rewrite text intact.
9. The **`Diary` is never mutated** by recall or reconsolidation under any branch — only the representation (seed +
   current text) changes ([I2][R7]).
10. The flow's **control-state** is one XState machine (`idle → confirmingSync → rewriting → reconsolidating → result`
    - error/cancel paths) with `reconsolidating` as the loading state; the **loaded memory ref, rewrite text, and result
      live in Zustand/Query**, not machine context (§3.2).
11. The flow **hardcodes no 별가루 price** and proceeds under the no-op allow-all spend gate; economy gating is deferred to
    Epic F (**CC2**), and it makes **no** direct clock/sync call (sync goes through [31]'s composition, [R1a]).
12. **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in a literary /
    restrained Korean voice; the distortion is never announced ([R8a]).
13. **web + mobile parity:** the same widget + feature run on both apps, sharing `model` / `api` verbatim; `*.native.tsx`
    siblings exist **only** for the sheet / rewrite-input primitives (§3.5).

## Policy / Values Impact

- **policy/ux** — the rules this flow makes true land in a new `policy/ux/recall-flow.md` once implemented (written at
  `/implement-job`; **do not** pre-create policy files):
  - **회고하기 = reconsolidation, not retrieval.** Recall is _summon-and-rewrite_; there is no "show me the original"
    recall. Viewing the faded current text is free ([R1]); rewriting it is the recall act.
  - **Sync requires consent, then stays server-atomic.** A recall while the clock is behind shows the sync-consent modal
    first; **아니오** cancels with the clock unmoved. On **예**, the subsequent `Recall` call performs sync + recall
