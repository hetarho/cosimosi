> the STAGE-RISING math, not text generation. Golden-parity for the timer + z-coord, mirrored TS↔Go.

## Purpose

Over universe-time an episodic memory does not only *fade* ([F], forgetting) — it also **abstracts**: the concrete "on
Tuesday I ate carbonara with Jinsu and it was so good" compresses toward "a warm dinner with a friend," and a **gist
version rises into the neocortex** as a separate, higher body ([C5]–[C7], complementary learning systems; McClelland
1995). This is the *systems consolidation* axis, deliberately **independent of forgetting** ([F]): forgetting randomly
*erases words*, semanticization *compresses meaning*. A memory can be both dim and highly gistified, or bright and still
concrete.

The 4 gist-stage **texts** are already written — the semanticize job pregenerates them into `semantic_stages` at launch
(plan 20/22, [C7]), so the *content* exists from day one. What is missing is the **model of rising through the stages
over time**: a per-memory countdown (the gist-timer, [C6a]) measured in universe-days that, when it fills, advances the
abstraction stage by one; a **z-axis coordinate** for the risen gist body that copies the hippocampal star's emergent
x,y and lifts only z into the reserved neocortex band ([C6][V9][I5]); and an append to the memory's provenance history as
each stage rises ([R8a], CC5). Because the client renders the gist stars every frame and the server later needs the same
stage/coordinate for cost-gating ([R8][G4]) and reads, the timer and z-coord math live once in Go and once in TS, pinned
by golden fixtures — the same discipline as plan 18. This unit owns that model as **pure, IO-free domain functions**; the
orchestration that *fires* the rise on a clock advance is Epic E's use-case ([41.consolidation-usecase](41.consolidation-usecase.md)),
and the render is [42.gist-star-rendering](42.gist-star-rendering.md).

## Scope / Non-goals

**In scope**
- **`Semanticize` — the pure abstraction-stage progression.** Given the current `semantic_stage` and the number of
  gist-timer units that have elapsed, return the new stage, monotone non-decreasing and clamped to the 5-value ladder
  `0..4` (stage 0 = concrete, no gist risen; stages 1..4 = the four pregenerated gist texts). Deterministic, side-effect-free.
- **The gist-timer model ([C6a]).** A per-memory countdown to the next gist stage: `elapsedUniverseDays / gist_units_per_stage`
  = whole units crossed since `semanticize_timer_reset_at`. One filled unit → one stage. The unit length
  (`semantic.gist_units_per_stage`, e.g. 10 universe-days) is a `values.yaml` scalar. Includes the **arousal / connection
  modulation** ([F6][F7]): higher-arousal and more-connected memories advance *slower* — the same read-time modulation
  factor forgetting uses, applied to the effective elapsed time.
- **The timer-reset contract.** `semanticize_timer_reset_at` is set to universe-time-now on **recall / reconsolidation**
  (plan 33, [C6a][F5]) so the next gist stage is *delayed*; already-risen stages are **kept** (z-axis one-way, [C7]). This
  unit defines the reset semantics + the read that consumes the anchor; the *write* on recall is plan 33's.
- **The z-axis 2-store gist coordinate ([C5][C6][V9]).** A pure `GistCoordinate(hippocampalX, hippocampalY, stage) →
  (x, y, z)` that **copies the hippocampal star's emergent x,y** and returns a z inside the **reserved neocortex band**
  (`force_sim.neocortex_z_min`..`neocortex_z_max`, 15..25). The neocortex layer has **no force-sim** ([C6][I5]); the gist
  body is placed by copy, not simulated. Higher stages may sit deeper into the band (stage → z within bounds) so the
  ascent reads as progressive.
- **Provenance append on stage rise (CC5).** Each time a stage rises, an append-only `memory_provenance` row
  (`kind = semanticized`, `source = system`, the risen stage's text, `universe_time`) is written — the semanticization
  half of the variant history ([R8a]). This unit specifies *what* is appended and *when*; the table + append port are
  owned by [32.reconsolidation-rules](32.reconsolidation-rules.md) (CC5); the actual write executes inside Epic E's
  consolidation transaction ([41.consolidation-usecase](41.consolidation-usecase.md)).
- **The `semantic.*` scalars** these functions read (`gist_units_per_stage`, the max-stage ladder length as a derived
  count, the modulation reuse) enumerated for `values.yaml`. The **formulas** (the unit-crossing division, the
  modulation shape, the stage→z placement) stay in code.
- **Golden-parity (TS↔Go)** for the timer (`Semanticize` + elapsed-unit math) and the z-coordinate — identical inputs
  produce identical outputs within a fixed tolerance, pinned by shared fixtures, so the client render and the server
  read agree.

**Non-goals**
- **No gist-text generation.** The 4 gist-stage texts are LLM-pregenerated at launch into `semantic_stages` (plan 20/22,
  [C7]); this unit **reads** the text for the risen stage and never writes or regenerates it. Reconsolidation's
  "regenerate only the remaining stages" ([C7]) is [33.recall-usecase](33.recall-usecase.md).
- **No advance orchestration / no cron ([T4]).** *When* stages rise is driven by the `AdvanceProgression` hook on the
  universe clock ([29.universe-clock](29.universe-clock.md)/[30.time-advance-usecase](30.time-advance-usecase.md), Epic B).
  Firing the rise over an advance interval `[from,to]`, running replay-reorg ([C2]), and `Downscale` (SHY [C4]) are
  [41.consolidation-usecase](41.consolidation-usecase.md). This unit is a pure model with no clock and no hook.
- **No rendering.** The neocortical-layer gist stars, the z-axis 2-layer depth, and the "빨려 올라가듯" ascent
  choreography are [42.gist-star-rendering](42.gist-star-rendering.md) ([V8][V9][C8]).
- **No force-sim.** The neocortex band has no simulation ([C6][I5]); x,y are *copied* from the hippocampal layout owned
  by [19.force-sim-layout](19.force-sim-layout.md). This unit does not place hippocampal stars and does not re-run layout.
- **No `Downscale` (SHY).** Homeostatic sleep-time synaptic renormalization ([C4][I9]) is a *different mechanism* from
  this stage-rising model; it is the `Downscale` fn introduced by [41.consolidation-usecase](41.consolidation-usecase.md).
  Seam only.
- **No forgetting math.** Brightness decay, decay-stage word-loss, and the arousal/connection *forgetting* modulation are
  [37.forgetting-decay](37.forgetting-decay.md) (Epic D). This unit reuses the **same modulation signal** ([F6][F7]) for
  the gist-timer but does not own it; the shared coefficient home is noted below.
- **No cost pricing.** `semantic_stage` is the **gist-depth signal** the view-cost curve consumes ([R8][G4], CC3), but
  the Twinkle price of *viewing* a gist star lives in Epic F's [43.stardust-ledger](43.stardust-ledger.md). This unit
  exposes the depth, not a price.
- **No schema reshape.** `semantic_stage`, `semanticize_timer_reset_at`, and `semantic_stages` already exist on
  `episodic_memories` (plan 16); this unit adds **no** column. `memory_provenance` is added by plan 32 (CC5).

## Design

### Placement (domain-first Go)

Semanticization is **core-domain behavior of the memory context** — it advances a stage on the aggregate root, reads its
timer anchor, and computes a coordinate for its gist body — so it lives in `internal/memory` beside plan 18's plasticity
math, **not** a new context (ARCHITECTURE §2.2; be-architecture: keep tightly-coupled memory behavior in one context).
The functions are pure (types + `Semanticize` + the elapsed-unit math + `GistCoordinate`) and mirror into
`packages/memory-logic` (TS). This unit adds **no** table, **no** query, and **no** RPC: the columns it reads already
exist (plan 16) and are already returned by `GetUniverse` (`semantic_stage`, `semanticize_timer_reset_at`,
`semantic_stages` — recon confirms all three are selected). The provenance **append port** is consumer-owned by Epic E's
use-case (ARCHITECTURE §2.4), not pre-declared here.

### The aggregate surface (given, not defined here)

From `episodic_memories` (owned by [16.memory-aggregate-schema](16.memory-aggregate-schema.md)):

- `semantic_stage SMALLINT` in `0..4` — the current abstraction stage (stored fact; advanced by `Semanticize` on
  consolidation). `0` = concrete only, no gist risen; `1..4` = the four pregenerated gist stars have risen.
- `semanticize_timer_reset_at DATE` — the universe-time anchor the gist-timer counts elapsed universe-days from ([C6a]).
  Set to `created_universe_time` at launch, and re-set to `now` on recall/reconsolidation (plan 33).
- `semantic_stages JSONB` — the 4 pregenerated gist texts ([C7]); read (never written) here to pick the risen stage's
  text for provenance/render.
- `arousal REAL`, plus the memory's aggregate connection strength — the modulation inputs ([F6][F7]).

The pure functions never mutate the aggregate; they take scalars and return scalars, so the same functions serve a Go
domain object and a TS mirror without either owning the other's struct shape (matching plan 18).

### `Semanticize` — pure abstraction-stage progression ([C6a][C7])

Signature (canonical; representation casing may differ per language):

```
Semanticize(currentStage int, unitsElapsed int) → int
```

`unitsElapsed` is the number of *whole* gist-timer units that have crossed since the timer anchor (computed by the timer
model below). The new stage is:

```
newStage = min(currentStage + unitsElapsed, SEMANTIC_MAX_STAGE)      // SEMANTIC_MAX_STAGE = 4 (derived ladder length)
```

Consequences the acceptance criteria pin:

- **Monotone non-decreasing.** `Semanticize(s, u) ≥ s` for `u ≥ 0` — a stage never *lowers*. This is the z-axis one-way
  rule ([C7]): once a gist star has risen it is kept, even across a recall that resets the timer.
- **Clamped to the ladder.** `Semanticize(s, u) ≤ SEMANTIC_MAX_STAGE = 4`; there is no stage beyond the four
  pregenerated texts. Stage 0 with no elapsed units stays at 0 (concrete only).
- **Deterministic & total.** Depends only on its arguments and `SEMANTIC_MAX_STAGE` (a derived array length, not a tuned
  value — the count of pregenerated gist texts). No clock, no store, no randomness.

**Decided (2026-07):** a single advance can cross **multiple** stages when the clock jumps far (Epic B's launch/recall
can advance the universe clock by many days at once, [T2]). `Semanticize` therefore adds *all* whole units elapsed and
clamps — the consolidation use-case appends one provenance row **per stage crossed** (CC5), preserving a continuous
변천사 even across a large jump. It does not silently skip intermediate gist stars.

### The gist-timer ([C6a]) — elapsed units in universe-days, arousal/connection-modulated

The timer converts elapsed universe-time into whole stage-units. Given the universe-time "now" (from the clock, plan 29),
the memory's `semanticize_timer_reset_at`, and the modulation inputs:

```
rawElapsedDays  = max(0, now − semanticize_timer_reset_at)                       // universe-days [I10]
effectiveDays   = rawElapsedDays × timerModulation(arousal, connectionStrength)  // [F6][F7]; ≤ 1, slows the timer
unitsElapsed    = floor(effectiveDays / semantic.gist_units_per_stage)
```

- **`semantic.gist_units_per_stage`** (universe-days per gist stage, e.g. 10 — confirmed at implement) is a `values.yaml`
  scalar; the *division* is a formula (code).
- **Modulation ([F6][F7]).** `timerModulation` returns a factor in `(0, 1]` that **slows** the timer for high-arousal
  and strongly-connected memories — the same signal forgetting uses to slow brightness decay. **Decided (2026-07):** the
  modulation **coefficients** live once with the forgetting axis ([37.forgetting-decay](37.forgetting-decay.md),
  `forgetting.arousal_slow_coefficient` / `forgetting.connection_slow_coefficient` — enumerated there), and semanticization *reuses*
  them so a highly-arousing memory forgets *and* gistifies slower with one consistent knob ([C6a] "frequently-recalled
  memories gistify and forget slower"). This unit references those keys; it does not introduce a second modulation
  coefficient. The modulation *shape* is a formula (code).
- **Reset delays the next stage ([C6a][F5]).** Because `unitsElapsed` counts from `semanticize_timer_reset_at`, setting
  that anchor to `now` on recall/reconsolidation (plan 33) makes `rawElapsedDays = 0` → `unitsElapsed = 0` → no new stage
  *for now* — the next gist is *delayed*, not undone. Already-risen stages (`semantic_stage`) are untouched by the reset
  (z-axis one-way, [C7]).
- **Independence from forgetting ([F]).** The gist-timer reads `semanticize_timer_reset_at`; forgetting decay reads
  `last_recalled_universe_time` (+ the neighbor `forgetting_offset_days`, CC4). They are separate anchors on separate
  axes — a memory can be deeply forgotten yet barely gistified, or vice versa. This unit computes only the gist axis.

### `GistCoordinate` — copy x,y, raise z ([C5][C6][V9][I5])

Signature:

```
GistCoordinate(hippocampalX float, hippocampalY float, stage int) → (x, y, z)
```

The risen gist body's position is **not simulated** — the neocortex band has no force-sim ([C6][I5]). Instead:

- **x,y are copied verbatim** from the hippocampal star's *emergent* settled coordinates (force-sim output, plan 19). The
  gist star sits directly *above* its concrete star. Meaning still comes from the hippocampal emergence ([I5] upheld: x,y
  are not re-derived and the neocortex is not a second coordinate authority).
- **z is raised into the reserved neocortex band** `force_sim.neocortex_z_min`..`force_sim.neocortex_z_max` (15..25),
  disjoint from the hippocampus band (0..10) by the reserved gap ([C5][V9]). **Decided (2026-07):** `z` is a function of
  `stage` within the band (deeper stages sit higher in the band, e.g. a linear map of `stage/SEMANTIC_MAX_STAGE` across
  `[neocortex_z_min, neocortex_z_max]`) so the ascent reads as progressive depth; the exact map is a formula (code). All
  four gist stages fall strictly inside `[neocortex_z_min, neocortex_z_max]`.

The neocortex layer is therefore **stable** ([V9]): its bodies move only when their hippocampal x,y move (the hippocampal
layout re-settles) or when a stage rises. There is no independent neocortical simulation and no neocortex→hippocampus
coordinate feedback.

### Provenance append on stage rise (CC5, [R8a])

Each rising stage is an event in the memory's variant history. When `Semanticize` advances the stage from `s` to `s'`,
**one `memory_provenance` row per crossed stage** is appended: `kind = semanticized`, `source = system`, `text` = that
stage's pregenerated gist text (from `semantic_stages`), `universe_time` = the advance's universe-time. This is the
semanticization half of 변천사 ([R8a]); reconsolidation appends `reconsolidated/user` rows (plan 33), and the
`created/original` baseline is synthesized at read (CC5). The `memory_provenance` table + append port are owned by
[32.reconsolidation-rules](32.reconsolidation-rules.md); the actual append executes inside Epic E's consolidation
transaction ([41.consolidation-usecase](41.consolidation-usecase.md)). This unit specifies the row *contents* and the
*one-per-crossed-stage* rule; it performs no IO itself.

### Golden-parity (TS↔Go)

Per ARCHITECTURE §2.5, purity exists so the two runtimes agree. The Go implementation in `internal/memory` and the TS
mirror in `packages/memory-logic` are pinned equal by **shared golden fixtures**
(`apps/api/internal/memory/testdata/semanticization-golden.json`), `(function, inputs) → expected output` rows with a
fixed tolerance. Coverage: `Semanticize` at each stage `0..4` including the clamp at `4`, a multi-unit single-advance
crossing several stages, `unitsElapsed = 0` (stage unchanged); the timer's elapsed-unit math at the reset anchor
(`elapsed = 0` → 0 units), across a range of universe-days, and under low/high modulation; and `GistCoordinate` asserting
x,y are copied unchanged and z lands strictly inside `[neocortex_z_min, neocortex_z_max]` for every stage. Both consumers
read the same generated `semantic.*` / `force_sim.*` constants (`pnpm gen:values` emits Go + TS), so a tuning change
cannot desync the two sides.

**Decided (2026-07):** golden-parity covers the **timer + z-coordinate** math (both the client render and the server read
must agree on stage and placement), matching plan 18's precedent. The LLM-gated pieces — gist-text *content* (plan 20/22)
and reconsolidation's remaining-stage regen (plan 33) — are **not** golden-parity (they are non-deterministic LLM output),
consistent with the brief's parity stance.

### Re-implementation note (e.g. Flutter)

Semanticization is fully specified by three rules — *stage rises by whole gist-units elapsed, clamped to 4 and never
lowered*; *a unit is `gist_units_per_stage` modulated universe-days from the reset anchor*; and *the gist body copies the
hippocampal x,y and raises z into the neocortex band* — plus the reused forgetting modulation. Any client reproduces it
from those rules + the `semantic.*` / `force_sim.*` constants; there is no hidden state beyond the per-memory stage +
timer anchor.

## Acceptance Criteria

1. `Semanticize(currentStage, unitsElapsed)` is **monotone non-decreasing** (`≥ currentStage` for `unitsElapsed ≥ 0`) and
   **clamped** to `SEMANTIC_MAX_STAGE = 4` (a derived ladder length = the count of pregenerated gist texts, not a tuned
   value); it is deterministic and IO-free ([C7], z-axis one-way).
2. A single advance may cross **multiple** stages (`unitsElapsed > 1`): `Semanticize` adds all whole units and clamps, and
   the consolidation use-case appends **one provenance row per crossed stage** so 변천사 stays continuous across a large
   clock jump (CC5, [R8a]).
3. The gist-timer computes `unitsElapsed = floor(effectiveDays / semantic.gist_units_per_stage)` where `effectiveDays =
   max(0, now − semanticize_timer_reset_at) × timerModulation(arousal, connectionStrength)`, measured in **universe-days**
   ([C6a][I10]); at the reset anchor (`now = semanticize_timer_reset_at`) `unitsElapsed = 0` (no stage rises) ([F5]).
4. Higher **arousal** and stronger **connection** *slow* the gist-timer (`timerModulation ∈ (0,1]`), **reusing the
   forgetting modulation coefficients** ([37.forgetting-decay](37.forgetting-decay.md), [F6][F7]) — no second modulation
   coefficient is introduced — and the modulation uses **arousal only, never valence** ([I3]).
5. Recall/reconsolidation resetting `semanticize_timer_reset_at` to `now` **delays** the next stage (`unitsElapsed`
   recomputes to 0) but **never lowers** `semantic_stage` — already-risen gist stars are kept ([C6a][C7], z-axis one-way).
   (The reset *write* is [33.recall-usecase](33.recall-usecase.md); this unit defines the read semantics.)
6. The gist axis is **independent** of forgetting: the gist-timer reads `semanticize_timer_reset_at`, while forgetting
   decay reads `last_recalled_universe_time`; a memory may be deeply decayed yet minimally gistified and vice versa ([F], [C]).
7. `GistCoordinate(hippocampalX, hippocampalY, stage)` **copies x,y verbatim** from the hippocampal star and returns a
   `z` strictly inside the reserved neocortex band `[force_sim.neocortex_z_min, force_sim.neocortex_z_max]` (15..25),
   disjoint from the hippocampus band (0..10); the neocortex has **no force-sim** and no independent coordinate authority
   ([C5][C6][V9][I5]).
8. Semanticization **never deletes** any memory, neuron, or synapse and never mutates the `Diary` — it only rises a
