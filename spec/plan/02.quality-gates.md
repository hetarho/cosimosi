# 02. quality gates

> Quality gates - reproducible local and CI checks for clean roots, architecture boundaries, and vocabulary drift.
> Scope: Infra | Status: built

## Purpose

The clean platform is only useful if every later unit can trust the same guardrails. This unit turns "does it still fit
the architecture?" into runnable gates: formatting, linting, typechecking, tests, generated-code freshness, FSD
boundaries, Go vet/test/build, and a ubiquitous-language drift check. It catches architectural mistakes before feature
work accumulates on top of them.

## Scope / Non-goals

**In scope**
- Root scripts for fast local checks and the full merge gate: `lint`, `typecheck`, `test`, `check`, plus app-specific
  scripts where each app needs them.
- Web gates: TypeScript build, oxlint, Vitest, Steiger FSD checks, and ESLint boundary checks.
- Mobile gates: React Native ESLint and TypeScript checking without requiring an emulator.
- Blog gates: Astro build and TypeScript/type generation through the existing build path.
- Backend gates: `gofmt` check, `go vet`, `go test ./...`, and `go build ./...`; Docker is the fallback path when the
  host Go toolchain is not available.
- Generated-code freshness checks for proto, sqlc, and values generation: a full check runs `pnpm gen` and fails if a
  generator changes committed generated outputs.
- Ubiquitous-language lint that prevents domain/visual vocabulary drift across active source roots.
- CI workflow updates so pull requests and direct pushes run the same logical gates against the clean roots only.

**Non-goals**
- No product feature tests, golden-parity tests, Playwright journeys, or visual regression. Those arrive with domain
  and presentation units.
- No deployment gate and no production secrets.
- No attempt to make reference app roots pass anything. They stay outside every active gate.
- No domain vocabulary expansion except the lint allowlist/rules needed to enforce the existing glossary.

## Design

### Gate shape

The repo has two levels of command:

- **Fast local gates**: commands a developer can run while building a slice, scoped by app/package when possible.
- **Full repo gate**: one root command, `pnpm check`, that runs every non-deployment validation required before a
  commit is considered healthy.

The full gate is composed from:

1. `pnpm lint`: web oxlint + FSD checks, mobile ESLint, api gofmt cleanliness, and ubiquitous-language lint.
2. `pnpm typecheck`: web TypeScript, mobile TypeScript, and blog Astro build/type generation.
3. `pnpm test`: web Vitest and api `go test ./...`.
4. `pnpm build:web`, `pnpm build:blog`, and `pnpm build:api`.
5. `pnpm vet:api`.
6. `pnpm check:gen`: run `pnpm gen` and fail if generated output paths are dirty.

The commands deliberately skip reference app roots and archived spec directories.

### Frontend and workspace checks

`apps/web` keeps oxlint as the fast lint. Steiger and ESLint boundaries enforce the FSD layer direction from the empty
shell onward:

- one-way layer imports (`app -> pages -> widgets -> features -> entities -> shared`);
- structural FSD rules from `@feature-sliced/steiger-plugin`;
- a probe command, `pnpm --filter @cosimosi/web lint:boundaries:probe`, that proves a forbidden `entities -> pages`
  import fails through `boundaries/dependencies`.

Vitest is the unit-test runner for pure TS and React logic. The gate does not invent product tests for empty slices;
`vitest run --passWithNoTests` proves the runner is wired until real tests land.

`apps/mobile` has a host-only gate: ESLint and TypeScript pass without booting iOS or Android. Native simulator
verification is a later implementation concern for mobile-facing slices.

`apps/blog` keeps the explicit Astro gate: `astro build` performs the static build and type generation used by CI.

### Backend checks

The Go gate is intentionally plain and lives behind `scripts/go-check.mjs`:

- `gofmt` must be clean;
- `go vet ./...` must pass;
- `go test ./...` must pass;
- `go build ./...` must pass.

When host Go is missing or blocked, the helper runs the same commands in `golang:1.26` with `apps/api` mounted at
`/app`. The fallback is not a weaker gate; it is the same command surface in a controlled toolchain.

### Generated-code freshness

`scripts/check-generated.mjs` runs `pnpm gen`, then checks the generated output pathspecs with `git status --porcelain`.
The current platform has no proto, sqlc schema, or `spec/values.yaml` source yet, so generation skips cleanly today and
the script lights up automatically as later units add those sources.

### Ubiquitous-language lint

The lint reads [ubiquitous-language.md](../ubiquitous-language.md) as the vocabulary source and scans active source
roots only. It catches:

- a rendering word (`star`, `cell-star`, `filament`, `constellation`) appearing outside visual/UI paths;
- a direct memory-to-memory edge type/table/message that would model an emergent relationship;
- known forbidden synonyms for canonical domain terms.

The probe modes `--probe=visual` and `--probe=edge` deliberately fail those two core drift classes without writing
fixture files into the tree. False positives are resolved by narrowing path allowlists, not by weakening the vocabulary
rule.

### CI

CI mirrors `pnpm check` as split jobs for speed: web, mobile, blog, api, codegen, and language lint. No job touches
reference app roots. No job needs production secrets.

## Acceptance Criteria

1. The root package exposes documented `lint`, `typecheck`, `test`, and `check` scripts; `pnpm check` runs every active
   non-deployment gate.
2. CI runs the same logical gates on pull requests and direct pushes to the active branches, with no references to
   reference app roots.
3. Web gates include TypeScript, oxlint, Vitest, Steiger, and ESLint boundary checks; a deliberate forbidden layer
   import fails.
4. Mobile gates include TypeScript and ESLint and run on a host machine without launching an emulator.
5. Backend gates include `gofmt` cleanliness, `go vet ./...`, `go test ./...`, and `go build ./...`; the Docker
   fallback runs the same checks.
6. Generated-code freshness is enforced: after `pnpm gen`, changed generated outputs fail the full gate.
7. Ubiquitous-language lint fails for domain/visual vocabulary drift and for a modeled memory-to-memory edge.
8. The checks do not import, install, lint, typecheck, test, or build reference app roots.
