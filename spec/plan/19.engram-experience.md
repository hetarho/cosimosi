# 19. 체험으로 보는 뇌과학 (engram-experience)

> 이미 구현된 6개 엔그램 이론을 체험(demo) 우주 안에서 눈으로 보고·직접 건드릴 수 있게 하는 "이론 렌즈" 오버레이를 만들고, 랜딩 카드를 라이브 체험으로 잇고, **랜딩 페이지의 설명·인터랙션을 (구현된·계획된) 뇌과학 이론에 맞게 정합**한다. | 범위: FE | 선행: — | 차단: —

## 목적 (Why)
MVP(01–13)에서 6개 엔그램 이론을 이미 구현해 두었지만, 사용자는 그것이 "신경과학을 옮긴 것"이라는 사실을 알아채기 어렵다. 우주는 그냥 예쁜 3D로만 보인다. 본 스펙은 **새 이론 로직을 추가하지 않고**, 이미 동작하는 이론들을 체험 우주 안에서 **관찰 가능(observable)** 하고 **직접 트리거(triggerable)** 하게 만든다. 핵심은 재사용 가능한 **"이론 렌즈/가이드" 오버레이**(HUD/legend + "관찰 포인트")로, 데모 모드에서 켜진다. 이 오버레이는 **specs 20–27이 각자의 새 이론 패널을 한 줄로 끼워 넣는 확장 프레임워크**이기도 하다 — 렌즈 레지스트리에 항목 하나를 더하면 새 이론이 가이드에 나타난다. 또한 오늘 "체험해보기"는 그냥 `/universe`로 떨어뜨리기만 하므로(HeroSection/CtaFooterSection), 이를 **안내가 있는 체험 진입**으로 격상하고, 이미 만들어진 랜딩 카드(EngramCard 등)에 "이 카드 체험하기" 링크를 달아 라이브 데모로 잇는다.

수면 위로 끌어올릴 6개 **완료된** 이론:
1. **엔그램=별** — 별 클릭 → 불변 원본 일기 + "이 별 하나가 기억 흔적(엔그램) 하나".
2. **의미+시간 시냅스 생성** — 새 일기 → 별 등장 + 가까운 기억과 연결("의미 유사도 τ=0.75·top-8 + 같은 날 +0.3").
3. **헵 공동회상 강화** — 두 별을 ≥2초씩 오가며 회상 → 사이 연결이 굵어짐("+0.05, 상한 1.0").
4. **능동 인출 게이팅** — 스침(<2초)은 강화 안 됨, ≥2초 능동 열람만.
5. **시간 감쇠 + 최소 밝기** — 오래된 별은 어두워지되 5% 바닥("반감기 30일").
6. **침묵 엔그램/잠든 별** — 어두운 별이 잔존 + `/dormant` 탐색 + 재점화.

또한 본 스펙은 **랜딩 페이지 정합 패스**를 포함한다. 랜딩은 cosimosi의 *의도된 최종 비전 showcase*이므로(정책: [policy/ux/landing.md](../policy/ux/landing.md)), 거기 적힌 이론 설명·인터랙션이 (a) **이미 구현된 이론**은 실제 동작과 일치하고, (b) **계획된 이론(20–27)**은 의도한 비전을 정확히 표현하며 각 카드가 *대응 이론·상태(구현됨/계획됨)*를 드러내고 라이브 체험으로 이어지도록 정리한다. (계획된 이론의 **실제 메커니즘**을 카드에 반영하는 갱신은 각 스펙의 T-landing 소관 — 19는 *지금 참인 것의 정합* + *비전 표현의 정확성·체험 연결*까지.)

## 범위
- **포함:**
  - FE `widgets/theory-lens`(신규): 데모에서 켜지는 **이론 렌즈 오버레이**(2D HUD/legend, R3F 밖 — Architecture §3.1). (a) 활성 이론 목록(legend)·토글·툴팁, (b) 우주에 앵커된 **"관찰 포인트"** 콜아웃, (c) **렌즈 레지스트리**(`TheoryLens[]`) — 이론 1개 = 레지스트리 항목 1개(확장점, 20–27이 소비).
  - FE `pages/experience`(신규, 선택): 랜딩 카드의 "이 카드 체험하기"가 도착하는 **안내 있는 체험 진입** — 데모 플래그를 켜고 특정 이론 렌즈에 포커스를 맞춘 채 `/universe`로 들어간다.
  - FE `shared/lib/demo`(수정): 관찰 훅 노출 — 데모 우주의 어느 별/엣지가 어느 이론을 예시하는지 가리키는 셀렉터(예: 가장 밝은 별, 가장 굵은 엣지, 대표 잠든 별, 같은 날 페어). 새 데이터·새 로직 아님, 기존 `DEMO_ENTRIES`/`DEMO_EDGES`에서 파생만.
  - FE `pages/landing/ui/section/*`(수정): EngramCard / HebbianCard / TimeWindowCard / SilentEngramCard / ConceptSection에 **"이 카드 체험하기"** 액션 추가 → 데모 진입(해당 이론 렌즈 포커스).
  - FE `widgets/universe-canvas/ui/UniverseCanvas.tsx`(수정): 데모 모드에서만 렌즈 오버레이(HUD)를 마운트. (오버레이는 캔버스 밖 DOM이라 R3F 트리에는 `<Html>`을 넣지 않는다 — 헌법4.)
  - FE `app/router.tsx`(수정): `/experience` 라우트를 추가하는 경우 등록(추가 시).
  - FE `pages/landing` **정합 패스**: (a) 이미 구현된 6개 이론의 카드 설명·인터랙션이 실제 동작과 일치하는지 감사·수정(예: 수치·문구·상호작용), (b) 계획된 이론(20–27)에 대응하는 카드가 의도한 비전을 정확히 표현하는지 확인, (c) **각 카드에 대응 이론 + 상태(✅구현됨/🚧계획됨 + plan NN) 표식**, (d) `LandingPage`의 JourneyAct 챕터 순서가 19–27 이론 흐름과 정합하는지 점검. 정전 기준은 [policy/ux/landing.md](../policy/ux/landing.md).
- **비목표(Out of scope):**
  - **어떤 새 이론 로직도 금지** — 조각화(21)·경쟁 할당(22)·재공고화(23)·변천사(24)·요즘 상태(25)·관련성 감쇠(26)·야간 공고화(27)는 각 스펙 소관. 본 스펙은 **이미 구현된 6개만** 수면 위로 올린다.

  - **AI(ai.Extractor) 와이어링 금지** — spec 20.
  - 실제 회상 강화 영속화(`ReinforceLinks`)는 11 소관이고 데모에선 no-op이다(`features/recall/api/recall.ts`). 본 스펙은 데모에서 **굵어짐을 클라 로컬로 보여주기만** 하고 서버로 쓰지 않는다(아래 설계 요점 참조).

## 설계 요점
**이론 렌즈 레지스트리(확장점, 헌법4 순수 — three/DOM 미의존).** 렌즈 1개 = 이론 1개. 데이터는 `widgets/theory-lens/model`에 순수 타입으로 두고(모바일 재사용), 우주의 어느 요소가 그 이론을 예시하는지는 `shared/lib/demo`가 노출하는 관찰 셀렉터의 id만 참조한다(좌표·three 의존 없음). **20–27은 이 배열에 항목 하나만 추가**하면 가이드에 등장한다.
```ts
// widgets/theory-lens/model/registry.ts (순수 — three/React/DOM import 금지, 헌법4)
export interface TheoryLens {
  id: string                      // 'engram' | 'synapse' | 'hebbian' | 'active-recall' | 'decay' | 'dormant' | (20–27이 추가)
  title: string                   // "엔그램 = 별"
  oneLine: string                 // 한 줄 설명(우주 안 표현)
  /** 사용자가 직접 해 볼 행동(관찰/트리거). */
  observe: string                 // "별을 클릭해 2초 머무르면 불변 원본 일기가 열려요"
  /** 데모 우주에서 이 이론을 가장 잘 보여주는 앵커(별/엣지 id). demo 셀렉터가 채운다. */
  anchor: () => { kind: 'star' | 'edge' | 'pair' | 'background'; ids: string[] }
  doi?: string                    // 근거 논문 DOI(있으면 툴팁/배지에 표기 — concept.md §과학적 근거)
  status: 'done' | 'planned'      // 19는 'done' 6개만; 20–27은 'planned'→'done'으로 자기 항목 갱신
}
export const THEORY_LENSES: TheoryLens[] = [/* 6개 done — 아래 6개 이론 */]
```

**데모 관찰 훅(`shared/lib/demo` 수정 — 순수, 기존 데이터에서 파생만).** 새 데이터·새 로직 없이 `DEMO_ENTRIES`/`DEMO_EDGES`에서 대표 요소만 골라 노출한다.
```ts
// shared/lib/demo/observe.ts (신규, 순수) — 기존 base 데이터에서 파생, 새 별/엣지 추가 없음
export function brightestStarId(): string        // 가장 최근(daysAgo 최소) = 가장 밝은 별 (엔그램=별 앵커)
export function thickestEdge(): { aId: string; bId: string }   // weight 최대 엣지 (헵 강화 앵커)
export function sameDayPair(): { aId: string; bId: string } | null  // 같은 날(daysAgo 근접) 페어 (시냅스 시간창 앵커)
export function dormantStarId(): string          // 가장 오래된(daysAgo 최대) 잠든 별 (침묵 엔그램 앵커)
```

**"체험하기" 진입(랜딩 카드 → 라이브 데모).** 각 카드에 `enterDemoMode()`→`navigate({to:'/universe', search:{lens:'<id>'}})`(또는 `/experience`) 액션을 단다. HeroSection의 `tryDemo()` 패턴과 동일하되 **어떤 이론 렌즈에 포커스할지**를 함께 넘긴다. 우주에 도착하면 렌즈 오버레이가 그 이론의 관찰 포인트를 강조한다(예: 헵 → `thickestEdge()` 엣지에 콜아웃 + "두 별을 ≥2초씩 오가 보세요").

**헵 강화의 "굵어짐"을 데모에서 보이기(서버 쓰기 없이).** 데모에선 `reinforceLinks`가 no-op이라(`features/recall/api/recall.ts`) 영속화는 없다. 본 스펙은 **클라 로컬**로만 보여준다: `features/recall/model`의 co-recall 누적(`onActiveView`/`drainDeltas`)을 구독해, 페어 delta가 쌓이면 `useSynapseStore`의 해당 엣지 `weight`를 **로컬 미리보기**로 증가시켜(상한 1.0) `UniverseSynapses`(09 머티리얼)가 즉시 굵어 보이게 한다. 이는 데모 세션 한정 시각 피드백이며(새로고침 시 소멸), **서버/proto는 건드리지 않는다**(헌법1·비목표). 일반(비데모) 모드에선 11의 영속 경로를 그대로 둔다.

**오버레이 마운트.** `UniverseCanvas`(또는 그 페이지 HUD 레이어)에서 `isDemoMode()`일 때만 `<TheoryLensOverlay/>`(2D DOM)를 렌더. 관찰 포인트의 화면 좌표는 별 force-sim 좌표를 `useThree` 카메라로 투영하지 않고(캔버스 밖 DOM이므로), **캔버스 모서리에 고정된 legend + "이 별로 데려가기"(focusStar 재사용)** 버튼으로 앵커한다 — 잠든 별 fly-to(`useCameraMode.focusStar`)와 동일 메커니즘. (3D 안 라벨 금지 — 헌법4.) 기존 HUD(MemoryForm 좌상단·MemoryPanel 우하단·NavPad·AppearanceSwitcher)와 z-index/위치가 겹치지 않게 좌하단 또는 접이식 패널로 둔다. SessionGate의 "체험 모드" 핀(우상단)과도 충돌하지 않게 한다.

**렌즈↔포커스 진입 파서.** 카드/Experience 진입은 `?lens=<id>`를 넘기고, 우주 셸은 마운트 시 그 값을 읽어 오버레이의 활성 렌즈로 설정한다(잘못된 id면 무시하고 legend 전체 표시). 데모 진입과 렌즈 포커스는 단일 헬퍼로 묶는다:
```ts
// 카드 onClick — HeroSection.tryDemo() 패턴 + 렌즈 포커스
function enterDemoWithLens(navigate: NavigateFn, lens: TheoryLens['id']) {
  enterDemoMode()
  void navigate({ to: '/universe', search: { lens } }) // 또는 '/experience'
}
```

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 사용자가 데모 우주에 진입하면 THE SYSTEM SHALL 이론 렌즈 오버레이(legend)를 띄워 **6개 완료 이론**(엔그램=별·시냅스 생성·헵 강화·능동 인출·시간 감쇠·잠든 별)을 각각 한 줄 설명 + 관찰 행동과 함께 나열한다. _(헌법4: 오버레이는 2D HUD)_
1.2 WHEN 사용자가 데모 우주에서 별을 클릭하고 ≥2초 머무르면 THE SYSTEM SHALL 그 별의 **불변 원본 일기**(`demoRecall` → 읽기 전용 패널)를 표시하고, 렌즈는 "이 별 하나가 기억 흔적(엔그램) 하나"임을 설명한다. _(헌법1)_
2.1 GIVEN 헵 렌즈가 가리키는 두 별(`thickestEdge`) WHEN 사용자가 두 별을 ≥2초씩 오가며 회상하면 THE SYSTEM SHALL 사이 연결을 데모 로컬로 굵게(weight↑, 상한 1.0) 다시 그려 굵어짐을 즉시 보여준다(서버 쓰기 없음). _(11 재사용, 데모 한정)_
2.2 WHEN 사용자가 별을 ﹤2초만 스치면 THE SYSTEM SHALL co-recall로 카운트하지 않으며(능동 인출 게이팅), 렌즈는 "2초 이상 머물러야 강화돼요"를 안내한다. _(능동 인출)_
3.1 WHEN 시간 감쇠 렌즈가 켜지면 THE SYSTEM SHALL 최근 별(`brightestStarId`)과 오래된 잠든 별(`dormantStarId`)의 밝기 차이를 가리키고, "반감기 30일·바닥 5%(별은 사라지지 않음)"를 설명한다. _(헌법2)_
3.2 WHEN 잠든 별 렌즈에서 "이 별로 데려가기"를 누르면 THE SYSTEM SHALL `useCameraMode.focusStar(dormantStarId())`로 카메라를 그 별로 fly-to하고 회상으로 재점화할 수 있게 한다(`/dormant`와 동일 메커니즘). _(헌법2, 잠든 별 탐색)_
4.1 WHEN 사용자가 랜딩에서 EngramCard/HebbianCard/TimeWindowCard/SilentEngramCard/ConceptSection의 "이 카드 체험하기"를 누르면 THE SYSTEM SHALL `enterDemoMode()` 후 그 카드의 이론 렌즈에 포커스한 채 데모 우주로 이동한다. _(체험 연결)_
5.1 WHEN 새 이론을 추가하려면 THE SYSTEM SHALL `THEORY_LENSES` 배열에 항목 1개를 더하는 것만으로 가이드에 그 이론이 나타나도록 한다(레지스트리 확장성 — 20–27 소비). _(확장점)_

7.3 WHEN `LandingPage`의 JourneyAct 챕터 순서를 점검하면 THE SYSTEM SHALL 19–27 이론 흐름(엔그램→조각→연결→강화→재공고화→망각→요즘 상태→야간)과 정합한다.

## 완료 정의 (DoD)
- 데모 진입(랜딩 "체험해보기" 또는 카드 "이 카드 체험하기") 후 **이론 렌즈 오버레이**가 6개 완료 이론을 legend로 표시한다(비데모 모드에는 나타나지 않음).
- 6개 이론 **수동 워크스루**가 모두 통과한다:
  1. 별 클릭→2초 머무름→불변 원본 일기 패널(읽기 전용). 렌즈: "별 하나 = 엔그램 하나".
  2. 시냅스 렌즈가 가까운 기억 연결(같은 날 페어/굵은 엣지)을 가리키고 τ=0.75·top-8·같은날+0.3을 설명.
  3. 헵 렌즈가 가리키는 두 별을 ≥2초씩 오가면 사이 엣지가 데모 로컬로 **눈에 띄게 굵어진다**(상한 1.0).
  4. 별을 <2초만 스치면 굵어지지 않는다(능동 인출 게이팅 — 렌즈 안내 일치).
  5. 시간 감쇠 렌즈가 최근 별 vs 잠든 별의 밝기 차를 보여주고 바닥 5%(소멸 없음)를 설명.
  6. 잠든 별 렌즈 "이 별로 데려가기"→fly-to→회상 재점화(`focusStar` 재사용, `/dormant`와 동일).
- **레지스트리 확장성**: `THEORY_LENSES`에 더미 항목 1개를 임시로 추가하면 가이드에 1개가 더 나타남을 확인(20–27의 끼움 방식 검증) — 확인 후 제거.
- **백엔드/proto 무변경**: `git diff --stat`에 `proto/`·`backend/` 변경 없음. grep 무결과:
  ```bash
  # Windows: Docker/WSL 또는 PowerShell — 새 RPC/proto/쿼리가 없어야 함(본 스펙은 FE 전용)
  git diff --name-only | grep -E '^(proto/|backend/)' || echo "no backend/proto changes (expected)"
  ```
- `widgets/theory-lens/model/registry.ts`·`shared/lib/demo/observe.ts`가 `three`/React/DOM import 없이 컴파일된다(헌법4, grep 확인).
- `pnpm --filter frontend build`·`pnpm lint`(eslint-boundaries/steiger FSD 경계 포함) 통과. 데모 한정 weight 미리보기는 비데모(11) 영속 경로를 변경하지 않는다(코드 확인).
