# 19. engram-experience - 기억 실험실(데모 가상 시계·시뮬) + 랜딩 재정비 (as-built)

> 현재 구현 기준 문서. 체험 우주(demo 데이터 우주)에 가상 시계와 기억 실험실 컨트롤을 넣어 이미 구현된 엔그램 이론을 사용자가 직접 트리거해 눈앞에서 보게 하고, 랜딩이 그 이론을 정확히 표현하도록 정합한다. | 범위: FE | 상태: 구현 완료, 반영: 21·22·23·24·25·26·27·37(데모 프레임 위 확장)·39(회상 flush 머신)

## 목적

MVP(01–13)가 엔그램 이론 여러 개를 이미 구현했지만, 시간 감쇠는 며칠을 기다려야 보이고 잠든 별은 오래 지나야 생긴다. 새 이론 로직을 더하지 않고, 이미 동작하는 이론을 데모 우주 안에서 시뮬레이션 가능(simulatable)하게 만드는 것이 목적이다.

핵심 메커니즘은 데모 가상 시계다. 밝기·잠듦 계산은 전부 `now`를 인자로 받는 순수 함수(`entities/memory/model/activation.ts`)이므로, 데모에서 `now`를 `Date.now() + offset`으로 바꾸고 "하루/한 달 지나기"로 offset을 키우면 별이 어두워지고, 임계를 넘은 별이 잠들고, 회상하면 다시 밝아진다. 시간이 실제 흐른 것과 동일한 코드 경로다(시뮬레이션 ≠ 연출).

이 가상 시계와 데모 데이터 모듈은 20–27의 확장 프레임이기도 하다. 조각화(21)·경쟁 할당(22)·재공고화(23)·변천사(24)·요즘 상태(25)·관련성 감쇠(26)·야간 공고화(27)·겹쳐보기(37)는 모두 같은 데모 데이터 모듈(`shared/lib/demo/data.ts`)에 자기 근사를 얹고, 같은 시간 보내기 버튼으로 시연된다. 본 스펙은 그 프레임(가상 시계·재점화·연결 생성·헵 로컬·기억 실험실·이론 레지스트리)을 소유한다.

## 현재 구현

### 가상 시계 (`shared/lib/demo/clock.ts`)

- 시간 시뮬레이션의 단일 주입 지점. 모듈 변수 `offsetMs` 하나가 전부다(새로고침 시 소멸 — 데모 수명 규칙).
- `virtualNowMs()`는 `Date.now() + (isDemoMode() ? offsetMs : 0)`. 비데모면 정확히 `Date.now()` — 일반 모드 무영향.
- `skipDemoDays(days)`는 `offsetMs += days·DAY_MS`(`DAY_MS=86_400_000`).
- `demoOffsetDays()`는 패널 표시용 "+N일째"(반올림 일수).
- `resetDemoClock()`은 offset을 0으로 — `resetDemo()`가 호출한다.
- three/React/DOM 미의존(헌법4).

렌더·파생 계층은 `Date.now()`가 아니라 이 함수를 읽는다. 실제 호출부: `entities/star/ui/StarField.tsx`(rebuild·spotlight 2곳), `entities/memory/api/map-star.ts`, `entities/memory/api/universe-query.ts`(`applyUniverse`·`refreshActivation`), `entities/memory/api/dormant-query.ts`, `entities/synapse/model/store.ts`, `features/recall/ui/MemoryPanel.tsx`, `features/recall/model/recall-flush.machine.ts`, `features/dormant-search/{ui/DormantSheet,api/list-dormant}.ts`, `widgets/universe-canvas/ui/{AmbientNebula,UniverseCanvas,overlay/OverlayUniverse}.tsx`, `pages/visit/ui/VisitPage.tsx`(비데모라 `=Date.now()`), `pages/home/ui/HomePage.tsx`(겹쳐보기 재시뮬 deps). 시계 주입이 한 군데로 모인 덕에 이후 스펙은 별도 시간 인자 없이 시간 의존 이론을 데모에 노출한다.

### 데모 데이터 모듈 (`shared/lib/demo/`)

데모 우주는 더 이상 손으로 박은 별·엣지 목록(옛 `DEMO_ENTRIES`/`DEMO_EDGES`)이 아니라 페르소나 일기 코퍼스를 시뮬레이션해서 만든다.

- `personas.ts` — 세 페르소나 코퍼스 `CORPORA`(`student`/`worker`/`homemaker`). 각 코퍼스는 일기(조각 N개)·회상 이력의 순수 데이터. `demoPersonaList()`가 `{id, label, tagline}`을 낸다(예: '20대 대학생' · '사소한 것에 크게 흔들리는 밤').
- `simulate.ts` — 코퍼스를 별·시냅스 그래프로 빚는 순수 시뮬레이터. 실서버 worker의 연결 규칙을 조각 단위로 근사한다:
  - 일내 결속 `intra_entry` 고정 weight `INTRA_WEIGHT=0.8`(같은 일기 조각 모든 쌍).
  - 의미 링크: 시간순으로 각 조각이 이전 조각 중 topic-cosine 유사도 top-`KNN_K=5`에 잇는다(자격 바닥 `SIM_TAU=0.4`). weight `= 0.42 + sim·0.4 + temporal_bonus`, 상한 `SEM_CAP=0.79`(< intra). 사람·개체 주제를 공유하면 `entity`, 아니면 `semantic`.
  - temporal 보너스: 같은 주(`TEMPORAL_DAYS=7`) 안에서 최대 `TEMPORAL_MAX=0.3`.
  - 회상 세션: 그 일기 조각을 재점화(밝아짐)하고 함께 읽은 다른 주제 일기끼리 `co_recall` 다리(`CO_RECALL_BASE=0.5`, 기존 선엔 `CO_RECALL_BUMP=0.1`).
  - 연결성 보장: 다른 일기와 잇는 선이 없는 일기는 시간순 이웃에 약한 다리(외톨이 방지 안전망).
  - `crossResonances(a, b, max=4)`: 두 페르소나 코퍼스 사이 공명 쌍(겹쳐보기 37 데모 — 1:1 매칭, 결정론적).
- `data.ts` — proto 어댑터 + 데모 변이의 집. 코퍼스를 `simulate`로 빚어 proto `Star`/`Record`/`Synapse`로 파생하고(`GetUniverse`·`RecallMemory`·`ListDormant`·`ListRecords`와 같은 id 공간 공유), 모듈 변수에 런타임 상태를 둔다. 본 스펙이 소유하는 함수:
  - `demoStars()` / `demoSynapses()` — base + 체험 중 추가분.
  - `demoMarkRecalled(id)` — 그 별의 `lastRecalledAt`을 가상 now로 전진(재점화). `renewStar`로 별을 **불변 교체**해 protobuf structural sharing이 변경을 감지하게 한다(제자리 변이는 refetch가 no-op이 됨).
  - `demoAddRecord(input)` — 일기를 조각 별 fan-out으로 추가하고 데모 연결을 생성한다(아래 §데모 연결 생성).
  - `demoAddStar(mood, date)` / `demoAddMultiSceneStar(date)` — 기억 실험실의 "우주 키우기" 입력(미리 쓴 일기 `QUICK_ENTRIES`/`MULTI_SCENE_ENTRIES`에서 본문 선택).
  - `demoToday()` — 별 띄우기 날짜 기본값(가상 시계 기준 YYYY-MM-DD).
  - `resetDemo()` — base·추가분·재성형 상태를 비우고 `resetDemoClock()`까지 호출.
  - 이후 스펙이 같은 모듈에 얹은 함수(본 스펙 소유 아님, 데모 프레임 위 확장): `demoReshape`(23)·`demoEvolution`(24)·`demoConsolidate`(27)·`demoOverlayData`/`buildPersonaUniverse`(37)·`demoListRecords`/`demoFragmentText`(28)·valence 근사(25).
- `flag.ts` — `isDemoMode`/`enterDemoMode`/`exitDemoMode`·페르소나 게터/세터·`setDemoModeListener`(app이 캐시·스토어 리셋 주입). sessionStorage 보관.
- `observe.ts` — 관찰 셀렉터 `brightestStarId`/`thickestEdge`/`sameDayPair`/`dormantStarId`(코퍼스 그래프에서 id만 파생, 순수). index에서 export되나 현재 소비처는 없다(패널이 fly-to 앵커 방식에서 이론 캐러셀로 바뀌며 호출이 사라짐 — 죽은 export).

### 재점화 — 데모에서 완결 (`features/recall`)

실서버는 `RecallMemory`가 `last_recalled_at=now`를 쓰고 refetch 병합으로 별이 밝아진다. 데모는 그 대칭으로:

- `api/recall.ts`의 `recallMemory(memoryId)`는 데모 분기에서 `demoRecall(id)`(원본 일기) 후 `demoMarkRecalled(id)`(가상 now로 전진)와 `demoReshape(id)`(23 PE 게이트 재성형)를 호출한다. 쓰기 게이트(겹쳐보기 37의 `isWriteBlocked()`)가 켜져 있으면 undefined.
- `ui/MemoryPanel.tsx`는 ≥2초 active dwell 후 `recallMemory`를 발화하고, 데모 회상 성공 시 `universeInvalidateKey()`로 universe 쿼리를 무효화한다(전진한 타임스탬프가 화면에 반영). 비데모는 staleTime이 소유.
- 잠재우기 → fly-to → 회상 → 재점화 루프가 데모에서 완결된다.

병합 단조성: `entities/memory/model/merge.ts`의 `mergeStars`는 `max(lastRecalledAt)`. 그래서 시간 시뮬레이션은 데이터를 과거로 미는 게 아니라 시계를 미래로 보낸다 — 전진한 타임스탬프는 병합을 그대로 통과한다.

### 헵 강화 — 데모 로컬 (`entities/synapse`, `features/recall`)

- `entities/synapse/model/store.ts`의 `bumpEdgeWeight(aId, bId, delta)`: 정규화 페어(a<b)로 엣지를 찾아 `weight = min(1, w+delta)`·`brightness = 1`·`coActivationCount++`·`lastActivatedAt = virtualNowMs()`로 갱신한다. 페어가 없으면 데모 한정 `co_recall` 엣지를 로컬 생성(`weight = min(1, delta)`·`coActivationCount = 1`). 서버 쓰기 없음.
- co-recall 누적·증분은 `features/recall/model/co-recall.ts`(`CO_RECALL_DELTA=0.05`·`DWELL_MS=2000`·간격 효과 `spacingBoost`)가 순수하게 한다.
- 데모 즉시 굵어짐 훅 지점은 **`recall-flush.machine.ts`의 `accumulate` 액션**이다(옛 zustand `recordActiveView`가 아니라 XState 머신). `RECORD_VIEW`마다 `onActiveView(session, id, virtualNowMs())`로 누적하고, 직전 active view와 다른 별이며 데모면 `useSynapseStore.getState().bumpEdgeWeight(prev, id, CO_RECALL_DELTA)`를 직접 부른다. 비데모 영속 경로(`reinforceLinks` flush)와 11의 강화 의미론은 불변. `<2초` 스침은 dwell 타이머가 끝나기 전에 취소돼 `RECORD_VIEW`가 안 가므로 어떤 연결도 굵어지지 않는다(능동 인출 게이팅).

### 데모 연결 생성 (`shared/lib/demo/data.ts`의 `demoAddRecord`)

새 일기가 별만 뜨고 끝나지 않게, 데모도 연결을 만든다. 빈 줄 문단을 사건 경계로 보고(`splitScenes`, 상한 `DEMO_MAX_FRAGMENTS=3`) 조각 별로 fan-out한 뒤(21 근사), 첫 조각을 다음 규칙으로 잇는다:

- 일내 결속: 모든 조각 쌍을 `intra_entry` weight 0.8로.
- 같은 날: `entryDate`가 같은 기존 일기 최신순 `ADD_SAME_DAY_LINKS=2`개와 `temporal` weight 0.55.
- 같은 mood: 첫 조각과 같은 감정의 최신 일기 `ADD_SAME_MOOD_LINKS=1`개와 `semantic` weight 0.6(같은 날로 이미 이어진 별 제외).
- 흥분성 편향(22 데모 근사): 최근 `HOT_WINDOW_MS=6h` 안에 회상해 "뜨거운" 별이 있으면 그 별과도 `semantic` weight 0.66로 잇는다 — 라이브 force-sim이 새 별을 hot 성단 곁으로 끌어간다.

추가 엣지는 `addedEdges`(가상 now `lastActivatedAt`)에 쌓이고 `resetDemo`가 비운다. 패널 문구가 "체험은 근사 — 실제 우주는 AI 의미 임베딩(τ=0.75·최대 8개·같은 날 +0.3)으로 연결"임을 밝힌다.

### 기억 실험실 (`widgets/demo-sim`)

`DemoSimPanel`(캔버스 밖 2D HUD, 체험 우주에서만 마운트)은 두 모달로 나뉜다.

- 진입 칩 두 개(좌하단): "기억 실험실"(경과일 표시) / "뇌과학 이론".
- `ControlsPanel` — 컨트롤러 패널:
  - 우주 키우기: 감정 드롭다운(7종 — 데모는 미리 쓴 일기가 있는 `JOY`/`CALM`/`SAD`/`ANGER`/`FEAR`/`LOVE`/`NEUTRAL`) + 날짜 → `demoAddStar` / "다감정 하루" `demoAddMultiSceneStar`(21). 별도 설명 카드 없이 새 별·새 연결이 우주에 바로 반영되고, 기억 실험실은 짧은 피드백 문구만 띄운다.
  - 시간 보내기: "하루"(+1) / "한 달"(+30) / "처음"(`resetDemoExperience`) + "+N일째" 표시. 누르면 하루 단위 배치(`demoApplyDayBatch`)와 force-sim 조용한 재안정화가 끝난 최종 좌표를 보여준다.
  - 다른 삶 보기: `demoPersonaList()` 페르소나 스위처(`switchDemoPersona` → 우주 재시드, 시간·추가 별 초기화) + "우주 겹쳐보기"(37, `useDemoOverlay.setOn(true)`).
  - 데스크톱은 펼친 채, 모바일은 칩으로 접어 시작. 시트 열림을 `onSheetChange`로 페이지에 알린다(모바일 카메라 시프트). 회상 패널·일기/잠든 별 오버레이가 열리면 `suppressed` 계약으로 HUD가 닫히거나 숨는다.
- `TheoryModal` — 이론 안내 캐러셀. `entities/theory`의 `THEORIES`를 한 번에 하나씩, 점 탭·‹›·방향키·드래그(터치/마우스)로 넘긴다. 각 페이지는 미니 비주얼(`TheoryDemo`)·`oneLine`·`howTo`·`status`(planned면 🚧 배지)·`doi` 링크. `?sim=<id>` 진입은 그 이론 페이지가 떠 있는 채로 시작.

시간 보내기 오케스트레이션은 `model/time-travel.ts`:

- `runTimeSkip(queryClient, days, onSettled?)` — `demoApplyDayBatch(days)`를 호출해 `skipDemoDays(1)` + `demoConsolidate()`를 일수만큼 반복하고, `refreshActivation()`과 universe/dormant 무효화를 수행한다. `한 달`은 30회 배치다.
- `demoApplyDayBatch(days)` — 체험 우주에서만 쓰는 하루 배치 근사. 별·시냅스 개수는 삭제하지 않고, 가지치기는 weight/밝기 하향으로만 드러난다.
- `runConsolidate(queryClient)` — 수동/호환 경로용 공고화 호출. 시간 이동 경로에서는 `demoApplyDayBatch`가 하루마다 이미 적용한다.
- `resetDemoExperience()` — `exitDemoMode`→`resetDemo`→`enterDemoMode`(랜딩 tryDemo와 같은 검증된 리셋 경로).
- `switchDemoPersona(id)` — 페르소나 영속 후 같은 리셋 경로(다음 GetUniverse가 새 코퍼스로 재시드).

`runTimeSkip`/`demoApplyDayBatch`/`refreshActivation`/`demoConsolidate`(데모 야간 공고화)는 본 스펙이 둔 자리를 27이 채운 것이다 — 시간이 지나야 보이는 이론이 기존 시간 보내기 버튼으로 트리거됨을 보여주는 사례.

### 즉시 재파생 (`entities/memory/api/universe-query.ts`의 `refreshActivation`)

시간 스킵 직후엔 refetch가 아니라 재파생이 맞다(데이터는 그대로, "지금"만 움직였다). `refreshActivation()`은 별 배열 identity를 갱신(`setStars([...stars])`)해 StarField가 새 now로 밝기를 다시 굽게 하고, 엣지 brightness를 `starBrightness(lastActivatedAt, virtualNow)`로 직접 재파생하며, ambient도 새 now로 다시 종합한다(요즘 상태가 식음). dormant 목록은 `dormantInvalidateKey()` 무효화로 갱신.

### 이론 레지스트리·비주얼 (`entities/theory`)

이론 레지스트리는 위젯에서 entities로 내려와 데모 모달과 랜딩이 함께 소비한다(랜딩이 pages를 import 못 하므로 공유 비주얼은 entity 레이어).

- `model/registry.ts`의 `THEORIES: TheoryEntry[]` — 현재 7개: `engram`·`fragmentation`·`synapse`·`hebbian`·`active-recall`·`decay`·`dormant`. 모두 `status: 'done'`. `decay` 항목 문구는 26(관련성·감정 가중 망각)이 갱신했다. `fragmentation` 항목은 21이 추가했다. 새 이론은 이 배열에 한 항목을 더하면 모달에 나타난다(확장점).
- `TheoryEntry`: `id`·`title`·`oneLine`·`howTo`(어떤 컨트롤러로 어떤 행위를 하면 확인되는지)·`doi?`·`status`.

## Flutter 동등성 기준

- 데모/시뮬은 순전히 웹 클라 로컬 기능이다(서버·proto·RPC 변경 없음). Flutter는 데모 모드 자체를 가질 필요는 없다.
- 포팅이 의미 있는 것은 가상 시계 패턴뿐이다: 밝기·잠듦 파생 함수(`activation`/`starBrightness`/`isDormant`)는 `now`를 주입받는 순수 함수여야 하고, 시간 시뮬을 붙인다면 "데이터를 과거로 밀지 말고 현재 시각을 미래로 보내라"는 단조성 규칙(병합이 `max(lastRecalledAt)`)을 따라야 한다.
- 시뮬 패널·이론 캐러셀·페르소나·랜딩 SVG 인터랙션은 웹 렌더 표면이며 동등성 대상이 아니다.

## 수용 기준

1. 체험 우주에 진입하면 기억 실험실이 뜬다(기억 실험실 칩 + 뇌과학 이론 칩). 비demo(로그인) 메인 우주에는 나타나지 않는다.
2. "하루"/"한 달"을 누르면 가상 시계가 +1/+30일 전진하고, 하루 단위 배치(`skipDemoDays(1)` → `demoConsolidate()`)가 일수만큼 적용된다. 사용자는 중간 force-sim 요동이 아니라 조용히 재안정화된 최종 우주를 본다(실제 감쇠 수식 `LAMBDA=ln2/30` 그대로).
3. 가상 시간이 충분히 지나 raw activation이 임계(`2·A_MIN`) 이하가 된 별은 잠든 별로 분류돼 잠든 별 목록에 나타나되, 밝기는 `A_MIN=0.05` 바닥 밑으로 내려가지 않는다.
4. 별을 클릭해 ≥2초 머무르면 불변 원본 일기가 열리고 그 별의 `lastRecalledAt`이 가상 now로 전진해(재점화) 별이 다시 밝아진다. universe 무효화로 화면에 반영된다.
5. 두 별을 각각 ≥2초씩 번갈아 회상하면 사이 연결 weight가 +0.05씩(상한 1.0) 즉시 굵어지고, 연결이 없던 페어면 `co_recall` 엣지를 로컬 생성한다(서버 쓰기 없음).
6. 별을 <2초만 스치면 어떤 연결도 굵어지지 않는다(능동 인출 게이팅 — dwell 타이머 취소).
7. 체험 우주에서 새 일기/별을 띄우면 같은 날(temporal)·같은 mood(semantic)·최근 회상 별(흥분성 편향) 연결이 생성된다. 기억 실험실은 별도 설명 카드 없이 짧은 피드백만 띄운다.
8. "다감정 하루"는 한 일기를 색이 다른 N개 조각 별로 갈라 강한 일내 선(0.8)으로 묶어 태운다.
9. 랜딩 카드의 "체험 우주에서 해보기"는 `enterDemoMode()` 후 `/?sim=<id>`로 이동하고, 해당 이론 페이지가 펼쳐진 채 시작한다(잘못된 id는 무시).
10. 비데모 모드에서 `virtualNowMs() === Date.now()`이고 패널·재점화·로컬 weight 분기가 일절 개입하지 않는다.
11. "처음"/페르소나 전환은 가상 시계 0 + demo 리셋으로 초기 우주 복귀. "체험 종료"·새로고침 시 offset과 추가분이 소멸한다.
12. 새 이론은 `THEORIES`에 항목 하나를 더하면 이론 모달에 나타난다(시간 의존 이론은 기존 시간 보내기 버튼이 트리거).
13. `shared/lib/demo/{clock,observe,simulate}.ts`·`entities/theory/model/registry.ts`·`widgets/demo-sim/model/time-travel.ts`는 three/React/DOM을 import하지 않는다.
14. 백엔드·proto·RPC·sqlc 변경 없음(FE 전용).
15. 랜딩 카드는 대응 이론 상태 배지(✅/🚧 + plan NN)를 달고, JourneyAct 챕터 순서가 19–27 이론 흐름(엔그램→분할→연결·강화→재공고화→망각→요즘 상태→야간→공명)과 정합한다.
