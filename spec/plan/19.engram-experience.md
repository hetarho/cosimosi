# 19. 체험으로 보는 뇌과학 — 기억 시뮬레이션 + 랜딩 재정비 (engram-experience)

> 데모(체험) 우주에 **가상 시계 + 시뮬레이션 패널**을 넣어, 이미 구현된 6개 엔그램 이론을 사용자가 **직접 트리거하고 눈앞에서 변하는 것을 보게** 한다 — "하루/한 달 지나기" 버튼으로 별이 어두워지고 잠들며, 회상하면 다시 밝아지고, 두 별을 오가면 연결이 굵어지고, 새 일기를 쓰면 별이 뜨며 연결이 생긴다. 그리고 **랜딩 페이지를 재정비**한다 — 카드 설명·인터랙션이 우리가 실제 구현한 것을 정확히 표현하는지 감사하고, 각 카드에 더 좋은 2D 인터랙션이 없는지 검토·업그레이드하며, 카드를 해당 시뮬레이션으로 바로 잇는다. | 범위: FE | 선행: — | 차단: —

## 목적 (Why)
MVP(01–13)에서 6개 엔그램 이론을 이미 구현했지만, 사용자는 그것이 "신경과학을 옮긴 것"임을 알아챌 수 없다 — 시간 감쇠는 실제로 며칠을 기다려야 보이고, 잠든 별은 100일이 지나야 생기며, 데모에서는 회상해도 별이 다시 밝아지지 않고 새 일기를 써도 연결이 생기지 않는다. 본 스펙은 **새 이론 로직을 추가하지 않고**, 이미 동작하는 이론들을 데모 우주 안에서 **시뮬레이션 가능(simulatable)** 하게 만든다.

핵심 메커니즘은 **데모 가상 시계**다. 밝기·잠듦 계산은 전부 `activation(lastRecalledAt, now)`에 `now`를 주입하는 순수 함수이므로(`entities/memory/model/activation.ts`), 데모에서 `now`를 `Date.now() + offset`으로 바꾸고 "하루 지나기 / 한 달 지나기" 버튼으로 offset을 키우면 — **눈앞에서** 별이 어두워지고, 임계를 넘은 별이 잠들고, 잠든 별을 회상하면 다시 밝아진다. 시간이 실제 흐른 것과 동일한 코드 경로다(시뮬레이션 ≠ 연출).

이 가상 시계·시뮬레이션 패널은 **specs 20–27의 확장 프레임워크**이기도 하다: 야간 공고화(27)·재공고화(23)·관련성 감쇠(26)처럼 "시간이 지나야 보이는" 이론은 구현 후 같은 "하루 지나기" 버튼으로 시연되고, 패널 레지스트리에 항목 하나를 더하면 새 이론의 시뮬레이션 액션이 나타난다.

수면 위로 끌어올릴 6개 **완료된** 이론과 각각의 시뮬레이션 수단:
1. **엔그램=별** — 별 클릭 ≥2초 → 불변 원본 일기(이미 동작). 패널이 가장 밝은 별로 데려간다.
2. **의미+시간 시냅스 생성** — 새 일기 작성 → 별 등장 + **데모에서도 가까운 기억과 연결 생성**(현재는 연결 없이 뜸 — 본 스펙이 채움).
3. **헵 공동회상 강화** — 두 별을 ≥2초씩 오가면 사이 연결이 **즉시 굵어짐**(데모 로컬, 상한 1.0).
4. **능동 인출 게이팅** — 스침(<2초)은 강화 안 됨 — 굵어지지 않는 대조를 직접 확인.
5. **시간 감쇠 + 최소 밝기** — **"하루/한 달 지나기"** → 전체가 어두워지되 5% 바닥에서 멈춤.
6. **침묵 엔그램/잠든 별** — 시간을 보내 별을 잠재우고 → fly-to → 회상 → **재점화로 다시 밝아짐**(현재 데모는 재점화가 없음 — 본 스펙이 채움).

또한 본 스펙은 **랜딩 페이지 재정비 패스**를 포함한다. 랜딩은 cosimosi의 *의도된 최종 비전 showcase*이므로(정책: [policy/ux/landing.md](../policy/ux/landing.md)):
- **정확성 감사** — (a) **이미 구현된 이론** 카드의 설명·수치·인터랙션이 실제 동작과 일치해야 한다. 알려진 불일치 예: `HebbianCard`는 슬라이더로 **양방향**(LTP/LTD) 변화를 시연하지만 MVP 구현은 +0.05 **단조 증가**뿐(양방향 약화는 23 재공고화 소관 — "계획됨"으로 표기하거나 시연을 단조로 조정), `TimeWindowCard`의 "10분~30일 간격" 연출 vs 실제 같은날+0.3 규칙, `SilentEngramCard`의 시연 바닥 `FLOOR=0.12` vs 정전 `A_MIN=0.05`(시연용 값임을 카드가 드러내거나 정합). (b) **계획된 이론(20–27)** 카드는 의도한 비전을 정확히 표현하며 상태(✅구현됨/🚧계획됨)를 드러낸다. (계획된 이론의 실제 메커니즘 반영은 각 스펙의 T-landing 소관.)
- **인터랙션 업그레이드** — 9개 챕터(ConceptSection~ResonanceSection) 각각에 대해 "이 이론을 더 잘 전달하는 2D 인터랙션은 없는가"를 검토하고, 가치 있는 개선을 적용한다. 가드레일은 [policy/ux/landing.md](../policy/ux/landing.md) §비주얼·모션: 트렌디 SVG/CSS(랜딩 본문 WebGL 신규 도입 금지), 항상-온 미세 모션(완전 정적 금지), coarse pointer/모바일 in-view 트리거, glow 클리핑 금지, 곡선 시냅스 전용, `prefers-reduced-motion` 존중.
- **체험 연결** — 각 카드가 "이 카드 체험하기"로 해당 시뮬레이션에 바로 잇는다.

## 설계 요점
**가상 시계(단일 주입 지점, 순수).** 시간 시뮬레이션의 전부는 "렌더 계층이 읽는 현재 시각"을 한 함수로 모으는 것이다. 실제 감쇠 수식·임계·병합은 한 줄도 바뀌지 않는다.
```ts
// shared/lib/demo/clock.ts (순수 — three/React/DOM import 금지, 헌법4)
let offsetMs = 0
/** 렌더·파생 계층의 "현재 시각". 비데모면 항상 Date.now() — 일반 모드 무영향. */
export function virtualNowMs(): number {
  return Date.now() + (isDemoMode() ? offsetMs : 0)
}
export function skipDemoDays(days: number): void   // offset += days·DAY_MS (데모에서만 의미)
export function demoOffsetDays(): number            // 패널 표시용 "+N일째"
export function resetDemoClock(): void              // exitDemoMode/resetDemo에서 호출
```
시간 스킵 직후에는 refetch가 아니라(쿼리 데이터는 그대로다) `refreshActivation()`이 스토어의 별 배열 identity를 갱신하고 엣지 brightness를 가상 now로 재파생해 즉시 다시 그린다. `/dormant` 목록은 `dormantInvalidateKey()` 무효화로 갱신.

**재점화를 데모에서 완성(서버 의미론과 동일하게).** 실서버는 RecallMemory가 `last_recalled_at=now`를 쓰고 refetch 병합으로 별이 밝아진다. 데모는 그 대칭으로: `recallMemory` 데모 분기가 `demoMarkRecalled(id)`(데이터의 타임스탬프를 가상 now로 전진)를 호출하고, 패널이 universe 쿼리를 무효화하면 기존 16 파이프라인(`demoStars()` refetch → `mergeStars`의 max가 전진을 통과)이 별을 다시 밝힌다. **"시간을 보내 잠재우고 → 회상해 다시 깨우는" 루프**가 데모에서 완결된다.

**헵 굵어짐을 즉시(서버 쓰기 없이).** `recordActiveView`에서 직전 active view와 새 view의 페어가 생기는 순간, 데모면 `bumpEdgeWeight(a, b, CO_RECALL_DELTA)`로 `useSynapseStore`를 직접 갱신한다(상한 1.0, 엣지가 없으면 `co_recall` 엣지 로컬 생성). `UniverseSynapses`가 엣지 배열 변경에 반응해 즉시 굵게 다시 그린다. flush(`reinforceLinks`)의 데모 no-op과 11의 비데모 영속 경로는 불변. (co-recall 세션의 `deltas`는 in-place 변이라 store 구독으로는 감지 불가 — 그래서 `recordActiveView` 내부 분기가 정확한 훅 지점이다.)

**시뮬레이션 패널 레지스트리(확장점, 헌법4 순수).** 항목 1개 = 이론 1개. 데이터는 `widgets/demo-sim/model`에 순수 타입으로 두고, 우주 요소 참조는 `shared/lib/demo/observe.ts` 셀렉터의 id만 쓴다(좌표·three 의존 없음). **20–27은 이 배열에 항목 하나를 추가**하면 패널에 등장한다 — 특히 시간이 지나야 보이는 이론(26 관련성 감쇠·27 야간 공고화)은 기존 "하루 지나기" 버튼이 그대로 트리거가 된다.

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 사용자가 데모 우주에 진입하면 THE SYSTEM SHALL 시뮬레이션 패널(접이식 2D HUD)을 띄워 시간 머신(하루/한 달 지나기·경과일)과 **6개 완료 이론**의 시뮬 액션을 나열한다(비데모 모드에는 나타나지 않음). _(헌법4: 2D HUD)_
1.2 WHEN "하루 지나기"/"한 달 지나기"를 누르면 THE SYSTEM SHALL 가상 시계를 +1일/+30일 전진시키고 모든 별·시냅스 밝기를 **즉시** 재파생해 어두워진 우주를 다시 그린다(실제 감쇠 수식 `LAMBDA=ln2/30` 그대로 — 연출 금지). _(시간 감쇠)_
1.3 WHEN 가상 시간이 충분히 지나 별의 raw activation이 임계 이하가 되면 THE SYSTEM SHALL 그 별을 잠든 별로 분류해 `/dormant` 목록에 나타내되, 밝기는 5% 바닥 밑으로 내려가지 않는다(별은 사라지지 않음). _(헌법2)_
2.1 WHEN 데모에서 (잠든) 별을 클릭해 ≥2초 머무르면 THE SYSTEM SHALL 불변 원본 일기를 표시하고 그 별의 `lastRecalledAt`을 가상 now로 전진시켜(재점화) 별이 **다시 밝아진 모습**을 보인다. _(헌법1·잠든 별 재점화 — 데모에서도 완결)_
2.2 WHEN 잠든 별 항목의 "이 별로 데려가기"를 누르면 THE SYSTEM SHALL `focusStar(dormantStarId())`로 카메라를 fly-to한다(`/dormant`와 동일 메커니즘). _(헌법2)_
3.1 WHEN 사용자가 두 별을 각각 ≥2초씩 오가며 회상하면 THE SYSTEM SHALL 사이 연결의 weight를 데모 로컬로 +0.05씩(상한 1.0) 올려 **즉시 굵어짐**을 보이고, 연결이 없던 페어면 `co_recall` 연결을 로컬 생성한다(서버 쓰기 없음 — `reinforceLinks` 데모 no-op 유지). _(헵 강화, 11 비데모 경로 불변)_
3.2 WHEN 별을 <2초만 스치면 THE SYSTEM SHALL 어떤 연결도 굵어지지 않으며, 패널은 "2초 이상 머물러야 강화돼요"를 안내한다(`DWELL_MS=2000` 근거). _(능동 인출 게이팅)_
4.1 WHEN 데모에서 새 일기를 작성하면 THE SYSTEM SHALL 새 별과 함께 같은 (가상)날 별과의 temporal 연결, 같은 mood 최신 별과의 semantic-근사 연결을 생성해 "기억은 가까운 기억과 이어진다"를 보이고, 패널은 실서버 기준(임베딩 τ=0.75·top-8·같은날+0.3)의 데모 근사임을 명시한다. _(시냅스 생성)_
5.1 WHEN 사용자가 랜딩에서 EngramCard/HebbianCard/TimeWindowCard/SilentEngramCard/ConceptSection의 "이 카드 체험하기"를 누르면 THE SYSTEM SHALL `enterDemoMode()` 후 `/universe?sim=<id>`로 이동해 해당 시뮬 항목이 펼쳐진 채 시작한다. _(체험 연결)_
6.1 WHEN 새 이론을 추가하려면 THE SYSTEM SHALL `SIM_ENTRIES`에 항목 1개를 더하는 것만으로 패널에 그 이론의 시뮬 액션이 나타나게 한다(확장점 — 20–27 소비; 시간 의존 이론은 기존 시간 머신 버튼이 트리거). _(확장점)_
7.1 WHEN 비데모(로그인) 모드로 동작하면 THE SYSTEM SHALL `virtualNowMs() === Date.now()`이고 패널·재점화 분기·로컬 weight 분기가 일절 개입하지 않는다(11·16 경로 불변). _(비데모 무영향)_
7.2 WHEN 본 스펙이 빌드되면 THE SYSTEM SHALL 새 `*.proto` 메시지·새 RPC·새 백엔드 쿼리를 추가하지 않는다(FE 전용 — grep 검증). _(비목표)_
7.3 WHEN `shared/lib/demo/{clock,observe}.ts`·`widgets/demo-sim/model/*`가 빌드되면 THE SYSTEM SHALL `three`/React/DOM을 import하지 않는다(순수·모바일 재사용). _(헌법4)_
8.1 WHEN 랜딩 재정비 패스를 수행하면 THE SYSTEM SHALL 구현된 6개 이론 카드의 설명·수치·인터랙션을 실제 동작과 일치시키고([policy/ux/landing.md](../policy/ux/landing.md) 기준 — HebbianCard 양방향 시연·TimeWindowCard 간격 연출·SilentEngramCard 바닥값 등 알려진 불일치 해소), 각 카드에 대응 이론 + 상태(✅구현됨/🚧계획됨 + plan NN)를 표시하며, `LandingPage` JourneyAct 순서가 19–27 이론 흐름과 정합하는지 점검한다. _(비전 정합)_
8.2 WHEN 랜딩 9개 챕터를 각각 검토하면 THE SYSTEM SHALL "이 이론을 더 잘 전달하는 2D 인터랙션은 없는가"를 카드별로 평가한 기록(채택/보류 사유)을 남기고, 채택된 개선을 policy §비주얼·모션 가드레일(트렌디 SVG·항상-온 미세 모션·모바일 in-view 트리거·glow 클리핑 금지·곡선 시냅스·reduced-motion) 안에서 적용한다. _(인터랙션 업그레이드)_

## 완료 정의 (DoD)
- 데모 진입(랜딩 "체험해보기" 또는 카드 "이 카드 체험하기") 후 **시뮬레이션 패널**이 표시된다(비데모 모드에는 나타나지 않음).
- 6개 이론 **시뮬 워크스루**가 모두 통과한다:
  1. "하루 지나기"×수회 / "한 달 지나기" → 모든 별·연결이 **눈에 띄게 어두워지고**, 충분히 지나면 잠든 별이 늘어나(`/dormant` 목록 갱신) 밝기는 5% 바닥에서 멈춘다.
  2. 잠든 별 "데려가기" → fly-to → ≥2초 회상 → 원본 일기 + **그 별이 다시 밝아진다**(재점화 — 시간 스킵→잠듦→재점화 루프 완결).
  3. 두 별을 ≥2초씩 오가면 사이 연결이 **즉시 굵어진다**(+0.05/회, 상한 1.0); 연결 없던 페어는 co_recall 연결이 생긴다.
  4. <2초 스침은 아무것도 굵어지지 않는다(능동 인출 게이팅 — 패널 안내 일치).
  5. 새 일기 작성 → 별 등장 + 같은 날 temporal·같은 mood 근사 연결 생성(패널이 "실서버는 임베딩 τ=0.75·top-8" 근사임을 명시).
  6. 별 클릭 ≥2초 → 불변 원본 일기(읽기 전용) + "별 하나 = 엔그램 하나" 설명.
- "처음으로" → 가상 시계 0 + 데모 리셋으로 초기 우주 복귀. "체험 종료"·새로고침 시 offset·로컬 변경분 소멸(기존 데모 수명 규칙).
- **레지스트리 확장성**: `SIM_ENTRIES`에 더미 항목 1개를 임시 추가하면 패널에 1개가 더 나타남을 확인(20–27 끼움 방식 검증) — 확인 후 제거.
