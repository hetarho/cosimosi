## Purpose

Forgetting in cosimosi is **not erasure ŌĆö it is losing access** (Ryan & Frankland 2022): an episodic-memory star that has
not been recalled in a long while **dims** and its **current-memory text loses words at random** ("ļéśļŖö ņśżļŖś ņ╣£ĻĄ¼ļ×æ
xxxxx ĒīīņŖżĒāĆļ×æ xxxx ņĀĢļ¦É ļ¦øņ×łņŚłļŗż" ŌåÆ progressively more `xxxx`), yet the trace is **never deleted** ŌĆö brightness and text
stop at a floor and wait to be revived by recall ([F1][F2], the silent engram, Roy 2017). This is the Ebbinghaus
forgetting curve made visible, and it is what makes recall meaningful: *use it or lose it*.

This is **domain math**, not UI: the client renders effective brightness **every frame** while the server needs the
identical value for later cost-gating ([38.forgetting-cost](38.forgetting-cost.md)) ŌĆö so it must be **the same math
everywhere it runs**, pinned by golden-parity like [18.synapse-plasticity](18.synapse-plasticity.md) ŌĆö and it must never
violate the model: the system deletes nothing ([I1][F3]), reading never mutates a stored fact ([I5]), and forgetting is a
**read-time function of universe time** ([T4], no cron), floored so a star fades in appearance but its row lives forever.

Plan 18 shipped `EffectiveBrightness(ŌĆ”)` as an Epic-A **stub** resolving to full brightness with the decay input
reserved. This unit **fills that stub**: elapsed universe-days since the last recall (offset by retrieval-induced
neighbor nudges) drive a floored brightness decay and a discrete **decay stage**; each stage removes more words from a
text; **arousal slows it** ([F6]) and **connection strength slows it** ([F7]). It is the load-bearing read-time
derivation of the whole forgetting axis, which [38](38.forgetting-cost.md)/[39](39.forgetting-visuals.md) consume.

## Scope / Non-goals

**In scope**
- **The read-time forgetting math** as pure, IO-free functions in `packages/memory-logic` (TS) Ōåö `internal/memory` (Go),
  over scalars drawn from the `EpisodicMemory` aggregate (no VO mutation):
  - **Effective elapsed universe-days** ŌĆö `effectiveElapsedDays(now, lastRecalled, createdUniverseTime, forgettingOffsetDays) ŌåÆ days`:
    universe-days since the last recall (or since creation, when never recalled), plus the signed neighbor
    `forgetting_offset_days` (CC4), floored at `0`.
  - **`EffectiveBrightness(ŌĆ”)`** ŌĆö the plan 18 stub, now **filled**: a floored, monotone-decreasing brightness in
    `[floor, 1]` computed from effective elapsed days, **slowed by arousal ([F6]) and by connection strength ([F7])**;
    `elapsed = 0 ŌåÆ 1.0` (full), never below `forgetting.brightness_floor`, never `0`, never mutating a stored fact
    ([F1][F2][I1]).
  - **`DecayStage(ŌĆ”)` ŌĆö the discrete forgetting stage** `0..N` a memory has reached: a monotone-non-decreasing step
    function of the same effective elapsed days (with the same arousal/connection slowing), so brightness and stage
    **move together** ([F1]) and the stage floors at the deepest stage (never a "gone" stage ŌĆö [F2]).
  - **The decay-stage word-removal algorithm** ŌĆö `decayStageText(currentText, stage) ŌåÆ text`: a **deterministic**,
    seeded random removal of a per-stage ratio of words from a text, replacing each removed word with the redaction
    token, **preserving structure** (POS/first-and-last heuristics) so the text degrades naturally ([F1][F9]). This is
    what *produces* the `decay_stages` strings ([R8a]).
- **Adding `decay_stages` to the universe read + DTO.** The `decay_stages JSONB` column exists (plan 16) but is **not
  currently selected** by `ListUniverseEpisodicMemories` (recon). This unit adds it to the read query and the
  `GetUniverse` DTO so the client has the stored decay texts (plus the fields to compute the current stage) ŌĆö the read's
  only change here.
- **Golden-parity (TSŌåöGo)** for every function above, pinned by a shared fixture table like plan 18, so a tuning change
  cannot desync client render from server gating.
- Enumerating the exact `forgetting.*` coefficients/floor/ratios these functions read.

**Non-goals**
- **No recall/reconsolidation writes.** *Resetting* brightness and *recovering* decay stage on recall ([F5][R2]) ŌĆö the
  anchor writes (`last_recalled_universe_time`, `recall_count`, timer reset) ŌĆö are [33.recall-usecase](33.recall-usecase.md)
  (CC1). This unit only *reads* those anchors; recovery is the natural consequence of `last_recalled` moving to now.
- **No neighbor-offset write.** The signed `forgetting_offset_days` (retrieval-induced ┬▒ on *neighbors*, [R5]) is
  **written** by [32.reconsolidation-rules](32.reconsolidation-rules.md) (CC4) on recall; this unit only **reads** it in
  the elapsed-days computation. The recalled star itself recovers wholly ([F5]); the offset applies to neighbors only.
- **No cost math.** *Deeper decay ŌåÆ costlier recall* ([F4]) and *recovery on recall* ([F5], cost side) are
  [38.forgetting-cost](38.forgetting-cost.md); this unit exposes the decay-depth signal (the stage/effective brightness),
  not a cost multiplier and never a Twinkle price ([CC3], no price constant outside Epic F).
- **No rendering.** Mapping effective brightness ŌåÆ star dimming and the word-loss text onto stars / the detail panel is
  [39.forgetting-visuals](39.forgetting-visuals.md) ([V2]); this unit returns numbers and strings, not pixels.
- **No LLM.** Decay texts are **random word removal**, an algorithm (not semantic compression) ŌĆö no LLM call. Contrast
  the **semantic** stage texts (`semantic_stages`, LLM-pregenerated, [C7], Epic E), a *separate axis* ([F] is
  independent of [C]).
- **No stage-text generation *scheduling*.** *When* a new decay-stage string is produced and persisted over an advance
  interval is bound to [41.consolidation-usecase](41.consolidation-usecase.md)'s concrete `AdvanceProgression` handler;
  this unit owns the *algorithm* that produces a stage text, and the read-time stage/brightness that need no persistence
  at all.
- **No new table, no schema reshape.** `decay_stages` already exists (plan 16); this unit only selects it. No column is
  added here (`forgetting_offset_days` is added by plan 32).
- **No emotionŌåÆposition/connection coupling.** Arousal slows decay ([F6]) and is read here for *that* only; valence/mood
  never enter (valence is color-only, [I3][M7]).

## Design

### Placement (domain-first Go)
Forgetting decay is **read-time behavior of the memory aggregate** ŌĆö it dims and erases a star as universe time passes ŌĆö
so it lives in `internal/memory` (Go) and `packages/memory-logic` (TS), **not** a new context (ARCHITECTURE ┬¦2.2,
be-architecture: keep the aggregate's read-time math with the aggregate). It sits in the **pure domain layer**: total,
deterministic, side-effect-free, depending only on its scalar arguments and the generated `forgetting.*` constants ŌĆö no
clock, repository, logger, or ambient randomness (the word-removal randomness is a *seeded PRNG*, an argument, not IO).
The single non-pure change ŌĆö selecting `decay_stages` in the universe read ŌĆö lands in `memory/pg` (the sole sqlc seam)
and the `GetUniverse` DTO/mapper.

### The inputs (given, not defined here)
All read from the `EpisodicMemory` root (plan 16) + the clock (plan 29); none is mutated by reading:
- `last_recalled_universe_time DATE` (NULL until first recall) ŌĆö the recency anchor. When NULL, decay is measured from
  `created_universe_time` (a never-recalled star still forgets ŌĆö `elapsed = now ŌłÆ created`).
- `created_universe_time DATE` ŌĆö the fallback recency anchor + the connection/age baseline.
- `arousal REAL` (Emotion VO, plan 17) ŌĆö the **only** emotion input; high arousal slows decay ([F6][I3]).
- `forgetting_offset_days REAL NOT NULL DEFAULT 0` ŌĆö the signed neighbor nudge (CC4), **added by plan 32**, read here.
- a **connection-strength signal** for [F7] ŌĆö see the note below.
- `now` ŌĆö the universe clock DATE ([T5], plan 29), passed in as the read-time "now"; the caller computes it, never this
  unit.
- `decay_stages JSONB` ŌĆö the stored per-stage texts ([R8a]) the read now returns (not an input to the math; produced by
  the algorithm below).

**Decided (2026-07):** the **connection-strength signal for [F7]** is the memory's **`EffectiveStrength`
(size, [V3])** ŌĆö recall-accumulated strength that already reflects how load-bearing/well-shared the memory is ŌĆö passed
in as a scalar, not recomputed here. This keeps [F7] a single clean modulation input, avoids this pure function reaching
into the synapse graph (which would break purity), and reuses the memory-level value plan 18 already defines. The exact
mapping strengthŌåÆslowdown is a coefficient (`forgetting.connection_slow_coefficient`), the shape a formula (code). The
caller (render / gating) already has `EffectiveStrength` in hand, so no extra query.

### `effectiveElapsedDays` ŌĆö the recency clock with the neighbor offset (CC4)
Signature (canonical; casing may differ per language):
```
effectiveElapsedDays(now DATE, lastRecalled DATE?, createdUniverseTime DATE, forgettingOffsetDays float) ŌåÆ float
```
```
anchor   = lastRecalled ?? createdUniverseTime          // never-recalled ŌåÆ decay from creation
elapsed  = max(0, universeDaysBetween(anchor, now))     // universe-days, forward-only ([I10])
effective = max(0, elapsed + forgettingOffsetDays)      // CC4 neighbor nudge, floored at 0
```
The signed `forgetting_offset_days` (written on a *neighbor's* recall by plan 32: `ŌłÆ` = slow forgetting for a
1-shared-semantic-neuron neighbor, `+` = speed forgetting for Ōēź2 shared ŌĆö [R5]) shifts the neighbor's effective age. It
is floored at `0` so an offset can never make a memory "younger than new". Universe-days are counted between DATEs (day
granularity, consistent with the whole engine ŌĆö plan 29). This is the single elapsed-days quantity both `EffectiveBrightness`
and `DecayStage` consume, so **brightness and stage move together** ([F1]).

### `EffectiveBrightness` ŌĆö the plan 18 stub, now filled ([F1][F2][F6][F7])
Signature (fills the reserved decay input on plan 18's stub):
```
EffectiveBrightness(effectiveElapsedDays float, arousal float, effectiveStrength float) ŌåÆ brightness   // in [floor, 1]
```
This **widens** plan 18's Epic-A stub `EffectiveBrightness(elapsedUniverseDays) ŌåÆ 1.0` from one reserved param to three:
the first param's meaning becomes the **offset-inclusive** `effectiveElapsedDays` (`= max(0, (now ŌłÆ last_recalled) +
forgetting_offset_days)`, [CC4]), plus `arousal` and `effectiveStrength`. As part of this unit, plan 18's stub
golden-fixture rows and its two call sites (web `packages/memory-logic`, Go `internal/memory`) are migrated to the new
signature ŌĆö so "no other read-time function changed" holds while `EffectiveBrightness`'s callers are re-threaded.
Behavior the acceptance criteria pin (the **behavior** is the contract; the **shape** stays in code):
- **`effectiveElapsedDays = 0 ŌåÆ 1.0`** (a just-recalled or brand-new star is full ŌĆö recall reset [R2] is the elapsed
  going to 0).
- **Monotone non-increasing** in `effectiveElapsedDays` (more time-since-recall ŌåÆ never brighter).
- **Floored, never 0.** The result never drops below `forgetting.brightness_floor` and never reaches `0` ŌĆö the silent
  engram ([F2][I1]); the base fact is never mutated by reading.
- **Arousal slows it ([F6]).** Higher `arousal` flattens the decay so a high-arousal memory stays bright longer, via
  `forgetting.arousal_slow_coefficient`. Arousal is the *only* emotion input ([I3]); valence never enters.
- **Connection strength slows it ([F7]).** Higher `effectiveStrength` flattens the decay too, via
  `forgetting.connection_slow_coefficient`.

**Decided (2026-07):** the initial decay **shape is exponential** ŌĆö
`brightness = floor + (1 ŌłÆ floor) ┬Ę decayFactor^(effectiveElapsedDays / slow)` where
`decayFactor = 1 ŌłÆ forgetting.brightness_decay_per_day` and `slow = 1 + arousal┬Ęarousal_slow_coefficient +
effectiveStrength┬Ęconnection_slow_coefficient` (the two modulators *stretch* the time axis, so both slow the fade
without changing the floor) ŌĆö matching plan 18's `EffectiveSynapseStrength` precedent. The shape is a **formula
(code, excluded from values)**; only the coefficients + floor are values. It is replaceable by any monotone-decreasing,
floored curve as a formula change if [F9]-style tuning demands.

### `DecayStage` ŌĆö the discrete forgetting stage ([F1][F2])
Signature:
```
DecayStage(effectiveElapsedDays float, arousal float, effectiveStrength float) ŌåÆ int   // 0 .. maxStage
```
A **step function** of the *same* effective elapsed days (with the *same* arousal/connection slowing), so the stage a
star is at is always consistent with its brightness ([F1] "ĒĢ©Ļ╗ś ņøĆņ¦üņ×ä"). Behavior:
- **Monotone non-decreasing** in `effectiveElapsedDays`; `effectiveElapsedDays = 0 ŌåÆ 0`.
- **Floors at the deepest stage.** The maximum stage is the **derived length** of the per-stage ratio array (below) ŌĆö it
  never exceeds it, and there is **no "gone" stage** past the last one ([F2]). The deepest stage is the most-redacted
  text, still readable-as-fragments, never empty.
- **Arousal + connection slow the crossing** ([F6][F7]) ŌĆö the same `slow` factor stretches the day thresholds, so a
  high-arousal / well-connected star reaches each stage later.

**Decided (2026-07):** stage boundaries are **universe-day thresholds** governed by `forgetting.stage_interval_days`
(days per stage, stretched by `slow`); `stage = min(maxStage, floor(effectiveElapsedDays / (stage_interval_days ┬Ę
slow)))`. The *number* of stages is the **derived length** of `forgetting.stage_word_removal_ratios` (not a separate
declared count ŌĆö array lengths are derived). The threshold spacing is a formula over one coefficient; the ratios are the
values.

### `decayStageText` ŌĆö the random word-removal algorithm ([F1][F9])
Signature:
```
decayStageText(currentText string, stage int, seed int64) ŌåÆ string
```
Produces the stage-`stage` decay text from a memory's `current_text` by **removing a per-stage ratio of words at random
and replacing each with the redaction token** ("ļéśļŖö ņśżļŖś ņ╣£ĻĄ¼ļ×æ xxxxx ĒīīņŖżĒāĆļ×æ xxxx ņĀĢļ¦É ļ¦øņ×łņŚłļŗż"). It is
**deterministic given `(currentText, stage, seed)`** ŌĆö the randomness is a **seeded PRNG** (an argument), so the same
inputs give the same redaction on client and server (golden-parity) and across re-derivations. Rules:
- **Ratio per stage** ŌĆö `forgetting.stage_word_removal_ratios[stage]` fraction of removable words is redacted, strictly
  increasing with stage (deeper stage ŌåÆ more `xxxx`).
- **Monotone / nested removal.** Stage *k+1* removes a **superset** of stage *k*'s words ŌĆö words already lost stay lost
  as decay deepens, so the progression reads as continuous erosion, and `decayStageText` at successive stages is
  consistent with the same seed.
- **Structure preservation ([F9]).** A **POS/position heuristic** keeps the text legible: prefer removing content words
  (nouns/verbs/adjectives) over function words, and **never remove the first and last word** of a sentence, so the
  sentence's skeleton survives. **Decided (2026-07):** v1 ships a **simple, language-agnostic heuristic** (word-index
  seeded selection + a stop-word/first-last guard); the PRD marks the *exact* ratios / POS rules a **"to refine"**
  ([F9]), so the ratios live in `values.yaml` (tunable without a code change) while the heuristic itself is a
  **formula/content (code)**, refinable later toward per-language POS without reshaping the contract.
- **Floor ([F2]).** Even the deepest stage keeps a **non-empty** skeleton (the first/last-word guard guarantees it) ŌĆö a
  faded memory is fragments, never a blank.
- **Redaction token** ŌĆö the visual `xxxx` marker is **UI content**, not a tuning number; it is code/content (excluded
  from values), owned with the algorithm (rendered by [39](39.forgetting-visuals.md)).

**Where the produced text is stored.** These strings are what fill `decay_stages JSONB` ([R8a]). *Producing and
persisting* a newly-reached stage's text over an advance interval is bound to [41.consolidation-usecase](41.consolidation-usecase.md)'s
`Consolidate` handler, the concrete `AdvanceProgression` hook (CC7) ŌĆö this unit owns the **algorithm**; [41] owns the
advance-time write that fills missing stage strings. The **read-time** `DecayStage`/`EffectiveBrightness` need no stored text at all (they are pure functions of
elapsed time), so the client can always show the correct dimming even before a stage string is persisted; the stored
strings are the exact word-loss text ([39](39.forgetting-visuals.md) shows the persisted string for the current stage,
falling back to `current_text` when a stage string is not yet filled).

### Adding `decay_stages` to the universe read (the one persistence change)
`ListUniverseEpisodicMemories` (`apps/api/db/queries/memory/universe.sql`) already selects `current_text`, `arousal`,
`recall_count`, `created_universe_time`, `last_recalled_universe_time`, `semantic_stage`, `semantic_stages`, ŌĆ” but
**not** `decay_stages` (recon). This unit adds `decay_stages` to that `SELECT` and threads it through the `memory/pg`
rowŌåÆdomain mapper and the `GetUniverse` DTO, so the client receives the stored decay texts. It also confirms the read
carries `arousal` + the fields to compute `EffectiveStrength` (already selected) so the client can compute
`EffectiveBrightness`/`DecayStage` from stored facts alone ([I5] ŌĆö derived, not stored). `forgetting_offset_days` is
added to the read by plan 32 (which adds the column); this unit consumes it once present and, until then, treats a
missing offset as `0` (the DEFAULT). No proto **message** change is required beyond the added `decay_stages` field on the
existing `GetUniverse` episodic-memory sub-message.

**Decided (2026-07):** the DTO carries the **stored `decay_stages` strings + the anchor facts**, and the client computes
the *current* stage/brightness from those facts via `packages/memory-logic` ŌĆö the server does **not** pre-render a
"current decay text" into the DTO (that would freeze a read-time value into transport, violating [I5] and desyncing from
the frame-by-frame client clock). One exception is not made: the read stays stored-facts-only, exactly like plan 16's
read stance.

### Golden-parity (TSŌåöGo)
Per ARCHITECTURE ┬¦2.5 the two runtimes must agree, because the client renders `EffectiveBrightness`/`DecayStage`/`decayStageText`
every read while the server computes them for cost-gating ([38](38.forgetting-cost.md)). The Go implementation in
`internal/memory` and the TS mirror in `packages/memory-logic` are pinned equal by a **shared golden fixture**
(`apps/api/internal/memory/testdata/forgetting-decay-golden.json`, `(function, inputs) ŌåÆ expected` rows + a fixed
tolerance), extending plan 18's harness. Fixtures cover: `effectiveElapsedDays` with NULL `lastRecalled` (creation
fallback), with signed offsets (both directions, and the `0` floor), `EffectiveBrightness` at `elapsed = 0` (= 1.0),
across a decay range hitting the floor, and at high vs low arousal / strength (slower fade), `DecayStage` at each
threshold boundary and floored at `maxStage`, and `decayStageText` proving determinism under a fixed seed, the nested
superset property, ratio-per-stage, and the first/last-word (non-empty) guard. Both sides read the same generated
`forgetting.*` constants (`pnpm gen:values` emits Go + TS), so a tuning change cannot desync render from gating.

### Purity & re-implementation
- The four functions live in the pure domain of `internal/memory` (Go) and `packages/memory-logic` (TS) ŌĆö no `pg`,
  `rpc`, SDK, clock, or ambient randomness; the word-removal PRNG is a seeded argument (still pure). They take/return
  domain scalars/strings, never proto DTOs or sqlc rows (┬¦2.4 anti-corruption boundary).
- No formula, curve shape, threshold spacing, redaction token, or POS heuristic is a value: only the coefficients,
  floor, and per-stage ratios go to `spec/values.yaml`; the shapes are code. Array lengths (stage count) are **derived**.
- The whole axis is reproducible on another client from: one elapsed-days clock (offset + floor), one floored monotone

## Acceptance Criteria
1. `effectiveElapsedDays(now, lastRecalled, created, offset)` returns universe-days from `lastRecalled` (or from
   `created` when `lastRecalled` is NULL), **plus** the signed `forgetting_offset_days`, **floored at `0`**; it is
   never negative and a never-recalled star still accrues elapsed days from creation ([F1][I10] + CC4).
2. `EffectiveBrightness(effectiveElapsedDays, arousal, effectiveStrength)` satisfies **`= 1.0` at `elapsed = 0`**, is
   **monotone non-increasing** in `effectiveElapsedDays`, and stays within **`[forgetting.brightness_floor, 1]` ŌĆö never
   below the floor and never `0`** (silent engram; stored fact never mutated) ([F1][F2][I1][I5]).
3. Higher `arousal` yields a **slower** brightness fade at the same elapsed days (`EffectiveBrightness` non-decreasing in
   `arousal`), governed by `forgetting.arousal_slow_coefficient`; **valence/mood are not inputs** ([F6][I3]).
4. Higher `effectiveStrength` (connection/size) yields a **slower** fade at the same elapsed days, governed by
   `forgetting.connection_slow_coefficient` ([F7]).
5. `DecayStage(effectiveElapsedDays, arousal, effectiveStrength)` is a **monotone-non-decreasing** step function of the
   same effective elapsed days (so stage and brightness move together), is `0` at `elapsed = 0`, and **floors at the
   maximum stage** = the derived length of `forgetting.stage_word_removal_ratios` ŌĆö there is **no stage past the last**
   ([F1][F2]).
6. `decayStageText(currentText, stage, seed)` is **deterministic** given its inputs, removes the
   `forgetting.stage_word_removal_ratios[stage]` fraction of words (strictly increasing with stage) replacing each with
   the redaction token, and its removed-word set at stage *k+1* is a **superset** of stage *k*'s ([F1]).
7. `decayStageText` **preserves structure**: it never removes the first or last word of a sentence and prefers content
   words over function words, so even the **deepest stage is non-empty** and legible-as-fragments ([F2][F9]). (Exact
   ratios/POS rules are a `values.yaml`-tunable "to refine" per [F9].)
8. Decay texts are produced by an **algorithm, not an LLM**, and are **independent of the semantic (`semantic_stages`)
   axis** ŌĆö forgetting removes words at random; semanticization compresses meaning ([F1], [C] independence).
9. `ListUniverseEpisodicMemories` (and the `GetUniverse` DTO) **selects and returns `decay_stages`**, plus the anchor
   facts (`arousal`, `created_universe_time`, `last_recalled_universe_time`, and the fields for `EffectiveStrength`) so
   the client computes current stage/brightness from stored facts alone; the server does **not** pre-render a current
   decay text ([I5]). `forgetting_offset_days` (plan 32's column) is read once present, treated as `0` when absent.
10. The Go (`internal/memory`) and TS (`packages/memory-logic`) implementations produce **identical outputs** for the
    shared golden-fixture table within a fixed tolerance, and both read the same generated `forgetting.*` constants
    (TSŌåöGo golden-parity, matching plan 18).
11. All four functions are **pure**: deterministic, IO-free (no clock, DB, transport, SDK, or ambient randomness ŌĆö the
    word-removal PRNG is a seeded argument), operating only on their arguments and generated constants; the plan 18
    `EffectiveBrightness` **stub is filled** here with its Epic-D dynamics, with no other read-time function changed.
12. Forgetting is **read-time and cron-free** ([T4]): stage/brightness are pure functions of universe time; persisting a
    newly-reached stage's decay text is bound to [41]'s concrete advance hook, not a scheduler, and the system
    deletes nothing ([F3][I1]).
