## Purpose

The universe is the product's primary, felt way of holding memory — but the star is a **trace**, deliberately lossy and
mutable. The objective record, the `Diary`, is immutable ([I2][D4]) and lives _outside_ the star: the emotional core of
cosimosi is the **contrast** between the trace (the star) and the original (the diary). This unit is the place that
contrast becomes visible: a plain, calm archive where a diarist can confirm _the record is still there_ — read exactly
what they wrote, unchanged, for free ([G4]).

It is deliberately a **complete supporting role** ([D2]): the universe stays the main navigation; this list is a
reader-app-style index, "your writing is preserved" and no more. Two secondary jobs ride on it. First, **split
visualization** ([D3]): each diary shows which episodic memories (stars) it split into at launch — the many-to-one
provenance from the objective entry to the traces it seeded. Second, the **"이 일기로 태어난 별 보기"** jump ([D3]): from
an entry, the user can pay to **whole-recall** all of that diary's memories at once and fly to them — a `RecallDiaryStars`
action (not a rewrite) that syncs the clock and recovers their brightness/decay/gist-timer through the shared reinforce
bundle.

It builds the **UI, the list read, and the jump composition only** — the recall/reinforce math, the cost curve, the clock
sync, and the acceleration are their own plans; this unit _reads_ the list over the model [23] already owns and
_composes_ those capabilities, rather than scattering them into the universe widget.

## Scope / Non-goals

**In scope**

- **`pages/diary-reader`** — a **new route/screen** (web route via [15.web-router-foundation](15.web-router-foundation.md)'s
  typed registry; mobile screen via [13.mobile-app-shell](13.mobile-app-shell.md)'s navigation registry), distinct from
  the universe page. The universe stays `/` and primary; this is a separate, plain archive ([D2]).
- **`widgets/diary-reader`** — the archive block: a **reverse-chronological list** of the user's `Diary` entries
  (reader-app-style), each row showing the diary date + body preview; opening an entry shows the **full immutable body**
  ([D4]) and its split view. Read-only.
- **`entities/diary`** — a NEW pure domain-mirror slice (no three; parity with [23]'s entity mirrors): `model` = the FE
  `Diary` type (`id, body, diary_date, created_universe_time`, + its `memory_ids[]` split membership) + a Zustand store;
  `api` = maps `DiaryDto` → the FE type from the `GetDiaries` response. Shared verbatim web+mobile via `packages/`.
- **The `GetDiaries` RPC contract** this unit consumes (owned here as a _shape_; brief §CC8 — the query implementation is
  the memory context's). Recon confirms **no diary-list read exists** on `memory.v1` (only `SplitDiary`/`ReviseSplit`/
  `LaunchStars`/`GetUniverse`). Unary, HTTP GET-eligible, `NO_SIDE_EFFECTS`, per-user-scoped, paginated.
- **Split visualization** ([D3]) — each diary entry shows the **2–5 episodic memories** it split into ([E2]): each as a
  small chip/row with its name + primary-emotion color (from the [23] `episodic-memory` mirror + [17]'s `moodColor`),
  linking the objective entry to the traces it seeded. Membership comes from `memory.diary_id` (the read joins it).
- **The "이 일기로 태어난 별 보기" jump** ([D3]) — a per-diary action that **whole-recalls** all of that diary's still-live
  episodic memories at once and **flies** to them in the universe:
  - **Cost = the SUM of per-memory recall costs** ([D3][G4]) — each memory is priced by its decay depth ([F4],
    [38.forgetting-cost](38.forgetting-cost.md) computes accessibility, [43.stardust-ledger](43.stardust-ledger.md) prices
    it), but the widget receives the **server-supplied sum** from [44]'s `QuoteSpend(kind=DIARY_RECALL, diary_id)` and
    displays that quote before gating the action behind the spend.
  - **Whole recall = `RecallDiaryStars`** ([33.recall-usecase](33.recall-usecase.md)) — after consent, the jump calls
    33's no-rewrite diary recall action once. Server-side it syncs the clock, spends, and applies `Reinforce` to each
    still-live memory born from the diary in **one transaction**. It is explicitly **NOT reconsolidation** ([R6]):
    rewriting is per-star only ([D3]). Each reinforced memory gets 33's recall-effects bundle — reset
    `last_recalled_universe_time`, bump `recall_count`/`EffectiveStrength` ([R3]), reset the semanticize timer ([C6a]),
    and **apply the neighbor forgetting ± ([R5])** — with **no** `current_text`, **no** `seed` change, and **no**
    provenance row (provenance records only created/semanticized/reconsolidated, [32]). **Decided (2026-07):** the jump
    gets [F5] recovery _and_ [R5] neighbor gain precisely because 33 makes them **recall effects inside `Reinforce`**
    (not reconsolidation-only); a no-rewrite diary recall therefore carries them, and the visible effect surfaces via the
    read-time D/E functions.
  - **Clock sync** ([R1a]) — the jump syncs the universe clock to today (recall advances time), so it opens the **shared
    sync-consent modal** from [31.time-acceleration-ui](31.time-acceleration-ui.md) first; **예** ⇒ call
    `RecallDiaryStars`, then acceleration + fly from the committed response; **아니오** ⇒ cancel, clock unmoved, nothing
    spent.
  - **Fly to the memories** — on success, navigate to the universe and frame/fly the camera to the recalled cluster
    (through [23]'s camera rig); the acceleration ([31]) plays over the sync interval before the reveal.
- **Two entry points** ([D5]) — the diary reader is reachable from **(1)** a persistent affordance on the **universe
  page UI** and **(2)** the star-detail **원본 일기 보기** button ([35.star-detail-panel](35.star-detail-panel.md)),
  which deep-links to _that memory's_ diary entry (via `memory.diary_id`).
- **Original reading is FREE** ([G4]) — listing, opening, reading the immutable body, and viewing the split all cost **no**
  별가루; only the whole-recall **jump** spends (it recalls, not reads).
- **UI copy via i18n** ([08.i18n-seam](08.i18n-seam.md)) — no hardcoded strings; a **literary, restrained Korean** voice.
  **web + mobile parity** — same page/widget/features/entity; `*.native.tsx` siblings **only** for list/scroll/row
  primitives that genuinely differ (§3.5).

**Non-goals**

- **No recall/reinforce/reconsolidation math.** `Reinforce`, `Recall`, `RecallDiaryStars`, `Reconsolidate`, the
  prediction-error gate, the anchor writes, and neighbor gain ([R5]) are [33.recall-usecase](33.recall-usecase.md)/

- **No cost curve or economy.** The decay→accessibility signal is [38.forgetting-cost](38.forgetting-cost.md); the

- **No clock/sync/acceleration logic.** `SyncToToday`, the interval, the consent modal, and the choreography are

- **No star detail / recall flow.** The meta panel, the 회고하기 rewrite, 변천사, and the free forgotten-text read are

- `widgets/diary-reader` composes the two `features/*` (each = **one user action**, §3.1) and owns the jump's control-state

### The `GetDiaries` RPC contract (owned here as a shape, consumed here — §2.7 unary)

Recon: **no diary-list read exists** on `memory.v1` today. This unit fixes the contract shape; the query implementation +
per-user scoping + proto↔domain server mapping are the memory context's ([16.memory-aggregate-schema](16.memory-aggregate-schema.md),
brief §CC8 — `memory/pg`). Package `memory.v1`, service `MemoryService`. **Unary, HTTP GET-eligible, `NO_SIDE_EFFECTS`,
per-user-scoped, paginated** (reverse-chronological by `diary_date`).

```
GetDiaries(GetDiariesRequest{ page_size, page_token }) → GetDiariesResponse {
  repeated DiaryDto diaries
  string            next_page_token
}
DiaryDto { id, body, diary_date, created_universe_time, repeated DiarySplitRef memories }
DiarySplitRef { episodic_memory_id, name, mood }   // the split membership ([D3]); mood → color via [17] moodColor
```

- `body` is the **immutable original** ([I2][D4]) — returned verbatim, never derived. The split refs come from
  `episodic_memories WHERE diary_id = $1 AND deleted_at IS NULL` (the read already excludes soft-deleted memories, brief
  as-built), so a diary whose stars were all deleted still lists (the record persists, [I1]) but shows no split chips.
- **Decided (2026-07):** the split-visualization data rides on `GetDiaries` (the `DiarySplitRef` list) rather than
  requiring a second per-diary fetch — the list needs name+mood per memory to draw chips, and that is small; the full
  memory facts already arrive on `GetUniverse` for the fly. `next_page_token` keeps the archive lazy for long histories.
- **Original reading is free** ([G4]): `GetDiaries` and opening an entry are pure reads; **no** spend, no clock advance,
  `NO_SIDE_EFFECTS`. Only the jump mutates.

### The whole-recall jump ("이 일기로 태어난 별 보기", [D3]) — compose, don't own

The jump is a **composition** of four capabilities this unit does not own; it wires them in order:

1. **Fetch the summed cost quote.** For the diary's still-live memory ids, call [44]'s
   `QuoteSpend(kind=DIARY_RECALL, diary_id)` and display the returned `{cost, covered, shortfall}` before the user commits
   ([D3][G4]). The server prices each memory by decay depth (decay depth → accessibility, [38]; priced by [43]) and returns
   the sum; the widget adds no price and holds no cost curve.
2. **Consent to the sync.** Because whole recall advances the clock to today ([R1a]), the widget opens the **shared
   sync-consent modal** ([31.time-acceleration-ui](31.time-acceleration-ui.md)) stating the consequence + the quoted cost.
   **아니오** ⇒ cancel: clock unmoved, **nothing spent**, no reinforce.
3. **Server-side whole recall.** On **예**, call [33]'s `RecallDiaryStars(diary_id)`. That server transaction verifies
   the spend through the same gate/quote discipline ([44]), composes `SyncToToday`, and applies the internal reinforce
   bundle over the diary's still-live memory ids. It is **NOT** reconsolidation — **no** `current_text` rewrite, **no** `seed`
   change ([R6][I8]); rewriting is per-star only ([D3]). The UI receives the affected memory ids + sync interval from the
   committed response, so there is no "clock moved, reinforce failed" partial state.
4. **Fly + accelerate.** On success, navigate to the universe, play the [31] acceleration over the sync interval, and
   fly/frame the camera ([23]) to the recalled cluster. The recovered brightness/decay/gist-timer ([F5]) surface on the
   **next** `GetUniverse` refetch (read-time D/E functions read the reset anchors, §2.8) — no polling, no streaming.

- **Decided (2026-07):** the jump uses [33]'s **`RecallDiaryStars`** server action rather than composing sync/spend/
  reinforce in the client. The action still reuses the existing `Reinforce` bundle internally, keeping the anchor-write
  path single-sourced while making sync + spend + reinforcement atomic.
- **Empty / all-deleted diary:** if a diary has no still-live memories (e.g. all soft-deleted, Epic H), the jump is
  disabled (cost 0, nothing to recall) and the entry lists as an original-only record ([I1] — the diary persists).

### Two entry points ([D5])

- **Universe page UI** — a persistent, restrained affordance on the universe page (e.g. an "일기장" entry in the page
  chrome) navigates to `pages/diary-reader` at the top of the list. This is the archive's front door.
- **Star-detail 원본 일기 보기** — [35.star-detail-panel](35.star-detail-panel.md)'s button deep-links into the reader at
  **that memory's** diary entry (resolved via `memory.diary_id`), scrolled/expanded to the original body. The widget's
  public API exposes an "open at diary id" entry for this. Both routes land on the **same** reader (§3.5 parity).

### Control-state (XState, §3.2)

`model/diary-reader.machine.ts` holds the **jump** control-state only (the list itself is data-driven — a Query fetch +
the `entities/diary` store, no machine phase needed for browsing):

| State        | Meaning                                                                 | Exits                                                         |
| ------------ | ----------------------------------------------------------------------- | ------------------------------------------------------------- |
| `browsing`   | list + entry detail shown; jump idle                                    | `JUMP` → `confirming`                                         |
| `confirming` | sync-consent modal open (quoted cost stated)                            | `ACCEPT` → `recalling`; `REJECT` → `browsing`                 |
| `recalling`  | `RecallDiaryStars` in flight (sync + spend + internal reinforce bundle) | `done` → `flying`; `error` → `browsing` (nothing spent/moved) |
| `flying`     | acceleration + camera fly to the cluster                                | `done` → `browsing` (on the universe)                         |

- The diary list, the selected entry, the server quote, and the target memory ids live in **Zustand/Query** (data); the
  machine holds the phase + id refs only (§3.2). A jump whose `RecallDiaryStars` fails returns to `browsing` with the
  clock unmoved and nothing spent (the server transaction leaves no partial state). Any per-frame fly
  effect reads the machine via `getSnapshot()`, never React state (§3.2).

### Copy (i18n) — literary, restrained Korean

All strings resolve through the i18n seam ([08.i18n-seam](08.i18n-seam.md)); none hardcoded (`lint:raw-strings`). The
archive framing is quiet and plain — the reader is a keeping-place, not a feature to sell — and the jump's consent copy
states the consequence honestly (the quoted cost, the clock sync) in the restrained voice (PRD [§3.1](../PRD.md)). Web +
mobile draw from the **same** message keys (parity).

  `/implement-job`; **do not** pre-create policy files):
  - **The original is a free, immutable keeping-place.** Listing, opening, reading the body, and viewing the split are
    free ([G4]); the reader never mutates a `Diary` ([I2][D4]).
  - **The universe is primary; the reader is supporting.** The archive is a plain reader-app list, not a second main
    surface ([D2]); it is reachable from two doors ([D5]).
  - **The jump whole-recalls but never rewrites.** "이 일기로 태어난 별 보기" = `RecallDiaryStars` (clock sync + spend +
    internal reinforce bundle for brightness/decay/gist-timer recovery + neighbor gain), **never** a reconsolidation —
    rewriting is per-star only ([D3][R6][I8]).
  - **The jump costs the sum of recall costs and requires sync consent.** The server quote is shown up front, the sync-
    consent modal precedes it, and "아니오" cancels with nothing spent and the clock unmoved ([D3][R1a]).
- **tech/** — the as-built FE rules (the `pages/diary-reader` + `widgets/diary-reader` composition over two features + the
  `entities/diary` mirror; the `diary-reader.machine.ts` jump phases with the list/quote/ids in Zustand/Query; the jump as
  a composition of [33] batch-reinforce + [43]/[44] spend + [31] sync/acceleration + [23] fly; the `GetDiaries`
  read/DTO shape) are recorded in [tech/state-machine.md](../tech/state-machine.md) and [tech/rendering.md](../tech/rendering.md)
  at `/implement-job`, reinforcing [ARCHITECTURE.md](../ARCHITECTURE.md) §3.1/§3.2/§2.7/§2.8 rather than adding a new concern.
