# 08. star-rendering — 도메인 모델(순수) + 별 시각화(InstancedMesh·TSL)

> 기억=별의 순수 도메인 모델과 별 시각화를 만든다. | 범위: MVP | 선행: 06, 07 | 차단: 10, 12

## 목적 (Why)
기억(별)을 표현할 **순수 도메인 모델**(타입·활성도·스토어)과, 수천 별을 **InstancedMesh 1개**로 그리는 시각화를 만든다. 별마다 다른 형태·색·밝기는 per-instance 시드(FE가 `memory_id`에서 결정론적 파생)를 TSL에서 변형해 표현하고, 좌표는 force-sim(07) 버퍼를 ref로 구독해 React 리렌더 없이 매 프레임 갱신한다. 이후 데이터 fetch(10)와 감쇠 렌더(12)가 이 모델·렌더 위에 올라간다.

## 범위

  - `shared/config/mood.ts` — 감정 팔레트(mood→색)
  - 각 슬라이스 Public API(`index.ts`)
- **비목표(Out of scope):**
  - 시냅스 렌더(→09), 데이터 fetch·`RecordMemory`/`GetUniverse` 바인딩(→10), 감쇠 애니메이션·잠든 별 탐색·`last_recalled_at` 운영(→12)
  - 회상 강화·재공고화·변천사(→11, v1+ #20·#21)
  - 시드 기반 **풍부한 생성 형태**(결정·성운 등) — MVP는 기본 형상+per-instance 변형까지(→v1 #20). InstancedMesh·시드 입력 경로만 지금 깐다.

## 설계 요점
**원칙4 — `model`은 순수.** `entities/memory/model/*`·`shared/config/mood.ts`는 `three`/React/DOM을 import하지 않는다(모바일 재사용). 타입에 json/db/proto 태그 없음(원칙5).

```ts
// entities/memory/model/types.ts  (순수 — three/React 미의존)
// Mood 7종은 02 proto `enum Mood`(JOY/CALM/SAD/ANGER/FEAR/LOVE/NEUTRAL + UNSPECIFIED)와 단일 출처.
// FE는 proto enum ↔ 아래 7 mood 문자열을 매핑한다(매핑 책임은 10). 본 enum은 임의로 늘리지 않는다.
export type Mood = 'joy' | 'calm' | 'sad' | 'anger' | 'fear' | 'love' | 'neutral';
export interface Memory {            // 별 도메인 객체(전송/영속 타입 아님)
  id: string;
  mood: Mood;
  intensity: number;                 // 0..1 (감정 강도 → 크기·존재감)
  lastRecalledAt: number;            // epoch ms (활성도 계산 입력)
  seed: number;                      // memory_id에서 결정론적 파생(해시) — 형태 변형용(아래 seed 노트)
}
export interface StarNode { id: string; memory: Memory; index: number; } // index=인스턴스 버퍼 슬롯
```

```ts
// entities/memory/model/activation.ts  (순수, 단위 테스트 대상)
export const HALF_LIFE_DAYS = 30;
export const LAMBDA = Math.LN2 / HALF_LIFE_DAYS;   // ≈ 0.0231 /day
export const A_MIN = 0.05;
const DAY_MS = 86_400_000;
export function activation(lastRecalledAt: number, now: number): number {
  const dtDays = Math.max(0, (now - lastRecalledAt) / DAY_MS);
  return Math.exp(-LAMBDA * dtDays);               // (0,1], Δt=0 → 1
}
export function starBrightness(lastRecalledAt: number, now: number): number {
  return Math.max(A_MIN, activation(lastRecalledAt, now)); // 별 유효 밝기, 바닥 a_min
}
```

**seed 파생(결정론적).** `Memory.seed`는 FE가 `memory_id`에서 **결정론적으로 파생**(해시)한다 — 서버 `visual_spec`(03 `memories.visual_spec JSONB`)은 **MVP 미사용, v1으로 연기**. 낙관적 임시 별은 `temp-` id로 파생하고, 서버 확정 시 server `memory_id`로 **재파생**한다(별이 저장되며 형태가 한 번 "굳는" 효과 — 허용). 같은 `memory_id`는 항상 같은 seed→같은 형태(재현성).
> **해시 함수 단일 출처.** 해시는 `entities/memory/model/seed.ts`의 순수 헬퍼 `seedFromId(id: string): number` 하나로 두고 `entities/memory` Public API로 노출한다. 10의 매퍼(`map-star`)와 낙관 흐름(`use-record-memory`)이 **이 헬퍼를 공유 import**한다(동일 id→동일 seed 보장; 각자 인라인 해시 금지).

**StarField (ui — 플랫폼 레이어, three/TSL 여기서만):**
- `InstancedMesh`(공유 지오메트리, count=별 수) **1개** → 소수 draw call(원칙8).
- per-instance 속성 버퍼: `aSeed`(시드), `aMood`(색 RGB), `aIntensity`, `aBrightness`. `store`+`mood.ts`+`activation.ts`로 CPU에서 채워 `InstancedBufferAttribute`로 올림.
- **TSL 머티리얼**(`MeshStandardNodeMaterial`/노드): 크기=`f(intensity)`(인스턴스 행렬 scale 또는 노드), 색=`aMood`, **밝기=`max(a_min, activation)`→emissive**, per-instance 시드로 정점/표면 미세 변형(형태 고유성). WebGL2 폴백 자동(§3.1).
- **좌표 구독:** `useFrame`에서 07 force-sim 출력 `Float32Array`(ref/worker 메시지)를 읽어 `instanceMatrix` position만 갱신 → `instanceMatrix.needsUpdate = true`. **React state로 60fps 구동 금지**(§3.2-2). 06 `<Canvas>` 자식으로 마운트.
- 좌표 버퍼 미연결 시(09/10 이전 독립 검증) 더미 좌표 생성기를 ui 내부 dev 경로로 허용(임시).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN N개 별이 스토어에 있을 때 THE SYSTEM SHALL 별을 **단일 `InstancedMesh`**로 렌더하여 별 지오메트리에 대한 draw call 수가 N에 비례해 증가하지 않게 한다(소수 draw call).
1.2 WHEN 별 mood·intensity·activation이 주어질 때 THE SYSTEM SHALL 색=mood 팔레트, 크기=f(intensity), emissive 밝기=`max(a_min, activation)`을 TSL 머티리얼로 반영한다.
1.3 GIVEN `lastRecalledAt`와 `now` WHEN `activation(...)`을 호출하면 THE SYSTEM SHALL Δt=0에서 1.0, 30일 경과 시 0.5±0.01을 반환한다.
1.4 WHEN 임의의 큰 Δt에서 THE SYSTEM SHALL `starBrightness(...)`가 `a_min`(0.05) 미만으로 내려가지 않게 한다(별은 사라지지 않음 — 원칙2).
1.5 IF mood 값이 팔레트에 없을 때 THEN THE SYSTEM SHALL `NEUTRAL_RGB`로 폴백하고 throw하지 않는다.
1.6 WHEN force-sim 좌표 버퍼가 갱신될 때 THE SYSTEM SHALL `useFrame` 내 `instanceMatrix` 갱신만 수행하고 React 리렌더(컴포넌트 재실행)를 유발하지 않는다(원칙3·§3.2).
1.7 THE SYSTEM SHALL `entities/memory/model/**`·`shared/config/mood.ts`가 `three`·`react`·`react-dom`·DOM 전역을 import/참조하지 않게 한다(원칙4).
