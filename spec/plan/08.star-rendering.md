# 08. 별 렌더링 - as-built

> 별 렌더링의 현재 권위는 `entities/memory`의 순수 star domain, `shared/config`의 13 mood palette, `entities/star`의 instanced 3D/2D visual이다. 좌표는 저장/전송하지 않고 force-sim live buffer에서 나온다.

## Mood와 색
- mood의 FE 단일 출처는 `frontend/src/shared/config/mood.ts`다.
- 13개 mood:
  - `joy`, `calm`, `sad`, `anger`, `fear`, `love`, `neutral`
  - `excitement`, `gratitude`, `relief`, `stress`, `tired`, `emptiness`
- proto `Mood`는 `map-star.ts#moodFromProto()`에서 domain mood string으로 매핑한다.
- unknown/unspecified mood는 `neutral`과 `NEUTRAL_RGB`로 fallback한다.
- 색은 linear RGB tuple `readonly [number, number, number]`이며 `three.Color`가 아니다.
- 사용자 감정색 override는 `resolveMoodRgb(mood, emotionColors)`로 적용한다. 별과 시냅스 endpoint 색은 같은 override를 읽는다.

## Memory domain object
`Memory`는 transport/persistence type이 아니다. 현재 필드:

```ts
interface Memory {
  id: string
  mood: Mood
  intensity: number       // 0..1
  valence: number         // -1..1
  relevance: number       // 0..1, recent-topic centroid alignment
  lastRecalledAt: number  // epoch ms
  recordId: string
  fragmentIndex: number
  seed: number
  brightnessOffset: number
  hueShift: number
  formSeedDelta: number
  version: number
  resonant: boolean
}

interface StarNode {
  id: string
  memory: Memory
  index: number
}
```

- `body`, full original text, `entryDate`, `createdAt`는 render star에 넣지 않는다. 원본은 `records`/`RecallMemory`/record queries의 책임이다.
- `recordId`와 `fragmentIndex`는 원본 일기 단위 조망/하이라이트에 사용한다.
- `brightnessOffset`, `hueShift`, `formSeedDelta`, `version`은 reconsolidation/reshaping state이며 server-authoritative 값이다.
- `relevance`는 server가 `GetUniverse`에서 계산한 recent-topic alignment다.
- `resonant`는 다른 우주와 공명으로 이어진 별임을 나타내고, StarField가 ring marker로 보여준다.

## Store 계약
- `useMemoryStore`는 `StarNode[]`를 저장한다. 좌표는 저장하지 않는다.
- 주요 state/action:
  - `stars`
  - `ambient`
  - `loadedEmpty`
  - `setStars(stars)`
  - `setLoadedEmpty(loadedEmpty)`
  - `setAmbient(ambient)`
  - `addStar(node)`
  - `addStars(nodes)`
  - `replaceStar(tempId, node)`
  - `replaceMany(tempIds, nodes)`
  - `removeStar(tempId)`
- `removeStar()`는 `temp-` id만 제거한다. 서버에서 온 별은 클라이언트가 삭제하지 않는다.
- `replaceMany()`는 fragment fan-out 이후 temp batch를 제거하고, 이미 refetch로 들어온 server star는 중복 append하지 않는다.
- `loadedEmpty`는 "로딩 전 0개"와 "실제로 빈 우주"를 구분한다. 첫 일기 별 birth animation이 이 값에 의존한다.
- `starsOfRecord(stars, recordId)`는 `recordId`가 같은 별을 `fragmentIndex` 순으로 반환한다.

## Focus state
- 별 선택은 memory store의 `selectedId`가 아니라 `focus.machine.ts`의 `focusActor`가 소유한다.
- 상태는 `idle`, `star`, `diary`, `pair`다.
- `StarField`는 `selectedId`와 `highlightedRecordId`를 prop으로 받아 표시만 한다. actor를 직접 import하지 않는다.
- `onSelect(id)` prop은 `UniverseCanvas`가 받아 `focusActor.send({ type: 'SELECT_STAR', id })`로 배선한다.

## activation/brightness
- 기본 decay:
  - `HALF_LIFE_DAYS = 30`
  - `LAMBDA = ln(2) / 30`
  - `A_MIN = 0.05`
  - `activation(lastRecalledAt, now) = exp(-lambda * dtDays)`
  - `starBrightness = max(A_MIN, activation)`
- 현재 StarField는 단순 brightness가 아니라 modulated brightness를 사용한다.
  - `degreeNormById(edges)`로 connection degree를 median-normalized한다.
  - `modulatedBrightness(lastRecalledAt, now, degreeNorm, relevance, intensity, valence)`를 계산한다.
  - `reshapedBrightness(base, brightnessOffset)`를 합성해 최종 brightness attribute로 올린다.
- decay modulation은 연결, 최근 topic relevance, 감정 강도, negative valence가 decay를 느리게 만드는 방향만 허용한다.

## proto Star -> StarNode
- `mapStar(star, index)`가 generated proto `Star`를 domain `StarNode`로 바꾼다.
- 매핑:
  - `memoryId -> id`
  - `mood -> moodFromProto()`
  - `intensity`
  - `valence`
  - `relevance`
  - `lastRecalledAt -> parseEpochMs(..., virtualNowMs())`
  - `recordId`
  - `fragmentIndex`
  - `seed -> seedFromId(id)`
  - `brightnessOffset`, `hueShift`, `formSeedDelta`, `version`
  - `resonant`
- 좌표는 절대 매핑하지 않는다. star coordinate는 force-sim buffer에서만 나온다.

## StarField 렌더링
- `StarField`는 별 본체를 하나의 `InstancedMesh`로 렌더한다.
- props:
  - `positionsRef`
  - `object`
  - `emotionColors`
  - `highlightedRecordId`
  - `selectedId`
  - `onSelect`
  - optional external `stars`, `edges`, `tint`, `tintStrength` for overlay.
- `externalStars`가 없으면 `useMemoryStore((s) => s.stars)`를 읽는다.
- `externalEdges`가 없으면 `useSynapseStore((s) => s.edges)`를 읽어 degree modulation에 사용한다.
- per-instance attributes:
  - `aMood`: mood RGB with optional override/tint.
  - `aSeed`: `reshapedSeed(seed, formSeedDelta)`.
  - `aBrightness`: modulated + reshaped brightness.
  - `aHueShift`: `hueShift` degrees converted to radians.
- instance matrix:
  - position은 매 프레임 `positionsRef.current`를 읽는다.
  - buffer가 없으면 `fibonacciStarPosition(i, count, seed)` fallback을 쓴다.
  - scale은 `0.6 + clamp01(intensity) * 1.4`.
  - wobble은 `wobbleUnit(seed, t, axis) * WOBBLE_AMP`이며 reduced motion이면 0이다.
- 매 프레임 React state를 갱신하지 않는다. matrix/attribute ref만 imperative update한다.

## 별 형태
- `StarObject` 종류:
  - `deepfield`
  - `aurora`
  - `liquid`
  - `ember`
- 선택지는 `STAR_OBJECTS`, 기본값은 `DEFAULT_OBJECT = 'deepfield'`.
- `buildStarForm(object)`가 geometry/material/update를 반환한다.
- WebGPU path:
  - `MeshStandardNodeMaterial` from `three/webgpu`
  - TSL nodes from `three/tsl`
- 형태별 요약:
  - `deepfield`: low-poly crystal, view-dependent facets, slow deterministic rotation.
  - `aurora`: icosahedron nebula cloud, fractal noise flow.
  - `liquid`: displaced glossy bead.
  - `ember`: octahedron ember/lava facets.
- animation time은 built-in TSL time node가 아니라 manual `uTime` uniform을 `StarField.useFrame`에서 갱신한다.

## birth, focus, resonance
- 새 별 birth:
  - 첫 full load는 애니메이션하지 않는다.
  - `loadedEmpty` 이후 생긴 첫 별 또는 기존 우주에서 새로 들어온 별은 scale birth animation을 받는다.
  - birth duration은 1.2s, burst layer는 1.6s다.
  - external overlay source는 birth animation을 끈다.
- focus/diary highlight:
  - 선택 별 또는 강조 일기 별은 `FOCUS_BOOST = 1.3`.
  - 나머지는 `FOCUS_DIM = 0.12`.
  - single star focus가 diary highlight보다 우선한다.
- resonance:
  - `memory.resonant === true`인 별은 additive billboard ring marker를 받는다.
  - marker는 최대 64개까지 렌더하고, reduced motion에서는 pulse를 정지한다.

## Ambient
- `Ambient` summary는 `hue`, `sat`, `arousal`, `valence`다.
- server `GetUniverseResponse.ambient`가 있으면 그것을 사용한다.
- 없으면 client가 loaded stars에서 `deriveAmbient()`를 수행한다.
- 다중 light pool 분포는 항상 client가 `ambientLights(stars, virtualNowMs())`로 파생한다.
- `AmbientNebula`는 이 pool을 far sphere 위 additive billboard로 렌더한다.

## Merge semantics
- `applyUniverse()`는 `mergeStars()`를 통해 incoming stars를 local render set에 병합한다.
- 같은 id:
  - `lastRecalledAt`은 max를 취한다.
  - `relevance`, reshaping fields, `resonant`는 server-authoritative라 항상 incoming 값을 반영한다.
- server-only star는 뒤에 append한다.
- local-only temp/just-confirmed star는 유지한다.
- index는 array slot과 같아야 하며, append/reindex 액션이 이 불변식을 지킨다.

## Flutter 동기화 포인트
- Flutter의 star model도 위 `Memory` 필드 전체를 포함해야 한다. 특히 `valence`, `relevance`, `recordId`, `fragmentIndex`, reshaping fields, `resonant`를 누락하면 web과 다른 결과가 나온다.
- 좌표는 domain/store에 저장하지 않는다. renderer가 force-sim live buffer row를 읽는다.
- focus state는 별 store 안의 boolean/string이 아니라 별/일기/쌍이 배타인 상태기계로 둔다.
- user emotion color override와 selected star object는 renderer 입력이다. mood 의미색은 유지하고 override만 우선 적용한다.
- birth animation은 first-load batch에는 적용하지 않고, 실제 empty universe를 확인한 뒤 들어온 첫 별에는 적용한다.
