# 07. force-sim — Barnes-Hut 순수 tick + Web Worker

> 연결 가중치 그래프에서 별 좌표를 창발시키는 순수 `tick(dt)` 레이아웃(Barnes-Hut)과 Web Worker 래퍼. 기존 별은 pin, 신규+직접 이웃만 이동. | 범위: MVP | 선행: — | 차단: 08

## 목적 (Why)
별의 위치는 임베딩(1536차원)이 아니라 **연결 관계에서 물리 시뮬레이션으로 창발**한다(concept §결정1). 강한 시냅스끼리 당기고 모든 별이 밀어내면 성단이 빚어진다. 새 기억이 추가될 때마다 전체를 재계산하면 "내 기억이 거기 있다"는 감각이 깨지므로(concept §결정1 제약), **기존 별은 고정(pin)하고 신규 별과 그 직접 이웃만** 움직인다. 수천 노드를 전제로 O(N²) 척력을 Barnes-Hut quadtree로 근사하고, 메인 스레드를 막지 않도록 Web Worker에서 돌린다.

## 범위
- **포함:**
  - `sim.ts` — 순수 함수. 입력=노드/엣지 배열, 출력=좌표 `Float32Array`. 인력(엣지 가중치)·척력(Barnes-Hut quadtree, θ 근사)·약한 중심 인력. `requestAnimationFrame`/`useFrame`/`three`/DOM 일절 미참조(원칙4).
  - `worker.ts` — Web Worker 래퍼. `init`(그래프 적재) / `tick`(N스텝 진행) / `dispose` 메시지 수신, 좌표 `Float32Array`를 transferable로 회신.
  - 부분 배치 정책 — `pinned` 노드는 좌표 불변, **신규 노드 + 그 직접 이웃(엣지로 연결된 1-hop)만** 자유 이동. 나머지는 고정.
  - `index.ts` 공개 API + `types.ts` 입출력 계약.
- **비목표(Out of scope):**
  - 렌더링(InstancedMesh/Line2/TSL) — **08·09**가 담당. 여기서는 좌표 버퍼만 생산.
  - 캔버스/카메라/Bloom — **06**.
  - WebGPU compute 오프로딩 — v1(현재는 Web Worker만).
  - 전체 재안정화(re-layout, "우주의 수면") — v1 백로그 **#22**. MVP는 부분 배치만.
  - 좌표 서버 캐싱·권위 저장 — 서버는 가중치 그래프만(원칙3). 좌표는 클라 산출.

## 설계 요점
좌표는 **3D**(별 우주). quadtree 대신 **octree** Barnes-Hut를 쓰되, 본 문서는 관용대로 "Barnes-Hut"로 통칭한다.

```ts
// types.ts — 순수 입출력 계약 (three/React/DOM 미의존)
export interface SimNode { id: string; pinned: boolean; x: number; y: number; z: number }
export interface SimEdge { source: string; target: string; weight: number } // weight ∈ [0,1]
export interface SimGraph { nodes: SimNode[]; edges: SimEdge[] }
export interface SimParams {
  theta: number;        // Barnes-Hut 근사 임계, 기본 0.9
  repulsion: number;    // 척력 계수 (음수 charge), 기본 -30
  linkDistance: number; // 엣지 목표 거리, 기본 30
  centerGravity: number;// 중심 인력, 기본 0.01
  velocityDecay: number;// 속도 감쇠(마찰), 기본 0.6
  alphaMin: number;     // 수렴 종료 임계, 기본 0.001
}

// sim.ts — 순수 모듈. 상태는 호출자가 보유, tick은 부수효과로 positions를 갱신.
export function createSim(graph: SimGraph, params?: Partial<SimParams>): SimState;
export function tick(state: SimState, steps?: number): Float32Array; // [x0,y0,z0, x1,y1,z1, ...] 노드 순서 보존
export function positions(state: SimState): Float32Array;            // 현재 좌표 스냅샷
export function alpha(state: SimState): number;                      // 현재 alpha (수렴도)
```

**부분 배치 정책(원칙3 핵심):**
- `createSim`에서 노드를 3분류: ① `pinned===true` → 고정(힘 적용·이동 없음, 단 척력 소스로는 참여) ② 신규 노드(pinned===false)와 그 1-hop 이웃 → **자유(free) 이동** ③ 그 외 free가 아닌 기존 노드 → 고정.
- free 집합 = `{ n | !n.pinned } ∪ { n | n이 free 노드와 엣지로 직접 연결 }`. 1-hop만(전파 금지).
- 매 tick: free 노드만 위치 갱신. 고정 노드 좌표는 입력값 그대로 유지(인력/척력의 **소스**로는 사용해 신규 별이 기존 성단에 끌리게 함 — concept §경쟁적 할당의 토대).
- 신규 노드 초기 좌표: 직접 이웃 free/pinned 노드들의 평균 ± 작은 지터(이웃 없으면 원점 근방 구면 랜덤).

**Barnes-Hut:** 매 tick octree 재구축 → 각 free 노드에 대해 셀 폭/거리 ≥ θ면 셀 질량중심으로 일괄 척력(O(N log N)). 인력은 엣지 순회(`weight`로 강도 스케일 → 강연결일수록 가깝게).

**Worker 계약(`worker.ts`):**
```ts
// main → worker
{ type: 'init'; graph: SimGraph; params?: Partial<SimParams> }
{ type: 'tick'; steps: number }   // 진행, alpha와 좌표 회신
{ type: 'dispose' }
// worker → main
{ type: 'positions'; buffer: ArrayBuffer; alpha: number; settled: boolean } // buffer는 transfer로 반환
```
- 좌표는 `Float32Array.buffer`를 **transferable**(`postMessage(msg, [buffer])`)로 넘겨 복사 비용 0. 메인은 받은 버퍼를 렌더(08)에 그대로 전달.
- 노드 순서는 `init`의 `nodes` 배열 순서로 고정 → 인덱스가 곧 InstancedMesh 인스턴스 인덱스(08과의 계약).
- Worker는 **자체 rAF 루프를 돌리지 않는다**(원칙4 정신: 스케줄링은 호출자). 메인이 `tick` 메시지로 펌프; 호출 측(08 `useFrame` 또는 setInterval)이 구동.

## 수용 기준 (EARS / Given-When-Then)
1.1 GIVEN 강하게 연결된 노드 쌍(weight=1.0)과 약하게 연결된 쌍(weight=0.1) WHEN `tick`을 alpha가 `alphaMin` 이하로 떨어질 때까지 반복 THE SYSTEM SHALL 강연결 쌍의 최종 거리를 약연결 쌍보다 짧게 수렴시킨다.
1.2 GIVEN `pinned===true` 노드 WHEN 임의 스텝 수만큼 `tick` THE SYSTEM SHALL 해당 노드의 (x,y,z)를 입력값과 동일하게 유지한다(부동소수 오차 ≤ 1e-6).
1.3 GIVEN pinned 기존 노드들 + 그에 연결된 신규 노드 1개 WHEN `tick` THE SYSTEM SHALL 신규 노드와 그 1-hop 이웃만 좌표를 변경하고, 1-hop 밖의 free 아닌 노드는 고정한다.
1.4 WHEN `positions(state)` 호출 THE SYSTEM SHALL 길이 `nodes.length*3`인 `Float32Array`를 노드 입력 순서대로 반환한다.
1.5 WHEN 노드 수 N≥3000인 그래프로 `createSim` 후 단일 `tick` THE SYSTEM SHALL O(N log N) 경로(octree 근사)로 처리해 1스텝을 한 프레임 예산(브라우저 환경 < 16ms 목표, 테스트 환경은 비차단)으로 완료한다.
1.6 WHEN worker에 `init`→`tick` 메시지를 보냄 THE SYSTEM SHALL `positions` 메시지로 좌표 버퍼를 transferable로 회신하고, 메인 스레드를 동기 차단하지 않는다.
1.7 IF 엣지의 `source`/`target`가 노드 목록에 없으면 THEN THE SYSTEM SHALL 해당 엣지를 무시하고(throw 금지) 나머지 그래프로 진행한다.
1.8 IF `tick` 누적으로 alpha ≤ `alphaMin` THEN THE SYSTEM SHALL `settled=true`를 보고하고 이후 free 노드 이동을 멈춘다(불필요한 연산 회피).
