# 31. time acceleration ui

> Time-acceleration UI — the presentation unit of Epic B: it makes the universe clock **visible and felt**. A persistent
> **"우주의 시간" HUD** ([T6]) shows the current universe time; a **time-acceleration choreography** plays when the clock
> advances ([T2] — "accelerate to the diary date, forgetting/consolidation playing through, then the star appears"); and
> a **sync-consent modal** ([T2] case 2 / [R1a]) asks before a recall pulls the clock to today. One XState machine owns
> the overlay's control-state; the clock value + advance interval are data.
> Scope: FE | Status: planning
>
> Layer P (presentation) — the third and final unit of Epic B. Home: `widgets/universe-time` (composing three
> `features/*`) + the `entities/universe-clock` mirror, over the running `widgets/universe-canvas`
> ([23.universe-canvas](23.universe-canvas.md)). It reads the clock + interval from
> [30.time-advance-usecase](30.time-advance-usecase.md) (via `GetUniverse` + the extended `LaunchStars` response). The
> live **recall** trigger of the sync-consent modal is Epic C (`recall-flow-ui`), which *composes* the modal built here.

## Purpose

The universe clock is invisible unless the product shows it. This unit is where Epic B becomes something the user can
**see and feel**: a diarist should always know *what time it is in their universe* ([T6]), should **feel** the jump when
a new star launches them forward through time ([T2]), and should be **asked before** a recall spends their past by
syncing the clock to today ([T2] case 2 / [R1a]). Without this unit the clock advances silently — stars would dim and
gist would rise with no felt cause, and the recall consent ([R1a]) — the one moment the product warns the user they are
about to close off past dates — would not exist.

It builds three presentation surfaces over the already-running universe canvas: the **HUD** that reads the current
universe time, the **acceleration choreography** that plays over an advance interval, and the **sync-consent modal**. It
consumes the backend Epic B built ([30.time-advance-usecase](30.time-advance-usecase.md)): the clock arrives on
`GetUniverse.universe_time`, and a launch's advance interval arrives on the extended `LaunchStars` response. It builds
the **UI and its control-state only** — the clock, the advance, and the sync capability are the backend's.

## Scope / Non-goals

**In scope**
- **`entities/universe-clock`** — the FE mirror of the clock value (pure, **no three**; parity with plan 23's
  domain-mirror entities): `model` = the `UniverseClock` type (`currentUniverseTime: Date | null`) + a small Zustand
  store; `api` = reads `universe_time` from the `GetUniverse` response. Shared verbatim web+mobile via `packages/`
  (the app slice exposes the store + public API), mirroring [23.universe-canvas](23.universe-canvas.md)'s entities.
- **`features/universe-clock-hud`** — the **"우주의 시간"** HUD ([T6]): a small, persistent, restrained overlay on the
  universe page showing the current universe time (= the last diary date) from `entities/universe-clock`. Shows nothing
  (or an empty-universe affordance) when the clock is nil. Live now — it reads a value already on `GetUniverse`.
- **`features/accelerate-time`** — the **acceleration choreography** ([T2]) played over an advance interval
  `[previous, current]` (from the extended `LaunchStars` response now; from `SyncToToday` via recall later): a
  "time-passing" transition — the HUD's date advancing, a dilation/sweep over the scene — after which the launched
  star's reveal lands ([T2] case 1: *accelerate, then the star appears*). This unit owns the **choreography shell + the
  interval seam**; the concrete forgetting-dimming ([F], Epic D) and gist-rising ([C]/[E], [V8][C8]) visuals are a
  **reserved slot** the acceleration plays into once those epics ship (the PRD's "reserve the slot" discipline, [V8]).
- **`features/confirm-time-sync`** — the **sync-consent modal** ([T2] case 2 / [R1a]): before a recall syncs the clock
  to today, a modal states the consequence ("회상하려면 우주 시간을 오늘로 맞춰야 해요. 그 사이 안 쓴 과거 날짜의 일기는
  이후 추가할 수 없게 됩니다. 진행할까요?" — i18n copy) with **예 / 아니오**. **예** proceeds (recall calls `SyncToToday`,
  then the acceleration plays over the sync interval); **아니오** cancels with the clock unmoved. This unit **builds the
  reusable modal + its control-state**; the **live 회고하기 trigger is Epic C** (`recall-flow-ui` composes it), the same
  way plan 27's launch triggers plan 25's awaken.
- **`widgets/universe-time`** — composes the three features over the universe page and owns the overlay's **XState
  machine** (`idle → confirming → accelerating → idle`; §3.2): control-state only, with the clock value + the advance
  interval held in Zustand/Query (ids/interval in data, phase in the machine).
- **Coordinating the launch reveal** ([T2] case 1) — on a launch that advances the clock, the launched star's *reveal*
  is played **through** the acceleration (accelerate → then the star appears), refining plan 27's immediate optimistic
  reveal. This unit hooks plan 27's launch event + plan 30's returned interval; it does **not** re-own the writing flow.
- **UI copy via i18n** ([08.i18n-seam](08.i18n-seam.md)) — no hardcoded strings; a **literary, restrained Korean** voice
  (the consent copy especially — plain, honest, unadorned). **web + mobile parity** — same widget/features/entity;
  `*.native.tsx` siblings **only** for the overlay/modal/HUD primitives that genuinely differ (§3.5).

**Non-goals**
- **No clock, advance, or sync logic.** `universe_state`, `AdvanceClock`, `AdvanceOnLaunch`, `SyncToToday`, and the
  progression hook are [29.universe-clock](29.universe-clock.md) / [30.time-advance-usecase](30.time-advance-usecase.md).
  This unit renders the clock and the interval and *calls* `SyncToToday` only through Epic C's recall composition.
- **No recall flow.** 회고하기, prediction-error, rewrite, reconsolidation result — all Epic C (`star-detail-panel`,
  `recall-flow-ui`). This unit provides the sync-consent modal Epic C's recall opens; it does not build recall.
- **No forgetting or consolidation visuals.** The dimming ([F], Epic D `forgetting-visuals`), gist stars rising / the
  consolidation replay ([C]/[E], [V8][C8]) are their own epics; the acceleration **reserves the slot** and plays a
  neutral time-passing transition until they fill it.
- **No writing flow.** Diary input, split, revise, launch confirm are [27.writing-flow-ui](27.writing-flow-ui.md). This
  unit only choreographs the *time* consequence of a launch, hooking that flow's launch event.
- **No new backend / RPC.** The clock is on `GetUniverse`; the interval is on the `LaunchStars` response — both owned by
  [30.time-advance-usecase](30.time-advance-usecase.md). This unit adds no proto.
- **No richer past-date experience.** A past-dated launch returns an empty interval (no acceleration); the one-time
  past-date confirmation notice stays [27.writing-flow-ui](27.writing-flow-ui.md)'s. The HUD makes the monotonic bar
  visible; a deeper past-date UX is a reserved slot, not built here.

## Design

### Placement (FSD, §3.1)
```
apps/web/src/widgets/universe-time/
├── ui/            UniverseTimeOverlay (HUD + acceleration + modal host) + *.native.tsx sibling for overlay primitives
├── model/         universe-time.machine.ts (XState) · the interval/clock store · pure interval helpers
└── index.ts       public API (the overlay + the "open sync consent" affordance Epic C recall composes)

apps/web/src/features/
├── universe-clock-hud/  ui ("우주의 시간" HUD) — reads entities/universe-clock
├── accelerate-time/     ui (the time-passing choreography over [previous, current]) · model (playback state)
└── confirm-time-sync/   ui (the 예/아니오 consent modal, i18n copy) · model (consent result)

apps/web/src/entities/universe-clock/
├── model/         UniverseClock type + Zustand store        (pure, shared via packages/, no three)
├── api/           maps GetUniverse.universe_time → the store
└── index.ts
```
The same slices exist on mobile; `model`/`api` are shared verbatim (pure), and only the `ui` overlay/modal/HUD
primitives fork via `*.native.tsx` (§3.5). The widget imports the `entities/universe-clock` mirror and the domain-mirror
entities from [23] only to *read*; it never imports `three` or a visual entity (§3.4) — the acceleration's scene effects
are handed to the canvas as discrete events, not driven from React per frame (§3.2).

### The clock mirror + HUD ([T6])
`entities/universe-clock` mirrors the single clock value: its `api` reads `universe_time` from the `GetUniverse`
response (the same fetch [23] already makes — no new request), its store holds `currentUniverseTime: Date | null`.
`features/universe-clock-hud` renders it as the persistent **"우주의 시간"** overlay (the last diary date). When the
clock is nil (empty universe), the HUD shows an empty-universe affordance rather than a date. The HUD updates whenever
`GetUniverse` refetches (e.g. after a launch) — the clock value is authoritative from the server ([30]).

### The acceleration choreography ([T2]) — book the slot
On an advance interval `[previous, current]` (non-empty ⇒ the clock moved):
- **On launch** ([T2] case 1): `LaunchStars` returns `previous_universe_time` / `universe_time` ([30]); the widget
  enters `accelerating` and plays a **time-passing transition** — the HUD date advancing from `previous` to `current`,
  a scene dilation/sweep — **then** reveals the newly launched star ([T2]: accelerate → then the star appears). This
  refines plan 27's immediate optimistic reveal: the ids/insert are still optimistic, but the *reveal* is gated behind
  the acceleration. **No warning** ([T2] case 1).
- **On recall** ([T2] case 2, Epic C): after consent, Epic C calls `Recall` ([33]), which server-side composes
  `SyncToToday` ([30]) and the recall write in one transaction. This widget then plays the same acceleration over the
  returned sync interval while revealing the already-committed recall result. The animation is a presentation of the
  authoritative result, not a pre-write step.
- **Reserved slot** ([V8][C8]): the *content* of "forgetting/consolidation playing through" — stars dimming ([F], Epic
  D), gist stars rising to the neocortex, the replay "pulled-upward" motion ([C]/[E]) — is filled by those epics. Epic B
  plays a **neutral, restrained** time-passing transition and exposes the interval + a choreography seam so Epics D/E
  slot their per-interval visuals in without re-plumbing. An empty interval (past-dated launch) plays **nothing**.

Animation timing is **presentation**, driven off the interval (a longer time-jump reads as a slightly longer transition,
capped), and does not block the optimistic data path — a slow or skipped animation never delays persistence.

### The sync-consent modal ([T2] case 2 / [R1a])
`features/confirm-time-sync` is the reusable consent modal: it states the consequence in restrained i18n copy and offers
**예 / 아니오**. It is **built here, triggered by Epic C**: `recall-flow-ui`'s 회고하기 opens it (through the widget's
public API); **예** resolves to "proceed" (Epic C then calls `Recall`, whose server transaction syncs the clock + applies
the recall, and this widget plays the returned interval); **아니오** resolves to "cancel" — the recall is abandoned and
the clock does not move ([T2] case 2). The modal never itself calls the backend; it returns a decision, keeping the
recall composition Epic C's.

### Overlay control-state (XState, §3.2)
`model/universe-time.machine.ts` holds control-state only:

| State | Meaning | Exits |
|---|---|---|
| `idle` | HUD showing; nothing animating | `CONFIRM_SYNC` → `confirming`; `ADVANCED` → `accelerating` |
| `confirming` | sync-consent modal open (opened by Epic C recall) | `ACCEPT` → wait for Epic C's committed recall interval → `accelerating`; `REJECT` → `idle` |
| `accelerating` | playing the acceleration over `[previous, current]` | `done` → `idle` |

The advance **interval** and the clock value live in Zustand/Query (data); the machine holds the phase + the interval
ref only (§3.2). Any per-frame scene effect the acceleration drives reads the machine via `getSnapshot()` in `useFrame`,
never via React state (§3.2). A launch with an empty interval skips `accelerating` entirely.

### Copy (i18n) — literary, restrained Korean
All strings resolve through the i18n seam ([08.i18n-seam](08.i18n-seam.md)); none hardcoded (`lint:raw-strings`). The
HUD label ("우주의 시간") and the consent copy use the restrained literary voice (PRD [§3.1](../PRD.md)) — the consent
especially is plain and honest (state the consequence, ask, no decoration). Web + mobile draw the **same** message keys.

## Acceptance Criteria
1. A persistent **"우주의 시간" HUD** on the universe page shows the current universe time (= the last diary date) read
   from `GetUniverse.universe_time` via `entities/universe-clock`; when the clock is nil (empty universe) it shows an
   empty-universe affordance, not a date ([T6]).
2. `entities/universe-clock` is a **pure** mirror (no three) with a `model` (type + store) and an `api` that maps
   `GetUniverse.universe_time`; it is shared verbatim web + mobile.
3. On a **launch that advances the clock**, an **acceleration choreography** plays over `[previous_universe_time,
   universe_time]` (from the `LaunchStars` response) and the launched star is revealed **after** it (accelerate → then
   the star appears); a **past-dated** launch (empty interval) plays **no** acceleration ([T2] case 1).
4. The acceleration is a **neutral time-passing transition with a reserved slot** for forgetting-dimming ([F]) and
   gist-rising / replay ([C]/[E], [V8][C8]); it exposes the interval + a choreography seam those epics fill without
   re-plumbing, and never blocks or delays the optimistic data path.
5. A reusable **sync-consent modal** states the consequence in i18n copy with **예 / 아니오** ([T2] case 2 / [R1a]); it
   returns a decision only (it calls no backend). Its **live trigger (회고하기) is Epic C** (`recall-flow-ui` composes
   it); **예** ⇒ proceed (recall syncs + the acceleration plays), **아니오** ⇒ cancel with the clock unmoved.
6. The overlay's **control-state** is one XState machine (`idle → confirming → accelerating → idle`); the clock value +
   advance interval live in **Zustand/Query**, not machine context; per-frame scene effects read via `getSnapshot()`,
   never React state (§3.2).
7. The overlay exposes **no** rewind, meaning-layer, or placement control — it only shows time, plays a forward-only
   acceleration, and consents to a sync ([I10][I11]); recall-sync/acceleration never mutates a `Diary` ([I2]).
8. **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in a literary /
   restrained Korean voice.
9. **web + mobile parity:** the same widget + three features + the entity mirror run on both apps, sharing `model`/`api`
   verbatim; `*.native.tsx` siblings exist **only** for the overlay / modal / HUD primitives (§3.5).
