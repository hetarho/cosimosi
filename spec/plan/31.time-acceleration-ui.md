# 31. time acceleration ui

> Time-acceleration UI — the presentation unit of Epic B: it makes the universe clock **visible and felt**. A persistent
> **"우주의 시간" HUD** ([T6]) shows the current universe time; a **time-acceleration choreography** plays when the clock
> advances ([T2] — "accelerate to the diary date, forgetting/consolidation playing through, then the star appears"); and
> a **sync-consent modal** ([T2] case 2 / [R1a]) asks before a recall pulls the clock to today. One XState machine owns
> the overlay's control-state; the clock value + advance interval are data.
> Scope: FE | Status: built (job 42)
>
> Layer P (presentation) — the third and final unit of Epic B. Home: `widgets/universe-time` (composing three
> `features/*`) + the `entities/universe-clock` mirror, over the running `widgets/universe-canvas`
> ([23.universe-canvas](23.universe-canvas.md)). It reads the clock + interval from
> [30.time-advance-usecase](30.time-advance-usecase.md) (via `GetUniverse` + the extended `LaunchStars` response). The
> live **recall** trigger of the sync-consent modal is Epic C (`recall-flow-ui`), which _composes_ the modal built here.

## Purpose

The universe clock is invisible unless the product shows it. This unit is where Epic B becomes something the user can
**see and feel**: a diarist should always know _what time it is in their universe_ ([T6]), should **feel** the jump when
a new star launches them forward through time ([T2]), and should be **asked before** a recall spends their past by
syncing the clock to today ([T2] case 2 / [R1a]). Without this unit the clock advances silently — stars would dim and
gist would rise with no felt cause, and the recall consent ([R1a]) — the one moment the product warns the user they are
about to close off past dates — would not exist.

It builds three presentation surfaces over the already-running universe canvas: the **HUD** that reads the current
universe time, the **acceleration choreography** that plays over an advance interval, and the **sync-consent modal**. It
consumes the backend Epic B built ([30.time-advance-usecase](30.time-advance-usecase.md)): the clock arrives on
`GetUniverse.universe_time`, and a launch's advance interval arrives on the extended `LaunchStars` response. It builds
the **UI and its control-state only** — the clock, the advance, and the sync capability are the backend's.

## Scope / Non-goals

**In scope**

- **`entities/universe-clock`** — the FE mirror of the clock value (pure, **no three**; parity with plan 23's
  domain-mirror entities): `model` = the `UniverseClock` type (`currentUniverseTime: string | null` — the domain
  mirror carries day-granular ISO date strings end to end, matching `UniverseSnapshot.universeTime`) + a small Zustand
  store (set / clear); `api` = maps the `GetUniverse` read's `universeTime` into the store (nil ⇒ clear). Shared
  verbatim web+mobile via `packages/` (the app slice exposes the store + public API), mirroring
  [23.universe-canvas](23.universe-canvas.md)'s entities.
- **`features/universe-clock-hud`** — the **"우주의 시간"** HUD ([T6]): a small, persistent, restrained overlay on the
  universe page showing the current universe time (= the last diary date) from `entities/universe-clock`. Shows nothing
  (or an empty-universe affordance) when the clock is nil. Live now — it reads a value already on `GetUniverse`.
- **`features/accelerate-time`** — the **acceleration choreography** ([T2]) played over an advance interval
  `[previous, current]` (from the extended `LaunchStars` response now; from `SyncToToday` via recall later): a
  "time-passing" transition — the HUD's date advancing, a dilation/sweep over the scene — after which the launched
  star's reveal lands ([T2] case 1: _accelerate, then the star appears_). This unit owns the **choreography shell + the
  interval seam**; the concrete forgetting-dimming ([F], Epic D) and gist-rising ([C]/[E], [V8][C8]) visuals are a
  **reserved slot** the acceleration plays into once those epics ship (the PRD's "reserve the slot" discipline, [V8]).
- **`features/confirm-time-sync`** — the **sync-consent modal** ([T2] case 2 / [R1a]): before a recall syncs the clock
  to today, a modal states the consequence ("회상하려면 우주 시간을 오늘로 맞춰야 해요. 그 사이 안 쓴 과거 날짜의 일기는
  이후 추가할 수 없게 됩니다. 진행할까요?" — i18n copy) with **예 / 아니오**. **예** proceeds (recall calls `SyncToToday`,
  then the acceleration plays over the sync interval); **아니오** cancels with the clock unmoved. This unit **builds the
  reusable modal + its control-state**; the **live 회고하기 trigger is Epic C** (`recall-flow-ui` composes it), the same
  way plan 27's launch triggers plan 25's awaken.
- **`widgets/universe-time`** — composes the three features over the universe page and owns the overlay's **XState
  machine** (`idle → confirming → accelerating → idle`; §3.2): control-state only, with the clock value + the advance
  interval held in Zustand/Query (ids/interval in data, phase in the machine).
- **Coordinating the launch reveal** ([T2] case 1) — on a launch that advances the clock, the launched star's _reveal_
  is played **through** the acceleration (accelerate → then the star appears), refining plan 27's immediate optimistic
  reveal. This unit hooks plan 27's launch event + plan 30's returned interval; it does **not** re-own the writing flow.
- **UI copy via i18n** ([08.i18n-seam](08.i18n-seam.md)) — no hardcoded strings; a **literary, restrained Korean** voice
  (the consent copy especially — plain, honest, unadorned). **web + mobile parity** — same widget/features/entity;
  `*.native.tsx` siblings **only** for the overlay/modal/HUD primitives that genuinely differ (§3.5).

**Non-goals**

- **No clock, advance, or sync logic.** `universe_state`, `AdvanceClock`, `AdvanceOnLaunch`, `SyncToToday`, and the
  progression hook are [29.universe-clock](29.universe-clock.md) / [30.time-advance-usecase](30.time-advance-usecase.md).
  This unit renders the clock and the interval and _calls_ `SyncToToday` only through Epic C's recall composition.
- **No recall flow.** 회고하기, prediction-error, rewrite, reconsolidation result — all Epic C (`star-detail-panel`,
  `recall-flow-ui`). This unit provides the sync-consent modal Epic C's recall opens; it does not build recall.
- **No forgetting or consolidation visuals.** The dimming ([F], Epic D `forgetting-visuals`), gist stars rising / the
  consolidation replay ([C]/[E], [V8][C8]) are their own epics; the acceleration **reserves the slot** and plays a
  neutral time-passing transition until they fill it.
- **No writing flow.** Diary input, split, revise, launch confirm are [27.writing-flow-ui](27.writing-flow-ui.md). This
  unit only choreographs the _time_ consequence of a launch, hooking that flow's launch event.
- **No new backend / RPC.** The clock is on `GetUniverse`; the interval is on the `LaunchStars` response — both owned by
  [30.time-advance-usecase](30.time-advance-usecase.md). This unit adds no proto.
- **No richer past-date experience.** A past-dated launch returns an empty interval (no acceleration); the one-time
  past-date confirmation notice stays [27.writing-flow-ui](27.writing-flow-ui.md)'s. The HUD makes the monotonic bar
  visible; a deeper past-date UX is a reserved slot, not built here.

## Design

### Placement (FSD, §3.1)

```
packages/universe/src/                     the pure, shared substance (§3.5 — both apps import it verbatim)
├── universe-clock-store.ts                UniverseClock type + Zustand store (set / clear)
├── advance-interval.ts                    AdvanceInterval + emptiness predicate + capped duration mapping +
│                                          date sampling + advanceAnnouncementFromLaunch (the launch consequence)
└── universe-time.machine.ts               the overlay machine (the writingFlowMachine parity precedent)

apps/{web,mobile}/src/widgets/universe-time/
├── ui/            UniverseTimeOverlay (hosts the machine; composes the three features) — per-platform ui
├── model/         release-advance.ts (the reveal released on DONE: awaken announce + clock landing)
└── index.ts       public API (the overlay + the "open sync consent" affordance the recall flow composes)

apps/{web,mobile}/src/features/
├── universe-clock-hud/  ui ("우주의 시간" HUD, sweep-override prop) — reads entities/universe-clock
├── accelerate-time/     model (the announce seam: pending announcement, merge-on-announce, take-to-consume)
│                        · ui (the veil + rAF date sweep over [previous, current])
└── confirm-time-sync/   model (the decision contract: deferred resolver store + requestTimeSyncConsent)
                         · ui (the 예/아니오 consent modal, i18n copy)

apps/{web,mobile}/src/entities/universe-clock/
├── api/           sync-universe-clock.ts — maps GetUniverse.universeTime → the store (nil ⇒ clear)
└── index.ts       re-exposes the shared store/type + the sync seam
```

The slices are named identically on both apps; every `model`/`api`/`index` file is a byte-identical mirror and the
pure substance lives in `packages/universe` — only the `ui` hosts differ per platform (DOM/Tailwind vs RN
View/Animated; the shared `Dialog`/`Button` primitives already carry the `.native` forks, so no new `*.native.tsx`
sibling was needed). The widget imports the `entities/universe-clock` mirror only to _read_; it never imports `three`
or a visual entity (§3.4) — the acceleration is a DOM/RN veil layer over the canvas, and the scene hears nothing
(the transition's content is neutral; per-frame veil intensity goes through a ref/Animated value, never React state).

### The clock mirror + HUD ([T6])

`entities/universe-clock` mirrors the single clock value: its `api` maps `universeTime` from the `GetUniverse`
response (the same fetch [23] already makes — no new request; every read syncs, an absent universe clears), its store
holds `currentUniverseTime: string | null` (`useUniverseClockStore` in `packages/universe` — day-granular ISO date
strings, the domain mirror's date type end to end). `features/universe-clock-hud` renders it as the persistent
**"우주의 시간"** pill (the last diary date; `tabular-nums`, glass HUD tone). When the clock is nil (empty universe),
the HUD shows the empty-universe line ("첫 일기와 함께 흐르기 시작해요") rather than a date. The HUD updates whenever
`GetUniverse` refetches (e.g. after a launch) — the clock value is authoritative from the server ([30]) — and takes a
sweep-override prop so the acceleration can drive the displayed date while it plays.

### The acceleration choreography ([T2]) — book the slot

On an advance interval `[previous, current]` (non-empty ⇒ the clock moved; `previous = null` is the first-ever launch,
whose interval is never empty):

- **On launch** ([T2] case 1): `LaunchStars` returns `previous_universe_time` / `universe_time` ([30]);
  `advanceAnnouncementFromLaunch` decides the consequence once for both apps — a moving interval announces
  `{ interval, revealNeuronIds }` to the `features/accelerate-time` seam, and the widget enters `accelerating` and
  plays the **time-passing transition**: a restrained radial veil dims the scene (per-frame intensity via a DOM ref /
  `Animated.Value`, never React state) while the HUD date sweeps `previous → current`. **The reveal gate is the
  deferred awaken**: the optimistic insert and the `GetUniverse` invalidate stay immediate (the memory row exists
  behind the veil), and the `newNeuronIds` awaken announce — the launched star's visible entry — fires only when the
  transition completes (accelerate → then the star appears). A **past-dated or same-day** launch (empty interval — no
  time passed) keeps the immediate awaken and plays nothing. **No warning** ([T2] case 1).
- **On recall** ([T2] case 2, Epic C): after consent, Epic C calls `Recall` ([33]), which server-side composes
  `SyncToToday` ([30]) and the recall write in one transaction. Epic C announces the returned sync interval through
  the **same seam** a launch uses; the widget plays the same acceleration over it while revealing the
  already-committed recall result. The animation is a presentation of the authoritative result, not a pre-write step.
- **Reserved slot** ([V8][C8]): the _content_ of "forgetting/consolidation playing through" — stars dimming ([F], Epic
  D), gist stars rising to the neocortex, the replay "pulled-upward" motion ([C]/[E]) — is filled by those epics. Epic B
  plays a **neutral, restrained** time-passing transition and exposes the interval + a choreography seam so Epics D/E
  slot their per-interval visuals in without re-plumbing. An empty interval plays **nothing**.

**The seam is loss-proof.** The announcement store is module-level (it survives the write sheet unmounting) and
merge-on-announce (back-to-back launches union into one sweep); an overlay unmounting mid-play releases its taken
announcement immediately, so an interrupted animation never loses the awaken. The release itself is idempotent: it
lands the clock mirror on the interval end **monotonically** (never behind a newer value a refetch or later release
already wrote — the mirror can't rewind, [I10]) and **unions** the reveal ids into the awaken store instead of
replacing them (a release can't clobber an immediate same-day announce, and the awaken registry keeps re-announced
ids idempotent). The OS reduced-motion preference resolves the sweep immediately.

Animation timing is **presentation**, driven off the interval (a longer time-jump reads as a slightly longer
transition, capped — `UNIVERSE_TIME_ACCELERATION`: 1600ms floor, +90ms/day, 3600ms cap, ≤24 HUD date steps;
code-level constants, the plan-23 camera-rig precedent), and does not block the optimistic data path — a slow or
skipped animation never delays persistence.

### The sync-consent modal ([T2] case 2 / [R1a])

`features/confirm-time-sync` is the reusable consent modal: it states the consequence in restrained i18n copy and offers
**예 / 아니오**. It is **built here, triggered by Epic C**: `recall-flow-ui`'s 회고하기 calls
`requestTimeSyncConsent()` (re-exported as the widget's public affordance) and awaits the decision — the consent store
parks a deferred resolver, the mounted overlay observes it, opens the modal, and settles it. **예** resolves to
"proceed" (Epic C then calls `Recall`, whose server transaction syncs the clock + applies the recall, and this widget
plays the returned interval); **아니오** — and every ambiguous exit: backdrop, ✕, escape, or the overlay unmounting
with the modal open — resolves to "cancel": the recall is abandoned and the clock does not move ([T2] case 2). A
second request while the modal is open shares the same decision. The modal never itself calls the backend; it returns
a decision, keeping the recall composition Epic C's.

### Overlay control-state (XState, §3.2)

`packages/universe/src/universe-time.machine.ts` holds control-state only (context is empty — the strictest form of
the context rule):

| State          | Meaning                                             | Exits                                                                                                                                                      |
| -------------- | --------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `idle`         | HUD showing; nothing animating                      | `CONFIRM_SYNC` → `confirming`; `ADVANCED` (guarded: the interval moves) → `accelerating`                                                                   |
| `confirming`   | sync-consent modal open (opened by Epic C recall)   | `ACCEPT` → `idle` (the decision resolves "proceed"; the committed recall interval later arrives via the same `ADVANCED` seam and plays); `REJECT` → `idle` |
| `accelerating` | playing the acceleration over `[previous, current]` | `DONE` → `idle` (the reveal releases)                                                                                                                      |

The advance **interval** and the clock value live in Zustand (data); the playing announcement is parked in a widget
ref between `ADVANCED` and `DONE` (§3.2: payload outside machine context). The acceleration drives no scene effect —
the veil is a DOM/RN layer whose per-frame intensity goes through a ref/`Animated.Value`, and the HUD date re-renders
at most once per sampled date; a scene effect added later reads the machine via `getSnapshot()` in `useFrame`, never
React state (§3.2). An empty interval never enters `accelerating` (its reveal releases immediately).

### Copy (i18n) — literary, restrained Korean

All strings resolve through the i18n seam ([08.i18n-seam](08.i18n-seam.md)); none hardcoded (`lint:raw-strings`). The
keys: `universe_time_hud_label` ("우주의 시간"), `universe_time_hud_empty` ("첫 일기와 함께 흐르기 시작해요"),
`universe_time_sync_consent_title` ("우주의 시간"), `universe_time_sync_consent_body` (the consequence, verbatim:
"회상하려면 우주 시간을 오늘로 맞춰야 해요. 그 사이 안 쓴 과거 날짜의 일기는 이후 추가할 수 없게 됩니다.
진행할까요?"), `universe_time_sync_accept` ("예") / `universe_time_sync_reject` ("아니오"); the dialog's close
affordance reuses `common_dismiss`. The restrained literary voice (PRD [§3.1](../PRD.md)) — the consent especially is
plain and honest (state the consequence, ask, no decoration). Web + mobile draw the **same** message keys.

## Acceptance Criteria

1. A persistent **"우주의 시간" HUD** on the universe page shows the current universe time (= the last diary date) read
   from `GetUniverse.universe_time` via `entities/universe-clock`; when the clock is nil (empty universe) it shows an
   empty-universe affordance, not a date ([T6]).
2. `entities/universe-clock` is a **pure** mirror (no three) with a `model` (type + store) and an `api` that maps
   `GetUniverse.universe_time`; it is shared verbatim web + mobile.
3. On a **launch that advances the clock**, an **acceleration choreography** plays over `[previous_universe_time,
universe_time]` (from the `LaunchStars` response) and the launched star is revealed **after** it (accelerate → then
   the star appears); a **past-dated** launch (empty interval) plays **no** acceleration ([T2] case 1).
4. The acceleration is a **neutral time-passing transition with a reserved slot** for forgetting-dimming ([F]) and
   gist-rising / replay ([C]/[E], [V8][C8]); it exposes the interval + a choreography seam those epics fill without
   re-plumbing, and never blocks or delays the optimistic data path.
5. A reusable **sync-consent modal** states the consequence in i18n copy with **예 / 아니오** ([T2] case 2 / [R1a]); it
   returns a decision only (it calls no backend). Its **live trigger (회고하기) is Epic C** (`recall-flow-ui` composes
   it); **예** ⇒ proceed (recall syncs + the acceleration plays), **아니오** ⇒ cancel with the clock unmoved.
6. The overlay's **control-state** is one XState machine (`idle → confirming → accelerating → idle`); the clock value +
   advance interval live in **Zustand/Query**, not machine context; per-frame scene effects read via `getSnapshot()`,
   never React state (§3.2).
7. The overlay exposes **no** rewind, meaning-layer, or placement control — it only shows time, plays a forward-only
   acceleration, and consents to a sync ([I10][I11]); recall-sync/acceleration never mutates a `Diary` ([I2]).
8. **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in a literary /
   restrained Korean voice.
9. **web + mobile parity:** the same widget + three features + the entity mirror run on both apps, sharing `model`/`api`
   verbatim; `*.native.tsx` siblings exist **only** for the overlay / modal / HUD primitives (§3.5).
