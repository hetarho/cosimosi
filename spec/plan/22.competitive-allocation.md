# 22. 경쟁적 할당 + 흥분성 (competitive-allocation)

> 새 조각 별은 빈 공간이 아니라 **최근 활성화된(흥분성 높은) 성단**으로 끌려가고, 연결도 그쪽으로 편향된다(~6h 창). 더불어 라이브 force-sim을 배선해 좌표가 그래프에서 진짜로 창발하게 한다. | 범위: FS | 선행: 21 | 차단: 27

## 목적 (Why)
지금까지 새 별은 의미 유사성(KNN cos_sim)만으로 이웃과 연결되고, 좌표는 `fibonacciStarPosition`(결정론적 정적 셸)로 박혀 있었다 — "연결에서 창발"한다는 concept §결정1·원칙3이 아직 비유에 그쳤다. 본 스펙은 신경과학의 **경쟁적 할당**(엔그램 형성은 흥분성 기반의 *경쟁*; concept §결정2)을 실제 기제로 박는다: 일기를 쓰기 직전 자주 떠올리거나 막 기록한 성단은 "흥분성"이 높아, 그 시간 창(~6h) 안에 들어온 **새 조각 별을 더 잘 포섭**한다. 서버는 흥분성 점수를 buildLinks 단계에서 계산해 연결 선택을 편향하고(서버가 그래프 권위 — 원칙3), 클라는 라이브 force-sim에 새 조각을 **가장 뜨거운 성단 중심 근처에 시드**해 좌표가 그래프에서 진짜로 빚어지게 한다(기존 별은 pin). 이로써 "요즘의 나"가 새 기억이 어디 깃들지에도 손을 댄다(concept §우주의 배경).

## 범위
- **포함:**
  - **BE 흥분성 점수.** worker `buildLinks` 단계에서 성단별 흥분성 `e(c,t)=Σ exp(-(t-t_event)/TAU_EXC)`(`TAU_EXC=6h`, 최근 회상·기록 이벤트 누적)을 계산. 클러스터는 KNN 이웃의 기존 시냅스 연결성분(connected component)으로 서버에서 도출(별도 cluster 컬럼 없이 파생 — 아래 설계).
  - **BE 편향 연결 선택.** 후보를 `k=8`보다 넉넉히 가져와(`KnnNearest` LIMIT 상향) `score = semantic_cos + W_EXC·norm_e(neighbor_cluster)`(`W_EXC=0.25`)로 재정렬해 상위 `k=5`만 링크. **소프트 억제**(soft inhibition): 한 성단이 그날 조각을 흡수할수록 그 성단의 `e`를 내려 한 성단 독식 방지(Delamare/Clopath 항).
  - **FE 라이브 force-sim 배선.** `UniverseCanvas`가 라이브 좌표 `Float32Array` 버퍼를 만들고, 순수 force-sim(`shared/lib/force-sim`, three 미의존 — 원칙4)을 `useFrame`로 펌프하며, **같은 버퍼를 네 readers 전부**(StarField `positionsRef` + UniverseSynapses + FlyToController + Focus)에 공급. 새 조각은 **argmax-e 성단 중심 근처**에 시드. 기존 별 pin, 신규 조각 + 직접 이웃만 relax.
- **비목표(Out of scope):**
  - 앰비언트 무드 gain(`g=1+0.3·arousal`이 `W_EXC`를 스케일) → **25(ambient-mood)**. 여기서는 `W_EXC` 상수.
  - 재공고화 재성형·변천사 → **23/24**.
  - 야간 전체 재안정화·재분배·가지치기 → **27(nightly-consolidation)**.
  - 조각화(1 일기 → N 조각 별) 자체 → **21(memory-fragmentation)**. 본 스펙은 21이 만든 조각 별·`extract`/`embed` 잡 흐름을 **전제**로 그 위에 할당 편향만 얹는다.
  - 관련성 가중 감쇠(λ_eff) → **26**.

## 설계 요점

### 흥분성을 어떻게 저장할까 — 새 컬럼 없이 파생 (권장)
흥분성 `e(c,t)`는 **이미 있는 타임스탬프에서 전부 파생**된다: 별의 마지막 활성 시각은 `memories.last_recalled_at`(11/12에서 운영), 시냅스의 마지막 공동활성은 `memory_links.last_activated_at`(11). 따라서 **별도 `excitability` 컬럼·테이블을 추가하지 않는다** — buildLinks가 후보 별과 그 성단의 위 타임스탬프를 읽어 그 자리에서 `Σ exp(-(t-t_event)/TAU_EXC)`를 계산한다. (장점: 마이그레이션 0건, 단일 출처 유지, 시간 경과에 따라 자동 감쇠. 비용: 매 buildLinks에서 성단 멤버 타임스탬프 조회 — 조각 fan-out 규모에서 무시 가능.) `TAU_EXC=6h`는 반감기 ~4h라 **3h 전 활성 성단은 강하게 co-할당, 24h 전 성단은 사실상 0**(아래 1.2).

### BE 흥분성 + 편향 연결 선택 (`buildLinks`, 순수 헬퍼 — 단위 테스트)
KNN은 의미 후보만 준다. 그 후보들을 흥분성으로 재정렬·선별한다.
```go
// worker.go — 새 상수
const (
    tauExc       = 6 * time.Hour // 흥분성 시간 상수(반감기 ≈ 4h)
    wExc         = 0.25          // 흥분성 편향 가중 (score = cos + W_EXC·norm_e)
    biasedK      = 5             // 편향 재정렬 후 최종 링크 수 (≤ knnK 후보)
    candidateK   = 16            // KnnNearest 후보 상한(=knnK*2): 재정렬 여지 확보
    inhibitDecay = 0.5           // soft inhibition: 한 조각 할당당 성단 e를 ×(1-?) 로 감쇠
)

// excitability: 한 성단의 이벤트 타임스탬프들로 e(c,t)=Σ exp(-(t-t_event)/TAU_EXC).
// t_event = 성단 멤버 별의 last_recalled_at + 멤버 간 시냅스 last_activated_at.
func excitability(now time.Time, events []time.Time) float64 {
    var e float64
    for _, ev := range events {
        dt := now.Sub(ev).Hours()
        if dt < 0 { dt = 0 }
        e += math.Exp(-dt / tauExc.Hours())
    }
    return e
}

// biasedLinks: KNN 후보(candidateK개)를 score=cos + W_EXC·norm_e(neighbor_cluster)로
// 재정렬해 상위 biasedK만 링크. norm_e = e / (max_e over candidate clusters), 0..1.
// soft inhibition: 한 성단에 조각이 할당될 때마다 그 성단의 e를 ×inhibitDecay 해 다음
// 조각이 같은 성단을 독식하지 못하게 한다(Delamare/Clopath 경쟁 항).
func biasedLinks(selfID, userID string, selfDate time.Time, now time.Time,
    cands []Neighbor, clusterOf map[string]string, clusterE map[string]float64) []LinkUpsert
```
- **클러스터 도출(서버, cluster 컬럼 없이).** buildLinks는 후보 별들의 기존 시냅스(`ListLinksByUser` 범위)로 연결성분을 묶어 `clusterOf[memoryID]=clusterID`를 만든다(union-find, 의미 그래프의 약한 클러스터링). 고립 후보는 자기 자신이 한 성단. (정밀 군집은 27 야간 배치 소관 — 여기선 buildLinks 한정 경량 파생.)
- **점수·억제.** `norm_e`는 후보 성단들 중 최대 `e`로 정규화(0..1). `score`로 내림차순 정렬 → 상위 `biasedK` 채택. 채택할 때마다 그 성단의 `clusterE` 항목을 `×inhibitDecay` → 다음 후보가 같은 성단이면 점수가 깎여 다른 성단도 기회를 얻는다(소프트 억제). 최종 `LinkUpsert.Weight`는 기존 `initialWeight(cos, temporalBonus)` 그대로(흥분성은 **선택 편향**이지 가중치 자체는 의미·시간이 정한다 — concept §"연결의 방향·상대는 임베딩이 정한다").
- 흥분성이 0이거나 후보 성단이 하나뿐이면 `biasedLinks`는 순수 cos_sim 정렬(=기존 동작)로 자연 폴백.

### SQL — KNN 후보 상향 + 이웃 클러스터 입력 (`embedding.sql`)
```sql
-- name: KnnNearest :many  (수정: LIMIT @k 를 candidateK 로 호출 — 쿼리문은 그대로,
-- 호출부 k만 16으로. 재정렬 여지 확보. cos_sim ≥ τ=0.75 / user 격리 불변.)
-- (행은 그대로: memory_id, cos_sim, entry_date — 클러스터는 ListLinksByUser로 서버 도출)

-- name: ListLinksForCluster :many  (신규, link.sql) — 후보 별들의 기존 시냅스(연결성분 도출용).
-- 후보 + 그 1-hop 이웃의 a_id/b_id/last_activated_at만; user 격리. (cluster 컬럼 없이 union-find 입력)
SELECT ml.a_id, ml.b_id, ml.last_activated_at
FROM memory_links ml
WHERE ml.user_id = @user_id
  AND (ml.a_id = ANY(@ids::text[]) OR ml.b_id = ANY(@ids::text[]));

-- name: ListLastRecalled :many  (신규, memory.sql) — 후보 별의 last_recalled_at(흥분성 이벤트).
SELECT m.id, m.last_recalled_at FROM memories m
WHERE m.user_id = @user_id AND m.id = ANY(@ids::text[]);
```
> `GraphStore`에 `ListLinksForCluster`·`ListLastRecalled`(또는 둘을 합친 `LoadExcitabilityInputs`)를 추가하고 `handle`이 KNN 후보 id로 한 번 조회 → `biasedLinks`에 주입. `BatchUpsertLinks`는 그대로(가중치·정규화 불변).

### FE 라이브 force-sim 배선 (`UniverseCanvas` — 네 readers 단일 버퍼)
07이 만든 순수 `createSim`/`tick`/`positions`는 좌표 `Float32Array`만 낸다(three 미의존). 22는 이걸 **UniverseCanvas에서 펌프**하고, 결과 버퍼를 모든 좌표 소비자가 공유하게 한다.
```ts
// widgets/universe-canvas/ui/UniverseCanvas.tsx (개념)
// 1) stars+edges → SimGraph(노드 pinned=기존 별; 신규 조각 pinned=false). 신규 조각 초기 좌표 =
//    argmax-e 성단(=새 조각의 최강 이웃이 속한 성단) centroid ± 지터(top-N hot 클러스터 blend).
// 2) const positionsRef = useRef<Float32Array|null>(null)  // 단일 라이브 버퍼
// 3) useFrame: tick(sim, steps); positionsRef.current = positions(sim)  // settled면 펌프 중단
// 4) 같은 positionsRef 를 네 readers 전부에:
//    <StarField positionsRef={positionsRef} .../>
//    <UniverseSynapses positionsRef={positionsRef} .../>   // posById 를 버퍼 인덱스로
//    <FlyToController positionsRef={positionsRef} .../>     // 타깃을 버퍼에서
//    <FocusController positionsRef={positionsRef} .../>     // 프레이밍 타깃을 버퍼에서
```
- **⚠️ 네 readers가 어긋나면 카메라가 별을 놓친다.** 현재 StarField(dummy)·UniverseSynapses·FlyToController·FocusController가 **각자 `fibonacciStarPosition(i, n, seed)`**를 호출한다. 라이브 버퍼로 바꾸려면 **네 곳을 동시에** 같은 버퍼·같은 인덱싱(stars 배열 위치 = 버퍼 슬롯, 07/08 계약)으로 교체해야 한다. 한 곳이라도 정적 fibonacci를 남기면 fly-to/focus가 화면에 그려진 별이 아닌 옛 좌표로 날아가 "별을 못 만나는" 버그가 난다.
- **버퍼 미수렴/미연결 폴백.** force-sim이 아직 좌표를 못 낸 초기 프레임엔 `fibonacciStarPosition`을 시드 초기값으로 써 깜빡임 없이 시작(07의 신규 노드 초기 좌표 규칙과 동일 정신). settled 후엔 정적이므로 매 프레임 재업로드 없음(StarField는 이미 버퍼 없을 때 no-op).
- **pin 정책(원칙3).** 기존 별은 `pinned=true`(좌표 불변, 척력 소스로만 참여) → "내 기억이 거기 있다"가 깨지지 않음. 신규 조각 + 그 1-hop 이웃만 free. 새 조각이 hot 성단 centroid 근처에서 출발하므로 인력이 그쪽으로 빚는다(시각적 경쟁적 할당).
- **순수성(원칙4).** 좌표 시드/centroid 계산 보조(예: `seedNearCluster(graph, hotClusterIds): SimNode[]`)는 `shared/lib/force-sim` 또는 `shared/lib/layout.ts`에 **three/DOM 미의존**으로 둔다. UniverseCanvas(ui)만 three에 닿는다.

## 수용 기준 (EARS / Given-When-Then)
1.1 GIVEN 새 조각 별의 KNN 후보 성단 중 하나가 최근(≤TAU_EXC) 활성화되었을 때 WHEN worker가 링크를 선택하면 THE SYSTEM SHALL `score = semantic_cos + W_EXC·norm_e(neighbor_cluster)`(W_EXC=0.25) 상위 `biasedK=5`를 채택해, 의미 동률이면 **더 흥분성 높은 성단**을 우선 연결한다. _(헌법3 — 서버가 그래프 권위)_
1.2 GIVEN 동일 의미 후보 두 성단 중 A는 3h 전, B는 24h 전 활성화 WHEN 흥분성을 계산하면 THE SYSTEM SHALL A의 `e`(≈exp(-3/6·ln2 보정)·강함)가 B(≈0)보다 명백히 크게 나와 A로 편향하고, **하루(24h) 지난 성단의 편향은 사실상 0**이 되게 한다. _(과학적 근거 — ~6h 창)_
1.3 GIVEN 한 일기의 여러 조각이 같은 hot 성단에 후보로 몰릴 때 WHEN 차례로 할당하면 THE SYSTEM SHALL **소프트 억제**(할당당 성단 `e`×`inhibitDecay`)로 한 성단이 모든 조각을 독식하지 못하게 분산한다. _(경쟁/Delamare-Clopath)_
1.4 GIVEN 후보 성단이 하나뿐이거나 흥분성이 전부 0일 때 WHEN 링크를 선택하면 THE SYSTEM SHALL 순수 `cos_sim` 정렬(기존 buildLinks 동작)로 폴백하고 throw하지 않는다.
1.5 WHEN 흥분성 입력을 모을 때 THE SYSTEM SHALL `memories.last_recalled_at`·`memory_links.last_activated_at`에서 **파생**하고 별도 excitability 컬럼을 추가하지 않는다(마이그레이션 0건). _(단일 출처)_
1.6 WHEN 새 조각이 우주에 떠오를 때 THE SYSTEM SHALL 라이브 force-sim에서 그 조각을 **argmax-e 성단 centroid 근처**에 시드해 빈 공간이 아니라 hot 성단 곁으로 끌리게 한다(좌표 창발). _(헌법3, concept §결정2)_
1.7 WHEN 라이브 좌표 버퍼가 갱신될 때 THE SYSTEM SHALL **StarField·UniverseSynapses·FlyToController·FocusController 네 readers가 동일 버퍼·동일 인덱싱**을 읽어, 카메라 fly-to/focus가 화면에 렌더된 바로 그 별 위치로 도달하게 한다(어긋남 0). _(§3.2-2)_
1.8 GIVEN 기존(settled) 별 WHEN force-sim이 tick될 때 THE SYSTEM SHALL 기존 별 좌표를 pin(불변)하고 신규 조각 + 그 1-hop 이웃만 이동시킨다. _(헌법3 pin 제약)_
1.9 WHEN 별·시냅스가 다른 성단으로 재배치/탈락되어도 THE SYSTEM SHALL 어떤 별·시냅스도 삭제하지 않는다(밝기/위치만 변함). _(헌법2)_
1.10 WHEN `shared/lib/force-sim`·신규 시드 헬퍼·synapse model을 빌드하면 THE SYSTEM SHALL `three`/React/DOM을 import하지 않는다(모바일 재사용). _(헌법4)_
1.11 WHEN 데모(체험)에서 한 성단을 회상한 직후 새 기억을 쓰면 THE SYSTEM SHALL 새 별이 라이브 force-sim으로 **그 성단 쪽으로 눈에 띄게 끌려가고**, 일정 시간(>TAU_EXC) 기다린 뒤 쓰면 끌림이 사라지게 한다. _(체험 — 시간 창 시연)_
