# 04. memory-api — RecordMemory · GetUniverse (저장·조회)

> 일기 저장(원본 불변 + 별 + job 적재)과 우주 조회(전체 가중치 그래프) RPC를 실제 DB로 구현. | 범위: MVP | 선행: 02, 03 | 차단: 05, 10, 11, 12

## 목적 (Why)
사용자가 일기를 쓰면 **원본을 불변으로 영구 저장**하고(원칙1), 동시에 별(`memories`) 행을 만들고, 임베딩/연결 생성을 비동기로 넘길 **job을 enqueue**한 뒤 `memoryId`를 즉시 돌려준다. 또 우주 화면이 그릴 **전체 그래프(별 + 시냅스 + last_*_at, 잠든 것 포함)** 를 한 번에 반환한다. 02가 깐 빈 Connect 핸들러를 실제 sqlc/pgx로 채워 코어 루프의 백엔드 절반을 완성한다.

## 범위
- **포함:**
  - `internal/entry/*` → `internal/memory/*`로 진화(엔티티 정리). 순수 도메인(`memory.go`), 소비자 측 인터페이스(`repository.go`), sqlc gen 매핑 + 동적 쿼리만 얇은 pgx(`repository_pg.go`), 비즈니스 정책(`service.go`), proto↔도메인 매핑 Connect 핸들러(`handler.go`).
  - `internal/link/`의 **그래프 read 일부**(`GetUniverse`가 쓰는 `memory_links` 조회만). 강화/감쇠 로직은 비목표.
  - `db/queries/{memory.sql, link.sql(read만)}` 작성 → `sqlc generate`로 `internal/db/gen` 산출.
  - **RecordMemory**: `records`(불변) insert + `memories` insert + `jobs` enqueue → `memoryId` 반환(한 트랜잭션).
  - **GetUniverse**: 인증 사용자의 모든 `memories` + 모든 `memory_links`(+ `last_recalled_at`/`last_activated_at`) 반환 — 잠든 별·시냅스 포함.
  - `cmd/api/main.go` 와이어링(memory/link 서비스 + 핸들러 마운트), 기존 `internal/entry/*` 삭제.
- **비목표(Out of scope):**
  - 임베딩 생성·top-k 최근접 연결 **생성** → 05.embedding-worker. 04는 `jobs` 행만 적재하고 끝.
  - 회상·공동 회상 **강화**(weight 증가, `last_*_at` 갱신) → 11.recall-reinforce.
  - 활성도 **감쇠**·`a_min`·잠든 별 탐색 RPC(`ListDormant`) → 12.decay-dormant. (04는 `last_*_at` raw 값만 반환, 밝기 계산은 클라.)
  - `ai.Extractor`·`visual_spec` LLM 채움 → v1(24). 04는 `visual_spec`을 INSERT하지 않는다(03에서 nullable, MVP 미사용 — FE가 memory_id seed 파생).
  - proto 정의·Connect 서버 부트스트랩·auth 인터셉터(`userID` 추출) → 02.rpc-contract. 04는 02가 만든 `MemoryService` 핸들러 인터페이스를 **구현만** 한다.
  - 스키마/마이그레이션(`records`,`memories`,`memory_links`,`embeddings`,`jobs` 테이블·HNSW) → 03.data-schema.

## 설계 요점
**도메인(순수, 태그 금지 — 원칙5).** `internal/memory/memory.go`:
```go
type Mood string // "joy"|"calm"|"sad"|"anger"|"fear"|"love"|"neutral" (7종, proto Mood enum과 매핑)
// RecordInput: RecordMemory가 records(불변)에 쓰는 원본 입력. 별 도메인과 분리(헌법1).
type RecordInput struct {
    UserID         string
    Body           string    // 일기 원본 — records에 영구 보관
    EntryDate      time.Time  // 사용자 지정 시점(기본 오늘)
    Mood           Mood      // 선택(null 허용)
    Intensity      float64   // 0~1, 선택(기본 0)
    IdempotencyKey string    // 선택(멱등 키, 비면 미적용)
}
// Memory: 별(GetUniverse Star 투영). mood/intensity 출처는 records(JOIN). body/entry_date 없음.
type Memory struct {
    ID         string         // = memory_id
    Mood       Mood           // records에서 JOIN
    Intensity  float64        // records에서 JOIN
    LastRecalledAt *time.Time // 회상 활성도용(GetUniverse가 반환, 04는 미갱신)
}
type Synapse struct { // link 기능 도메인(read 부분만 04 소관)
    AID, BID          string
    Weight            float64
    LinkType          string
    CoActivationCount int
    LastActivatedAt   *time.Time
}
type Universe struct { Memories []Memory; Synapses []Synapse }
```
> ⚠️ 도메인 타입에 `json:`/`db:`/proto 태그 절대 금지. proto·sqlc row 타입은 인프라; 핸들러가 proto↔도메인, repository_pg가 row↔도메인 매핑.

**인터페이스(소비자=service 측).** `internal/memory/repository.go`:
```go
type Repository interface {
    // 한 트랜잭션: (멱등 체크) → records insert(불변) → memories insert → jobs enqueue
    RecordMemory(ctx context.Context, in RecordInput) (memoryID string, err error)
    ListByUser(ctx context.Context, userID string) ([]Memory, error)
}
```
`internal/link/repository.go`(read만): `ListByUser(ctx, userID) ([]memory.Synapse, error)` — `memory_links.user_id = $1` 스코프.

**service 정책.** `memory.Service.RecordMemory`: ID 생성(서버 권위 — 클라는 ID 미생성, 원칙/§7), `EntryDate` 비면 `time.Now().UTC()`, repo에 위임(record→memory→job 트랜잭션은 repository_pg가 수행). `visual_spec`은 MVP 미사용(03에서 nullable, INSERT 안 함). `GetUniverse(ctx, userID)`는 `memory.ListByUser` + `link.ListByUser`(service↔service, 인터페이스 의존)를 합쳐 `Universe` 반환.

**handler(proto↔도메인, 얇게).** `RecordMemory(ctx, req)`: `rpcserver.UserIDFromContext(ctx)` → 없으면 `connect.NewError(connect.CodeUnauthenticated, …)`. proto req → `RecordInput`(UserID 주입, proto `Mood` enum→도메인 `Mood` 7종 매핑, `entry_date` YYYY-MM-DD 파싱, `idempotency_key` 전달) → `svc.RecordMemory` → `RecordMemoryResponse{memory_id}`. `GetUniverse(ctx, req)`: userID 추출 → `svc.GetUniverse` → `GetUniverseResponse{stars[], synapses[]}`(별마다 `mood`/`intensity`/`last_recalled_at`, 시냅스마다 `weight`/`last_activated_at` 포함 — **밝기 계산은 클라**, 원칙2·3).

**repository_pg(sqlc gen + 트랜잭션).** `RecordMemory`는 `pool.Begin` → `gen.New(tx)`로 (1) `idempotency_key`가 있으면 `FindMemoryByIdempotencyKey`로 기존 `(user_id, key)` 조회 → 있으면 그 `memory_id`를 반환(멱등, INSERT 생략, commit). 없으면 (2) `record_id` 생성 → `InsertRecord`(records, 불변) → (3) `memory_id` 생성 → `InsertMemory`(memories.record_id = 위 record_id) → (4) `EnqueueJob`(jobs.memory_id = 위 memory_id) 후 commit(원자성). 나머지는 sqlc 정적 쿼리 직접 호출. 동적 정렬/N-홉이 없으므로 04는 raw pgx 거의 불필요(정적 쿼리만).

**SQL.** `db/queries/memory.sql` (record→memory→job 순서, FK는 memories.record_id → records.id):
```sql
-- name: FindMemoryByIdempotencyKey :one
SELECT m.id AS memory_id
FROM records r JOIN memories m ON m.record_id = r.id
WHERE r.user_id = $1 AND r.idempotency_key = $2;   -- 멱등: 기존 memory_id 반환
-- name: InsertRecord :exec
INSERT INTO records (id, user_id, body, entry_date, mood, intensity, idempotency_key)
VALUES ($1, $2, $3, $4, $5, $6, $7);            -- 불변(UPDATE/DELETE 쿼리 없음), memory_id 컬럼 없음
-- name: InsertMemory :one
INSERT INTO memories (id, user_id, record_id)
VALUES ($1, $2, $3) RETURNING id;               -- record_id NOT NULL FK 필수, entry_date/mood/intensity 없음
-- name: EnqueueJob :exec
INSERT INTO jobs (id, memory_id, kind, status) VALUES ($1, $2, 'embed', 'pending');
-- name: ListMemoriesByUser :many
SELECT m.id AS memory_id, r.mood, r.intensity, m.last_recalled_at
FROM memories m JOIN records r ON r.id = m.record_id
WHERE m.user_id = $1 ORDER BY m.created_at;   -- 잠든 것 포함(WHERE 밝기 필터 없음, 원칙2); mood/intensity 출처는 records
```
`db/queries/link.sql`(read만):
```sql
-- name: ListLinksByUser :many
SELECT ml.a_id, ml.b_id, ml.weight, ml.link_type, ml.co_activation_count, ml.last_activated_at
FROM memory_links ml
WHERE ml.user_id = $1;
```
> records에 UPDATE/DELETE 쿼리를 **절대 만들지 않는다**(원칙1). memories에도 04 범위에선 mutate 쿼리 없음(강화/감쇠는 11·12).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 인증된 사용자가 `RecordMemory(body, entry_date?, mood?, intensity?, idempotency_key?)`를 호출 THE SYSTEM SHALL 단일 트랜잭션으로 record→memory→job 순서로 `records` 1행(불변)·`memories` 1행(`record_id` = 그 record id)을 insert하고 `jobs`에 `memory_id` = 그 memory id로 `kind='embed', status='pending'` 1행을 enqueue한 뒤 새 `memory_id`를 응답한다.
1.2 IF 인증 컨텍스트에 `userID`가 없으면 THEN THE SYSTEM SHALL `connect.CodeUnauthenticated` 오류를 반환하고 아무 행도 쓰지 않는다.
1.3 IF 트랜잭션 중 어느 insert가 실패하면 THEN THE SYSTEM SHALL 롤백하여 `records`/`memories`/`jobs` 어디에도 부분 행을 남기지 않는다.
1.4 WHEN `entry_date`가 비어 호출되면 THE SYSTEM SHALL `now()`(UTC) 날짜를 `records.entry_date`에 저장한다(mood/intensity 비면 null/0, `visual_spec`은 INSERT 안 함 — 03 nullable).
1.5 WHEN `idempotency_key`가 주어지고 같은 `(user_id, idempotency_key)`의 record가 이미 존재하면 THE SYSTEM SHALL 새 행을 만들지 않고 기존 `memory_id`를 그대로 반환한다(멱등).
2.1 WHEN 인증된 사용자가 `GetUniverse`를 호출 THE SYSTEM SHALL 그 사용자의 **모든** `memories`(`record_id`로 `records` JOIN해 mood/intensity 채움, 밝기·활성도와 무관, 잠든 별 포함)와 `user_id`가 일치하는 **모든** `memory_links`를 반환한다(모든 쿼리 `WHERE user_id=$ctx` 스코프).
2.2 THE SYSTEM SHALL 각 별에 `last_recalled_at`, 각 시냅스에 `weight`·`last_activated_at`을 **raw 값 그대로** 포함하고, 활성도/밝기/좌표는 계산하지 않는다(밝기=클라 렌더, 좌표=클라 force-sim — 원칙2·3).
2.3 IF 인증 컨텍스트에 `userID`가 없으면 THEN THE SYSTEM SHALL `connect.CodeUnauthenticated`를 반환한다.
3.1 THE SYSTEM SHALL `records`에 대한 UPDATE/DELETE 경로(쿼리·핸들러·service 메서드)를 일절 제공하지 않는다(원본 불변·원칙1).
3.2 THE SYSTEM SHALL 도메인 타입(`Memory`,`Synapse`,`Universe`)에 `json:`/`db:`/proto 태그를 두지 않는다(원칙5).

## 완료 정의 (DoD)
- `docker run --rm -v ${PWD}/backend:/src -w /src golang:1.26 go build ./...` 가 통과한다(entry 잔재 없음, gen 컴파일됨).
- Connect 클라(buf curl 또는 02의 TS/Go 클라)로 **왕복**: 인증 토큰으로 `RecordMemory{body:"오늘…"}` → `memory_id` 수신 → `GetUniverse` 호출 시 응답 `stars[]`에 그 `memory_id`가 포함된다.
- DB 검증: 위 호출 후 `records`·`memories`·`jobs` 각 테이블에 1행씩 생기고, `jobs.status='pending'`·`jobs.kind='embed'`이다.
- 인증 없는 `RecordMemory`/`GetUniverse` 호출이 `Unauthenticated`로 거부되고, 그때 `records` 행이 증가하지 않는다.
- `db/queries/*.sql`·`internal/memory/*`·`internal/link/*` 어디에도 `records` UPDATE/DELETE가 없고(`rg "UPDATE records|DELETE FROM records"` 무결과), 도메인 타입에 struct 태그가 없다.
