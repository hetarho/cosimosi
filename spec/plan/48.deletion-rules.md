# 48. deletion rules

> Deletion rules — the domain model for the only two ways a memory ever leaves the universe: **full delete** (soft-delete
> every star born from the diary + seal their neurons/activations, keep shared neurons, weaken each removed memory's
> synaptic contribution) and
> **letting-go** (seal the semantic neurons only — a content-less silent engram). Sealing/soft-delete is the live-graph
> mechanism; system dynamics never delete ([I1]), while the user's full-delete path may hard-delete after the restore
> window ([X2]). Sealed things are excluded from **all** dynamics ([X3]).
> Scope: BE | Status: planning
>
> Layer D (domain) — the first unit of Epic H (Deletion & Letting-go). Home: the core-domain `internal/memory` context —
> the pure sealing/soft-delete predicates + the neuron-classification rule, with the exclusion filters that live in
> `memory/pg`'s read/compute queries. It **activates** the `deleted_at` / `sealed_at` columns that
> [16.memory-aggregate-schema](16.memory-aggregate-schema.md) reserved and the `Depress` (LTD) seam that
> [18.synapse-plasticity](18.synapse-plasticity.md) defined. The _flows_ (the delete/restore transaction, the 30-day
> sweep, the AI-suggest→approve letting-go loop) are the use-case [49.release-usecase](49.release-usecase.md); the delete
> UI + reassurance copy are [50.deletion-ui](50.deletion-ui.md).

## Purpose

cosimosi's founding promise is that **the system does not delete as forgetting** — forgetting is dimming, not deletion,
and the trace survives at the floor as a silent engram ([I1][F3]). The single exception is the user's own explicit choice
to remove a diary. This unit defines what that choice _means_ to the model, precisely enough that the
removal honors the same neuroscience the engine does: shared neurons (little stars belonging to _many_ memories) are
never destroyed, and each removed memory only _withdraws its own synaptic contribution_.

There are exactly two removals, different in kind:

- **Full delete** ([X1][X2]) — "I want this whole diary gone." Soft-delete **all** `EpisodicMemory` rows born from that
  `Diary`, seal their memberships, seal the neurons that no live memory outside that diary still uses, keep the shared
  neurons but weaken the removed memories' contribution to their synapses. Reversible for 30 days.
- **Letting-go** ([X4][X5]) — "I want to keep the star but drain what it means." Seal the memory's **semantic** neurons
  only; every spatial/entity/emotion/color/star fact stays. The star still shines its emotion color at its emergent
  position, but its content is hollowed out — a silent engram, symbolic release, not therapy ([X7]). Permanent, no timer.

Both leave a coherent universe behind: once sealed, a fact exerts **no** force, decays for nobody, consolidates into
nothing, and never appears — "no ghost that pulls other stars while invisible" ([X3]). The launch/read loop (Epic A)
already excludes soft-deleted memories and sealed neurons from `GetUniverse`; this unit makes that exclusion **total**
(force-sim, decay, consolidation too) and defines the **write** side (what gets sealed, how the shared-neuron
contribution is weakened) that Epic H turns on.

## Scope / Non-goals

**In scope**

- **The neuron-classification rule** (pure domain predicate): given a set of memories being removed and the activation
  graph, partition their neurons into **orphan** (activated by _no non-sealed, non-deleted memory outside the removal
  set_) vs **shared** (still activated by at least one live memory outside the removal set). This is the rule that
  decides seal-vs-keep, for both removals.
- **Full-delete sealing semantics** ([X1]): for one `Diary`, soft-delete every linked `EpisodicMemory` (`deleted_at`),
  seal their `NeuronActivation` memberships, seal the **orphan** neurons (`sealed_at`), keep the **shared** neurons, and
  **weaken each removed memory's synaptic contribution** to the shared neurons' synapses via LTD (`Depress`,
  [18.synapse-plasticity](18.synapse-plasticity.md)).
- **Letting-go sealing semantics** ([X4]): seal only the memory's **semantic** neurons (with the same orphan/shared
  classification + contribution-weakening applied to _those_ neurons); spatial, entity, emotion, color, and the star
  itself are untouched. The memory is **not** soft-deleted (the star lives).
- **The total exclusion of sealed/soft-deleted facts from ALL dynamics** ([X3]): confirm the already-present
  `GetUniverse` read exclusion, and **extend the same predicate** to force-sim input, read-time decay/effective-value
  computation, and consolidation/replay selection. One canonical "is this alive?" filter, applied everywhere.
- **The [X4] "the star's position/connections naturally recompute"** consequence: because positions are emergent
  ([I5]) and force-sim reads only live neurons/synapses, sealing semantic neurons _automatically_ reshapes the layout —
  no explicit reposition write.
- The single new tuning number this feature introduces: the **LTD contribution-weakening amount** applied on removal
  (enumerated for `values.yaml`).
- The **golden-parity stance** for the sealing/classification math (it is server-authoritative, not a FE↔BE mirror —
  stated explicitly, below).

**Non-goals**

- **No flows / orchestration.** The delete transaction, the 30-day soft-delete → hard-delete **sweep**, restore, the
  permanent letting-go action, and the **AI-suggest → user-approve** semantic-neuron identification loop ([X6]) are all
  [49.release-usecase](49.release-usecase.md). This unit provides the _rules_ those flows execute.
- **No RPC / proto changes.** The `Release` / restore / letting-go RPC surface is [49.release-usecase](49.release-usecase.md);
  `GetUniverse` already filters sealed/deleted facts and needs no message change.
- **No UI.** The delete flow, letting-go flow, export-reassurance copy, and professional-resource notice ([X6][X7][W6])
  are [50.deletion-ui](50.deletion-ui.md).
- **No new `Depress`/`Downscale` math.** `Depress` (LTD, associative/local) already exists
  ([18.synapse-plasticity](18.synapse-plasticity.md)); this unit only _wires the deletion seam_ to it. `Downscale`
  (SHY, sleep/global) is a different mechanism ([I9]) and belongs to Epic E consolidation — never conflated here.
- **No hard-delete on any system dynamic** ([I1][F3]). The only hard delete anywhere is the user-initiated 30-day sweep
  of already-user-soft-deleted release groups (owned by [49.release-usecase](49.release-usecase.md)); no decay,
  consolidation, or force-sim path ever deletes a row.
- **No AI port.** The semantic-neuron _suggestion_ is an AI concern ([X6]) owned by [49.release-usecase](49.release-usecase.md);
  this unit only defines that the _target set_ of a letting-go is "this-memory's semantic neurons," and the domain
  executes the seal.

## Design

### Placement (domain-first Go)

Removal is **core-domain behavior of the memory aggregate** — it operates on `EpisodicMemory`, `Neuron`,
`NeuronActivation`, and `Synapse`, all owned by `internal/memory` — so it lives in `internal/memory`, not a new context
(ARCHITECTURE §2.2; be-architecture: keep tightly-coupled aggregate behavior in one context). Split into layers:

- **Pure domain** (context package, no proto/sqlc/pgx — §2.4): the neuron-classification predicate and the "is this
  fact alive?" predicate. Deterministic, IO-free, testable with no infrastructure.
- **`memory/pg`** (§2.6, the only sqlc/pgx package): the sealing writes (set `deleted_at` / `sealed_at`), the
  contribution-weakening synapse updates (calling `Depress` on the affected edges), and the **exclusion filters** baked
  into every read/compute query. Repository ports are declared by the consumer ([49.release-usecase](49.release-usecase.md),
  §2.4 consumer-owned ports); this unit provides the concrete queries + the classification/exclusion rules they run.

### The neuron-classification rule (pure)

The heart of both removals. Given a removal set `R` (one or more `EpisodicMemory` ids) and the set of neurons those
memories activate, classify each neuron:

```
Classify(neuron, removingMemoryIds, liveActivations) → { orphan | shared }
```

- **orphan** — after all memories in `R` are removed, the neuron is activated by **no live memory outside `R`** (a memory
  that is neither soft-deleted nor whose activation is sealed). → **seal it** (`sealed_at`).
- **shared** — the neuron is still activated by at least one live memory outside `R`. → **keep it**, and weaken only the
  removed memories' synaptic contribution (below).

"Live" is the same predicate as the dynamics-exclusion filter: a memory counts if `deleted_at IS NULL`, and an
activation counts if it is not sealed and its neuron's `sealed_at IS NULL`. **Decided (2026-07):** orphan-ness is
evaluated **as of the removal** against the live memories **outside the removal set**; a neuron used only by the diary's
released memories becomes an orphan the instant that diary is released. The classification is pure — the _data_ (who
else activates this neuron) is supplied by `memory/pg`; the _decision_ is domain math.

This rule is identical for full delete and letting-go; they differ only in **which memories/neurons enter it**: full
delete feeds all neurons from every `EpisodicMemory` born from the diary, letting-go feeds only one memory's approved
semantic neurons.

### Full delete ([X1][X2])

The rule set (the _transaction_ that runs it is [49.release-usecase](49.release-usecase.md)):

1. **Soft-delete the diary's memories** — set `episodic_memories.deleted_at = now()` for every `EpisodicMemory` born
   from the target `Diary`. All of those stars vanish from every read/dynamic immediately; their rows (and
   `current_text`, stage texts, seed) are preserved for 30-day restore ([X2]). The `Diary` is **not** mutated in place
   ([I2]); its body remains intact through the restore window and [49.release-usecase](49.release-usecase.md)'s
   RetentionSweeper may hard-delete it after that window as the user-origin [X2] exception. Its export-reassurance is
   [50.deletion-ui](50.deletion-ui.md).
2. **Seal the memberships** — the removed memories' `NeuronActivation` rows are excluded from the live graph (they hang
   off now-soft-deleted memories, so the exclusion filter already drops them; no separate column is needed — see the
   activation note below).
3. **Classify the union of removed memories' neurons** (rule above). **Orphan** neurons → `sealed_at = now()`. **Shared**
   neurons → kept.
4. **Weaken each removed memory's synaptic contribution** — for every synapse between two neurons co-activated by a
   removed memory where at least one endpoint is a **shared** (kept) neuron, apply LTD:
   `Depress(strength, deletion.contribution_weaken_amount)` ([X1], via
   [18.synapse-plasticity](18.synapse-plasticity.md)). The synapse is **not** deleted — its base strength is lowered, so
   the filament thins/fades but the edge endures ([I1] spirit; shared neurons keep their other bonds).

**Decided (2026-07):** "weaken the removed memory's contribution" is realized as a deterministic `Depress` step for each
removed memory's affected synapse contribution (the synapse stores one cumulative base strength [L5], not a permanent
per-memory ledger — [16.memory-aggregate-schema](16.memory-aggregate-schema.md)), so the contribution is withdrawn by
proportional LTD decrements, not by subtracting stored per-memory shares. Synapses between two neurons that both became
**orphans** need no `Depress` — both endpoints are sealed and already excluded, so the edge leaves the dynamics via the
exclusion filter (weakening a doubly-orphaned edge would be redundant).

### Letting-go ([X4][X5])

"Keep the star, drain the meaning." The rule set:

1. **Target = the memory's semantic neurons only.** Spatial, entity, emotion, color, and the star (`EpisodicMemory`
   itself) are **never** touched ([X4][I11]). The memory is **not** soft-deleted — no `deleted_at`; the star lives.
2. **Classify + seal** exactly as above, but only over the semantic-neuron subset: **orphan** semantic neurons → seal;
   **shared** semantic neurons → keep + `Depress` this memory's contribution on their synapses.
3. **Position/connections recompute for free.** Because positions are emergent ([I5]) and force-sim + centrality read
   only live neurons/synapses ([19.force-sim-layout](19.force-sim-layout.md)), removing semantic neurons from the live
   graph _automatically_ reshapes the star's radius/angle/cluster — this unit writes **no** position ([X4] "위치·연결은
   자연 재계산"). The star keeps its emotion color and shape (seed unchanged), so it still shines — a **content-less
   trace, a silent engram** ([X4], Roy 2017).
4. **Permanent, no timer** ([X5]) — unlike full delete, letting-go has no 30-day restore window (a deliberate, considered
   act). That permanence is enforced by the flow ([49.release-usecase](49.release-usecase.md)); this unit states the
   rule.

**Decided (2026-07):** letting-go seals neurons but leaves the `EpisodicMemory` row fully intact (including its emotion
columns and seed), so the star's [V4] color / [V3] size / [V5] shape are preserved by construction; only content-bearing
semantic neurons and their synaptic pull are withdrawn. There is no "partial `current_text`" rewrite here — the
`current_text` fate on letting-go (if any) is a flow/UI concern, not a domain-sealing rule.

### Total exclusion from ALL dynamics ([X3])

The single most important invariant this unit enforces: **one canonical "alive" predicate, applied at every read and
every compute.** A memory is alive iff `deleted_at IS NULL`; a neuron is alive iff `sealed_at IS NULL`; a synapse is
alive iff **both** endpoints are alive; an activation is alive iff its memory and neuron are both alive.

| dynamic                            | owner                                                                                            | exclusion status                                                                                                                                                                                                         |
| ---------------------------------- | ------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `GetUniverse` read                 | [23.universe-canvas](23.universe-canvas.md)                                                      | **already excludes** `deleted_at`/`sealed_at` on all four sub-queries (as-built, [16.memory-aggregate-schema](16.memory-aggregate-schema.md) partial indexes) — this unit **confirms** it holds for neuron _sealing_ too |
| force-sim layout input             | [19.force-sim-layout](19.force-sim-layout.md)                                                    | force-sim consumes the `GetUniverse` graph, so excluded neurons/synapses never enter layout — **no ghost pull** ([X3]); confirmed                                                                                        |
| read-time decay / effective values | [37.forgetting-decay](37.forgetting-decay.md), [18.synapse-plasticity](18.synapse-plasticity.md) | sealed neurons / soft-deleted memories are **excluded from the decay + effective-value computation set** — this unit **extends** the filter to the compute queries                                                       |
| consolidation / replay             | [41.consolidation-usecase](41.consolidation-usecase.md)                                          | replay-reorg + Downscale select only **live** synapses/neurons — this unit **extends** the filter to the consolidation selection                                                                                         |
| nebula color field                 | [26.nebula-color-field](26.nebula-color-field.md)                                                | color bleed weights only live memories' `EffectiveStrength`; excluded via the same read filter — confirmed                                                                                                               |

**Decided (2026-07):** rather than each dynamic re-deriving its own filter, the exclusion is expressed **once** as the
canonical predicate above and reused — `GetUniverse` already encodes it, and Epics D/E's compute queries (decay,
consolidation) `JOIN`/`WHERE` against the same `deleted_at IS NULL` / `sealed_at IS NULL` conditions. A sibling plan's
compute query that forgets the filter is the failure mode this unit exists to prevent; its acceptance criteria assert the
filter is present in **every** dynamic, not just the read.

### The `NeuronActivation` "sealing" note

[16.memory-aggregate-schema](16.memory-aggregate-schema.md) gives `neuron_activations` no `sealed_at` column of its own.
**Decided (2026-07):** an activation is "sealed" **transitively** — excluded whenever its memory is soft-deleted
(`deleted_at`) or its neuron is sealed (`sealed_at`). Full delete soft-deletes every memory in the diary release set, so
_all_ of their activations drop out through the memory filter; letting-go seals specific neurons, so activations to
_those_ neurons drop out through the neuron filter. This needs **no new column** — the two reserved columns express both
removals' membership exclusion. [49.release-usecase](49.release-usecase.md) owns the short-lived release audit needed
for exact restore within the retention window.

### Golden-parity stance

Sealing/soft-delete and neuron classification are **server-authoritative writes over persisted state**, not read-time
math the client re-derives — so they are **not** a FE↔BE golden-parity mirror (contrast [18.synapse-plasticity](18.synapse-plasticity.md)'s
`Depress`, golden-parity because both runtimes compute effective strength). A removal reaches the client purely through
`GetUniverse`: the sealed/deleted facts stop appearing, and the client's existing force-sim / effective-value math runs
unchanged over the smaller live graph. The client never seals, classifies, or `Depress`es on delete — it only stops
seeing what the server excluded. The one shared quantity, `Depress`, is already parity-pinned in plan 18; the removal
_decision_ is single-writer server state, like the universe clock ([29.universe-clock](29.universe-clock.md)).

### Re-implementation note

The rules are fully specified by four statements: (1) _classify_ each removed neuron orphan-vs-shared against live
memories outside the removal set; (2) _seal orphans, keep shared_; (3) _`Depress` each removed memory's contribution on
shared neurons' synapses by one amount_; (4) _exclude every `deleted_at`/`sealed_at` fact from every dynamic_. Full
delete feeds all neurons from the diary's memories + soft-deletes those memories; letting-go feeds only semantic neurons
and keeps the memory. Any client/runtime reproduces the model from those rules plus the single LTD amount.

## Acceptance Criteria

1. **Neuron classification** is a pure predicate that partitions a removal set's neurons into **orphan** (activated by no
   live memory outside the removal set) and **shared** (still activated by ≥1 live memory outside the removal set),
   evaluated as-of removal; it is deterministic and IO-free (data supplied by `memory/pg`) [X1][X4].
2. **Full delete** is diary-scoped: it soft-deletes every `EpisodicMemory` born from the target `Diary`
   (`episodic_memories.deleted_at = now()`), seals the removal set's **orphan** neurons (`neurons.sealed_at = now()`),
   and **keeps** its **shared** neurons; no neuron, memory, synapse, or activation row is hard-deleted by this operation
   [X1][X2][I1].
3. **Full delete weakens each removed memory's synaptic contribution** to shared neurons' synapses via
   `Depress(strength, deletion.contribution_weaken_amount)` ([18.synapse-plasticity](18.synapse-plasticity.md)); the
   synapse's base strength is lowered but the edge is **never deleted**, and shared neurons retain their other bonds
   [X1][I6].
4. **Letting-go** targets **semantic neurons only** — spatial, entity, emotion, color, and the star (`EpisodicMemory`
   incl. its emotion columns + seed) are untouched, and the memory is **not** soft-deleted (the star lives); orphan
   semantic neurons are sealed, shared ones kept + contribution-weakened [X4][I11].
5. After letting-go, the star's position/connections **recompute naturally** with **no** position write — force-sim reads
   only live neurons/synapses, so removing semantic neurons reshapes layout automatically; the star still renders its
   emotion color + shape (a content-less silent engram) [X4][I5].
6. **Sealed and soft-deleted facts are excluded from ALL dynamics** — the `GetUniverse` read (confirmed), force-sim
   layout, read-time decay/effective-value computation, consolidation/replay selection, and nebula color — via one
   canonical alive-predicate (`deleted_at IS NULL` for memories, `sealed_at IS NULL` for neurons, both endpoints alive
   for synapses); there is no "invisible-but-still-pulling" ghost [X3].
7. **The system never originates deletion** — no decay, consolidation, force-sim, or read path performs a hard delete; the only hard
   delete is the user-initiated 30-day sweep owned by [49.release-usecase](49.release-usecase.md), and it is out of scope
   here [I1][F3].
8. The rules use `Depress` (LTD) and **never** `Downscale` (SHY) — deletion contribution-weakening is associative/local,
   distinct from homeostatic sleep-time renormalization [I9].
9. The classification/sealing/exclusion behavior is **server-authoritative** (not FE↔BE golden-parity); the client learns
   of a removal only by facts disappearing from `GetUniverse`, and its unchanged force-sim/effective-value math runs over
   the smaller live graph.
10. No new column is added to `neuron_activations` — activation exclusion is transitive via the memory's `deleted_at` and
    the neuron's `sealed_at` (schema of [16.memory-aggregate-schema](16.memory-aggregate-schema.md) unchanged) [X1][X4].
11. The only tuning number introduced is `deletion.contribution_weaken_amount`; the classification rule, the sealing
    writes, and the exclusion predicate are formulas/logic that stay in code (not `values.yaml`).
