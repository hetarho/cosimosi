# 39. forgetting visuals

> Forgetting visuals — the presentation unit of Epic D: it makes forgetting **seen**. A star that has not been recalled
> in universe-time **dims** (the reserved brightness channel from [24.star-neuron-rendering](24.star-neuron-rendering.md)
> now driven by real `EffectiveBrightness`), and its **current-memory text loses words** — the decay-stage text (random
> word removal) shown on the star's hover/label and in the star-detail panel. Both read the read-time values Epic D's
> [37.forgetting-decay](37.forgetting-decay.md) computes; no new domain, no new write.
> Scope: FE | Status: planning
>
> Layer P (presentation) — the third and final unit of Epic D. Home: the `entities/star` visual body
> ([24.star-neuron-rendering](24.star-neuron-rendering.md)) for the dimming channel + `widgets/star-detail`
> ([35.star-detail-panel](35.star-detail-panel.md)) for the word-loss text, over the running `widgets/universe-canvas`
> ([23.universe-canvas](23.universe-canvas.md)). It **fills** the brightness seam reserved in plan 24 and the
> forgetting-visual slot reserved in the acceleration choreography ([31.time-acceleration-ui](31.time-acceleration-ui.md)
> [V8]); it **consumes** the read-time `EffectiveBrightness` + decay-stage text produced by

## Purpose

Epic D's [37.forgetting-decay](37.forgetting-decay.md) makes forgetting *true* at read time — it computes a floored
`EffectiveBrightness` that falls the longer a star goes unrecalled, and it produces the **decay-stage text** (the
current-memory text with words randomly removed, "나는 오늘 친구랑 xxxxx 파스타랑 xxxx 정말 맛있었다" → progressively
more `xxxx`). But until this unit, that decay is invisible: the star renders at full brightness (Epic A only *reserved*
the brightness channel, [24.star-neuron-rendering](24.star-neuron-rendering.md)) and the star-detail panel shows a
whole, un-eroded text.

This unit is where **forgetting becomes something the diarist sees and feels** — the core product sensation that a
memory not returned to *fades and loses its words* ([F1]), yet never disappears (it stops at the floor — the silent
engram, [F2]). It does two presentation things, both read-only over values [37.forgetting-decay](37.forgetting-decay.md)
already computes: it **drives the star's dimming** (fills the reserved brightness channel with real
`EffectiveBrightness`) and it **shows the word-loss text** (the decay-stage text on the star's hover/label and as the
free current-memory text in the star-detail panel, [D1][G1]). Recall reverses both — brightness resets and the words
return ([F5][R2]) — but that recovery is produced by the read-time values recomputing after recall writes its anchors
([33.recall-usecase](33.recall-usecase.md)); this unit only re-renders.

## Scope / Non-goals

**In scope**
- **Star dimming — fill the reserved brightness channel** ([V2][F1][F2]). The `entities/star` body already carries a
  brightness channel/uniform that resolved *full* in Epic A ([24.star-neuron-rendering](24.star-neuron-rendering.md)
  reserved it as "the Epic-D decay seam"). This unit binds that channel to the **real** read-time `EffectiveBrightness`
  from `packages/memory-logic` (plan 18's stub, now implemented by [37.forgetting-decay](37.forgetting-decay.md)),
  mapped within the existing `rendering.star_brightness_min`…`star_brightness_max` range. **No new body, no new shader
  graph** — the channel exists; this unit changes what feeds it (from full → `EffectiveBrightness`).
- **Word-loss text on the star hover/label** ([F1][R8a]). When a star is hovered/focused on the canvas, its label shows
  the **decay-stage text** (the current-memory text with words removed per the current decay stage), not the whole
  text — so the eroded memory reads *as eroded* even before opening the panel. Read from the `decay_stages` text now on
  the `GetUniverse` DTO (Epic D adds `decay_stages` to the read; see Data below).
- **Word-loss text in the star-detail panel** ([D1][G1]). The star-detail panel ([35.star-detail-panel](35.star-detail-panel.md))
  shows the **forgotten current-memory text for free** ([G1]: "흐려진 내 기억"은 무료). This unit supplies that panel
  section with the decay-stage text (the same read-time source as the hover), and surfaces the panel's **current
  forgetting degree** (the decay stage / brightness) as read-only meta ([D1]). The panel *shell* and its buttons
  (회고하기 / 변천사 / 원본 일기 보기) are [35.star-detail-panel](35.star-detail-panel.md)'s; this unit fills the
  forgetting-text section + the forgetting-degree meta.
- **Recall reversal is a re-render, not new logic** ([F5][R2]). After a recall writes its anchors
  ([33.recall-usecase](33.recall-usecase.md): `last_recalled_universe_time` reset, decay-stage recovery), the next
  `GetUniverse` read recomputes `EffectiveBrightness` to full and the decay-stage text back to whole; this unit's star
  and panel simply re-render the recovered values. It owns **no** recall write.
- **The acceleration slot** ([V8][31.time-acceleration-ui](31.time-acceleration-ui.md)). The Epic-B acceleration
  choreography reserved a slot for "forgetting playing through" during a time-jump. This unit provides the
  **per-star dimming** the acceleration plays into: when the clock advances over `[from,to]`, stars that crossed a decay
  threshold read a lower `EffectiveBrightness` and dim; the acceleration surfaces that as felt fade. This unit fills the
  reserved slot with the dimming channel; it does not re-own the choreography shell.
- **web + mobile parity** — the brightness channel is the shared TSL star body ([24.star-neuron-rendering](24.star-neuron-rendering.md))
  fed the same read-time value on both apps; the hover-label + panel text sections are the same slices, with
  `*.native.tsx` siblings **only** for the label/panel primitives that genuinely differ (§3.5). All copy via i18n.

**Non-goals**
- **No decay math, no read-time functions.** `EffectiveBrightness` (floored, arousal- & connection-modulated), the
  decay-stage index, and the **random-word-removal rule** ([F9]) that produces the decay-stage text are all
  [37.forgetting-decay](37.forgetting-decay.md) (`packages/memory-logic` + the `decay_stages` content). This unit
  **renders** the values; it computes none of them.
- **No new write, no recall.** Brightness reset + decay recovery on recall are [33.recall-usecase](33.recall-usecase.md);
  the accessibility cost of a decayed recall is [38.forgetting-cost](38.forgetting-cost.md). This unit issues no write
  and never resets brightness itself — it re-renders whatever the read model carries.
- **No star-detail shell, no buttons, no provenance.** The panel container, the 회고하기 / 변천사 / 원본 일기 보기
  buttons, the 변천사 (provenance) view, and the free-vs-paid gating are [35.star-detail-panel](35.star-detail-panel.md).
  This unit fills one section (the forgotten current-memory text) + the forgetting-degree meta.
- **No semanticization / gist visuals.** Forgetting (random word loss, brightness) is an axis **independent** of
  gist/semanticization (meaning compression, z-rise) ([F], §5.3.5). The gist-stage text and the neocortical gist star
  are [40.semanticization](40.semanticization.md) / [42.gist-star-rendering](42.gist-star-rendering.md); a decay-stage
  text is never a gist-stage text.
- **No cost display / stardust.** Whether a *recall* of a deeply-decayed star costs more 별가루 ([F4][G4]) is
  [38.forgetting-cost](38.forgetting-cost.md) (depth signal) + Epic F [43.stardust-ledger](43.stardust-ledger.md)
  (price). Viewing the forgotten text is **free** ([G1]); this unit shows it with no gate.
- **No consolidation replay visual** ([V8] gist/replay half) — that is [42.gist-star-rendering](42.gist-star-rendering.md);
  this unit fills only the *forgetting-dimming* half of the reserved acceleration slot.
- No hardcoded tuning scalars in FE (the brightness range is generated config; §Policy).

## Design

### Placement (FSD, §3.1)
This unit adds **no new slice** — it fills seams reserved by sibling plans:
```
apps/web/src/entities/star/                 (from 24.star-neuron-rendering)
├── ui/            the TSL star body — the brightness channel now fed EffectiveBrightness (was full in Epic A)
└── @x/episodic-memory  reads decay_stage / EffectiveBrightness inputs from the domain mirror (one-way, §3.4)

apps/web/src/entities/episodic-memory/       (from 23.universe-canvas)
└── api/           GetUniverse mapper extended to carry decay_stages + the current decay-stage text (Epic D read)

apps/web/src/widgets/star-detail/            (from 35.star-detail-panel)
└── ui/            the "forgotten current-memory text" section + the forgetting-degree meta filled here
                   (+ *.native.tsx for the label/panel text primitives where they differ, §3.5)
```
The star body imports the domain mirror **only via `@x`** and reads shared read-time functions
(`packages/memory-logic`); it exports no visual word back into the domain slice (§3.4). The hover label + panel text are
plain read-only presentation — no flow, no XState machine (there is no multi-phase control-state here; the recovery
animation, if any, is a discrete re-render driven off the read model, §3.2).

### Star dimming — the brightness channel, now driven by `EffectiveBrightness` (§3.3)
- The `entities/star` body's **brightness** channel/uniform — reserved by [24.star-neuron-rendering](24.star-neuron-rendering.md)
  ("resolves full today, the range is the decay seam Epic D fills") — is now fed the **real** read-time
  `EffectiveBrightness(base, recall_count, effectiveElapsed)` from `packages/memory-logic`
  ([37.forgetting-decay](37.forgetting-decay.md) implements the stub plan 18 reserved). The value is **floored**
  ([F2] — a decayed star dims to `rendering.star_brightness_min`, never to 0) and mapped within the existing
  `rendering.star_brightness_min`…`rendering.star_brightness_max` range (no new range; the min *is* the silent-engram
  floor).
- Per §3.2/§3.3 the brightness value rides the existing per-instance brightness attribute/uniform; it **recomputes only
  when the read model / universe time changes** (a `GetUniverse` refetch or a clock advance), not per frame — the
  coordinate buffer is what `useFrame` reads per frame, exactly as Epic A. **Decided (2026-07):** filling the channel
  requires **no body/shader change** — Epic A already authored the uniform; this unit changes only the *source* of the
  value (full → `EffectiveBrightness`), so the star body diff is at the projection layer, not the TSL graph.
- **Golden-parity stance (inherited, not owned).** The brightness *math* is golden-parity between
  `packages/memory-logic` (TS) and `internal/memory` (Go), owned by [37.forgetting-decay](37.forgetting-decay.md)
  (matching plan 18's precedent). This presentation unit is **not** golden-parity — it renders whatever the shared
  function returns; it introduces no math to keep in parity.

### Word-loss text — the decay-stage text on hover + in the panel ([F1][R8a][D1][G1])
- The **decay-stage text** (the current-memory text with words randomly removed at the current decay stage) is produced
  by [37.forgetting-decay](37.forgetting-decay.md) and stored per-stage in `decay_stages` ([R8a]); Epic D adds
  `decay_stages` to the `GetUniverse` read + DTO (the column exists on `episodic_memories` since
  [16.memory-aggregate-schema](16.memory-aggregate-schema.md) but was unselected — see brief §"As-built truth"). The
  read model exposes the **text at the current decay stage** (the stage index is derived from `EffectiveBrightness` /
  the decay curve by [37.forgetting-decay](37.forgetting-decay.md); this unit reads the resolved current-stage text, it
  does not index into stages itself).
- **On the canvas hover/label:** a hovered/focused star shows its **current decay-stage text** (eroded), so the fade
  reads before the panel opens. **Decided (2026-07):** the hover label shows a **truncated** current decay-stage text
  (a short preview), the **full** forgotten current-memory text lives in the star-detail panel — the label is a glimpse,
  the panel is the read.
- **In the star-detail panel:** the panel ([35.star-detail-panel](35.star-detail-panel.md)) shows the **forgotten
  current-memory text for free** ([G1]) — this unit fills that section with the full current decay-stage text — plus a
  read-only **forgetting-degree** meta (the current decay stage / brightness as an unobtrusive indicator, [D1]: "현재
  망각 정도"). The panel's buttons and the 변천사/원본 views stay [35.star-detail-panel](35.star-detail-panel.md)'s.
- **The distortion is not separately announced** ([R8a]: 왜곡을 따로 알리지 않음) — the eroded text simply *is* the
  memory as it now reads; the diarist discovers the loss by reading, not by a warning banner. This unit renders the
  eroded text plainly, with no "this memory has decayed" callout.

### Recall reversal — a re-render off recovered read-time values ([F5][R2])
- Recall ([33.recall-usecase](33.recall-usecase.md)) resets `last_recalled_universe_time` and recovers the decay stage;
  on the next `GetUniverse` read, `EffectiveBrightness` recomputes to full and the current decay-stage text resolves back
  to the whole current-memory text. This unit's star **brightens** and its panel/hover text **fills back in** — a pure
  re-render of recovered values. No recall write, no brightness reset, no text mutation lives here ([I8]: recall resets
  accessibility, not content — the *content* returning here is the un-erosion of the decay representation, not a Diary
  restore).
- **Decided (2026-07):** a recovery *animation* (a star brightening, words fading back) is **optional presentation** and,
  where present, is a discrete re-render off the read-model change (§3.2) — not a per-frame React-driven loop and never a
  blocker on the recall data path. Epic A's launch-reveal / Epic B's acceleration precedents govern the timing.

### The acceleration slot ([V8], filling the forgetting half)
- [31.time-acceleration-ui](31.time-acceleration-ui.md) reserved a slot in the time-jump choreography for "forgetting
  playing through". When the clock advances over `[from,to]`, stars whose unrecalled-elapsed crossed a decay threshold
  read a lower `EffectiveBrightness` and **dim** as the acceleration plays. This unit supplies that per-star dimming (via
  the same brightness channel); [31.time-acceleration-ui](31.time-acceleration-ui.md) owns the choreography shell + the
  interval seam. The **gist-rising / replay** half of [V8] is [42.gist-star-rendering](42.gist-star-rendering.md), not
  here — this unit fills only the forgetting-dimming half.

### Copy (i18n) — literary, restrained Korean
All user-visible copy (the forgetting-degree meta label, any empty/floor affordance) resolves through the i18n seam
([08.i18n-seam](08.i18n-seam.md)); none is hardcoded (`lint:raw-strings`). The **decay-stage text itself is content**
generated by [37.forgetting-decay](37.forgetting-decay.md) (the eroded memory), not an i18n message. Web + mobile draw
the same message keys (parity).

## Acceptance Criteria
1. A star that has not been recalled in universe-time renders **dimmer**: the `entities/star` brightness channel is fed
   the **real** read-time `EffectiveBrightness` (from `packages/memory-logic`, [37.forgetting-decay](37.forgetting-decay.md)),
   replacing Epic A's full-brightness placeholder, mapped within the existing `rendering.star_brightness_min`…
   `star_brightness_max` range ([V2][F1]).
2. Brightness is **floored** — a fully-decayed star dims to `rendering.star_brightness_min`, **never to 0 / never
   removed** (the silent engram) ([F2][I1]); the star stays renderable at the floor.
3. Filling the brightness channel requires **no new star body or shader graph** — the reserved uniform from
   [24.star-neuron-rendering](24.star-neuron-rendering.md) is bound to `EffectiveBrightness`; the change is at the
   projection layer, and the value recomputes on read-model / universe-time change, not per frame (§3.2/§3.3).
4. A hovered/focused star's label shows its **current decay-stage text** (the current-memory text with words removed),
   not the whole text — the eroded memory reads as eroded before the panel opens ([F1][R8a]).
5. The star-detail panel shows the **forgotten current-memory text for free** (the full current decay-stage text) plus a
   read-only **forgetting-degree** meta ("현재 망각 정도"); the panel shell, buttons, and 변천사/원본 views remain
   [35.star-detail-panel](35.star-detail-panel.md)'s ([D1][G1]).
6. The decay-stage text is read from the `decay_stages`-backed **current-stage text** on the `GetUniverse` DTO (Epic D's
   read extension); this unit **does not** compute the removal rule or index the stages — it renders the resolved
   current-stage text ([R8a][F9]).
7. The word-loss is **not separately announced** — the eroded text is shown plainly, with no "this memory decayed"
   warning; the diarist discovers the loss by reading ([R8a]).
8. After a recall ([33.recall-usecase](33.recall-usecase.md)) the star **brightens back to full** and its hover/panel
   text **fills back to whole** on the next `GetUniverse` read — a pure re-render of recovered read-time values; this
   unit issues **no** recall write and never resets brightness or mutates text itself ([F5][R2][I8]).
9. Dimming and word-loss **move together** — a star reading a lower `EffectiveBrightness` also reads a deeper
   decay-stage text, because both derive from the same read-time decay in [37.forgetting-decay](37.forgetting-decay.md)
   ([F1]).
10. Forgetting (dimming + word-loss) is **independent of gist/semanticization** — a decay-stage text is never a
    gist-stage text, and this unit renders no z-rise / gist star ([F], §5.3.5); those are
    [40.semanticization](40.semanticization.md) / [42.gist-star-rendering](42.gist-star-rendering.md).
11. Viewing the forgotten text (hover + panel) is **free** — no 별가루 gate; the recall-cost of a decayed star stays out
    of this unit ([G1]).
12. This unit fills the **forgetting-dimming half** of the reserved acceleration slot ([V8],
    [31.time-acceleration-ui](31.time-acceleration-ui.md)): on a clock advance, stars that crossed a decay threshold dim
    as the acceleration plays; it does not re-own the choreography shell nor the gist/replay half.
13. **All** user-visible copy (the forgetting-degree meta, any floor affordance) resolves through the i18n seam (no raw
    strings; `lint:raw-strings` passes); the decay-stage text is generated content, not an i18n message.
14. **web + mobile parity:** the brightness channel is the shared TSL star body fed the same read-time value on both
    apps; the hover-label + panel-text sections share their read logic, with `*.native.tsx` siblings **only** for the
    label/panel primitives that differ (§3.5).

## Policy / Values Impact
