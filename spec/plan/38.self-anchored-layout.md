# 38. 자아 중심 + 활성도 반지름 (self-anchored radial layout)

> 우주 중심에 **"나" 별**(앵커)을 세우고, 각 기억 별의 *중심으로부터의 거리*를 그 기억의 **강함(활성도 + 감정 강도)** 으로 정한다. 자주·최근 떠올린 강렬한 기억일수록 "나"에 가깝고, 잊혀갈수록 바깥으로 멀어진다. 회상하면 미세하게 중앙으로, 시간이 지나면 서서히 바깥으로, 새 기억은 가장 강하므로 중앙 근처에서 태어난다. | 범위: FE | 선행: 22 | 차단: —

## 목적 (Why)
22가 라이브 force-sim을 배선했지만, 좌표가 순수 그래프 스프링/척력에서만 빚어져 **직관이 약했다**: 새 별이 hot 무리 곁에 붙긴 해도 `linkDistance`·척력이 별 셸 스케일에 안 맞아 별들이 중심에서 사슬처럼 바깥으로 행진하고, "강한 기억 = 가까운 기억"이라는 읽기가 없었다. 본 스펙은 우주에 **자아(나)** 라는 중심을 도입해 거리에 의미를 부여한다 — **"거리는 강함, 방향은 연결"**. 거리(반지름)는 활성도(최근성, 12의 망각 곡선)와 감정 강도의 합성으로, 방향(각도·이웃)은 여전히 그래프와 22의 흥분성 편향으로 정해진다. 이로써 회상의 *공고화*(재점화 11/12)가 "중앙으로 당겨짐"으로, 망각이 "바깥으로 멀어짐"으로 **공간에 드러난다** — concept §우주의 배경(요즘 상태)·§망각이 시각이 된다.

## 범위
- **포함:**
  - **자아 별(나).** 우주 중심(원점)에 고정된 단일 앵커 별. 그래프 **비참여**(연결·KNN·시냅스 없음, `memories`가 아님). 새 appearance 차원 `selfObject`로 **2–3개 폼 중 선택**(기본 `nebula-heart`). 색은 25(ambient) 도입 시 요즘 감정색, 그 전엔 테마 accent 폴백.
  - **활성도 반지름.** 순수 `strength(activation, intensity)`·`targetRadius(strength)`(`shared/lib/layout.ts`, three/DOM 미의존). `strength = clamp(W_ACT·activation + W_INT·intensity, 0, 1)`, `targetRadius = lerp(R_MIN, R_MAX, 1 − strength)`.
  - **force-sim 반지름 셸 힘.** `shared/lib/force-sim`에 노드별 **목표 반지름 셸로의 라디얼 스프링**(`SimNode.radius` + `SimParams.radialStrength`) 추가. 척력이 같은 셸 위 별들을 각도로 흩고, 스프링이 연결된 별을 각도로 모은다. `radius` 미지정(0)이면 기존 `centerGravity`(원점 당김) 거동 — 07 계약 보존.
  - **창발 거동(전부 activation 파생, 별도 상태·서버 변경 없음).** 회상 → activation↑ → targetRadius↓ → 중앙으로 미끄러짐. 시간 경과 → activation↓ → targetRadius↑ → 바깥으로. 새 기억 → activation=1 → 중앙 근처 출생. 매 프레임 현재 시각(`virtualNowMs`)으로 반지름을 갱신하고, 의미 있게 움직였을 때만 sim을 재이완(re-kick)해 부드럽게 활강.
  - **네 reader 단일 버퍼(22) 유지.** 자아 별은 중심 고정이라 버퍼 밖. 별·시냅스·카메라는 22의 공유 버퍼를 그대로 읽는다.
- **비목표(Out of scope):**
  - 앰비언트 무드 색·gain(`g=1+0.3·arousal`) → **25**. 여기선 nebula-heart 색은 테마 accent 폴백, 반지름 가중은 상수.
  - 재공고화 재성형·변천사 → **23/24**. 회상은 activation 재점화(11/12)까지만 — 본 스펙은 그 *공간* 표현(중앙 당김).
  - 관련성 가중 감쇠 `λ_eff`(R_conn·R_recent·R_emo) → **26**. 여기선 단일 λ(12) activation + intensity만 강함으로 쓴다.
  - 야간 재안정화·재분배 → **27**.
  - `selfObject` **서버 per-user 동기**(30 패턴) → 후속. 일단 기기 로컬(localStorage)로 `object`와 같은 취급.
  - 자아 별 **클릭 상호작용**(자기소개·설정 패널) → 후속. 본 스펙은 렌더·폼 선택까지.

## 설계 요점

### 강함 → 반지름 (순수, 단위 테스트)
```ts
// shared/lib/layout.ts
export const R_MIN = 8    // 가장 강한 기억이 자아 곁에 머무는 최소 반지름(자아 별 반경 밖)
export const R_MAX = 60   // 잊힌(잠든) 기억이 떠도는 최외곽
export const W_ACT = 0.7  // 강함 = W_ACT·활성도 + W_INT·감정강도
export const W_INT = 0.3
/** activation∈(0,1](최근성, 12)·intensity∈[0,1](감정 강도) → 강함 0..1. */
export function strength(activation: number, intensity: number): number
/** 강함 0..1 → 중심 거리. 강함=1 → R_MIN(중앙), 강함=0 → R_MAX(외곽). */
export function targetRadius(strength: number): number   // lerp(R_MIN, R_MAX, 1−strength)
```
- 활성도는 12의 `activation(lastRecalledAt, now)` 그대로 — 회상하면 1로 재점화, 시간 지나면 감쇠. 감정 강도는 `Memory.intensity`(불변). **활성도가 주도(0.7), 감정 강도가 보정(0.3)** — "활성도 + 감정 강도 합성".
- 컨트롤러(widgets)가 `activation`(entities/memory)·`intensity`를 모아 `strength`/`targetRadius`(shared/lib)를 호출 — FSD 단방향(shared는 entities 미import) 유지.

### force-sim 반지름 셸 힘 (`shared/lib/force-sim`, 순수)
`step()`의 중심 중력 블록을 일반화한다:
```ts
// SimNode: + radius?: number   // 목표 셸 반지름(미지정/0 → 원점 중력 = 기존 거동)
// SimParams: + radialStrength  // 셸 스프링 계수(기본 0.08; radius>0일 때만 작동)
// SimState: + radius: Float32Array  // 노드별 목표 반지름(미지정 0)
// step(), free 노드의 위치 힘:
//   if radius[i] > 0:  len=|p|; if len>eps: v += (p/len)·(radius[i]−len)·radialStrength·alpha
//   else:              v += −p·centerGravity·alpha            // 기존
```
- 척력(Barnes-Hut)이 **같은 목표 반지름의 별들을 그 셸 위에서 각도로 고르게 흩는다** — 사슬이 아니라 동심 셸. 스프링은 연결된 별을 각도로 모은다. → 강한 기억은 안쪽 셸의 고리, 잊힌 기억은 바깥 셸.
- `radius` 전부 0이면 `radialStrength`는 비활성, `centerGravity` 거동 유지 → **07 단위 테스트 불변**.
- 컨트롤러가 매 프레임 `sim.radius[i]`를 현재 `targetRadius`로 갱신하고, 변화량이 임계 이상이면(회상 점프·시간 누적) `alpha`를 올려 재이완 → 부드러운 활강. 변화 없으면 settled로 두어 매 프레임 N체 재계산 안 함(07의 settle 조기 종료).

### 각도 안정성 — 헌법3 pin 절 정제
헌법3의 "기존 별은 pin"은 **"각도는 안정, 반지름은 활성도로 변동"** 으로 정제된다. 모든 기억 별은 이제 free(셸로 이완)지만, 22의 `prevPos` resume이 직전 위치(방향)에서 이어받아 **각도 연속성**을 지킨다 — 회상·시간으로 *반지름만* 미끄러지고 방향(이웃 관계)은 보존된다. "내 기억이 그 방향에 있다"는 유지하면서 강함에 따라 안/밖으로 호흡한다. (서버는 여전히 그래프만 — 헌법3의 "좌표는 클라 창발·서버 그래프만"은 불변.)

## 수용 기준 (EARS / Given-When-Then)
1.1 GIVEN 활성도·감정 강도가 다른 두 별 WHEN 레이아웃이 정해지면 THE SYSTEM SHALL `strength = clamp(W_ACT·activation + W_INT·intensity, 0, 1)`로 더 강한 별을 **중심에 더 가깝게**(작은 반지름) 둔다. _(거리=강함)_
1.2 WHEN 강함이 1이면 THE SYSTEM SHALL 반지름 `R_MIN`(자아 곁), 강함이 0이면 `R_MAX`(외곽)로 두고 그 사이를 선형 보간한다.
1.3 WHEN 한 별을 회상하면(activation→1) THE SYSTEM SHALL 그 별의 목표 반지름이 줄어 **중앙으로 미끄러진다**(재점화의 공간 표현). _(회상→중앙)_
1.4 WHEN 시간이 흘러 활성도가 감쇠하면 THE SYSTEM SHALL 별의 목표 반지름이 늘어 **바깥으로 멀어진다**(망각의 공간 표현). _(시간→바깥)_
1.5 WHEN 새 기억이 태어나면(activation=1) THE SYSTEM SHALL 가장 강하므로 **중앙 근처에서 출생**한다(빈 공간/사슬 아님). _(새 기억=중앙)_
1.6 GIVEN 같은 목표 반지름의 여러 별 WHEN force-sim이 이완되면 THE SYSTEM SHALL 척력으로 그 별들을 셸 위에서 **각도로 흩어** 사슬이 아닌 동심 분포를 만든다. _(거리=강함, 방향=연결)_
1.7 WHEN `SimNode.radius`가 0/미지정이면 THE SYSTEM SHALL 기존 `centerGravity` 원점 거동을 그대로 유지한다(07 계약 불변, 폴백). _(하위호환)_
1.8 WHEN 우주가 그려질 때 THE SYSTEM SHALL 중심에 **자아 별**을 그리되 그래프(연결·KNN·시냅스)에 넣지 않는다. _(자아=앵커, 비참여)_
1.9 WHEN 사용자가 `selfObject`를 고르면 THE SYSTEM SHALL 2–3개 폼(기본 `nebula-heart`) 중 그 형태로 자아 별을 그리고 선택을 기기에 지속한다. _(폼 선택)_
1.10 WHEN 별의 반지름이 활성도로 변해도 THE SYSTEM SHALL `prevPos` resume으로 **각도(방향)를 보존**하고 어떤 별·시냅스도 삭제하지 않는다(밝기·거리만 변함). _(헌법2·3 정제)_
1.11 WHEN `shared/lib/force-sim`·`shared/lib/layout.ts`를 빌드하면 THE SYSTEM SHALL `three`/React/DOM을 import하지 않는다. _(헌법4)_
