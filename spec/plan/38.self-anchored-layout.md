# 38. 자아 중심 레이아웃 - 자아 앵커 + 거리=강함 + 표상 부동 (as-built)

> 현재 구현 기준 문서. 우주 중심에 "나" 별(앵커)을 세우고, 각 기억 별의 중심 거리를 강함(활성도 + 감정 강도)으로, 방향을 연결 그래프로 정하며, force-sim 접선/radial 분리로 거리=강함을 강제하고 별의 방향을 밤마다 한 스텝 표류시키는 클라이언트 레이아웃의 SSOT다. | 범위: FE | 상태: 구현 완료(거리=강함 강제·표상 부동 통합)
>
> **(spec 03 — changes/03, job 08):** 중심 "나" 별이 이제 **광원**이 된다. "거리=강함(주로 최근성)"이 self-light **반사 밝기로도** 보인다 — 가까운 별이 더 환히 비춰짐(반사는 위치를 광학적으로 읽을 뿐 레이아웃·`strength` 식·`W_INT`를 바꾸지 않는다). 별 빛 3채널은 [policy/domain/star.md](../policy/domain/star.md) §밝기·감쇠.

## 목적

22가 라이브 force-sim을 배선했지만 좌표가 순수 그래프 스프링/척력에서만 빚어져 "강한 기억 = 가까운 기억"이라는 읽기가 없었고, 별들이 중심에서 사슬처럼 바깥으로 행진했다. 38은 우주에 자아(나)라는 중심 앵커를 도입해 거리에 의미를 부여한다 — "거리는 강함, 방향은 연결". 반지름은 활성도(12 망각 곡선의 최근성)와 감정 강도의 합성으로, 방향(각도·이웃)은 그래프와 22의 흥분성 편향으로 정해진다. 회상의 공고화가 "중앙으로 당겨짐"으로, 망각이 "바깥으로 멀어짐"으로 공간에 드러난다. 좌표는 여전히 클라 force-sim에서 창발하고 서버는 가중치 그래프만 안다(헌법3). 헌법3의 "기존 별은 pin"은 38에서 "각도는 안정, 반지름은 활성도로 변동"으로 정제된다.

## 현재 구현

### 강함 → 반지름 (순수 헬퍼)

`frontend/src/shared/lib/layout.ts`에 three/React/DOM 미의존 순수 함수로 산다(헌법4).

- `R_MIN=6` — 가장 강한 기억이 머무는 최소 반지름(자아 별 반경 밖). `R_MAX=40` — 잠든 기억이 떠도는 최외곽. R_MAX는 카메라 별 셸 프레이밍(~46)보다 안쪽이라 구름이 흩어지지 않고 응집한다.
- `W_ACT=0.7`(활성도 가중)·`W_INT=0.3`(감정 강도 가중). 활성도가 주도, 감정 강도가 보정.
- `strength(activation, intensity) = clamp01(W_ACT·clamp01(activation) + W_INT·clamp01(intensity))`.
- `targetRadius(strength) = R_MIN + (R_MAX − R_MIN)·(1 − clamp01(strength))`. 강함 1 → R_MIN(중앙), 강함 0 → R_MAX(외곽), 선형 보간·단조 감소.
- 정전 수치(R_MIN/R_MAX/W_ACT/W_INT)의 값 SSOT는 `spec/policy/domain/universe.md`다.

### 활성도/감정 입력

- 활성도는 12의 `activation(lastRecalledAt, now)=exp(−LAMBDA·Δt_days)` 그대로다(`frontend/src/entities/memory/model/activation.ts`). `LAMBDA=Math.LN2/30`, `A_MIN=0.05`. 회상하면 1로 재점화, 시간 지나면 감쇠.
- 감정 강도는 `Memory.intensity`(불변).

### 반지름 합성 (widgets, FSD 단방향)

`frontend/src/widgets/universe-canvas/model/radial-layout.ts`가 entities(memory)와 shared(layout)를 조합한다 — shared는 entities를 import하지 않아 FSD 단방향이 유지된다. 순수 TS(three/React/DOM 미의존).

- `radiusOf(mem, now)`: `act = max(A_MIN, activation(lastRecalledAt, now))` 후 `targetRadius(strength(act, intensity))`. 활성도를 A_MIN으로 바닥 처리해 가장 잠든 별들이 모두 동일 외곽 셸로 붕괴하지 않고 감정 강도로 여전히 흩어진다.
- `atRadius(pos, r)`: 시드 방향을 보존한 채 길이만 목표 반지름 셸로 스케일(별이 군집의 각도에서 오르되 강함의 거리에 놓인다). 원점 퇴화 시 고정 축 `[r,0,0]`.
- `RADIAL_SIM_PARAMS = { repulsion: -18, linkDistance: 14, radialStrength: 0.1 }` — 강함 셸 우주용으로 조인 파라미터. 약한 척력 + 짧은 링크로 연결 별이 사슬이 아니라 응집하고, 굳은 라디얼 스프링으로 각 별이 강함 셸을 끌어안는다.
- 이 모듈은 단일 우주 캔버스(`LiveLayoutController`)와 37 오버레이(`overlay/OverlayUniverse.tsx`)가 공유한다 — 양쪽이 동일 셸·동일 sim 파라미터를 써 "내 우주"와 오버레이 사본이 조용히 어긋나지 않는다.

### force-sim 반지름 셸 힘 (순수 코어)

`frontend/src/shared/lib/force-sim/{types.ts,sim.ts}`. 07 계약을 일반화한다.

- `SimNode.radius?: number` — 목표 셸 반지름. 미지정/0이면 기존 `centerGravity` 원점 당김(07 거동 보존).
- `SimParams.radialStrength` — 셸 스프링 계수. `DEFAULTS.radialStrength=0.08`(`radial-layout`의 공유 파라미터는 0.1로 덮어쓴다). `radius>0`인 노드에만 작동.
- `SimState.radius: Float32Array` — 노드별 목표 반지름. 호출자가 틱 사이에 갱신하고 `alpha`를 올려 새 셸로 활강시킬 수 있다.
- `step()` 위치 힘 분기(`RADIAL_MIN_DIST=1e-3`):
  - `radius[i]>0`이고 `|p|>RADIAL_MIN_DIST`: 단위 라디얼 `r̂=p/len`. 외력(척력+링크, `fbuf`에 누적)에서 라디얼 성분을 빼 접선 성분만 `vx`에 더한다(40: 링크/척력은 별을 셸 위에서 각도로만 돌리고 거리는 안 바꾼다). 라디얼 셸 스프링 `f=(r−len)·radialStrength·alpha`이 거리축을 소유한다(셸 안이면 바깥으로, 밖이면 안으로 → `|p|=r`에서 정착).
  - 원점 퇴화(`len≤RADIAL_MIN_DIST`): 라디얼 방향이 없어 외력 전부 + `+x` 넛지로 셸로 탈출.
  - `radius[i]=0`: `vx += fbuf − p·centerGravity·alpha`(07 거동).
- 외력은 `fbuf`(틱당 외력 누산기)에 모으고 `vx` 밖에 둔다 — 셸 노드가 접선 성분만 취하게 하기 위함. 이 분리가 없으면 링크 스프링(계수 ≈ weight 0.6–0.8)이 셸 스프링(radialStrength ≈ 0.1)을 압도해 새 별을 이웃의 셸로 끌어낸다(38의 빈틈).
- `createSim`은 노드 `radius`를 수집한다(`radius[i] = node.radius ?? 0`). `MAX_SPEED_FACTOR=2` 변위 상한·octree 발산 방지는 07 그대로.

### 자아 별 (SelfStar)

`frontend/src/widgets/universe-canvas/ui/SelfStar.tsx`. 원점 고정 단일 mesh — 그래프 비참여(연결·KNN·시냅스 없음, `memories`가 아님).

- 지오메트리는 `IcosahedronGeometry(1, 6)`, `scale=SELF_RADIUS=5`(가장 강한 기억 셸 R_MIN=6 바로 안쪽이라 가까운 기억이 자아를 휘감되 삼키지 않는다).
- `selfObject` 폼별 TSL 머티리얼(`MeshBasicNodeMaterial`, three/webgpu·three/tsl). 씬 방향광이 없어 emissive only(StarField/forms 관용):
  - `nebula-heart`(기본): `mx_noise_float` 흐름 글로우 + 부드러운 볼류메트릭 가장자리(형체 없는 빛의 소용돌이).
  - `core`: 밝은 준-고체 태양, 뜨거운 중심 + 약간 밝은 림, `sin` 호흡.
  - `well`: 어두운 중심 + 빛이 휘어 밝은 림 링(`pow(rim, 2.2)`).
  - 공통: `AdditiveBlending`·`toneMapped=false`(HDR 유지)로 BloomPass가 글로우를 키운다(헌법8 — 셰이더 TSL).
- 색 = 테마 accent(`themeAccent(theme)` → linear). ambient hue는 25 도입 시.
- 시간 uniform `uTime`은 수동 클록(`useFrame`의 `state.clock.elapsedTime`) — built-in `time` 노드가 BloomPass 아래서 멈추기 때문. reduced-motion이면 `uTime=0`으로 내부 흐름 정지(여전히 렌더).
- `raycast={() => undefined}`로 raycast 비활성(클릭 상호작용은 후속). `frustumCulled={false}`.

### 컨트롤러 배선 (`LiveLayoutController`)

`frontend/src/widgets/universe-canvas/ui/UniverseCanvas.tsx`. 별 집합 + 시냅스로 하나의 가중치 그래프를 만들어 매 프레임 하나의 positions 버퍼로 이완한다(22의 네 reader 단일 버퍼: StarField·FlyTo·Focus가 직접 읽고, 시냅스는 settle 시 publish된 snapshot에 베이크).

- 모든 기억 별은 free(`pinned:false`) — 강함으로 호흡(반지름 변동). 각도 연속성은 22의 `prevPos` resume이 직전 라이브 위치(방향)에서 이어받아 지킨다.
- sim 재구축은 그래프 토폴로지(별 id + 엣지 쌍 `topo` 문자열)가 바뀔 때만. 데모 시간 스킵의 `refreshActivation`이 밝기 재계산만으로 stars/edges 배열 참조를 ~12×/tween 갈아끼워도 같은 토폴로지면 sim을 재구축하지 않는다(별 churn·시냅스 stretch 방지).
- 노드 시드: resume이 있으면 그 위치에서, 없으면(신규·첫 로드) 연결된 가장 hot한 군집의 각도(`seedNearCluster`, 22)로 오르되 강함의 거리(`radiusOf` → `atRadius`)에. 배치 이웃이 없으면 `scatterDirection(seed)`로 흩뿌린 방향(38 — 별을 하나씩 추가해도 fibonacci 황금각 나선 호를 그리지 않게)을 강함 셸로 스케일. heat는 ~6h 지수 감쇠(`HOT_TAU_MS`, 22 흥분성 창 미러).
- `createSim(..., RADIAL_SIM_PARAMS, { seedNewNodes: false })` — 위 dir·radius 배치를 이웃 평균으로 덮지 않게.
- 매 프레임 `useFrame`:
  - (데모 전용) 표상 부동: 클록이 밤(`floor(virtualNow/DAY_MS)`)을 넘을 때마다 free 노드의 방향을 `applyAngularDrift(pos, seed, dn)`로 회전(|p|=강함 보존), 속도 0 리셋, `alpha`를 `REKICK_ALPHA`로 올림. `nightRef`로 야간 인덱스 베이스라인을 잡아 경계 사이엔 안 움직인다. 실서비스는 좌표가 세션마다 새로 창발(헌법3 비영속)해 중간 애니메이션이 없다.
    - `applyAngularDrift`는 **고정 per-seed 축**(`scatterDirection(seed)`) 둘레 Rodrigues 회전이라 가환군이다(`drift(N)=N×drift(1)` — 시계가 점프하든 트윈하든 같은 결과, 스킵=대기·경로 무관). 축을 매번 pos에서 구하면 부호 반전으로 정체·진동하므로 고정축이 핵심. `DRIFT_STEP_RAD=0.08`(~4.6°/밤, 손튜닝).
    - **차등 드리프트(무비용 창발):** 회전 후 접선력이 재이완하며 잘 연결된 성단은 부분 복원(덜 흐름)되고 고립 별은 자유로이 표류(더 흐름)한다 — 고립 기억일수록 더 흐른다는 표상 부동 연구(Audrain & McAndrews 2022)를 별도 코드 없이 (A)의 접선력 + re-kick에서 창발시킨다.
  - 매 프레임 `radiusOf(mem, virtualNowMs())`로 목표 반지름 재계산(`targetScratchRef` 재사용, 무할당). 갱신 전 `sim.radius`(마지막 이완 셸)와 비교해 최대 변화 `maxDelta` 산출 — 직전 프레임 값이 아닌 마지막 이완 값과 비교해야 느린 감쇠가 누적돼 임계를 넘는다.
  - `maxDelta > REKICK_THRESHOLD(0.5)`이면 모든 목표를 `sim.radius`에 한 번에 commit하고 `alpha`를 `REKICK_ALPHA(0.3)`로 재상승(re-kick) → sim 전체(라디얼+링크+척력)가 새 균형으로 함께 이완. 회상 점프는 항상 임계를 넘고, 느린 시간 감쇠는 가끔 넘어 단계적 활강. 임계 미만이면 settled 유지(매 프레임 N체 재계산 안 함).
  - settle 시에만 positions snapshot을 publish — 시냅스는 이완된 좌표에서 재연결, 중간 이완 중엔 안 함.
- `<SelfStar selfObject={…} />`는 `<UniverseDrift>` 안(별·시냅스와 함께 부유, StarDust는 밖)·`<group visible={ready}>` 안에 마운트된다. `useAppearance((s) => s.selfObject)` 구독. 자아 별은 sim 그래프 밖이라 positions 버퍼에 없고 원점 고정으로 부유한다.
- settle 시 `onReady`로 첫 레이아웃 공개(8초 안전망 타임아웃). 중간 세션 소스 리셋(데모 "처음으로")은 `onReset`으로 다시 가린다.

## Flutter 동등성 기준

38은 클라이언트 렌더 레이아웃이다(서버 계약 없음 — 좌표 필드는 영원히 proto에 없다, 헌법3). Flutter도 다음을 자체 force-sim/레이아웃으로 재현해야 한다.

- 중심에 단일 "나" 앵커를 그리되 그래프(연결·KNN·시냅스)에 넣지 않는다.
- 각 별의 중심 거리(반지름)를 `strength = clamp(0.7·activation + 0.3·intensity, 0, 1)` → `targetRadius = lerp(R_MIN, R_MAX, 1−strength)`로 정한다. activation은 12의 `exp(−λ·Δt)`를 A_MIN으로 바닥 처리.
- 방향(각도)은 연결 그래프에서 창발시키되, 레이아웃 재구축 시 직전 위치에서 resume해 각도 연속성을 지킨다.
- 회상 → activation↑ → 반지름↓(중앙), 시간 → activation↓ → 반지름↑(외곽), 새 기억 → activation=1 → 중앙 근처 출생.
- `selfObject` 폼 선택(기본 nebula-heart)은 기기 로컬 선호로 지속(서버 동기 아님).
- 정전 수치(R_MIN/R_MAX/W_ACT/W_INT)는 `spec/policy/domain/universe.md`를 따른다.

## 수용 기준

1. 활성도·감정 강도가 다른 두 별 중 `strength = clamp(W_ACT·activation + W_INT·intensity, 0, 1)`가 큰 별이 중심에 더 가깝다(작은 반지름).
2. 강함이 1이면 반지름 `R_MIN`(자아 곁), 0이면 `R_MAX`(외곽), 그 사이는 선형 보간이며 강함에 단조 감소다.
3. 한 별을 회상하면(activation→1) 목표 반지름이 줄어 중앙으로 미끄러진다.
4. 시간이 흘러 활성도가 감쇠하면 목표 반지름이 늘어 바깥으로 멀어진다.
5. 새 기억은 가장 강하므로 중앙 근처에서 출생한다(빈 공간/사슬 아님).
6. 같은 목표 반지름의 여러 별은 force-sim 이완 시 척력으로 셸 위에서 각도로 흩어져 사슬이 아닌 동심 분포를 만든다.
7. `SimNode.radius`가 0/미지정이면 기존 `centerGravity` 원점 거동을 그대로 유지한다(07 계약 불변).
8. 우주가 그려질 때 중심에 자아 별이 보이되 그래프에 넣지 않으며, sim positions 버퍼에 들어가지 않는다.
9. 사용자가 `selfObject`를 고르면 3개 폼(기본 nebula-heart) 중 그 형태로 자아 별을 그리고 선택을 기기에 지속한다(서버로 push하지 않는다).
10. 별의 반지름이 활성도로 변해도 `prevPos` resume으로 각도(방향)를 보존하고 어떤 별·시냅스도 삭제하지 않는다.
11. `shared/lib/force-sim/**`·`shared/lib/layout.ts`는 three/React/DOM을 import하지 않는다(헌법4).
12. 단일 우주 캔버스와 37 오버레이가 같은 `radial-layout`(셸 수식·sim 파라미터)을 공유해 두 우주가 동일 셸에 별을 놓는다.
