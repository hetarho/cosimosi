# cosimosi spec — index (00.overview)

> `spec/` is cosimosi's **single source of truth (SSOT)**. plan · concept · values describe *what is currently true*
> (as-built), in enough detail to re-implement on any client (incl. Flutter/RN). It holds "what is true now," not work
> history — per-step checklists live in [jobs/](../jobs/), and history beyond that lives in git.

One line: **"You write a diary → it splits into event-cells and rises as stars → memories that share cells link with
light → recalling together strengthens them, recall and time re-sculpt the stars, and neglected stars dim but never
vanish."**

> **This is the rebuilt roadmap.** The MVP validated the core idea (diary → memory-universe). The MVP-era spec
> (plan 01–55, policy, tech, changes, jobs, code-review) is archived under [`spec/.mvp/`](../.mvp/) for reference.
> This roadmap re-derives the build cleanly from the [rewritten concept](../concept.md): foundation → core domain →
> impl layer. Numbers here are a **fresh series** (stable IDs — never reused/renumbered).

---

## How this roadmap is built (re-plan, not rewrite)

The code from the MVP stays. We author this clean plan set and **build only the delta**; already-built units are
captured, not rebuilt. The one discipline that makes that safe:

> **verify-and-capture — never "skip on assumption."** Before declaring a unit already-implemented, do a cheap
> spec↔code reconciliation (grep/read). Three outcomes: **matches** → write the plan as-built, done · **drifts** →
> the plan states the target, the gap becomes a change/refactor job · **missing/inert** → real build. The SSOT-AUDIT
> already proved code drifts from its own specs (R_MIN/MAX, 1:1 vs 1:N, 7 vs 13 moods) — capturing without
> re-grounding would re-paint that drift at the foundation.

**Plan altitude must match build intent:** `CAPTURE` units → write the plan as **as-built** (mirror code). `NEW`/
`CONCEPT` units → write the plan as **target** (these are the actual work; never skipped).

**Workflow.** `/cosimosi:create-plan` fills a stub via interview → `/cosimosi:create-plan-job NN` → `/cosimosi:implement-job MM`.
All spec docs are English. Config numbers go in [values.yaml](../values.yaml) → `pnpm gen:values` (never hardcoded).

**Tags.** `NEW` (doesn't exist yet) · `CONCEPT` (new concept changes behavior vs current code) · `DEBT` (exists, needs
refactor) · `CAPTURE` (already clean — spec mirrors code, little/no code work) · `DEFER` (planned, after single-user track).

## Invariants (constitution — every plan honors these)

Engineering/architecture invariants. For **behavioral** laws (forgetting = dimming, recall partial-restore, sleep
consolidation, etc.) and the neuroscience guardrails, [concept.md](../concept.md) §9 (laws) + §10 (must-not-implement)
are authoritative.

1. **Original diary is immutable & permanent.** User text is never edited/deleted; stored apart from stars/links.
2. **Stars & synapses are never deleted.** Decay only dims (floor `A_MIN`); they persist as dormant.
3. **Coordinates are emergent client-side (force-sim), never server-authoritative.** Server owns only the weighted
   graph. Radius = connectedness/self-relevance, angle = relationship; time/forgetting is **never a position axis**.
4. **FSD one-way imports.** Slices import only lower layers; entity↔entity only via `@x`. `model`/`shared/lib` import
   no three/React/DOM (mobile reuse). Platform branching only in `ui`.
5. **proto = transport DTO, domain is pure, sqlc = persistence.** No json/db/proto tags on domain types.
6. **Connect is unary-only** (RN has no streaming). Recall reinforcement = client-local + idempotent unary batch.
7. **AI behind provider ports** (Embedder/Extractor) — any LLM/embedding is swappable.
8. **Shaders in TSL; stars = InstancedMesh buckets; synapses = Line2** — thousands-of-nodes premise.
9. **Two-layer engram model.** Information unit = ensemble (big star) made of cells (small stars); connection = shared
   cells. No "grandmother cell" (one node = one fact). *(concept §5; implementation depth — full vs visual-only —
   is settled when authoring plan 12.)*
10. **Emotion lives only in color** — never in position, links, or strength.
11. **Visual bodies come through an asset-source port** (shader | model | …); domain never imports a concrete
    renderer or asset format.
12. **Config is build-time, git-versioned** (`values.yaml` → `gen:values` → FE/BE constants). No hardcoded tuning;
    canonical constants are not mutated at runtime/admin. *(decided 2026-06-26.)*

## Cross-cutting decisions

- **Numbering.** Fresh contiguous series (01–43). Stable IDs — never reused/renumbered. The MVP series (01–55) is
  frozen under `.mvp/`. New units append at 44+.
- **i18n = Paraglide** (compile-time, type-safe, tree-shaken; RN-shareable). String seam from day one.
- **values philosophy = build-time (status quo).** Considered DB/admin-managed runtime config; rejected because the
  client + offline demo need the numbers at runtime (→ fetch + dual source), half the constants are science-tied
  (DOIs in concept §12) and must stay git-reviewed, and a DB layer is *more* infra than the build step, not less.
- **tech/ & policy/** are re-established incrementally as plans land (the as-built rules each plan needs); the MVP
  versions remain in `.mvp/` for reference until superseded.
