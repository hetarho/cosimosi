# cosimosi 스펙 — 색인 (00.overview)

> `spec/`는 cosimosi의 **단일 진실 출처(SSOT)** 다. plan·policy·tech·concept은 *현재 구현된 것*을 기술하는 **as-built** 기준이며, 모바일(Flutter)을 포함한 어떤 클라이언트로도 같은 동작을 재구현할 수 있을 만큼 충분히 적는다. "지금 무엇이 참인가"를 담고 작업 이력·체크리스트는 담지 않는다 — 구현 단계의 체크리스트는 [jobs/](../jobs/) 작업 문서가, 그 너머 이력은 git이 안다.

한 문장: **"일기를 쓰면 → AI가 사건 조각으로 나눠 별로 띄우고 → 의미가 비슷한 기억과 빛으로 잇고 → 함께 떠올린 기억끼리 강해지고, 회상·시간이 별을 다시 빚고, 안 본 별은 어두워지되 사라지지 않는다."**

## spec/ 구조

| 디렉터리 | 무엇 | 질문 |
|---|---|---|
| [concept.md](../concept.md) | 비전·헌법 | **왜** 이렇게 만드나 (충돌 시 여기로 돌아간다) |
| [plan/](.) | 번호별 **as-built 기능 스펙** | **무엇이** 구현돼 있나 (한 파일 = 한 기능 영역, 체크박스 없음) |
| [jobs/](../jobs/) | **구현 작업 문서**(시퀀셜) | plan·changes를 *어떤 단계로* 만드나 (인수조건 + 구현 체크리스트) |
| [changes/](../changes/) | **변경 제안**(기존 수정 WHAT) | 구현된 걸 *어떻게 바꾸나* (사람 작성; 구현은 job, 완료 시 archive) |
| [tech/](../tech/) | 횡단 **엔지니어링 설계** | **어떻게** 짜나 (여러 기능 스펙이 참조) |
| [policy/](../policy/) | 정전(canonical) **규칙·수치** | 무엇이 **참이어야** 하나 (값의 단일 출처) |
| [values.yaml](../values.yaml) | 정전 **스칼라 단일 출처** | 튜닝 수치 1곳 (FE/BE 상수가 여기서 생성 — `pnpm gen:values`) |
| [ops/](../ops/) | 배포·인프라·브랜치 | 도메인 외적 **운영** |

- **plan/ · changes/ · jobs/ (워크플로)** — plan=신규 WHAT, changes=수정 WHAT(둘 다 사람이 인터뷰로 작성), jobs=구현 작업(인수조건+구현 체크리스트, 시퀀셜). **신규:** `/create-plan` → `/create-new-job NN` → `/implement-job MM`. **수정:** `/create-change` → `/create-change-job NN` → `/implement-job MM`. `/implement-job`이 job frontmatter(`type: new|change`)로 분기하고, 끝나면 바뀐 현실을 plan(+policy/tech/values)에 반영한다(변경이면 `changes/` 문서를 `changes/archive/`로). 동시 작업 충돌은 아래 §진행 현황 보드로 막는다.
- **이름 주의** — 여기서 `plan/`은 *as-built 기능 스펙(WHAT)* 이다. Spec Kit·BMAD의 `plan`(=기술 설계 HOW)과 **반대**이며, 우리의 HOW는 [tech/](../tech/)에 있다(모델 대응: plan ≈ OpenSpec `specs/`(living) · changes ≈ OpenSpec `changes/`(델타) · jobs ≈ tasks).
- **tech/** — 한 기능에 속하지 않는 횡단 설계는 여기 한 곳에 두고 기능 스펙이 참조한다(리팩토링용 스펙을 따로 두지 않는다 — 처음부터 이렇게 짠다는 전제). [architecture](../tech/architecture.md)(FSD·렌더링·백엔드·데이터모델) · [state-machines](../tech/state-machines.md)(XState 제어상태) · [xstate-guide](../tech/xstate-guide.md)(작성 규약) · [overlay-shell](../tech/overlay-shell.md)(리스트/탐색 오버레이 셸) · [quality-gates](../tech/quality-gates.md)(빌드·린트·불변 게이트·코어 루프 검증) · [llm](../tech/llm.md)(LLM 공급자 추상화) · [neuroscience](../tech/neuroscience.md)(뇌과학 근거 지도).
- **policy/** — 같은 수치(반감기·τ·A_MIN·R_MIN/MAX·delta 등)가 스펙 곳곳에서 표류하지 않도록 **값의 단일 출처**를 도메인·UX별로 못 박는다. 기능 스펙은 값을 인용하되 SSOT는 policy다. (현재 구현된 상태만 담는다 — 상태 배지 없음.)
- **values.yaml** — 정전 튜닝 스칼라(A_MIN·반감기·공고화 노브 등)의 **기계가 읽는 단일 출처**. FE(TS)·BE(Go) 상수가 여기서 생성된다(`pnpm gen:values`, `pnpm gen`에 포함) — 숫자는 코드가 아니라 이 "밸런스 패치" 파일에서 고친다. 코드·문서가 같은 숫자를 베껴 표류하던 걸 막는다(현재 decay·consolidation부터; 점진 확장). 공식·배열 콘텐츠(테마 CSS·mood 색표)·proto/DB는 제외.
- **ops/** — [deploy-cicd](../ops/deploy-cicd.md)(자동 배포 파이프라인) · [staging-isolation](../ops/staging-isolation.md)(스테이징 환경 격리, 예정) · [observability-activation](../ops/observability-activation.md)(Sentry·PostHog 실계정 연결, 예정) · [branch-strategy](../ops/branch-strategy.md).

> 번호 공백(13·14·16·31·32·33·39·40)은 결번이다 — 그 작업들은 횡단 설계(tech/)나 운영(ops/)으로 흡수됐다. 아래 "흡수된 스펙" 참고. 번호는 안정적 식별자라 재배치하지 않는다.

## 불변 원칙 (constitution — 모든 스펙이 지킨다)

concept.md·tech/architecture.md에서 추출한, 절대 깨면 안 되는 것들:

1. **원본 일기는 불변·영구.** 무엇이 변해도 사용자가 쓴 텍스트는 수정·삭제하지 않는다(별/연결과 분리 저장).
2. **별·시냅스는 우주에서 삭제하지 않는다.** 감쇠는 *밝기*만(최소 밝기 바닥 `A_MIN=0.05`). 잠든 별로 잔존.
3. **별 좌표는 서버가 아니라 클라 force-sim에서 창발.** 서버는 가중치 그래프만 권위. 거리=강함·각도=연결(38), 좌표는 비영속.
4. **FSD 임포트 방향 단방향.** 슬라이스는 아래 레이어만 import, entities 간 교차는 `@x` 공개 API로. `model`/`shared/lib`은 `three`/React/DOM 미의존(모바일 재사용). 플랫폼 분기는 `ui`에서만.
5. **proto=전송 DTO, 도메인은 순수, sqlc=영속.** 도메인 타입에 json/db/proto 태그 금지.
6. **Connect는 unary 전용**(RN 스트리밍 미지원). 회상 강화는 클라 로컬 + unary 배치.
7. **AI는 공급자 추상화 뒤.** 어떤 LLM/임베딩도 교체 가능([tech/llm.md](../tech/llm.md)).
8. **셰이더는 TSL**, 별은 **InstancedMesh**, 시냅스는 **Line2** — 수천 노드 전제.

## 횡단 설계 결정 (충돌 방지 단일 기준)

- **조각 fan-out(21):** `memories.record_id`는 non-unique FK(1 record→N memories). 원본 일기는 1행 불변(헌법1); 조각별 감정은 가변 `memories`에(records 아님). spec 10의 단일 낙관 별은 21이 "조각내는 중 → refetch로 N 별"로 대체.
- **마이그레이션 번호(빌드/배포 순):** `00001` engram 스키마(03, 수정 금지·append-only) · `00002` user_settings·user_emotion_colors(30) · `00003` llm_provider_configs·llm_selection·llm_usage_daily(34) · `00004` memories에 mood/intensity/fragment_index/fragment_text/valence + jobs 키잉(21) · `00005` evolution_history + memories 재성형 4컬럼(23) · `00006` stable_x/y/z 안정 좌표 캐시(27) · `00007` universe_shares(35) · `00008` star_gifts·resonances(36).
- **job 키잉(00004):** `jobs.memory_id` nullable + `record_id`(extract)·`user_id`(consolidate). `KindExtract`(20)·`KindEmbed`·`KindConsolidate`(27).
- **proto `Star` 필드 번호(충돌 금지):** `1–4` memory_id/mood/intensity/last_recalled_at · `5–8` brightness_offset/hue_shift/form_seed_delta/version(23) · `9–10` record_id/fragment_index(28) · `11` resonant(36) · `12` valence(21) · `13` relevance(26). `RecallMemoryResponse`: `1` record · `2` fragment_text(28). 좌표 필드는 영원히 없음(헌법3).
- **밝기 바닥 단일화:** 헌법2의 단일 바닥은 `A_MIN=0.05`. 26의 변조 감쇠도 이 바닥을 재사용.
- **값 SSOT는 policy:** 반감기 λ=ln2/30·τ=0.75·top-k·R_MIN/MAX·co-recall delta 등 정전 수치는 [policy/](../policy/)가 권위.
