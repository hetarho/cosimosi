# 02. RPC 계약 — proto + Connect 서버 · 전송 · 인증 인터셉터

> proto 단일 계약 + Connect 서버(Echo 대체) + FE 전송 + Supabase JWT 검증 인터셉터를 깐다. | 범위: MVP | 선행: 01 | 차단: 04, 05, 10, 11, 12

## 목적 (Why)
엔그램 페이로드(노드 + 가중치 엣지 + 진화하는 비주얼 속성)는 살아있는 그래프라 수기 JSON DTO 동기화보다 protobuf 필드-번호 진화가 안전하다. Echo REST를 connect-go로 교체하고, 웹·React Native가 공유할 단일 `.proto` 계약과 Supabase JWT 검증 인터셉터를 깐다. RPC 비즈니스 로직은 후속 스펙으로 넘긴다.

## 범위
- **포함:**
  - `proto/cosimosi/v1/*.proto`: `Mood` enum(7종) + 메시지 `Star`/`Synapse`/`Record`/`LinkDelta` + 각 Request/Response + 단일 `service MemoryService`의 unary RPC **시그니처만**(`RecordMemory`, `GetUniverse`, `ReinforceLinks`, `RecallMemory`, `ListDormant`). LinkService 등 별도 서비스 없음.
  - `buf.yaml` + `buf.gen.yaml` + Docker codegen(`pnpm buf:gen`) → Go(`protoc-gen-go`/`protoc-gen-connect-go`) + TS(`protobuf-es`/`connect-es`) 타입 생성. 생성 connect 패키지명 = `cosimosiv1connect`.
  - 백엔드 `platform/rpcserver`: `net/http` ServeMux + `h2c` 평문 HTTP/2 + `connectrpc.com/cors` + db 상태 반환 `/health`. **Echo 제거.**
  - connect 핸들러 마운트 **골격**(메서드는 `connect.NewError(connect.CodeUnimplemented, ...)` 스텁).
  - FE `shared/api` connect transport(`createConnectTransport` + JWT `Authorization` 헤더 주입).
  - 백엔드 Supabase JWT 검증 connect interceptor(`user_id` → `ctx`).
- **비목표(Out of scope):**
  - RPC 비즈니스 로직(저장·조회는 04, 회상·강화는 11, 감쇠·잠든 별은 12). 본 스펙은 **시그니처 + 스텁**까지.
  - 스트리밍(원칙 6, unary 전용) — 정의하지 않는다.
  - DB 스키마·마이그레이션(03), AI 워커(05), FE 화면(06–10).
  - FE 로그인 플로우 자체(01) — 본 스펙은 01이 발급한 토큰을 헤더로 실어 보내는 것까지.

## 설계 요점
- **계약(proto, `proto/cosimosi/v1/memory.proto`):** 도메인이 아니라 **전송 DTO**(원칙 5). 좌표 필드 없음 — 좌표는 클라 force-sim에서 창발(원칙 3). 시그니처만, 로직은 후속. 단일 `MemoryService` 5 RPC(LinkService 등 별도 서비스 없음). `Mood` enum 7종이 BE/FE 단일 출처. 필드 번호는 구현 시 부여.
  ```proto
  syntax = "proto3";
  package cosimosi.v1;
  option go_package = "github.com/cosimosi/backend/internal/gen/cosimosiv1;cosimosiv1";

  // Mood enum 단일 출처 — BE/FE 생성 타입 공유. records.mood TEXT는 이 이름 문자열 저장(null 허용).
  enum Mood { MOOD_UNSPECIFIED = 0; JOY = 1; CALM = 2; SAD = 3; ANGER = 4; FEAR = 5; LOVE = 6; NEUTRAL = 7; }

  message RecordMemoryRequest { string body = 1; Mood mood = 2; double intensity = 3;
                                string entry_date = 4; string idempotency_key = 5; } // entry_date=YYYY-MM-DD
  // 별 즉시 등장은 클라 낙관적(memory_id + 폼값으로 StarNode 구성). body/mood는 클라가 폼값 보유.
  message RecordMemoryResponse { string memory_id = 1; }

  // Star = 별 시각 입력(body 없음). 밝기=max(a_min,activation)는 클라 계산, 서버는 last_recalled_at만 권위.
  message Star { string memory_id = 1; Mood mood = 2; double intensity = 3; string last_recalled_at = 4; }
  // Synapse = 가중치 시냅스(무방향 a<b). weight만 권위, 두께/밝기는 클라 셰이더.
  message Synapse { string a_id = 1; string b_id = 2; double weight = 3;
                    string link_type = 4; string last_activated_at = 5; }
  message GetUniverseRequest {}
  message GetUniverseResponse { repeated Star stars = 1; repeated Synapse synapses = 2; }

  message RecallMemoryRequest { string memory_id = 1; }
  // Record = 불변 원본(records 출처). body/entry_date/mood/intensity/created_at.
  message Record { string memory_id = 1; string body = 2; string entry_date = 3;
                   Mood mood = 4; double intensity = 5; string created_at = 6; }
  message RecallMemoryResponse { Record record = 1; }

  // 회상 강화: 클라 로컬 누적 후 unary 배치 flush. delta_weight는 증분.
  message LinkDelta { string a_id = 1; string b_id = 2; double delta_weight = 3; }
  message ReinforceLinksRequest { repeated LinkDelta items = 1; string batch_id = 2; }
  message ReinforceLinksResponse {}

  message ListDormantRequest {}
  message ListDormantResponse { repeated Star stars = 1; }   // Star 재사용(별도 DormantStar 금지)

  service MemoryService {
    rpc RecordMemory(RecordMemoryRequest) returns (RecordMemoryResponse);   // 04
    rpc GetUniverse(GetUniverseRequest) returns (GetUniverseResponse);      // 04 (idempotent)
    rpc ReinforceLinks(ReinforceLinksRequest) returns (ReinforceLinksResponse); // 11 배치 업서트
    rpc RecallMemory(RecallMemoryRequest) returns (RecallMemoryResponse);   // 11 재공고화
    rpc ListDormant(ListDormantRequest) returns (ListDormantResponse);      // 12 잠든 별
  }
  ```
- **codegen(`buf.gen.yaml`):** Go 출력 `backend/internal/gen/cosimosiv1`, TS 출력 `frontend/src/shared/api/gen`. 모듈은 `proto/buf.yaml`, 템플릿은 `backend/buf.gen.yaml`에 있으므로 **리포 루트에서 인자 없는 `buf generate`는 실패**(루트에 buf.gen.yaml 없음). 실행은 `buf generate --template backend/buf.gen.yaml proto`로 템플릿·입력 모듈을 명시한다. ⚠️ `buf`는 **Docker**에서: `docker run --rm -v "${PWD}":/work -w /work bufbuild/buf:latest generate --template backend/buf.gen.yaml proto`.
- **서버 플럼빙(`platform/rpcserver`):** rpcserver는 **서버 플럼빙 전용**(net/http mux + h2c + CORS + 인증 인터셉터 + slog 로깅 인터셉터). MemoryService **구현은 보유하지 않는다** — 실제 핸들러는 `internal/memory/handler.go`(04)다. connect-go 핸들러는 `http.Handler` → `http.NewServeMux`에 `mux.Handle(path, handler)`. 마운트는 생성 패키지 **`cosimosiv1connect`**(proto package `cosimosi.v1`; `memoryv1connect` 아님)의 `cosimosiv1connect.NewMemoryServiceHandler(...)`. `h2c.NewHandler(mux, &http2.Server{})`로 평문 HTTP/2, `connectrpc.com/cors`로 CORS preflight. `/health`는 mux에 직접 등록(`db.Ping` → `{status,version,db}`).
- **인증 인터셉터(`platform/rpcserver/auth.go`):** `connect.UnaryInterceptorFunc`. `Authorization: Bearer <jwt>` 파싱 → Supabase JWT 검증(`SUPABASE_JWT_SECRET` HS256 또는 JWKS) → `sub`(=`user_id`)를 `context`에 주입. 실패 시 `connect.NewError(connect.CodeUnauthenticated, ...)`. user_id 추출 헬퍼는 **`rpcserver.UserIDFromContext(ctx)`**(04/11/12가 이를 import). `/health`는 mux 직접 등록이라 인터셉터 미경유.
- **로깅 인터셉터(`platform/rpcserver/logging.go`):** `connect.UnaryInterceptorFunc`로 구조화 로깅(`log/slog`). 각 RPC의 procedure 이름·user_id·소요시간·결과 코드(에러 시 `connect.CodeOf`)를 slog로 남긴다. 인증 인터셉터와 함께 `connect.WithInterceptors(logging, auth)`로 체이닝(로깅이 바깥, 인증이 안쪽).
- **FE 전송(`frontend/src/shared/api/transport.ts`):** `createConnectTransport({ baseUrl, interceptors:[authHeader] })`. `authHeader`는 01의 `getAccessToken`(`import { getAccessToken } from '@/shared/api'`)으로 access token을 읽어 `req.header.set("Authorization", "Bearer "+token)`. `createClient(MemoryService, transport)`를 `memoryClient`로 노출하되, `index.ts`는 **01이 이미 생성**(supabase·getAccessToken export)하므로 **수정**해 `memoryClient`를 추가 export한다(01의 export 보존, `export *` 금지).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN `pnpm buf:gen`을 실행하면 THE SYSTEM SHALL Docker 안에서 `buf generate --template backend/buf.gen.yaml proto`로 `backend/internal/gen/cosimosiv1/*.go`와 `frontend/src/shared/api/gen/*`(`Mood` enum + 메시지 + 단일 `MemoryService` 클라이언트)를 생성한다.
1.2 IF `.proto`에 스트리밍 RPC가 정의되면 THEN THE SYSTEM SHALL 그것을 거부한다(unary만 — 원칙 6, 본 스펙은 unary 5종만 정의).
2.1 WHEN connect 서버가 기동하면 THE SYSTEM SHALL `net/http` + `h2c`로 평문 HTTP/2를 서비스하고 `MemoryService` 핸들러를 마운트한다.
2.2 WHEN `GET /health`를 호출하면 THE SYSTEM SHALL `{status,version,db}`를 반환하고 `db`는 `db.Ping` 성공 시 `"up"`, 실패 시 `"down"`이다.
2.3 WHEN 미구현 RPC(04/11/12 담당)를 호출하면 THE SYSTEM SHALL `connect.CodeUnimplemented`를 반환한다.
3.1 GIVEN 유효한 Supabase JWT WHEN 보호 RPC를 호출하면 THE SYSTEM SHALL `sub`를 `user_id`로 `ctx`에 주입하고 핸들러로 통과시킨다.
3.2 IF `Authorization` 헤더가 없거나 토큰 서명·만료가 유효하지 않으면 THEN THE SYSTEM SHALL `connect.CodeUnauthenticated`를 반환한다(핸들러 미실행).
4.1 WHEN FE transport가 RPC를 호출하면 THE SYSTEM SHALL 01 세션의 access token을 `Authorization: Bearer`로 주입한다.
4.2 WHEN FE가 임시 unary 호출(예: `GetUniverse`)을 보내면 THE SYSTEM SHALL 서버 스텁이 `Unimplemented`(또는 빈 `GetUniverseResponse{stars:[],synapses:[]}`)를 반환하고 transport·codegen 연동이 동작함을 확인한다.
