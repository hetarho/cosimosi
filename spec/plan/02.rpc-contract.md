# 02. RPC 계약 — proto + Connect 서버 · 전송 · 인증 인터셉터

> proto 단일 계약 + Connect 서버(Echo 대체) + FE 전송 + Supabase JWT 검증 인터셉터를 깐다. | 범위: MVP | 선행: 01 | 차단: 04, 05, 10, 11, 12

## 목적 (Why)
엔그램 페이로드(노드 + 가중치 엣지 + 진화하는 비주얼 속성)는 살아있는 그래프라 수기 JSON DTO 동기화보다 protobuf 필드-번호 진화가 안전하다. 지금은 버릴 코드가 없는 유일한 전환 창이므로 Echo REST를 connect-go로 교체하고, 웹·React Native가 공유할 단일 `.proto` 계약과 Supabase JWT 검증 인터셉터를 깐다. RPC 비즈니스 로직은 후속 스펙으로 넘긴다.

## 범위
- **포함:**
  - `proto/cosimosi/v1/*.proto`: 메시지 `Memory`/`Star`/`Synapse`/`Universe` + `service MemoryService`의 unary RPC **시그니처만**(`RecordMemory`, `GetUniverse`, `ReinforceLinks`, `RecallMemory`, `ListDormant`).
  - `buf.yaml` + `buf.gen.yaml` + Docker codegen(`pnpm buf:gen`) → Go(`protoc-gen-go`/`protoc-gen-connect-go`) + TS(`protobuf-es`/`connect-es`) 타입 생성.
  - 백엔드 `platform/rpcserver`: `net/http` ServeMux + `h2c` 평문 HTTP/2 + `connectrpc.com/cors` + db 상태 반환 `/health`. **Echo 제거.**
  - connect 핸들러 마운트 **골격**(메서드는 `connect.NewError(connect.CodeUnimplemented, ...)` 스텁).
  - FE `shared/api` connect transport(`createConnectTransport` + JWT `Authorization` 헤더 주입).
  - 백엔드 Supabase JWT 검증 connect interceptor(`user_id` → `ctx`).
- **비목표(Out of scope):**
  - RPC 비즈니스 로직(저장·조회는 04, 회상·강화는 11, 감쇠·잠든 별은 12). 본 스펙은 **시그니처 + 스텁**까지.
  - 스트리밍(원칙 6, unary 전용) — 정의하지 않는다.
  - DB 스키마·마이그레이션(03), AI 워커(05), FE 화면(06–10).
  - FE 로그인 플로우 자체(01) — 본 스펙은 01이 발급한 토큰을 헤더로 실어 보내는 것까지.

## 설계 요점
- **계약(proto, `proto/cosimosi/v1/memory.proto`):** 도메인이 아니라 **전송 DTO**(원칙 5). 좌표 필드 없음 — 좌표는 클라 force-sim에서 창발(원칙 3). 시그니처만, 로직은 후속.
  ```proto
  syntax = "proto3";
  package cosimosi.v1;
  option go_package = "github.com/cosimosi/backend/internal/gen/cosimosiv1;cosimosiv1";

  message Memory { string id = 1; string body = 2; string mood = 3;
                   double intensity = 4; string entry_date = 5; string created_at = 6; }
  // Star = 별 시각 상태(밝기=max(a_min,activation)는 클라 계산, 서버는 last_recalled_at만 권위)
  message Star { string memory_id = 1; string mood = 2; double intensity = 3;
                 string last_recalled_at = 4; }
  // Synapse = 가중치 시냅스(무방향 a<b). weight만 권위, 두께/밝기는 클라 셰이더.
  message Synapse { string a_id = 1; string b_id = 2; double weight = 3;
                    string link_type = 4; int32 co_activation_count = 5; string last_activated_at = 6; }
  message Universe { repeated Star stars = 1; repeated Synapse synapses = 2; }

  service MemoryService {
    rpc RecordMemory(RecordMemoryRequest) returns (RecordMemoryResponse);   // 04
    rpc GetUniverse(GetUniverseRequest) returns (GetUniverseResponse);      // 04 (idempotent)
    rpc ReinforceLinks(ReinforceLinksRequest) returns (ReinforceLinksResponse); // 11 배치 업서트
    rpc RecallMemory(RecallMemoryRequest) returns (RecallMemoryResponse);   // 11 재공고화
    rpc ListDormant(ListDormantRequest) returns (ListDormantResponse);      // 12 잠든 별
  }
  ```
- **codegen(`buf.gen.yaml`):** Go 출력 `backend/internal/gen/cosimosiv1`, TS 출력 `frontend/src/shared/api/gen`. ⚠️ `buf`는 **Docker**에서: `docker run --rm -v ${PWD}:/work -w /work bufbuild/buf:latest generate`.
- **서버:** connect-go 핸들러는 `http.Handler` → `http.NewServeMux`에 `mux.Handle(path, handler)`. `h2c.NewHandler(mux, &http2.Server{})`로 평문 HTTP/2, `connectrpc.com/cors`로 CORS preflight. `/health`는 mux에 직접 등록(`db.Ping` → `{status,version,db}`).
- **인증 인터셉터(`platform/rpcserver/auth.go`):** `connect.UnaryInterceptorFunc`. `Authorization: Bearer <jwt>` 파싱 → Supabase JWT 검증(`SUPABASE_JWT_SECRET` HS256 또는 JWKS) → `sub`(=`user_id`)를 `context`에 주입. 실패 시 `connect.NewError(connect.CodeUnauthenticated, ...)`. `/health`는 mux 직접 등록이라 인터셉터 미경유.
- **FE 전송(`frontend/src/shared/api/transport.ts`):** `createConnectTransport({ baseUrl, interceptors:[authHeader] })`. `authHeader`는 01의 세션에서 access token을 읽어 `req.header.set("Authorization", "Bearer "+token)`. `createClient(MemoryService, transport)`를 `index.ts`로만 노출(FSD Public API).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN `pnpm buf:gen`을 실행하면 THE SYSTEM SHALL Docker 안에서 `buf generate`로 `backend/internal/gen/cosimosiv1/*.go`와 `frontend/src/shared/api/gen/*`(메시지 4종 + `MemoryService` 클라이언트)를 생성한다.
1.2 IF `.proto`에 스트리밍 RPC가 정의되면 THEN THE SYSTEM SHALL 그것을 거부한다(unary만 — 원칙 6, 본 스펙은 unary 5종만 정의).
2.1 WHEN connect 서버가 기동하면 THE SYSTEM SHALL `net/http` + `h2c`로 평문 HTTP/2를 서비스하고 `MemoryService` 핸들러를 마운트한다.
2.2 WHEN `GET /health`를 호출하면 THE SYSTEM SHALL `{status,version,db}`를 반환하고 `db`는 `db.Ping` 성공 시 `"up"`, 실패 시 `"down"`이다.
2.3 WHEN 미구현 RPC(04/11/12 담당)를 호출하면 THE SYSTEM SHALL `connect.CodeUnimplemented`를 반환한다.
3.1 GIVEN 유효한 Supabase JWT WHEN 보호 RPC를 호출하면 THE SYSTEM SHALL `sub`를 `user_id`로 `ctx`에 주입하고 핸들러로 통과시킨다.
3.2 IF `Authorization` 헤더가 없거나 토큰 서명·만료가 유효하지 않으면 THEN THE SYSTEM SHALL `connect.CodeUnauthenticated`를 반환한다(핸들러 미실행).
4.1 WHEN FE transport가 RPC를 호출하면 THE SYSTEM SHALL 01 세션의 access token을 `Authorization: Bearer`로 주입한다.
4.2 WHEN FE가 임시 unary 호출(예: `GetUniverse`)을 보내면 THE SYSTEM SHALL 서버 스텁이 `Unimplemented`(또는 빈 `Universe`)를 반환하고 transport·codegen 연동이 동작함을 확인한다.
