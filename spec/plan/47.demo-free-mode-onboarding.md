# 47. demo-free-mode-onboarding - 데모 자유모드 온보딩 + change09 HUD 통합 (as-built)

> 데모 체험은 "누구의 우주를 볼지" 먼저 고르고, 튜토리얼/자유 탐험을 고른 뒤 자유모드로 들어간다. 자유모드는 **빈 우주에서 30일을 배속으로 살아가는 genesis 시뮬레이션**으로 시작해(정적 코퍼스 일괄 시드 폐기 — change 28), 30일을 마치면 환영 안내와 함께 자유 작성·배속 컨트롤이 열린다. 자유모드 HUD는 change09 우주 셸과 같고, 데모 페르소나/시간 버튼을 얹으며 기존 기억 실험실 패널은 내린다. | 범위: FE | 상태: 구현 완료(자유모드·온보딩·genesis; 튜토리얼모드는 plan 48)

## 목적

체험 우주(spec 19)는 비로그인 사용자가 엔그램 우주를 빠르게 만져 보게 하는 장치다. 현재 구현은 `/` 진입과 동시에 기본 페르소나 우주가 열리고, 좌하단 `기억 실험실` 패널 안에 별 추가·시간 이동·페르소나 전환·이론 모달이 한꺼번에 들어 있다. 기능은 풍부하지만, 처음 온 사용자는 "내가 누구의 우주를 보고 있는지"와 "튜토리얼을 따라갈지, 그냥 탐험할지"를 고르기 전에 도구 패널부터 마주한다.

이 plan의 목적은 체험 진입을 두 단계로 바꾸는 것이다.

1. 먼저 `누구의 우주를 탐험해볼까요?`에서 세 페르소나 중 하나를 고른다.
2. 다음으로 `기능 하나하나 알아보기`와 `자유롭게 탐험해보기` 중 하나를 고른다.

본 plan은 **자유롭게 탐험해보기(자유모드)** 와 온보딩을 소유한다. 다른 선택지인 **기능 하나하나 알아보기**는 [plan 48](48.demo-guided-tour.md)의 스포트라이트 투어(튜토리얼모드)로 진입한다 — 그 단계·콘텐츠는 plan 48이 소유한다. (`tutorial_tbd` 상태는 투어 진입 전 안내용 fallback으로만 남아 있다.)

## 범위 / 비목표

**범위**

- 랜딩에서 `enterDemoMode()` 후 `/`로 들어온 최초 체험 세션에 데모 온보딩을 표시한다.
- 온보딩 1단계에서 기존 세 페르소나(`student`/`worker`/`homemaker`) 중 하나를 선택한다.
- 온보딩 2단계에서 튜토리얼 모드와 자유모드 선택을 보여주고, 자유모드 선택 시 change09 이후의 우주 셸로 진입한다.
- 자유모드에서는 기존 `SessionGate`의 체험 안내 문구와 우상단 `체험 종료` pin을 제거하고, 종료 동선은 햄버거 사이드바 안으로 수렴한다.
- 자유모드에서는 기존 좌하단 `기억 실험실`/`뇌과학 이론` 칩과 큰 데모 실험실 컨트롤러를 제거한다.
- change09의 좌상단 테마 버튼 아래에 페르소나 버튼과 시간 버튼을 세로로 배치한다.
- 페르소나 버튼은 팝오버로 세 페르소나를 보여주고, 선택 시 해당 페르소나 우주로 전환한다.
- 시간 버튼은 팝오버로 가상 시계 배속 셀렉터(`1초=1·2·4시간`/정지)와 `처음으로`를 제공한다(연속 흐름, change 24).
- 하단 중앙 `새 별 띄우기` floating 버튼은 데모 자유모드에서도 production 기록 폼을 read-only 프리셋 일기로 열어, 작성 → `별 나누기` 조각화 → `별 띄우기` 흐름을 그대로 체험하게 한다(change 25).

**비목표**

- 튜토리얼 모드의 실제 단계, 코치마크, 이론별 미션, 진행 저장을 만들지 않는다.
- 랜딩의 마케팅 구조나 일반 로그인 우주 진입을 바꾸지 않는다.
- change09에서 정의한 햄버거 사이드바·망원경 시트·HUD 숨김·하단 floating 버튼 자체를 다시 설계하지 않는다. 이 plan은 그 셸이 존재한다고 가정하고 데모 동작만 얹는다.
- 백엔드, proto, RPC, sqlc, DB schema를 바꾸지 않는다.
- 데모 별/시냅스 그래프를 실계정 데이터에 저장하지 않는다.
- 원본 일기 수정/삭제, 별/시냅스 삭제, 서버 좌표 영속화 같은 불변식은 그대로 금지다.

## 설계 요점

### 데모 진입 상태

데모 모드는 기존처럼 `sessionStorage('cosimosi:demo'='1')`를 세션 경계로 삼는다. 여기에 체험 흐름 상태를 하나 더 둔다.

- `not_started`: 데모 플래그는 켜졌지만 아직 최초 온보딩을 완료하지 않은 상태.
- `persona_selected`: 페르소나를 골랐고 모드 선택을 기다리는 상태.
- `free`: 자유모드로 들어간 상태.
- `tutorial`: 사용자가 `기능 하나하나 알아보기`를 골라 [plan 48](48.demo-guided-tour.md) 스포트라이트 투어가 진행 중인 상태.
- `tutorial_tbd`: 투어 안내 fallback(준비 중 카드 + 자유모드 복귀) — 정상 경로에서는 `tutorial`로 진입한다.

이 상태는 데모 세션 전용이며 실계정 세션·서버에 저장하지 않는다. `exitDemoMode()`는 demo flag와 함께 이 온보딩 상태도 비운다. 같은 데모 세션에서 새로고침했을 때는 이미 `free`를 고른 사용자가 다시 온보딩으로 튕기지 않게 유지한다. 단, 랜딩에서 새로 체험을 시작하는 경로는 기존처럼 demo reset을 거쳐 온보딩부터 시작한다.

### 최초 온보딩

`/` 보호 라우트가 데모 모드이고 flow 상태가 `free`가 아니면, 우주 자유모드 HUD를 바로 마운트하지 않고 온보딩 표면을 먼저 보여준다.

1단계는 `누구의 우주를 탐험해볼까요?`라는 질문과 세 페르소나 카드다. 카드는 기존 `demoPersonaList()`의 label/tagline을 사용한다. 선택 즉시 `setDemoPersona(id)`를 저장하고 `resetDemo()`로 해당 코퍼스를 깨끗하게 시드할 준비를 한다.

2단계는 모드 선택이다.

- `기능 하나하나 알아보기`: flow를 `tutorial`로 두고 [plan 48](48.demo-guided-tour.md) 스포트라이트 투어를 자유모드 셸 위에 띄운다.
- `자유롭게 탐험해보기`: flow 상태를 `free`로 저장하고, 빈 우주에서 30일 genesis를 재생하는 자유모드 셸을 렌더한다(아래 "자유모드 = 30일 genesis").

온보딩은 앱 사용법을 길게 설명하는 화면이 아니라 선택 화면이다. 기능 해설, 키보드 단축키, 신경과학 이론 캐러셀은 이 plan의 자유모드 첫 화면에 노출하지 않는다.

### 자유모드 = 30일 genesis (change 28)

자유모드는 정적 페르소나 코퍼스를 한꺼번에 시드하지 않는다. 진입 시 우주는 **비어 있고**, change 24 연속 시계를 `demo_genesis.hours_per_second` 배속(예 24 — 하루≈1초)으로 자동 재생하며 30 simulated days를 하루씩 살아간다.

- **하루 루프**: simulated day마다 페르소나별 `demo_genesis.daily_write_prob`(배열 순서 = `PERSONA_ORDER` student·worker·homemaker)로 난수를 굴려 그 날 일기를 쓸지 정한다(하루 최대 1편). 쓰는 날엔 per-persona 사전분절 **프리셋 풀**(`diary-presets.ts` — 자유모드 `새 별 띄우기`와 같은 콘텐츠 출처)에서 페르소나 토픽 분포(weight)대로 한 편을 뽑아, change 25 조각화 fan-out + change 27 production 동치 엔진(일내 결속·의미 KNN 근사·감정 유사도 링크·거리=강함·밝기=반경)으로 별·연결이 태어난다. `demo_genesis.daily_recall_prob`로 과거 별을 다시 떠올려(재점화) 거리/밝기 드리프트를 살린다.
- **야간 공고화**: 30일 안의 매 밤(04:00 KST 경계)마다 change 24 시계가 야간 공고화(`demoConsolidate`)를 발화해 별이 정리·드리프트한다.
- **난수 경계**: 난수는 genesis **입력**(작성 여부·템플릿 선택·회상 대상)에만 있고(`genesis.ts`가 세션마다 다른 시드로 쥔다 — 데모마다 다른 우주), 엔진 함수는 입력이 같으면 결정론이라 change 27 골든 parity가 깨지지 않는다.
- **관전 잠금**: genesis 동안 `새 별 띄우기` 버튼과 배속 셀렉터는 잠긴다. 카메라 이동/줌/별 클릭(읽기·회상)은 허용된다. 페르소나 전환·`처음으로`는 genesis를 빈 우주에서 처음부터 다시 돌린다(새 난수).
- **30일 후**: 시계가 멈추고 **환영 안내**("30일을 함께 보냈어요 — 이제 당신의 별을 띄워보세요")가 뜬다. 닫으면 `새 별 띄우기`(아래 흐름)와 사용자 배속 컨트롤이 열려 자유모드가 평소대로 동작한다.
- **튜토리얼·겹쳐보기 예외**: 튜토리얼(plan 48)과 겹쳐보기(spec 37)는 회상·잠든 별·일기 목록·공명을 시연하므로 genesis가 아니라 **성숙한 정적 코퍼스**(`simulate(CORPORA)`)를 그대로 시드한다. 데이터 시드는 `getDemoFlow() === 'free'`에서만 genesis 분기를 탄다. 자유모드에서 "둘러보기 다시 보기"로 튜토리얼에 진입하면 정적 코퍼스를 다시 시드하고 genesis를 멈춘다.

### 자유모드 HUD 정리

change09가 완료된 뒤 자유모드의 기본 HUD는 일반 우주 모드와 같은 구조를 공유한다.

- 우측 상단 햄버거 버튼은 남긴다. 데모에서는 사이드바 첫 항목을 `로그아웃` 대신 `체험 종료`로 표시하거나, 동일 위치에서 `exitDemoMode()` + `/landing` 복귀를 수행한다.
- 기존 `SessionGate`가 렌더하던 우상단 체험 안내 문구(`체험 모드 ...`)와 별도 `체험 종료` pin은 제거한다.
- 좌하단 `기억 실험실` 칩, `뇌과학 이론` 칩, 큰 컨트롤러 패널은 자유모드에서 제거한다. 이론 안내와 튜토리얼성 콘텐츠는 후속 튜토리얼 plan으로 넘긴다.
- change09의 상단 중앙 `UI 숨기기`를 누르면 데모 페르소나/시간 버튼과 하단 `새 별 띄우기`도 함께 숨는다. `UI 보이기` 버튼만 남는다.

### 좌상단 페르소나/시간 팝오버

좌상단에는 change09의 테마 버튼이 유지된다. 그 아래에 데모 자유모드 전용 작은 아이콘 버튼 두 개를 세로로 둔다.

- 페르소나 버튼: `UserRound` 성격의 아이콘 버튼. 클릭하면 팝오버가 열리고 세 페르소나를 label/tagline과 함께 보여준다. 현재 선택된 페르소나는 선택 상태로 표시한다. 다른 페르소나를 고르면 `switchDemoPersona(id)`와 같은 리셋 경로를 사용해 가상 시계 0, 추가 별 0, 해당 코퍼스 우주로 전환한다. 자유모드 상태는 유지한다.
- 시간 버튼: `Clock3` 성격의 아이콘 버튼. 클릭하면 팝오버가 열리고 `하루 후로 이동`, `한 달 후로 이동`, `처음으로`를 제공한다. 하루/한 달은 기존 `runTimeSkip(queryClient, 1/30)`과 조용한 force-sim 재안정화 경로를 사용한다. `처음으로`는 선택된 페르소나를 유지한 채 가상 시계와 추가 별을 초기화한다. 온보딩으로 돌아가지는 않는다.

두 팝오버는 바텀시트가 아니라 작은 transient surface다. 다른 우주 표면(망원경 시트, 회상 패널, 사이드바)이 열리면 닫히고, FSD상 순수 demo model이 React/DOM/three를 import하지 않도록 UI와 상태 유틸을 분리한다.

### 하단 중앙 데모 `새 별 띄우기`

change09의 하단 중앙 floating `새 별 띄우기` 버튼은 일반 모드·데모 모드 모두 production 기록 폼(`MemoryForm`)을 연다. 데모 자유모드에서는 같은 폼·상태머신을 **read-only 프리셋 일기** 모드로 재사용한다 — 즉시 랜덤 별을 뱉지 않고, 사용자가 실제 작성 흐름(작성 → `별 나누기` 조각화 → 검토 → `별 띄우기`)을 그대로 체험한다. (genesis 30일 관전 중에는 이 버튼이 잠긴다 — 30일을 마쳐야 열린다. change 28.)

- 버튼을 누르면 페르소나별 프리셋 일기 한 편(다음 편으로 회전)을 골라 본문·날짜를 폼에 주입한다. 본문·날짜는 read-only(프리셋), 날짜는 `demoToday()`(가상 시계 기준 오늘).
- `별 나누기`를 누르면 그 프리셋의 **사전분절 조각**(조각 텍스트 + 조각별 mood/intensity/valence)이 production과 동일한 segmenting → review → fan-out 시퀀스로 펼쳐진다. AI(`SegmentMemory`)만 프리셋으로 대체하고 UI·상태머신·표현은 production 그대로다.
- `별 띄우기`로 확정하면 조각 별이 우주에 태어난다. 생성·연결·밝기·거리는 production 규칙·정전 값을 그대로 쓴다 — 일내 결속(`connection.intraEntryWeight`), 의미 KNN(임베딩 없는 데모는 같은 감정으로 게이트하는 명시적 근사), 감정 유사도 링크(job 37), 거리=강함·밝기=반지름. 데모 전용 링크 가중치·키워드 추정·`DEMO_MAX_FRAGMENTS` 상한은 없다(콘텐츠 프리셋만 데모 고유).
- 제출은 compose 머신의 `submitted` emit → 우주·일기 목록 쿼리 무효화로 즉시 반영한다(production과 같은 경로).

프리셋 작성 흐름은 실서버 `RecordMemory`·`SegmentMemory`·AI 추출·DB 쓰기·별가루 경제와 연결하지 않는다(데모는 서버 미호출). 즉시 랜덤 별 생성(`demoAddRandomStars`)은 폐기됐다.

## 수용 기준 (인수 조건)

1. 랜딩에서 체험 우주를 시작해 `/`에 최초 진입하면 일반 우주 HUD나 기존 데모 실험실 컨트롤러가 먼저 뜨지 않고, `누구의 우주를 탐험해볼까요?` 온보딩이 보인다.
2. 온보딩 1단계에는 기존 세 페르소나(`student`, `worker`, `homemaker`)가 모두 표시되고, 선택한 페르소나가 데모 우주의 데이터 출처가 된다.
3. 페르소나 선택 후에는 `기능 하나하나 알아보기`와 `자유롭게 탐험해보기` 두 선택지가 보인다.
4. `기능 하나하나 알아보기`는 flow를 `tutorial`로 두고 plan 48 스포트라이트 투어로 진입한다(투어 단계·완료/건너뛰기는 plan 48 소유).
5. `자유롭게 탐험해보기`를 누르면 데모 flow 상태가 `free`가 되고, 빈 우주에서 30일 genesis가 재생되는 자유모드 셸이 열린다(genesis 동작은 아래 23~30).
6. 자유모드에서는 기존 `SessionGate`의 체험 안내 문구와 우상단 별도 `체험 종료` pin이 보이지 않는다.
7. 자유모드에서 체험 종료 동작은 햄버거 사이드바 안의 항목으로 가능하며, 실행 시 `exitDemoMode()`와 demo reset 경계를 거쳐 `/landing`으로 돌아간다.
8. 자유모드에서는 좌하단 `기억 실험실` 칩, `뇌과학 이론` 칩, 기존 큰 컨트롤러 패널이 보이지 않는다.
9. 좌상단 테마 버튼 아래에 페르소나 버튼이 있고, 클릭하면 세 페르소나를 고르는 팝오버가 열린다.
10. 자유모드에서 페르소나를 바꾸면 선택된 페르소나의 초기 우주로 전환되고, 가상 시계와 체험 중 추가한 별은 초기화된다. 자유모드 상태는 유지된다.
11. 페르소나 버튼 아래 시간 버튼을 클릭하면 `하루 후로 이동`, `한 달 후로 이동`, `처음으로` 팝오버가 열린다.
12. `하루 후로 이동`과 `한 달 후로 이동`은 기존 하루 단위 배치/조용한 재안정화 경로를 사용하고, 시간 이동 전후 별·시냅스 삭제를 만들지 않는다.
13. `처음으로`는 현재 선택된 페르소나를 유지한 채 가상 시계 0과 추가 별 0 상태로 돌아간다. 온보딩으로 돌아가지 않는다.
14. genesis 30일을 마친 뒤(아래 23~30) 데모 자유모드에서 하단 중앙 `새 별 띄우기` 버튼을 누르면 production 기록 폼이 read-only 프리셋 일기(현재 페르소나·`demoToday()`)로 열린다(change 25). genesis 관전 중에는 이 버튼이 잠겨 있다.
15. `별 나누기`를 누르면 프리셋 사전분절 조각이 production과 동일한 segmenting → review → fan-out으로 펼쳐지고, `별 띄우기`로 조각 별이 태어난다.
16. 펼쳐진 조각 별·연결은 production 규칙·정전 값으로 우주에 반영된다 — 일내 결속·의미 KNN(클라 근사)·감정 유사도 링크·거리=강함·밝기=반지름. 데모 전용 링크 가중치·키워드 추정·상한은 없다.
17. 작성 날짜는 `demoToday()` 기준 오늘이며, 시간 이동 후에는 이동된 가상 날짜가 적용된다. 데모는 서버를 호출하지 않는다.
18. 상단 중앙 `UI 숨기기`를 누르면 데모 페르소나 버튼, 시간 버튼, 하단 `새 별 띄우기`, 햄버거/망원경 등 우주 HUD가 숨고, `UI 보이기` 버튼만 남는다.
19. 실계정 우주에서는 온보딩, 데모 페르소나/시간 버튼이 나타나지 않고, 실계정 `새 별 띄우기`는 빈 작성 폼(자유 입력)을 연다.
20. 데모 flow 상태와 추가 별은 실서버에 기록되지 않고, demo exit/출처 경계 리셋 후 실계정 우주에 섞이지 않는다.
21. 백엔드·proto·RPC·sqlc 변경 없이 FE와 values 생성물만으로 구현된다.
22. build·lint·typecheck·steiger를 통과하고, 기존 데모 회상, 잠든 별, 일기 목록, 시간 이동, 페르소나 전환, 일반 모드 `새 별 띄우기` 작성 흐름에 회귀가 없다.

### genesis (change 28)

23. 페르소나를 고르고 자유모드로 들어가면 정적 코퍼스가 한꺼번에 뜨지 않고, 우주가 **빈 상태에서 시작**해 30일 genesis가 배속(`demo_genesis.hours_per_second`)으로 재생된다.
24. genesis는 simulated day마다 페르소나별 `demo_genesis.daily_write_prob`(순서=`PERSONA_ORDER`)로 일기 작성 여부를 정하고(하루 최대 1편), 페르소나마다 빈도가 눈에 띄게 다르다.
25. 작성하는 날엔 per-persona 프리셋 풀(`pickGenesisDiary`, 자유모드 작성과 공유)에서 토픽 분포대로 한 편을 뽑아 change 25 fan-out + change 27 엔진으로 별·연결이 태어난다. 매 밤(04:00 KST) 야간 공고화가 발화한다. 데모마다 다른 우주가 자란다.
26. genesis 동안 `새 별 띄우기`와 배속 셀렉터는 잠긴다(카메라/줌/별 클릭=읽기·회상은 허용).
27. 30 simulated days를 마치면 시계가 멈추고 **환영 안내**가 뜬 뒤 `새 별 띄우기`·배속 컨트롤이 열린다.
28. genesis 난수는 입력(작성 여부·템플릿·회상 대상)에만 있고 엔진은 결정론을 유지해 change 27 골든 parity가 깨지지 않는다.
29. 페르소나 전환·`처음으로`는 genesis를 빈 우주에서 처음부터 다시 돌린다(새 난수). demo exit 시 genesis 상태가 정리돼 실계정 우주에 섞이지 않는다.
30. 튜토리얼(plan 48)·겹쳐보기(spec 37)는 genesis가 아니라 정적 코퍼스를 시드한다(`getDemoFlow()==='free'`에서만 genesis). 자유모드에서 "둘러보기 다시 보기"로 튜토리얼 진입 시 정적 코퍼스를 다시 시드하고 genesis를 멈춘다.
