# 47. demo-free-mode-onboarding - 데모 자유모드 온보딩 + change09 HUD 통합

> 데모 자유모드를 "누구의 우주를 볼지" 먼저 고르고, 튜토리얼/자유 탐험 선택 뒤 자유모드로 들어가게 재구성한다. change09의 우주 HUD 재배치가 완료됐다는 전제에서, 기존 기억 실험실 패널을 자유모드용 작은 컨트롤로 흡수한다. | 범위: FE | 상태: 기획

## 목적

체험 우주(spec 19)는 비로그인 사용자가 엔그램 우주를 빠르게 만져 보게 하는 장치다. 현재 구현은 `/` 진입과 동시에 기본 페르소나 우주가 열리고, 좌하단 `기억 실험실` 패널 안에 별 추가·시간 이동·페르소나 전환·이론 모달이 한꺼번에 들어 있다. 기능은 풍부하지만, 처음 온 사용자는 "내가 누구의 우주를 보고 있는지"와 "튜토리얼을 따라갈지, 그냥 탐험할지"를 고르기 전에 도구 패널부터 마주한다.

이 plan의 목적은 체험 진입을 두 단계로 바꾸는 것이다.

1. 먼저 `누구의 우주를 탐험해볼까요?`에서 세 페르소나 중 하나를 고른다.
2. 다음으로 `기능 하나하나 알아보기`와 `자유롭게 탐험해보기` 중 하나를 고른다.

이번 범위는 **자유롭게 탐험해보기(자유모드)**만 구현 대상으로 한다. 튜토리얼 모드는 사용자가 볼 수 있는 선택지로 남기되 `TBD`/준비 중 상태이고, 실제 튜토리얼 콘텐츠와 단계 진행은 별도 plan에서 다룬다.

## 범위 / 비목표

**범위**

- 랜딩에서 `enterDemoMode()` 후 `/`로 들어온 최초 체험 세션에 데모 온보딩을 표시한다.
- 온보딩 1단계에서 기존 세 페르소나(`student`/`worker`/`homemaker`) 중 하나를 선택한다.
- 온보딩 2단계에서 튜토리얼 모드와 자유모드 선택을 보여주고, 자유모드 선택 시 change09 이후의 우주 셸로 진입한다.
- 자유모드에서는 기존 `SessionGate`의 체험 안내 문구와 우상단 `체험 종료` pin을 제거하고, 종료 동선은 햄버거 사이드바 안으로 수렴한다.
- 자유모드에서는 기존 좌하단 `기억 실험실`/`뇌과학 이론` 칩과 큰 `DemoSimPanel` 컨트롤러를 제거한다.
- change09의 좌상단 테마 버튼 아래에 페르소나 버튼과 시간 버튼을 세로로 배치한다.
- 페르소나 버튼은 팝오버로 세 페르소나를 보여주고, 선택 시 해당 페르소나 우주로 전환한다.
- 시간 버튼은 팝오버로 `하루 후로 이동`, `한 달 후로 이동`, `처음으로`를 제공한다.
- 하단 중앙 `새 별 띄우기` floating 버튼은 데모 자유모드에서 기록 폼을 열지 않고, 시뮬레이션상 오늘 날짜(`demoToday()`)에 감정 랜덤 별 1~5개를 즉시 추가한다.

**비목표**

- 튜토리얼 모드의 실제 단계, 코치마크, 이론별 미션, 진행 저장을 만들지 않는다.
- 랜딩의 마케팅 구조나 일반 로그인 우주 진입을 바꾸지 않는다.
- change09에서 정의한 햄버거 사이드바·망원경 시트·HUD 숨김·하단 floating 버튼 자체를 다시 설계하지 않는다. 이 plan은 그 셸이 존재한다고 가정하고 데모 동작만 얹는다.
- 백엔드, proto, RPC, sqlc, DB schema를 바꾸지 않는다.
- 데모 별/시냅스 그래프를 실계정 데이터에 저장하지 않는다.
- 원본 일기 수정/삭제, 별/시냅스 삭제, 서버 좌표 영속화 같은 불변식은 그대로 금지다.

## 설계 요점

### 데모 진입 상태

데모 모드는 기존처럼 `sessionStorage('cosimosi:demo'='1')`를 세션 경계로 삼는다. 여기에 체험 흐름 상태를 하나 더 둔다.

- `not_started`: 데모 플래그는 켜졌지만 아직 최초 온보딩을 완료하지 않은 상태.
- `persona_selected`: 페르소나를 골랐고 모드 선택을 기다리는 상태.
- `free`: 자유모드로 들어간 상태.
- `tutorial_tbd`: 사용자가 튜토리얼 모드를 눌렀지만 이번 plan에서는 실제 진입하지 않는 상태. 화면은 준비 중/TBD 안내를 보여주고 자유모드로 돌아갈 수 있게 한다.

이 상태는 데모 세션 전용이며 실계정 세션·서버에 저장하지 않는다. `exitDemoMode()`는 demo flag와 함께 이 온보딩 상태도 비운다. 같은 데모 세션에서 새로고침했을 때는 이미 `free`를 고른 사용자가 다시 온보딩으로 튕기지 않게 유지한다. 단, 랜딩에서 새로 체험을 시작하는 경로는 기존처럼 demo reset을 거쳐 온보딩부터 시작한다.

### 최초 온보딩

`/` 보호 라우트가 데모 모드이고 flow 상태가 `free`가 아니면, 우주 자유모드 HUD를 바로 마운트하지 않고 온보딩 표면을 먼저 보여준다.

1단계는 `누구의 우주를 탐험해볼까요?`라는 질문과 세 페르소나 카드다. 카드는 기존 `demoPersonaList()`의 label/tagline을 사용한다. 선택 즉시 `setDemoPersona(id)`를 저장하고 `resetDemo()`로 해당 코퍼스를 깨끗하게 시드할 준비를 한다.

2단계는 모드 선택이다.

- `기능 하나하나 알아보기`: 이번 plan에서는 준비 중/TBD 상태다. 버튼은 선택 가능하되 실제 튜토리얼 우주로 보내지 않고, "튜토리얼 모드는 다음 단계에서 준비 중" 성격의 짧은 상태를 보여준다.
- `자유롭게 탐험해보기`: flow 상태를 `free`로 저장하고 change09 이후의 우주 셸을 렌더한다.

온보딩은 앱 사용법을 길게 설명하는 화면이 아니라 선택 화면이다. 기능 해설, 키보드 단축키, 신경과학 이론 캐러셀은 이 plan의 자유모드 첫 화면에 노출하지 않는다.

### 자유모드 HUD 정리

change09가 완료된 뒤 자유모드의 기본 HUD는 일반 우주 모드와 같은 구조를 공유한다.

- 우측 상단 햄버거 버튼은 남긴다. 데모에서는 사이드바 첫 항목을 `로그아웃` 대신 `체험 종료`로 표시하거나, 동일 위치에서 `exitDemoMode()` + `/landing` 복귀를 수행한다.
- 기존 `SessionGate`가 렌더하던 우상단 체험 안내 문구(`체험 모드 ...`)와 별도 `체험 종료` pin은 제거한다.
- 좌하단 `기억 실험실` 칩, `뇌과학 이론` 칩, 큰 컨트롤러 패널은 자유모드에서 제거한다. 이론 안내와 튜토리얼성 콘텐츠는 후속 튜토리얼 plan으로 넘긴다.
- change09의 상단 중앙 `UI 숨기기`를 누르면 데모 페르소나/시간 버튼과 하단 `새 별 띄우기`도 함께 숨는다. `UI 보이기` 버튼만 남는다.

### 좌상단 페르소나/시간 팝오버

좌상단에는 change09의 테마 버튼이 유지된다. 그 아래에 데모 자유모드 전용 작은 아이콘 버튼 두 개를 세로로 둔다.

- 페르소나 버튼: `UserRound` 성격의 아이콘 버튼. 클릭하면 팝오버가 열리고 세 페르소나를 label/tagline과 함께 보여준다. 현재 선택된 페르소나는 선택 상태로 표시한다. 다른 페르소나를 고르면 `switchDemoPersona(id)`와 같은 리셋 경로를 사용해 가상 시계 0, 추가 별 0, 해당 코퍼스 우주로 전환한다. 자유모드 상태는 유지한다.
- 시간 버튼: `Clock3` 성격의 아이콘 버튼. 클릭하면 팝오버가 열리고 `하루 후로 이동`, `한 달 후로 이동`, `처음으로`를 제공한다. 하루/한 달은 기존 `runTimeSkip(queryClient, 1/30)`과 조용한 force-sim 재안정화 경로를 사용한다. `처음으로`는 선택된 페르소나를 유지한 채 가상 시계와 추가 별을 초기화한다. 온보딩으로 돌아가지는 않는다.

두 팝오버는 바텀시트가 아니라 작은 transient surface다. 다른 우주 표면(망원경 시트, 회상 패널, 사이드바)이 열리면 닫히고, FSD상 순수 demo model이 React/DOM/three를 import하지 않도록 UI와 상태 유틸을 분리한다.

### 하단 중앙 데모 `새 별 띄우기`

change09의 하단 중앙 floating `새 별 띄우기` 버튼은 일반 모드에서는 기록 폼을 연다. 데모 자유모드에서는 같은 버튼이 즉시 랜덤 별 생성 액션이 된다.

- 클릭 1회마다 랜덤 개수 1~5개를 뽑는다.
- 각 별의 감정은 13종 canonical mood에서 랜덤으로 고른다.
- 날짜는 사용자가 입력하지 않고 `demoToday()`(가상 시계 기준 오늘)로 고정한다.
- 본문은 데모 전용 미리 쓴 문장 코퍼스 또는 fallback 생성 규칙으로 채운다. 기존 `demoAddStar(mood, date)`가 7종 mood만 지원한다면, 구현 단계에서 13종 mood를 모두 처리하는 `demoAddRandomStars({count, date})` 같은 helper를 추가한다.
- 추가 후 `universeInvalidateKey()`와 기존 탄생 애니메이션/연결 생성 경로를 사용해 새 별과 연결을 즉시 보여준다.

랜덤 별 생성은 체험 놀이용이다. 실서버 `RecordMemory`, AI 추출, DB 쓰기, 별가루 경제와 연결하지 않는다.

### Public Interfaces

- `frontend/src/shared/lib/demo/flag.ts`
  - 기존 `isDemoMode`/`enterDemoMode`/`exitDemoMode`/`getDemoPersona`/`setDemoPersona`에 더해 데모 체험 flow 상태를 읽고 쓰는 순수 helper를 둔다.
  - `exitDemoMode()` 또는 demo reset 경계에서 flow 상태를 함께 비운다.
- `frontend/src/shared/lib/demo/data.ts`
  - `demoAddRandomStars(count, date)` 또는 동등 helper를 제공한다. 내부에서 13종 mood와 데모 본문/fallback을 처리하고 기존 연결 생성 규칙을 재사용한다.
- `frontend/src/widgets/demo-sim/model/time-travel.ts`
  - `runTimeSkip`, `resetDemoExperience`, `switchDemoPersona`는 팝오버 컨트롤에서도 재사용한다.
  - `처음으로`가 온보딩이 아니라 현재 페르소나 자유모드 초기 우주로 돌아가는 의미임을 명확히 한다.
- `frontend/src/pages/home/ui/HomePage.tsx` 또는 동등 우주 셸
  - 데모 flow gate를 우주 HUD보다 앞에서 판단한다.
  - change09 HUD 영역에 데모 페르소나/시간 컨트롤과 랜덤 별 생성 동작을 연결한다.

## 수용 기준 (인수 조건)

1. 랜딩에서 체험 우주를 시작해 `/`에 최초 진입하면 일반 우주 HUD나 기존 `DemoSimPanel`이 먼저 뜨지 않고, `누구의 우주를 탐험해볼까요?` 온보딩이 보인다.
2. 온보딩 1단계에는 기존 세 페르소나(`student`, `worker`, `homemaker`)가 모두 표시되고, 선택한 페르소나가 데모 우주의 데이터 출처가 된다.
3. 페르소나 선택 후에는 `기능 하나하나 알아보기`와 `자유롭게 탐험해보기` 두 선택지가 보인다.
4. `기능 하나하나 알아보기`는 이번 plan에서 튜토리얼을 시작하지 않는다. 준비 중/TBD 상태를 보여주고, 사용자가 자유모드 선택으로 돌아갈 수 있다.
5. `자유롭게 탐험해보기`를 누르면 데모 flow 상태가 `free`가 되고, change09 이후의 우주 셸 자유모드가 열린다.
6. 자유모드에서는 기존 `SessionGate`의 체험 안내 문구와 우상단 별도 `체험 종료` pin이 보이지 않는다.
7. 자유모드에서 체험 종료 동작은 햄버거 사이드바 안의 항목으로 가능하며, 실행 시 `exitDemoMode()`와 demo reset 경계를 거쳐 `/landing`으로 돌아간다.
8. 자유모드에서는 좌하단 `기억 실험실` 칩, `뇌과학 이론` 칩, 기존 큰 컨트롤러 패널이 보이지 않는다.
9. 좌상단 테마 버튼 아래에 페르소나 버튼이 있고, 클릭하면 세 페르소나를 고르는 팝오버가 열린다.
10. 자유모드에서 페르소나를 바꾸면 선택된 페르소나의 초기 우주로 전환되고, 가상 시계와 체험 중 추가한 별은 초기화된다. 자유모드 상태는 유지된다.
11. 페르소나 버튼 아래 시간 버튼을 클릭하면 `하루 후로 이동`, `한 달 후로 이동`, `처음으로` 팝오버가 열린다.
12. `하루 후로 이동`과 `한 달 후로 이동`은 기존 하루 단위 배치/조용한 재안정화 경로를 사용하고, 시간 이동 전후 별·시냅스 삭제를 만들지 않는다.
13. `처음으로`는 현재 선택된 페르소나를 유지한 채 가상 시계 0과 추가 별 0 상태로 돌아간다. 온보딩으로 돌아가지 않는다.
14. 데모 자유모드에서 하단 중앙 `새 별 띄우기` 버튼을 누르면 기록 폼이나 날짜/감정 선택 UI 없이 즉시 별이 추가된다.
15. 데모 `새 별 띄우기` 1회 클릭은 랜덤 개수 1~5개의 별을 추가하고, 각 별은 13종 canonical mood 중 하나를 랜덤으로 가진다.
16. 데모 랜덤 별의 날짜는 모두 `demoToday()` 기준 오늘이며, 시간 이동 후에는 이동된 가상 날짜가 적용된다.
17. 데모 랜덤 별은 기존 데모 연결 생성 규칙(같은 날, 같은 mood, 최근 회상 hot 별 등)을 재사용해 우주와 시냅스에 즉시 반영된다.
18. 상단 중앙 `UI 숨기기`를 누르면 데모 페르소나 버튼, 시간 버튼, 하단 `새 별 띄우기`, 햄버거/망원경 등 우주 HUD가 숨고, `UI 보이기` 버튼만 남는다.
19. 실계정 우주에서는 온보딩, 데모 페르소나/시간 버튼, 데모 랜덤 별 생성 동작이 나타나지 않는다.
20. 데모 flow 상태와 추가 별은 실서버에 기록되지 않고, demo exit/출처 경계 리셋 후 실계정 우주에 섞이지 않는다.
21. 백엔드·proto·RPC·sqlc 변경 없이 FE와 values 생성물만으로 구현된다.
22. build·lint·typecheck·steiger를 통과하고, 기존 데모 회상, 잠든 별, 일기 목록, 시간 이동, 페르소나 전환, 일반 모드 `새 별 띄우기` 작성 흐름에 회귀가 없다.
