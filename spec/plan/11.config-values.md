# 11. config values

> Config values - `spec/values.yaml` as the build-time source for generated TypeScript and Go constants.
> Scope: FS | Status: built

## Purpose

cosimosi will have many science, rendering, cache, and product-balance numbers. Those numbers must not drift between
frontend and backend or hide inside feature code. This unit formalizes `spec/values.yaml` as the single source for
build-time tuning constants and generates typed constants for Go and TypeScript.

## Scope / Non-goals

**In scope**

- `spec/values.yaml` creation with inline schema notes and grouping convention.
- `pnpm gen:values` generation to TypeScript and Go outputs, with invalid fixture coverage.
- Root `pnpm gen` integration and generated-code freshness checks through [02.quality-gates](02.quality-gates.md).
- Validation that values are finite and serializable; invalid numbers fail generation.
- Support for numeric scalars, numeric arrays, string scalars, and one-level scalar maps when a feature explicitly
  needs stable id -> value tables.
- Initial Phase 1 entries required by [06.client-data-cache](06.client-data-cache.md).
- Documentation for what belongs in values versus runtime environment, proto, DB schema, CSS/theme content, or code.

**Non-goals**

- No runtime remote config.
- No feature-flag system; flags are owned by [10.observability-and-flags](10.observability-and-flags.md).
- No secrets, URLs, DSNs, API keys, deployment settings, or Supabase configuration in `values.yaml`.
- No product domain formulas, proto contracts, SQL schemas, or theme CSS content.
- No generated mobile-native constants unless mobile has a concrete consumer; TypeScript package output must be
  platform-pure so mobile can import it.

## Grounding

- [ARCHITECTURE.md](../ARCHITECTURE.md) §4 requires build-time generated config from `values.yaml`.
- [00.overview](00.overview.md) §4 says config numbers go in `values.yaml` and are generated, never hardcoded.
- [scripts/gen-values.mjs](../../scripts/gen-values.mjs) is the generator. It exports a testable generator function and
  also runs as the `pnpm gen:values` CLI.

## Design

### Values source

`spec/values.yaml` is grouped by domain/platform area and documents what can be stored there:

```yaml
client_cache:
  default_stale_ms: 30000
  default_gc_ms: 300000
  default_retry_count: 1
  optimistic_rollback_ms: 10000
```

Groups and generated keys are snake_case. Keys are stable once shipped because they generate public constants used by FE
and BE.

### What belongs here

Use `values.yaml` for:

- coefficients;
- thresholds;
- caps;
- defaults;
- debounce/delay windows that shape product behavior;
- numeric arrays used by deterministic calculations;
- stable id -> price/free-kind style maps when a product plan explicitly owns them.

Do not use it for:

- runtime environment;
- secrets or deployment config;
- proto/DB schema;
- formulas;
- UI copy;
- theme CSS content or mood palette content tables;
- feature flags;
- array lengths that can be derived from the array content itself.

### Generated outputs

Generation emits committed, marked files:

- TypeScript constants in the pure `@cosimosi/config` package at `packages/config/src/values.gen.ts`;
- Go constants under the API platform package at `apps/api/internal/platform/values/values_gen.go`.

Generated files are never edited by hand. The root `pnpm gen` script runs values before messages/tokens/proto/sql so
later generated surfaces can depend on build-time constants.

### Validation

The generator fails when:

- the YAML is empty after the file exists;
- a value is non-finite (`Infinity`, `NaN`);
- a numeric array is empty or contains anything other than finite numbers;
- a nested object is not an allowed one-level scalar map;
- a scalar map is empty, nested, or mixes string and numeric values;
- key casing is invalid;
- two accepted keys would generate the same TypeScript or Go identifier;
- generated output is not formatted deterministically.

The generator may skip cleanly only before `spec/values.yaml` exists. Once the file exists, invalid content fails.
`scripts/gen-values.test.mjs` covers non-finite numbers, invalid group/key casing, unsupported nested arrays/maps, and
deterministic output.

## Acceptance Criteria

1. `spec/values.yaml` exists with documented grouping and at least the Phase 1 client-cache entries.
2. `pnpm gen:values` validates the YAML and generates deterministic TypeScript and Go constants.
3. Generated TypeScript constants are platform-pure and importable by both web and mobile.
4. Generated Go constants live under an API platform package, not a product domain package.
5. Non-finite numbers, invalid key casing, and unsupported nested structures fail generation with clear errors.
6. `pnpm gen` runs values generation before proto/sql generation.
7. The full quality gate fails when generated values output is stale.
8. Documentation states what belongs in `values.yaml` and what must remain env/proto/sql/CSS/code.
9. No runtime remote config, secrets, feature flags, or product formulas are introduced.
