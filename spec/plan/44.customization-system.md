# 44. 커스터마이즈 시스템 — 4축 외형 + 소유권·별가루 (customization system)

> 우주의 외형을 **4개의 독립 축**(배경·별·나·시냅스)으로 커스터마이즈하는 통합 체계와, 그 아이템을 **별가루(화폐)로 사서 소유·선택**하는 모델을 정의한다. plan 30(개인 설정)이 "선택한 값"만 서버화했다면, 44는 그 위에 **아이템 카탈로그 + 소유권(unlock) + 지갑**을 얹어 유료화의 골격을 세운다. 색은 의미를 가진 축(별=감정·시냅스=감정)에선 mood를 유지하고, 감정이 없는 축(나)에선 요즘 감정에서 파생한다 — **이번 plan은 "모양/스타일"만 판매**한다. | 범위: FS

> **용어 통일(체계).** 지금까지 "테마(theme)"가 우주 전반 색을 담당했고 별/나/시냅스 커스텀은 제각각이거나 없었다. 이를 **4축 커스터마이즈**로 통일한다:
>
> | 축(axis) | 한국어 | 영문 식별자 | 판매 대상 | 색의 출처 | 무료 1종(기본) |
> |---|---|---|---|---|---|
> | Background | **배경** | `background` | **효과/질감(번들)** | 요즘 mood/감정색 **파생**(고정 hue 없음·검은 우주, change 11) | `vast` |
> | Star | **별** | `star` | **모양(form)** | mood(13감정) — **불변** | `deepfield` |
> | Self | **나** | `self` | **모양(form)** | ambient mood(요즘 감정) **파생** | `mirrorball` |
> | Synapse | **시냅스** | `synapse` | **스타일(style)** | 양끝 별 mood 블렌드 — **불변** | `filament`(현재 렌더) |
>
> - **화폐는 `별가루`(영문 Stardust).** 한국어는 `별가루`로 두고, 기존 배경 점구름 "별먼지"(count 1500)와 **영문만 분리**해 충돌을 없앤다 — 화폐 = 한국어 `별가루`·영문 **Stardust**, 배경 점구름 = 한국어 `별먼지`·영문 **cosmic dust**(코드 `StarDust` 컴포넌트의 영문 명명 정리는 구현 재량). 별먼지는 화폐가 아니다. 별가루 시작 잔액 100, **충전(top-up)·결제(PG)는 이번 plan 비목표** — 처음부터 가진 100으로 사보는 것까지만.
> - **아이템(item) = (축, 종류 kind).** 식별자 `"<axis>:<kind>"`(예 `background:lively`). 축마다 **무료 1종 + 유료 N종**. 무료 종은 누구나 선택 가능(소유 행 없이 묵시적 소유).
> - **[델타 — plan 52 형태 스킨]** 형태 있는 3축(별·나·시냅스)은 판매 단위가 단일 모양/스타일에서 **형태(form)×표면(surface) 2 아이템**으로 분리됐다(`"<axis>:form:<id>"`·`"<axis>:surface:<id>"`, 각 축당 무료 1+유료 N). 선택은 두 축을 합성 wire id `"<form>+<surface>"`로 *기존* 필드(`star_object`·`self_object`·`synapse_style`)에 직렬화해 proto/DB 무변경, 합성의 소유 = 양쪽 sub-item 소유(BE 합성 검증). 분리 전 단일 paid id 구매는 BE가 sub-item으로 확장해 호환. 배경은 단일 effect 유지. 시각/디컴포지션 출처는 [plan/52](52.form-skins.md).
> - **소유권·선택·지갑 = 서버 권위. 시각 정의 = 코드(id로 키잉). 가격·무료여부 = `spec/values.yaml`(FE/BE 생성).** plan 30의 "기본은 클라 소유, 서버는 오버라이드만" 정신을 잇되, 돈이 걸린 소유권/잔액은 클라가 못 속이게 서버가 권위로 든다.

## 목적

cosimosi의 시각 언어는 사용자의 것이고, 그걸 **꾸미고 소유하는 즐거움**을 제품의 수익 모델로 삼는다. 지금은 (1) 배경(테마)·별 모양·나 모양이 서로 다른 체계로 흩어져 있고 용어가 통일돼 있지 않으며, (2) 모든 옵션이 무료라 유료화 여지가 없고, (3) 시냅스는 커스텀이 아예 없다. 44는 이 셋을 **하나의 4축 커스터마이즈 체계 + 아이템/소유권/지갑 모델**로 정돈한다:

- **용어·체계 통일** — "테마"를 **배경**으로 정명하고, 4축(배경·별·나·시냅스)을 같은 아이템/소유권/선택 모델 아래 둔다. 다른 클라이언트(Flutter)도 같은 4축·같은 아이템 id·같은 RPC로 재구현 가능하게.
- **유료화 골격** — 축마다 무료 1종 + 유료 N종. 유료 아이템은 **별가루로 구매(unlock)** 해야 선택할 수 있다. 충전·결제는 후속 plan에 맡기고, 이번엔 **시작 잔액 100**으로 구매 흐름까지 검증한다.
- **색의 철학 보존** — "감정의 색"(별=mood)은 깨지 않는다. 색을 파는 대신 **모양/스타일**을 판다. 감정이 없는 "나"만 요즘 감정(ambient)에서 색을 파생해 테마 귀속을 끊는다.
- **맛보기 = 전환 동력** — 랜딩/사인인을 **전체 라이브 플레이그라운드**로 만들어, 미가입자가 4축 전부(유료 포함)를 자유로이 바꿔보고 "가입하면 내 우주에 적용"으로 넘어오게 한다.

## 범위 / 비목표

**범위**
- **4축 커스터마이즈 체계**: 배경·별·나·시냅스를 같은 아이템/소유권/선택 모델로 통일(용어·식별자 포함).
- **아이템 카탈로그**: 축별 종류(kind)와 그 시각 정의(코드), 무료여부·가격(values.yaml). 아이템 id `"<axis>:<kind>"`.
- **소유권·지갑(서버)**: 별가루 잔액(시작 100), 소유한 유료 아이템 집합, 구매(별가루 차감 + 소유 부여) RPC. 선택(per-axis 활성 id)을 서버화(plan 30 `UpdateSettings` 확장 — 나·시냅스 축 추가).
- **배경(Background) 확장**: 배경이 색만이 아니라 **텍스처·요소를 포함한 번들**이 된다(현재 `theme`의 bg+palette를 backdrop 번들로 일반화).
- **나(Self) 전면 개편**: 별처럼 독특한 형태들로 재설계, **색은 ambient mood 파생**(테마 귀속 제거), 선택을 서버화.
- **시냅스(Synapse) 스타일 커스텀**: 연결선 스타일(예: 가닥/흐름/입자/빔)을 아이템으로. 색은 양끝 mood 블렌드 유지.
- **별(Star)**: 이미 모양 4종이 선택되므로 **아이템/소유권 모델로 편입** + 신규 유료 모양 추가 여지. 색은 mood 유지(변경 없음).
- **스위처(인벤토리) UI**: `AppearanceSwitcher`를 4축 인벤토리로 확장 — 잔액 표시, 잠금 배지·가격·구매 버튼, 소유/무료만 선택 가능.
- **사인인/초대 플레이그라운드**: 4축 전부를 라이브로 미리보는 미니 코스모스(배경 + 별 + 나 앵커 + 시냅스 표본). 랜딩은 배경 + 히어로 별만(나·시냅스 표본 제외 — 순수 마케팅 백드롭). 미인증은 전부 잠금 해제·미저장(스위처는 어디서나).

**비목표**
- **충전(top-up)·결제(PG/인앱결제)·환불·정산은 안 한다.** 별가루는 시작 100 고정, 잔액이 0이면 더는 못 산다. 충전 흐름은 후속 plan.
- **별·시냅스 "색" 판매는 안 한다.** 색은 의미(감정)라 팔지 않는다 — 별=mood, 시냅스=양끝 mood. (감정이 없는 "나"는 ambient 파생이라 색 판매가 아니라 자동 파생.)
- **감정색(`MOOD_PALETTE`) 자체의 판매/커스텀은 비목표**(plan 30의 per-mood 오버라이드 경로는 그대로 둔다 — 44는 모양/스타일 축만).
- **구체적인 신규 시각 디자인(셰이더·형태)의 최종본은 plan이 못 박지 않는다** — 축 구조·아이템 모델·색 규칙만 확정하고, 형태/텍스처/스타일의 픽셀 디테일은 각 job의 디자인 반복으로(육안 튜닝).
- **관리자 콘솔(34)에서 아이템/가격 편집은 비목표**(가격은 values.yaml 단일 출처 — 후속에 콘솔이 얹힐 수 있게 데이터만 정돈).
- **선물/공유 우주(35·36)에서 타인 커스텀 적용 변경은 비목표**(공유 우주는 이미 소유자 `Settings` 스냅샷을 입힌다 — 새 선택 축이 늘어날 뿐 모델 동일).

## 설계 요점

### A. 아이템·소유권·지갑 모델 (체계)

- **아이템 id**: `"<axis>:<kind>"`. 축은 `background|star|self|synapse`, kind는 코드 카탈로그가 정의하는 종류 슬러그(영문 kebab/소문자). 예: `background:vast`(무료)·`background:aurora-veil`(유료)·`star:liquid`·`self:prism-cube`·`synapse:dendrite`. id는 **안정 식별자**(소유권·proto·values 키)이므로 재명명/재배치 금지(별 `memory_id` 시드 재현성과 동급 규율). 카탈로그에서 제거된 레거시 id(`self:core`·`self:well`·`synapse:beam`·`synapse:flow`)는 신규 구매/노출 대상이 아니고 로드/렌더에서 축 기본값으로 정규화된다(데이터 행 삭제 없음, change 11).
- **무료 vs 유료**: 축마다 **정확히 1종 무료**(아래 기본 1종 표). 무료 종은 소유 행 없이 **묵시적 소유** — 누구나 선택 가능. 유료 종은 구매 전엔 **잠금**(선택 불가, 스위처에서 가격·구매 버튼만).
- **소유(ownership)**: 사용자가 **구매한 유료 아이템 id 집합**. 서버 권위(`user_owned_items`). 무료 종은 이 집합에 넣지 않는다(묵시).
- **선택(selection)**: 축마다 **현재 활성 아이템 id 1개**. plan 30의 `user_settings`(theme·star_object)에 **`self_object`·`synapse_style` 두 축을 추가**해 4축 전부 서버화한다(나 축은 지금 localStorage 전용 → 서버로 승격). null = 클라 기본.
- **지갑(wallet)**: 별가루 잔액(`user_wallet.stardust`). 사용자가 처음 인벤토리를 조회할 때 행이 없으면 **시작 잔액(`starting_stardust`=100) 1회 시드**(멱등 — 없을 때만 INSERT). 구매 시 차감, 차감 결과 음수가 되는 구매는 거부.
- **권위 분리(사용자 답변 반영 — 혼합)**:
  - **서버(DB) 권위**: 잔액·소유권·선택. 돈/잠금은 클라가 못 속인다.
  - **코드(FE) 정의**: 각 아이템의 **시각 정의**(셰이더/형태/스타일/`swatch`/표시 이름/태그라인) — id로 키잉.
  - **values.yaml(FE+BE 생성)**: **가격·무료여부·시작 잔액** = config 스칼라. 서버가 구매 검증에 쓰고 FE가 가격/잠금 표시에 쓴다(단일 출처 — 가격이 FE/BE로 갈라져 표류하지 않게). 아이템 *종류의 enumeration*은 코드 카탈로그(시각 정의)가 가지며, BE는 "유효 id 화이트리스트"만 미러(plan 30 `validThemes`/`validObjects` 확장).

### B. RPC 계약 (`SettingsService` 확장 — `proto/cosimosi/v1/memory.proto`)

plan 30의 `SettingsService`를 커스터마이즈 바운디드 컨텍스트로 확장한다(모두 인증 필수·unary).

- **`GetInventory`**(`NO_SIDE_EFFECTS`, HTTP GET): `GetInventoryResponse { int64 stardust = 1; repeated string owned_item_ids = 2; }`. 지갑 없으면 100 시드 후 반환. 무료 종은 응답에 넣지 않는다(묵시 소유 — 클라가 카탈로그로 안다).
- **`PurchaseItem`**: `PurchaseItemRequest { string item_id = 1; }` → `PurchaseItemResponse { int64 stardust = 1; repeated string owned_item_ids = 2; }`(구매 후 새 상태). 서버 검증(한 트랜잭션, 부분 적용 금지):
  1. `item_id`가 알려진 유료 아이템인가(values 가격표에 존재 & 무료 아님). 아니면 `InvalidArgument`.
  2. 이미 소유면 `FailedPrecondition`(`ErrAlreadyOwned`) — 이중 차감 금지.
  3. 잔액 ≥ 가격인가. 아니면 `FailedPrecondition`(`ErrInsufficientFunds`).
  4. 통과 시 `stardust -= price` + `user_owned_items` UPSERT(멱등). 검증 실패 시 어떤 행도 안 바뀐다.
- **`GetSettings`/`UpdateSettings`**(plan 30 확장): 선택 4축. `UpdateSettings`는 **선택하려는 아이템을 소유(또는 무료)하지 않으면 거부**(`FailedPrecondition` `ErrNotOwned`) — 잠긴 아이템을 API로 우회 선택 못 한다.

proto 메시지 변경(필드 번호 안정 — 추가만):
- `Settings { string theme = 1; string star_object = 2; repeated EmotionColor emotion_colors = 3; string self_object = 4; string synapse_style = 5; }` (4·5 신규 append; `theme`=배경 선택 id로 의미 재정의, 와이어 이름은 호환 위해 유지).
- `UpdateSettingsRequest`에 `optional string self_object = 4; optional string synapse_style = 5;` append.
- 신규: `GetInventoryRequest{}`/`GetInventoryResponse`·`PurchaseItemRequest`/`PurchaseItemResponse`.

### E. 4축 상세

#### 배경 (Background) — 색 + 텍스처 + 요소 번들
- 현재 `theme`은 우주 캔버스 배경색(`themeBg`) + 랜딩/사인인 fluid 팔레트(`paletteForTheme`, 43)만 담당. 배경 아이템은 이를 **하나의 backdrop 번들**로 일반화: `{ bg(딥 베이스색), palette(fluid 색), texture/elements(독특한 요소 — 예: 추가 nebula 결, 별먼지 밀도·결, grain 강도, 특수 요소 레이어) }`.
- 무료 `vast`는 현재의 최소 배경. 유료 배경은 **독특한 텍스처/요소**를 더한다(구체 디자인은 job). 배경 변경은 여전히 **별의 mood 색을 바꾸지 않는다**(universe.md §테마↔별색 분리 유지).
- 소비처: `widgets/universe-canvas`(우주 배경)·`widgets/cosmos-scene`(비-우주 배경, 43) 둘 다 배경 번들을 prop으로 받게 일반화.

#### 별 (Star) — 모양만
- 색=mood 유지(변경 없음, star.md). 모양(form) 4종이 이미 선택되므로 **아이템/소유권 모델로 편입**(무료 `deepfield` + 유료 `aurora`/`liquid`/`ember`, 이후 신규 모양 추가). 잠긴 모양은 스위처에서 잠금·구매.

#### 나 (Self) — 모양 개편 + ambient 색
- **모양 전면 개편**: "나"가 별처럼 독특해지도록 형태를 재설계(change 11 — `mirrorball`(무료·반사구)·`prism-cube`(굴절 큐브)·`neuron-bloom`(soma+dendrite); 레거시 nebula-heart/core/well 제거·mirrorball 정규화). 셰이더 디테일은 후속 비주얼 폴리시.
- **색 = ambient mood 파생**(테마 귀속 제거): 자아 별의 **자기 몸체 색**을 `themeAccent(theme)` 대신 **요즘 감정(ambient, 25)** 에서 파생(`deriveAmbient`/`ambientToRgb`). "나 = 지금의 나"를 색으로 드러낸다. 데이터 없음(빈 우주·랜딩)·미인증이면 중립/배경 accent 폴백.
  - **주의(spec 03 보존)**: 자아 별이 다른 별에 **던지는 빛(self-light 반사 채널)은 중립색 유지** — 우주의 "무드 색" 소유권은 `AmbientNebula` 풀이 가진다(이중 주입 금지). 44는 **자아 별 자신의 몸체 색만** ambient 파생으로 바꾼다(던지는 빛 아님).
- 선택 서버화: localStorage 전용 → `user_settings.self_object` + `UpdateSettings.self_object`로 승격.

#### 시냅스 (Synapse) — 스타일
- 색=양끝 별 mood 블렌드 유지(synapse.md). **스타일(style)** 을 아이템으로: 무료 `filament`(꼬인 가닥 다발) + 유료 `beam`(곧고 두꺼운 한 줄기)·`flow`(크게 휜 흐름)·`particle`(가는 점선). 스타일은 선의 **생김새(지오메트리 + 셰이더 표현)** 를 바꾸되 색·가중치 시각(weight→밝기/alpha/펄스)·삭제금지(점선도 비드 사이 바닥 불투명) 불변식은 유지. Line2/TSL·전역 스칼라 한계(per-edge 셰이더 두께 변조 불가·스타일당 단일 머지 드로우·정점 attribute ≤8, synapse.md)는 그대로.
- 선택 서버화: `user_settings.synapse_style` + `UpdateSettings.synapse_style`(신규 축).

### I. 견고성·미인증 (헌법 견고성)
- 미인증·오프라인·체험·로딩 중: 4축 모두 클라 기본값(배경=vast·별=deepfield·나=mirrorball·시냅스=filament)으로 렌더, 별가루/소유권 없이도 안 깨진다.
- 알 수 없는/손상 선택 id는 축 기본값으로 폴백(plan 30 `merge` 정신 — store가 merge/applyServerSettings에서 검증).
- **잠금 강제의 위치(as-built):** 우주 렌더는 **선택값을 그대로 그린다**(알 수 없는 id만 축 기본값 폴백). 잠긴(미소유 유료) 아이템 강제는 **저장 시점**(드래프트 저장 = 미구매 아이템 일괄 구매 후에야 영속; 잔액 부족이면 저장 막힘) + **서버**(`UpdateSettings`가 미소유 선택을 `ErrNotOwned`로 거부)에서 한다 — 홈 편집기는 잠긴 아이템도 **미리보기 선택**을 허용하되 저장 전까진 서버에 안 올라간다. 렌더에서 소유권으로 다시 폴백하지 **않는다** — 그러면 (a) 기존 사용자의 우주가 무료 기본값으로 무더기 회귀하고, (b) 공유 우주(방문)에서 *소유자* 선택이 *방문자* 빈 소유로 가려지기 때문(회귀). 플레이그라운드(미인증·체험)는 전부 잠금 해제·미리보기 허용·미저장.

### J. job 분할 제안
규모가 크니 독립 검증 단위로(43처럼):
- **J1 — 골격**: 별가루 지갑 + 소유권 + 구매 RPC + 선택 4축 서버화(proto·BE·migration 00010·FE store/inventory) + 스위처 잠금·가격·구매 UI. 배경·별 선택은 이미 있으니 자연 편입. values `customization` 그룹.
- **J2 — 배경(Background)**: theme→Background 정명, backdrop 번들 일반화(텍스처/요소), 유료 배경 추가.
- **J3 — 나(Self) 개편**: 형태 재설계·확장 + ambient 색 파생 + 선택 서버화.
- **J4 — 시냅스(Synapse) 스타일**: 스타일 축 신설 + 스타일 아이템.
- **J5 — 플레이그라운드**: 랜딩/사인인 미니 코스모스(self·synapse 표본) + 전체 잠금 해제·미저장 + CTA.

## 수용 기준 (인수 조건)

1. 인증 사용자가 `GetInventory`를 처음 호출하면 별가루 잔액 `starting_stardust`(=100)가 시드되어 반환되고, 소유 아이템은 빈 목록이다(무료 종은 응답에 없음 — 묵시 소유).
2. `PurchaseItem`은 한 트랜잭션에서 (a) 알 수 없거나 무료인 아이템 → `InvalidArgument`, (b) 이미 소유 → `FailedPrecondition`, (c) 잔액 < 가격 → `FailedPrecondition`로 거부하고 **어떤 행도 바꾸지 않는다**. 통과 시 잔액이 가격만큼 줄고 소유 집합에 해당 id가 추가된 새 상태를 반환한다.
3. 별가루 잔액은 구매로만 줄고 음수가 되지 않으며, 충전·재설정 경로는 존재하지 않는다(이번 plan 비목표 — 시작 100 외 증가 수단 없음).
4. `UpdateSettings`로 **소유하지 않은(무료도 아닌) 유료 아이템**을 선택하려 하면 `FailedPrecondition`(ErrNotOwned)로 거부되고 선택이 바뀌지 않는다. 무료 종·소유한 유료 종은 정상 선택·영속된다.
5. 선택은 **4축 전부**(배경=`theme`·별=`star_object`·나=`self_object`·시냅스=`synapse_style`) 서버에 저장되어 새로고침·타 기기에서 같은 선택으로 로드된다(나 축이 더는 기기-로컬이 아니다).
6. 별 색은 mood(13감정) 팔레트로만 정해지고, 어떤 별 모양 아이템을 선택해도 색이 감정을 따른다(모양만 바뀐다 — "감정의 색" 보존).
7. 자아 "나" 별의 **몸체 색**은 요즘 감정(ambient mood)에서 파생되고 테마/배경에 귀속되지 않는다. 데이터 없음·미인증이면 중립/배경 accent로 폴백한다. 자아 별이 **다른 별에 던지는 빛은 중립색**으로 남는다(spec 03 보존 — mood 이중 주입 없음).
8. 시냅스 스타일을 아이템으로 선택할 수 있고 스타일별로 선의 생김새(가닥 다발/곧은 빔/크게 휜 흐름/점선)가 또렷이 갈린다. 색은 여전히 양끝 별 mood 블렌드이며 weight→밝기/alpha/펄스 시각과 삭제금지(헌법2 — 점선도 비드 사이 바닥 불투명) 불변식이 유지된다.
9. 배경 아이템은 색이 아니라 **효과/질감 번들**로 적용되고(change 11 — 고정 hue 없음·검은 우주), 배경 변경이 별의 mood 색을 바꾸지 않는다. 선택 배경의 `effect`가 `UniverseNebula`에서 서로 다른 절차적 셰이더 경로(haze·nebula·waves·aurora·static·caustics·ridges)를 고르고, 그 위에 요즘 감정색(상위 `emotionSlots`개·Bjork R-비중·사용자 감정색)이 주 색으로 칠해진다(spec 07; 별색·깊이 불간섭). 미인증·빈 우주·로딩은 중립 딥스페이스 fallback.
10. 홈 편집기는 좌상단 별가루·테마 알약(실로그인은 잔액 표시)이 여는 **집중 모달**(change 06)이며, 최상위 `스킨`/`감정 색` 2탭으로 나뉜다(독립 `외형` 헤더 텍스트 없음, change 09). **`스킨` 탭** = 상단 **고정 라이브 프리뷰**(나 + 별 3개 + 강·약 시냅스, 4축 즉시 반영) + 축 탭(`배경/별/광원/시냅스`) + 항목별 **실제 모형 썸네일**(라이브 미니 3D, 그라디언트 swatch 아님) + 하단 **저장** command bar; 구매/별가루/저장 규칙은 무변. **`감정 색` 탭** = 같은 표면에서 13감정 색 편집(45 `EmotionColorEditor` 재사용)·`UpdateSettings.emotion_colors` 풀세트 저장. 모달이 열리면 우주 캔버스는 언마운트된다(컨텍스트 양보). 잠긴 유료 아이템엔 자물쇠 + `저장 시 N 별가루`, 잠긴 것도 **미리보기 선택**된다. **저장**(하단, 구매 포함): 미구매가 있으면 `저장 · N 별가루`(합계가)·없으면 `저장`, 미구매분을 먼저 사고 4축을 영속·닫음(잔액 부족이면 막힘). **뒤로**(헤더): 드래프트가 마지막 저장과 다르면 미저장 경고 모달, "나가기"는 드래프트를 버린다. `감정 색` 저장으로 모달을 떠날 때도 커밋되지 않은(미구매) 스킨 드래프트는 되돌려 미결제 스킨이 활성으로 새지 않는다.
11. 축마다 정확히 1종이 무료(묵시 소유)이고 나머지는 유료(잠금)다 — 무료 1종 매핑(배경 vast·별 deepfield·나 mirrorball·시냅스 filament)이 values 생성 상수로 단일 출처화된다.
12. 사인인·초대 플레이그라운드에서 미인증자가 4축 전부(유료 포함)를 자유로이 바꿔 미니 코스모스(배경+별+나 앵커+시냅스 표본)에 즉시 반영된다(랜딩은 배경+별만 미리보인다). 아무것도 서버에 저장되지 않고 별가루도 차감되지 않는다.
13. 미인증·오프라인·체험·로딩 중에도 4축 클라 기본값으로 렌더가 깨지지 않고, 알 수 없는/잠긴 선택은 축 기본값으로 폴백한다.
14. 가격·무료여부·시작 잔액은 `spec/values.yaml`에서 생성된 상수로 FE/BE가 공유한다(FE/BE 하드코딩·이중 출처 없음). 아이템 id는 `"<axis>:<kind>"` 안정 식별자다.
15. 모든 신규 RPC(GetInventory·PurchaseItem)와 확장 RPC(UpdateSettings)는 미인증이면 `Unauthenticated`로 거부하고 unary다(헌법6). build·lint·vet·steiger 통과, 우주 셸·기존 화면 회귀 없음.

## 정책 영향 (policy / values)

> 아래는 **구현 완료 시** 각 policy 문서에 반영될 규칙(지금은 plan이 출처, 미리 policy에 기성사실로 적지 않음). 구현은 /cosimosi:create-plan-job 44 → /cosimosi:implement-job.

- **신규 `policy/domain/customization.md`(구현 시 생성)** — 정전 규칙:
  - 4축(배경·별·나·시냅스)·아이템 id `"<axis>:<kind>"`·축당 무료 1종(묵시 소유)·유료 N종(별가루 구매로 unlock).
  - 별가루: 시작 100, 충전·결제 없음(차감만), 음수 불가. 소유권·선택·잔액=서버 권위, 시각 정의=코드, 가격·무료=values.
  - 색 규율: 별=mood 불변·시냅스=양끝 mood 불변·나=ambient 파생(던지는 빛은 중립)·배경=요즘 mood/감정색 파생(고정 hue 없음·검은 우주, change 11; 별색 불간섭). "모양/스타일만 판매".
  - 선택은 소유(또는 무료)한 아이템만. 잠긴 아이템 선택 거부.
- **`policy/domain/universe.md`** — "테마"→"배경(Background)" 정명, 배경이 색+텍스처+요소 번들임. 화폐 "별가루"와 배경 점구름 "별먼지"는 별개(별먼지는 화폐 아님). 자아 별 몸체 색=ambient 파생(던지는 self-light 반사는 중립 유지).
- **`policy/domain/star.md`** — 별 모양이 커스터마이즈 아이템(무료 deepfield + 유료), 색=mood 불변(변경 없음 명시).
- **`policy/domain/synapse.md`** — 시냅스 스타일이 커스터마이즈 아이템(무료 filament + 유료), 색=양끝 mood·삭제금지 불변식 유지.
- **`spec/values.yaml` — 신규 `customization` 그룹(config, 코드 하드코딩 금지 → 생성 상수)**:
  - `customization.starting_stardust: 100` — 신규 사용자 시작 별가루.
  - `customization.free.background|star|self|synapse` — 축별 무료 아이템 kind(`vast`·`deepfield`·`mirrorball`·`filament`, change 11). 4 스칼라(문자열).
  - `customization.price.<item_id>` — 유료 아이템 id별 가격(별가루) 스칼라(예 `price.background:aurora-veil: 40`). 가격 수치는 구현 시 균형 조정(시작 100으로 2~3개 구매 가능 수준).
  - (셰이더 계수·팔레트 hex·`swatch` 등 시각 *콘텐츠*는 values 제외 범주 — 코드/머티리얼 소관. 카탈로그 *길이*는 배열 파생.)
- **`tech/architecture.md`** — `SettingsService`가 커스터마이즈 컨텍스트로 확장(지갑·소유·구매), `entities/appearance`가 선택 4축+인벤토리 보유, 카탈로그 시각 정의=entities·가격=values·소유=DB 분리 메모.
