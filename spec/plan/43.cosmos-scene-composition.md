# 43. CosmosScene — 공유 WebGL 배경 합성 (cosmos-scene-composition)

> 앱의 **모든 비-우주 배경(사인인·초대·랜딩)** 을, 우주 셸이 이미 쓰는 "한 R3F 캔버스" 패턴으로 끌어올린다. 절차적 fluid 배경(dim nebula 뒤 + 어두운 구름 앞) + 별 프리미티브(plan 42) + halo + 트윙클 별먼지 + grain을 한 씬에 담는 **재사용 합성 widget `CosmosScene`** 을 만들고, 페이지들이 prop으로 데이터·팔레트만 주입해 공유한다. | 범위: FE | 상태: 구현됨(J1·J2·J3, jobs 05·06·07)

> **구현된 형태(as-built):** `widgets/cosmos-scene/`(`CosmosScene` + `StarVisual`/`CosmosSceneProps`)가 한 R3F 캔버스에 `renderOrder`로 [**뒤 fluid**(radial, `VALUES.cosmos.backBrightness`로 어둡게 — 딥스페이스) → 트윙클 InstancedMesh 별먼지 → **별**(`buildStarBody` uniform 바인딩) + **halo**(또렷한 소프트 글로우) → **앞 fluid**(dark 모드 = 어두운 구름이 별 앞을 안개처럼 가림, 가산광 아님)]를 합성한다. **글로우는 halo가** 낸다 — bloom은 기본 OFF(과한 글로우로 별이 묻히지 않게; `bloom` prop으로 켤 수 있음). 풀스크린 fit ortho 카메라(`manual=true`), `frameloop='demand'` + 스로틀 rAF(`VALUES.cosmos.fpsCap`), `quality`(또는 기기 등급: WebGL2 폴백/저코어)·reduced-motion 다운그레이드.
>
> **(spec 03 — changes/03, job 08):** 두 가지가 갱신됐다. (1) **배경 별 조명:** 자아-별·그래프가 없는 배경 씬은 `StarMesh`가 **우상단 평행광**(positional=0, `VALUES.starLighting.backdrop_light_dir`)을 주입해 crystal 면/엣지를 드러낸다 — 위치 고정 점광·fake key-light는 안 굽고, 평행광이라 화면-비대칭 없음. 브랜드 별은 자가발광 full(glow=1) + 반사. (2) **fluid 배경 비율 고정:** plane이 `scale=[aspect,1]`이라 셰이더 노이즈 도메인 x를 같은 aspect로 보정(`setAspect`) — 화면 너비가 바뀌어도 구름이 안 늘어난다(어떤 비율에서도 형태 고정). 별 빛 모델 = [policy/domain/star.md](../policy/domain/star.md) §밝기·감쇠 3채널.
>
> 공유 부품: `BloomPass`·`halo`를 `widgets/*` → **`shared/ui`로 이동**(universe-canvas와 공유). fluid 머티리얼은 `shared/ui/fluid-material.ts`(`buildFluidMaterial` — `palette`/`brightness`/`dark`/`radial`/`octaves` 옵션) + `paletteForTheme(theme)`(vast/lively/calm 팔레트, appearance 미import — 디커플드). grain은 CosmosScene DOM 래퍼의 CSS 오버레이(WebGPU 컨텍스트 1개 유지).
>
> 적용: **사인인·초대·랜딩** 모두 `CosmosScene` 하나로(테마별 `paletteForTheme(theme)`, 별 앵커는 엠블럼 자리 — 정밀 정렬·미감은 육안 튜닝 대상). 사인인·초대에 **테마 플로팅 스위처**(`AppearanceSwitcher`) 추가. 노브는 `spec/values.yaml`의 `cosmos` 그룹(`scene_dpr`·`fps_cap`·`fluid_octaves`·`back_brightness`·`twinkle_count`·`quality_low_max_cores`). **구 배경 제거 완료:** `CosmosBackdrop`·`FluidGradient`·`BrandMark`·`StarFieldCanvas`·`NebulaBackground`·`pages/landing/ui/background/*`·`widgets/star3d/*` 전부 삭제.

## 목적
지금 비-우주 배경은 화면마다 제각각이다 — 사인인·초대는 `CosmosBackdrop`(WebGPU `FluidGradient` + 2D `StarFieldCanvas` + grain)에 별도 `StarCanvas`(브랜드 별)를 겹치고, 랜딩은 테마별 `LandingBackground`(CSS 그라디언트 + blur orb + 2D `StarFieldCanvas` + grain)를 쓴다. 그래서:

1. **별이 배경과 한 공간에 못 산다.** 캔버스가 분리돼 있으면 "배경 일부는 별 앞으로·일부는 뒤로 흐른다" 같은 깊이 합성이 불가능하다(plan 42 Part B에서 박스 한정으로 시도했더니 별 둘레에 **동그란 패치**로 떠 페이지에 안 녹았다).
2. **자가발광 별이 안 빛난다.** glow(번짐)는 emissive가 아니라 **BloomPass**나 halo가 만든다. 사인인·랜딩이 쓰는 `StarCanvas`엔 bloom이 없어, plan 42에서 별을 자가발광으로 통일한 뒤 불투명 aurora가 흐릿하게 죽었다.
3. **중복·불일치.** 같은 "우주 배경" 인상을 세 군데가 다른 코드로 만든다. 우주 셸은 이미 정답(한 캔버스 + bloom + 별 프리미티브)을 갖고 있는데 공유되지 않는다.

목표는 그 정답을 **디커플드 재사용 컴포넌트 `CosmosScene`** 으로 추출해 모든 비-우주 배경을 그 위로 옮기는 것이다. 페이지는 데이터(별 concept/color·팔레트·레이어·카메라 프리셋)만 주입한다. 이는 장차 우주 셸 채택과 npm 라이브러리화의 토대이기도 하다.

## 범위 / 비목표
**범위**
- 새 공유 widget **`CosmosScene`** — 한 R3F `<Canvas>`(WebGPU/WebGL2, `createRenderer`)에 다음을 깊이순으로 합성: ① fluid 배경 **뒤** 레이어(들) → ② 트윙클 별먼지(3D point) → ③ 별 프리미티브 0~N개(`buildStarBody`, plan 42) + halo → ④ fluid 배경 **앞** 레이어(들, 투명/additive) → ⑤ `BloomPass`. grain은 씬 내(또는 셰이더) 처리. 단순 **카메라 프리셋**(emblem / fullscreen), `manual=true`(리사이즈 안전).
- **`BloomPass`를 `widgets/universe-canvas` → `shared/ui`로 이동**해 universe-canvas와 CosmosScene이 공유한다.
- **모든 비-우주 배경을 CosmosScene으로 이주(전부 WebGL):**
  - **사인인·초대** — 배경 + 브랜드 별 + 트윙클 + grain + bloom을 **한 씬으로 완전 통합**(2D `StarFieldCanvas`를 3D point로, grain을 씬 내로). 화면당 WebGPU 컨텍스트 1개.
  - **랜딩** — 테마(vast/lively/calm)별 **팔레트로 페이지 전체 배경**을 CosmosScene으로 교체(동그란 패치 없음), 히어로 별이 bloom으로 글로우.
- **디커플링** — CosmosScene은 `appearance` 스토어·navigation/focus FSM·전송 계층을 import하지 않고 prop으로만 받는다. 페이지가 어댑터(테마→팔레트, object/accent→StarVisual)를 둔다.
- **성능 예산** — 노브(fluid fps 캡·dpr·fbm octave·트윙클 카운트·레이어 수)를 `spec/values.yaml`로 옮겨 단일 출처화. `prefers-reduced-motion`이면 정적(시간 정지·demand). **품질(quality) prop**으로 화면별 부하 조절(배경 전용 화면은 bloom·레이어 축소).

**비목표**
- **우주 셸 재구축은 안 한다.** 우주는 이미 이 패턴(한 캔버스 + bloom)이라 **레퍼런스로 둔다**. CosmosScene이 *나중에* 우주에 채택될 수 있게 디커플드로 설계만 하되, navigation/focus FSM·8개 카메라 컨트롤러 일반화는 이 plan 밖이다.
- **npm 패키지 실제 추출·배포는 비목표**(디커플링이 그걸 *가능하게*만 한다).
- 별 프리미티브 자체 변경은 비목표(plan 42 그대로 소비). aurora 투명 복원도 비목표 — **bloom이 글로우를 복원**하므로 불투명 유지.
- 동시 다중 컨텍스트 문제 아님 — 배경은 라우트별이라 한 번에 한 CosmosScene만 산다.

## 설계 요점

### CosmosScene 컴포넌트 (공유 widget)
prop으로만 구동되는 디커플드 합성 씬:

```ts
interface StarVisual {            // 장식용 별 한 개 (우주의 StarNode와 별개, 단순화)
  concept: StarObject             // 형태(deepfield/aurora/liquid/ember)
  color: string                   // mood/accent hex
  position: [number, number, number]
  size: number                    // 반지름
  seed?: number
}
interface CosmosSceneProps {
  stars?: StarVisual[]            // 0개면 순수 배경(랜딩 일반 섹션), N개면 별+배경
  palette?: CosmosPalette         // 배경 fluid 색(테마별). 기본 = 현재 fluid 팔레트
  layers?: { back?: number; front?: number }  // 별 앞/뒤 fluid 레이어 수
  twinkle?: number                // 3D 트윙클 point 수(0=없음)
  bloom?: boolean                 // 기본 true(VALUES.bloom)
  grain?: boolean
  camera?: 'emblem' | 'fullscreen'
  quality?: 'high' | 'low'        // 저사양/배경전용 다운그레이드
  className?: string
}
```

- **깊이 합성:** 한 씬 안에서 `renderOrder`로 [배경 뒤 → 트윙클 → 별+halo → 배경 앞]을 쌓는다(배경 머티리얼 `depthTest=false`, 앞 레이어 `depthWrite=false`·additive). 배경이 **화면/섹션 전체**를 채우므로 plan 42 Part B의 "동그란 박스 패치" 문제가 사라진다(전면 fluid는 경계가 없다).
- **Bloom으로 글로우:** `BloomPass`(RenderPipeline)가 최종 출력을 소유 → 자가발광(HDR, `toneMapped=false`) 별이 실제로 번진다(불투명 aurora 포함). `StarCanvas`엔 없던 그 글로우를 여기서 제공.
- **카메라 프리셋:** `emblem`=별 중심 fit ortho(사인인·초대·랜딩 히어로), `fullscreen`=배경 채움. 둘 다 `manual=true`(R3F 리사이즈 frustum 덮어쓰기 방지, plan 42에서 확인된 이슈).
- **grain:** 씬 내 풀스크린 오버레이 패스 또는 셰이더로(현재 CSS `GrainOverlay` 대체 검토).

### FSD 배치 & BloomPass 이동
- `CosmosScene` = **widget**(`widgets/cosmos-scene`). `buildStarBody`(entities/star) + `BloomPass`(shared/ui) + fluid 머티리얼(shared/ui) + `createRenderer`(shared/lib/r3f)를 조립. widget이라 entities를 import할 수 있다(§4 합법).
- **`BloomPass` → `shared/ui`** 로 이동(현재 `widgets/universe-canvas/ui/BloomPass.tsx`). 자족적(three/webgpu + `VALUES.bloom`)이라 FluidGradient처럼 shared/ui에 두면 universe-canvas와 cosmos-scene 두 widget이 공유. universe-canvas는 import 경로만 바뀐다(무회귀).
- **디커플링:** CosmosScene·BloomPass는 앱 store/FSM을 모른다. 페이지가 어댑터로 주입(사인인: `appearance.object`+`themeAccent`→StarVisual; 랜딩: `theme`→palette).

### 페이지 이주
- **사인인·초대:** `CosmosBackdrop` + `BrandMark`(StarCanvas) → `<CosmosScene camera="emblem" stars={[브랜드별]} twinkle=… grain bloom />` 하나로. 컨텍스트 2→1.
- **랜딩:** `LandingBackground`(vast/lively/calm) → `<CosmosScene camera="fullscreen" palette={themePalette} … />` 페이지 배경 + 히어로 섹션은 별 포함. 비-히어로 섹션은 별 0개 배경(같은 엔진, 가벼운 quality).
- **정리:** 이주 완료 후 `CosmosBackdrop`·`FluidGradient`(CosmosScene 내부로 흡수)·landing `background/*`·미사용 `NebulaBackground` 제거 검토. 2D `StarFieldCanvas`는 다른 소비처가 없으면 제거(트윙클이 3D point로 흡수).

### 성능 (사용자 우려 반영)
랜딩 배경은 현재 **CSS/2D(아주 가벼움)** 인데 WebGL fluid+bloom으로 바꾸면 화면당 GPU 부하가 는다. 그래서:
- **노브를 values.yaml로:** fluid fps 캡(현 30)·dpr(현 0.6)·fbm octave(현 3)·트윙클 카운트·레이어 수. 코드 하드코딩 제거(plan 42 perf 커밋이 박아둔 값 포함).
- **품질 단계:** `quality='low'`(또는 기기 등급 감지 — `deviceMemory`/`hardwareConcurrency`/WebGL2 폴백)에서 bloom·앞레이어·트윙클을 축소하거나 정적. 배경 전용 화면은 기본 저부하.
- **reduced-motion:** 시간 정지 + `frameloop='demand'`(현 FluidGradient 거동 유지).
- 라우트당 컨텍스트 1개라 화면 간 누적은 없다.

### 단계(job) 분할 제안
규모가 크니 독립 검증 가능한 단위로: **J1** CosmosScene + BloomPass 이동 + 사인인 이주. **J2** 초대 이주. **J3** 랜딩 전체 배경 이주(+ 구 배경 정리). 각 job이 끝날 때 해당 화면 무회귀를 확인.

## 수용 기준 (인수 조건)
1. `CosmosScene` 한 widget이 fluid 배경(앞/뒤 레이어) + 트윙클 point + 별 프리미티브(`buildStarBody`)+halo + grain + `BloomPass`를 **한 R3F 캔버스**에 깊이순(renderOrder)으로 렌더한다.
2. `CosmosScene`·`BloomPass`는 `appearance`·navigation/focus FSM·전송 계층을 import하지 않는다(prop 주입만) — 디커플드·라이브러리화 가능.
3. `BloomPass`가 `shared/ui`에 있고 universe-canvas와 CosmosScene이 **공유**한다(중복 구현 없음). 우주 셸은 시각·동작 무회귀.
4. 사인인·초대가 `CosmosScene` **하나**로 배경+브랜드별+트윙클+grain+bloom을 렌더 — 화면당 동시 WebGPU 컨텍스트 **1개**(이전 2개). 자가발광 별이 bloom으로 **글로우**(불투명 aurora도 빛난다).
5. 랜딩 배경이 테마(vast/lively/calm)별 팔레트로 `CosmosScene`(fullscreen)으로 교체되고, 배경이 **페이지 전체**에 깔려 별 둘레 동그란 패치가 없다. 히어로 별이 글로우.
6. 한 씬 안에서 배경 레이어가 **별 앞·뒤 양쪽**에 존재하고 깊이로 정렬된다("일부는 별 앞으로·일부는 뒤로 흐른다").
7. `prefers-reduced-motion`이면 정적(시간 정지·demand). 성능 노브(fps·dpr·octave·트윙클/레이어 카운트)는 **`spec/values.yaml`에서 생성**된 상수로 온다(FE 하드코딩 없음). `quality='low'`(또는 기기 등급)에서 부하가 실제로 준다.
8. 셰이더 TSL·별먼지 InstancedMesh/Points 유지(헌법8). build·lint·steiger 통과. 이주한 모든 화면(사인인·초대·랜딩)과 우주 셸에 회귀가 없다.
