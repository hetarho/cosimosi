# 39. 상태 머신 리팩토링 (XState 도입)

> 흩어진 제어 상태(카메라 모드·포커스·작성·회상·세션)를 **XState v5 상태 머신**으로 단일화한다. | 범위: FE | 선행: 28, 31, 38 | 차단: 없음

## 목적 (Why)

cosimosi는 일기 앱이 아니라 **우주를 항해하는 게임**이다 — 카메라가 성운/회상/비행/조망 사이를 오가고, "지금 무엇에 집중하나"(별/일기/리스트/변천사)가 서로 배타다. 이 "모드·단계·수명주기"가 지금은 **여러 스토어의 nullable 필드 + 동기화 이펙트**로 흩어져 있어, 같은 논리 상태를 두 진입점이 다른 변수로 표현해 **불일치**가 난다(실증: "일기로 별 보기"를 `frameDiary`와 `onSeeDiaryStars`가 다르게 다뤄 한쪽만 카드 표시·`NearFarHighlightGuard`가 다른 쪽을 지움). 제어 상태를 명시적 FSM으로 모델링해 **불가능한 상태를 표현 불가능하게** 만들고, 동기화 이펙트를 전환 함수로 흡수한다.

설계 전반은 [state-machines.md](../state-machines.md), 작성 규약은 [xstate-guide.md](../xstate-guide.md)가 단일 출처다. 이 스펙은 **그 설계를 코드로 옮기는 단계적 작업 순서**다.

## 범위

**포함** — 아래 7개 영역을 XState로 승격(2 Core + 3 Lifecycle + evolution 흡수 + 선택적 time-skip):

- 세션(P0·레퍼런스), 포커스(P1), 항행(P2), 작성(P3), 회상 flush(P4), (선택) leaf 흡수/평가(P5).
- XState v5 + @xstate/react 도입, `createActorContext` 앱 패턴, R3F per-frame `getSnapshot` 패턴, 머신 간 통신 배선.

**비목표 (Out of scope)**

- **데이터를 머신으로 옮기지 않는다** — `stars[]`·`edges[]`·`ambient`·`emotionColors`·쿼리 캐시·force-sim 좌표 버퍼는 zustand/Query/ref 유지. 머신은 `id` 참조만.
- **동작·시각 변경 금지(behavior-preserving)** — 카메라 감각·UX는 그대로. 단 P1은 의도적으로 "일기로 별 보기" 두 진입점을 **동일**하게 만든다(이게 목표).
- **백엔드·proto·DB 무변경.** 순수 FE 리팩토링.
- **leaf UI 일괄 전환 금지** — MemoryPanel phase·SignIn step·BottomSheet snap·DemoSimPanel·랜딩 카드 데모는 가치가 증명될 때만 승격(기본 유지).
- 새 카메라/포커스 기능 추가 금지(리팩토링이지 기능 확장이 아니다).

## 설계 요점

- **승격 기준:** "N개 배타 상황의 명시적 수명주기"만 머신. 값/컬렉션은 데이터로 둔다([state-machines §0](../state-machines.md)).
- **배치:** 머신은 소유 레이어의 `model/<name>.machine.ts`(순수 TS). React 바인딩·R3F 연동은 `ui`. entities는 머신 스냅샷 파생 props로 렌더만(헌법4).
- **두 Core의 관계:** `navigation` 액터 먼저 생성(ref 불필요, 도착을 `emit`) → `focus` 액터가 nav ref를 `input`으로 받아 `sendTo` → 페이지가 `navActor.on('arrived.star', …) → focusActor.send(…)` 한 줄 브리지(순환 ref 회피, 둘 다 top-level standalone이라 per-frame getSnapshot 빠름).
- **R3F:** 컨트롤러는 `useActorRef`로 ref만 얻어 `useFrame`에서 `getSnapshot()`; 임계 도달 시 `send(ARRIVED)`. 연속 lerp·`move`·drag는 머신 밖. `useMachine` 금지.
- **단계 독립성:** 각 Phase는 빌드/리뷰/검증을 독립으로 통과하고 따로 커밋 가능. P0가 패턴·도구를 깔고, P1→P2 순서(포커스가 나브 계약을 정의)로 의존. P3·P4는 Core와 독립이라 순서 자유.

## 수용 기준 (EARS/GWT)

1. **도입**
   - 1.1 WHEN `pnpm --filter ./frontend build`·`lint`·`steiger`를 돌리면 THE SYSTEM SHALL xstate/@xstate/react가 설치된 상태로 0 에러로 통과한다.
   - 1.2 THE SYSTEM SHALL 모든 `*.machine.ts`를 `model` 세그먼트에 두고, `model`에서 React/three import가 없다(헌법4·steiger 통과).
2. **세션(P0)**
   - 2.1 WHEN 앱이 부팅하면 THE SYSTEM SHALL `loading`→(세션 있음)`authed`/(없음·실패)`anon`으로 전이한다(현 auth-store와 동일 동작).
   - 2.2 WHEN 사용자 경계가 바뀌면(다른 uid) THE SYSTEM SHALL 우주 데이터 리셋을 1회 트리거한다(기존 `syncIdentity` 동작 보존).
3. **포커스(P1)**
   - 3.1 WHEN 일기 목록에서 일기를 고르거나(`SELECT_DIARY`) 회상 패널의 "이 일기의 다른 별들 보기"를 누르면(`SEE_DIARY_STARS`) THE SYSTEM SHALL **동일하게** `diaryFocus` 상태로 들어가 일기 카드 표시 + 별 조망을 보인다(두 진입점 동작 일치 — 본 스펙의 핵심 수용).
   - 3.2 THE SYSTEM SHALL 별 포커스와 일기 조망이 동시에 활성일 수 없다(머신 단일 상태 — `NearFarHighlightGuard` 삭제 후에도 배타 유지).
   - 3.3 WHEN 빈 캔버스를 탭하거나 Esc를 누르면 THE SYSTEM SHALL `idle`로 복귀한다(별·일기·리스트·변천사 어느 상태에서든).
   - 3.4 WHEN `?panel=dormant|diary`로 딥링크하면 THE SYSTEM SHALL 해당 리스트 상태로 시작하고, 뒤로가기로 복귀한다.
4. **항행(P2)**
   - 4.1 WHEN `FLY_TO_STAR`/`FRAME_DIARY`/`TOGGLE_MODE` 이벤트를 받으면 THE SYSTEM SHALL 각각 `flyingToStar`/`framingDiary`/`modeTransition`(tag `transitioning`)으로 전이하고, 도착 시 `recall`/`nebula`/`to`로 정착한다.
   - 4.2 WHILE `transitioning` tag가 켜져 있으면 THE SYSTEM SHALL 카메라 거리·함선 경계 클램프를 완화한다(기존 동작 보존).
   - 4.3 THE SYSTEM SHALL 매 프레임 카메라 보간을 `useFrame` 안에서 수행하고 머신은 `getSnapshot`으로만 읽는다(React 리렌더로 60fps 구동 금지 — Architecture §3.2).
5. **작성(P3)·회상(P4)**
   - 5.1 WHEN 작성이 분절/제출되면 THE SYSTEM SHALL `compose`↔`review` × `idle/segmenting/submitting/error`를 한 머신으로 표현하고 불가능 조합(예: review 아닌데 submitting)을 만들 수 없다.
   - 5.2 WHEN 공동 회상이 누적되면 THE SYSTEM SHALL `after` 디바운스(유휴 5s)로 flush하고, 실패 시 같은 `batch_id`로 재시도(멱등)한다(기존 동작 보존).
6. **회귀**
   - 6.1 THE SYSTEM SHALL 기존 vitest를 모두 통과시킨다(91+개), 머신별 신규 단위테스트를 추가한다.

## 리뷰 합산 (codex 교차 — 전체 P0–P4 diff, `-m gpt-5.5`)

per-phase `/code-review`(엔진 내) 외에, 완료 시점에 전체 diff를 **codex**(다른 엔진)로 1회 교차 리뷰했다. 4건 중 3건 반영, 1건 문서화:

- **반영(HIGH)** — 비행 상태(flyingToStar/framingDiary)가 이벤트로 즉시 진입하나 컨트롤러가 타깃을 못 풀면(우주 미로드·id 부재) `ARRIVED` 미발송 → `transitioning` 영구 고착(카메라 동결). `after` 안전 타임아웃(10s)으로 settled 강제 복귀.
- **반영(MED)** — `modeTransition` 중 `FLY_TO_STAR`/`FRAME_DIARY` 유실(특히 "이 일기의 다른 별들" 요청). modeTransition에 두 핸들러 추가 + ModeTransitionController useFrame에 nav-state 가드(양보).
- **반영(MED-HIGH)** — compose 'submitted' 무효화 구독이 HomePage에 있어, 제출 응답 전 라우트 이탈 시 캐시 무효화 유실. App 루트로 이전(라우트 이동과 무관·앱 queryClient 단일 출처).
- **문서화(LOW, dev 전용)** — StrictMode 이중 마운트 시 세션 식별/`sign_in` 분석 중복(`lastUserId`가 액터 context라 재마운트에 리셋). 프로덕션은 단일 마운트라 정상이고 dev PostHog는 no-op이라 무해 — 머신 순수성 유지를 위해 미수정.
