# 06. 우주 캔버스 - as-built

> 현재 `/universe` 첫 화면은 R3F 캔버스 하나가 아니라, `HomePage`가 데이터 쿼리와 DOM HUD를 소유하고 `UniverseCanvas`가 3D 장면과 카메라 컨트롤러를 소유하는 우주 셸이다. Flutter 구현도 이 경계를 그대로 맞춘다.

## 현재 범위
- `frontend/src/pages/home/ui/HomePage.tsx`
  - `useQuery(universeQueryOptions())`로 `GetUniverse`를 읽고 `applyUniverse()`로 memory/synapse store에 병합한다.
  - `useQuery(settingsQueryOptions())`로 theme, star object, emotion color 설정을 읽고 `applySettings()`로 appearance store를 갱신한다.
  - 캔버스 바깥 DOM HUD를 소유한다(IA: home-ia revamp — 진입·결과·뷰 컨트롤 분리, 상세는 [tech/overlay-shell](../tech/overlay-shell.md)):
    - **상시 버튼 둘** — 카메라 시점 토글 + "메뉴" 런처(우상단). 나머지 기능은 메뉴 뒤에.
    - **메뉴** — 런처가 여는 `Surface`: 새 일기·일기·잠든 별·우주 공개·주고받은 별·테마·외형.
    - **결과 표면** 통일 비차단 `Surface`(모바일 바텀시트 / 데스크톱 떠있는 카드) — `MemoryForm`(작성)·`MemoryPanel`(회상)·`EvolutionPanel`(변천사)·`ShareUniverseBody`(공유)·`StarGiftsBody`(선물 목록)·`SendStarBody`(보내기). 모두 body-only 콘텐츠를 호스트가 감싼다.
    - **브라우즈 목록** `OverlayHost`(peek) — `DiarySheet`·`DormantSheet`·`DiaryCard`.
    - **뷰 컨트롤**(진입과 분리) — 카메라 성운/회상 토글·`AppearanceSwitcher`(우상단)·`NavPad`(회상 모드, 화면 가장자리).
    - 데모 패널, error/loading/empty overlay.
  - 캔버스 바운더리는 `Sentry.ErrorBoundary`로 캔버스만 감싼다. 렌더러 실패 시에도 작성 폼 같은 HUD는 살아 있다.
- `frontend/src/widgets/universe-canvas/ui/UniverseCanvas.tsx`
  - R3F `<Canvas>` 내부 3D 장면과 모든 카메라/레이아웃 컨트롤러를 소유한다.
  - 씬 안에 DOM을 넣지 않는다. DOM grain/HUD/패널은 모두 canvas sibling이다.
- `frontend/src/shared/lib/r3f`
  - `types.ts`는 `StarNode`, `SynapseEdge`, `CameraMode`, `GraphRenderer` 타입만 담는 순수 포트다.
  - `renderer.ts`는 Web 전용 `createRenderer()`, `rendererBackend()`, `RendererUnavailableError`를 제공한다.

## 렌더러 계약
- `createRenderer(props)`는 `three/webgpu`의 `WebGPURenderer`를 생성하고 `await renderer.init()`을 수행한다.
- `navigator.gpu`가 없으면 생성 시점에 `forceWebGL: true`를 넣어 WebGL2 백엔드를 바로 사용한다. 이렇게 해야 fallback warning 없이 렌더된다.
- WebGPU와 WebGL2가 모두 실패하면 `RendererUnavailableError`를 throw한다. `UniverseCanvas`는 async `gl` factory reject를 state로 받아 렌더 중 throw로 바꾸고, `HomePage`의 캔버스 바운더리가 전용 안내를 보여준다.
- `rendererBackend(gl)`은 `webgpu` 또는 `webgl2`를 반환한다. `UniverseCanvas.onCreated`에서 analytics/reporting과 dev console log에 사용한다.
- 언마운트 시 `BloomPass`가 먼저 pipeline/node target을 dispose하고, `UniverseCanvas` 부모 cleanup이 `ResizeObserver.disconnect()`와 `gl.dispose()`를 수행한다.

## Bloom과 장면 레이어
- Bloom은 `frontend/src/widgets/universe-canvas/ui/BloomPass.tsx`의 `RenderPipeline` 기반이다.
- 구성은 `pass(scene, camera)`와 `bloom(scenePass, 0.9, 0.5, 0.1)`를 합친 `pipeline.outputNode = scenePass.add(bloomNode)`다.
- `useFrame(..., 1)` 우선순위로 R3F 기본 렌더를 대체하고 `pipeline.render()`가 최종 출력을 소유한다.
- 현재 장면 순서는 다음과 같다.
  - `<color attach="background" args={[themeBg(theme)]} />`
  - `<ambientLight intensity={0.4} />`
  - `<AmbientNebula />`: 최근 감정 요약과 별 분포로 만든 additive light pools.
  - `<StarDust count={1500} />`: 그래프와 무관한 배경 별먼지.
  - `<group visible={ready}> <UniverseDrift> ... </UniverseDrift> </group>`
  - `<SelfStar />`: 중심 "나" 별.
  - `<UniverseSynapses />`: live layout snapshot과 positions buffer를 함께 읽는 시냅스 레이어.
  - `<StarField />`: memory store의 별을 그리는 인스턴스 별 레이어.
  - layout/camera/focus/view-offset controllers.
  - `<BloomPass />`

## 데이터와 로딩
- `HomePage`는 `universeQueryOptions()` 성공 시 `applyUniverse(res)`를 호출한다.
- `applyUniverse()`는 별과 시냅스를 wholesale replace하지 않고 병합한다. 제출 중 temp 별, 기존 instance slot, 로컬이 앞선 timestamp를 보존한다.
- 첫 레이아웃이 안정화되기 전 `UniverseCanvas`는 "별들이 제자리를 찾고 있어요..." veil을 보여주고 별/시냅스 group을 `visible={ready}`로 가린다.
- 별 0개인 우주가 실제로 로드되면 즉시 ready 처리한다. 초기 pending 또는 출처 reset으로 별 배열이 비면 veil을 다시 arm한다.
- 8초 safety timer가 있어 레이아웃 settle 이벤트가 오지 않아도 사용자가 영구 veil에 갇히지 않는다.
- `HomePage`는 `universe.isPending`, 첫 로드 `universe.isError`, `starCount === 0` 빈 상태를 별도 DOM overlay로 표시한다.

## 카메라와 항행 상태
- 카메라 상태의 단일 출처는 `frontend/src/widgets/universe-canvas/model/navigation.machine.ts`의 XState actor다.
- 상태:
  - `nebula`: 멀리서 전체 조망.
  - `recall`: 근접 항해.
  - `flyingToStar`: 특정 별로 이동 후 `recall`에 도착.
  - `framingDiary`: 원본 일기의 모든 별을 조망 후 `nebula`에 도착.
  - `modeTransition`: nebula/recall 토글 이동.
  - `overlay.viewing` / `overlay.framingPair`: 데모 겹쳐보기 전용.
- 주요 이벤트:
  - `TOGGLE_MODE`, `FLY_TO_STAR`, `FRAME_DIARY`, `ARRIVED`, `SET_MOVE`, `ENTER_OVERLAY`, `EXIT_OVERLAY`, `FRAME_PAIR`.
- `#transitioning` tag가 켜진 동안 clamp를 완화한다. `flyingToStar`, `framingDiary`, `overlay.framingPair`에는 10초 safety timeout이 있어 target 미해결 시 settled 상태로 빠진다.
- `selectHeadingMode()`는 HUD/NavPad 라벨용으로 현재 또는 향하는 모드를 반환한다.

## 컨트롤러별 동작
- `CameraRig`
  - mode에 따라 `OrbitControls` 거리 clamp를 바꾼다.
  - nebula는 `OBSERVE_MIN_DIST = 58` 이상에서 조망한다.
  - recall은 근접 항해용 거리 범위를 사용한다.
- `NebulaOrbitController`
  - pointer/touch/wheel을 직접 처리하는 custom arcball이다.
  - 별 focus 또는 transition 중에는 개입하지 않는다.
- `NavController`
  - recall mode에서 D-pad/keyboard `move`를 읽어 thrust/yaw/pitch를 적용한다.
  - `SHIP_BOUNDARY = 46 * 0.85` 안에 카메라를 clamp하고 벽 충돌 recoil/shake를 준다.
- `FlyToController`
  - live force-sim buffer를 매 프레임 다시 읽어 이동 중인 별을 추적한다.
  - 도착 시 `focusActor.SELECT_STAR`를 보내고 navigation에 `ARRIVED`를 보낸다.
- `FrameAllController`
  - `recordId`의 별들을 live buffer slot으로 풀어 `frameTarget()`으로 전체 조망 위치를 계산한다.
  - portrait/landscape 모두 맞도록 vertical/horizontal fov 중 더 좁은 값을 사용한다.
- `FocusController`
  - 별 focus 중 camera target을 별에 lock한다.
  - nebula에서는 현재 거리감을 유지한 채 별 radial side로 orbit하고, recall에서는 position은 유지하고 aim만 돌린다.
- `ModeTransitionController`
  - nebula->recall은 중심으로 dive, recall->nebula는 저장한 overview pose로 복귀한다.
- `ViewOffsetController`
  - 모바일 compose/recall sheet 또는 일기 조망 카드가 하단을 가릴 때 perspective `setViewOffset()`과 `zoom`으로 우주 중심을 화면 위쪽으로 올린다.
  - 입력 플래그는 `model/use-viewport.ts`의 작은 zustand store(`sheetOpen`)가 소유한다.

## 포커스와 셸 오버레이
- "지금 무엇에 집중하나"의 단일 출처는 `frontend/src/entities/memory/model/focus.machine.ts`다.
- 상태는 `idle`, `star`, `diary`, `pair`이며 서로 배타다.
- 별 focus는 `MemoryPanel`, diary focus는 `DiaryCard`와 frame-all highlight, pair focus는 겹쳐보기 비교 패널을 구동한다.
- 빈 우주 탭, Esc, 출처 reset은 실제 이벤트명 `DISMISS`를 사용한다.
- `/universe?panel=dormant|diary`는 라우트 이동이 아니라 영속 캔버스 위 셸 패널 상태다. 캔버스는 패널 열고 닫기로 언마운트되지 않는다.
- `/universe?fly=<memoryId>`는 선물 수락 후 해당 별이 `GetUniverse`에 실릴 때 한 번 `FLY_TO_STAR`를 보낸 뒤 search param을 제거한다.

## 겹쳐보기
- 데모 겹쳐보기는 `UniverseOverlay`가 별도 scene을 렌더한다.
- `HomePage`가 데모 persona 데이터를 `mapStar`/`toSynapseEdge`로 domain props로 바꿔 넘긴다.
- 겹쳐보기 진입 시 navigation은 `ENTER_OVERLAY`를 받고 write gate를 켠다. 이 상태에서는 쓰기 RPC 경로가 막혀야 한다.
- overlay를 끄면 `EXIT_OVERLAY`로 들어오기 전 mode로 돌아가고 `focusActor.send({ type: 'DISMISS' })`로 focus를 비운다.

## Flutter 동기화 포인트
- Flutter도 "canvas/scene"과 "HUD/query/error shell"을 분리한다. 3D scene 내부에 플랫폼 UI를 넣는 방식으로 옮기지 않는다.
- star/synapse 좌표는 `positionsRef`에 해당하는 단일 live buffer를 모든 독자가 공유한다. 별, 시냅스, fly-to, focus가 서로 다른 좌표 계산을 하면 안 된다.
- navigation/focus는 boolean 조합이 아니라 명시적 FSM으로 맞춘다. 특히 star focus와 diary focus는 동시에 켜질 수 없다.
- renderer fallback UX도 맞춘다. 렌더러 불가와 일반 캔버스 크래시는 다른 안내다.
- 모바일 bottom sheet가 열릴 때 view offset/zoom으로 우주를 위로 올리는 동작을 구현한다.

## 현재 DoD
- `/universe`는 `fixed inset-0` + `data-lenis-prevent` 컨테이너에서 렌더된다.
- `RendererUnavailableError`면 캔버스 안내만 뜨고 HUD는 유지된다.
- 첫 레이아웃 settle 전에는 별/시냅스 snapping을 숨긴다.
- nebula/recall 전환, star fly-to, diary frame-all, focus dismiss, mobile view offset이 모두 navigation/focus actor를 통해 수렴한다.
- `GraphRenderer` 포트와 r3f renderer 타입 파일은 three/React/DOM을 import하지 않는다.
