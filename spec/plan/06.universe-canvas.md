# 06. 우주 캔버스 — WebGPU 렌더러·Bloom·카메라 셸

> R3F `<Canvas>` + three `WebGPURenderer`(WebGL2 자동 폴백) + 노드 기반 Bloom + 씬 셸 + 카메라 모드(nebula/recall)를 더미 데이터로 검증한다. | 범위: MVP | 선행: — | 차단: 08, 09

## 목적 (Why)
엔그램 우주를 그릴 **렌더링 토대**를 깐다. 별·시냅스(08/09)·force-sim(07)·데이터 연동(10)을 얹기 전에, WebGPU 렌더러 비동기 생성과 WebGL2 폴백, Bloom 후처리, 카메라 모드 전환이 더미 점만으로도 안정적으로 동작함을 먼저 확정한다. 또한 모바일 이식을 위한 **렌더러 추상화 경계(`GraphRenderer` 포트)** 와 순수/플랫폼 레이어 분리를 처음부터 박는다(원칙 4).

## 설계 요점
**렌더러 격리(원칙 4):** 순수 레이어(`shared/lib/r3f/types.ts`)는 `three`/React를 import하지 않는다. 플랫폼 구현(`renderer.ts`, `UniverseCanvas.tsx`)에만 `three`/R3F를 가둔다.

```ts
// shared/lib/r3f/types.ts  — 순수, three/React 미의존 (모바일 재사용)
export interface StarNode { id: string; intensity: number; mood: string }
export interface SynapseEdge { aId: string; bId: string; weight: number }
export type CameraMode = 'nebula' | 'recall'
// 렌더러 추상화 경계: 웹은 R3F/WebGPU, 모바일은 WebGPU/Filament로 교체
export interface GraphRenderer {
  setGraph(nodes: StarNode[], edges: SynapseEdge[]): void
  setCameraMode(mode: CameraMode): void
  dispose(): void
}
```

```ts
// shared/lib/r3f/renderer.ts — three 의존 (플랫폼 레이어)
import * as THREE from 'three/webgpu'
// R3F <Canvas gl={createRenderer}> 에 넘기는 비동기 팩토리.
// WebGPURenderer는 WebGPU 미지원 환경에서 내부적으로 WebGL2 백엔드로 자동 폴백.
export async function createRenderer(props: { canvas: HTMLCanvasElement }) {
  const renderer = new THREE.WebGPURenderer({ canvas: props.canvas, antialias: true })
  await renderer.init()                 // ← 비동기 init 필수 (§3.1)
  return renderer
}
export function rendererBackend(r: THREE.WebGPURenderer): 'webgpu' | 'webgl2' {
  return (r.backend?.isWebGPUBackend) ? 'webgpu' : 'webgl2'  // 콘솔 로그용
}
```

```tsx
// widgets/universe-canvas/ui/UniverseCanvas.tsx (요지)
```

**Bloom(노드 후처리) — three `0.184` 핀 기준:**
- import 경로(비대칭 주의): 후처리 클래스는 코어(`three/webgpu`), `bloom` 헬퍼는 addons:
  ```ts
  import { PostProcessing } from 'three/webgpu'              // 클래스: three/webgpu
  import { pass } from 'three/tsl'
  import { bloom } from 'three/addons/tsl/display/BloomNode.js' // bloom 노드: addons(jsm)
  ```
- 후처리 클래스명 주석: three `0.184`는 **`PostProcessing`**. three dev(r185+)에서 **`RenderPipeline`로 리네임 진행 중**이므로 three 버전 업그레이드 시 import/클래스명 재확인 필요(`PostProcessing`↔`RenderPipeline`). 채택 버전은 `package.json` `three ^0.184.0`에 고정.
- 구성(단일 확정 방식): `const scenePass = pass(scene, camera)` → `postProcessing.outputNode = scenePass.add(bloom(scenePass))`. `<Canvas frameloop="never">` 또는 `useFrame`에서 **R3F 기본 `gl.render` 대신 `postProcessing.render()`(WebGPU 경로는 내부적으로 `renderAsync` 펌프) 1회 호출**(`renderer.render(scene,camera)` 아님). useFrame `priority>0`로 기본 렌더 패스를 가로채 후처리가 최종 출력을 소유한다.
- 공식 예제(`webgpu_postprocessing_bloom`)는 `renderer.setAnimationLoop(animate)` + `postProcessing.render()` 기반임 — R3F에선 그 animate 루프를 `useFrame`(priority>0)로 대치한다(주석으로 명시).
- WebGPU/WebGL2 양쪽에서 동작해야 한다(TSL→GLSL 컴파일). emissive 강한 더미 점에 발광이 보여야 한다. 통합 리스크가 있으면 bloom은 선택 후처리로 격리하고 무후처리 렌더(1.1)부터 green 확인.

**성능 전제(§3.2):** 더미 점도 `Points`/`InstancedMesh`로. React state로 60fps 구동 금지 — 카메라/포스트만 `useFrame`에서 갱신.

**전체화면 사이징 + Lenis 상호작용:** `/universe` HomePage는 RootLayout의 Lenis `SmoothScroll`(ReactLenis root) 안에서 마운트되므로 콘텐츠 주도 높이로 `h-full`이 0으로 접히지 않도록 캔버스 래퍼를 `fixed inset-0`(또는 고정 `100dvh`)로 두고, OrbitControls 휠이 Lenis `smoothWheel`에 가로채이지 않게 캔버스 영역에서 Lenis를 비활성(`data-lenis-prevent`/이벤트 분리)한다.

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 사용자가 우주(`/universe`)에 진입 THE SYSTEM SHALL `WebGPURenderer`를 비동기 생성·`init()` 후 전체 화면 `<Canvas>`를 렌더한다. (랜딩 `/`는 LandingPage 유지.)
1.2 IF 브라우저가 WebGPU를 미지원 THEN THE SYSTEM SHALL 예외 없이 **WebGL2 백엔드로 자동 폴백**하여 동일 씬을 렌더하고, 콘솔에 사용 중인 백엔드(`webgpu`|`webgl2`)를 1회 로그한다.
1.3 WHEN 씬이 렌더됨 THE SYSTEM SHALL 짙은 남색(`#070b1e`) 배경과 옅은 별 먼지(point cloud)를 표시한다.
1.4 WHEN 더미 점이 그려짐 THE SYSTEM SHALL 노드 기반 Bloom 후처리를 적용해 발광(glow)이 보이게 한다.
1.5 GIVEN 카메라 모드가 `nebula` WHEN 사용자가 줌 THE SYSTEM SHALL `min/maxDistance`로 줌 범위를 제한(전체 조망)한다.
1.6 GIVEN 카메라 모드가 `recall` WHEN 사용자가 줌·이동 THE SYSTEM SHALL 줌 제한을 해제하여 자유 근접 항해를 허용한다.
1.7 WHEN 캔버스가 마운트·언마운트 THE SYSTEM SHALL 콘솔 에러·경고 없이 동작하고 언마운트 시 렌더러를 `dispose`한다.
1.8 THE SYSTEM SHALL 씬 내부에 DOM `Html`을 넣지 않는다(원칙 4 — 모바일 이식성).
1.9 THE SYSTEM SHALL `shared/lib/r3f/types.ts`에서 `three`/React/DOM을 import하지 않는다(원칙 4 — 순수 레이어).
1.10 GIVEN 별이 0개(빈 우주) WHEN 캔버스가 마운트 THE SYSTEM SHALL 크래시·콘솔 에러 없이 배경·별 먼지만 렌더한다(더미 점 0개여도 무크래시).
