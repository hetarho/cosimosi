# 55. public universe

> Public universe — a **deferred placeholder** presentation unit (Epic J): browsing another user's _shared_
> universe as a read-only 3D render, reusing the plan 14/23 render substrate. v1 reserves the seam; it does not build it.
> Scope: FE | Status: planning (deferred — v1 out of scope, PRD §8.3)
>
> Placement: FSD `pages/public-universe` + a thin **read-only variant of `widgets/universe-canvas`** (plan 23), over a
> future `internal/sharing` read (CC8). This plan fills **no** v1 slice; it only fixes what the canvas / read-model / route
> substrate must **not** preclude so the browse feature drops in later without a redesign. Pairs with
> [54.star-sharing](54.star-sharing.md) (the share act that produces the browsable subset).

## Purpose

The PRD names two social capabilities, both explicitly deferred ([S1] send a star, **[S2] browse a public universe**,
PRD §5.8, §8.3). Social is v1 **placeholder** discipline (PRD §8.2): defer the _build_, but reserve the _slot_ — so the v1
structure knows "a public universe browse lands here", later fills it without a redesign, and no intervening decision
silently collides with the empty seat.

This unit is that reserved seat for **[S2]**. Where [54.star-sharing](54.star-sharing.md) reserves the _act_ of sharing an
episodic unit one-way to a friend, this unit reserves the _destination_: rendering the shared subset as a **read-only**
universe the recipient can navigate. The whole value of the pairing is that browsing a public universe is **not a new
engine** — it is the plan 23 canvas + plan 14 renderer substrate, pointed at _another user's shared, read-only_ graph
instead of the viewer's own. The only real work v1 must do now is make sure the render substrate, the read model, and the
route registry admit a **second, foreign, read-only** graph source without any of them assuming "the universe is always
mine and always writable."

## Scope / Non-goals

**In scope (reserve-the-slot only)**

- **Name the deferred capability and its shape.** Browsing a public universe is **async, one-way, read-only** (PRD §5.8
  note; CC12): the owner shares a subset (via [54.star-sharing](54.star-sharing.md)); a viewer later opens it as a
  navigable 3D render they cannot mutate. No live session, no realtime, no back-channel from viewer to owner.
- **The reused substrate, stated as a contract.** A public universe renders through the **same** `@cosimosi/3d-renderer`
  boundary (plan 14) and the **same** `widgets/universe-canvas` scene composition (plan 23) — mounted in a **read-only
  mode**: the camera/navigation rig (zoom · rotate · pan · focus · fly) works; every write affordance (recall, launch,
  detail-panel action buttons, letting-go, palette) is **absent or inert**. Positions stay emergent ([I5]) — the viewer's
  client runs the same `packages/force-sim` layout (plan 19) over the shared graph.
- **The seam the read model must hold.** The FE read model (domain-mirror `entities/{episodic-memory,neuron,synapse}`,
  plan 23) must be able to load a graph whose **owner is not the viewer** and whose provenance is "shared subset, not my
  aggregate." **Decided (2026-07):** a public universe is a _distinct read_ (a future `sharing.v1` RPC returning the same
  DTO **shapes** as `GetUniverse`), not a parameter on `GetUniverse` — `GetUniverse` stays the viewer's own,
  `NO_SIDE_EFFECTS`, self-scoped read (plan 23), and never grows an `owner_id`/`is_public` parameter that would let a
  caller read arbitrary users. The seam is "the canvas accepts a graph from an injected source," already true in plan 23
  (the widget consumes DTOs; it does not assume a single fetch).
- **The route slot.** Reserve a `pages/public-universe` route (web) / screen (mobile) addressed by a share handle, behind
  the auth gate ([53.auth-universe-gate](53.auth-universe-gate.md)) — enumerated so plan 15's typed route registry knows
  the slot exists. No page body is built.
- **The non-goals that protect v1.** State the invariants a later build must honor so nothing in v1 forecloses them
  (below), and the abuse/privacy surface v1 must not accidentally open.

**Non-goals (all deferred to the v-next social epic)**

- **The share act itself** — producing the shared subset (which episodic units, packaging, delivery) is
  [54.star-sharing](54.star-sharing.md), not here. This unit only _renders_ what 54 produced.
- **The `sharing.v1` read RPC + `internal/sharing` server context** — the public-universe read query, its access-control
  (who may open which handle), and the shared-subset assembly are backend, deferred with the epic (CC8, CC12). This plan
  fixes only the **FE-visible contract shape** (same DTO shapes as `GetUniverse`) it will consume.
- **Any write from a viewer.** No recall, reconsolidation, launch, letting-go, palette change, or comment on a public
  universe — it is strictly read-only. A viewer _recalling into their own_ universe from a shared star is a separate future
  question, not reserved here.
- **Discovery / feeds / public directories / permalinks / social graph.** No "browse all public universes," no follower
  model, no notifications. Only opening a specific shared handle.
- **Realtime / collaborative / live co-presence** — out; the model is async one-way (CC12).
- **Friend INVITE-for-growth** — that is v1 and lives in Epic F earn ([44.earn-spend-usecase](44.earn-spend-usecase.md),
  [G3]); it is **not** social sharing and **not** here (CC12).
- **New tuning scalars, new schema, new proto, policy/tech docs** — a placeholder introduces none (Values Impact below).

## Design

> **This is a placeholder plan.** The design records the _seam_ and the _constraints on v1_, not an implementation. The
> deferred build is authored later via `/create-plan` (or a change) when the social epic is scheduled.

### The deferred capability, precisely

A **public universe** is a read-only 3D render of a **shared subset** of another user's memory graph. The owner shares
episodic units (via [54.star-sharing](54.star-sharing.md)); the recipient later opens a **share handle** and navigates the
resulting universe. It is **async** (no live session), **one-way** (viewer cannot write back), and **read-only** (viewer
cannot mutate the owner's stars or their own via this view). Everything visual is the plan-23 universe; only the _source_
and the _mutability_ differ.

### The reserved read seam (FE contract shape only)

**Decided (2026-07):** the public universe is a **distinct read**, not a widened `GetUniverse`. A future `sharing.v1`
service exposes a read (e.g. `GetSharedUniverse(handle)`) returning the **same DTO shapes** as `GetUniverse`
(`EpisodicMemoryDto` / `NeuronDto` / `SynapseDto` + `universe_time`, plan 23) — the _stored facts only, no positions_
([I5]), scoped by the share grant rather than by the caller's own id. This keeps the invariant that `internal/memory`'s
own-universe read stays strictly self-scoped (CC8): a public read can never be spoofed by passing someone else's id into
`GetUniverse`. The FE need not change its **mappers** — the domain-mirror `entities/*` `api` maps the same DTO shapes; only
the _client call_ and the _ownership tag on the store_ differ. What v1 must guarantee now: the graph-builder / force-sim
feeder / canvas consume DTOs **from an injected source**, never assuming exactly one self-scoped fetch (already true in
plan 23 — the widget is a reader of whatever graph it is handed).

### The reserved render seam (read-only canvas)

A public universe mounts the **same** renderer boundary (plan 14) and the **same** `widgets/universe-canvas` scene (plan 23) in a **read-only mode**. Concretely, the mode the later build flips:

- **Navigation stays on** — zoom · rotate · pan · focus · fly ([V0]); the viewer explores freely.
- **Every write affordance is off** — no recall / reconsolidation / launch / letting-go / palette on this view; the
  star-detail panel ([35.star-detail-panel](35.star-detail-panel.md)) opens in a **read-only projection** (meta + shape +
  color only; the action buttons — 회고하기 / 변천사 보기 / 원본 일기 보기 — are absent, because provenance, Diary, and recall
  are the _owner's_ and not shared).
- **Layout is the viewer's own compute** — the viewer's client runs `packages/force-sim` (plan 19) over the shared graph;
  positions stay emergent ([I5]), never shipped from the owner. Center=self / radius=connectivity ([V1]) applies to the
  _shared subset's_ structure.
- **Nebula/palette follow the owner's stored emotion, but the viewer's active palette** — color=emotion is immutable
  ([I3][I11]); a viewer browsing sees the owner's stars' emotions honestly, mapped through the viewer's own palette
  (rendering is palette-agnostic; the mapping is the single entry, CC10). The viewer cannot re-emotion or re-palette the
  owner's stars.

### Placement (FSD) + parity — the slot only

- **`pages/public-universe`** (web) / a `PublicUniverseScreen` (mobile) — a reserved route/screen addressed by a share
  handle, behind the auth gate ([53.auth-universe-gate](53.auth-universe-gate.md)). Enumerated so plan 15's typed route
  registry and plan 13's navigation registry carry the slot; **no page body built in v1**.
- **A read-only variant of `widgets/universe-canvas`** — the later build adds a `readOnly` composition of the plan-23
  widget (same machine, graph-builder, sim bridge, entity `api`, shared verbatim through `@cosimosi/universe`) with write
  affordances omitted. It is **not** a fork of the engine; it is a mode. Web/mobile parity and `*.native` discipline are
  inherited from plan 23 unchanged.
- **`internal/sharing`** (backend) is the reserved server context (CC8) for both 54 and 55; it is **not** touched in v1.

### Golden-parity stance

None. This is a placeholder with no math and no shared FE↔BE domain logic. When built, it _reuses_ the golden-parity
layout ([19.force-sim-layout](19.force-sim-layout.md)) and read-time value fns ([18.synapse-plasticity](18.synapse-plasticity.md))
unchanged — a public universe derives brightness/strength/decay from the shared stored facts + `universe_time` exactly as
the owner's universe does; it introduces no new parity surface.

### What v1 must NOT preclude (the load-bearing part of this plan)

1. **The canvas reads from an injected graph source** — never hardwire "the universe = my `GetUniverse` fetch." A second,
   foreign, read-only source must drop in behind the same widget (plan 23 already satisfies this; keep it so).
2. **`GetUniverse` stays self-scoped** — do **not** add `owner_id`/`is_public`/`user_id` parameters to the own-universe
   read to "reuse it" for public browsing; the public read is a _separate_ grant-scoped RPC (`internal/sharing`). Widening
   the self read would open an authorization hole.
3. **Read-only is expressible without forking the widget** — write affordances (recall, launch, detail-panel actions,
   letting-go, palette) must be **compositional** (mode/props), not baked so deep that a read-only universe needs a
   parallel widget. Slices added by later P plans (35, 36, 45, 47, 50, 51) should keep their write actions injectable, not
   assumed-present.
4. **No viewer write can reach the owner's aggregate** — a browse is a pure read; it must be structurally impossible for
   opening a public universe to trigger any owner-side dynamic (recall, decay, clock advance). [I1]/[I10]: viewing never
   advances anyone's universe time.
5. **Positions/Diary/provenance are never shipped to a viewer** — only stored facts (the `GetUniverse` DTO shapes); the
   viewer re-derives layout ([I5]) and never sees the owner's original Diary ([I2]) or mutable provenance.

## Acceptance Criteria

> For a placeholder, "true" = **the reserved seam holds and nothing in v1 forecloses the deferred build.**

1. This plan **records the deferred capability** — browsing a public universe is **async, one-way, read-only**, deferred
   from v1 (PRD §5.8 **[S2]**, §8.3), reserving the slot per the placeholder discipline ([PRD.md](../PRD.md) §8.2) — and
   builds **no** v1 slice, RPC, schema, proto, or values key.
2. The plan states that a public universe reuses the **same** `@cosimosi/3d-renderer` boundary (plan 14) and
   `widgets/universe-canvas` scene (plan 23) in a **read-only mode** — navigation on, all write affordances off — and is
   **not** a new engine [V0].
3. The plan fixes the read seam: a public universe is a **distinct grant-scoped read** returning the **same DTO shapes** as
   `GetUniverse`, and **[S2]** must **not** be satisfied by adding `owner_id`/`is_public` parameters to `GetUniverse`, which
   stays strictly **self-scoped** and `NO_SIDE_EFFECTS` (CC8) [I1].
4. The plan reserves a `pages/public-universe` route (web) / screen (mobile), behind the auth gate
   ([53.auth-universe-gate](53.auth-universe-gate.md)), enumerated for plan 15/13's registries — with **no page body**.
5. The plan enumerates the invariants the deferred build must honor and v1 must not foreclose: positions stay emergent and
   are never shipped [I5]; the owner's Diary is never exposed/mutated [I2]; emotion→color and the meaning layer stay
   immutable for a viewer [I3][I11]; browsing triggers **no** owner-side dynamic and never advances universe time [I1][I10].
6. The plan names the **paired** deferred act ([54.star-sharing](54.star-sharing.md)) as the producer of the browsable
   subset and the reserved backend context (`internal/sharing`, CC8) as the future home — and marks INVITE-for-growth as
   **v1 / Epic F** ([44.earn-spend-usecase](44.earn-spend-usecase.md) [G3]), explicitly **not** this social slot (CC12).
7. The plan lists the write-affordance seams later P plans must keep **injectable/compositional** (recall/detail actions/
   launch/letting-go/palette) so a read-only universe needs no forked widget.
