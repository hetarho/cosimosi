# 24. star neuron rendering

> Star / neuron rendering — the three **visual (rendering) entities** that project the domain-mirror graph to bodies:
> `entities/star` (the episodic-memory big star — size · brightness · color · seed-form), `entities/cell-star` (the
> neuron point), and `entities/filament` (the synapse fat-line). They import the domain mirror via `@x`, project it
> one-way through the plan-14 asset-source port, and refine plan 23's generic instanced nodes / line-segment edges into
> the specific star / cell-star / filament bodies.
> Scope: FE | Status: built
>
> This is the presentation unit where the memory graph *becomes the universe*: the read-time appearance rules ([V3][V5]
> [V6][M3]) are applied to real stars, neurons, and synapses, on web (open beta) + mobile (MVP) from one TSL source.

## Purpose

Plan [23.universe-canvas](23.universe-canvas.md) mounts the renderer, wires the force-sim worker, and draws the domain
graph as **generic** instanced nodes + line-segment edges — placeholders with no product meaning. Plan
[14.rendering-foundation](14.rendering-foundation.md) provides the renderer boundary, the TSL shader-art toolkit, and
the **asset-source port** (`VisualBodySource`) through which bodies arrive without any slice importing `three`. What is
still missing is the **projection itself**: how an `EpisodicMemory` becomes a *big star* whose size is its strength,
whose brightness is its recall recency, whose color is its emotion, and whose shape is its unique seed; how a `Neuron`
becomes a *seedless point*; and how a `Synapse` becomes a *fat line* whose width and brightness track its effective
strength.

This unit builds those three visual entities and the TSL shader bodies behind them. It is the one place (ARCHITECTURE
§3.4) where the domain vocabulary (`EpisodicMemory` / `Neuron` / `Synapse`) and the visual vocabulary (`star` /
`cell-star` / `filament`) meet — and the projection is strictly **one-way**: visual words never travel back up into the
domain-mirror slices or the API mapper, and anything the domain treats as *emergent* (position, constellation) stays
emergent here too — rendered, never modeled.

## Scope / Non-goals

**In scope**
- **Three visual (rendering) entities**, each projecting one domain-mirror entity from plan 23 through the plan-14
  asset-source port, importing the mirror **only via `@x`** (`entities/star/@x/episodic-memory`, etc.):
  - **`entities/star`** — the big star body for `EpisodicMemory`. Maps four independent visual channels to domain facts
    (each cross-referenced to PRD §6):
    - **SIZE = EffectiveStrength** — arousal-modulated initial strength ([E8a]) + accumulated recall, read at render
      time by the shared `packages/memory-logic` function over the memory's stored facts ([V3]).
    - **BRIGHTNESS = EffectiveBrightness** — recall recency, read at render time by the shared `packages/memory-logic`
      function ([V2]). Forgetting (the decay curve that dims a star over universe time) is **Epic D**; Epic A renders
      **full brightness** and only reserves the seam — this unit notes the seam, it does not build decay.
    - **COLOR = the star's primary emotion** — obtained from the single swappable `mood → color` palette seam owned by
      plan [17.emotion-model](17.emotion-model.md) (`packages/emotion`) ([M3][V4]). Color comes from emotion **and
      nothing else** ([I3]).
    - **SHAPE = a unique per-star `seed`** — a TSL shader body that produces a coherent-but-varied big-star form driven
      by the stored `seed` (a meaningless visual value giving each star its identity) ([V5]).
  - **`entities/cell-star`** — the neuron body for `Neuron`: a **simple, seedless point** (no per-neuron form, no
    emotion color — a neuron carries information, not emotion) ([V5]). Positioned by the force-sim (plan
    [19.force-sim-layout](19.force-sim-layout.md)); this unit only renders the point at the coordinate the canvas feeds.
  - **`entities/filament`** — the synapse body for `Synapse`: a **batched camera-billboarded ribbon fat-line** whose
    **width and brightness = effective synapse strength** read at render time by plan 18's `EffectiveSynapseStrength`
    ([V6]). Filaments connect **neurons only** — never star↔star ([I4][I6][V6]).
- **TSL shader bodies, instanced.** Star bodies are authored in **TSL** (one source → WGSL for WebGPU + GLSL for the
  WebGL2 fallback, per §3.3), composed from the plan-14 `shader-art/` toolkit, and drawn as instanced meshes; cell-stars
  are instanced points; filaments are batched fat-lines. Per-instance appearance (size/brightness/color/seed) is fed
  through instance attributes / uniforms, not per-frame React state (§3.2/§3.3).
- **Refinement of plan 23, not a re-mount.** This unit swaps plan 23's generic node/edge bodies for the star /
  cell-star / filament bodies **inside** the existing `widgets/universe-canvas` scene and the existing asset-source
  wiring. It does not bootstrap the renderer, the force-sim worker, the camera rig, or the `GetUniverse` read.
- **Web (open beta) + mobile (MVP) from one TSL source** (§3.5). Mobile MVP may **lower instance counts / use fewer
  post-FX**; the shader bodies and the projection logic are shared verbatim.

**Non-goals**
- **No domain math and no read model.** `EffectiveStrength` / `EffectiveBrightness` (memory-level, `packages/memory-logic`,
  owned by plan 18), the synapse `EffectiveSynapseStrength` (plan 18), the `mood → color` palette (plan 17), the
  coordinates (plan 19), and the
  `GetUniverse` mapper (plan 23) are all consumed, not defined here.
- **No star (or universe-body) visual *design*.** This unit owns the **projection** — how domain facts map to the
  star's visual channels (size/brightness/color/shape-`seed`), the seedless neuron point, the synapse fat-line, and the
  instanced TSL body pipeline. It does **not** own the *designed aesthetic*: the star's silhouette/material, the
  seed-form grammar, the body/nebula/background look, and the emotion palette **values** are all owned by
  [57.3d-assets-and-background-design](57.3d-assets-and-background-design.md). Epic A renders a **first-pass,
  placeholder form** to prove the pipeline; plan 57 re-designs these bodies **within this same asset-source pipeline** —
  the same reserve-the-slot pattern [09.design-system](09.design-system.md) uses to defer the 2D visual identity to
  [56.2d-ui-design-language](56.2d-ui-design-language.md).
- **No forgetting / dimming.** The decay curve that actually dims a star over time is Epic D
  ([forgetting-visuals](00.overview.md)); Epic A renders full brightness and marks the seam only.
- **No seed change / reconsolidation.** The `seed` changes **only on reconsolidation** (`Reshape`, Epic C [R6]) — this
  unit renders whatever `seed` the read model carries and treats it as immutable input; it does not mutate or animate a
  seed change. Seam noted, not built.
- **No nebula, no latent stars.** The emotion color-blend nebula is plan [26.nebula-color-field](26.nebula-color-field.md);
  the gray latent-neuron background + awaken animation is plan [25.latent-star-field](25.latent-star-field.md).
- **No z-axis gist / abstraction.** Abstraction is expressed by z-axis rise + gist text (Epic C/E [C5]–[C7]), **not** by
  star shape ([V5]); no gist stars or neocortical layer here.
- **No product navigation / detail panel / writing flow** — camera rig ([U3][V0]) is plan 23; the star-detail panel is
  Epic C; the writing flow is plan [27.writing-flow-ui](27.writing-flow-ui.md).
- No hardcoded tuning scalars in FE code (visual ranges live in `values.yaml`; shader graphs + palette stay code).

## Design

### The three visual entities & the one-way projection (§3.4)
Each visual entity lives in `entities/<visual-noun>/` and its **only** upstream dependency on domain vocabulary is the
`@x` public API of a plan-23 domain-mirror entity:

| Visual entity | Projects (domain mirror, via `@x`) | Body | Channels |
|---|---|---|---|
| `entities/star` | `episodic-memory` | instanced TSL big-star mesh | size, brightness, color, shape(seed) |
| `entities/cell-star` | `neuron` | instanced seedless point | (position only; no color, no seed) |
| `entities/filament` | `synapse` | batched camera-billboarded ribbon fat-line | width + brightness = strength |

- **Projection is one-way.** A visual entity reads the domain-mirror type (via `@x`), reads shared read-time functions
  (`packages/memory-logic`, plan 18) and the palette seam (plan 17), and produces a **body**. It exports nothing back
  into the domain slice or the `api` mapper. No visual word (`star`, `cell-star`, `filament`, `radius`, `brightness`,
  `color`) becomes a domain symbol (enforced by the §1 cross-vocabulary lint).
- **Emergent stays emergent.** Position and constellation are *not* projected as stored facts: the body is placed at the
  coordinate the canvas (plan 23) feeds from the force-sim worker (plan 19). This unit never writes a coordinate back,
  never types a constellation.
- **Bodies arrive through the asset-source port.** Each entity's body is supplied via plan 14's `VisualBodySource`
  (`shader | glTF | primitive`) — the star is a `shader` source (TSL), the cell-star a `primitive`/instanced-point
  source, the filament a `shader` source (a custom camera-billboarded ribbon) — so no slice imports `three` directly
  (`three` stays confined to `@cosimosi/3d-renderer`, per tech/rendering.md).

### `entities/star` — the episodic-memory big star
Four **independent** visual channels, each a pure function of stored facts carried by the `episodic-memory` mirror
(from the `GetUniverse` DTO, plan 23) — so the same inputs always draw the same star:

- **Size (`EffectiveStrength`, [V3]).** Read at render time by the shared memory-level `EffectiveStrength` (plan 18) in
  `packages/memory-logic` from the memory's `base_strength` (arousal-modulated initial size, [E8a]) + `recall_count`.
  Size grows slowly (accumulation). Mapped to instance scale within a `rendering.star_size_min`…`star_size_max` range.
- **Brightness (`EffectiveBrightness`, [V2]).** Read at render time by the shared memory-level `EffectiveBrightness` (plan 18) in
  `packages/memory-logic`. **In Epic A the forgetting decay is not built**, so brightness resolves to full and this unit
  renders at full brightness; the channel + the shader uniform exist so Epic D's decay drops in without a body change.
  Mapped within `rendering.star_brightness_min`…`star_brightness_max`.
- **Color (emotion, [M3][V4]).** The memory's primary `Emotion` (mood) → a color via the single swappable `mood → color`
  palette seam (`resolvePalette()` / `usePalette()`-style) owned by plan 17 (`packages/emotion`). The star entity is
  **palette-agnostic** — it calls the seam, never a color table. Color is bound to emotion only ([I3]); no other channel
  feeds color.
- **Shape (`seed`, [V5]).** A **TSL shader body** parameterized by the star's stored `seed` (a `BIGINT` on the memory
  root) so every star shares a coherent visual grammar but takes a slightly different form — a meaningless visual value
  that gives the star its identity. Authored from the plan-14 `shader-art/` toolkit (noise/field/sdf → form), TSL-only
  so it compiles to WGSL + GLSL. **The `seed` is immutable input here**: it changes **only on reconsolidation**
  (`Reshape`, Epic C [R6][V5]); this unit renders the current seed and neither mutates nor animates a seed change
  (seam noted). Abstraction is **not** a shape channel — it is z-axis gist rise (Epic C/E [C5]–[C7]).

### `entities/cell-star` — the neuron point
- A **simple, seedless point** ([V5]). A neuron carries information, not emotion, so it has **no emotion color** and
  **no per-neuron seed-form** — it is not a reconsolidation target. Drawn as an instanced point body; per-instance
  appearance is minimal (a small fixed/derived point size, e.g. from connectivity/degree the `GetUniverse` DTO already
  carries for radius [V1]) — but **never emotion color**.
- **Position** comes from the force-sim (plan 19) via the canvas; this entity only renders the point at the fed
  coordinate. **Decided (2026-07):** Epic A ships a **small constant point size**; the degree fact stays reserved for a
  later subtle degree-driven size if the universe earns it.

### `entities/filament` — the synapse fat-line
- A **batched camera-billboarded ribbon fat-line** between the two neurons a `Synapse` connects. **Neuron↔neuron only**
  — a filament is **never** drawn between two stars ([I4][I6][V6]); the `synapse` mirror carries
  `neuron_a_id`/`neuron_b_id`, so a filament's two endpoints are always cell-star coordinates, structurally excluding a
  star↔star line.
- **Width + brightness = effective synapse strength ([V6]).** Read at render time by plan 18's `EffectiveSynapseStrength(base,
  elapsedUniverseDays)` from the synapse's stored `strength` + `last_activated_universe_time` (golden-parity with the Go
  domain). Mapped to line width within `rendering.filament_width_min`…`filament_width_max` and to line opacity/glow
  within `rendering.filament_brightness_min`…`filament_brightness_max`. Filaments are batched into one ribbon mesh per §3.3.

### Instancing, batching & the frame loop (§3.3)
- **Stars** = one (or a few bucketed) `InstancedMesh` of the TSL big-star body; per-star size/brightness/color/seed ride
  instance attributes + uniforms. **Cell-stars** = instanced points. **Filaments** = batched fat-lines.
- Per-frame updates read the coordinate buffer (force-sim worker output) and the read-time channel values into
  `InstancedMesh.matrix` / instance attributes / uniforms inside `useFrame` — **never** via React state (no 60fps
  re-renders, §3.2). The read-time channels themselves recompute only when the read model / universe time changes, not
  per frame.
- **Bucket capacity** reuses the existing `rendering.instance_bucket_size` (plan 14); star/cell-star/filament bucketing
  strategy (one bucket per body kind vs. shared) is an implementation detail confirmed at implement — the contract is
  only that bodies are instanced/batched and no slice touches `three`.

### Web + mobile parity (§3.5)
- One TSL source for the star body serves both platforms (WGSL on web WebGPU, GLSL on the WebGL2 fallback, the RN
  WebGPU host on mobile). The projection logic (`entities/*`) is pure/shared; only the canvas host wiring is already
  forked in plan 14.
- **Mobile MVP** may lower instance counts and drop some post-FX; the exact mobile reduction (instance caps, which
  post-FX are dropped) is **confirmed at implement via on-device RN profiling** (decided 2026-07) — the shared body and
  projection do not change.

### Seams noted, not built (so later epics drop in without a re-body)
- **Forgetting dimming** (Epic D) reuses the brightness channel/uniform already present.
- **Seed change / `Reshape`** (Epic C reconsolidation [R6]) reuses the seed uniform; a seed transition animation is a
  later choice.
- **z-axis gist stars** (Epic C/E) are a separate visual entity/layer — abstraction is z-rise, not star shape.

### As-built decisions
- **Bodies live in `packages/3d-renderer/src/assets/bodies/`** (`star-body.ts`, `cell-star-body.ts`,
  `filament-body.ts`), each a `VisualBodySource` factory. `three` stays confined to the package; the entity `ui`
  segments consume the package's layers/sources, never `three`.
- **`VisualBodyKind` is unchanged** (`shader | glTF | primitive`): the star and filament resolve as `shader`, the
  cell-star as `primitive`. No new kind was needed — the fat-line body is carried by a `Mesh` like the others.
- **Fat-line = a batched camera-billboarded ribbon, not three's `Line2`.** `Line2NodeMaterial`'s transparent path
  samples the opaque viewport texture the package's custom PostFX pipeline never exposes (WebGPU then rejects the bind
  group — recorded in plan 23's as-built). The `FatLineLayer` instead draws one batched ribbon mesh (4 verts / 2 tris
  per edge), billboarding each quad toward the camera with a half-width = synapse strength; it is genuinely a batched
  fat-line with variable width + additive-glow brightness, and survives the pipeline. Endpoints are neuron
  coordinate slots only, so a star↔star line is structurally impossible [I4][I6].
- **Per-instance star channels** ride `InstancedNodeLayer` (extended with an optional `channels` = per-instance scale +
  named instance attributes): size → per-instance matrix scale; tint (emotion, linear RGB) / brightness / seed → TSL
  `attribute()` reads. Channels recompute only on read-model / universe-time change; the coordinate buffer is read per
  frame in `useFrame`. The brightness attribute exists though it resolves full today — the Epic-D decay seam [V2].
- **`elapsedUniverseDays` (ISO-date → floored days) lives in `@cosimosi/memory-logic`**, the companion to
  `effectiveBrightness` / `effectiveSynapseStrength` (which take days as input) — one universe clock, no per-entity copy.
- **Boundary tooling:** the FSD `@x` cross-import is admitted by an `entities-x` element + `entities → entities-x` rule
  in each app's eslint boundaries config; the §1 ubiquitous-language lint treats an `entities/<rendering-term>` slice
  (star/cell-star/filament) as a visual path, so rendering vocabulary is native there while still forbidden in the
  domain-mirror slices and their api mappers.

## Acceptance Criteria
1. Three visual (rendering) entities exist — `entities/star`, `entities/cell-star`, `entities/filament` — each
   projecting exactly one plan-23 domain-mirror entity (`episodic-memory` / `neuron` / `synapse`) imported **only via
   `@x`**, and each emitting its body through the plan-14 asset-source port (`VisualBodySource`); no slice imports
   `three` directly.
2. The projection is **one-way**: no visual vocabulary (`star`/`cell-star`/`filament`/`radius`/`brightness`/…) appears
   as a domain symbol in the domain-mirror slices or their `api` mappers, and no visual entity writes back into the
   domain slice or the API mapper (the §1 cross-vocabulary lint passes).
3. A `star` body's **size** is its `EffectiveStrength` (arousal-initial [E8a] + recall accumulation) read at render time
   from `packages/memory-logic` — larger strength renders a larger star ([V3]).
4. A `star` body's **brightness** is its `EffectiveBrightness` read at render time from `packages/memory-logic`; in Epic
   A (no forgetting) it renders at full brightness and the brightness channel/uniform is present for Epic D to fill
   ([V2]) — the seam is noted, decay is not built.
5. A `star` body's **color** is its primary emotion via the single `mood → color` palette seam owned by plan 17
   (`packages/emotion`); the star entity is palette-agnostic and color is bound to emotion **only** — no other channel
   sets color ([M3][V4][I3]).
6. A `star` body's **shape** is a TSL shader form parameterized by the star's stored `seed`, giving a coherent-but-varied
   big-star form; two stars with different seeds render different forms ([V5]).
7. The `seed` is treated as **immutable input**: this unit renders the current seed and never mutates or animates a seed
   change — seed change happens **only on reconsolidation** (Epic C `Reshape` [R6]); abstraction is z-axis gist, **not**
   star shape ([V5][C5]–[C7]).
8. A `cell-star` (neuron) body is a **simple, seedless point** with **no emotion color** and no per-neuron seed-form,
   positioned at the force-sim coordinate the canvas feeds ([V5][I5]).
9. A `filament` (synapse) body is a **batched camera-billboarded ribbon fat-line** whose **width and brightness track its
   effective synapse strength** (plan 18 `EffectiveSynapseStrength`, read-time); a filament connects **two neurons only** and is **never** drawn
   between two stars ([V6][I4][I6]).
10. Emotion drives **only** color — it never sets a `cell-star`/`filament` geometry, a coordinate, or an edge ([I3]);
    positions/constellations stay emergent (drawn at fed coordinates, never stored or reverse-projected, [I5]).
11. Star bodies are authored in **TSL** (compiling to WGSL + GLSL) and drawn **instanced**; cell-stars are instanced
    points and filaments are batched fat-lines; per-instance appearance rides instance attributes/uniforms and per-frame
    updates run in `useFrame`, never React state (§3.3/§3.2).
12. The star / cell-star / filament bodies render on **web (open beta) from one TSL source shared with mobile (MVP)**;
    mobile is source-gate verified (typecheck · lint · test) with on-device render pending (`pnpm ios`). Mobile MVP
    may lower instance counts / drop post-FX without changing the shared body or projection (§3.5).
13. This unit **refines** the plan-23 scene: the generic instanced nodes / fat-line edges are replaced by the specific
    star / cell-star / filament bodies inside the existing `widgets/universe-canvas`; the renderer, force-sim worker,
    camera rig, and `GetUniverse` read are consumed, not rebuilt.
