# 53. 기억-상태 기반 별 형태 — 고유 모양 + 추상화 단계 변형

> 별마다 고유한 모양을 갖고, 잊혀(추상화)갈수록 형태가 단계적으로 변한다 | 범위: FE | 상태: 기획
> 지금은 모든 별이 사실상 같은 실루엣(기본 lowpoly는 회전만 다름)이다. 기억이 저마다 다르게 생기고, 시간이 지나며 형태가 변하는 걸 표현한다.

## 목적

기억은 저마다 다른 모양이어야 하고, 잊혀갈수록 윤곽이 뭉개져 "요지"만 남아야 한다(요지화). 현재는 형태(form) 빌더가 seed를 **회전/애니메이션**에만 쓰고 **지오메트리(실루엣)** 엔 거의 안 써서, 같은 form이면 모든 별이 같아 보인다(liquid만 예외). 이 plan은 (1) 별마다 고유한 형태, (2) 추상화 단계(1~4)에 따른 형태 변형, (3) 볼 때마다의 미세 변형을 도입해 "살아 변하는 기억"을 시각화한다.

## 범위 / 비목표

**포함:**
- 별마다 **고유 seed**(탄생 시 결정). 형태 빌더가 seed로 *지오메트리*(정점 변위/면 수/실루엣)를 바꿔 별마다 다른 모양.
- 탄생 시 **3개가량의 seed 성분**을 부여(예: 형태 jitter·비대칭·표면 거칠기 축) — form별로 의미 있는 변형 축.
- **추상화 단계 1~4**(`abstraction_stage`, change 20 야간 요지화가 승급)에 따라 형태가 단계적으로 변형 — 단계가 높을수록 더 단순/추상(요지화). form별로 4단계 모델이 사실상 존재.
- 별을 **볼 때마다** 형태가 미세하게 변함(재공고화의 시각 표현 — seed 미세 jitter).

**비목표:**
- 색(감정)·크기(intensity)·밝기(거리) 규칙 변경 — 불변(plan 44 색, change 18·19).
- form/surface **선택 UI**(plan 52 2축 스킨) 변경 — 이 plan은 "기억 상태가 굴리는 형태"이지 "사용자가 고르는 스킨"이 아니다. 둘은 직교(사용자가 고른 form 위에서 고유 seed·추상화가 변형).
- AI 기반 *내용* 변형 — 별도(plan 54). 여기선 *형태(지오메트리)* 만.

## 설계 요점

- **seed 다축화.** 현재 단일 `seed`(FNV-from-id) + `form_seed_delta`를 **다성분 seed**로 확장(탄생 시 결정론 3축). 각 form 빌더가 축을 받아 지오메트리를 변형(정점 변위·비대칭·면 수).
- **form 빌더가 지오메트리에 seed 반영.** lowpoly 등 고정 지오메트리 form도 seed로 정점을 흔들거나 detail을 조절해 별마다 실루엣이 다르게. (liquid의 변위 패턴을 일반화.)
- **추상화 단계 → 형태.** `abstraction_stage`(0~4)가 form의 단순화 정도를 구동: 높을수록 면 수↓·디테일↓·윤곽 뭉개짐("요지"). form별 단계 4개가 사실상 4개의 모델 변형.
- **볼 때마다 미세 변형.** 회상/열람 시 seed에 미세 jitter를 더해(재공고화 시각화) 같은 별이 조금씩 다르게 보임. 누적 캡으로 폭주 방지. (서버 영속 여부·트리거는 plan 54의 재공고화 경로와 정합.)
- 전부 셰이더/CPU 지오메트리에서 **결정론**(같은 seed·stage → 같은 모양), Math.random 비사용 — 우주가 매번 안 뒤집힘.

## 수용 기준 (인수 조건)
1. 같은 form을 쓰는 두 별이라도 seed가 다르면 **실루엣(지오메트리)이 눈에 띄게 다르다**(회전만 다른 게 아님).
2. `abstraction_stage`가 오르면 형태가 단계적으로 더 단순/추상화된다(1~4, 단조).
3. 같은 memory id·stage·seed면 항상 같은 모양(결정론, 세션·기기 무관).
4. 별을 회상/열람하면 형태가 미세하게 변한다(누적 캡 내).
5. 색·크기·밝기는 이 변경으로 안 바뀐다(회귀 경계).
6. 사용자가 고른 form/surface(plan 52)는 그대로 존중되며, 그 위에서 고유 seed·추상화가 변형으로 얹힌다.
