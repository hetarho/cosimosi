# 11. 회상·공동 회상 강화 (recall-reinforce)

> 별을 열어 불변 원본 일기를 읽고 이웃 시냅스로 항해하며, 함께 본 두 별 사이 연결을 클라 로컬+unary 배치로 강화한다. | 범위: MVP | 선행: 04, 09, 10 | 차단: 12

## 목적 (Why)
탐험(회상)이 우주를 강화하는 핵심 루프를 닫는다. 별을 클릭하면 그날의 **불변 원본 일기**가 뜨고(수정 불가), 이웃 시냅스를 따라 다른 별로 항해한다. 두 별을 함께/오가며 회상할 때마다 그 사이 연결이 **헵 규칙**으로 강해지고(공동 회상), 회상한 별은 재점화되어(`last_recalled_at=now`) 다시 밝아질 채비를 한다. 강화는 RN 미지원 스트리밍 대신 **클라 로컬 누적 + unary 배치 업서트**로 영속화한다.

## 범위
- **포함:**
  - FE `features/recall`: 별 클릭→원본 일기 패널(읽기 전용), 이웃 시냅스 항해(클릭한 별의 연결 이웃을 강조·이동), 회상 강화 증분 delta 로컬 누적+디바운스(유휴 ~5s)+`beforeunload`→`ReinforceLinks` 배치 호출(`batch_id` 포함), 별 ≥2초 능동 열람→`RecallMemory` 호출.
  - FE `entities/synapse`(또는 `entities/memory`) 엣지 스토어: `GetUniverse` synapses→`SynapseEdge` 매퍼 + 엣지 적재 스토어 + "선택 별 이웃 셀렉터"(10이 무시한 synapses를 11이 적재).
  - BE `memory`(단일 `MemoryService`): `ReinforceLinks` RPC — `UNNEST` 배치 + `ON CONFLICT` 원자적 업서트(`weight=LEAST(1.0, weight+delta)`, `co_activation_count++`, `last_activated_at=now`), `processed_batches`로 batch_id 멱등. `RecallMemory` RPC — `last_recalled_at=now` 갱신 + records JOIN으로 불변 원본 `Record` 반환(수정 경로 없음, `RETURNING *` 금지).
  - BE `db/queries/link.sql`(04 생성됨, 수정/확장): 공동 회상 증분 delta 배치 업서트 쿼리 추가.
- **비목표(Out of scope):**
  - 감쇠 시각·잠든 별 탐색·강화 후 "밝아짐" 렌더 검증 → **12(decay-dormant)**. (여기선 데이터 영속까지만; 밝아짐 확인은 12에서.)
  - 별 재성형(형태 변형)·변천사 로그(`evolution_history`) → **v1 #20/#21**. 여기서 갱신하는 건 `last_recalled_at`뿐.
  - proto 계약·서버 와이어링·인증 인터셉터 정의 → **02**. `ReinforceLinks`/`RecallMemory`는 02의 단일 `MemoryService`에 이미 정의됨(LinkService 신설 금지). 본 스펙은 02 계약을 구현한다.
  - 시냅스 렌더·`weight·brightness` 셰이더 → **09**. 09는 `entities/synapse`의 순수 `SynapseEdge` 타입·매핑·`SynapseLines`(edges props)만 만들고 **엣지 스토어/매퍼는 만들지 않는다**(엣지 출처를 04 GetUniverse로 위임). 따라서 GetUniverse synapses→`SynapseEdge` 매퍼·엣지 스토어·이웃 셀렉터는 **본 스펙(11)이 신설**한다.

## 설계 요점
**계약(proto, 02가 권위 — 단일 `MemoryService`).** 두 RPC 모두 unary(원칙 6). `ReinforceLinks`/`RecallMemory`는 02의 `MemoryService`에 이미 정의됨(별도 서비스 신설 금지 — 단일 `MemoryService` 유지). 본 스펙은 02 계약을 구현한다.
```proto
// proto/cosimosi/v1/ — service MemoryService (02 권위)
message LinkDelta { string a_id = 1; string b_id = 2; double delta_weight = 3; } // 클라가 정규화: a_id < b_id, delta는 증분
message ReinforceLinksRequest { repeated LinkDelta items = 1; string batch_id = 2; } // 디바운스 윈도의 증분 누적 + 멱등 batch_id
message ReinforceLinksResponse {}
message RecallMemoryRequest { string memory_id = 1; }
// Record = 불변 원본(body 포함, records 출처). Memory(별 투영)와 다름.
message Record { string memory_id = 1; string body = 2; string entry_date = 3; Mood mood = 4; double intensity = 5; string created_at = 6; }
message RecallMemoryResponse { Record record = 1; }
```

**BE 원자적 배치 업서트(`db/queries/link.sql`, 04 생성됨 → 수정/확장, sqlc).** `:copyfrom`은 ON CONFLICT 불가 → `UNNEST` 사용. delta는 **증분**, 상한 1.0(§6). 신규 행은 `weight=delta`, `link_type='co_recall'`, `user_id`는 컨텍스트에서 주입. batch_id는 `processed_batches`로 멱등(이미 본 batch_id면 skip — 재전송 이중 가산 방지).
```sql
-- name: BatchAlreadyProcessed :one  (link.sql) — 멱등 가드(이미 본 batch_id면 true)
SELECT EXISTS (SELECT 1 FROM processed_batches WHERE batch_id = @batch_id AND user_id = @user_id);
-- name: MarkBatchProcessed :exec  (link.sql)
INSERT INTO processed_batches (batch_id, user_id) VALUES (@batch_id, @user_id)
ON CONFLICT (batch_id) DO NOTHING;
-- name: ReinforceLinks :exec  (link.sql) — 증분 delta UNNEST 업서트
INSERT INTO memory_links (a_id, b_id, user_id, weight, link_type, co_activation_count, last_activated_at, created_at)
SELECT a, b, @user_id, d, 'co_recall', 1, now(), now()
FROM unnest(@a_ids::text[], @b_ids::text[], @deltas::float8[]) AS t(a, b, d)
ON CONFLICT (a_id, b_id) DO UPDATE
SET weight              = LEAST(1.0, memory_links.weight + EXCLUDED.weight),
    co_activation_count = memory_links.co_activation_count + 1,
    last_activated_at   = now();
-- name: RecallMemoryTouch :exec  (memory.sql, 04 생성 파일) — 별만 가변(영속)
UPDATE memories SET last_recalled_at = now() WHERE id = @id AND user_id = @user_id;
-- name: GetRecordByMemory :one  (memory.sql) — 불변 원본 읽기(records JOIN, RETURNING * 금지)
SELECT r.body, r.entry_date, r.mood, r.intensity, r.created_at
FROM memories m JOIN records r ON r.id = m.record_id
WHERE m.id = @id AND m.user_id = @user_id;
```
> 한 배치는 `batch_id`로 멱등: `BatchAlreadyProcessed`가 true면 전체 skip, 아니면 `ReinforceLinks` 후 `MarkBatchProcessed`(같은 트랜잭션). 재전송(1.7 재시도) 시 이중 가산 없음. `EXCLUDED.weight`는 그 호출의 증분 delta. 동시 호출 원자성은 행 단위 `ON CONFLICT`가 보장하되, 같은 batch_id 동시 중복은 `processed_batches.batch_id PK`가 직렬화. `RecallMemory`는 (a) `RecallMemoryTouch`(memories last_recalled_at만 갱신) → (b) `GetRecordByMemory`(records JOIN으로 불변 원본 읽기)의 **2단계**(memories에는 body 없음 — `RETURNING *` 금지).

**공동 회상 트리거(조작적 정의).** 별 상세를 **≥2초 능동 열람**하고 **직전 능동 열람 별과 페어링**될 때 1 co-recall 이벤트(`delta_weight = 0.05`). 단순 스크롤·스침·2초 미만 열람은 카운트하지 않는다(능동 인출 기준 — concept §회상). 페어는 정규화(`a_id<b_id`)·세트화하므로 한 윈도에서 같은 페어 중복은 1 이벤트(증분 delta 0.05)로 합산.

**FE 로컬 누적·디바운스(`features/recall/model`, 순수 — three/DOM 미의존, 원칙 4).** 페어별 `delta_weight`를 로컬 **증분 누적**, **디바운스(유휴 ~5s) + `beforeunload`**에 `ReinforceLinks` 1회 배치(`batch_id` 부여). 실패 시 누적 유지·재시도(1.7), 단 같은 batch_id 재전송은 BE가 멱등 처리.
```ts
// model/co-recall.ts (순수)
type Pair = `${string}|${string}`;            // 정규화된 a|b (a<b)
const CO_RECALL_DELTA = 0.05;                 // 이벤트당 증분
const DWELL_MS = 2000;                        // ≥2초 능동 열람 임계
const DEBOUNCE_IDLE_MS = 5000;                // 유휴 ~5s flush
interface RecallSession { deltas: Map<Pair, number>; lastViewedId: string | null; batchId: string; }
function onActiveView(s: RecallSession, id: string): void; // ≥2초 열람 확정 시 호출: 직전 능동열람 별과 페어에 +0.05 누적
function drainDeltas(s: RecallSession): { items: { aId: string; bId: string; deltaWeight: number }[]; batchId: string }; // 배치 페이로드로 비우기(flush 시)
```
**FE 엣지 스토어·이웃 셀렉터(`entities/synapse/model` 또는 `entities/memory/model` — 11 신설).** 10이 무시한 `GetUniverse` synapses를 11이 적재한다: (a) `GetUniverse` synapses→`SynapseEdge`(09 타입) 매퍼, (b) 엣지 적재 zustand 스토어(`edges: SynapseEdge[]; setEdges`), (c) "선택 별 이웃 셀렉터"(`neighborsOf(memoryId): SynapseEdge[]` — `aId|bId`에서 선택 별이 끼인 엣지 추출). NeighborNav가 이 셀렉터를 소비.
```ts
// entities/synapse/model/store.ts (또는 mapping.ts) — 11 신설
function toSynapseEdge(s: GetUniverseSynapse): SynapseEdge;  // synapses[]→SynapseEdge[] 매퍼(weight/linkType, brightness/reinforcedRecency는 12 배선 전 기본값)
const useSynapseStore: () => { edges: SynapseEdge[]; setEdges(e: SynapseEdge[]): void };
const neighborsOf: (memoryId: string) => SynapseEdge[];      // 선택 별이 끼인 엣지(이웃 항해)
```
**FE UI(`features/recall/ui`, 플랫폼 레이어).** 별 선택→**≥2초 dwell 확정** 시 `onActiveView` 누적 + `RecallMemory`로 원본 `Record` 패널 표시(읽기 전용, 편집 컨트롤 없음 — 원칙 1). 패널의 이웃 목록은 **11이 신설한 엣지 스토어의 `neighborsOf(selectedId)`**에서 선택 별 엣지를 읽어 렌더, 클릭 시 카메라 포커스 이동.

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 사용자가 별을 클릭하면 THE SYSTEM SHALL `RecallMemory`가 반환한 **불변 원본 `Record`(본문·작성일·감정 — records 출처, JOIN)**를 읽기 전용 패널로 표시하고, 편집·삭제 UI를 노출하지 않는다. _(원칙 1)_
1.2 WHEN 별이 ≥2초 능동 열람되면 THE SYSTEM SHALL `RecallMemory`를 호출해 `memories.last_recalled_at=now`만 갱신하고(별만 가변), 원본은 records JOIN으로 읽어 반환한다(`memories RETURNING *` 금지 — body 없음). _(원칙 1, §6 재점화)_
1.3 GIVEN 한 회상 세션에서 두 별 A·B를 함께/오가며 ≥2초씩 능동 열람했을 때 WHEN 디바운스 윈도가 닫히면(유휴 ~5s/언마운트/`beforeunload`) THE SYSTEM SHALL 정규화·합산한 증분 delta 페어를 `batch_id`와 함께 `ReinforceLinks`로 **단일 unary 배치** 호출한다. _(원칙 6)_
1.4 WHEN `ReinforceLinks`가 페어 (a,b,delta)를 받으면 THE SYSTEM SHALL `a_id<b_id`로 정규화된 행에 `weight=LEAST(1.0, weight+delta)`, `co_activation_count++`, `last_activated_at=now`를 원자적으로 적용하고, 행이 없으면 `weight=delta, link_type='co_recall', user_id=$ctx`로 생성한다. _(§6, §4.5)_
1.5 WHEN 동일 페어로 `ReinforceLinks`가 동시에 다중 호출되면 THE SYSTEM SHALL 행 단위 `ON CONFLICT`로 갱신을 직렬화하여 손실 없이 누적한다(weight 상한 1.0 유지). _(§4.8 함정 5)_
1.6 WHEN 페이지를 새로고침하면 THE SYSTEM SHALL 강화된 `weight`가 영속되어 `GetUniverse`(04)가 굵어진 시냅스(`synapses[]`)를 반환한다. _(영속)_
1.7 IF `ReinforceLinks` 배치가 네트워크 오류로 실패하면 THEN THE SYSTEM SHALL 로컬 증분 누적(Map)을 비우지 않고 같은 `batch_id`로 다음 윈도에서 재시도한다(강화 이벤트 유실 방지).
1.8 IF `RecallMemory`의 `memory_id`가 (해당 user_id로) 존재하지 않으면 THEN THE SYSTEM SHALL `NotFound`를 반환하고 별을 생성·수정하지 않는다.
1.9 WHEN `features/recall/model`이 빌드되면 THE SYSTEM SHALL `three`/React/DOM을 import하지 않는다(순수·모바일 재사용). _(원칙 4)_
1.10 WHEN 같은 `batch_id`의 `ReinforceLinks`가 재전송(재시도/중복)되면 THE SYSTEM SHALL `processed_batches`로 이미 본 batch_id를 식별해 **skip**하여 이중 가산을 방지한다. _(멱등)_
1.11 WHEN 사용자가 별을 단순 스크롤·스침하거나 2초 미만만 열람하면 THE SYSTEM SHALL co-recall 이벤트로 카운트하지 않는다(능동 인출 기준). _(공동 회상 조작적 정의)_
1.12 WHEN `GetUniverse`(04) 응답이 도착하면 THE SYSTEM SHALL `synapses[]`를 `SynapseEdge`로 매핑해 엣지 스토어에 적재하고, `neighborsOf(memory_id)`로 선택 별의 이웃 엣지를 셀렉트해 NeighborNav에 공급한다(10이 무시한 synapses를 11이 적재).

## 완료 정의 (DoD)
- 별 클릭 시 **불변 원본 일기**(`RecallMemory` 응답 `Record` — body는 records JOIN으로 읽음)가 읽기 전용으로 뜬다(편집 경로 없음). `RecallMemory` 후 `memories.last_recalled_at`만 갱신되되 `body`·`entry_date`·`mood`는 불변(원칙 1). memory.sql/link.sql에 `memories RETURNING *`가 없다(grep 확인).
- 두 별을 ≥2초씩 함께/오가며 회상→디바운스(유휴 ~5s)/`beforeunload` 후 **단일 unary `ReinforceLinks` 배치**(`items[]` + `batch_id`) 1회 호출(네트워크 탭에서 streaming 없음 — 원칙 6 확인), 사이 `weight`가 증분 delta(`+0.05`, 상한 1.0) 증가·`co_activation_count++`. 단순 스크롤·2초 미만 열람은 co-recall 미카운트.
- **새로고침 후에도 강화가 유지**된다: `GetUniverse`(04) 응답의 해당 엣지(`synapses[]`) `weight`가 증가해 있다(영속).
- 멱등: 같은 `batch_id`로 `ReinforceLinks`를 2회 보내도 `processed_batches` 가드로 weight가 한 번만 가산된다(이중 가산 없음).
- 동시 호출 원자성: 동일 페어로 `ReinforceLinks`를 동시에 2회(서로 다른 batch_id) 호출해도 weight가 누적·직렬화되고 1.0을 넘지 않는다.
  ```bash
  # Docker/WSL — 단일 MemoryService; 증분 delta + batch_id
  docker run --rm fullstorydev/grpcurl -plaintext -d '{"items":[{"aId":"m1","bId":"m2","deltaWeight":0.05}],"batchId":"b-001"}' host.docker.internal:8080 cosimosi.v1.MemoryService/ReinforceLinks
  ```
- 이웃 항해: `GetUniverse` `synapses[]`가 `SynapseEdge`로 적재되고(11 신설 엣지 스토어), NeighborNav가 선택 별의 `neighborsOf(memory_id)` 이웃을 렌더한다.
- `pnpm --filter frontend build`·`pnpm lint` 통과. `features/recall/model`이 `three`/React/DOM을 import하지 않음(원칙 4, grep 확인).
- Docker/WSL: `docker run --rm -v ${PWD}/backend:/src -w /src golang:1.26 go build ./...` 성공(`.exe` 호스트 직접 실행 안 함).
