# 27. 우주의 수면 — 야간 공고화 (nightly-consolidation)

> 야간 배치가 4단계(전체 재안정화·재분배·요지 추상화·약한 선 가지치기)로 우주를 한 번 정리하고 흥분성을 리셋해, 아침에 살짝 달라진 풍경을 보게 한다 — 무엇도 삭제하지 않는다. | 범위: FS | 선행: 22, 23, 26 | 차단: —

## 목적 (Why)
concept §야간 공고화("우주의 수면")를 실제 배치로 구현하는 **캡스톤**이다. 낮 동안 부분 배치(07)로 위치 오차가 쌓이고, 새 별·재공고화(23)·감쇠(26)로 그래프가 어긋난다. 수면 중 뇌가 최근 기억을 재활성화해 큰 구조로 **재분배**하고 **요지를 추출**하며 약한 연결을 정리하듯(능동적 체계 공고화), 야간에 사용자별로 한 번 `consolidate` 잡을 돌려 우주를 다시 정돈한다. 단계는 4개로 순서가 있다: ① 전체 재안정화(force-sim 재수렴 + 안정 좌표 캐시) ② 호스트 성단 중심으로 재분배(스키마 적합 보너스) ③ 오래된 저회상 별의 형태를 한 단계 더 단순화(요지) ④ 약하고 안 쓰인 선의 **밝기**를 바닥으로(가지치기). 그 뒤 성단 흥분성(22)을 리셋해 24h 회전 주기를 만든다. 사용자는 다음 접속 때 "밤사이 우주가 한 번 정리됐어요"라는 살짝 달라진 풍경(morning diff)을 본다. **별·선은 절대 삭제하지 않는다(헌법2). 원본 기록은 손대지 않는다(헌법1).**

## 범위
- **포함:**
  - BE: `internal/job/job.go`에 `KindConsolidate Kind = "consolidate"` 추가(05의 `KindEmbed`·20의 `KindExtract` 옆).

  - BE: `internal/job/worker.go`에 `consolidate` 스테이지 + 4패스 순차 실행. 좌표 재안정화는 **순수 `shared/lib/force-sim` 알고리즘과 동치인 서버 측 tick**(좌표는 캐시 권위가 아니라 보조 — 헌법3은 클라 산출이 권위, 서버는 재진입을 빠르게 하는 안정 좌표 *캐시*만 둔다)으로 수렴값을 얻어 캐시.

## 설계 요점
**파라미터(단일 출처 — `worker.go` 상수).**
```go
// internal/job/worker.go — consolidate 패스 상수
const (
    redistributeLerp     = 0.6              // ② 호스트 성단 중심으로 끌어들이는 보간 비율
    schemaBonus          = 0.15             // ② 도식(반복 성단)에 적합한 별 추가 가중(흥분성·연결도 기반)
    gistAgeDays          = 30.0             // ③ 이보다 오래되고 저회상이면 요지 대상
    gistFormSimplify     = 0.4              // ③ form_seed_delta를 이 비율만큼 단조 증가(형태 단순화)
    gistRecallCutoffDays = 14.0             // ③ "저회상" 판정: last_recalled_at이 이보다 과거
    weakEdgeThreshold    = 0.2              // ④ weight < 0.2 이고…
    weakEdgeIdleDays     = 14.0             // ④ …last_activated_at이 이보다 과거인 선이 가지치기 대상
    // 흥분성은 22 소관(TAU_EXC=6h); 야간에 e(c,t)를 0으로 리셋해 24h 회전 주기를 만든다.
)
```
- 모든 임계는 비교(시각·weight)만 사용 — SQL `WHERE`에 `exp()`/감쇠식 금지(sargable; 밝기·`λ_eff`는 26 클라 모델 권위).

**야간 티커(`cmd/worker/main.go`).** 별도 워커 프로세스가 매일 정해진 시각(또는 cron)에 활성 사용자별로 `consolidate` 잡을 enqueue한다. enqueue는 RecordMemory 트랜잭션 밖이므로(05 주석: 소비자 `Repository`엔 `Enqueue`가 없음) **티커 전용 enqueue 경로**(`internal/job`에 풀 스코프 `EnqueueConsolidate(ctx, userID)` 또는 `db/queries/job.sql`의 `EnqueueJob` 재사용)를 둔다. `consolidate` 잡엔 별 단위 `memory_id`가 필요 없으므로 `jobs.memory_id`는 사용자 단위 sentinel을 쓰거나 `jobs`에 nullable `user_id` 컬럼을 더해 사용자 스코프로 claim한다(마이그레이션에서 선택).
```
ticker(daily) → 각 active user → EnqueueConsolidate(user)
worker.Claim(ctx, KindConsolidate) → consolidate 4패스 → Complete   |  실패 → 지수 백오프(기존 failWithBackoff 재사용)
```

**4패스(순서 보장, `worker.go` `handleConsolidate`).**
1. **전체 재안정화(re-layout, 결정1).** 사용자 전 노드/엣지를 로드해 force-sim과 **동치인** 서버 측 순수 tick으로 alpha가 임계 이하가 될 때까지 수렴 → 안정 좌표를 캐시(아래 SQL). 좌표는 권위가 아니라 클라 재진입 가속용 캐시(헌법3). 기존 별 pin 정책은 야간엔 풀어 **전체** 재안정화.
2. **재분배·통합.** 각 별을 호스트 성단(최강 연결 군집)의 centroid로 `lerp≈0.6` 당기고, 도식(반복 인물·장소·주제로 형성된 성단)에 적합한 별엔 `schemaBonus`만큼 추가로 끌어들인다(흥분성·연결도 기반 — 22의 `e(c,t)` 활용). 결과는 ①의 캐시 좌표에 반영.
3. **요지 추출(gist).** `now - created_at > gistAgeDays` 이고 `last_recalled_at < now - gistRecallCutoffDays`인 별의 `form_seed_delta`를 `gistFormSimplify`만큼 **단조 증가**(이미 단순화된 별은 더 단순화만, 절대 복잡화 금지)해 형태를 한 단계 추상화. 각 변경은 `evolution_history`에 `trigger='nightly_gist'`로 **append**(23 테이블 재사용, INSERT-only — 헌법1/2).
4. **가지치기(prune).** `weight < weakEdgeThreshold` 이고 `last_activated_at < now - weakEdgeIdleDays`인 선의 **밝기를 바닥으로**(weight를 floor 부근으로 reweight; 26 brightness FLOOR 정신) — **행은 남기고 클릭 가능**. `DELETE` 절대 없음(헌법2).
5. **흥분성 리셋(24h 회전).** 성단별 `e(c,t)`(22)를 0으로 리셋해 다음 낮의 경쟁적 할당이 새로 시작되게 한다.

**안정 좌표 캐시 마이그레이션(헌법3 — 캐시일 뿐 권위 아님).**
```sql
-- db/migrations/00006_nightly_consolidation.sql (예시; 03 스키마 비파괴 ALTER)
-- +goose Up
-- 별의 형태 진화 상태(23이 Star에 노출): 야간 요지/재성형이 갱신하는 가변 layer.
ALTER TABLE memories ADD COLUMN IF NOT EXISTS form_seed_delta REAL NOT NULL DEFAULT 0;
-- 야간 재안정화가 캐시하는 안정 좌표(권위 아님; 클라 force-sim 재진입 가속용). NULL이면 클라가 처음부터 산출.
ALTER TABLE memories ADD COLUMN IF NOT EXISTS stable_x REAL;
ALTER TABLE memories ADD COLUMN IF NOT EXISTS stable_y REAL;
ALTER TABLE memories ADD COLUMN IF NOT EXISTS stable_z REAL;
-- evolution_history(23 신설)는 재사용 — 야간은 trigger='nightly_gist'로 INSERT만 한다.
-- +goose Down
ALTER TABLE memories DROP COLUMN IF EXISTS stable_z;
ALTER TABLE memories DROP COLUMN IF EXISTS stable_y;
ALTER TABLE memories DROP COLUMN IF EXISTS stable_x;
-- form_seed_delta는 23이 추가했다면 23 마이그레이션 소관 — 본 스펙은 부재 시에만 보강.
```
> ⚠️ `form_seed_delta`는 23이 먼저 `memories`에 추가할 수 있다. 그 경우 본 스펙은 `ADD COLUMN IF NOT EXISTS`로 **중복 정의를 피하고** stable 좌표만 신설한다(23과의 순서 충돌 방지). evolution_history 테이블 자체는 23이 만든다.

**SQL(재안정화 캐시·요지·가지치기·흥분성 리셋).**
```sql
-- db/queries/memory.sql
-- name: CacheStableCoords :exec
-- 야간 재안정화(①②) 결과 좌표 캐시. 권위 아님(헌법3) — 클라 재진입 가속용. UNNEST 배치 upsert.
UPDATE memories AS m SET stable_x = c.x, stable_y = c.y, stable_z = c.z
FROM (SELECT unnest(@ids::text[]) AS id,
             unnest(@xs::float4[]) AS x, unnest(@ys::float4[]) AS y, unnest(@zs::float4[]) AS z) AS c
WHERE m.id = c.id AND m.user_id = @user_id;

-- name: GistSimplifyStars :exec
-- ③ 요지: 오래되고 저회상인 별의 form_seed_delta를 단조 증가(GREATEST로 후퇴 금지). 원본 record 불변(헌법1).
UPDATE memories
SET form_seed_delta = GREATEST(form_seed_delta, LEAST(1.0, form_seed_delta + @simplify::float4))
WHERE user_id = @user_id
  AND created_at < sqlc.arg(age_cutoff)::timestamptz
  AND last_recalled_at < sqlc.arg(recall_cutoff)::timestamptz;

-- name: ListStarsForConsolidate :many
-- ①② 입력: 전 별 좌표 캐시·연결도 재계산용. mood/intensity는 records JOIN(별=가변, 원본=불변).
SELECT m.id AS memory_id, m.created_at, m.last_recalled_at, m.form_seed_delta, r.mood, r.intensity
FROM memories m JOIN records r ON r.id = m.record_id
WHERE m.user_id = $1 ORDER BY m.created_at;
```
```sql
-- db/queries/link.sql
-- name: PruneWeakLinks :exec
-- ④ 가지치기: 약하고 안 쓰인 선의 weight를 바닥으로(밝기↓). 행은 남긴다 — DELETE 금지(헌법2).
-- 밝기는 26 클라 모델이 weight로 산출하므로 weight를 floor 부근으로 내리면 선이 어두워질 뿐 사라지지 않는다.
UPDATE memory_links
SET weight = LEAST(weight, sqlc.arg(floor)::float4)
WHERE user_id = @user_id
  AND weight < sqlc.arg(weak_threshold)::float4
  AND last_activated_at < sqlc.arg(idle_cutoff)::timestamptz;
```
> ⚠️ 흥분성 `e(c,t)`(22)는 워커 메모리에서 계산되는 파생 상태이므로 "리셋"은 영속 테이블이 있으면 0 UPDATE, 없으면 워커 인메모리 캐시 클리어다(22가 영속화하지 않았다면 SQL 없음 — 22 구현에 맞춰 택일).

**Star/evolution 진화 노출(23 계약 재사용).** 요지 단계는 `Star.form_seed_delta`(23이 proto에 추가)를 갱신하고 `evolution_history`에 `trigger='nightly_gist'` 스냅샷을 남긴다. 본 스펙은 **proto를 새로 바꾸지 않는다**(23이 정의한 필드·테이블 그대로 소비). 24의 변천사 UI가 야간 스냅샷을 그대로 보여준다.

**FE morning diff(`UniverseCanvas.tsx`).** 다음 `GetUniverse` refetch에서 갱신된 `form_seed_delta`(요지)·가지치기로 어두워진 선·안정 좌표 캐시가 반영돼 장면이 살짝 달라진다. 좌표 캐시(`stable_*`)가 오면 force-sim **초기 좌표**로 써서(없으면 기존 `fibonacciStarPosition` 폴백) 재진입이 빠르고 일관되게 — 단, 좌표 권위는 여전히 클라 force-sim(헌법3). 마지막 공고화 이후 처음 여는 세션이면 은은한 "밤사이 우주가 한 번 정리됐어요" 노트를 1회 표시(매 프레임 React state 재계산 금지 — §3.2, ref/플래그로 1회). 요지 형태는 `StarField`가 `form_seed_delta`를 per-instance attribute로 받아 `buildStarForm`/seed에 반영(필요 시 `entities/star`에 gist form step 추가).

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 야간 티커가 동작할 때 THE SYSTEM SHALL 활성 사용자별로 `kind='consolidate'` 잡을 enqueue하고, 워커가 `Claim(ctx, KindConsolidate)`로 그 잡을 한 건씩 처리한다. _(헌법6 — 스트리밍 아님, 배치)_
1.2 WHEN `consolidate` 잡을 처리할 때 THE SYSTEM SHALL 4패스를 **순서대로**(재안정화 → 재분배 → 요지 → 가지치기) 실행하고, 끝나면 성단 흥분성을 리셋한 뒤 잡을 `done`으로 완료한다.
2.1 WHEN 전체 재안정화 패스가 끝날 때 THE SYSTEM SHALL force-sim과 동치인 수렴 좌표를 `memories.stable_*`에 캐시하되, 이 좌표를 **권위가 아닌 캐시**로만 두고 클라가 여전히 force-sim으로 좌표를 산출하게 한다. _(헌법3)_
2.2 WHEN 재분배 패스가 동작할 때 THE SYSTEM SHALL 각 별을 호스트 성단 centroid로 `lerp≈0.6` 당기고 도식 적합 별엔 `schemaBonus`를 추가 적용한다.
3.1 GIVEN `created_at`이 `gistAgeDays`(30일)보다 오래되고 `last_recalled_at`이 `gistRecallCutoffDays`보다 과거인 별 WHEN 요지 패스가 동작할 때 THE SYSTEM SHALL `form_seed_delta`를 **단조 증가**(GREATEST — 후퇴 금지)로 `gistFormSimplify`만큼 단순화한다.
3.2 WHEN 요지 패스가 별을 단순화할 때 THE SYSTEM SHALL 그 변경을 `evolution_history`에 `trigger='nightly_gist'`로 **append-only** 기록하고, 원본 `records` 행은 한 글자도 바꾸지 않는다. _(헌법1)_
4.1 GIVEN `weight < weakEdgeThreshold`(0.2) 이고 `last_activated_at`이 `weakEdgeIdleDays`보다 과거인 선 WHEN 가지치기 패스가 동작할 때 THE SYSTEM SHALL 그 선의 `weight`를 바닥 부근으로 낮춰 **어둡게**만 하고, 행은 우주에 남겨 클릭 가능하게 둔다.
4.2 WHEN 가지치기를 수행할 때 THE SYSTEM SHALL `memory_links`/`memories`에 대해 어떤 `DELETE`도 실행하지 않는다(코드 grep으로 `DELETE FROM memory_links`·`DELETE FROM memories` 0건). _(헌법2)_
5.1 WHEN 4패스가 끝날 때 THE SYSTEM SHALL 성단 흥분성 `e(c,t)`(22)를 리셋해 다음 낮의 경쟁적 할당이 0에서 시작되게 한다(24h 회전).
5.2 IF 어떤 패스가 실패하면 THEN THE SYSTEM SHALL 기존 `failWithBackoff`로 잡을 재시도/보존(`failed`)하고, 별·선·원본을 삭제하거나 손상시키지 않는다. _(헌법1·2)_
6.1 WHEN 사용자가 공고화 이후 처음 우주를 열 때 THE SYSTEM SHALL 갱신된 형태(요지)·어두워진 약한 선·안정 좌표가 반영된 "살짝 달라진" 장면을 보여주고, 1회 "밤사이 우주가 한 번 정리됐어요" 노트를 표시한다(매 프레임 재계산 금지 — §3.2).
7.1 WHEN 랜딩 `NightlyConsolidationCard`를 볼 때 THE SYSTEM SHALL 가지치기가 선을 *삭제*가 아니라 *밝기 바닥*으로 정리함과 24h 흥분성 리셋의 생물학적 근거를 카피로 명시한다.
8.1 GIVEN 데모 모드 WHEN 사용자가 "밤 보내기"를 누를 때 THE SYSTEM SHALL 라이브 데모 우주에 4단계 애니메이션을 돌리고, 정돈됐지만 별·선이 그대로 남은 장면으로 끝낸다(삭제 0 — 헌법2).
