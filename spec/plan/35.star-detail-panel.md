# 35. star detail panel

> Star detail panel — the panel opened when a node is clicked on the universe canvas: it shows a star's **meta** (shape ·
> emotion color · brightness · 작성일 · 강도 · current forgetting state) and, for an episodic star, its **forgotten
> current memory text — free, no time advance** ([R1][T3]), with three actions — **회고하기** (→ recall flow), **변천사
> 보기** (the provenance / 변천사 view), **원본 일기 보기** (→ diary reader). It reads the selection state already owned
> by the universe-canvas machine and consumes the `GetProvenance` read.
> Scope: FE | Status: planning
>
> Layer P (presentation), Epic C's first presentation unit. Home: `widgets/star-detail` composing `features/*`, over the
> running `widgets/universe-canvas` ([23.universe-canvas](23.universe-canvas.md)) whose XState machine already holds the
> selected-node id. It fills the **panel + provenance view**; it leaves the **recall behavior** to
> [33.recall-usecase](33.recall-usecase.md) (via [36.recall-flow-ui](36.recall-flow-ui.md)), the **provenance read** to
> [46.provenance-export](46.provenance-export.md), and the **diary reader** to [47.diary-reader-page](47.diary-reader-page.md).

## Purpose

The universe canvas ([23.universe-canvas](23.universe-canvas.md)) can select a node — clicking a star stores its id in
the canvas machine — but nothing yet _opens_ on that selection. This unit is the **read surface of a single star**: the
first place a diarist can look _at_ one memory rather than _across_ the universe, and the launchpad for every act that
touches it (recall, seeing its history, reading its origin diary).

It realizes PRD [§5.6](../PRD.md) [D1]: click a star → a **meta-only** detail (no time advance, no reinforcement), and,
for an episodic star, its **forgotten current memory text shown for free** ([R1][G1]). This is the deliberate emotional
center of the product — the user sees their _own faded memory_ (words progressively erased by forgetting) at no cost,
and only _re-remembering_ (회고 = reconsolidation) or _reading a gist_ costs 별가루. The panel makes that free/paid line
visible: viewing is free and the clock does not move ([T3][I10]); the three buttons are where the paid / navigational
acts begin. It builds the **panel UI + the provenance view + the button seams only** — the recall behavior, the
provenance read, and the diary reader are their own plans.

## Scope / Non-goals

**In scope**

- **`widgets/star-detail`** — a panel / side-sheet opened when the canvas selection becomes non-nil (a node clicked,
  [D1][U3]). It mounts **over** the running `widgets/universe-canvas` (it does not remount the renderer, [23]/[14]); it
  reads the selected node id from the canvas machine (§3.2) and closes on deselect / a close affordance. Composes the
  features below and owns the panel's own control-state.
- **`features/star-meta`** — the meta block ([D1]). For a **big star** (episodic memory): its unique **shape** (seed
  render), **emotion color**, **brightness**, **작성일** (`created_universe_time`), **강도** (effective strength), and
  **current forgetting state** (decay stage / word-loss extent). For a **small star** (neuron): **info only, no
  emotion** ([D1] — neurons carry no mood). All meta values are **read-time derived** from stored facts on `GetUniverse`
  by `packages/memory-logic` (plan 18 / Epic D), never re-derived here.
- **`features/current-memory-text`** — for an episodic star, the **forgotten current memory text**, shown **free**
  ([R1][G1]): the `current_text` rendered with the decay-stage word erasure ([F1] — "나는 오늘 친구랑 xxxxx …"). The
  text is a stored fact ([R8a]); this feature _renders_ it faded. Viewing it **advances no clock and reinforces
  nothing** ([R1][T3][I10]) — it is a pure read.
- **`features/star-provenance`** — the **변천사 (provenance) view** ([R8a][D1]): the star's **shape** plus a
  **time-ordered list of stage texts**, each with a **kind** label (생성 / 요지화 / 재공고화) and a **source** (원본 /
  시스템 / 사용자). It **consumes** the `GetProvenance` read (owned by [46.provenance-export](46.provenance-export.md)),
  whose response = a synthesized **created/original baseline** (derived at read from creation facts, CC5) **+** the
  append-only `memory_provenance` entries. Distortion is **not** separately flagged — the user discovers it by expanding
  ([R8a]).
- **The three action buttons** ([D1]) as **seams**, wired but delegating:
  - **회고하기** → opens the recall flow ([36.recall-flow-ui](36.recall-flow-ui.md)); this panel emits the "recall
    requested" event for the selected episodic star and does **not** itself recall / reconsolidate / spend 별가루.
  - **변천사 보기** → toggles the in-panel provenance view above.
  - **원본 일기 보기** → navigates to the diary reader ([47.diary-reader-page](47.diary-reader-page.md)) at the star's
    origin diary; this panel emits the navigation intent (the diary id) and does not build the reader.
- **The panel's control-state as one XState machine** — `model/star-detail.machine.ts` (§3.2): `closed → meta →
provenance` (+ the `meta`/`provenance` states are the two views), driven off the canvas selection; the selected id and
  the fetched provenance live in Zustand/Query, ids/phase only in machine context.
- **UI copy via i18n** ([08.i18n-seam](08.i18n-seam.md)) — no hardcoded strings; a **literary, restrained Korean** voice
  (the button labels 회고하기 / 변천사 보기 / 원본 일기 보기 are the poetic UI vocabulary, PRD [§3.1](../PRD.md)).
  **web + mobile parity** — the same widget + features; `*.native.tsx` siblings **only** for the panel / sheet primitives
  that genuinely differ (§3.5).

**Non-goals**

- **No recall / reconsolidation behavior.** 회고하기, the sync-consent modal, prediction-error compare, rewrite,
  the reconsolidation result, and the 별가루 spend are [33.recall-usecase](33.recall-usecase.md) /
  [36.recall-flow-ui](36.recall-flow-ui.md). This panel only **opens** that flow for the selected star.
- **No provenance read or write.** The `GetProvenance` RPC, the `memory_provenance` table, the created-baseline synthesis
  rule, and CSV/MD export ([W6]) are owned by [32.reconsolidation-rules](32.reconsolidation-rules.md) (the table +
  first write) and [46.provenance-export](46.provenance-export.md) (the read + export). This panel **consumes** the read.
- **No gist-star viewing.** Reading a **gist (요지) star's** text ([R8][G1], a paid read) is
  [34.view-semantic-usecase](34.view-semantic-usecase.md) + its own view surface; this panel is the **episodic** star
  detail (a clicked gist star routes to that flow, not this panel's free-text block).
- **No diary reader / split view / "이 일기로 태어난 별 보기" jump.** [D2][D3][D5] are
  [47.diary-reader-page](47.diary-reader-page.md); this panel only **navigates** there via 원본 일기 보기.
- **No forgetting math or visuals engine.** The decay-stage / brightness / word-erasure computation is
  [37.forgetting-decay](37.forgetting-decay.md) and the on-star dimming/word-loss is
  [39.forgetting-visuals](39.forgetting-visuals.md); this panel **reads** the effective values and renders the faded
  text. Until Epic D ships, effective values equal the Epic-A stubs (full brightness, `current_text` unerased) — the
  panel is fully buildable now and the fade appears when D lands (CC1).
- **No economy.** Whether an action costs 별가루 is Epic F ([43.stardust-ledger](43.stardust-ledger.md) /
  [45.stardust-ui](45.stardust-ui.md)); free viewing here is free by requirement ([R1][G1]), and the paid acts price
  themselves inside the recall / gist-view flows, not here. No price constant appears in this panel (CC2/CC3).
- **No selection engine.** Node hover/click → selected id is [23.universe-canvas](23.universe-canvas.md)'s canvas
  machine; this unit **reads** the selection, it does not own picking or the camera focus.
- No new tuning scalars hardcoded in FE (see §Policy).

## Grounding

- Constitution ([I1]–[I11]): [00.overview](00.overview.md) §3, owned by [PRD.md](../PRD.md) §7. This unit is bound by:
  **[I2]** the `Diary` is immutable — the provenance view shows _representation_ history (current text / shape stage
  texts), never a mutable original; the objective record is reached only via 원본 일기 보기 (the reader / export). **[I3]**
  emotion never drives position/connection — the meta block _displays_ emotion (color) and shows **no** position /
  connection / placement control; a neuron's meta shows **no** emotion at all. **[I8]** viewing resets nothing and
  restores no content — the free current-text read is pure (accessibility recovery happens only on **recall**, not on
  open). **[I10]** time is monotonic and **viewing does not advance it** ([T3]) — opening the panel and reading the free

  emotion / strength / position, only reads them.

## Design

### Placement (FSD, §3.1)

```
apps/web/src/widgets/star-detail/
├── ui/            StarDetailPanel (panel/side-sheet host, meta+text+provenance) + *.native.tsx sibling for the sheet
├── model/         star-detail.machine.ts (XState) · panel view store · the "resolve selection → domain star" selector
└── index.ts       public API (the panel; it reads the canvas selection, needs no explicit open affordance)

apps/web/src/features/
├── star-meta/            ui (meta block: shape · color · brightness · 작성일 · 강도 · forgetting state; neuron = info-only)
├── current-memory-text/  ui (episodic forgotten current_text, rendered faded per decay stage — free)
└── star-provenance/      ui (변천사 list: kind/source-labelled stage texts) · api (GetProvenance call + proto↔domain map)
```

- `widgets/star-detail` composes the three `features/*` (each = one user surface, §3.1), owns the **panel machine** and
  the view store, and imports the domain-mirror entities (`entities/episodic-memory`, `entities/neuron`) from [23] only
  to _read_ the selected star's stored facts + neuron kind. It never imports `three` or a visual entity (§3.4): the
  star's rendered **shape** in the panel is drawn through the same asset-source body seam plan 24 owns (a small preview),
  not re-implemented here.
- **entities mirror already exists** (CC brief / [23]): this unit adds **no** new domain-mirror slice — it reuses
  `entities/episodic-memory` and `entities/neuron`. `entities/episodic-memory/@x/` cross-imports already exist for
  sibling slices; the panel needs none beyond the public mirror API.
- **web + mobile parity (§3.5):** the same widget + three features on both apps; `model` and `api` shared verbatim
  (pure, no DOM/native). Only `ui` forks where a primitive differs — the **panel / side-sheet host** (web side-panel /
  DOM ↔ RN bottom-sheet) — via a `*.native.tsx` sibling. web = open-beta scope, mobile = MVP.

### Selection → panel (reads the canvas machine, §3.2)

The universe-canvas machine ([23]) already stores the selected node id (`kind:id`) in its context and reads it in
`useFrame` for the highlight. This panel **subscribes** to that selection (via the shared `@cosimosi/universe` machine's
`getSnapshot()` / a selector, not per-frame React state) and opens when the selection becomes non-nil, closes on
deselect. **Decided (2026-07):** selection ownership stays in the **canvas machine** (single source of the selected id);
`star-detail.machine.ts` holds only the _panel's own_ view phase (`meta` vs `provenance`) + a close flag, deriving
open/closed from the canvas selection. This avoids two machines racing to own "which star is selected".

The selected id resolves to a domain star through a pure selector in `model/`: it looks the id up in the
`entities/episodic-memory` / `entities/neuron` stores ([23]) and yields either an **episodic** star (full meta + free
text + buttons) or a **neuron** star (info-only meta, no emotion, no episodic buttons). A **gist (요지) star** selection
routes to [34.view-semantic-usecase](34.view-semantic-usecase.md)'s paid-view surface rather than this panel's free-text
block (CC7 — gist texts are pregenerated; the paid gate is Epic F).

### Meta block ([D1]) — display-only, read-time derived

`features/star-meta` renders, for an **episodic** star:

- **shape** — the seed-driven star body preview ([V5]; drawn via the asset-source seam, not re-implemented);
- **emotion color** — from `moodColor(mood)` (the single palette entry, plan 17 / CC10) — display only ([I3]);
- **brightness** — the effective brightness (read-time, [V2]);
- **작성일** — `created_universe_time` (the star's diary date);
- **강도** — the effective strength ([V3][R3]);
- **current forgetting state** — the decay stage / word-loss extent ([F1]).

For a **neuron** (small star): **info only, no emotion** ([D1]) — name / type / connectivity, no mood color, no episodic
buttons. **All** numeric/derived meta comes from `packages/memory-logic`'s read-time functions (plan 18 stubs now;
[37.forgetting-decay](37.forgetting-decay.md)'s `EffectiveBrightness` / decay-stage and effective strength once Epic D
ships, CC1) over the stored facts on `GetUniverse`. This panel **re-derives no math** — it displays what the shared
functions return, so web/mobile show identical values (the same golden-parity guarantee [23] relies on).

### Free forgotten current text ([R1][G1][F1]) — the emotional center

`features/current-memory-text` shows the episodic star's `current_text` (a stored fact, [R8a]) rendered with the
**decay-stage word erasure** ([F1]): as the star forgets, random words are shown removed. **Decided (2026-07):** the
panel renders the erasure from the **decay-stage forgetting-text** that Epic D stores per stage ([R8a]/`decay_stages`);
until Epic D ships, the Epic-A stub returns the text unerased, so this feature shows the full `current_text` now and the
progressive fade appears when [37.forgetting-decay](37.forgetting-decay.md) / [39.forgetting-visuals](39.forgetting-visuals.md)
land (CC1). This read is **free and pure**: opening the panel and reading this text **advance no clock** ([T3][I10]),
**spend no 별가루** ([R1][G1]), and **restore no content / reset no accessibility** ([I8]) — accessibility recovery is a
consequence of **recall** (33), never of viewing.

### 변천사 (provenance) view ([R8a][D1]) — consumes GetProvenance (CC5)

`features/star-provenance` renders the **shape + a time-ordered stage-text list**, each entry labelled by **kind**
(생성 / 요지화 / 재공고화) and **source** (원본 / 시스템 / 사용자). It **consumes** the `GetProvenance` read owned by
[46.provenance-export](46.provenance-export.md):

```
GetProvenance(GetProvenanceRequest{ episodic_memory_id }) → GetProvenanceResponse {
  repeated ProvenanceEntry entries    // time-ordered; each { kind, source, text, universe_time }
}
```

Per **CC5**: the first entry is the **created/original baseline synthesized at read** from the memory's creation facts
(`created_universe_time`, the initial account) — there is **no** Epic-A backfill; the remaining entries are the
append-only `memory_provenance` rows (`reconsolidated/user` appended by recall [33], `semanticized/system` appended as
gist stages rise [40]). This panel does not write provenance and does not know the synthesis rule — it renders the
ordered list the read returns. Distortion is **not** separately flagged; expanding the list is how the user discovers
change ([R8a]). The list is fetched on entering the `provenance` view (a Query read, cached), not on every panel open.

### The three buttons ([D1]) — seams, not behavior

| Button             | This panel does                                                                 | Owned by                                                                                                                                                                                          |
| ------------------ | ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **회고하기**       | emits "recall requested" for the selected episodic star → opens the recall flow | [36.recall-flow-ui](36.recall-flow-ui.md) (UI) over [33.recall-usecase](33.recall-usecase.md) (behavior); the sync-consent modal is [31.time-acceleration-ui](31.time-acceleration-ui.md) ([R1a]) |
| **변천사 보기**    | toggles the in-panel `provenance` view (fetches `GetProvenance`)                | read owned by [46.provenance-export](46.provenance-export.md)                                                                                                                                     |
| **원본 일기 보기** | emits the origin-diary navigation intent (diary id)                             | [47.diary-reader-page](47.diary-reader-page.md) ([D5] — the second path into the diary reader)                                                                                                    |

The 회고하기 button is shown only for an **episodic** star (a gist star's read is paid-view 34; a neuron has no recall).
This panel neither prices nor performs any action behind these buttons — it hands off (CC2: no price constant here).

### Panel control-state (XState, §3.2)

`model/star-detail.machine.ts` holds the panel's **own** control-state; the selected id (from the canvas machine) and
the fetched provenance live in Zustand/Query (data), phase in the machine:

| State        | Meaning                                        | Exits                                                                                                                 |
| ------------ | ---------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| `closed`     | no selection / panel dismissed                 | canvas selection non-nil → `meta`                                                                                     |
| `meta`       | meta block + free current text + buttons shown | `SHOW_PROVENANCE` → `provenance`; `RECALL` → (emit; stays); `OPEN_DIARY` → (emit; stays); deselect/`CLOSE` → `closed` |
| `provenance` | 변천사 list shown (GetProvenance fetched)      | `BACK` → `meta`; deselect/`CLOSE` → `closed`                                                                          |

The panel derives open/closed from the canvas selection (single source, above); `RECALL` / `OPEN_DIARY` are emitted
intents consumed by the composing page (recall flow / router) and leave the panel state intact. No per-frame scene work
here — the panel is DOM/RN UI over the canvas, reading selection via a snapshot/selector (§3.2).

### Golden-parity stance

This is a **presentation** unit; it holds **no** FE↔BE shared math. Every derived value it shows (brightness, strength,
decay stage, word-erased text) comes from `packages/memory-logic` — the golden-parity pure functions owned by
[18.synapse-plasticity](18.synapse-plasticity.md) / [37.forgetting-decay](37.forgetting-decay.md). The panel is a pure
**reader** of those outputs, so web and mobile render identical meta without this unit re-implementing any formula.

### Copy (i18n) — literary, restrained Korean

All strings resolve through the i18n seam ([08.i18n-seam](08.i18n-seam.md)); none hardcoded (`lint:raw-strings`). The
button labels (회고하기 / 변천사 보기 / 원본 일기 보기) and the kind/source labels (생성·요지화·재공고화 / 원본·시스템·사용자)
use the restrained literary voice (PRD [§3.1](../PRD.md)) — no decoration, no translation-ese. Web + mobile draw the
**same** message keys (parity).

## Acceptance Criteria

1. Clicking a node on the universe canvas (selection non-nil in the canvas machine, [23]) opens `widgets/star-detail`
   **over** the running canvas without remounting the renderer; deselecting / a close affordance dismisses it ([D1][U3]).
2. For an **episodic** (big) star the meta block shows **shape · emotion color · brightness · 작성일 · 강도 · current
   forgetting state**; for a **neuron** (small) star it shows **info only, with no emotion** ([D1][I3]); all derived
   values come from `packages/memory-logic` read-time functions, not re-derived in the panel.
3. An episodic star's **forgotten current memory text** is shown (rendered faded per decay stage where Epic D provides
   it; full text until then), and viewing it is **free** — it **advances no clock**, **spends no 별가루**, and **restores
   no content** ([R1][G1][F1][T3][I8][I10]).
4. A **변천사 보기** view renders the star's shape + a **time-ordered stage-text list** with **kind** (생성/요지화/재공고화)
   and **source** (원본/시스템/사용자) labels, populated by the `GetProvenance` read (created baseline synthesized at read
   - `memory_provenance` entries, CC5); distortion is **not** separately flagged ([R8a][D1]).
5. **회고하기** (shown only for an episodic star) **opens the recall flow** ([36.recall-flow-ui](36.recall-flow-ui.md)) —
   this panel emits the request and does **not** itself recall, reconsolidate, price, or spend 별가루 ([R1][R6]).
6. **원본 일기 보기** emits the navigation intent to the diary reader at the star's origin diary
   ([47.diary-reader-page](47.diary-reader-page.md), [D5]); this panel does not build the reader.
7. A **gist (요지) star** selection routes to the paid gist-view surface ([34.view-semantic-usecase](34.view-semantic-usecase.md),
   [R8]), not this panel's free-text block; a clicked **neuron** shows the info-only meta with no episodic buttons.
8. The panel exposes **no** override of a star's emotion / strength / position and **no** placement or meaning-layer
   control — it only reads ([I3][I11]); the provenance view shows representation history only and never a mutable
   original ([I2]).
9. The panel's **control-state** is one XState machine (`closed → meta → provenance`); the selected id stays owned by the
   **canvas** machine (single source) and the fetched provenance lives in Zustand/Query, not in machine context (§3.2).
10. **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in a literary /
    restrained Korean voice.
11. **web + mobile parity:** the same widget + three features run on both apps, sharing `model` / `api` verbatim;
    `*.native.tsx` siblings exist **only** for the panel / sheet primitives (§3.5).

## Policy / Values Impact

- **policy/ux** — the rules this panel makes true land in a new `policy/ux/star-detail.md` once implemented (written at
  `/implement-job`; **do not** pre-create policy files):
  - **Viewing is free and moves no clock.** Meta + the forgotten current memory text are always free to view; opening the
    panel advances no universe time and spends no 별가루 ([R1][G1][T3][I10]).
  - **The free text is the faded memory, not the original.** The panel shows the decay-erased `current_text`; the
    immutable original is reached only via 원본 일기 보기 (the reader / export) ([R8a][I2][D4]).
  - **Neurons show information, not emotion.** A small star's detail carries no mood color and no episodic actions ([D1]).
  - **변천사 shows representation history, distortion unflagged.** The kind/source-labelled stage list is the star's
    change history; distortion is discovered by expanding, never separately announced ([R8a]).
  - **Actions hand off; the panel neither prices nor performs them.** 회고하기 → recall flow (paid, reconsolidation),
    변천사 → the read, 원본 일기 보기 → the reader — no price or behavior lives in the panel ([D1][R6][G1]).
- **tech/** — the as-built FE rules (the `widgets/star-detail` composition over three features; the
