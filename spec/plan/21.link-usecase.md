# 21. link usecase

> Link usecase вҖ” the core `internal/memory` use-case that grows the synapse graph after a launch: same-memory Hebbian
> wiring, shared-neuron connection, and a narrow temporal-proximity bonus, all as neuronвҶ”neuron edges only.
> Scope: BE | Status: built (job 27)

## Purpose

A launched diary produces an `EpisodicMemory` and its co-activated `Neuron`s (plan 20's `Encode`/`PersistEncoded`).
On its own that is a set of isolated stars. cosimosi's constellations вҖ” the _relational_ structure a user later
navigates by meaning and context rather than by date вҖ” grow from **synapses between neurons**, not from any
memoryвҶ”memory relationship. `Link` is the use-case that lays and strengthens those synapses.

It exists so that the moment a user launches stars, the neurons that fired together in one memory are wired together
(Hebbian [L1]); two memories that reuse a neuron are thereby already connected through it (shared neuron [L2]); and
neurons that co-fire in memories written close in time are strengthened a little more (temporal-proximity bonus [L4]).
Every edge is neuronвҶ”neuron ([L6][I6]) вҖ” there is never a memoryвҶ”memory edge ([I4][L7]). The "nearby-in-time memories
cluster together" behavior is not implemented as a force between memories; it **emerges** from shared neurons + near
seed + the temporal bonus, and is realised as layout by the force-sim (plan 19), never stored.

`Link` reuses the pure plasticity functions from [18.synapse-plasticity](18.synapse-plasticity.md) for all strength
math and persists through the atomic weighted upsert defined by [16.memory-aggregate-schema](16.memory-aggregate-schema.md).
It computes _no_ similarity/embedding linking ([L3]) and owns no rendering.

## Scope / Non-goals

**In scope**

- The `Link` use-case in `internal/memory`, invoked by `PersistEncoded` (plan 20) at launch, that grows the neuron
  synapse graph for the launched memory (and its relation to the user's existing neurons).
- **Same-memory Hebbian wiring [L1]:** within one launched `EpisodicMemory`, every pair of co-activated neurons gets a
  synapse (all pairwise edges).
- **Shared-neuron connection [L2]:** when a launched memory reuses a neuron that already belongs to earlier memories,
  the shared neuron **is** the link вҖ” no extra edge is created; the connection is the neuron's existing
  `neuron_activations` membership, which `GetUniverse` (plan 23) already exposes.
- **Temporal-proximity bonus [L4]:** neurons co-activated across memories whose diary dates fall inside a narrow window
  (`synapse.temporal_window_days`) receive extra strengthening; time attaches to the **memory's date**, never to a
  neuron ([E6]).
- Strength math delegated to plan 18 pure functions: tiered `InitialStrength` (same-memory > shared-neuron > temporal,
  [L10]) for a first-time pair; saturating `Potentiate` toward the `1.0` cap ([L8][L9]) for a repeat co-firing.
- Persistence via the atomic weighted upsert into `synapses`
  (`INSERT вҖҰ ON CONFLICT (user_id, neuron_a_id, neuron_b_id) DO UPDATE`, ARCHITECTURE В§2.6): incrementing
  `co_activation_count`, advancing `last_activated_universe_time`, and writing the new stored base `strength`. Canonical
  undirected ordering `neuron_a_id < neuron_b_id`; per-user scoped.
- The consumer-owned `SynapseStore` port surface `Link` needs (declared where consumed), backed by the queries plan 16
  provisions.

**Non-goals**

- **No similarity / embedding linking [L3].** `Link` never creates an edge from vector nearness. Embeddings assist
  neuron _identity_ (dedup [E10], plan 20) and the latent-field, never connection.
- **No memoryвҶ”memory edge or memoryвҶ”memory attraction [I4][L7][E9].** There is no memory-to-memory table, force, or
  synapse. Inter-memory proximity is emergent, not modelled.
- **No stored positions, constellations, or coordinates [I5].** Constellations are an emergent read-side relationship
  computed by the force-sim (plan 19) from shared neurons + synapse strength вҖ” not a domain source-of-truth and not a
  stored table (ARCHITECTURE В§2.9 #3).
- **No plasticity math redefinition.** `Link` calls plan 18's pure functions; it does not re-derive initial values,
  potentiation curves, the cap, or read-time effective strength.
- **No read-time effective strength / decay.** `Link` writes the stored _base_ only; time-decayed **effective** strength
  is a read-time pure-function derivation ([L5]), owned by plan 18 / `packages/memory-logic` and consumed by plan 23.
- **No LLM extraction, embedding, semanticization, or persistence of the aggregate root** вҖ” those are plan 20 (`Encode`/
  `PersistEncoded`) and plan 22 (worker). `Link` receives already-persisted neurons/activations and only writes synapses.
- **No async queue mechanics.** Whether cross-memory refinement runs in the `link` job is plan 22's concern; this plan
  fixes the _rules_ and picks launch-time execution for correctness (see Design).
- **No universe-clock authority.** Universe time is derived from the launched memory's `created_universe_time` (= its
  `diary_date`); the per-user clock is Epic B (seam only).

## Design

### Where `Link` lives and when it runs

`Link` is application behavior published by the core `internal/memory` context (ARCHITECTURE В§2.2). Per В§2.3 it starts
in the context package alongside `Encode`/`PersistEncoded` (plan 20) and moves to `memory/app` only if the context
later earns the `{domain,app,pg,rpc}` split. It is a pure orchestration over the domain plasticity functions plus one
persistence port вҖ” it holds no IO of its own.

**`Link` runs synchronously as the last step of `PersistEncoded` at launch, in the same transaction that persisted the
`Diary`, `EpisodicMemory`, `Neuron`s, and `NeuronActivation`s.** The launch path is:

```
LaunchStars (rpc) вҶ’ PersistEncoded:
    persist Diary + EpisodicMemory + Neuron (dedup applied) + NeuronActivation
    вҶ’ Link(user, launchedMemory, itsNeurons)        // synapses, same tx
    вҶ’ enqueue `embed` + `semanticize` jobs          // async (plan 22)
    вҶ’ return memory ids                             // optimistic (В§2.8)
```

Rationale for launch-time (not deferred to the `link` job): the synapse graph is a **correctness** fact of the write,
not a slow external call. Running it inside the launch transaction keeps the atomic upserts idempotent under the write's
own retry, guarantees `GetUniverse` (plan 23) reads a consistent graph on the very next read, and needs no LLM/embedding
call. The async `link` job (plan 22) is reserved for _heavier cross-memory refinement_ if it is ever needed (for
example, re-scanning a large existing graph for temporal bonuses beyond the launched memory's own neurons). The brief
(В§7) allows same-memory Hebbian at launch and cross-memory shared/temporal linking in the `link` job; **decided
(2026-07): for Epic A this plan runs all three rules at launch, synchronously inside the `PersistEncoded`
transaction** вҖ” the async `link` job stays a reserved seam, promoted only if a launch is observed to touch too much of
the graph to stay in the request path.

### Inputs

`Link(ctx, scope platform.UserScope, launched EpisodicMemory, neurons []Neuron)` receives, from `PersistEncoded`:

- the authenticated user (every read and write is scoped to `user_id`, ARCHITECTURE В§4);
- the just-persisted `EpisodicMemory` вҖ” needed for its `created_universe_time` (= `diary_date`), which drives the
  temporal-window comparison and the `last_activated_universe_time` written on each edge;
- the launched memory's co-activated neurons (after dedup [E10]) вҖ” the set the Hebbian rule wires and the set whose
  prior co-activations the temporal bonus examines.

`Link` does not receive or read embeddings, proto types, or sqlc rows вҖ” only domain values and the `SynapseStore`/
`NeuronStore` ports (ARCHITECTURE В§2.4).

### Rule 1 вҖ” same-memory Hebbian wiring [L1]

For the launched `EpisodicMemory`, every **unordered pair** of its co-activated neurons `(a, b)`, `a вү  b`, gets a
synapse. With _n_ co-activated neurons this is the full `nВ·(nвҲ’1)/2` set of pairwise edges. For each pair, `Link`:

1. normalises the pair to canonical undirected order `neuron_a_id < neuron_b_id`;
2. classifies the signal as **same-memory** (this pair co-fired within one memory) and asks plan 18 for the tier;
3. upserts the edge (see _Strength & persistence_).

Same-memory is the **strongest** initial tier ([L10]) and always applies when both neurons are members of the launched
memory, regardless of whether either neuron is new or reused.

### Rule 2 вҖ” shared-neuron connection [L2] (the shared neuron _is_ the link)

When the launched memory reuses a `Neuron` that already belongs to earlier memories (dedup [E10] made it the _same_
neuron, not a new one), those memories are **thereby connected** вҖ” through the shared neuron. `Link` creates **no extra
edge** for this: the connection is the neuron's `neuron_activations` membership across memories, which the read side
(`GetUniverse`, plan 23) already returns and the force-sim (plan 19) already uses to pull those memories toward the
shared neuron's centroid.

**As-built (job 27):** at launch, sharing is expressed _purely through activation membership_ вҖ” `Link` writes **no extra
edge** for it (A2). The launch-time edges `Link` creates are the same-memory pairs (Rule 1); two memories that reuse a
neuron are joined by the `neuron_activations` membership `GetUniverse` already returns and the force-sim already pulls
toward the shared neuron's centroid ([L2][L7]). The **shared-neuron initial tier** (`synapse.initial_shared_neuron`,
same-memory > **shared-neuron** > temporal, [L10]) remains a plan-18 plasticity value, reserved for the heavier
cross-memory refinement the async `link` job would perform (В§2.8) вҖ” it is not synthesised at launch, precisely so the
"no extra edge for sharing" rule ([I4][I6][L7]) holds absolutely.

### Rule 3 вҖ” temporal-proximity bonus [L4] (narrow window, on the memory's date)

Time is **not a neuron** ([E6]): it attaches to a memory's date. The bonus rewards a pair that _fires together again_
close in time вҖ” PRD [L4] "к°Җк№Ңмҡҙ мӢңк°Ғм—җ мғқкёҙ мқјнҷ”кё°м–өм—җм„ң **н•Ёк»ҳ мјңм§„** лүҙлҹ°лҒјлҰ¬" (neurons that **co-fire** in
episodic-memories-created-at-close-times), i.e. co-activation **plus** memory-time proximity.

**As-built (job 27):** for each launched co-firing of a pair, `Link` reads the pair's co-activation memberships
(`neuron_activations` вҶ’ `episodic_memories` dates) and applies `synapse.temporal_bonus` when **this launch's** co-firing
memory is within `synapse.temporal_window_days` of _another_ memory that also co-activated the pair. The comparison is
anchored on the current launch (not on any two historical co-firings), so a distant repeat of an old near-in-time
cluster earns nothing. A pair's _first-ever_ co-firing (a single co-activation memory) is same-memory only and takes no
bonus; the bonus arises on the second and later in-window co-firings, added on top of that launch's `Potentiate`. The
comparison is on the memories' dates; nothing is written onto a neuron.

The window is **narrow** by design ([L4]): the bonus models competitive/excitability-based allocation (recently active
neurons are likelier to be reused by a near-in-time next memory вҖ” [E7], Han 2007), not a broad "same era" effect. The
temporal-only tier (a pair connected _only_ by temporal proximity, with neither same-memory co-firing nor sharing) is
the **weakest** initial tier ([L10]). Outside the window, no temporal bonus is applied вҖ” plain Hebbian/shared rules
still govern the edge.

The bonus never introduces a memoryвҶ”memory edge and never stores time on a neuron. It only adjusts a neuronвҶ”neuron
synapse's stored base strength, keyed on the memories' dates.

### Rule 4 вҖ” no similarity / embedding linking [L3]

`Link` never creates or strengthens an edge from embedding nearness. Semantic similarity guides neuron **identity**
(conservative dedup [E10], done in `Encode`, plan 20) and later the latent-field assist вҖ” never _connection_. The
constellation graph grows only from neuron reuse ([L2]) + Hebbian co-firing ([L1]) + the temporal bonus ([L4]). This is
an explicit guard against constellation collapse (over-merging by similarity), and it means `Link` reads no
`embeddings` rows.

### Rule 5 вҖ” emergent constellations, never stored [I5][L7]

"Nearby-in-time memories cluster" is **not** implemented as a memory-attraction force ([L7][E9]). It emerges from three
stored/seed facts, combined only at read/layout time:

- **shared neurons** ([L2]) вҖ” memories that reuse a neuron are pulled toward its centroid by the force-sim;
- **near seed** ([E7]) вҖ” a new neuron's seed starts near reused / recently-active neurons (a presentation seed, final
  position emergent);
- **temporal bonus** ([L4]) вҖ” near-in-time co-firing makes the neuronвҶ”neuron synapses stronger, so the force-sim's
  link force draws them tighter.

`Link` owns only the third (and contributes the neuronвҶ”neuron edges the first two also rely on). Constellations
themselves are an **emergent relationship**: not a domain source-of-truth, not a memoryвҶ”memory table, computed by the
force-sim (plan 19) from shared neurons + synapse strength (ARCHITECTURE В§2.5, В§2.9 #3, [I4]). `Link` writes no
coordinate and no cluster.

### Strength & persistence (plan 18 pure fns + atomic upsert)

`Link` decides _which_ pairs and _which tier/bonus_ apply; **all numeric strength math is plan 18's pure functions**
([18.synapse-plasticity](18.synapse-plasticity.md)), so the same math is reproducible client-side (golden-parity):

- **First-time pair** вҶ’ `InitialStrength(signal)` returns the tier's low starting base
  (`synapse.initial_same_memory` > `synapse.initial_shared_neuron` > `synapse.initial_temporal`, [L10]), plus
  `synapse.temporal_bonus` when Rule 3's window applies.
- **Repeat co-firing (pair already has a row)** вҶ’ `Potentiate(current, вҖҰ)` increases the stored base **in proportion to
  the remaining headroom `(1 вҲ’ current)`** ([L8]), saturating smoothly toward the single cap `synapse.strength_cap = 1.0`
  ([L9]); the temporal bonus, when in window, feeds the potentiation. Frequent pairs converge to the cap without
  runaway.

Persistence is one **atomic weighted upsert** per edge into `synapses`, using the query plan 16 provisions
(ARCHITECTURE В§2.6):

```sql
INSERT INTO synapses (id, user_id, neuron_a_id, neuron_b_id,
                      strength, co_activation_count, last_activated_universe_time, created_at)
VALUES (@id, @user_id, @a, @b, @strength, 1, @universe_time, now())
ON CONFLICT (user_id, neuron_a_id, neuron_b_id) DO UPDATE
SET strength                     = @strength,          -- Potentiate(existing.strength, вҖҰ), computed by Link (plan 18)
    co_activation_count          = synapses.co_activation_count + 1,
    last_activated_universe_time = GREATEST(synapses.last_activated_universe_time, @universe_time);
```

- The `ON CONFLICT` target is the `UNIQUE (user_id, neuron_a_id, neuron_b_id)` constraint (plan 16). Because `Link`
  canonicalises `a_id < b_id` before writing, an unordered neuron pair has exactly one row per user вҖ” the invariant that
  makes the upsert **idempotent** ([L8]): re-running `Link` for the same launch (write retry) advances the same row
  rather than creating duplicates.
- On the **repeat** path, `Link` first reads the existing stored base (through `SynapseStore`) to compute
  `Potentiate(existing, вҖҰ)` in the domain, then writes the result вҖ” the stored `strength` is always the **base**, never
  the read-time effective value ([L5]). The base is read then written **absolutely**, while `co_activation_count`
  increments **relatively** (plan 16's baseline upsert); this is correct under the product's per-user **serial** launch
  model (the diary-monotonic universe clock, Epic B), matching `PersistEncoded`'s other optimistic reads (В§2.8) вҖ”
  concurrent same-user launches of the same pair are not a supported path.
- `last_activated_universe_time` and `co_activation_count` are the _accumulated history_ facts ([L5]) the read-time
  `EffectiveSynapseStrength` decay consumes; `Link` writes them, plan 23 reads them, plan 18 derives the decay.
- Every statement is scoped by the authenticated `user_id` (ARCHITECTURE В§4); `Link` never touches another user's edges.

### The `SynapseStore` port

`Link` consumes a small consumer-owned port declared in `internal/memory` (ARCHITECTURE В§2.4, brief В§4), implemented by
`memory/pg` (the only package that touches sqlc/pgx). The surface `Link` needs:

- `SynapseStrengths` вҖ” read, in **one query**, the stored base strength of every synapse whose both endpoints are among
  the launched neurons (the `Potentiate` repeat-path bases, keyed by canonical pair). Batched rather than a read per
  pair, so a launch touching _n_ neurons costs one round-trip, not `nВ·(nвҲ’1)/2`.
- the atomic upsert above (`UpsertSynapse`), taking the canonical pair, the computed base strength, and the universe
  time;
- `CoActivations` вҖ” read, for the launched memory's neurons, their memory memberships with each memory's
  `created_universe_time`, so `Link` times the temporal bonus on the memories' dates (backed by `neuron_activations`
  joined to `episodic_memories`, plan 16).

The port is embedded in `LaunchTx`, so every `Link` read/write runs on `PersistEncoded`'s transaction. No sqlc row
escapes `memory/pg`; the port speaks domain values only. `Link` is a **required** `Service` dependency (`NewService`
rejects a nil `Linker`), so no composition root can launch memories without growing the synapse graph.

## Acceptance Criteria

1. After `PersistEncoded` launches a memory with _n_ co-activated neurons, **all** `nВ·(nвҲ’1)/2` unordered neuron pairs of
   that memory have a synapse row (same-memory Hebbian wiring) [L1].
2. When a launched memory reuses an existing neuron, **no extra edge** is created to represent the sharing; the two
   memories are connected purely through the shared neuron's `neuron_activations` membership, and `GetUniverse` returns
   that shared membership вҖ” the shared neuron _is_ the link [L2].
3. Two neurons that **co-fire again** in a memory within `synapse.temporal_window_days` of a prior co-firing receive the
   `synapse.temporal_bonus` ([L4] "н•Ёк»ҳ мјңм§„"); a co-firing outside that window вҖ” or a first, lone co-firing вҖ” applies no
   bonus. The window is anchored on this launch's co-firing and compared on the memories' dates; no time value is
   written onto any neuron [L4][E6].
4. `Link` creates or strengthens **no** synapse from embedding/semantic similarity, and reads no `embeddings` rows;
   the graph grows only from neuron reuse + Hebbian co-firing + the temporal bonus [L3].
5. Every synapse `Link` writes is between two neurons; there is no memoryвҶ”memory edge, table, force, or attraction
   anywhere in the use-case, and inter-memory proximity remains emergent [I4][I6][L6][L7][E9].
6. A first-time pair is initialised via plan 18's `InitialStrength` at the correct tier, with
   `synapse.initial_same_memory` > `synapse.initial_shared_neuron` > `synapse.initial_temporal` [L10]; the tier of a
   pair is determined by its strongest applicable signal. At launch every wired pair co-fires within a memory, so its
   strongest signal вҖ” and its tier вҖ” is same-memory; the shared-neuron and temporal initial tiers remain plan-18 values
   reserved for the async `link` job's cross-memory refinement (В§2.8).
7. A repeat co-firing strengthens the stored base via plan 18's `Potentiate`, increasing in proportion to
   `(1 вҲ’ current)` and never exceeding `synapse.strength_cap = 1.0`, converging smoothly toward the cap [L8][L9].
8. Persistence uses `INSERT вҖҰ ON CONFLICT (user_id, neuron_a_id, neuron_b_id) DO UPDATE`, storing the canonical
   `neuron_a_id < neuron_b_id`, incrementing `co_activation_count`, advancing `last_activated_universe_time`, and writing
   the stored **base** strength (not the read-time effective value); re-running `Link` for the same launch is idempotent
   (advances the same row, creates no duplicate) [L5][L8].
9. Constellations are never stored: `Link` writes no coordinate, cluster, or memoryвҶ”memory relationship; clustering is
   computed by the force-sim (plan 19) from shared neurons + synapse strength at layout time [I5][L7] (ARCHITECTURE В§2.9 #3).
10. Every read and write is scoped to the authenticated `user_id`; `Link` never reads or writes another user's neurons,
    activations, or synapses [U1] (ARCHITECTURE В§4).
11. All strength values come from plan 18's pure functions (no strength arithmetic is re-implemented in `Link`), so the
    same inputs produce the same stored base under the client-side golden-parity mirror.
12. `Link` performs no LLM/embedding call and no hard delete of any neuron or synapse; it only inserts or strengthens
    edges [I1].
