# 38. forgetting cost

> Forgetting cost — the pure read-time relationship that turns a memory's forgetting depth into an **accessibility
> multiplier**: a deeper-decayed episodic star is harder to reach ([F4]), and recall normalizes it back ([F5]). It
> exposes an accessibility / cost-**weight** (a derived, golden-parity read value), not a Twinkle price — the price
> lives in Epic F. IO-free math, mirrored TS ↔ Go, consumed by F's cost curve and the recall spend gate.
> Scope: BE | Status: planning
>
> Layer U — the second unit of Epic D, built after [37.forgetting-decay](37.forgetting-decay.md) (D). Home: the pure
> domain of `internal/memory` (Go) + `packages/memory-logic` (TS), alongside the effective-value functions seated in
> [18.synapse-plasticity](18.synapse-plasticity.md). It reads 37's decay depth and writes no table, defines no RPC —
> it is a derived value on the existing read / gating path. It fills the [F4]/[F5] _accessibility_ seam and leaves the
> _pricing_ seam to Epic F ([43.stardust-ledger](43.stardust-ledger.md) / [44.earn-spend-usecase](44.earn-spend-usecase.md)).

## Purpose

Forgetting in cosimosi is not erasure — it is a memory becoming _hard to reach_ (접근성 감소, reversible; Ryan &
Frankland 2022). [37.forgetting-decay](37.forgetting-decay.md) makes that visible: an un-recalled episodic star dims and
its decay stage advances, deleting words. This unit fixes the _other half_ of that idea — the part the user feels as
effort. **The more a star has decayed, the more it costs to pull it back** ([F4]); and **recall recovers it wholly**, so
the cost normalizes the moment you retrieve it ([F5]). Together they are the "use it or lose it" loop given a price
signal: a memory you keep visiting stays cheap; one you let fade becomes expensive to summon, but never lost.

This unit owns exactly the **pure relationship** between forgetting depth and accessibility — one monotone function from
"how far gone is this star" to "how much harder is it to reach", plus the rule that recall resets the input to zero
(because recall resets decay in 37). It is deliberately small and deliberately math-only: it produces an
**accessibility / cost-weight** in a normalized range, a _derived read-time value_ like `EffectiveBrightness`, and
nothing else. It is **not** the Twinkle economy. It states no price, holds no balance, spends nothing. Epic F's cost
curve consumes this weight to compute a Twinkle price, and Epic F's spend gate deducts it (CC2/CC3). Because both the
client (to preview "회고하기" cost) and the server (to gate the spend) must agree on the weight, the function lives once
in Go and once in TS, pinned equal by golden-parity fixtures — the same discipline as
[18.synapse-plasticity](18.synapse-plasticity.md).

## Scope / Non-goals

**In scope**

- A pure, IO-free domain function `AccessibilityCostWeight(decayDepth) → weight` in `internal/memory` (Go) +
  `packages/memory-logic` (TS): **monotone non-decreasing** in decay depth — a deeper-decayed episodic star yields a
  higher weight ([F4]) — over a normalized, floored/capped range.
- The **input contract**: `decayDepth` is 37's read-time forgetting depth for the episodic memory (the continuous decay
  progress / stage fraction 37 computes from `effectiveElapsed`, arousal- and connection-modulated). This unit does
  **not** recompute decay; it consumes 37's depth signal.
- The **recovery-on-recall** rule ([F5]): because recall resets decay in 37 (brightness reset + decay-stage recovery +
  gist-timer reset, all together — [R2][F5][C6a]), `decayDepth → 0` after recall, and therefore
  `AccessibilityCostWeight` returns its **floor** (fully accessible / cheapest). Recovery is expressed _through the input_
  — no separate recovery function, no stored accessibility field.
- The **golden-parity** contract (TS ↔ Go identical outputs within tolerance) so the client's cost preview and the
  server's spend gate never disagree, plus the fixtures that pin the monotonicity, the floor at `decayDepth = 0`, and the
  cap.
- The **seam to Epic F** stated precisely: this weight is what F's recall cost curve consumes to derive a Twinkle price;
  and it is the same weight the recall spend gate ([44.earn-spend-usecase](44.earn-spend-usecase.md)) reads to size its
  charge. Enumerating the `forgetting.*` value keys this function reads (floor/cap/shape coefficient).

**Non-goals**

- **No Twinkle price, ledger, balance, or spend.** The recall cost _amount_ in Twinkle, the two-tier balance
  (basic/additional), spend order, and every price constant are Epic F ([43.stardust-ledger](43.stardust-ledger.md) /
  [44.earn-spend-usecase](44.earn-spend-usecase.md), CC3). This unit emits a normalized weight; F prices it. **No price
  constant appears here** (CC3).
- **No decay computation.** `effectiveElapsed`, the decay stages, the word-deletion, and the arousal/connection-strength
  modulation of the decay _rate_ are [37.forgetting-decay](37.forgetting-decay.md) ([F1][F2][F3][F6][F7][F9]). This unit
  takes 37's depth as input; it does not derive it.
- **No gist-view cost.** Gist-star _viewing_ is cheaper the deeper the gist ([R8][G4]) — a _different_ axis (gist-depth =
  `semantic_stage`, owned by [40.semanticization](40.semanticization.md)) priced by F's cost curve, not this unit
  ([34.view-semantic-usecase](34.view-semantic-usecase.md) is the read). Forgetting cost is the _episodic recall_ axis
  only.
- **No recall use-case, sync, or reconsolidation.** `Recall`/`Reinforce`/`Reconsolidate`, the sync-confirm gate, and the
  actual anchor writes that reset decay are [33.recall-usecase](33.recall-usecase.md) (Epic C). This unit only defines
  the accessibility-vs-depth relationship recall _recovers through_.
- **No neighbor forgetting.** The signed `forgetting_offset_days` that recall nudges on _neighbors_ ([R5], CC4) is
  [32.reconsolidation-rules](32.reconsolidation-rules.md)'s write and [37.forgetting-decay](37.forgetting-decay.md)'s
  read (it feeds `effectiveElapsed` → depth). It reaches this unit only through 37's depth input; this unit adds no
  offset math.
- **No RPC or DB column.** No new proto message and no new `episodic_memories` column — accessibility is a _derived_
  read-time value ([I5]), like `EffectiveBrightness`, not a stored one. (Whether the derived weight is surfaced on the
  `GetUniverse` DTO or computed at the gate is a placement note below, not a schema change.)
- **No UI.** The "회고하기" cost display and the dim/word-loss visuals are
  [35.star-detail-panel](35.star-detail-panel.md) / [45.stardust-ui](45.stardust-ui.md) /
  [39.forgetting-visuals](39.forgetting-visuals.md).

## Design

### Placement (domain-first Go)

`AccessibilityCostWeight` is a **pure domain function**, not a use-case, so per [ARCHITECTURE.md](../ARCHITECTURE.md)
§2.5 it lives at the innermost pure layer of the core-domain `internal/memory` context — the same home as
[18.synapse-plasticity](18.synapse-plasticity.md)'s `EffectiveSynapseStrength` / `EffectiveStrength` /
`EffectiveBrightness`, and next to 37's decay-depth function it reads. The registry lists this unit at layer U because
it is _consumed on_ the use-case / gating path, but it introduces no use-case, RPC, or repository of its own — it is a
scalar transform. It depends on nothing but its `decayDepth` argument and the generated `forgetting.*` constants: no
clock, no repository, no proto, no SDK, no randomness. It never sees a sqlc row or a proto DTO
([ARCHITECTURE.md](../ARCHITECTURE.md) §2.4 #4).

**Decided (2026-07):** this unit adds **no new `internal/memory` sub-package and no RPC**. It is a function seated
beside 37's decay math (both born in Epic D, both read at the same read); the two ship as one small pure-math surface.
The use-case wiring (which caller passes the weight where) is Epic C recall + Epic F pricing, not here.

### `AccessibilityCostWeight` — the pure relationship ([F4])

Signature (canonical; representation casing may differ per language):

```
AccessibilityCostWeight(decayDepth float) → weight float
```

- **Input** `decayDepth ∈ [0, 1]` — the episodic memory's read-time forgetting depth from
  [37.forgetting-decay](37.forgetting-decay.md): `0` = freshly recalled / not decayed (fully accessible), `1` = maximally
  decayed (the silent-engram floor, [F2]). 37 derives it from `effectiveElapsed` (which already folds neighbor
  `forgetting_offset_days` [R5]/CC4 and the arousal/connection-strength modulation [F6][F7]); this unit takes it as
  given. **Decided (2026-07):** the contract is the _normalized decay depth_ (a `[0,1]` fraction), not a raw stage index
  — so the two axes speak the same normalized language and the weight shape is independent of how many decay stages 37
  defines (stage count is a 37 concern, and array lengths are derived, not a value here).
- **Output** `weight ∈ [forgetting.cost_weight_floor, forgetting.cost_weight_cap]` — a normalized accessibility /
  cost-weight, **not** a Twinkle price. `cost_weight_floor` is the fully-accessible baseline (a freshly recalled star is
  still recalled at the base weight — recall is _never free_, [G1]; the floor is the cheap end, not zero-cost); the
  weight **caps** so a maximally decayed star is _expensive but bounded_ — never unreachable ([I1][F2]).

Behavioral contract (the acceptance criteria pin the _behaviour_, not the curve shape):

- **`AccessibilityCostWeight(0) = forgetting.cost_weight_floor`** — a fully accessible (freshly recalled) star returns
  the floor. This is the [F5] recovery endpoint expressed through the input.
- **Monotone non-decreasing** in `decayDepth`: deeper decay never _lowers_ the weight, and strictly increases it across
  the interior — deeper decay ⇒ harder to reach ⇒ costlier ([F4]).
- **Capped**: `AccessibilityCostWeight(1) = forgetting.cost_weight_cap`, and the result stays within
  `[floor, cap]` for all inputs — the silent engram is expensive, never infinite ([I1][F2]).

The **shape** between floor and cap stays in **code** (a formula, excluded from `values.yaml` per the config rule).
**Decided (2026-07):** the initial shape is a monotone convex ease — cost rises gently for the first stages of decay and
steepens as a memory approaches the floor of silence, matching [F4]'s "많이 흐려진 기억일수록 떠올리기 어렵다" — governed
by a single tunable curvature coefficient `forgetting.cost_weight_curve` (its number set at implement). The convexity is
a formula choice, replaceable by another monotone-non-decreasing curve without changing the contract; only the floor,
cap, and curvature coefficient are values.

### Recovery on recall ([F5]) — expressed through the input, no separate function

[F5] is that recall recovers a star **wholly and together**: brightness reset, decay-stage recovery (deleted words come
back), and gist-timer reset happen as one act ([R2][F5][C6a]). In this model **recovery is not a second function** — it
is the _consequence_ of recall resetting decay:

1. [33.recall-usecase](33.recall-usecase.md) (Epic C) writes the recall anchors (`last_recalled_universe_time =
universe_time`, `recall_count += 1`, `semanticize_timer_reset_at = universe_time`) — CC1.
2. [37.forgetting-decay](37.forgetting-decay.md) then computes `effectiveElapsed ≈ 0` at the next read, so
   `decayDepth → 0`.
3. `AccessibilityCostWeight(0) = cost_weight_floor` — the star is fully accessible / cheapest again.

So this unit's [F5] obligation is exactly the **floor guarantee at `decayDepth = 0`**: recall, by zeroing decay,
normalizes accessibility. There is no stored `accessibility` to reset and no recovery path independent of decay — which
is why recovery is _reversible_ and _whole_, and why "use it or lose it" holds without a second source of truth. **The
recalled star itself recovers wholly ([F5]); the neighbor ± offset (CC4) affects only _neighbors_ and reaches this unit
only as a shifted `decayDepth` for those neighbors** — the recalled star's own depth goes to zero.

### The seam to Epic F — weight in, price out (CC2 / CC3)

This unit ends at the **weight**. Epic F picks it up:

- **[43.stardust-ledger](43.stardust-ledger.md)** owns the **recall cost curve** — `recallTwinkleCost(weight) → Twinkle`
  — mapping this normalized weight to a Twinkle price, alongside the gist-view curve (keyed by gist-depth, a _different_
  axis, CC3). Every price constant (the Twinkle floor/cap of a recall, the curve coefficients) is a `stardust.*`
  (or `twinkle.*`) value in Epic F, **not here**.
- **[44.earn-spend-usecase](44.earn-spend-usecase.md)** owns the **spend gate** (the consumer-owned `SpendGate` port from
  CC2): on recall it derives `weight = AccessibilityCostWeight(decayDepth)`, prices it through the ledger curve, and
  checks-and-spends against the two-tier balance (basic → additional). Epics C–E bind a **no-op allow-all** gate so the
  recall loop works costlessly until F (CC2); F binds the real one. The gate **re-derives** the weight server-side rather
  than trusting a client number ([ARCHITECTURE.md](../ARCHITECTURE.md) §2.9 #8) — the client computes the _same_ weight
  (golden-parity) only to _preview_ the cost in "회고하기".

Stated as the invariant seam: **D computes accessibility; F prices it; C spends it.** No price crosses into `internal/memory`'s
forgetting math, and no accessibility formula crosses into `internal/twinkle`.

### Golden-parity (TS ↔ Go)

Per [ARCHITECTURE.md](../ARCHITECTURE.md) §2.5, purity exists so the two runtimes agree. The Go implementation in
`internal/memory` and the TS mirror in `packages/memory-logic` are pinned equal by **shared golden fixtures** (the same
harness [18.synapse-plasticity](18.synapse-plasticity.md) established): `(function, input) → expected output` rows plus a
fixed tolerance. The fixture set covers `decayDepth = 0` (asserting the floor), `decayDepth = 1` (asserting the cap),
a sweep across the interior asserting monotonic non-decrease and the convex steepening, and out-of-range inputs clamped
into `[floor, cap]`. Both sides read the _same_ generated `forgetting.*` constants (`pnpm gen:values` emits Go and TS),
so a tuning change cannot desync the client cost-preview from the server spend-gate. **Decided (2026-07):** this unit is
**golden-parity** (matching [18.synapse-plasticity](18.synapse-plasticity.md) / [37.forgetting-decay](37.forgetting-decay.md)),
because the client must preview a recall cost that equals what the server will charge; it is _not_ LLM-gated (unlike
[32.reconsolidation-rules](32.reconsolidation-rules.md)'s prediction-error predicate), so exact parity is both required
and achievable.

### Purity & placement (ARCHITECTURE §2.4, §2.5)

- The function lives in the pure domain of `internal/memory` (Go) and `packages/memory-logic` (TS) — no `pg`, `rpc`,
  SDK, clock, or framework import. It is the innermost layer; nothing depends inward on it except by calling.
- It takes and returns a domain scalar, not a proto DTO or sqlc row (§2.4 anti-corruption boundary).
- No formula is a value: only the floor, cap, and curvature coefficient go to `spec/values.yaml`; the curve shape and
  the clamps are code.

## Acceptance Criteria

1. `AccessibilityCostWeight(decayDepth)` is a **pure, IO-free** function (deterministic; no clock, DB, transport, SDK, or
   randomness) in both `internal/memory` (Go) and `packages/memory-logic` (TS), reading only its argument and the
   generated `forgetting.*` constants. ([ARCHITECTURE.md](../ARCHITECTURE.md) §2.5)
2. It is **monotone non-decreasing** in `decayDepth` over `[0, 1]` and strictly increasing across the interior — a
   deeper-decayed episodic star yields a weight ≥ (and, in the interior, >) that of a shallower one: deeper decay ⇒
   harder to reach ⇒ costlier. ([F4])
3. `AccessibilityCostWeight(0) = forgetting.cost_weight_floor` — a fully accessible (freshly recalled) star returns the
   **floor** (cheapest, but non-zero: recall is never free [G1]); this is the [F5] recovery endpoint expressed through
   the decay input. ([F5][G1])
4. The weight is **capped**: `AccessibilityCostWeight(1) = forgetting.cost_weight_cap` and every result stays within
   `[forgetting.cost_weight_floor, forgetting.cost_weight_cap]` — a maximally decayed (silent-engram) star is expensive
   but **bounded**, never unreachable/infinite. ([I1][F2])
5. **Recovery on recall is expressed through the input, not a separate function or a stored field**: because recall
   resets decay in [37.forgetting-decay](37.forgetting-decay.md) (`decayDepth → 0`), the weight returns to its floor;
   there is no `accessibility` column and no independent recovery path. ([F5][I5][I8])
6. The function's **input is 37's normalized decay depth** (`[0,1]`), not a raw stage index; this unit does **not**
   recompute decay, does not read `effectiveElapsed`/stages/arousal/connection directly, and adds no neighbor-offset math
   (the offset reaches it only via 37's depth). ([F1][F6][F7][R5])
7. The unit exposes a **weight, not a price**: it references no Twinkle constant, ledger, or balance; the recall Twinkle
   price is derived from this weight by [43.stardust-ledger](43.stardust-ledger.md)'s cost curve and spent by
   [44.earn-spend-usecase](44.earn-spend-usecase.md)'s gate (which **re-derives** the weight server-side, §2.9 #8). D
   computes accessibility, F prices it, C spends it. ([G4][G1], CC2/CC3)
8. It is the **episodic-recall axis only** — it does **not** compute the gist-view cost (that is the gist-depth axis,
   `semantic_stage`, priced by F and read by [34.view-semantic-usecase](34.view-semantic-usecase.md)). ([R8][G4])
9. The Go function and the TS mirror produce **identical outputs** for a shared golden-fixture table within a fixed
   tolerance (TS ↔ Go golden-parity), covering the floor at `0`, the cap at `1`, interior monotonicity, and out-of-range
   clamping; both read the same generated `forgetting.*` constants. ([ARCHITECTURE.md](../ARCHITECTURE.md) §2.5)
10. No formula, curve shape, or clamp is stored in `spec/values.yaml`; only the floor, cap, and curvature coefficient are
    values (the convex shape stays code). ([00.overview](00.overview.md) config rule)
