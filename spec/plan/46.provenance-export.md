# 46. provenance export

> Provenance export — the backend read/export use-case for a star's _변천사_ (variant history) and the reassurance
> that live/retained records can be taken out: a `GetProvenance` RPC returning the time-ordered variant history (kind +
> source), the "created" baseline synthesized at read, and an `Export` RPC that emits retained original `Diary` records +
> memories as downloadable CSV/MD ("일기가 사라질까 걱정 마세요"). The `Diary` is the objective record and is never mutated.
> Scope: BE | Status: planning
>
> Layer U (use-cases) — the read/export unit of Epic G. Home: `internal/memory` use-cases (`GetProvenance`, `Export`)
> over the `memory_provenance` append-only table. This unit is the table's **owner-of-record for reading and export**;
> the table itself is introduced by [32.reconsolidation-rules](32.reconsolidation-rules.md) and _written_ by
> [33.recall-usecase](33.recall-usecase.md) (reconsolidation) and [40.semanticization](40.semanticization.md) (gist
> stages). It fills the read/export seam left open by those plans; it leaves the _panel that renders 변천사_
> ([35.star-detail-panel](35.star-detail-panel.md)) and the _diary reader page_ ([47.diary-reader-page](47.diary-reader-page.md)) to Layer P.

## Purpose

cosimosi's central emotional contract is a contrast: the **star is a trace** — it dims, loses words, and is rewritten
each time you recall it — while the live/retained **`Diary` is the objective record**, immutable by the system and by
reconsolidation ([D4][I2]). For that contrast to be honest, two things must be legible on the backend. First, a user must
be able to _see how a memory changed_ — the 변천사, a time-ordered list of every variant the star has held, each tagged
with **what kind of change** made it (created / semanticized / reconsolidated) and **who authored** it (original / system
/ user) ([R8a][D1]).
Second, a user must be able to _walk away with retained data_ — the original diaries and their memories that have not
been hard-deleted by the user's own full-delete flow, exported to a plain, portable file (CSV/MD), so that the fear "what
if my diary disappears" is answered before it is felt ([W6]).

This unit owns both reads on the backend. It exposes the append-only variant history for a single star (`GetProvenance`)
and the whole-account export (`Export`), and it fixes the one property both depend on: **the `Diary` is never a mutable
row in provenance** — the original lives in the `Diary` table and in the export while retained, and the variant history
records only the _representation's_ evolution ([I2][R7]). The rendering of the 변천사 view and the download UX belong to Layer P;
this plan specifies the read contracts, the synthesized baseline, and the export shape, not the screens.

## Scope / Non-goals

**In scope**

- **`GetProvenance(episodic_memory_id)`** — a per-user read returning one star's variant history in **universe-time
  order**: the synthesized **created/original baseline** first, then every appended `memory_provenance` row (kind +
  source + text + universe-time). Read-only; advances no clock ([T3]); costs no stardust (metadata-tier, free [R1][G1]).
- **The synthesized "created" baseline** — the first entry is **derived at read** from the memory's creation facts
  (`created_universe_time` + the original account = the immutable `Diary` body via `diary_id`), **not** stored as a row
  and **not** backfilled into Epic-A data ([CC5]). It is present for every retained star, even one that has never been
  reconsolidated or semanticized.
- **`Export(format)`** — a per-user whole-account export of the **retained immutable original `Diary` records + their
  episodic memories** as **CSV or MD** ([W6]), returned as a downloadable payload. The export is the canonical "your
  retained data can leave with you" artifact; it draws from the objective record, never from the mutable representation.
- **The read/export ownership of `memory_provenance`** — this unit reads and exports the append-only history; it does
  not define the table (that is [32.reconsolidation-rules](32.reconsolidation-rules.md)) and does not append rows.
- **Both RPCs on `memory.v1.MemoryService`** (Connect, unary) with thin handlers (proto↔domain map + call), all read
  policy in the use-case/domain.
- **Per-user scoping** on every provenance read and every export query (§4); `GetProvenance` refuses a star that is not
  the caller's.

**Non-goals**

- **No `memory_provenance` table or migration** — the table (`id, user_id, episodic_memory_id, kind, source, text,
universe_time, created_at`, append-only) is introduced by [32.reconsolidation-rules](32.reconsolidation-rules.md)
  ([CC5]). This unit consumes a read repository against it.
- **No provenance _writes_** — the `reconsolidated/user` row is appended by [33.recall-usecase](33.recall-usecase.md)
  on prediction-error rewrite ([R6]); the `semanticized/system` rows are appended by
  [40.semanticization](40.semanticization.md) as gist stages rise ([C6]). This unit never writes a row.
- **No 변천사 rendering or panel** — the star-detail panel that shows the history list (star shape + per-stage texts,
  no separate distortion notice) is [35.star-detail-panel](35.star-detail-panel.md) ([R8a][D1]); it _calls_
  `GetProvenance`.
- **No diary reader page / split visualization / "이 일기로 태어난 별 보기" jump** — that is
  [47.diary-reader-page](47.diary-reader-page.md) ([D2][D3][D5]). This unit provides the export, not the reader UI.
- **No recall, reconsolidation, forgetting, or semanticization math** — those are Epics C/D/E. This unit reads the
  facts they produce.
- **No deletion / letting-go / restore** — soft-delete + sealing are Epic H ([48.deletion-rules](48.deletion-rules.md)/
  [49.release-usecase](49.release-usecase.md)); this unit's export honors the exclusion (see below) but does not build
  it. (The reassurance _copy_ on the deletion flow is [50.deletion-ui](50.deletion-ui.md); the export it points at is
  this unit.)
- **No stardust cost** — reading provenance and exporting are free (metadata/archive tier, [G1][G4]); the economy is
  Epic F and is not touched here.

## Design

### Placement (domain-first Go)

`GetProvenance` and `Export` are application read use-cases in the core-domain `internal/memory` context, alongside the
other memory reads ([CC8]: `internal/memory` owns provenance + diary reads). They start in the context package
(ARCHITECTURE §2.3). The append-only `memory_provenance` table and the `diaries`/`episodic_memories` reads are reached
only through **consumer-owned ports** declared here — a `ProvenanceReader` and an `ExportReader` — bound to
[16.memory-aggregate-schema](16.memory-aggregate-schema.md)'s `memory/pg` concrete at the composition root (§2.4).
Dependency direction: `memory/rpc → GetProvenance/Export (use-case) → pure domain (baseline synthesis, ordering,
CSV/MD serialization rules)`, with the readers reached only through interfaces (§2.9#4/#6). Handlers stay thin
(§2.7/§2.9#7). **Decided (2026-07):** provenance + export live in `internal/memory` (not a separate context) because
both are pure reads over the memory aggregate's own tables; there is no cross-context port here.

### `memory_provenance` — the read model (defined by [32], read here)

The append-only table, per [CC5], is:

```sql
-- memory_provenance — append-only-while-retained variant history. NO UPDATE; DELETE only via Epic H user full-delete sweep.
-- Owned/created by 32.reconsolidation-rules; written by 33 (reconsolidation) + 40 (semanticization); READ + EXPORTED here.
CREATE TABLE memory_provenance (
    id                 TEXT PRIMARY KEY,
    user_id            TEXT NOT NULL,
    episodic_memory_id TEXT NOT NULL REFERENCES episodic_memories(id) ON DELETE CASCADE,
    kind               TEXT NOT NULL,   -- 'created' | 'semanticized' | 'reconsolidated'  (see note below)
    source             TEXT NOT NULL,   -- 'original' | 'system' | 'user'
    text               TEXT NOT NULL,   -- the variant text at this event
    universe_time      DATE NOT NULL,   -- when it happened in universe-time [T]
    created_at         TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

The `kind`/`source` enums and this DDL are **content/schema owned by [32]** (excluded from `values.yaml` per the config
rule); this plan consumes them. **Decided (2026-07):** the `created` kind is a _valid value of the column_ but the
`created/original` baseline is **not persisted** — no `created` row is written at launch and no Epic-A backfill occurs
([CC5]). The baseline is **synthesized at read** (below), so a star that has only ever been created and never
reconsolidated/semanticized still shows a one-entry 변천사. This keeps the write side of [32]/[33]/[40] free of a
launch-time provenance write and keeps the `Diary` out of the mutable history entirely ([I2]).

### `GetProvenance` — the 변천사 read ([R8a][D1])

`GetProvenance(scope, episodic_memory_id) → ProvenanceHistory`:

1. **Load the memory** (per-user scoped). If it is not the caller's, return `not found` (no cross-user leak, §4). A
   **soft-deleted** memory ([X2], Epic H) is treated per the caller's context — the panel does not open a deleted star,
   so the default read filters `deleted_at IS NULL`; the restore surface (Epic H) is out of scope here.
2. **Synthesize the created/original baseline** from the memory's creation facts: `kind = created`, `source = original`,
   `universe_time = created_universe_time`, and `text` = the memory's **original account**. **Decided (2026-07):** the
   baseline `text` is read from the **immutable `Diary` body** via the memory's `diary_id` (the objective record,
   [I2][D4]) — **not** from `current_text` (which reconsolidation overwrites) and **not** from any `memory_provenance`
   row (32/33 append the representation text _at_ each event — the new `current_text` — never a pre-image, so no prior
   narrative is stored). Sourcing the baseline from the Diary keeps the "created/original" entry honest and needs no
   Epic-A backfill. This entry is **always first**.
3. **Load the appended rows** for the memory from `memory_provenance` via the `ProvenanceReader` port, per-user scoped.
4. **Merge + order by `universe_time` ascending** (ties broken by `created_at`), baseline first. The result is the
   time-ordered variant history: `[created/original, …, semanticized/system, …, reconsolidated/user, …]` interleaved by
   when they happened.
5. **Return** each entry as `{kind, source, text, universe_time}` — the shape the 변천사 panel renders. **No separate
   "this was distorted" flag** is emitted ([R8a]: the distortion is _found by expanding_, not announced).

`GetProvenance` is **read-only**: it advances no clock ([T3]), writes nothing, and is free (metadata tier [R1][G1]) —
it carries **no** `SpendGate` call. It is GET-eligible / `NO_SIDE_EFFECTS` (§2.7).

### `Export` — the CSV/MD download ([W6][D4])

`Export(scope, format) → ExportPayload`. The export is the **immutable original**: the user's `Diary` records and the
episodic memories launched from them, serialized so the account can be carried away in a plain, portable file.

- **Source = the objective record.** The export reads `diaries` (append-only [I2]) and their `episodic_memories`,
  per-user scoped. **Decided (2026-07):** the export's primary content is the `Diary` body (the objective, un-distorted
  record — this is what "일기가 사라질까 걱정 마세요" promises); each diary lists the memories launched from it (name,
  mood, `created_universe_time`) so the structure is legible, but the _diary body is verbatim and un-mutated_. The
  mutable representation (`current_text`, decay/gist stage texts) is **not** the export's authoritative content — the
  point of the export is the record, not the trace ([D4]).
- **`format` ∈ {CSV, MD}** ([W6]):
  - **CSV** — one row per diary (or per diary×memory, flattened): `diary_date, diary_body, memory_name, mood,
created_universe_time`. A machine-portable table.
  - **MD** — a human-readable document: each diary as a dated section with its verbatim body, then its memories as a
    short list. The reading-first form.
    The exact column set / MD layout is **content owned by this plan's implementation** (a serialization format, not a
    tuning scalar — excluded from `values.yaml`).
- **Exclusion honored, retained data preserved until the sweep ([I1][X3]).** The live export excludes **soft-deleted** memories
  (`deleted_at IS NULL`) and their sealed neurons — consistent with the universe read's existing exclusion
  ([16.memory-aggregate-schema](16.memory-aggregate-schema.md)) — so a "letting-go" ([X4]) is respected in what is
  handed out. The rows are not deleted by this read/export unit; Epic H owns soft-delete, restore, and the post-window
  user-delete sweep. A retained `Diary` whose star was never launched (past-dated [T1][I10]) is still exported — the diary
  was saved, and the export is the archive that proves it.
- **Delivery.** `Export` returns the serialized bytes + a content-type/filename hint; streaming vs. one payload and any
  size cap are an implementation detail behind this contract. **Decided (2026-07):** v1 exports the _whole account_ in
  one call (no date-range/selection params) — the reassurance is "you can take _all_ of it"; range/selective export is a
  later refinement behind this same RPC.

### RPC contract (`memory.v1.MemoryService`, Connect, unary — §2.7)

Both RPCs are added to the existing `memory.v1.MemoryService`; handlers in `memory/rpc` map proto↔domain and call the
use-case only (§2.9#7).

```proto
// 변천사 read — the star's variant history (created/semanticized/reconsolidated × original/system/user), time-ordered.
rpc GetProvenance(GetProvenanceRequest) returns (GetProvenanceResponse);   // read-only, NO_SIDE_EFFECTS [T3], free [R1]
// whole-account export of the immutable Diary + memories as a downloadable file [W6][D4].
rpc Export(ExportRequest) returns (ExportResponse);                        // read-only, NO_SIDE_EFFECTS

message GetProvenanceRequest  { string episodic_memory_id = 1; }
message GetProvenanceResponse { repeated ProvenanceEntry entries = 1; }    // baseline first, then universe-time order
message ProvenanceEntry       { string kind = 1;                          // created|semanticized|reconsolidated [R8a]
                                string source = 2;                        // original|system|user [R8a]
                                string text = 3;
                                string universe_time = 4; }               // DATE (universe-time [T])

message ExportRequest  { ExportFormat format = 1; }                       // CSV | MD [W6]
enum ExportFormat      { EXPORT_FORMAT_UNSPECIFIED = 0; EXPORT_FORMAT_CSV = 1; EXPORT_FORMAT_MD = 2; }
message ExportResponse { bytes content = 1; string content_type = 2; string filename = 3; }
```

Both are **read-only** and carry no side effect: neither advances the clock ([T3]), writes a row, nor spends stardust.
The provenance read returns a **history that never contains the `Diary` as a mutable entry** — the `original` source
here labels the _synthesized baseline_ (the memory's first narrative), while the objective `Diary` itself is reached
only through `Export` and the reader ([D2]). This is the structural expression of [I2]: the immutable record is not
inside the variant history.

### Thin handler / dependency rule

The `memory/rpc` handlers map the proto DTOs to domain inputs and call `GetProvenance` / `Export`; the baseline
synthesis, the ordering, the exclusion filter, and the CSV/MD serialization all live in the use-case/domain (§2.9#7).
No proto message or sqlc row is seen by the use-case or the pure domain (§2.9#4); the `ProvenanceReader` and
`ExportReader` are consumer-owned interfaces (§2.9#6). Every provenance read and export query is scoped to the
authenticated user (§4).

## Acceptance Criteria

1. `GetProvenance(episodic_memory_id)` returns the star's variant history **time-ordered by `universe_time`** with the
   synthesized **created/original baseline first**, each entry carrying `{kind, source, text, universe_time}` where
   `kind ∈ {created, semanticized, reconsolidated}` and `source ∈ {original, system, user}`. ([R8a][D1])
2. The **created/original baseline is synthesized at read** from the memory's creation facts — it is **not** a stored
   `memory_provenance` row and is **never** backfilled into Epic-A data; a star that has never been
   reconsolidated/semanticized still returns a one-entry history. ([CC5][R8a])
3. `GetProvenance` emits **no separate distortion flag** — the history is a plain time-ordered list; distortion is found
   by reading it, not announced. ([R8a])
4. `Export(format)` returns the user's **retained immutable original `Diary` records + their memories** serialized as
   **CSV or MD**, with the diary body **verbatim and un-mutated**; the mutable representation (`current_text`, stage texts)
   is not the export's authoritative content. User-hard-deleted release groups are absent because Epic H has removed them.
   ([W6][D4][I2])
5. The **`Diary` never appears as a mutable entry in provenance** — the variant history records only the representation's
   evolution; the objective record is reached only via `Export` (and the reader [D2]). This unit issues **no `UPDATE`**
   to `diaries`. ([I2][R7])
6. Both `GetProvenance` and `Export` are **read-only**: they advance no clock ([T3]), append no `memory_provenance` row,
   and spend **no stardust** (metadata/archive tier); they are GET-eligible / `NO_SIDE_EFFECTS`. ([T3][R1][G1])
7. This unit **does not create, migrate, or write** `memory_provenance` — the table is owned by
   [32.reconsolidation-rules](32.reconsolidation-rules.md) and written by [33](33.recall-usecase.md)/[40](40.semanticization.md);
   this unit only reads and exports it. ([CC5])
8. The export **honors the deletion/sealing exclusion** (soft-deleted memories `deleted_at IS NULL`, sealed neurons
   excluded) — respecting a "letting-go" in what is handed out — and performs no deletion itself; a saved-but-past-dated
   retained `Diary` (no star) is still exported. ([I1][X3][X4][T1])
9. Handlers are **thin** (proto↔domain map + call); baseline synthesis, ordering, exclusion, and CSV/MD serialization
   live in the use-case/domain; no proto/sqlc type leaks into the use-case or pure domain; the `ProvenanceReader` and
   `ExportReader` are consumer-owned interfaces. (§2.4/§2.7/§2.9#4/#6/#7)
10. Every provenance read and export query is **scoped to the authenticated user**; `GetProvenance` on another user's
    star returns `not found`, and `pnpm lint:persistence` passes. (§4/[U1])
