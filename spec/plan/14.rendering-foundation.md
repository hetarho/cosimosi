## Purpose

Every presentation unit in Epic A (`universe-canvas`, `star-neuron-rendering`, `nebula-color-field`,
`latent-star-field`) and beyond renders a 3D memory universe. They must not each bootstrap the renderer, the
WebGPU→WebGL2 fallback, the cross-platform shader pipeline, the domain↔renderer boundary, or the background look.
This unit creates that substrate **and** a customizable universe background — so the look can be re-skinned by editing
a few constants (a deliberate "tweak constants → re-skin" affordance), and so Epic A consumes a ready seam.

## Scope / Non-goals

**In scope**
- **One renderer boundary, two hosts.** three.js `WebGPURenderer` (automatic WebGL2 fallback) driven by R3F v9, with all
  bootstrap (async `init`, frame `present`/submit, resize, color space, pixel-ratio cap, post pipeline) behind a single
  boundary in `shared/lib` — a web impl and a `*.native` RN impl (`react-native-webgpu`, MVP fidelity). No slice imports
  `three` directly.
- **`packages/shaders`** — the pure, cross-platform TSL toolkit (no DOM/native deps): shader/material/geometry helpers
  authored in **TSL** so one source compiles to WGSL (WebGPU, web + future native) and GLSL (WebGL2 fallback).
- **Asset-source port** (ARCHITECTURE §3.4): visual bodies arrive through a `VisualBodySource` port
  (shader | glTF | primitive); the domain/mirror layer never imports a concrete renderer.
- **Post-processing seam.** A bloom/glow pass via three's node (TSL) post pipeline, behind the boundary, verified on
  the WebGPU path with graceful WebGL2 degrade.
- **Skinnable universe background/environment.** A "skin" = a named preset of *non-domain* ambiance (background/
  starfield/fog/exposure/bloom defaults, camera mood). Presets are code content; the **active skin is one build-time
  constant** (`values.yaml` → `gen:values`); read through **one seam** designed so a later end-user runtime switcher
  ([P4]) replaces the constant source without touching consumers (runtime-ready, runtime-not-built).
- **`/test` demo.** A panel (via [12.test-harness](12.test-harness.md)) renders the skinned background + a few
  placeholder instanced meshes + the bloom pass + a preset switcher + a demo orbit camera, on web, exercising the
  WebGL2 fallback. No domain data.

**Non-goals**
- **No domain rendering** — real stars (`EpisodicMemory`), neurons, synapses, and the emotion-driven nebula are Epic A
  (`star-neuron-rendering`, `nebula-color-field`, `universe-canvas`, `latent-star-field`).
- **No force-sim** — `packages/force-sim` is its own Epic A unit; the demo uses static placeholder coordinates.
- **No product navigation UX** — the navigable camera rig for the real universe is `universe-canvas` ([U3][V0], Epic A);
  here only a minimal demo camera.
- **No end-user customization UI** — the runtime skin/theme switcher ([P4]) stays deferred; this unit ships only the
  build-time preset + the runtime-ready seam.
- **No mobile *feature* UI** (writing flow, panels, detail) — that is Epic A. The RN *renderer host* itself **is** in
  scope here (MVP fidelity).
- No hardcoded tuning scalars in FE code (they live in `values.yaml`).

## Design

### Stack & the renderer boundary
- Web renderer is three.js `WebGPURenderer` (auto WebGL2 fallback) under R3F v9. All renderer/canvas lifecycle —
  `await renderer.init()`, frame submit/`present`, resize, color management, pixel-ratio cap, the post pipeline — lives
  behind **one** boundary in `shared/lib`. Slices mount a `<UniverseCanvas>`-style host and never touch `three`.
- The RN host (`react-native-webgpu`) is the `*.native` sibling of that boundary — **built at MVP fidelity** (lower
  instance counts / fewer post-FX acceptable). The pure shader toolkit below the boundary is shared verbatim.

### Shaders — TSL only, one source
- `packages/shaders` holds TSL nodes/materials/geometry helpers — **pure, no DOM/native**. Authoring in TSL is the
  cross-platform contract: one source → WGSL (WebGPU) + GLSL (WebGL2). **Raw GLSL via `onBeforeCompile` is forbidden**
  (it forks shaders per backend and breaks the web↔native single-source property).

### Asset-source port (§3.4)
- A `VisualBodySource` port yields a renderable body from `shader | glTF | primitive`; concrete sources are injected at
  the composition boundary. The domain/mirror entities import the port, never `three`. This is the FE anti-corruption
  boundary that Epic-A rendering entities later project through.

### Skin system (the customizable universe background)
- A **skin** is a named preset: a structured object of non-domain ambiance params (background/starfield, fog, exposure,
  bloom defaults, camera mood) + the TSL node graph that draws the background/environment.
- **Presets are code content** (a `packages/shaders` / render-config module) — like a theme table, they are *array/
  object content*, so they stay in code, not `values.yaml`.
- The **active selection is a single build-time constant** (`rendering.active_skin` in `values.yaml` → `gen:values`).
  Re-skinning = change that constant (and/or edit a preset's params). This is the intended "tweak constants → re-skin"
  affordance.
- Consumers read the active skin through **one seam** (`SkinProvider` + `useSkin()`). The seam is designed so a future
  end-user runtime switcher ([P4]) swaps the constant source for a runtime store **without changing any consumer** —
  the placeholder pattern (reserve the slot now, build the UI later).
- **Invariant guard:** the skin is a pure *연출/presentation* layer. It cannot set or override per-memory emotion,
  position, or strength ([I3], [I11]); it only styles non-domain ambiance.

### Post-processing
- Bloom/glow through three's node (TSL) post pipeline, behind the boundary, verified on WebGPU with WebGL2 degrade.
  (Some `@react-three/drei` / `postprocessing` passes assume GLSL/WebGL — prefer TSL/node passes; pin `three`.)

### `/test` verification
- A test panel mounts the boundary with the active skin + 2–3 placeholder `InstancedMesh` buckets + a `Line2`-style fat
  line + the bloom pass + a preset switcher, with a demo orbit camera, on web, exercising the fallback path. No domain
  data, no force-sim, no real stars/neurons.

### Platform posture & version risk
- `packages/shaders` is cross-platform-pure (the single shared shader source); **both** the web and RN renderer hosts
  are built here — web at open-beta fidelity, mobile at MVP fidelity (web = open beta, mobile = MVP).
- RN caveats apply **now**: `react-native-webgpu` is pre-1.0 → **pin exact versions** (TSL API also shifts
  release-to-release → budget migration on `three` bumps); **New Architecture, RN ≥ 0.81 only**; **no Expo Go**
  (custom dev client / prebuild); **test on physical iOS + Android** (emulators fall back to a slow software adapter);
  `present()` + async `init()` differ from web.

## Acceptance Criteria

1. A single renderer boundary in the `@cosimosi/3d-renderer` package brings up three.js `WebGPURenderer` with automatic
   WebGL2 fallback; no slice imports `three` directly.
2. The `@cosimosi/3d-renderer` package's `shader-art/` toolkit is pure TSL (no DOM/native deps) and provides the shared
   shader/material/geometry helpers; shaders are authored in TSL (no raw-GLSL `onBeforeCompile`).
3. An asset-source port supplies visual bodies; the domain/mirror layer imports no concrete renderer.
4. A post-processing seam renders a TSL bloom/glow pass on the WebGPU path and degrades on WebGL2.
5. A skin is a named preset of non-domain background/environment params; presets are code content; the active skin is
   selected by one build-time constant (`values.yaml` → `gen:values`); changing it re-skins the universe background.
6. Skins are read through one seam designed so a runtime switch can replace the constant source without changing any
   consumer ([P4]-ready, not built).
7. The skin/presentation layer cannot mutate domain (emotion / position / strength) — presentation-only ([I3], [I11]).
8. A `/test` panel renders the skinned background + placeholder instanced meshes + the bloom pass + a preset switcher on
   web (and a mobile dev build), with graceful fallback; no domain data, no force-sim, no real stars/neurons.
9. The RN renderer host is built at MVP fidelity on the shared TSL source (web = open beta · mobile = MVP), with pinned
   versions and physical-device verification; `packages/shaders` is the single shared shader source for both.
