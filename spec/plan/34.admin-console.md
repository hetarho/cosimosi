# 34. 관리자 콘솔 (admin-console)

> 관리자 전용 `/admin` 콘솔을 신설한다: ① **LLM 공급자 관리** — 공급자별 지원 모델 리스트·API 키를 웹에서 관리하고(키는 **AES-256-GCM 봉투 암호화** — DB가 통째로 탈취돼도 평문 불가, API는 평문을 절대 반환하지 않음), 활성 공급자·모델을 **재배포 없이 런타임 교체**한다. ② **운영 대시보드** — Sentry(에러)·PostHog(행동)와 겹치지 않는 서비스 합계·성장 추이·잡 큐·**LLM 토큰 사용량**(수익화 단위경제·포트폴리오 용도)만 담는다. | 범위: FS | 선행: 16, 20 | 차단: 없음(21 전 수행 권장)

## 목적 (Why)

- **키/공급자 교체가 지금은 재배포다.** 20이 5사 공급자 추상화를 깔았지만 선택·키가 전부 env라, 교체 = VPS `.env` 수정 + 컨테이너 재생성이다([llm.md](../../llm.md) §순서). 21부터 실 LLM 추출이 **일기마다** 돌기 시작하면 키 로테이션·모델 변경·공급자 전환(비용·품질 튜닝)이 잦아진다 — 21 전에 웹 콘솔로 옮겨 운영 마찰을 없앤다.
- **키는 탈취돼도 무의미해야 한다.** DB에는 AES-256-GCM **암호문만** 저장하고 마스터키(`LLM_KEY_ENCRYPTION_KEY`)는 서버 env에만 둔다(봉투 암호화). DB 덤프·백업 유출 단독으로는 복호화 불가. API·로그·Sentry 어디에도 평문이 나가지 않는다(write-only — 응답은 `key_set`+`last4`뿐).
- **대시보드는 "안 겹치는 것만".** 에러는 Sentry(18), 사용자 행동·퍼널은 PostHog(18), CPU/메모리는 Lightsail/CF 콘솔이 이미 담당 — 그걸 복제하지 않는다. 빈 곳은 **비즈니스/AI 비용 뷰**다: 사용자·일기·별·시냅스 합계와 성장 추이(포트폴리오 스크린샷·수익화 판단), 잡 큐 건강(failed 적체 감지), **LLM 토큰 사용량**(공급자·모델·일 단위 — 사용자당 AI 원가 = 수익화 단위경제의 핵심인데 어떤 외부 도구도 못 잡는다). 전부 관리자 전용 on-demand 집계 + 인덱스 카운트라 베타 규모(~50명)에서 ms 단위 — 서버 부담은 무시 가능.

## 범위

- **포함:**
  - proto 신설 `AdminService`(7 rpc) + 관리자 게이트 인터셉터(`ADMIN_USER_IDS` allowlist, fail-closed).
  - DB 마이그레이션 `00003_admin_llm.sql`: `llm_provider_configs`(공급자별 모델 리스트·암호화 키) · `llm_selection`(단일행 활성 선택) · `llm_usage_daily`(일×공급자×모델×kind 토큰 누적).
  - `internal/admin` 바운디드 컨텍스트(30 settings 미러: handler/service/repository/crypto) — 키 암복호화·공급자 설정·대시보드 집계 소유.
  - `internal/llm` 확장: `Response.Usage`(토큰), 공급자 매트릭스 데이터화(`providers.go` — factory·resolver·admin 공유), **`Resolver`**(DB 활성 선택을 TTL 캐시로 읽어 어댑터 위임, DB 비면 env 폴백 — 기존 동작 보존), usage 싱크.
  - FE `/admin` 페이지(lazy 코드 스플릿): 활성 LLM 선택 · 공급자 5 카드(키 입력/삭제/테스트·모델 리스트 편집) · 대시보드(합계·30일 스파크라인·토큰 사용 — 외부 차트 의존성 없이 경량 SVG).
- **비목표(Out of scope):**
  - **임베딩(05 `openai.go`) usage 계측** — 추출 대비 비용 비중이 수십분의 일. 컬럼(`kind`)은 확장 대비로 두되 배선은 후속.
  - **마스터키 회전 자동화** — 암호문에 버전 바이트(`0x01`)를 박아 포맷만 대비. 재암호화 스크립트는 필요해질 때.
  - **사용자 관리(정지·삭제)·공지·피처 플래그** — admin의 다음 마일스톤. 이 스펙은 LLM 운영 + 대시보드까지만.
  - **요금 청구/결제** — 수익화는 토큰 원가 *가시화*까지만(단가표는 FE 정적 상수). 빌링은 별도 스펙.
  - **인프라 모니터링 복제**(에러율·세션·CPU) — Sentry·PostHog·호스팅 콘솔 소관.
  - 21의 추출 fan-out 자체 — 34는 21이 쓸 **동적 LLM 클라이언트와 계측 싱크를 준비**할 뿐, extract job 적재·소비는 21.

## 설계 요점

**권한 — `ADMIN_USER_IDS` allowlist, fail-closed.** 별도 사용자 테이블·role이 없으므로(인증은 Supabase JWT `sub`) config에 `AdminUserIDs []string`(쉼표 구분 Supabase user UUID)을 추가하고, `rpcserver.NewAdminGateInterceptor(ids)`가 `UserIDFromContext`로 검사해 불일치·미설정 시 `CodePermissionDenied`를 반환한다(**빈 allowlist = 전원 거부**). AdminService 핸들러에만 기존 체인(logging→auth) 뒤에 추가 장착한다 — `connect.WithInterceptors`는 옵션 누적이므로 `append(opts, connect.WithInterceptors(adminGate))`. FE는 PermissionDenied를 받으면 **NotFound 화면을 렌더**해 admin 표면의 존재를 비관리자에게 광고하지 않는다.

**키 암호화 — AES-256-GCM 봉투(envelope), write-only.**
- 마스터키 `LLM_KEY_ENCRYPTION_KEY` = base64 32바이트(생성: `openssl rand -base64 32`), **서버 env에만** 존재. 미설정 상태에서 키 저장 시도 → `CodeFailedPrecondition`(env 이름 명시).
- 저장 포맷(BYTEA): `0x01 ‖ nonce(12B) ‖ ciphertext+tag`. **AAD = provider명** — 다른 행의 암호문을 복사해 끼워도 복호화가 실패한다(행 간 스왑 방지). 버전 바이트는 키 회전 대비 포맷 진화용.
- 평문 키의 생애: `SetProviderKey`/`TestProviderKey` 요청 본문(TLS — CF 에지 종단) → 메모리에서 암호화/1회 호출 → 끝. **어떤 RPC 응답·로그·Sentry에도 평문 미포함**(응답은 `key_set bool`+`key_last4`+`updated_at`만; logging 인터셉터는 본문을 안 찍는 기존 동작 유지 — 18의 본문 금지 가드와 동일 원칙).
- `internal/admin/crypto.go`가 `Seal(provider, plaintext)`/`Open(provider, blob)` 소유. 단위 테스트: 라운드트립·AAD 불일치 실패·잘린 blob 실패·키 미설정 에러.

**DB(`00003_admin_llm.sql`) — overrides-only(30 철학).** 공급자 5종 목록·기본 모델·엔드포인트의 SSOT는 **코드 매트릭스**(`llm/providers.go`)다. DB는 사용자가 바꾼 것만 담는다(시드 행 없음 — `GetLLMConfig`가 매트릭스 위에 DB 행을 병합해 항상 5 카드를 채운다).

```sql
CREATE TABLE llm_provider_configs (
  provider      TEXT PRIMARY KEY,          -- openai|gemini|claude|deepseek|grok (+미래 확장)
  models        TEXT[] NOT NULL DEFAULT '{}', -- 관리자가 추가한 지원 모델(기본 모델은 코드 매트릭스)
  api_key_enc   BYTEA,                     -- 0x01‖nonce‖ct (NULL=키 없음)
  api_key_last4 TEXT,
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE TABLE llm_selection (               -- 활성 추출 LLM(단일행)
  id         SMALLINT PRIMARY KEY DEFAULT 1 CHECK (id = 1),
  provider   TEXT NOT NULL,
  model      TEXT NOT NULL DEFAULT '',     -- '' = 공급자 기본 모델
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE TABLE llm_usage_daily (             -- upsert 누적(일×공급자×모델×kind)
  day           DATE NOT NULL,             -- UTC
  provider      TEXT NOT NULL,
  model         TEXT NOT NULL,
  kind          TEXT NOT NULL,             -- 'extract' (embed는 후속)
  calls         BIGINT NOT NULL DEFAULT 0,
  input_tokens  BIGINT NOT NULL DEFAULT 0,
  output_tokens BIGINT NOT NULL DEFAULT 0,
  PRIMARY KEY (day, provider, model, kind)
);
```

⚠️ **마이그레이션 번호 시프트(이 계획이 확정한 단일 기준 — 00.overview §공유 설계 결정 동기화 완료):** 34가 21보다 먼저 빌드되므로 `00003`=34. 기존 예약은 21→`00004`, 23→`00005`, 27→`00006`으로 민다(21·23·27 문서 동기화 완료). `00001`·`00002`는 불변.

**`internal/llm` 확장 — 매트릭스 데이터화 + Resolver + Usage.**
- `providers.go`(신규): 공급자 매트릭스(이름→엔드포인트·기본 모델·키 env명·json_schema 지원·max_tokens 파라미터명)를 **데이터(맵)** 로 옮기고 `NewForProvider(provider, model, apiKey string, hc *http.Client) (Client, error)` 헬퍼를 둔다. `factory.New`(env 경로)·`Resolver`(DB 경로)·admin의 `TestProviderKey`(임시 키 경로) 셋이 같은 테이블을 재사용 — factory의 switch 중복이 사라진다. 매트릭스의 공급자 키 집합이 곧 `AdminService`가 허용하는 provider 값 도메인이다.
- `llm.go`: `Response`에 `Usage struct{ InputTokens, OutputTokens int }` 추가. 각 어댑터가 공급자 응답의 usage 필드를 매핑한다(openai-compat: `usage.prompt_tokens/completion_tokens` · anthropic: `usage.input_tokens/output_tokens` · gemini: `usageMetadata.promptTokenCount/candidatesTokenCount`). 없으면 0(베스트에포트).
- `resolver.go`(신규): `NewResolver(src ConfigSource, cfg *config.Config, sink UsageSink) Client`.
  - `ConfigSource`는 llm 패키지의 작은 포트 `ActiveLLM(ctx) (provider, model, apiKey string, ok bool, err error)` — **admin 패키지가 구현**(selection 조회 + 키 복호화). llm은 DB·admin을 모른다(의존 역전 — 헌법7 유지).
  - `Complete` 시 활성 설정을 **짧은 TTL(~30s) 캐시**로 읽어 해당 어댑터(`NewForProvider`)에 위임 → **`SetActiveLLM`·키 교체가 재시작 없이 ≤30s 내 반영**된다. `ok=false`(DB 미설정)면 env 기반 `factory.New` 클라이언트로 폴백 — 기존 동작·키리스 mock 경로 회귀 없음.
  - 성공한 `Complete` 후 `sink(day, provider, model, usage)`를 베스트에포트 호출(에러는 로그만 — **계측 실패가 추출을 실패시키지 않는다**).
- `ai.NewExtractor`는 `llm.New(cfg)` 대신 Resolver를 받도록 조립 경로만 바뀐다(현재 호출부 없음 — 21이 워커에 주입할 때 자동으로 동적 클라이언트를 얻는다).

**proto — `proto/cosimosi/v1/admin.proto`(신규 파일, 같은 패키지).** 헌법5(DTO)·6(unary). 요지:

```proto
message ProviderConfig {            // 평문 키 필드는 어떤 메시지에도 없다
  string provider = 1;              // 매트릭스 키(openai|gemini|claude|deepseek|grok)
  string default_model = 2;         // 코드 매트릭스 기본 모델(읽기 전용)
  repeated string models = 3;       // 관리자 추가 모델 리스트
  bool key_set = 4;
  string key_last4 = 5;
  string key_updated_at = 6;
}
message GetLLMConfigResponse {
  repeated ProviderConfig providers = 1;   // 항상 5종(매트릭스 ∪ DB 병합)
  string active_provider = 2;
  string active_model = 3;                 // '' = 공급자 기본
  bool encryption_ready = 4;               // LLM_KEY_ENCRYPTION_KEY 설정 여부(FE 안내 배너)
}
message SetProviderKeyRequest { string provider = 1; string api_key = 2; }   // write-only
message TestProviderKeyRequest {        // 저장 전 검증용 api_key(선택; 비면 저장된 키 사용)
  string provider = 1; string model = 2; string api_key = 3;
}
message TestProviderKeyResponse { bool ok = 1; string message = 2; int64 latency_ms = 3; }
message GetAdminOverviewResponse {
  int64 users = 1; int64 records = 2; int64 memories = 3; int64 synapses = 4;
  int64 jobs_pending = 5; int64 jobs_processing = 6; int64 jobs_failed = 7; int64 jobs_done_24h = 8;
  repeated DayCount record_series = 9;     // 최근 30일 일기 수
  repeated UsageRow llm_usage = 10;        // 최근 30일 day×provider×model×kind
}
service AdminService {
  rpc GetLLMConfig(GetLLMConfigRequest) returns (GetLLMConfigResponse) { option idempotency_level = NO_SIDE_EFFECTS; }
  rpc SetProviderKey(SetProviderKeyRequest) returns (SetProviderKeyResponse);
  rpc DeleteProviderKey(DeleteProviderKeyRequest) returns (DeleteProviderKeyResponse);
  rpc UpdateProviderModels(UpdateProviderModelsRequest) returns (UpdateProviderModelsResponse); // 리스트 교체
  rpc SetActiveLLM(SetActiveLLMRequest) returns (SetActiveLLMResponse);   // model ∈ models ∪ {default, ''} 검증
  rpc TestProviderKey(TestProviderKeyRequest) returns (TestProviderKeyResponse); // MaxTokens 16 ping 1회
  rpc GetAdminOverview(GetAdminOverviewRequest) returns (GetAdminOverviewResponse) { option idempotency_level = NO_SIDE_EFFECTS; }
}
```

**대시보드 집계(`internal/admin` + `queries/admin.sql`).** 합계 4종은 인덱스 카운트, 시리즈는 최근 30일 `GROUP BY day`, usage는 `llm_usage_daily` 범위 조회 — 전부 관리자 클릭 시 1회(on-demand, 폴링 없음). **가입자 수**: 우리 스키마엔 사용자 테이블이 없으므로 기본은 `COUNT(DISTINCT user_id) FROM records`(기록 보유 사용자). Supabase DB에선 `to_regclass('auth.users')` 존재 시에만 `auth.users` 카운트를 곁들인다(로컬 도커 pg엔 auth 스키마가 없어도 에러 없이 생략). **비용 추정은 FE**: `pages/admin/lib/pricing.ts`의 정적 단가표(USD/1M tokens, 주석으로 갱신처 안내) × 토큰 — 단가는 자주 바뀌므로 서버·DB에 두지 않고 "추정" 라벨을 단다.

**FE — `/admin`(lazy) + pages/admin(FSD).** `router.tsx`에 `lazyRouteComponent`로 코드 스플릿 라우트 추가(SessionGate 보호 — 메인 번들 영향 0). 페이지 구성(다크 우주 톤, 모바일 1열):
1. **활성 LLM 카드** — 공급자·모델 셀렉트(`GetLLMConfig`의 models ∪ default) + 저장(`SetActiveLLM`). `encryption_ready=false`면 마스터키 설정 안내 배너.
2. **공급자 카드 ×5** — 키 상태 배지(`key_set`·last4·updated_at), 키 입력(submit 후 입력값 즉시 클리어 — 화면에도 안 남김)·삭제·**테스트 버튼**(`TestProviderKey` — 저장 전 입력 중 키도 검증 가능), 모델 리스트 편집(추가/제거 → `UpdateProviderModels`).
3. **대시보드** — 합계 카드 4 + 잡 큐 상태 + 30일 일기 스파크라인 + LLM 토큰/추정 비용 바 — **차트 라이브러리 추가 없이 경량 인라인 SVG**.
데이터 훅은 connect-query(16) 패턴(`pages/admin/api/admin-queries.ts`), 뮤테이션 성공 시 `GetLLMConfig` invalidate. 비관리자: 첫 쿼리의 PermissionDenied → `NotFoundScreen` 렌더.

## 수용 기준 (EARS / Given-When-Then)

1.1 WHEN `SetProviderKey`로 키를 저장하면 THE SYSTEM SHALL DB에 AES-256-GCM 암호문(`0x01‖nonce‖ct`, AAD=provider)만 기록하고 평문 컬럼·평문 로그를 남기지 않는다.
1.2 WHEN 어떤 AdminService RPC가 응답하더라도 THE SYSTEM SHALL 평문 API 키를 포함하지 않는다(`key_set`/`key_last4`/`updated_at`만).
1.3 IF `LLM_KEY_ENCRYPTION_KEY`가 미설정이면 THEN `SetProviderKey`는 `FailedPrecondition`(env 이름 명시)으로 거부하고, `GetLLMConfig.encryption_ready=false`로 FE가 안내한다.
1.4 WHEN 한 provider 행의 암호문을 다른 provider 행으로 복사하면 THE SYSTEM SHALL 복호화에 실패한다(AAD 불일치 — 단위 테스트).
1.5 WHEN `DeleteProviderKey`를 호출하면 THE SYSTEM SHALL `api_key_enc`/`api_key_last4`를 NULL로 비운다.
2.1 WHEN `SetActiveLLM`으로 공급자·모델을 바꾸면 THE SYSTEM SHALL **프로세스 재시작 없이** ≤30s(TTL) 내 다음 `Complete` 호출부터 새 공급자·모델을 사용한다.
2.2 IF `llm_selection`이 비어 있으면(미설정) THEN Resolver는 env(`LLM_PROVIDER`/키)로 폴백하고, `AI_EXTRACTOR=mock` 키리스 경로는 일절 변하지 않는다(회귀 0).
2.3 WHEN `SetActiveLLM.model`이 그 공급자의 `models ∪ {default_model, ''}` 밖이면 THE SYSTEM SHALL `InvalidArgument`를 반환한다. provider가 매트릭스 밖이어도 동일.
2.4 WHEN `TestProviderKey`를 호출하면 THE SYSTEM SHALL MaxTokens 16 완성 1회로 키를 검증해 `{ok, message, latency_ms}`를 반환하고, 실패 message에 키 원문을 포함하지 않는다.
3.1 WHEN 호출자 `sub`가 `ADMIN_USER_IDS`에 없으면 THE SYSTEM SHALL 모든 AdminService RPC를 `PermissionDenied`로 거부한다(미인증은 기존대로 `Unauthenticated`).
3.2 IF `ADMIN_USER_IDS`가 비어 있으면 THEN THE SYSTEM SHALL 전원을 거부한다(fail-closed).
3.3 WHEN 비관리자가 `/admin`에 진입하면 THE SYSTEM SHALL NotFound 화면을 렌더한다(admin 표면 비노출, 콘솔 무에러).
4.1 WHEN `GetAdminOverview`를 호출하면 THE SYSTEM SHALL 합계(users·records·memories·synapses)·잡 카운트(pending/processing/failed/done_24h)·최근 30일 일기 시리즈·LLM usage를 반환한다(NO_SIDE_EFFECTS — HTTP GET).
4.2 WHEN Resolver 경유 `Complete`가 성공하면 THE SYSTEM SHALL `llm_usage_daily`에 (UTC day, provider, model, kind='extract') 행을 upsert 누적하고, 싱크 실패는 호출을 실패시키지 않는다(로그만).
4.3 WHEN `auth.users`가 없는 DB(로컬 도커 pg)에서 집계해도 THE SYSTEM SHALL 에러 없이 동작한다(`to_regclass` 가드 — 기본 users는 `COUNT(DISTINCT user_id) FROM records`).
