# 05. embedding-worker — AI 포트 + 임베딩 워커 + 연결 생성

> 일기 텍스트를 임베딩으로 바꿔 pgvector에 저장하고 top-k 최근접 별과 시냅스를 생성한다. | 범위: MVP | 선행: 03, 04 | 차단: 의미있는 연결(09 synapse, 11 recall)

## 목적 (Why)
04에서 일기 저장 시 `jobs`에 적재만 되고(별은 떠도 연결이 없음), 우주에 시냅스가 없다. 이 스펙은 비동기 워커를 붙여 **임베딩 생성 → pgvector 저장 → top-k 최근접 연결(memory_links) 생성**을 완성한다. AI 공급자는 포트 뒤에 두어(원칙7) 키 없이도 dummy 임베더로 돌아가게 한다.

## 범위
- **포함:** `internal/ai`(Embedder 포트 + OpenAI 어댑터 + dummy + Extractor 포트만), `internal/job`(claim `FOR UPDATE SKIP LOCKED` + 지수 백오프), 워커 파이프라인(MVP는 `cmd/api` 내 고루틴 권장), `db/queries/{embedding,link,job}.sql`, 연결 규칙(τ=0.75, top-k=8, `w0=clamp(cos_sim+temporal_bonus,0,1)`), 비용 가드(토큰 상한·캐싱 스텁, 임베딩/LLM 분리 계측), config(`AI_EMBEDDER`,`OPENAI_API_KEY`,차원 1536 고정).
- **비목표(Out of scope):**
  - **LLM Extractor 구현** — 포트(`Extractor` 인터페이스)만 두고 MVP는 no-op. 실제 분류·개체·풍부 비주얼 추출은 v1 `24.llm-extraction`.
  - **proto/RPC·서버 와이어링** — 02·04 담당. 이 스펙은 04가 만든 `RecordMemory` 핸들러가 적재한 job을 소비할 뿐.
  - **스키마(테이블·HNSW 인덱스) 정의** — 03 담당. 여기선 03이 만든 `embeddings`/`memory_links`/`jobs`에 대한 쿼리만 작성.
  - **별·시냅스 렌더링, 감쇠 시각화** — 08·09·12 담당. 워커는 가중치 그래프만 채운다.
  - **공동 회상 강화(weight += 0.05)** — 11 `recall-reinforce` 담당. 여기선 **초기 연결 생성**만.

## 설계 요점
**AI 포트(원칙7 — 어떤 임베딩/LLM도 교체 가능). 도메인·태그 금지(원칙5).**
```go
// internal/ai/embedder.go
type Embedder interface {
    // 반환 벡터 길이는 어댑터가 Dim()으로 보장(MVP 1536 고정).
    Embed(ctx context.Context, text string) ([]float32, error)
    Dim() int
}
// internal/ai/extractor.go — 포트만. MVP no-op 구현(NoopExtractor).
type Extractor interface {
    Extract(ctx context.Context, text string) (Extraction, error)
}
```
- `openai.go`: `OPENAI_API_KEY`로 `text-embedding-3-small`(1536) 호출. 응답 차원 ≠ `Dim()`이면 에러. 토큰 상한 초과 입력은 잘라내고(또는 거부) 카운터 증가. 호출 결과는 텍스트 해시 키 인메모리 캐시 스텁(중복 호출 절약).
- `dummy.go`: 키 불필요. 텍스트 해시 시드의 결정론적 정규화 벡터(`[]float32` 길이 1536, L2 정규화). `AI_EMBEDDER=dummy`로 선택 → 키 없는 E2E.
- `metrics.go`(또는 service 내 카운터): 임베딩 호출 수·토큰·캐시 적중을 LLM과 **분리 계측**(임베딩=저비용, LLM=고비용). MVP는 구조화 로그 카운터로 충분.

**job 큐(Architecture §3.6).** `internal/job`은 `jobs` 테이블 기반. `claim`은 `SELECT ... FOR UPDATE SKIP LOCKED LIMIT 1`로 동시 워커 안전. 실패 시 `attempts++`·`error` 저장·`status='pending'` 복귀 + **지수 백오프**(`next_run_at = now + base*2^attempts`, 상한). `attempts ≥ max`면 `status='failed'`로 보존(삭제 금지).

**연결 규칙(Architecture §5, 그대로).** 새 별의 임베딩으로 `ORDER BY embedding <=> $1 LIMIT 8`(자기 자신·`cos_sim < 0.75` 제외). 각 후보에 `w0 = clamp(cos_sim + temporal_bonus, 0, 1)`, `temporal_bonus`는 같은 주 `+0.3` 선형 감소(7일 경과 시 0). 정규화: `a_id < b_id`로 무방향 1행. **UNNEST 배치 업서트**(`ON CONFLICT (a_id,b_id) DO UPDATE SET weight = GREATEST`)로 다건 한 번에.

**파이프라인(MVP 고루틴).** `cmd/api` 기동 시 워커 고루틴 1개 띄움(폴링 간격 짧게). `cmd/worker/main.go`는 동일 로직을 별도 프로세스로 감싸는 **선택적 분리**(MVP 필수 아님 — 골격만).
```
job claim(pending,kind=embed) → memory 본문 로드 → Embedder.Embed
  → embeddings upsert(memory_id, vector, model)
  → knn(≥0.75, ≤8, self 제외) → link UNNEST 배치 업서트(w0)
  → job complete   |  실패 → 백오프 재시도 → 한계 시 failed 보존
```

## 수용 기준 (EARS / Given-When-Then)
1.1 WHEN 워커가 `pending`·`kind=embed` job을 claim하면 THE SYSTEM SHALL `SELECT ... FOR UPDATE SKIP LOCKED LIMIT 1`로 한 건만 잠그고 `status='running'`으로 표시한다.
1.2 WHEN job 처리가 성공하면 THE SYSTEM SHALL 해당 memory의 1536차원 임베딩을 `embeddings`에 업서트하고 `model`(어댑터·차원)을 기록한다.
1.3 GIVEN 임베딩이 저장된 상태에서 WHEN 워커가 최근접을 조회하면 THE SYSTEM SHALL `cos_sim ≥ 0.75`인 후보를 자기 자신 제외 **최대 8개** `embedding <=> $1` 오름차순으로 가져온다.
1.4 WHEN 후보 연결을 영속화하면 THE SYSTEM SHALL `a_id < b_id` 정규화 무방향 1행으로, `w0 = clamp(cos_sim + temporal_bonus, 0, 1)`(같은 주 `+0.3` 선형 감소)을 UNNEST 배치로 `ON CONFLICT (a_id,b_id)` 업서트한다.
1.5 WHEN 연결 생성이 끝나면 THE SYSTEM SHALL job을 `status='done'`으로 완료 처리한다.
2.1 IF `Embed` 호출이 실패하면(네트워크·키·5xx) THEN THE SYSTEM SHALL `attempts++`·`error` 기록 후 `next_run_at = now + base·2^attempts`(상한 적용)로 재시도 예약하고 job을 삭제하지 않는다.
2.2 IF `attempts`가 최대치에 도달하면 THEN THE SYSTEM SHALL `status='failed'`로 상태를 **보존**한다(원본 일기·memory는 불변, 원칙1).
2.3 IF OpenAI 응답 벡터 길이가 `Dim()`(1536)과 다르면 THEN THE SYSTEM SHALL 에러로 처리하고 임베딩을 저장하지 않는다.
2.4 IF 입력 토큰이 상한을 초과하면 THEN THE SYSTEM SHALL 잘라내거나 거부하고 비용 가드 카운터를 증가시킨다.
3.1 WHEN `AI_EMBEDDER=dummy`이면 THE SYSTEM SHALL `OPENAI_API_KEY` 없이도 결정론적 1536차원 벡터를 반환해 전체 파이프라인을 완주한다(임베더 교체 가능, 원칙7).
3.2 WHEN top-k에 자기 자신·임계 미만 후보만 있으면 THE SYSTEM SHALL 연결을 0개 만들고도 job을 정상 완료한다(별은 고립된 채 우주에 잔존, 원칙2).
4.1 THE SYSTEM SHALL 별/시냅스를 삭제하지 않으며(원칙2) 워커는 좌표를 저장하지 않는다(좌표는 클라 force-sim 권위, 원칙3).

## 완료 정의 (DoD)
- `AI_EMBEDDER=dummy`로 백엔드 기동 후 **`OPENAI_API_KEY` 없이** 일기 2건을 `RecordMemory`(04)로 저장하면, 워커가 두 건 모두 임베딩을 채우고 유사도 ≥0.75일 때 `memory_links`에 행이 생긴다.
- `docker compose exec postgres psql -U postgres -d cosimosi -c "select count(*) from embeddings;"` ≥ 2, `... "select count(*) from memory_links;"` ≥ (연결 수). 모든 row는 `a_id < b_id`.
- `GetUniverse`(04) 응답에 시냅스(edges)가 등장한다(09 렌더 전제 데이터 충족).
- Embedder를 `openai`로 바꿔도 동일 파이프라인이 동작한다(교체 가능, 원칙7). 키 오류 시 job은 `failed`로 보존되고 별·원본 일기는 그대로(원칙1·2).
- `docker run ... golang:1.26 go build ./...` 와 `go test ./internal/ai/... ./internal/job/...` 통과.
