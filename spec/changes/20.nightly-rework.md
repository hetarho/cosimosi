# 20. 변경: 야간 배치 재작성

> [plan/27.nightly-consolidation](../plan/27.nightly-consolidation.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> "우주의 수면"을 새 망각 모델(거리 단일 변수, change 18·19)에 맞춰 다시 짠다. 패스 추가/수정이 많아 가장 큰 변경.

## 현재 (as-is)

- 매일 1회(18:00 UTC ≈ 03:00 KST) 사용자별 `consolidate` 잡이 깨어나 **4패스**를 돈다([consolidate.go](../../backend/internal/job/consolidate.go)). 삭제 없음·원본 불변.
  - ① **재안정화**: 전체 그래프 서버 force-sim 재수렴(낮 부분배치 오차 보정), 좌표 캐시.
  - ② **재분배**: 별을 소속 성단 중심으로 `lerp 0.6` 끌어당김, **도식 적합 별엔 +0.15(schemaBonus)**.
  - ③ **요지화**: 오래되고(>30일) 안 떠올린(>14일) 별의 `form_seed_delta`를 고정 스텝(연속값) 단조 증가.
  - ④ **가지치기**: 약하고(weight<0.2) 오래 안 쓴(>14일) 링크의 weight를 0.05 바닥으로(밝기만).
- 전체 그래프 대상(반지름 스코프 없음). 링크는 link_type 구분 없이 동일 취급(재가중 없음).

## 목표 (to-be)

1. **시각(04시 KST).** 깨어나는 시각을 04:00 KST(= 19:00 UTC)로.
2. **재안정화·재분배 — 반지름 스코프.** 전체가 아니라 **특정 반지름 안(중심 가까운 활성 영역)에 있는 별들만** 재안정화/재분배한다. 멀리 표류한(거의 잊힌) 별은 밤마다 다시 끌어당기지 않고 가만둔다(망각 존중).
3. **링크 재가중(시간↓·의미↑).** 재분배/정리 시 **시간 기반 연결(`temporal`/시간보너스 파생)은 약화**하고, **의미 기반 연결(`semantic`)은 강화**한다. 시간이 지나며 "그때 같이 썼다"는 묶임은 풀리고 "의미가 닮았다"는 묶임이 남는다(과학: 시간창은 짧고 의미·도식 연결이 장기 보존).
4. **랜덤 spread(뭉침 방지).** 별 무리가 한 곳으로만 수렴하지 않게, 재분배 후 성단들을 3D 공간에 고루 퍼지도록 **약한 랜덤 변위**를 준다(군집 간 반발/지터). 단 결정론 재현이 필요하면 seed 기반 의사난수(프레임 무관).
5. **망각(거리 표류).** 별이 중심에서 멀어지는 차등 표류는 change 18(거리 공식)이 라이브로 담당한다 — 이 배치는 그 새 반지름으로 재안정화만 한다(중복 구현 금지).
6. **요지화 — 반지름 트리거 + 1~4 이산 단계.** 트리거를 "나이/회상"에서 **"반지름 임계 초과"** 로 바꾸고, 연속 `form_seed_delta` 대신 **추상화 단계 1~4(이산)** 로 승급한다. 별이 특정 반지름들을 넘을 때마다 단계가 +1(최대 4). 단계는 별 형태(신규 plan 53)와 재공고화 AI 변형(신규 plan 54)의 입력.
7. **가지치기 — 전반 약화 + 마지막 1링크 보호.** 안 쓴 링크는 전반적으로 weight를 낮추되(끊긴 듯 처리 가능), **별마다 가장 강한 링크 1개는 보호**해 완전 고립을 막는다("선 하나라도 남긴다"). 삭제(DELETE) 대신 weight 바닥/`severed` 처리(헌법2 유지) — row는 보존해 재연결(아래) 시 되살아남.
8. **주기적 재-KNN 패스(신규).** 옛 별도 주기적으로 의미 KNN을 다시 돌려, 그새 생긴 닮은 기억과 **뒤늦게 새 링크**를 맺는다. 끊겼거나 원래 고립이던 별의 재연결 안전망(가지치기·끊김의 짝).

## 범위 / 비목표 (회귀 경계)

**바꾼다:**
- [consolidate.go](../../backend/internal/job/consolidate.go) 패스 전반: 시각 상수, 반지름 스코프, 링크 재가중 로직(신규), spread(신규), 요지화 트리거·단계(신규), 가지치기 마지막-링크 보호, 재-KNN 패스(신규).
- DB: 별 `abstraction_stage`(0~4) 컬럼 신규(요지화 단계 영속). `memory_links`에 `severed`/약화 표현(끊김 처리 방식 따라).
- `spec/values.yaml`: 시각, 반지름 스코프 임계, 링크 재가중 계수(시간↓·의미↑), spread 강도, 요지화 단계별 반지름 임계 4개, 재-KNN 주기/임계, schemaBonus 제거.
- plan 27 문서 + [policy/domain/universe.md](../policy/domain/universe.md) §야간 공고화.

**안 바꾼다(회귀 경계):**

- **도식/패턴분리(schemaBonus) 제거** — 비목표가 아니라 명시적 삭제 대상.

## 수용 기준 (인수 조건)

1. 잡이 04:00 KST(19:00 UTC)에 깨어난다.
2. 재안정화·재분배가 **반지름 임계 안 별에만** 적용된다 — 멀리 표류한 별의 좌표는 밤에 안 끌려온다.
3. 야간 후 `temporal` 계열 링크 weight는 낮아지고 `semantic` 링크 weight는 (상한 내) 높아진다.
4. 재분배 후 성단들이 한 점에 안 뭉치고 3D로 분산된다(같은 입력 → 같은 결과, 결정론).
5. 요지화: 별이 반지름 임계를 넘으면 `abstraction_stage`가 +1(0→…→4, 단조, 최대 4에서 멈춤). 나이/회상 트리거는 더는 안 쓴다.
6. 가지치기 후에도 **모든 별이 최소 1개 링크를 유지**한다(degree ≥ 1). 어떤 링크 row도 DELETE되지 않는다.
7. 재-KNN 패스가 고립/끊긴 옛 별에 닮은 기억이 생겼을 때 새 링크를 만든다.
8. schemaBonus가 코드·values에서 사라진다.
9. 원본 record·기존 별 row는 한 건도 안 바뀌고 안 지워진다(잡 재실행에도 안전).
