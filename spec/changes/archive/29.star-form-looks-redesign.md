# 29. 변경: 별 형태 3종 룩 재설계 — 추상화 단계 시각화·랜덤성 확대

> [plan/53.memory-state-star-form](../plan/53.memory-state-star-form.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 추상화(`abstraction_stage`)가 지금은 "부드러운 변위 약화"라 육안으로 단계가 안 보인다. 형태 선택지를 **단일 축 3종 룩**으로
> 줄이고, 각 룩이 단계마다 **눈에 띄게 다른 실루엣**으로 변하며, 별마다 랜덤성을 크게 키워 모두 달라 보이게 한다.
> ⚠️ **헌법8 개정 동반**(별 단일 InstancedMesh → 룩×단계 버킷별 소수 고정 InstancedMesh) — §범위 참고.

## 현재 (as-is)
<!-- plan/53 + plan/52 중 이 변경이 건드리는 현재 동작. 출발점. -->

- **형태 모델 = form×surface 2축**(plan 52). form 5종(`lowpoly`·`octa`·`smooth`·`cloudy`·`liquid`) × surface 5종
  (`facet`·`glossy`·`lava`·`cloud`·`pulse`) = 25 조합. 합성 wire id `"<form>+<surface>"`. ([entities/star/model/forms.ts](../../frontend/src/entities/star/model/forms.ts))
- **추상화 = in-shader 부드러운 변위 약화**(plan 53). 단일 공유 지오메트리(헌법8) 위에서 `seedShape`가 per-instance
  정점을 변위하고, `abstraction_stage`(0~4, [values.yaml](../values.yaml) `consolidation.gist_stage_radii`=`[40,55,68,78]`)가
  올라가면 `stageSimplify`(0.75) 비율로 변위·비대칭을 **줄인다**. 면 수·기본 실루엣은 단계와 무관하게 그대로 →
  **단계 차이가 거의 안 보인다.** ([star-body.ts](../../frontend/src/entities/star/ui/star-body.ts) `seedShape`·`stageFactors`)
- **랜덤성** = `starForm.displace_amp`·`detail_amp`·`asymmetry`(values) 진폭이 작아 별끼리 실루엣 차이가 미묘하다.
- **렌더 = 단일 InstancedMesh**(헌법8). 모든 별이 한 지오메트리를 공유, per-instance attribute(seed/stage/mood)만 다르다.
- **선택은 전역 1개** — 합성 id 하나가 모든 별에 적용(감정 무관). (감정별 선택은 별도 change 30이 다룬다.)
- 정식 출시 전 — 레거시 소유/프리셋 호환 부담 없음(미지 id면 디폴트로 폴백만 하면 됨).

## 목표 (to-be)
<!-- 변경 후 참이어야 하는 동작·계약. 구체적으로. -->

### 1. 단일 축 3종 룩 (form×surface 2축 폐기)
형태 선택을 **단일 축 "별 형태" 3종**으로 통합한다. 각 룩은 **지오메트리(모양) + 셰이딩/질감(표면) + 4단계 추상화
변형**을 한 덩어리로 정의한다(2축 조합 폭발 폐기 — 선택지를 확실히 줄여 추상화가 또렷이 보이게).

| 룩 id | 이름(가칭) | 단계 0(또렷) | 단계 1 | 단계 2 | 단계 3~4(요지) |
|---|---|---|---|---|---|
| `polyhedron` | 다면체 | 20면체(icosahedron) | 12면체(dodecahedron) | 8면체(octahedron) | 4면체(tetrahedron) |
| `liquid` | 액체→구름 | 리퀴드(불투명·또렷 변위 구) | 좀 더 투명 | 더 투명 | 가장 투명·구름처럼 흩어짐 |
| `spiky` | 고슴도치 | 뾰족뾰족 가시 촘촘 | 가시 듬성 | 더 듬성 | 가시 없는 다각형(뿔 사라짐) |

- **단계 = 추상화(`abstraction_stage`)가 구동**(사용자 선택 아님). 별이 잊혀(반지름이 `gist_stage_radii` 임계를
  넘어)갈수록 단계가 올라 형태가 단조롭게 단순화/추상화된다(plan 53 요지화 의미 보존).
- 단계 변형은 **육안으로 또렷이 구분**되어야 한다(부드러운 변위 약화가 아니라 실루엣 자체가 바뀜).

### 2. 단계별 실제 지오메트리 교체 (멀티 InstancedMesh — 헌법8 개정)
`polyhedron`은 면 수(20→12→8→4)가 단계마다 **실제로 다른 지오메트리**다. 단일 InstancedMesh 한 개로는
표현 불가하므로, **별을 (룩 × 단계 버킷)별 InstancedMesh로 분할**해 렌더한다. 별이 단계를 넘으면 해당 버킷
메시로 이동한다. 메시 개수는 **소수 고정**(룩 3 × 단계 5 = 최대 15, 실제 구현은 같은 지오메트리 단계는 공유 가능)
이라 "수천 노드" 전제(헌법8 정신)는 유지된다.

### 3. 랜덤성 대폭 확대 (별마다 다 다르게)
같은 룩·같은 단계라도 별마다 실루엣이 **확연히** 다르게 한다. 변형 강도(진폭/비대칭)를 크게 키우고, 룩별로
의미 있는 per-star 변형 축을 둔다 — 예: `polyhedron` 정점 변위·비대칭 스트레치, `spiky` 가시 개수·길이·분포,
`liquid` 변위 주파수·투명도 미세 변동. 전부 **별 고유 seed**(`shape` 3축, 탄생 시 결정)에서 결정론적으로 파생
(같은 seed·stage → 항상 같은 모양, Math.random 비사용 — 우주가 매번 안 뒤집힘).

## 범위 / 비목표 (회귀 경계)

**바꾼다**
- 별 형태 카탈로그: form×surface 2축(5×5) → **단일 축 3종 룩**. `STAR_FORMS`·`STAR_SURFACES`·`STAR_PRESETS`·
  합성 id 파싱을 단일 축 룩 모델로 교체. [forms.ts](../../frontend/src/entities/star/model/forms.ts)·
  [kinds.ts](../../frontend/src/entities/star/model/kinds.ts)·`types.ts`.
- `abstraction_stage`→형태 매핑: 부드러운 변위 약화 → **단계별 실제 지오메트리 교체**(룩별 4단계 모델). [star-body.ts](../../frontend/src/entities/star/ui/star-body.ts).
- 렌더: 단일 InstancedMesh → **(룩×단계) 버킷별 InstancedMesh**. [widgets/universe-canvas](../../frontend/src/widgets/universe-canvas/) `StarField`·별 인스턴싱.
- per-star 랜덤성 진폭 대폭 확대(룩별 변형 축 추가). [values.yaml](../values.yaml) `starForm.*`.
- 정식 출시 전 정리: 폐기된 form/surface의 **유료 아이템(`star:form:*`·`star:surface:*`)·무료 슬롯 키를 3종 룩 기준으로 교체**. [values.yaml](../values.yaml) `customization.price`·`customization.free`.

**비목표(회귀 경계)**
- **색 규칙 불변** — 별 색 = mood(13감정, plan 29·44). 룩/단계는 형태·질감만 바꾼다.
- **크기·밝기 불변** — 크기 = intensity(`sizeFor`), 밝기 = 거리(`brightnessFromRadius`, A_MIN 바닥). 형태 변형은
  반지름 1 대비로 잡아 크기 규칙을 침범하지 않는다(plan 53 A5).
- **결정론·서버 계약 불변** — 서버는 seed·`abstraction_stage` 원자료만 권위(헌법3). 형태는 클라 파생. proto/DB 스키마 무변경.
- **감정별 선택은 이 change 비범위** — 전역 단일 선택 유지(감정별 오버라이드는 change 30). 단, change 30이 얹힐
  단일 축 룩 모델을 이 change가 먼저 정립한다(30은 29에 의존).
- **레거시 호환 비목표** — 미지/구 합성 id는 디폴트 룩으로 폴백(폴백 테이블·재구매 호환 불필요, 정식 출시 전).
- 다른 entity(나·시냅스)의 form×surface 2축은 **이 change 비범위**(별만). 나/시냅스 모델은 그대로 둔다.

### 헌법 개정 메모 (deliberate amendment)
이 change는 **헌법8을 의도적으로 개정**한다(사용자 승인): "별은 InstancedMesh(단수)" → "별은 **소수 고정 개수의
InstancedMesh 집합**(룩×단계 버킷)". 정신("수천 노드 전제·인스턴싱")은 유지 — 메시 개수가 노드 수와 무관하게
상수(≤15)로 묶이므로 인스턴싱 이득이 보존된다. 00.overview §불변 원칙 §8 문구를 구현 시 갱신한다.

## 수용 기준 (인수 조건)
<!-- 변경 완료의 판정 기준. job의 "인수 조건" 섹션으로 복사된다. -->
1. 별 형태 선택이 **단일 축 3종 룩**(`polyhedron`·`liquid`·`spiky`)이다. form×surface 2축·25 조합·합성 id는 사라진다.
2. `abstraction_stage`가 오르면 형태가 **육안으로 또렷이** 단계 변형된다 — `polyhedron`은 면 수 20→12→8→4로 실제
   지오메트리가 바뀌고, `liquid`는 점점 투명·구름화, `spiky`는 가시가 듬성→소멸(다각형). 단조(단계↑ = 더 추상).
3. 같은 룩·같은 단계라도 **별마다 실루엣이 확연히 다르다**(랜덤성 확대 — 회전만 다른 게 아님).
4. 같은 memory id·stage·seed면 항상 같은 모양(결정론, 세션·기기 무관, Math.random 비사용).
5. 색(mood)·크기(intensity)·밝기(거리)는 이 변경으로 안 바뀐다(회귀 경계).
6. 렌더가 (룩×단계) 버킷별 InstancedMesh로 분할되고 메시 개수가 상수(≤15)로 묶인다(별 수와 무관). 별이 단계를
   넘으면 해당 버킷으로 이동한다. 수천 별에서 프레임 회귀가 없다.
7. 미지/구 형태 id는 디폴트 룩(`polyhedron` 단계 0)으로 폴백하고 크래시하지 않는다.
8. FE `build`·`lint`·`steiger`·test 통과, 공유 우주(35/37)·근접 탐험(49)·데모(47/48) 회귀 없음.
