# 07. 변경: 기억 가중치(Bjork)로 빚는 배경 — 무드 오브 제거·감정색을 배경에 녹임

> [plan/25.ambient-mood](../plan/25.ambient-mood.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 별도로 떠 있는 투박한 "무드 오브"(`AmbientNebula`)를 없애고, **하나의 과학적 기억 가중치**(Bjork 저장/인출 강도)로
> (a) 별의 자기-중심 거리와 (b) 감정 순위를 함께 정한 뒤, 그 순위를 **배경 스킨이 직접 짜 넣는** 모델로 재구성한다.
> plan 25(요즘 배경)를 주로 고치며 **plan 44(배경 축)·38(자기근접 반지름)을 함께 수정**한다(아래 회귀 경계).

## 현재 (as-is)

지금 "요즘 감정"과 "강한 기억"은 **세 갈래로 흩어진 가중치**로 따로 계산되고, 배경 위에 별도 자산으로 떠 있다.

- **세 가중치(파편화).**
  - **자기근접(38)**: `strength = W_ACT·activation + W_INT·intensity`(`w_activation 0.7`·`w_intensity 0.3`), `activation = exp(−ln2/30·Δt)`(망각 곡선, 반감기 30일). `targetRadius(strength)`로 별의 중심 거리(반지름)를 정한다(강함=중앙).
  - **감정 순위/무드(25)**: `w = intensity·exp(−Δt/τ)`(`tau_mood_days 7`). `AggregateAmbient`가 `{hue,sat,arousal,valence}` 한 묶음으로 종합하고, `ambientLights`가 상위 K(`lights_k 6`)개 우세 mood를 가산광 풀로 흩뿌린다.
  - **밝기 감쇠(26)**: `λ_eff`(연결·관련성·감정 변조) — self-glow 채널 감쇠. (이 변경의 직접 대상 아님.)
- **빈도(frequency)는 어디에도 안 쌓인다.** 회상은 `RecallMemoryTouch`로 `last_recalled_at`만 `now()`로 재설정한다. **회상 횟수 컬럼이 없다** — "여러 번 떠올린 기억일수록 오래 간다"(공고화)가 모델에 없다.
- **무드 오브(`AmbientNebula`, 25).** `UniverseCanvas`(:1706)가 항상 `lights_k 6`개 가산 빌보드를 그린다. 풀 색 = **고정 의미 팔레트 `moodRgb`**(valence 보정) — **사용자가 고른 감정색(plan 45)을 무시한다.** 밝기는 `arousal`(`base_bright 0.06 + arousal_bright 0.2·arousal`). 별 셸 밖 반경 120~200에 떠 있는 "투박한 구형 그라디언트".
- **배경 스킨(`UniverseNebula`, 44).** 같은 캔버스에 도메인워프 fbm 성운 워시를 한 겹 더 깐다. 색은 선택한 배경(Background) 번들의 fluid 팔레트(`backgrounds.ts` palette). **모든 배경이 같은 fbm 텍스처를 쓰고 팔레트(색)로만 구분**된다(`aurora-veil`만 얇은 veil 한 겹). 감정색은 이 워시에 안 들어간다.
- **서버 권위 종합(25).** 서버가 `GetUniverse`에서 7일 윈도로 `AggregateAmbient` → `AmbientMood{hue,sat,arousal,valence}`를 응답 `ambient`에 싣고, 클라는 미수신 시 `deriveAmbient`/`ambientLights`로 폴백.
- **랜딩(25).** `AmbientMoodCard`가 위 무드 오브의 2D DOM 대응(radial-gradient orb 크로스페이드)을 보여준다.

요컨대: 색은 고정 팔레트, 무드는 별개 오브, 가중치는 세 갈래. "받쳐주는 색감 + 무늬/질감 + 사용자 감정색"이라는 결이 없다.

## 목표 (to-be)

### 1. 하나의 과학적 기억 가중치 — Bjork 저장/인출 강도

세 갈래 대신 **단일 인출 강도 R**을 정의해 자기근접과 감정 순위를 함께 구동한다. (근거: Bjork & Bjork *New Theory of Disuse* — 저장강도·인출강도 분리, 분산효과·정서 공고화.)

- **저장강도 S — 누적·단조 비감소.** 기억은 처음 부호화될 때 기본 강도로 시작하고, **회상할 때마다 누적**된다. **정서 강도(intensity)가 회상당 공고화 증분을 키운다**(정서적 기억일수록 깊이 새겨짐). S는 절대 줄지 않는다.
  - 형태(계수는 values): `S = (1 + recall_count) · (1 + emo_consolidation · intensity)` 류의 누적값. `recall_count`는 새 영속 신호.
- **인출 강도 R — 현재 접근성, 시간 감쇠.** `R = exp(−Δt / τ(S))`, `Δt` = `last_recalled_at` 이후 경과(일), `τ(S)`는 **S가 클수록 길어진다**(분산효과/멱법칙: 잘 공고화된 기억은 천천히 잊힌다). `τ(S) = tau0_days · (1 + tau_storage_gain · ln(1+S))` 류. R ∈ (0,1], 막 회상하면 R≈1.
- **R이 곧 가중치.** 
  - **자기근접(38 재배선)**: `targetRadius`가 옛 `strength(0.7·act+0.3·int)` 대신 **R**을 쓴다 — `targetRadius = R_MIN + (R_MAX−R_MIN)·(1−R)`. "거리=강함"은 보존(강할수록 중앙). **각도(방향)는 여전히 그래프**, 반지름만 R로. 회상→중앙 당김, 망각→바깥, **자주 떠올린 기억은 더 오래 중앙에 머문다**(τ↑).
  - **감정 순위(배경)**: mood별 `Σ R`(해당 감정 별들의 R 합)로 감정 순위·비중을 매긴다. 옛 `intensity·exp(−Δt/7일)` 종합을 대체.
- **계산 위치 = 클라(헌법3).** R·S·순위는 순수 클라 모듈에서 계산한다(좌표·렌더 입력은 클라 권위). 서버는 **원자료 `recall_count`만 영속·전달**한다(`relevance`(26)와 동급의 서버 권위 원자료 — `Star`에 append). `last_recalled_at`/`intensity`/`mood`는 이미 별에 있다.

### 2. 배경이 감정색을 짜 넣는다 — 무드 오브 제거

- **`AmbientNebula` 완전 제거.** 떠 있는 가산광 풀(투박한 구형 그라디언트)을 없앤다. "요즘 감정"은 이제 **배경 스킨의 텍스처/톤에 녹는다**(별개 오브 아님).
- **배경 스킨 = 받침색 + 무늬/질감 + 감정색 슬롯 N.** 배경 아이템(44 Background 축)이 다음을 정의한다:
  - **받침색(base/supporting color)** — 뒤를 받치는 깊은 색(예: 검정, 깊은 인디고). **원색이 아니라 절제된 색감 위주**.
  - **무늬/질감(texture/pattern)** — 스킨마다 **다른** fbm/패턴 결(지금처럼 색만 다른 게 아니라 무늬 자체가 다르다).
  - **감정색 주입 슬롯 N(`emotionSlots`)** — 이 스킨이 **상위 몇 개 감정색을 짜 넣는가**(스킨마다 고정, 카탈로그 정의). 예: `1`(주요 감정 1개만), `3`(상위 3개), `13`(비중대로 전부), 심지어 `0`(감정 무관 순수 텍스처)도 가능. **창의적·축을 살림.**
- **주입되는 색 = 사용자 감정색(plan 45).** 고정 `moodRgb`가 아니라 `resolveMoodRgb(mood, emotionColors)` — 사용자가 확정한 13감정 색이 배경에 녹는다(45와 일관·개인화, 기존 드리프트 해소). 순위 상위 N개 mood의 (색, R-비중)을 스킨 텍스처에 가중 합성한다.
- **arousal 유지·재배선.** `arousal`(이제 클라가 `Σ R`에서 도출, 예 `1−exp(−ΣR)`)은 **배경 스킨의 전역 생동**(밝기·움직임·휘젓는 세기)을 정한다. 격동한 요즘=생동하는 배경, 평온=잔잔. **22 흥분성 게인 seam(`ExcitabilityGain = 1 + 0.3·arousal`)은 헬퍼로 보존**(현재처럼 라이브 미배선, 27 야간 seam으로 흘려보냄).

### 3. 계약 정리

- **proto `Star` += `recall_count`**(append, 서버 권위 원자료). 클라가 S/R 계산.
- **`GetUniverseResponse.ambient`(AmbientMood) 은퇴.** 클라가 로드된 별(이제 `recall_count` 포함)에서 순위·arousal을 전부 파생하므로 서버 종합이 불필요. 필드 번호는 **reserved**(재사용 금지). 서버측 `AggregateAmbient`/`ListRecentForAmbient`/ambient 합성 제거(단순화). `ExcitabilityGain` 헬퍼는 seam으로 잔존(필요 시 arousal을 회상 데이터에서 재도출).

**비목표(절대 안 바꾼다 = 회귀 경계)**
- **별 색 = 감정색 불변.** 배경이 같은 감정색을 빌려 쓰되, **별 자체의 색 규칙(`resolveMoodRgb`, 색=감정)은 그대로**(star.md, 45). 모양/배경만 변한다(44 "색은 안 판다").
- **26 밝기/`λ_eff`·self-glow 모델 불변.** R은 **반지름·배경 순위만** 구동한다 — 별 밝기 감쇠는 26 그대로.
- **spec 03 별빛 3채널 불변.** 자기 별이 다른 별에 **던지는 빛(반사 채널)은 중립색** 유지, 자아 몸체 색=ambient 파생(44 J3)도 그대로. mood 이중 주입 금지.
- **헌법2(삭제 금지)·헌법3(좌표·렌더=클라 권위, 서버는 그래프/원자료)·헌법6(unary)·헌법8(InstancedMesh/TSL/Line2)** 불변.
- **별 각도(방향) 안정** — R은 반지름만 바꾼다(38 "각도 안정, 반지름 변동" 정신 유지). 표상 부동(밤 1스텝 표류)도 유지.
- **44 소유권/별가루/4축·45 감정색 온보딩 게이트·29 13감정 enum** 불변. 배경은 여전히 4축의 한 축(아이템 = `background:<kind>`)이고, `emotionSlots`는 시각 정의(코드 카탈로그)에 속한다(가격은 values 그대로).
- 35/36 공유·선물 우주의 owner Settings 스냅샷 모델 불변(배경 스킨 정의만 풍부해질 뿐).

## 수용 기준 (인수 조건)

1. **빈도 영속.** `memories.recall_count`가 생기고 회상(`RecallMemoryTouch`)마다 1 증가하며, `Star`(proto) payload로 클라에 전달된다. 기존 별은 합리적 기본값(예 1)으로 백필된다.
2. **R 공식.** 막 회상한 별은 R≈1(중앙 근처), 오래 휴면한 별은 R→낮음(외곽). **회상 횟수가 많은 별은 한 번만 회상한 별보다 같은 Δt에서 천천히 잊힌다**(τ가 저장강도 S로 길어짐). R·S는 순수 클라 모듈에서 계산된다(three/React/DOM 미의존).
3. **자기근접 = R.** `targetRadius`가 옛 `strength(0.7·act+0.3·int)` 대신 R을 쓴다. "거리=강함"(강할수록 중앙)이 보존되고, **별의 각도(방향)·force-sim 파라미터·표상 부동은 바뀌지 않는다**.
4. **배경이 감정색을 짜 넣는다.** 배경 스킨이 순위 상위 N개(스킨별 `emotionSlots`) 감정색을 **사용자 감정색(45)**·R-비중으로 텍스처에 합성한다. `emotionSlots:1` 스킨은 주요 감정 1색만, `13`(또는 전부) 스킨은 비중대로 다색을 녹인다. `0`이면 감정 무관 순수 텍스처.
5. **무드 오브 제거.** `AmbientNebula`(떠 있는 가산광 풀)가 제거되어 어떤 화면에도 투박한 구형 그라디언트가 없다. 요즘 감정은 배경 텍스처/톤으로만 드러난다.
6. **배경 차별화.** 배경 아이템들이 **색만이 아니라 받침색 + 무늬/질감**으로 갈린다(같은 fbm·색만 다른 상태가 아니다). 색감은 원색이 아닌 절제된 톤 위주다.
7. **arousal 재배선.** arousal이 `Σ R`에서 도출되어 배경 스킨의 전역 생동(밝기·움직임)을 정한다. 22 흥분성 게인 seam(`1 + 0.3·arousal`)은 헬퍼로 보존된다.
8. **계약 정리.** 서버는 `GetUniverse`에서 `AmbientMood`를 더는 싣지 않고(필드 reserved), 클라가 별(+`recall_count`)에서 순위·arousal을 파생한다. 서버 ambient 종합 코드가 제거되어도 코어 그래프 조회는 회귀 없이 동작한다.
9. **색·밝기 회귀 없음.** 별 색은 여전히 감정색(`resolveMoodRgb`)이고, 26 밝기 감쇠·self-glow·spec 03 반사 중립이 그대로다(이 변경이 별색/밝기를 안 건드림).
10. **데모·랜딩 재조정.** 데모 우주가 같은 R-순위·배경 짜임을 미러하고, 랜딩 `AmbientMoodCard`가 옛 오브가 아닌 새 배경-짜임 모델을 반영한다(또는 배경 카드로 흡수). 38 레이아웃 읽기·44/45 동작에 회귀가 없다.
11. **값 단일 출처.** 신규 가중치 계수(`tau0_days`·`tau_storage_gain`·`emo_consolidation` 등)와 arousal·배경 생동 계수가 `spec/values.yaml`에서 생성된 상수로 FE/BE 공유된다(하드코딩·이중 출처 없음). `emotionSlots`(스킨별 N)는 코드 카탈로그(시각 정의)에 둔다.
12. build·lint·vet·typecheck·steiger 통과, `prefers-reduced-motion`에서 배경 모션 정지(색 유지), 우주 셸·기존 화면 회귀 없음.
