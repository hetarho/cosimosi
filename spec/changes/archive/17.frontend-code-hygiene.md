# 17. 변경: frontend code hygiene

> [plan/42.star-visual-primitive](../../plan/42.star-visual-primitive.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> /create-change가 사용자 인터뷰로 채운다 — 자연어 한 줄이 아니라 검토 가능한 문서. STEPS는 /create-change-job이 만드는 job에.

## 현재 (as-is)
`plan/42`는 TSL 별 프리미티브를 "입력 바인딩과 분리된 단일 시각 프리미티브"로 정리했고, 현재 구현은 `entities/star/ui/star-body.ts`의 `buildStarBody`가 그 핵심이다. 이후 배경/시냅스/공유 씬에도 같은 TSL·R3F 관용구가 퍼졌지만, 타입 경계와 설명 주석은 앱 전반에서 아직 정리되지 않았다.

전수 스캔 범위는 `frontend/src`의 non-generated `*.ts`/`*.tsx` 303개 파일이다(`frontend/src/shared/api/gen/**`, `*.gen.ts`, 빌드 산출물 제외).

- **타입 assertion 남발:** TypeScript AST 기준 type assertion 192건/63파일. 이 중 `as never`가 54건이며 대부분 TSL node 변환 경계에 집중되어 있다.
  - `frontend/src/entities/star/ui/star-body.ts`: 17건 전부 `as never`.
  - `frontend/src/entities/synapse/ui/SynapseFilaments.tsx`: 11건 중 `as never` 6건 + `Float32Array` buffer cast 반복.
  - `frontend/src/entities/appearance/ui/self-form.ts`: 8건 전부 `as never`.
  - `frontend/src/shared/ui/fluid-material.ts`: 8건 전부 `as never`.
  - `frontend/src/widgets/universe-canvas/ui/UniverseNebula.tsx`: 8건 전부 `as never`.
  - `frontend/src/widgets/cosmos-scene/ui/CosmosScene.tsx`, `UniverseCanvas.tsx`, `UniverseOverlay.tsx`, `BloomPass.tsx`: `as unknown as GLProps`/`WebGPURenderer`류 R3F renderer cast 반복.
  - `appearance/model/store.ts`, `AppearanceSwitcher.tsx`, `shared/config/mood.ts`, `demo/flag.ts`: 문자열 id를 `Set.has`로 확인한 뒤 같은 표현을 다시 `as X`로 캐스팅하는 패턴이 반복된다.
- **주석 과다/불필요 주석:** generated 제외 후 주석 후보 89파일. raw count만으로 삭제 대상을 정하지는 않지만, 다음 파일들은 긴 블록·인라인 설명·change/spec 이력성 문구가 눈에 띄게 많아 수동 정리 대상이다.
  - 렌더링/항해: `StarField.tsx`(158 comment lines), `controllers/navigation.tsx`(142), `controllers/flights.tsx`(129), `SynapseFilaments.tsx`(121), `UniverseCanvas.tsx`(102), `UniverseNebula.tsx`(59), `star-body.ts`(53), `fluid-material.ts`(44).
  - 모델/도메인: `memory/model/activation.ts`(58), `memory/model/merge.ts`(44), `memory/model/ambient.ts`(38), `memory/api/universe-query.ts`(36), `shared/lib/layout.ts`(35).
  - 라우팅/데모: `app/router.tsx`(57), `shared/lib/demo/data.ts`(136), `shared/lib/demo/simulate.ts`(52), `widgets/demo-tour/model/tour.machine.ts`(44).

현 상태의 문제는 "주석이 많다" 자체가 아니라, 보존 가치가 있는 설계/불변식 주석과 한때의 변경 이력·코드 반복 설명·수치 조정 내역이 섞여 유지보수자가 중요한 경고를 찾기 어렵다는 점이다. `as never`도 런타임 버그라기보다 Three TSL 타입 정의와 실제 DSL 사이의 마찰을 파일마다 직접 노출한 상태다.

## 목표 (to-be)
프론트엔드 코드의 타입 경계와 주석 정책을 정리한다.

- TSL/R3F 타입 경계는 파일마다 `as never`/`as unknown as ...`를 흩뿌리지 않고, 작고 이름 있는 helper/API 뒤로 숨긴다.
- domain id/string normalization은 검증 함수·normalizer가 타입 narrowing을 책임지게 하고, 같은 값에 대한 중복 `as X`를 제거한다.
- 유지할 주석은 "코드만 봐서는 알 수 없는 불변식/브라우저·R3F·TSL 제약/회귀 방지 이유"로 좁힌다.
- 제거하거나 축약할 주석은 "코드가 이미 말하는 내용", "과거 change/spec 이력의 장문 복붙", "수치가 과거에 몇에서 몇으로 바뀌었는지 같은 구현 일지"다.
- 기능·시각 룩·도메인 동작은 그대로 유지한다. 이 변경은 behavior change가 아니라 maintainability change다.

## 범위 / 비목표 (회귀 경계)
**범위**

- `frontend/src`의 non-generated TypeScript/TSX.
- TSL node conversion helper 정리: `float(uTime as never)`, `vec3(uniform(Color) as never)`, `attribute(...) as never`, `dot(...) as never` 같은 패턴을 중앙화한다.
- R3F/WebGPU renderer helper 정리: `as unknown as GLProps`, `as unknown as WebGPURenderer`, `useThree((s) => s.controls) as ...` 반복을 이름 있는 helper/hook으로 묶는다.
- storage/API/settings 문자열 id normalization 정리: `Set.has(value as Type) ? value as Type : fallback` 같은 중복 cast를 local type guard 또는 parser로 바꾼다.
- 주석 정리 수동 패스: 위 as-is 후보를 우선순위로 하되, 삭제보다 "보존할 이유가 있는 주석만 남기기"를 기준으로 한다.

**비목표**

- generated 파일(`frontend/src/shared/api/gen/**`, `values.gen.ts`)은 직접 수정하지 않는다.
- `as const`는 이번 문제 범위가 아니다. tuple/object literal inference를 위한 정상 사용은 유지한다.
- 테스트에서 mock shape를 만들기 위한 국소 `as unknown as ExternalType`은 helper화 가치가 없으면 예외로 둘 수 있다. 단, 예외는 job에서 명시한다.
- CSS/className·카피·UI 재디자인, Three 셰이더 수치 튜닝, 카메라/항해 동작 변경은 하지 않는다.
- 새 values 수치 추가는 원칙적으로 없다.

## 수용 기준 (인수 조건)
1. `frontend/src` non-generated 파일 기준 `as never`가 렌더/TSL 소비 파일에 직접 남지 않는다. 필요한 캐스트는 이름 있는 TSL helper 모듈 한 곳에 격리되고, 그 helper가 왜 필요한지 짧게 설명한다.
2. `star-body.ts`, `SynapseFilaments.tsx`, `SynapseDust.tsx`, `self-form.ts`, `fluid-material.ts`, `UniverseNebula.tsx`, `CosmosScene.tsx`, `halo.ts`는 새 helper를 사용해 TSL node conversion을 표현한다.
3. R3F/WebGPU renderer 관련 `as unknown as GLProps`/`WebGPURenderer`/controls shape 반복은 공용 helper 또는 hook으로 정리되어, `UniverseCanvas.tsx`, `UniverseOverlay.tsx`, `CosmosScene.tsx`, `BloomPass.tsx`, `controllers/navigation.tsx`, `controllers/flights.tsx`에서 같은 타입 assertion 문장이 반복되지 않는다.
4. catalog/settings/demo id 검증 코드(`appearance/model/store.ts`, `AppearanceSwitcher.tsx`, `shared/config/mood.ts`, `shared/lib/demo/flag.ts`, `entities/synapse/model/store.ts`)는 type guard/parser를 통해 narrowing한다. 검증 전후 같은 식에 `as X`를 두 번 쓰는 패턴이 남지 않는다.
5. `as const`와 명시된 테스트/mock 예외를 제외하고, 새로 추가되는 type assertion은 helper/guard 내부 또는 경계 모듈에만 존재한다. job은 `rg " as " frontend/src -g "*.ts" -g "*.tsx" -g "!*.gen.ts"` 결과를 검토해 예외 목록을 남긴다.
6. 주석 정리는 최소한 다음 렌더링 hotspot을 수동 검토한다: `StarField.tsx`, `controllers/navigation.tsx`, `controllers/flights.tsx`, `SynapseFilaments.tsx`, `UniverseCanvas.tsx`, `UniverseNebula.tsx`, `star-body.ts`, `fluid-material.ts`.
7. 주석 정리 후에도 회귀 방지에 필요한 설명은 유지된다: frozen-time `uTime`, BackSide/depth/renderOrder/raycast 없음, R3F/WebGPU renderer 제약, FSD import 경계, 보안/세션/쓰기 게이트, 도메인 수식의 출처.
8. change/spec 번호나 과거 수치 변경 내역을 설명하는 장문 주석은 코드에서 제거하거나 한 문장 이하의 현재형 의도 설명으로 축약한다. 필요한 이력은 spec/change/job 문서에 남긴다.
9. 기능 동작과 시각 룩은 변경하지 않는다. 특히 우주 렌더링, 별/시냅스 모양, 항해/카메라, demo/tour, settings normalization의 사용자-facing 결과가 기존과 같다.
10. 검증: `pnpm --filter ./frontend lint`, `pnpm --filter ./frontend build`, 관련 테스트(`appearance`, `synapse`, `demo flag`, `navigation`/`tour` 중 변경 파일에 닿는 것)를 실행한다. 타입 helper 변경으로 TSL 렌더가 깨지지 않는지 최소 smoke 확인을 남긴다.
