# 06. 변경: 외형 편집기 모달 — 고정 프리뷰 + 탭 + 실모형 썸네일

> [plan/44.customization-system](../plan/44.customization-system.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 미리보기가 없어 선택이 어렵다는 문제를 해결한다 — 에디터를 모달로 재구성하고, 항목을 의미 약한 그라디언트가 아니라 **실제 모형**으로 보여준다. STEPS는 /create-change-job이 만드는 job에.

## 현재 (as-is)
<!-- plan/44.customization-system 중 이 변경이 건드리는 현재 동작. 출발점. -->
- 홈 외형 편집은 `AppearanceControls`(features/switch-appearance)를 **좌상단 별가루·테마 알약**(또는 메뉴)이 여는 **비차단 Surface**(place="center", width="sm")로 띄운다. 4축(배경·별·나·시냅스) 라디오그룹이 세로로 쌓인다.
- 각 항목 칩은 **CSS 그라디언트 swatch**(`BackgroundMeta.swatch`·`STAR_OBJECTS.swatch`·`SELF_OBJECTS.swatch`·`SYNAPSE_STYLES.swatch`)로만 표현된다 — 실제 3D 모형이 아니라 의미 약한 색 견본이라, 고른 게 우주에서 어떻게 보일지 알기 어렵다.
- 별도 미리보기 영역이 없다. 선택을 바꾸면 뒤 우주(`UniverseCanvas`)가 라이브로 바뀌지만 Surface가 작고 4축을 한눈에 비교하기 어렵다.
- 드래프트→저장 모델(이미 구현): 홈은 선택을 라이브 미리보기(드래프트)로 두고 `AppearanceSaveBar`(플로팅)가 커밋한다(미구매 유료 아이템 일괄 구매·잔액 가드·되돌리기). 잠긴 항목은 자물쇠 + `저장 시 N 별가루`. 체험(데모)은 전부 잠금 해제·무상 저장.
- 재사용 가능한 합성 위젯 `CosmosScene`(widgets/cosmos-scene)이 이미 존재: `stars`/`self`/`synapses`/`palette`/`texture` prop으로 나 앵커 + 별 표본 + 시냅스 표본을 **한 WebGPU 캔버스**에 그린다(사인인·랜딩·초대 플레이그라운드에서 사용). 형태 빌더 `buildStarBody`·`buildSelfForm`·`SynapseFilaments`/`SampleStrand`가 단일 출처.

## 목표 (to-be)
<!-- 변경 후 참이어야 하는 동작·계약. 구체적으로. -->
- 좌상단 알약을 누르면 **집중 모달**(화면을 덮는 오버레이)이 열린다. 비차단 Surface 대신, 모달 내부의 고정 프리뷰가 "내 우주" 역할을 해 편집에 집중하게 한다. 구조:
  - **[고정 라이브 미리보기 영역]**(상단 고정): `CosmosScene` 재사용 — **나(self) + 시냅스로 연결된 별 3개**, 시냅스는 **강한 연결·약한 연결이 섞이게**(weight 차등) 구성한다. 현재 드래프트 선택(4축)을 라이브 반영해, 탭/항목을 바꿀 때 변화를 한눈에 본다.
  - **탭**: `배경 / 별 / 광원 / 시냅스` 4개. **광원 탭 = 나(self) 축**(중심 자아 별 = 우주의 광원). 탭 선택 시 해당 축의 항목 리스트로 전환.
  - **스크롤 가능한 항목 리스트**: 각 항목을 **실제 모형 썸네일**로 보여준다(그라디언트 swatch 폐기). 별 = 별 형태, 광원 = 자아 형태, 시냅스 = 표본 가닥, 배경 = 백드롭(성운/베일) 룩.
- **항목 썸네일 = 사전 렌더(오프스크린) 이미지·캐시**: 각 모형을 **공유 렌더러로 한 번 렌더 → 이미지로 캐시**한 뒤 칩에 `<img>`로 표시(정지 이미지, 회전 없음). 같은 형태는 캐시 재사용. **칩마다 라이브 WebGPU 캔버스를 만들지 않는다** — 모달이 더하는 GPU 컨텍스트는 고정 프리뷰 1개(+ 썸네일 렌더용 1개)로 제한해 모바일/성능 안전.
- 잠금/가격/드래프트·저장은 **그대로 유지**: 잠긴 유료 항목은 자물쇠 + 가격, 선택은 드래프트 미리보기, `AppearanceSaveBar`가 커밋(미구매분 일괄 구매·잔액 가드). 체험(데모)은 전부 잠금 해제·무상 "저장".

## 범위 / 비목표 (회귀 경계)
<!-- 무엇을 바꾸고, 무엇은 안 바꾸나. -->
- **범위**: `features/switch-appearance`의 에디터 UI를 **모달 + 고정 프리뷰 + 탭 + 실모형 썸네일 리스트**로 재구성. 항목 썸네일 **사전 렌더 파이프라인**(오프스크린 1회 렌더 → 이미지 캐시) 추가. 고정 프리뷰는 `CosmosScene` 재사용(나 + 별 3개 + 강·약 시냅스 표본 구성). plan 44 §F(스위처 UI) 갱신.
- **비목표 — 이번 변경 아님**: **색 체계 재설계**(색=주 감정 최대 4색 / 배경=모양 전환 / 투박한 `AmbientNebula` mood 풀 제거)는 **별도 후속 change**로 분리한다. 이번 모달은 *현재의 4축·현재의 색 규율* 위에서 표현만 바꾼다.
- **불변(회귀 경계)**: 별가루 경제·소유권·구매 검증·서버 계약(proto/RPC/DB migration)·드래프트→저장 모델·잠금 강제 위치(저장 시점 + 서버)·렌더 폴백은 **안 바꾼다**. 4축 색 규율(별=mood·시냅스=양끝 mood 블렌드·나=ambient 파생·배경=자기 색)·삭제금지(헌법2) 등 8 불변 원칙 유지. 플레이그라운드 FAB(랜딩/사인인/초대)·미니 코스모스 어댑터 동작 유지(이 모달은 홈 전용; FAB에 동일 모달을 재사용할지는 job 재량, 단 미인증=전부 잠금 해제·미저장 유지).

## 수용 기준 (인수 조건)
<!-- 변경 완료의 판정 기준. job의 "인수 조건" 섹션으로 복사된다. -->
1. 좌상단 별가루·테마 알약을 누르면 **집중 모달**이 열리고, 상단 **고정 프리뷰**에 나 + 별 3개가 **시냅스로 연결(강한 연결·약한 연결 혼합)** 되어 현재 선택(4축)으로 렌더된다.
2. 모달에 `배경 / 별 / 광원 / 시냅스` 4개 탭이 있고, 탭 전환 시 해당 축의 항목 리스트가 **스크롤**된다. 광원 탭은 나(self) 축을 조정한다.
3. 항목 칩이 그라디언트 swatch가 아니라 **실제 모형 썸네일**(사전 렌더 이미지)로 표시된다 — 별/광원/시냅스는 형태, 배경은 백드롭 룩.
4. 항목을 고르면 고정 프리뷰가 즉시 그 선택을 반영하고(라이브 드래프트), 저장 시 우주에 반영된다. **드래프트→저장(미구매분 일괄 구매·잔액 부족 시 막힘)·되돌리기**가 기존대로 동작한다.
5. 잠긴 유료 항목은 자물쇠 + `저장 시 N 별가루`로 표시되고, 체험(데모)에선 전부 잠금 해제·라벨 `저장`(무상)이다.
6. 항목 썸네일은 **사전 렌더 1회 + 이미지 캐시**로 그려진다 — 칩마다 라이브 WebGPU 캔버스를 만들지 않는다(모달이 더하는 GPU 컨텍스트는 고정 프리뷰 + 썸네일 렌더로 제한).
7. 모달을 닫아도 드래프트·`AppearanceSaveBar` 상태가 기존 규칙대로 유지·동작한다(저장/되돌리기 회귀 없음).
