# 14. 변경: 죽은 코드·미사용 정리 + 구조 리팩터

> [plan/06.universe-canvas](../plan/06.universe-canvas.md)를 1차 앵커로 두되, MVP를 거치며 누적된 **죽은 코드·미사용 표면·중복·god-file을 코드베이스 전반에서 정리**하는 횡단 변경이다. 동작은 바꾸지 않는다 — 순수 삭제·이관·분해.

## 현재 (as-is)

MVP를 지나며 은퇴했으나 코드에 남은 잔여물이 누적돼 있다(정적분석 knip + 사용처 적대적 검증으로 확인).

- **죽은 파일·피처** — `features/dormant-search`(3파일, change 09에서 은퇴·star-explorer가 대체)·`features/universe` 셸 스토어(3파일, HomePage가 로컬 state로 재구현)·`shared/ui/OverlayHost`(현 셸은 Surface)·`entities/synapse/ui/SynapseLines`(SynapseFilaments로 대체)·`entities/appearance/model/themes.ts`(`@deprecated` shim)·`shared/lib/demo/observe.ts`·`shared/lib/r3f` 추상 렌더러 타입.
- **죽은 BE 코드** — `internal/ai/seed.go`(`SegmentSeed`/`normalizeText`, prod 호출 0 — 실 시드는 `memory/service.go:directionFor`)·`internal/job/worker.go:buildLinks`(prod는 `biasedLinks`)·`admin.Repository.UserExists`(prod는 unexported `r.userExists`).
- **미사용 의존성** — `@tanstack/react-virtual`·`class-variance-authority`·`date-fns`·(devDep)`@tanstack/router-plugin`.
- **미사용 barrel/export** — FSD `index.ts` re-export 다수가 소비자 0(`selectIsOverlay`·`selectIsFramingPair`·spec-26 decay 상수 barrel·appearance barrel 9개 등). 정의는 살아있고 re-export 라인만 죽은 경우가 대부분.
- **중복** — `newID()`가 6패키지 복붙·`moodToProto/From`이 3핸들러 복붙·"missing auth user" 가드가 ~21곳 인라인·nullable pgtype 매퍼가 5패키지 재선언·FE FNV-1a 해시가 SynapseFilaments/SynapseDust 2벌·13-mood quadrant 배열이 2벌.
- **god-file** — `widgets/universe-canvas/ui/UniverseCanvas.tsx`(2062L)·`pages/home/ui/HomePage.tsx`(1055L)·`internal/job/worker.go`(613L)에 다수 책임이 섞여 있다.

## 목표 (to-be)

- knip이 **미사용 파일·export·의존성 0**을 보고한다(생성코드 `*/gen/`·`*.gen.ts`·`*_pb.ts`, 외부 CLI가 소비하는 `steiger.config.ts`·`vite.config` 등은 명시적 예외).
- 위 죽은 BE 코드·중복 헬퍼가 제거되거나 단일 출처로 수렴한다.
- god-file 3개가 책임 단위(컨트롤러·훅·순수 빌더)로 분해돼 각 파일이 한 가지를 한다.
- **동작·시각·카피는 한 글자도 안 바뀐다** — 빌드·테스트·lint가 그대로 green.

## 범위 / 비목표 (회귀 경계)

**범위 (삭제·이관·분해)**
- 죽은 파일·피처·BE 코드·미사용 의존성 삭제(현재 섹션 목록).
- barrel/export 정리 — re-export만 죽은 건 해당 라인만, 정의까지 죽은 건 정의째.
- 중복 수렴 — `newID`→`internal/platform/id`, `moodToProto/From`→단일 변환, auth 가드→`rpcserver` 헬퍼, pgtype 매퍼→`internal/db`, FE FNV→`synapse/lib`, mood quadrant→파생 상수.
- god-file 분해 — UniverseCanvas(상수→model, 컨트롤러→ui/controllers, NavController/LiveLayout 분리)·HomePage(useDemoFlow/useTutorialTour/useUniverseSurfaces 훅 추출)·worker.go(excitability/link 순수 헬퍼 분리).
- 삭제 대상 심볼을 인용하던 spec/policy/tech `.md` 라인 동기화(문서=규칙).

**비목표 (회귀 경계)**
- **inert seam 코드는 삭제 금지** — `memory.ExcitabilityGain`·재공고화 reshape/peGate·`ambient` 각성 게인 관련은 [change 15](15.wire-inert-core-logic.md)에서 **배선**하므로 보존한다. (`force-sim` Web Worker는 inline `createSim`이 대체하므로 삭제 대상이나, 15와 무관.)
- 살아있는데 미사용처럼 보이는 오탐 보존 — `R_MIN`/`R_MAX`·`A_MIN`·`CameraMode`·`dormantInvalidateKey`·`MOOD`/`MoodKey`·`selectFramingPair`·`UserEmailFromContext`·FE 전용 `values_gen.go` 튜닝값.
- 동작·시각·애니메이션·카피 변경 없음. 새 기능 없음. 튜닝값 변경 없음.
- 헌법 8 불변 원칙 유지.

## 수용 기준 (인수 조건)

1. `knip`이 미사용 파일·export·의존성 0 보고(예외 목록 외).
2. `eslint .` green, `pnpm --filter ./frontend build`·`vitest run` green.
3. BE `go build ./...`·`go vet ./...`·`go test ./...` green(`buildLinks` 제거에 따른 테스트 재작성 포함).
4. UniverseCanvas·HomePage·worker.go가 책임 단위로 분해되고, 분해 전후 **시각·동작 회귀 없음**(데모·실유니버스 수동 확인).
5. 중복 헬퍼(`newID` 등)가 단일 출처로 수렴하고 호출부가 전부 이관된다.
6. 삭제 심볼을 인용하던 `spec/**` `.md` 라인이 갱신돼 dangling 참조 0.
7. inert seam 코드(`ExcitabilityGain`·재공고화 reshape)가 보존된다([change 15] 입력).
