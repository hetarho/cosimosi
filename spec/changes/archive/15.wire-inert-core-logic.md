# 15. 변경: inert 핵심 로직 실배선 + SSOT 하드코딩 제거

> [plan/23.reconsolidation-reshaping](../plan/23.reconsolidation-reshaping.md)을 1차 앵커로 두되, **MVP 단계에서 골격만 두고 런타임 무효과로 남은 핵심 이론을 실제로 작동하게 배선**하고, 랜딩 카드의 SSOT 위협 하드코딩·거짓 메타데이터를 제거하는 변경이다. MVP는 끝났다 — 랜딩이 단언하는 모든 핵심 이론은 인증 우주에서 실제로 돌아야 한다.

## 현재 (as-is)

랜딩 신경과학 청구를 실코드로 대조하면, 12개 핵심 이론 중 9개는 실가동하나 둘은 골격만 있고 런타임 효과가 0이다(실가동 감사·적대적 재추적으로 확인).

- **재공고화(plan 23) — inert.** recall마다 `memory/service.go`가 reshape를 호출하지만, `internal/memory/repository_pg.go:335-336`이 `RecallEmbedding`·`ConsolidatedEmbedding`에 **동일 임베딩(`emb`)을 할당** → cos=1 → `pe=0` → `service.go:368 if pe < peThreshold { return nil }`에서 매 회상 early-return. 인증 우주에서 별 재성형·`evolution_history`(변천사) 기록이 **0건**이다. 랜딩 V장 "별은 회상마다 다시 빚어지고, 변천사가 그 길을 남겨요"는 단언형이나, 카드 애니메이션은 FE 시뮬레이션(`ReconsolidationCard.tsx`)뿐이다. helper·proto·SQL·마이그레이션(`00005`)·테스트 골격은 이미 존재한다.
- **각성 게인(plan 25) — inert.** `memory.ExcitabilityGain(arousal)=1+arousalGainCoef·arousal`이 `internal/memory/memory.go:232`에 정의됐으나 **호출 0건**. 경쟁 할당 점수 `worker.go:543 score = CosSim + wExc·norm_e`는 상수 `wExc(=values.ExcitabilityWExc)`만 쓴다(`worker.go:42-47` 주석이 미배선임을 명시). 랜딩 VII장 "격동한 요즘일수록 새 기억을 더 강하게 끌어당김"은 효과 0이다. (배경 빛 변조는 별개로 ✅live.)
- **SSOT 위협 하드코딩.** `AmbientMoodCard.tsx:96`이 계수 `0.3`을 인라인(`VALUES.ambient.arousalGain` 미사용) → 재튜닝 시 조용히 드리프트. `ReconsolidationCard`가 `PE_THRESHOLD=0.15`를 리터럴로 두고, 카드 magnitude/hue 공식이 BE와 다르다.
- **거짓 메타데이터.** `ResonanceSection.tsx:123` 배지가 `status="planned" plan="30"`(🚧 계획된 비전) — 실제 공명은 풀스택 출시됨(plan 36·37, plan 30은 무관). VIII장 footnote "흥분성을 0으로 되돌려요"는 해당 컬럼이 없다(tau=6h 자연감쇠 메타포).
- **MVP 게이트.** 회상 재점화는 DB에 즉시·권위적으로 반영되나, `MemoryPanel.tsx:104`의 `universeInvalidateKey()` 무효화가 `isDemoMode`로 게이트돼 **프로덕션 캔버스는 5m staleTime echo를 기다린다**.

> `peThreshold`·`wExc` 등 튜닝 상수 값 자체는 이미 `spec/values.yaml→gen`에서 온다 — 문제는 값이 아니라 **배선**과 **카드의 인라인 하드코딩**이다.

## 목표 (to-be)

- 랜딩이 단언하는 **모든 핵심 이론이 인증 우주에서 실제로 작동**한다. MVP 임시 no-op·seam이 남지 않는다.
- **재공고화 활성** — 야간 consolidation이 별의 공고화 임베딩 스냅샷을 별도 컬럼에 영속하고, `GetReshapeContext`가 `ConsolidatedEmbedding`으로 그 스냅샷을 쓴다. recall-context 임베딩 ≠ 마지막 공고화 임베딩이 되어 `pe>0`이 가능 → `peThreshold` 초과 시 별이 경계 안에서 재성형(brightness/hue/form_seed_delta, version++)되고 `evolution_history`에 append된다.
- **각성 게인 활성** — `worker.go` 경쟁 점수가 `wExc ← wExc·ExcitabilityGain(arousal)`로 스케일되고, `LoadExcitabilityInputs`가 사용자 단위 arousal을 동반 로드한다. 격동한 요즘일수록 새 기억이 활성 성단으로 더 강하게 끌린다.
- **카드 수치가 전부 SSOT에서 파생** — 인라인 하드코딩 0(`AmbientMoodCard`·`ReconsolidationCard`가 `values.gen.ts`에서 import). 카드 공식이 BE와 어긋나면 정렬하거나 "예시"로 명시한다.
- **메타데이터가 실제와 일치** — 공명 배지 `done`/plan 36·37, VIII footnote가 미존재 write를 암시하지 않는다.
- **회상 재점화가 프로덕션 캔버스에 즉시 반영**(`isDemoMode` 게이트 제거).

## 범위 / 비목표 (회귀 경계)

**범위**
- **BE 재공고화** — 마이그레이션으로 공고화 스냅샷 임베딩 컬럼 신설 → 야간 워커(`consolidate.go`)가 채움 → `GetReshapeContext`/관련 쿼리가 `ConsolidatedEmbedding`을 그 스냅샷으로 반환(현재 중복된 `emb` 대신). 게이트(`service.go:367-368`)는 무수정.
- **BE 각성 게인** — `worker.go`에서 `wExc` 스케일 + `LoadExcitabilityInputs`(`repository_pg.go`) arousal 동반 로드.
- **FE SSOT** — `AmbientMoodCard`·`ReconsolidationCard` 수치를 `values.gen.ts`/헬퍼에서 파생, 인라인 리터럴 제거.
- **FE 카피·메타** — `ResonanceSection` 배지 done/plan 36·37, VIII footnote 문구.
- **FE 캔버스** — `MemoryPanel`의 `universeInvalidateKey` 무효화에서 `isDemoMode` 게이트 제거.
- 새 튜닝값(있으면)은 `spec/values.yaml`에만 추가하고 gen으로 흘린다 — 코드 하드코딩 금지.

**비목표 (회귀 경계)**
- **헌법 §1 원본 불변** — `records`는 절대 안 바뀐다. 재공고화는 별(star) layer만 빚는다.
- 이미 ✅live인 9개 이론(엔그램·분할·시냅스·헵·능동인출·망각·침묵엔그램·야간공고화·공명)의 동작·수치 무변경.
- `peThreshold`·`baseStep`·`arousalGainCoef` 등 튜닝 **상수 값 자체는 안 바꾼다** — 배선만 한다.
- 죽은 코드 삭제는 [change 14](14.dead-code-and-refactor-cleanup.md) 소관.

## 수용 기준 (인수 조건)

1. 인증 우주에서 새 회상이 `pe ≥ peThreshold`이면 별이 재성형되고(brightness/hue/form_seed_delta 변동·version++) `evolution_history` 행이 기록된다 — `pe`는 더 이상 구조적 0이 아니다(통합테스트로 두 임베딩이 다름을 확인).
2. `arousal>0` 사용자의 새 기억 경쟁 점수에서 흥분성 항이 `g=ExcitabilityGain(arousal)`배로 스케일된다(arousal=0 대비, worker 테스트).
3. 랜딩 카드가 표시하는 모든 수치가 `values.gen.ts`/SSOT에서 온다 — grep으로 `AmbientMoodCard`의 `0.3`·`ReconsolidationCard`의 `0.15` 인라인 리터럴 0 확인.
4. `ResonanceSection` 배지 = `done`·plan 36·37. VIII footnote가 미존재 write를 암시하지 않는다.
5. 프로덕션에서 별 회상 직후 캔버스 밝기가 즉시 갱신된다(staleTime echo 대기 없음).
6. 9개 live 이론에 회귀 없음(기존 FE·BE 테스트 green), 원본 record 불변 유지.
7. `go build/vet/test ./...`·FE `build`·`vitest run`·`eslint .` green.
