# 08. 변경: 메인 우주 제스처 항행 UX 전면 재생성

> [plan/06.universe-canvas](../plan/06.universe-canvas.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 메인 우주 항행을 버튼 중심에서 **캔버스를 만져 우주를 보는 UX** 로 재구성한다.
> 사용자-facing 용어는 `성운/회상`을 버리고 **"멀리서 내 우주 보기" / "별들 가까이서 탐험하기"** 로 바꾼다.
> 가까이서 탐험할 때는 사용자가 곧 **이동하는 광원** 이 되고, 멀리서 볼 때는 회전 중심 고정 대신 pan 가능한 조망을 제공한다.

## 현재 (as-is)

현재 메인 우주 항행은 두 카메라 모드와 버튼 입력에 기대고 있다.

- **용어.** `HomePage` 상단 토글과 `plan/06`·`policy/domain/navigation`은 사용자-facing 이름으로 `성운`(전체 조망) / `회상`(근접 항해)을 쓴다. 코드 내부 상태도 `nebula` / `recall`이다.
- **근접 항해(`recall`).** `NavPad`가 화면 위 2D DOM 버튼으로 전진/후진과 상하좌우 시선 회전을 제공한다. 키보드는 `W/S` 전후진, `A/D` 좌우 yaw, `ArrowUp/Down` pitch를 `navigationActor.SET_MOVE`로 보낸다. `NavController`는 그 move 값을 매 프레임 읽어 추력, 관성, 시선 회전, 벽 반동을 적용한다.
- **근접 터치 제스처 부재.** `recall` 모드에서 캔버스를 드래그해 고개를 돌리거나, 두 손가락으로 전진/후진하는 입력은 없다. 모바일에서는 사실상 `NavPad` 버튼을 눌러야 한다.
- **원거리 조망(`nebula`).** `NebulaOrbitController`가 캔버스 포인터 이벤트를 직접 듣는다. 한 손가락/마우스 드래그는 커스텀 arcball 회전이고, wheel 및 두 손가락 pinch는 dolly zoom이다. `controls.target`을 중심으로 orbit하므로 조망의 기준점이 사실상 중앙 또는 현재 focus target에 묶인다.
- **원거리 pan 부재.** `NebulaOrbitController` 주석과 구현 모두 pan을 의도적으로 제외한다. 두 손가락을 같은 방향으로 움직여도 화면을 지도처럼 이동할 수 없다.
- **탭/드래그 충돌 회피.** 현재 원거리 조망은 포인터 캡처와 8px deadzone으로 탭(별 선택)과 회전 드래그를 나눈다. 별 `onClick`은 R3F delta가 작은 탭만 통과한다.
- **광원.** spec 03 이후 별 반사는 진짜 `THREE.PointLight`가 아니라 `StarField`의 `selfLightPos` uniform과 TSL 그래프에서 계산한다. 메인 `UniverseCanvas`는 현재 `selfLightPos` 기본값을 원점으로 쓰며, 중심 `SelfStar`가 자아-광원의 시각 표상이다. 즉 근접 항해로 카메라가 움직여도 별을 비추는 광원은 사용자를 따라오지 않는다.
- **웹 구현 가능성.** 브라우저 표준은 이 변경을 구현할 수 있는 기반을 이미 제공한다. Pointer Events는 마우스/펜/터치를 하나의 이벤트 모델로 다루고, pointer capture로 손가락이 캔버스 밖으로 나가도 up/cancel을 회수할 수 있다. 커스텀 게임/지도 표면은 `touch-action: none`으로 브라우저 기본 스크롤·핀치와 충돌을 줄이는 것이 정석이다. Touch Events는 멀티터치를 표현하지만 Baseline이 아니므로, 구현 주력은 Pointer Events가 맞다.

## 목표 (to-be)

### 1. 사용자 용어 전면 교체

사용자에게 보이는 카메라 모드 이름은 다음 두 개만 쓴다.

| 기존 | 새 사용자-facing 이름 | 의미 |
|---|---|---|
| `성운`, `nebula`, 전체 조망 | **멀리서 내 우주 보기** | 우주 전체를 밖에서 둘러보고, 회전·pan·zoom으로 풍경을 잡는 모드 |
| `회상`, `recall`, 근접 항해 | **별들 가까이서 탐험하기** | 별 사이로 들어가 사용자가 광원이 되어 직접 탐험하는 모드 |

- UI copy, `title`, aria-label, overlay 안내, policy 문서의 사용자-facing 설명에서 `성운/회상` 용어를 제거한다.
- 코드 내부 `CameraMode = 'nebula' | 'recall'`은 구현 안정성을 위해 유지해도 된다. 단 그 이름은 개발자 식별자일 뿐, 화면과 정책의 사용자 언어는 새 이름을 따른다.
- "회상"은 여전히 도메인 행위(별을 2초 이상 바라봐 `RecallMemory`를 발생시키는 능동 인출) 이름으로 남을 수 있다. 카메라 모드 이름으로만 쓰지 않는다.

### 2. 별들 가까이서 탐험하기 = 내가 움직이는 광원

근접 탐험에서는 사용자가 **빛을 들고 별 사이를 움직이는 존재** 로 느껴져야 한다.

- `StarField.selfLightPos`는 모드에 따라 달라진다.
  - **멀리서 내 우주 보기:** 기존처럼 중심 자아 별, 즉 원점 또는 `SelfStar`의 실제 drift 위치를 광원 위치로 쓴다. 거리=강함이라는 38의 광학적 읽기를 보존한다.
  - **별들 가까이서 탐험하기:** 광원 위치가 카메라/탐험자 위치를 따라간다. 카메라가 전진하면 가까운 별 표면의 반사도 함께 이동한다. 사용자는 "광원 캐릭터"를 조종하는 느낌을 받는다.
- 근접 광원은 별의 **반사 채널** 만 바꾼다. `selfGlow`(의미 밝기), `activation`, `λ_eff`, 별 색, 별 좌표는 바꾸지 않는다.
- 근접 광원 위치는 shake offset이 누적된 화면 떨림이 아니라 **실제 항행 기준 위치** 를 따라야 한다. 벽 recoil과 관성 이동은 반영하되, idle shake가 광원을 미세하게 흔들어 별 반사가 지저분하게 떨리는 것은 피한다.
- 필요하면 근접 모드에서 아주 작은 전방 글로우/reticle 또는 보이지 않는 "light avatar"를 추가할 수 있다. 하지만 씬 안 DOM은 금지하고, 별·시냅스 렌더 프리미티브는 유지한다.
- 겹쳐보기(`UniverseOverlay`)는 각 우주의 기존 self-light 규칙을 유지한다. 이 변경의 주 대상은 메인 단일 우주 항행이다.

### 3. 별들 가까이서 탐험하기 터치 문법

모바일/터치에서 버튼을 누르는 대신 캔버스를 직접 조작한다.

| 입력 | 동작 | 세부 |
|---|---|---|
| 한 손가락 드래그 | 고개 돌리기 | 좌우 드래그 = yaw, 상하 드래그 = pitch. 현재 화살표 버튼의 시선 회전을 직접 드래그로 대체한다. |
| 두 손가락을 올린 뒤 위로 쓸기 | 전진 | 두 손가락 centroid가 위로 움직이면 `z > 0` 추력. 이동량에 비례해 0..1로 ramp한다. |
| 두 손가락을 올린 뒤 아래로 쓸기 | 후진 | 두 손가락 centroid가 아래로 움직이면 `z < 0` 추력. |
| 두 손가락 유지 중 좌우 흔들림 | 무시 또는 미세 yaw 금지 | 전후진 제스처와 시선 회전이 섞여 멀미 나지 않도록 두 손가락 모드에서는 thrust만 잠근다. |
| 키보드 | 유지 | 데스크톱 접근성과 빠른 조작을 위해 `W/S/A/D/Arrow`는 유지한다. |

- 손가락 수가 1에서 2로 바뀌는 순간, 한 손가락 look gesture는 종료하고 두 손가락 thrust gesture로 lock한다. 2에서 1로 줄어들면 새 deadzone을 통과하기 전까지 look을 재개하지 않는다.
- pointermove를 React state로 몰아넣지 않는다. 캔버스 gesture controller가 ref 기반 입력 버퍼를 갱신하고, `NavController`가 매 프레임 `keyboard/pad` 입력과 `gesture` 입력을 합성한다.
- 모바일에서 `NavPad`는 더 이상 기본 조작 표면이 아니다. 터치 지원 기기에서는 숨기거나 보조 접근성 컨트롤로 낮춘다. 데스크톱/비터치 환경에서는 현재 키보드와 필요 시 버튼 fallback을 유지한다.
- 탭과 드래그의 deadzone은 유지한다. 짧은 탭은 별 선택, deadzone을 넘긴 한 손가락 이동은 look, 두 손가락은 선택을 발생시키지 않는다.

### 4. 멀리서 내 우주 보기 터치 문법

원거리 조망은 "내 우주 전체를 손으로 잡고 보는" 느낌이어야 한다.

| 입력 | 동작 | 세부 |
|---|---|---|
| 한 손가락 드래그 | 시선 회전 | 기존 custom arcball 회전을 유지하되, 새 용어와 gesture controller 구조에 맞춘다. |
| 두 손가락 이동 | pan | 두 손가락 centroid 이동을 화면 평면 pan으로 해석해 `controls.target`과 `camera.position`을 같은 delta로 이동한다. 회전 기준이 중앙에 고정되지 않고 사용자가 조망 기준점을 옮길 수 있다. |
| 더블탭 후 유지하며 위로 드래그 | zoom in | 두 손가락 pan과 충돌하지 않게 zoom scrub은 **double-tap-hold + vertical drag** 로 lock한다. 위로 올리면 반경 감소, 즉 zoom in. |
| 더블탭 후 유지하며 아래로 드래그 | zoom out | 아래로 내리면 반경 증가. |
| wheel / pinch | 유지 | 데스크톱 wheel과 기존 pinch zoom은 보조 입력으로 유지한다. 사용자가 이미 익숙한 브라우저/지도식 조작을 깨지 않는다. |

두 손가락 pan과 "두 손가락 세로 이동 zoom"은 모두 centroid vertical 이동을 쓰기 때문에 같은 손동작으로 동시에 해석되면 충돌한다. 따라서 구현 문법은 다음처럼 잠근다.

- 두 손가락이 올라오면 기본은 **pan**이다.
- 두 손가락 사이 거리가 변하면 기존 pinch zoom을 수행할 수 있다.
- 사용자가 요구한 "더블터치 후 위/아래 zoom"은 웹에서 충돌이 적은 **double-tap-hold vertical scrub** 로 구현한다. 이 제스처가 lock되면 pan은 발생하지 않는다.
- double-tap의 시간/거리 임계, zoom scrub deadzone, pan speed, zoom speed는 모두 `spec/values.yaml`에 둔다.

### 5. 입력/상태 설계

이번 변경은 단순 copy 교체가 아니라 입력 계층을 분리한다.

- `navigation.machine.ts`는 카메라의 이산 상태(`nebula`/`recall` 등)를 계속 소유한다.
- 연속 gesture delta는 XState context에 고주파로 넣지 않는다. 새 `CanvasGestureController` 또는 `navigation-input` ref store가 다음을 담당한다.
  - active pointers map
  - gesture lock(`tapCandidate` / `look` / `closeThrust` / `farPan` / `farZoomScrub` / `pinchZoom`)
  - deadzone과 pointer capture
  - 브라우저 기본 제스처 차단(`touch-action: none`, wheel passive false)
- `NavController`는 `navigationActor.context.move`(keyboard/fallback)와 gesture buffer를 합성해 기존 추력·관성·벽 반동 물리를 재사용한다.
- `NebulaOrbitController`는 회전만 소유하던 구조에서 pan/zoom scrub까지 같은 controller가 소유하도록 확장하거나, 새 gesture controller로 입력만 분리하고 기존 arcball math를 재사용한다.
- 별 선택은 계속 `StarField.onSelect`와 `focusActor.SELECT_STAR`가 소유한다. 제스처 controller는 "이 포인터 시퀀스가 클릭 가능한 탭인가"만 판단한다.

### 6. 값과 튜닝

새로 생기거나 조정되는 숫자는 코드에 박지 않고 `spec/values.yaml`에 둔다.

- `gesture.drag_deadzone_px` = 기존 8px 계승
- `gesture.close_thrust_deadzone_px`
- `gesture.close_thrust_full_px`
- `gesture.close_look_sensitivity`
- `gesture.far_pan_speed`
- `gesture.far_zoom_scrub_deadzone_px`
- `gesture.far_zoom_scrub_speed`
- `gesture.double_tap_ms`
- `gesture.double_tap_max_dist_px`
- 기존 `NEBULA_ZOOM_SPEED`, `NEBULA_ROTATE_SPEED`, `NEBULA_DAMP`도 가능하면 values로 이동해 policy와 코드가 같은 값을 보게 한다.

## 범위 / 비목표 (회귀 경계)

**범위(바꾼다)**

- 메인 우주 사용자-facing 카메라 모드 용어: `성운/회상` → **멀리서 내 우주 보기 / 별들 가까이서 탐험하기**.
- `HomePage` 카메라 토글 copy, title, aria-label, 안내 문구, `NavPad` 기본 노출 정책.
- `UniverseCanvas` 입력 계층: 터치/마우스 Pointer Events 기반 gesture controller, `touch-action: none`, pointer capture, double-tap-hold, pan, zoom scrub.
- 근접 탐험: 한 손가락 look, 두 손가락 전진/후진, 광원 위치가 탐험자와 함께 이동.
- 원거리 조망: 두 손가락 pan, double-tap-hold vertical zoom, 기존 wheel/pinch 유지.
- `StarField.selfLightPos` 공급: 원거리 = 중심 자아 광원, 근접 = 탐험자 광원.
- `policy/domain/navigation.md`, `policy/ux/interaction.md`, `policy/domain/star.md`, `policy/domain/universe.md`에 구현 완료 시 새 사실 반영.
- `spec/values.yaml`에 gesture tuning 값 추가, `pnpm gen:values`로 FE 상수 생성.

**비목표(안 바꾼다)**

- 별/시냅스 좌표 생성 방식은 바꾸지 않는다. force-sim, 반지름=강함, 각도=연결, 좌표 서버 미저장은 그대로다.
- 회상 도메인 행위(`RecallMemory`, 2초 dwell, 공동 회상 강화)는 바꾸지 않는다. 카메라 모드 명칭에서만 `회상`을 제거한다.
- 원본 일기 불변, 별/시냅스 삭제 금지, unary RPC, FSD import 방향은 그대로다.
- 별 렌더 구조는 유지한다. 수천 별은 계속 단일 `InstancedMesh`, 시냅스는 `Line2`/TSL 계열, 반사는 TSL uniform 계산이다. 진짜 `THREE.PointLight`를 별마다 만들지 않는다.
- 별 색은 여전히 mood/emotion color에서 온다. 이동 광원은 색을 바꾸지 않는 중립 반사 채널이다.
- `FlyToController`, `FrameAllController`, `FocusController`, `ModeTransitionController`의 유도 비행 의미는 유지한다. 제스처 입력은 transition/focus 중 stand down한다.
- 겹쳐보기 전용 `UniverseOverlay` 카메라는 이 change의 필수 범위가 아니다. 필요 시 별도 change에서 맞춘다.
- 브라우저 기본 페이지 스크롤/뒤로가기 제스처를 전역으로 망가뜨리지 않는다. 차단은 우주 캔버스 표면에 한정한다.

## 수용 기준 (인수 조건)

1. **용어 교체.** 메인 우주 화면에서 카메라 모드 이름, 버튼 title, aria-label, 안내 copy에 `성운`/`회상`이 카메라 모드명으로 노출되지 않는다. 토글은 **멀리서 내 우주 보기** 와 **별들 가까이서 탐험하기** 로 표시된다.
2. **회상 도메인 보존.** 별 선택 후 2초 dwell로 `RecallMemory`가 발생하는 도메인 행위 이름과 동작은 그대로다. 카메라 모드 이름 변경이 회상 패널, 공동 회상, 잠든 별 재점화에 회귀를 만들지 않는다.
3. **근접 광원 이동.** 별들 가까이서 탐험하기 모드에서 카메라/탐험자가 이동하면 `StarField.selfLightPos`가 같은 기준 위치로 갱신되어 별 표면 반사가 함께 이동한다. 원거리 모드에서는 중심 자아 광원 규칙이 유지된다.
4. **광원 채널 경계.** 이동 광원은 반사 채널만 바꾼다. `selfGlow`, `activation`, `λ_eff`, 별 색, 별 좌표, `A_MIN` 밝기 바닥은 변하지 않는다.
5. **근접 한 손가락 look.** 별들 가까이서 탐험하기에서 한 손가락/마우스 드래그가 고개 회전으로 동작한다. 좌우는 yaw, 상하는 pitch이며, 기존 화살표 버튼과 같은 방향 감각이다.
6. **근접 두 손가락 전후진.** 별들 가까이서 탐험하기에서 두 손가락을 올린 뒤 위로 움직이면 전진, 아래로 움직이면 후진한다. 추력은 deadzone 이후 이동량에 비례해 ramp하고, 손을 떼면 기존 관성/제동으로 멈춘다.
7. **근접 버튼 비주도화.** 터치 지원 모바일에서 전진/후진/화살표 `NavPad`가 기본 조작 표면으로 고정 노출되지 않는다. 데스크톱 키보드와 접근성 fallback은 유지된다.
8. **원거리 한 손가락 회전 유지.** 멀리서 내 우주 보기에서 기존 한 손가락/마우스 arcball 회전, wheel zoom, pinch zoom은 회귀 없이 동작한다.
9. **원거리 두 손가락 pan.** 멀리서 내 우주 보기에서 두 손가락을 같은 방향으로 움직이면 화면 평면 기준으로 camera와 target이 함께 이동해 조망 기준점이 중앙에 고정되지 않는다.
10. **원거리 double-tap-hold zoom.** 멀리서 내 우주 보기에서 double-tap-hold 후 위로 드래그하면 zoom in, 아래로 드래그하면 zoom out이 된다. 이 gesture가 lock된 동안 pan은 발생하지 않는다.
11. **제스처 충돌 방지.** 탭/드래그/별 선택/두 손가락/pinch/double-tap-hold가 lock 규칙으로 분리된다. 별을 탭하면 선택되고, 드래그 중 별이 잘못 선택되지 않으며, 두 손가락 gesture가 `onPointerMissed` dismiss를 유발하지 않는다.
12. **브라우저 gesture 제어.** 우주 캔버스는 Pointer Events 기반으로 동작하고, 캔버스 표면에 `touch-action: none` 또는 동등한 처리를 적용해 브라우저 스크롤/핀치와 커스텀 gesture가 충돌하지 않는다. 차단 범위는 우주 캔버스에 한정된다.
13. **전환/포커스 중 stand down.** `flyingToStar`, `framingDiary`, `modeTransition`, 별 focus 중에는 사용자 gesture가 유도 비행/포커스 컨트롤러와 싸우지 않는다. 전환이 끝나면 해당 모드의 gesture가 다시 활성화된다.
14. **값 단일 출처.** gesture deadzone, sensitivity, double-tap 시간/거리, pan/zoom 속도는 `spec/values.yaml`에서 생성된 `VALUES`를 사용한다. 새 하드코딩 튜닝 숫자가 없다.
15. **성능/아키텍처.** pointermove가 React state 리렌더를 유발하지 않는다. 연속 입력은 ref/useFrame 경로로 처리되고, `shared/lib`/model 순수 모듈은 three/React/DOM을 import하지 않는다.
16. **모바일 검증.** Playwright 또는 수동 모바일 시뮬레이션으로 최소 390x844와 데스크톱에서 다음을 확인한다: 근접 look, 근접 두 손가락 전/후진, 원거리 pan, 원거리 zoom scrub, 별 탭 선택, 빈 우주 탭 dismiss.
17. build·lint·steiger·관련 vitest가 통과하고, 우주 셸 첫 로드/빈 우주/렌더러 fallback/일기 조망/fly-to에 회귀가 없다.
