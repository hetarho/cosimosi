# 24. 변경: 데모 연속 가상 시계 + 배속 컨트롤 (이산 시간 버튼 대체)

> [plan/19.engram-experience](../plan/19.engram-experience.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 데모에서 시간을 *이산 점프*가 아니라 **배속을 골라 자연스럽게 계속 흐르게** 해, 별이 멀어지고 어두워지고 밤마다
> 우주가 정리되는 모습을 **실시간으로** 보게 한다. 시간 흐름이 구동하는 로직은 production과 동일하게 맞춘다.

## 현재 (as-is)

데모 가상 시계는 [shared/lib/demo/clock.ts](../../frontend/src/shared/lib/demo/clock.ts)의 모듈 변수 `offsetMs`로,
`virtualNowMs() = Date.now() + offsetMs`다. 시간은 오직 **이산 점프**로만 흐른다:

- `skipDemoDays(days)`가 `offsetMs += days·DAY_MS`. 시간버튼 팝오버([DemoFreeModeControls.tsx](../../frontend/src/pages/home/ui/DemoFreeModeControls.tsx))는
  `하루 후로 이동`·`한 달 후로 이동`·`처음으로` 3개.
- 시간 점프 시 [widgets/demo-sim/model/time-travel.ts](../../frontend/src/widgets/demo-sim/model/time-travel.ts) `runTimeSkip`가
  `demoApplyDayBatch(days)`(day별 루프로 `demoConsolidate()` 호출 = 야간 공고화) → `refreshActivation()` → universe/dormant 쿼리 invalidate.
- 별의 밝기·반지름은 refetch 시 `virtualNowMs()`로 재계산되지만, 점프 사이에는 **정지**(연속 흐름 없음).
- 데모 `demoConsolidate()`(in [shared/lib/demo/data.ts](../../frontend/src/shared/lib/demo/data.ts))는 옛 요지/가지치기 근사라
  production change 20 재설계(반경 스코프·링크 reweight·spread·`abstraction_stage`·last-link 보호·re-KNN)와 다르다.

즉 "하루가 지나면 어떻게 되나"를 버튼으로 한 번에 건너뛸 뿐, 시간이 흐르며 우주가 *변해가는 과정*을 볼 수 없다.

## 목표 (to-be)

데모 가상 시계를 **연속 시계 + 배속 컨트롤**로 바꾼다.

- **배속 옵션 4종**: `1초 = 1시간`(기본) · `1초 = 2시간` · `1초 = 4시간` · `정지`(배속 없음). UI는 시간 버튼 팝오버의 배속
  셀렉터(또는 칩)로 고른다.
- **연속 흐름**: 데모 모드(`free`/`tutorial`)에서 시계는 선택 배속으로 **계속 흐른다**. rAF(또는 단일 타이머) 틱마다 경과 실시간을
  배속으로 곱해 `offsetMs`에 누적한다(`정지`면 멈춤). 시계가 흐르면 별의 반지름·밝기·드리프트가 **실시간으로** 갱신돼(거리=f(time),
  change 18·19) 별이 서서히 멀어지고 어두워지는 게 보인다.
- **야간 공고화 자동 발화**: 가상 시계가 **매 simulated 04:00 KST 경계를 지날 때마다** production 야간 공고화와 **동일한 로직**(change 20:
  반경 스코프 ≤60·시간링크 약화/의미링크 강화·spread·`abstraction_stage` 요지화·마지막 연결 보호·re-KNN)을 1회 실행한다. 빠른 배속에서
  한 틱에 여러 날 경계를 건너뛰면 각 경계마다 1회씩(누락·중복 없이 멱등) 실행한다. 사용자는 "밤이 오면 우주가 스스로 정리되는" 걸 본다.
- **이산 버튼 제거**: `하루 후로 이동`·`한 달 후로 이동` 점프 버튼을 제거하고 배속 컨트롤로 대체한다. `처음으로`(가상 시계 0 + 추가 별 0,
  페르소나 유지)는 유지한다.
- **production 로직 정렬**: 데모 `demoConsolidate()`를 위 change 20 모델로 맞추고, 데모 전용 요지/가지치기 근사 상수(옛 `DEMO_GIST_*`·
  `DEMO_WEAK_*` 등)는 production 정전 값(`spec/values.yaml`)을 따르도록 정리한다(데모 강조/단축 제거 — 자세한 생성·밝기 정렬은 change 25와 공유 경계).

## 범위 / 비목표 (회귀 경계)

**범위**: 데모 가상 시계(`shared/lib/demo/clock.ts`)·시간 컨트롤 UI·`time-travel.ts`의 시간 흐름/야간 발화 경로·데모 야간
공고화 로직(production 정렬). 배속 틱은 데모 세션에서만 돈다.

**비목표 (회귀 경계)**:
- 실서버 야간 배치 스케줄·cron·백엔드 동작을 바꾸지 않는다(데모는 클라 시뮬레이션). backend/proto/RPC/sqlc/DB 변경 없음.
- 실계정(비데모) 우주에서는 배속 시계·자동 야간 발화가 **나타나지 않는다**(실시간은 진짜 시간).
- 페르소나 전환·`처음으로`·망원경·회상·일반 모드 동작은 회귀 없이 유지. `처음으로`는 온보딩으로 돌아가지 않는다.
- 헌법: 별·시냅스 삭제 없음, 원본 불변, 서버 비좌표, `A_MIN=0.05` 단일 바닥 유지. 데모 데이터는 실서버에 안 쓴다.
- 배속이 흐르는 동안에도 force-sim·드리프트·밝기 식은 production과 **동일 공식**을 쓴다(데모 전용 식 신설 금지).

## 수용 기준 (인수 조건)

1. 데모 시간 컨트롤에 배속 4종(`1초=1시간` 기본 · `2시간` · `4시간` · `정지`)이 있고, 고른 배속이 즉시 적용된다.
2. `정지`가 아니면 데모 우주의 가상 시계가 선택 배속으로 **계속 흐르고**, 별의 반지름·밝기·드리프트가 실시간으로 갱신돼 별이 서서히
   멀어지고 어두워지는 변화가 끊김 없이 보인다(점프 아님).
3. 가상 시계가 simulated 04:00 KST를 지날 때마다 야간 공고화가 **production change 20 로직과 동일하게** 1회 실행된다(반경 스코프 ≤60·
   시간링크 약화/의미링크 강화·spread·`abstraction_stage` 승급·마지막 연결 보호·re-KNN). 빠른 배속에서 한 틱이 여러 경계를 건너도
   경계마다 정확히 1회(누락·중복 없음).
4. `하루 후로 이동`·`한 달 후로 이동` 이산 버튼이 사라지고 배속 컨트롤로 대체된다. `처음으로`는 가상 시계 0·추가 별 0으로 되돌리되
   페르소나는 유지하고 온보딩으로 가지 않는다.
5. 데모 야간 공고화가 옛 요지/가지치기 근사가 아니라 production 정전 값(`spec/values.yaml`)·change 20 모델을 따른다(데모 전용 요지/가지치기 상수 제거).
6. 실계정(비데모) 우주에는 배속 시계·자동 야간 발화·배속 컨트롤이 나타나지 않는다.
7. 별·시냅스·원본 일기 삭제·수정 없음. 데모 데이터는 실서버에 기록되지 않고 demo exit 시 정리된다.
8. 배속이 흐르는 동안 매 프레임 과도한 React 리렌더·머신 이벤트를 만들지 않는다(rAF/단일 타이머 + 좌표·밝기는 기존 렌더 경로, 헌법4·8).
9. 백엔드·proto·RPC·sqlc 변경 없이 FE + values 생성물만으로 구현되고, build·lint·typecheck·steiger 통과. 페르소나 전환·`처음으로`·
   망원경·회상·일반 모드 작성에 회귀 없음.
