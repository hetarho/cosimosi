# 09. 변경: 우주 모드 UX 재구성 — 햄버거 사이드바·탐색 탭·일기장·HUD 숨김

> [plan/06.universe-canvas](../plan/06.universe-canvas.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 우주 셸의 상시 HUD와 기능 진입 IA를 다시 잡는 변경이다. 캔버스·카메라·별/시냅스 렌더링의 도메인 동작은 그대로 두고, 사용자가 우주 안에서 "보기/탐색/작성/설정/계정"을 찾는 방식만 재구성한다. STEPS는 /create-change-job이 만드는 job에.

## 현재 (as-is)

루트 `/`는 `HomePage`가 영속 우주 캔버스와 DOM HUD를 함께 소유하는 우주 셸이다.

- 우상단에는 카메라 시점 토글과 "메뉴" 런처가 한 줄로 있고, 메뉴 `Surface` 안에 `새 일기 쓰기`·`일기`·`잠든 별`·`우주 공개`·`주고받은 별`·`테마·외형`·`감정색`이 들어 있다.
- `SessionGate`는 보호 라우트 공통으로 우상단 로그아웃 pin을 렌더한다. 그래서 우주 셸 안에는 메뉴 버튼과 로그아웃 pin이 서로 다른 소유권으로 공존한다.
- 일기 목록은 별도 일기 앱 화면이 아니라 `/?panel=diary` 오버레이다. `DiarySheet`는 날짜/본문 발췌 검색, 날짜, 별 개수, 발췌를 보여주고, 항목 선택 시 그 일기의 별들을 frame-all로 조망한다. 원본 전문은 별 회상 패널에서만 열린다.
- 잠든 별은 `/?panel=dormant` 오버레이다. `DormantSheet`는 잠든 별만 조회하고, 감정/메모리 id 검색과 밝기 dot, 마지막 회상 경과일을 보여준다.
- 우주 공개와 주고받은 별은 메뉴 항목이 여는 별도 `Surface`다.
- 외형 편집은 좌상단 테마/별가루 버튼 또는 메뉴의 `테마·외형`에서 `AppearanceModal`을 열며, 현재 4축 외형(배경·별·나·시냅스)을 한 화면의 탭으로 편집한다. 감정색 편집은 별도 `/emotion-colors` 라우트로 보낸다.
- 새 별/새 일기 작성은 메뉴 안의 `새 일기 쓰기` 항목으로 들어간다.
- 우주를 감상하기 위해 전체 HUD를 숨기는 단일 토글은 없다.

## 목표 (to-be)

우주 모드의 기본 화면은 우주 자체를 더 크게 남기고, 기능 진입은 성격별로 분리한다.

### A. 우상단 햄버거 사이드바

- 우측 상단 최상단에는 햄버거 버튼만 둔다. 누르면 우측 사이드바/드로어가 열린다.
- 사이드바 항목 순서는 다음 그룹을 따른다.
  - `로그아웃`
  - `마이페이지`
  - 구분선
  - `우주 공개`
  - `주고받은 별`
  - 구분선
  - `일기`
- `로그아웃`은 기존 `SessionGate` sign-out 동작과 같은 출처 경계 리셋을 수행한다. 우주 셸에서는 독립 로그아웃 pin을 더 이상 별도로 노출하지 않고, 사이드바 안의 항목으로 수렴한다. 다른 보호 라우트(`/admin`, `/gift/$token` 등)의 인증 UX는 별도 요구가 없으면 기존 동작을 유지한다.
- `우주 공개`·`주고받은 별`은 현재의 `ShareUniverseBody`·`StarGiftsBody`를 계속 쓰되 진입점만 사이드바로 옮긴다.
- `마이페이지`는 보호 라우트의 계정/내 정보 진입점이다. 이 변경에서는 우주 셸에서 갈 수 있는 최소 표면/라우트를 제공하는 것까지만 범위로 두고, 프로필 편집·탈퇴·알림 설정 같은 세부 계정 기능은 후속 범위다.

### B. 우상단 세로 보기 컨트롤

- 햄버거 버튼 아래에 시점 전환 버튼을 세로로 배치한다. 동작과 용어는 현재 카메라 모드 토글(멀리서 내 우주 보기 / 별들 가까이서 탐험하기)을 유지한다.
- 시점 전환 버튼 아래에 망원경 아이콘 버튼을 둔다. 누르면 우주 탐색 바텀시트가 열린다.
- 버튼들은 아이콘 중심으로 작고 안정적인 크기를 가지며, 낯선 아이콘에는 tooltip/aria-label을 제공한다.

### C. 망원경 바텀시트: `일기` / `별` 탭

- 망원경 바텀시트는 탭 2개를 가진다: `일기`, `별`.
- `일기` 탭은 현재 `DiarySheet`가 제공하던 원본 일기 목록/선택/별 조망 흐름을 유지하면서 검색과 필터를 강화한다.
  - 검색: 날짜·본문 발췌.
  - 필터: 감정, 날짜 범위 또는 날짜 빠른 선택.
  - 일기 선택 시 기존처럼 그 일기에서 태어난 별들을 frame-all로 조망하고 강조한다.
- `별` 탭은 우주의 별 목록을 제공한다.
  - 별 목록에는 일반 별과 잠든 별을 함께 보여준다. 잠든 별은 별도 `잠든 별` 메뉴/패널로 분리하지 않는다.
  - 필터: 감정, 날짜/기간, 잠든 별 포함/잠든 별만, 필요 시 공명/밝기 같은 현재 데이터로 판정 가능한 상태.
  - 별 선택 시 기존 `FLY_TO_STAR`/회상 흐름으로 수렴한다.
- 기존 `/?panel=dormant` 진입은 신규 UI에서는 제거한다. 호환이 필요하면 구현 단계에서 redirect 또는 무시 정책을 정하되, 제품 표면에는 `잠든 별`이라는 별도 진입점을 남기지 않는다.

### D. 독립 일기 페이지

- 사이드바의 `일기`는 신규 일기 페이지로 이동한다. 우주 안에서 별을 찾는 바텀시트와 별개로, 일반적인 일기 앱처럼 읽고 훑는 전용 화면이다.
- 일기 페이지는 날짜 중심 목록/검색/필터, 일기 상세 읽기, 해당 일기에서 태어난 별 개수와 우주에서 보기 동선을 제공한다.
- 원본 일기 불변 원칙은 유지한다. 이 변경은 일기 편집/삭제를 만들지 않는다. 필요하면 "새 별 띄우기" 작성 흐름으로 새 일기를 추가하고, 기존 일기는 읽기 전용이다.
- 일기 상세에서 `우주에서 보기`를 누르면 루트 `/`로 돌아와 해당 record의 별들을 조망한다.

### E. 상단 중앙 HUD 숨김/보이기

- 상단 중앙에 `UI 숨기기` / `UI 보이기` 토글 버튼을 둔다.
- `UI 숨기기`를 누르면 해당 토글 버튼을 제외한 우주 HUD가 모두 숨겨진다: 좌상단 테마 버튼, 우상단 햄버거/시점/망원경 버튼, 하단 중앙 새 별 버튼, NavPad, 데모 HUD, 열려 있는 탐색/결과 표면.
- 숨김 상태에서도 우주 캔버스의 회전/탐험/별 클릭 등 직접 상호작용은 가능해야 한다.
- `UI 보이기`를 누르면 기본 HUD가 복귀한다. 숨김 상태는 라우트 이동·로그아웃·계정 전환을 넘어 영속할 필요는 없다.

### F. 좌상단 테마 버튼과 꾸미기 표면

- 좌측 상단 테마 버튼은 유지한다.
- 테마 버튼이 여는 꾸미기 표면에서 현재 헤더/페이지의 `외형` 텍스트는 제거한다.
- 꾸미기 표면은 크게 `스킨`과 `감정 색`으로 나눈다.
  - `스킨`: 현재 4축 외형(배경·별·나·시냅스) 선택, 별가루/잠금/저장 흐름을 유지한다. 사용자-facing 이름은 `스킨`을 사용하되, 도메인 문서의 4축 커스터마이즈와 충돌하지 않게 내부 식별자는 기존 `background/star/self/synapse` 체계를 유지한다.
  - `감정 색`: 기존 `/emotion-colors`의 13개 감정색 설정 UX를 이 꾸미기 표면 안에서 편집할 수 있게 한다. 저장은 기존 `UpdateSettings.emotion_colors` full-set 규칙을 따른다.
- 독립 `/emotion-colors` 라우트는 최초 설정 게이트와 직접 접근 편집용으로 유지할 수 있지만, 우주 모드에서 감정색 재설정 진입은 꾸미기 표면 안으로 수렴한다.

### G. 하단 중앙 floating 새 별 버튼

- 하단 중앙에는 floating 버튼 `새 별 띄우기`를 둔다.
- 이 버튼은 현재 `MemoryForm` 작성 흐름을 연다. 메뉴/사이드바 안의 `새 일기 쓰기` 항목은 제거한다.
- 데모 모드에서는 기존처럼 서버 작성 폼을 숨기거나, 데모 전용 작성 흐름이 있는 경우 같은 위치에 데모용 새 별 동선을 둔다.

## 범위 / 비목표 (회귀 경계)

**범위**
- `HomePage`의 HUD 배치와 진입 IA: 우상단 사이드바, 세로 보기 컨트롤, 망원경 탐색 시트, 상단 중앙 HUD 숨김 토글, 하단 중앙 새 별 floating 버튼.
- 일기/별 탐색 표면 통합: `DiarySheet` 강화, 신규 별 목록 탭, 잠든 별 별도 진입 제거.
- 신규 보호 일기 페이지: 읽기·검색·필터·상세·우주에서 보기.
- 꾸미기 표면 재구성: `스킨` / `감정 색` 분리, 우주 모드 감정색 편집 진입 통합.
- `SessionGate`/우주 셸 로그아웃 노출 정리: 우주 셸에서는 사이드바 항목으로 수렴.

**비목표**
- 우주 캔버스 렌더러, force-sim, 별/시냅스 셰이더, 카메라 상태 머신의 물리·수학 변경.
- 원본 일기 수정/삭제. 원본 일기는 계속 불변·영구다.
- 감정 taxonomy(13종), 감정색 저장 계약, 별 색=mood 원칙 변경.
- 우주 공개/별 선물의 서버 계약 변경. 진입점만 사이드바로 옮긴다.
- 별가루 가격/소유권/아이템 카탈로그 변경.
- 마이페이지의 세부 계정 기능 전체 구현. 이번 범위는 우주 셸에서 접근 가능한 최소 보호 표면/라우트까지다.
- Flutter 구현. 단, 문서에는 Flutter가 같은 IA를 재현할 때 필요한 동등성 메모를 남긴다.

## 수용 기준 (인수 조건)

1. 루트 `/` 우주 셸의 우측 상단 최상단에는 햄버거 버튼이 있고, 누르면 우측 사이드바가 열린다.
2. 사이드바에는 `로그아웃`, `마이페이지`, 구분선, `우주 공개`, `주고받은 별`, 구분선, `일기`가 이 순서로 표시된다.
3. 사이드바의 `로그아웃`은 기존 sign-out과 동일하게 세션·우주 데이터·일기 캐시 출처 경계를 리셋한다. 우주 셸에서는 별도 우상단 로그아웃 pin이 보이지 않는다.
4. 사이드바의 `우주 공개`와 `주고받은 별`은 기존 기능 본문을 열며, 서버 계약과 공개/선물 도메인 동작은 변하지 않는다.
5. 햄버거 버튼 아래에는 시점 전환 버튼, 그 아래에는 망원경 버튼이 세로로 배치된다. 시점 전환은 기존 카메라 모드 토글 동작을 유지한다.
6. 망원경 버튼을 누르면 모바일에서는 바텀시트, 데스크톱에서는 기존 overlay-shell 원칙에 맞는 비차단 탐색 표면이 열리고, `일기`와 `별` 탭을 제공한다.
7. 망원경 `일기` 탭은 날짜·본문 발췌 검색과 감정/날짜 필터를 제공하고, 일기 선택 시 기존처럼 그 일기의 별들을 frame-all 조망 및 강조한다.
8. 망원경 `별` 탭은 일반 별과 잠든 별을 한 목록에서 보여주며, 감정/날짜/잠든 상태 필터를 제공한다. 별 선택은 기존 `FLY_TO_STAR` 후 회상 패널로 수렴한다.
9. 제품 UI에서 별도 `잠든 별` 메뉴/패널 진입점은 제거되고, 잠든 별 탐색은 망원경 `별` 탭으로 통합된다.
10. 사이드바의 `일기`는 신규 보호 일기 페이지로 이동한다. 이 페이지는 일기 목록, 검색, 날짜/감정 필터, 읽기 전용 상세, `우주에서 보기` 동선을 제공한다.
11. 일기 페이지와 망원경 일기 탭 모두 원본 일기를 수정·삭제하지 않는다. 운영 코드에 `UPDATE records` 또는 `DELETE FROM records` 성격의 흐름을 새로 만들지 않는다.
12. 일기 상세의 `우주에서 보기`는 루트 우주 셸로 돌아와 해당 record의 별들을 조망한다. 우주 캔버스의 좌표 권위는 여전히 클라이언트 force-sim이다.
13. 상단 중앙 `UI 숨기기`를 누르면 그 버튼을 제외한 우주 HUD와 열린 표면이 숨겨지고, `UI 보이기`를 누르면 기본 HUD가 복귀한다.
14. HUD 숨김 상태에서도 우주 캔버스 직접 조작과 별 클릭은 가능하다. 숨김/보이기 전환으로 WebGPU 캔버스가 재초기화되지 않는다.
15. 좌상단 테마 버튼은 유지되고, 열리는 꾸미기 표면에는 단독 헤더 텍스트 `외형`이 보이지 않는다.
16. 꾸미기 표면은 `스킨`과 `감정 색`으로 나뉜다. `스킨`은 기존 4축 커스터마이즈와 구매/저장 규칙을 유지하고, `감정 색`은 13개 mood 색을 기존 full-set 저장 규칙으로 편집한다.
17. 하단 중앙 floating `새 별 띄우기` 버튼이 현재 작성 폼을 열고, 햄버거 사이드바에는 작성 항목이 없다.
18. 어떤 진입/표면 전환에서도 별·시냅스 렌더링, 회상, 공유, 선물, 감정색, 외형 저장의 기존 도메인 불변식이 깨지지 않는다.
19. FSD 임포트 방향과 캔버스 밖 DOM HUD 원칙이 유지된다. 3D 씬 안에 `<Html>`을 추가하지 않고, 순수 model은 React/DOM/three에 의존하지 않는다.
20. build·lint·typecheck·steiger를 통과하고, 기존 `/?fly=`, 일기 frame-all, 별 회상, 우주 공개, 주고받은 별, 감정색 최초 게이트에 회귀가 없다.
