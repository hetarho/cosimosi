# 21. 변경: 링크 가중치에 감정 유사도 추가

> [plan/05.embedding-worker](../plan/05.embedding-worker.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 별이 이어지는 기준에 "감정이 닮음"을 *직접* 더한다. 지금은 감정이 글(임베딩)에 쓰인 만큼만 간접 반영된다.

## 현재 (as-is)

- 새 별 임베딩으로 의미 KNN(`cos ≥ 0.75`)을 돌려 `semantic` 링크를 만든다([excitability.go](../../backend/internal/job/excitability.go) `initialWeight`/`temporalBonus`).
- 초기 weight = `clamp(weightAlpha·cos_sim + temporal_bonus, 0, 1)`, 상한 `semanticWeightCap=0.79`. 시간 보너스(같은날 +0.3→7일 0)만 의미 유사도에 더해진다.
- **감정(mood·valence·intensity)은 링크 가중치에 직접 안 들어간다.** 색·크기·(기존)감쇠엔 쓰지만 연결 기준엔 없음 — 두 기억이 "같은 감정이라서" 이어지는 규칙은 부재.

## 목표 (to-be)

1. **감정 유사도 항 추가.** 링크 초기 weight에 두 별의 **감정 근접도** 보너스를 더한다 — 비슷한 감정(가까운 mood circumplex 좌표 / 가까운 valence·intensity)일수록 가산.
   - `weight = clamp(weightAlpha·cos_sim + temporal_bonus + emoAlpha·emoSim, 0, 1)`, `semanticWeightCap` 상한은 유지(일내 결속 0.8이 늘 최강이라는 불변 보존).
   - `emoSim ∈ [0,1]`: mood의 circumplex 좌표(`MOOD_AFFECT`, valence×arousal) 거리 또는 (valence, intensity) 벡터 코사인으로 근사.
2. **선택 임계는 의미 유사도 유지.** 후보 게이트(`cos ≥ 0.75`)는 그대로 — 감정만 닮고 의미 무관한 별이 무더기로 이어지지 않게. 감정은 *이미 의미로 후보가 된* 링크의 weight를 미세 조정.

## 범위 / 비목표 (회귀 경계)

**바꾼다:**
- `initialWeight` 계산에 감정 유사도 항 추가([excitability.go](../../backend/internal/job/excitability.go) — change 18로 함수가 이전될 수 있으니 job에서 위치 확인).
- 감정 유사도 헬퍼(서버) 신규 — mood→circumplex 좌표 or (valence,intensity) 기반.
- `spec/values.yaml`: `emoAlpha`(감정 유사도 가중) 신규.

**안 바꾼다(회귀 경계):**
- KNN 후보 게이트(`cos ≥ 0.75`)·`semanticWeightCap=0.79`·일내 결속 0.8 불변.
- 시간 보너스(change 무관) 유지.
- 경쟁적 할당(흥분성 편향)은 유지 — 이 변경과 독립.
- 헌법3(weight는 의미 그래프 파생, 좌표는 클라).

## 수용 기준 (인수 조건)

1. 의미 유사도·시간이 같고 감정만 다른 두 후보 중, **감정이 더 닮은 쪽의 링크 weight가 더 크다.**
2. 감정 유사도 항을 더해도 weight가 `semanticWeightCap(0.79)`를 못 넘어(일내 결속 0.8 < 보장).
3. 감정만 닮고 의미 유사도가 0.75 미만이면 **여전히 안 이어진다**(후보 게이트 불변).
4. `emoAlpha=0`이면 기존 동작과 동일(회귀 경계 — 항을 끄면 현행).
