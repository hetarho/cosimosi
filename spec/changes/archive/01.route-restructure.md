# 01. 변경: 라우트 재구성 (`/`=우주 · `/landing`=랜딩 · `/sign-in`=사인인)

> [plan/01.sign-in](../plan/01.sign-in.md)의 이미 구현된 라우트 보호·진입면 동작을 바꾸는 **변경 제안**(WHAT 델타).
> /create-change가 사용자 인터뷰로 채운다 — 자연어 한 줄이 아니라 검토 가능한 문서. STEPS는 /create-change-job이 만드는 job에.

## 현재 (as-is)
<!-- plan/01.sign-in 중 이 변경이 건드리는 현재 동작. 출발점. -->

라우트 테이블([frontend/src/app/router.tsx](../../frontend/src/app/router.tsx)):

- `/` → `LandingPage` — 공개 마케팅 랜딩, 게이트 없음 (spec 15).
- `/universe` → `SessionGate`(`HomePage`) — 보호 라우트. **사인인은 별도 라우트가 아니다**: 미인증이면 `SessionGate`가 같은 `/universe` URL에서 `SignInScreen`을 **인라인 렌더**한다([SessionGate.tsx:51](../../frontend/src/app/ui/SessionGate.tsx#L51)). `loading`→스플래시, `demo`→통과+"체험 모드" 배지, `authed`→children+로그아웃 핀.
- `/dormant` → `/universe?panel=dormant`로 영구 redirect (옛 풀페이지 호환 폴백, spec 31).
- Google OAuth `redirectTo` = `origin + '/universe'` ([session-context.tsx:10](../../frontend/src/app/ui/session-context.tsx#L10)).
- 진입 동선: 랜딩 "우주 만들어보기" → `enterDemoMode()` + navigate `/universe`, "로그인하기" → navigate `/universe`(인라인 사인인 표시) ([HeroSection.tsx](../../frontend/src/pages/landing/ui/section/HeroSection.tsx), [CtaFooterSection.tsx](../../frontend/src/pages/landing/ui/section/CtaFooterSection.tsx)). 데모 "체험 종료" → navigate `/`(현재 랜딩). gift/visit/notfound 링크가 `/universe`·`/`를 가리킴.
- 우주 셸의 search param(`sim`·`panel`·`fly`) 스키마와 `useSearch({from:'/universe'})`·`useNavigate({from:'/universe'})`가 `/universe` 라우트에 묶여 있다 ([HomePage.tsx:296-297](../../frontend/src/pages/home/ui/HomePage.tsx#L296-L297)).

## 목표 (to-be)
<!-- 변경 후 참이어야 하는 동작·계약. 구체적으로. -->

- **`/` → 우주 셸**(`SessionGate`(`HomePage`)). 인증 사용자에게 우주가 보인다. `sim`·`panel`·`fly` search param 스키마와 딥링크는 옛 `/universe`에서와 동일하게 `/`에서 동작한다.
- **`/landing` → 마케팅 랜딩**(`LandingPage`). 공개·게이트 없음. **직접 진입으로만** 도달한다(루트가 더는 랜딩이 아님). spec 15 동작은 경로만 바뀌고 무회귀.
- **`/sign-in` → 독립 공개 사인인 페이지.** `SignInScreen`을 라우트 콘텐츠로 렌더한다. `?redirect=<내부경로>`를 읽어 인증 성공 후 그 경로로 복귀(없거나 외부 URL이면 `/`). 이미 `authed`면 곧장 `redirect`(또는 `/`)로 보낸다.
- **보호 라우트 미인증 처리 = 인라인 → 리다이렉트.** `/`·`/admin`·`/gift/$token`에 미인증 진입 시 `SessionGate`가 같은 URL에 `SignInScreen`을 그리는 대신 `/sign-in?redirect=<원래경로>`로 **리다이렉트**한다. 인증 후 원래 경로로 복귀.
  - 메커니즘: `SessionGate`는 유지하되 **`anon` 분기만** 인라인 렌더 → `/sign-in` 리다이렉트로 교체한다. `loading`(스플래시)·`demo`(통과+배지)·`authed`(children+로그아웃)는 그대로. 세션 상태는 여전히 XState 머신/React 컨텍스트가 소유한다 — 게이트를 router `beforeLoad`로 옮기지 않는다(불변 4·tech/state-machines와 일관, 머신을 router context에 끌어들이지 않음).
- **미인증이 `/`에 도착하면** `SessionGate`가 `/sign-in?redirect=/`로 보낸다(로그인 우선). 마케팅 랜딩은 `/landing` 직접 진입으로만 만난다.
- **옛 `/universe` 라우트 제거** → NotFound(404). 리다이렉트하지 않는다. 모든 내부 navigate/Link·OAuth `redirectTo`를 `/`로 갱신한다.
- **`/dormant` redirect 라우트 제거** → NotFound(404). 런치 전이라 보존할 옛 북마크가 없어 폴백이 불필요하다. dormant **오버레이 기능은 그대로**이며 `/?panel=dormant`로만 도달한다.
- 진입 동선 갱신: 랜딩 "우주 만들어보기" → `enterDemoMode()` + navigate `/`. "로그인하기" → navigate `/sign-in`. 데모 "체험 종료" → navigate `/landing`.

## 범위 / 비목표 (회귀 경계)
<!-- 무엇을 바꾸고, 무엇은 안 바꾸나. -->

**바꾼다:**

- `router.tsx` 라우트 테이블 — index(`/`)=`SessionGate`(`HomePage`), 신규 `/landing`=`LandingPage`, 신규 `/sign-in` 페이지, `/universe`·`/dormant` 라우트 삭제. `sim`·`panel`·`fly` `validateSearch`를 index 라우트로 이동.
- `SessionGate`의 `anon` 분기 → `/sign-in?redirect=<현재경로>` 리다이렉트.
- `/sign-in` 페이지/라우트 — `redirect` search param 검증(내부 경로만; 오픈 리다이렉트 방지)·`authed`면 복귀. (FSD 불변 4상 라우트는 화면 UI를 pages에 위임 — 사인인을 `pages/sign-in`으로 승격할지 app/ui 유지할지는 job이 정합성으로 결정.)
- OAuth `redirectTo` → `origin + '/'` ([session-context.tsx](../../frontend/src/app/ui/session-context.tsx)).
- 내부 경로 갱신 `/universe`→`/`: `HeroSection`·`CtaFooterSection`·`GiftPage`·`VisitPage`·`NotFoundScreen`·`ErrorScreens`·`TryInUniverse`(sim)·`DemoSimPanel`·데모 종료 동선, 그리고 `HomePage`의 `useSearch`/`useNavigate` `from:'/universe'`→`from:'/'`.
- 스펙 문서(구현 단계 SSOT 반영): 01(라우트 보호 표·수용 기준)·15(랜딩 경로 `/`→`/landing`)·06(우주 셸 경로 표기)·12(`/dormant` redirect 제거·AC#9·진입 라우트 줄)·tech/overlay-shell(딥링크 예시 `/universe?panel=`→`/?panel=`)·policy/domain/navigation·policy/ux/landing의 경로 표기.

**안 바꾼다 (회귀 경계):**

- `SessionGate`의 `loading`(스플래시)·`demo`(통과+체험 배지)·`authed`(children+로그아웃) 동작. 데모 모드 우회는 그대로 — 보호 라우트는 `isDemoMode()`면 사인인으로 보내지 않고 통과한다(게이트가 데모를 먼저 검사하므로 자연 보존).
- dormant 오버레이 **기능 자체**(시트·`dormant-query`·`FLY_TO_STAR`·회상 후 무효화)와 `panel`/`sim`/`fly` search param의 **의미·딥링크 동작**. 라우트만 `/universe`→`/`로 옮기고, 제거하는 건 `/dormant` *경로*뿐.
- 우주 캔버스·오버레이 셸·회상·작성 등 셸 내부 동작.
- `/u/$slug`(무인증 공개 방문)는 그대로 공개. `/admin`·`/gift/$token`의 보호 *정책*은 그대로(미인증 처리만 인라인→리다이렉트로 통일).
- 인증 머신(`session.machine`)·토큰 주입(`Authorization: Bearer`)·계정 전환 시 캐시 격리 — 01의 기존 계약 그대로.

**비목표:**

- 사인인 UI 자체 재설계(이메일 OTP·Google OAuth 흐름) — 동작은 그대로, 위치만 라우트로.
- 새 튜닝 수치 없음 → **values.yaml 영향 없음**(라우트 경로는 튜닝 스칼라가 아니라 문자열 식별자).

## 수용 기준 (인수 조건)
<!-- 변경 완료의 판정 기준. job의 "인수 조건" 섹션으로 복사된다. -->
1. 로그아웃 상태로 `/`에 진입하면 `/sign-in?redirect=/`로 리다이렉트되고 사인인 화면이 보인다. 인라인 사인인은 더 이상 어떤 보호 라우트 URL에서도 렌더되지 않는다.
2. 로그아웃 상태로 `/admin`·`/gift/$token`에 진입하면 `/sign-in?redirect=<해당 경로>`로 리다이렉트된다.
3. `/sign-in`에서 이메일 OTP 검증 또는 Google OAuth 복귀로 인증되면 `redirect` 경로(없으면 `/`)로 이동하고 우주 셸이 렌더된다.
4. 이미 인증된 사용자가 `/sign-in`을 직접 열면 `/`(또는 `redirect`)로 곧장 이동한다.
5. `/landing`은 세션 없이 열리고 기존 랜딩(배경·히어로·여정·스위처)이 그대로 렌더된다(spec 15 동작 무회귀).
6. `/landing` 히어로 "우주 만들어보기"는 데모 모드로 `/`에 진입(로그인 없이 더미 우주), "로그인하기"는 `/sign-in`으로 이동한다.
7. 데모 모드 "체험 종료"는 데모 플래그를 비운 뒤 `/landing`으로 이동한다.
8. 옛 `/universe`로 진입하면 NotFound(404) 화면이 뜬다(리다이렉트하지 않는다). 내부 어디에도 `/universe`로 향하는 링크·OAuth `redirectTo`가 남지 않는다.
9. `/dormant`로 진입하면 NotFound(404) 화면이 뜬다. dormant 오버레이는 `/?panel=dormant`로 열리며 시트·`FLY_TO_STAR`·회상 후 무효화는 그대로 동작한다.
10. `redirect` search param은 내부 경로(`/`로 시작, `//`·외부 절대 URL 아님)만 허용한다 — 그 외는 무시하고 `/`로 폴백한다(오픈 리다이렉트 없음).
11. 인증 사용자의 `/` 진입·새로고침 시 우주 셸이 보이고, `sim`·`panel`·`fly` 딥링크 search param이 `/`에서 옛 `/universe`와 동일하게 동작한다.
12. `prefers-reduced-motion`·계정 전환 캐시 격리·토큰 주입 등 01의 기존 수용 기준이 회귀 없이 유지된다.
