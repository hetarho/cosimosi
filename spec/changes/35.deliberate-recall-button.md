# 35. 변경: 회상을 의도적 행동으로 — 클릭은 읽기, 버튼이 회상, 재회상 쿨다운

> [plan/11.recall-reinforce](../plan/11.recall-reinforce.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 지금은 별을 2초만 바라봐도(둘러보기) 무조건 회상이 발화해 재점화·재공고화·재성형·공동회상 강화가 일어난다.
> 그래서 우주를 **둘러보기만 해도 모든 별이 재성형/재점화**되어, 안 본 별의 자연 망각·요지화(추상화)가 무너진다.
> 회상을 **의도적 행동**으로 바꾼다: 클릭은 부작용 없이 내용을 읽기만, 명시적 **"이 별 자세히 보고 회상하기"** 버튼이
> 진짜 회상(부작용)을 일으키고, 재회상은 **마지막 회상 후 쿨다운(기본 1시간, 또는 하룻밤 공고화)** 경과 후에만 가능하다.

## 현재 (as-is)
<!-- plan/11.recall-reinforce 중 이 변경이 건드리는 현재 동작. 출발점. -->

- **2초 dwell = 무조건 회상.** 별을 클릭하면 [MemoryPanel](../../frontend/src/features/recall/ui/MemoryPanel.tsx)이 `dwelling`으로 열리고, `DWELL_MS`(2초, `values.recall.dwell_ms`) 능동 dwell이 지나면 **자동으로** `RecallMemory`를 발화한다([MemoryPanel.tsx:~145](../../frontend/src/features/recall/ui/MemoryPanel.tsx)). 캐시돼 있어도 touch는 매번 재발화한다(재점화 의미론, spec 16).
- **`RecallMemory` 한 호출이 부작용을 다 한다.** [handler.go RecallMemory](../../backend/internal/memory/handler.go)→`svc.RecallMemory`가 한 번에: ① 내용 반환(`Record`·`fragment_text`·`derived_text`), ② **재점화**(`last_recalled_at = now` — 잠든 별을 깨워 다시 밝게, spec 11·12), ③ **PE 게이트 재공고화 재성형**(`brightness_offset`·`hue_shift`·`form_seed_delta` 누적 + `evolution_history` append, spec 23), ④ `abstraction_stage ≥ stage_threshold`면 **AI 내용 재작성** 잡 적재(spec 54). proto에선 `RecallMemory`가 `idempotency_level = NO_SIDE_EFFECTS`로 선언돼 있으나 **실제로는 별 레이어를 변형**한다(원본 `records`만 불변).
- **공동 회상 강화도 같은 dwell에.** FE가 2초 dwell에서 `RECORD_VIEW`를 보내 공동 회상 쌍을 누적하고 unary 배치(`reinforceLinks`)로 헵 강화한다(spec 11). 즉 둘러보다 두 별을 각각 2초씩 보면 그 사이 시냅스가 강해진다.
- **결과:** 별을 클릭해 2초만 머무르면 — 읽으려던 것뿐이어도 — 그 별이 재점화(망각 리셋)·재성형(형태/색 지터)되고, 추상화된 별이면 내용까지 다시 흐려진다. 우주를 훑어보는 행위가 곧 모든 별을 "다시 떠올린 것"으로 처리돼, 안 본 별일수록 어두워지고 단순해지는(요지화) 자연 곡선이 둘러보기만으로 무너진다.

## 목표 (to-be)
<!-- 변경 후 참이어야 하는 동작·계약. 구체적으로. -->

### 1. 클릭 = 읽기전용 열람 (부작용 0)
- 별을 클릭하면 그 별의 내용(조각/흐려진 현재 내용/원본 토글, change 32의 3D 포트레이트·메타와 함께)을 **즉시 읽기전용으로** 보여준다 — 2초 dwell 자동 회상은 **폐기**한다.
- 이 열람은 **어떤 부작용도 일으키지 않는다**: `last_recalled_at` 불변, `evolution_history` 새 행 없음, AI 재작성 잡 미적재, 공동 회상 쌍 미누적. 둘러보기는 별을 절대 안 바꾼다.
- 내용을 부작용 없이 받기 위해 **읽기전용 조회 경로**가 필요하다(읽기전용 RPC 신설 또는 `RecallMemory`에 무부작용 모드 — 구현 재량, 단 진짜 `NO_SIDE_EFFECTS`·`records` 불변). `derived_text`(최신 AI 변형 표시 내용)도 읽기전용으로 반환.

### 2. "이 별 자세히 보고 회상하기" 버튼 = 진짜 회상
- 패널에 **`이 별 자세히 보고 회상하기`** 버튼을 둔다. 누르면 비로소 `RecallMemory`(부작용 포함)가 발화한다 — 재점화(`last_recalled_at=now`)·PE 게이트 재공고화 재성형·공동 회상 강화·(stage≥임계면) AI 재작성.
- 회상 후 패널은 회상 결과(갱신된 내용·형태)로 반영한다(현재 `shown` 동작 계승).
- 공동 회상 강화의 트리거가 **2초 dwell → 버튼 누름**으로 바뀐다(강화 식·배치·멱등은 불변, 시점만 의도적 행동으로).

### 3. 재회상 쿨다운 (1시간 / 하룻밤)
- 같은 별의 **재회상은 마지막 회상 후 쿨다운이 지나야** 가능하다 — `now − last_recalled_at ≥ recall_cooldown`(values 신규, 기본 **1시간**). 야간 공고화를 한 번 보내면(하룻밤) 당연히 쿨다운보다 길어 재회상이 열린다. 데모(가상 시계)도 같은 값으로 `virtualNowMs` 기준 판정.
- 한 번도 회상한 적 없는 별의 **첫 회상은 항상 가능**(쿨다운은 *재*회상에만).
- 쿨다운 중에는 버튼이 **비활성**이고 "아직 회상할 수 없어요(다음 회상 가능까지 ~N분)" 안내를 보인다. 쿨다운은 **BE가 권위로 강제**한다(쿨다운 중 회상 요청은 서버가 거부/무시 — FE 비활성은 UX 보조). 읽기전용 열람은 쿨다운과 무관하게 언제나 가능.

## 범위 / 비목표 (회귀 경계)
<!-- 무엇을 바꾸고, 무엇은 안 바꾸나. -->

**바꾼다**
- FE 회상([features/recall](../../frontend/src/features/recall)): 2초 dwell 자동 회상 제거 → 클릭 즉시 읽기전용 내용 표시 + `이 별 자세히 보고 회상하기` 버튼 + 쿨다운 상태(비활성·안내). `MemoryPanel`·`co-recall`(트리거 시점)·`recall.ts`(읽기전용 vs 회상 호출 분리).
- BE: ① 내용 **읽기전용 조회 경로**(부작용 없이 `Record`·`fragment_text`·`derived_text` 반환), ② `RecallMemory`에 **회상 쿨다운 게이트**(마지막 회상 후 `recall_cooldown` 미경과면 재회상 거부/무시). `internal/memory`(handler/service/repo).
- proto(`memory.proto`): 읽기전용 RPC 신설 **또는** `RecallMemory` 모드 + 쿨다운 응답(다음 가능 시각/거부 사유) — **append-only**(필드/RPC 추가만, 헌법5).
- 값(`spec/values.yaml`): `recall_cooldown`(신규, 기본 1시간 — ms 또는 hours). `recall.dwell_ms`는 회상 트리거에서 폐기되나 다른 용도(애널리틱스 등)가 있으면 유지·없으면 정리.

**안 바꾼다 (회귀 경계)**
- 회상이 *실제로 일어날 때*의 **재공고화 물리는 불변**(PE 게이트·reshape 식·`brightness/hue/form` 누적 한계·AI 재작성 게이트 `stage ≥ stage_threshold`, spec 23·54). 이 변경은 회상의 **트리거/빈도**만 바꾼다(2초 dwell·매 열람 → 버튼·쿨다운).
- 원본 `records` 불변·편집/삭제 경로 없음·원본 전체는 "원본 일기 전체 보기"로만(헌법1). `evolution_history` append-only(헌법1·2). user_id 격리(plan 11·24).
- 공동 회상 강화 식·`reinforceLinks` 배치·`batch_id` 멱등은 그대로(트리거 시점만 버튼으로). 헵 강화 자체는 유지.
- change 32의 별 3D 포트레이트·메타(추상화 라벨/게이지)·발췌 펼치기·변천사(24)·메인 우주 렌더(StarField)·잠든 별 탐색(12)은 그대로. 단, **잠든 별 재점화는 이제 회상(버튼)에서만** 일어난다(둘러보기로는 안 깸 — 의도된 효과).

## 수용 기준 (인수 조건)
<!-- 변경 완료의 판정 기준. job의 "인수 조건" 섹션으로 복사된다. -->
1. 별을 클릭하면 2초 dwell 자동 회상이 **더는 없다** — 클릭 즉시 그 별 내용(조각/흐려진/원본 토글)이 읽기전용으로 뜬다(3D 포트레이트·메타와 함께, change 32 유지).
2. **클릭만으로는 어떤 부작용도 없다**: 클릭 전후로 `last_recalled_at` 불변, `evolution_history`에 새 행 없음, AI 재작성 잡 미적재, 공동 회상 쌍 미누적, 별 레이어(밝기/형태/색) 무변(검증: 클릭 전후 GetUniverse·변천사 길이 동일).
3. 패널에 **`이 별 자세히 보고 회상하기`** 버튼이 있고, 누르면 `RecallMemory`가 발화해 재점화·재공고화(PE 통과 시 재성형)·공동 회상 강화가 일어나며 패널이 회상 결과로 갱신된다.
4. 회상 직후 **쿨다운**(values 신규, 기본 1시간) 안에는 같은 별 재회상이 막힌다 — 버튼 비활성 + 다음 가능 안내. 쿨다운 경과(또는 야간 공고화 1회) 후 다시 가능. 한 번도 회상 안 한 별의 첫 회상은 가능.
5. 쿨다운·무부작용은 **BE가 권위로 강제**한다 — 쿨다운 중 회상 요청을 서버가 거부/무시하고(FE 비활성은 보조), 읽기전용 조회는 records/memories/evolution_history를 수정하지 않는다(진짜 NO_SIDE_EFFECTS). user_id 격리 유지.
6. 쿨다운 임계는 `spec/values.yaml`(신규 `recall_cooldown`)에서 오며 코드 하드코딩이 아니다. 데모 가상 시계도 같은 값으로 판정한다(하룻밤 빨리감기로 쿨다운 해제 체험 가능).
7. 회귀 없음: 원본 불변·"원본 일기 전체 보기" 토글·변천사(24)·별 3D 포트레이트/메타/발췌(change 32)·재공고화 물리(23/54)·공동 회상 강화 식·메인 우주 렌더가 그대로. FE build·lint·steiger·test / BE vet·build·test 통과.
