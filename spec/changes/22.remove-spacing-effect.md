# 22. 변경: 간격 효과(spacing) 제거

> [plan/11.recall-reinforce](../plan/11.recall-reinforce.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 불필요 이론 솎기. 헵 강화를 "같이 읽으면 연결/강화" 한 줄로 단순화한다.

## 현재 (as-is)

- 별을 2초+ 능동 열람하고 다른 별을 열람하면 두 별 쌍에 `co_recall` 강화 이벤트가 쌓인다([co-recall.ts](../../frontend/src/features/recall/model/co-recall.ts)).
- 강화량 = `CO_RECALL_DELTA(0.05) · spacingBoost(gapDays)`, `spacingBoost = 1 + SPACING_GAIN·clamp(gapDays/SPACING_REF_DAYS,0,1)` (1~2×) — **마지막 회상 이후 간격이 클수록 더 큰 부스트**(간격 효과, Katz 2021).
- 별 쌍별 마지막 강화 시각(`pairLastSeen`)을 세션이 추적해 gap을 계산.

## 목표 (to-be)

1. **간격 부스트 제거.** 강화량을 고정 `CO_RECALL_DELTA(0.05)` 로 단순화. 같이 2초+ 읽은 두 별은 항상 동일량 강화(이미 연결돼 있으면 +0.05, 상한 1.0).
2. **간격 추적 제거.** `pairLastSeen`·gap 계산·`spacingBoost`·관련 상수 삭제.

## 범위 / 비목표 (회귀 경계)

**바꾼다:**
- [co-recall.ts](../../frontend/src/features/recall/model/co-recall.ts): `spacingBoost`·`pairLastSeen`·gap 로직 제거, 강화량 고정.
- `spec/values.yaml`: `SPACING_GAIN`·`SPACING_REF_DAYS` 제거.
- plan 11 + [policy/domain/synapse.md](../policy/domain/synapse.md) §헵 강화의 간격 효과 서술 삭제.

**안 바꾼다(회귀 경계):**
- 능동 인출 게이트(`DWELL_MS=2000`, 2초+ 열람) 유지 — 인출 연습 효과는 남긴다.
- `CO_RECALL_DELTA=0.05`·상한 1.0·`batch_id` 멱등·디바운스 flush 유지.
- 헵 강화의 핵심(같이 떠올리면 연결 강해짐)은 그대로.

## 수용 기준 (인수 조건)

1. 두 별을 같이 2초+ 읽으면 간격과 무관하게 항상 `+0.05` 강화된다(몰아보기 1× = 하루 띄움 1×).
2. 코드·values에서 `spacing`/`SPACING_*`/`pairLastSeen`이 사라진다.
3. dwell 2초 게이트와 멱등 강화는 회귀 없이 동작한다.
