# 27. 변경: 데모 시뮬레이션 = 실제 로직과 동치 (백엔드 비즈니스 로직의 충실한 FE 포트)

> [plan/19.engram-experience](../plan/19.engram-experience.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 데모가 "비슷해 보이는 별도 구현"이라 백엔드가 바뀌면 조용히 드리프트한다(추상화 단계가 데모에서 영원히 0인 버그가 그 예).
> 데모를 **실제 제품과 동일한 로직으로 시뮬레이션**하게 — 백엔드 비즈니스 로직을 FE에 *충실히 포팅*하고 단일 출처에 묶어 — 다시는 어긋나지 않게 한다.

## 현재 (as-is)

데모 우주는 백엔드 비즈니스 로직(Go: `backend/internal/memory`·`internal/job`)을 FE에서 **느슨하게 근사**한다
([shared/lib/demo/data.ts](../../frontend/src/shared/lib/demo/data.ts)·[simulate.ts](../../frontend/src/shared/lib/demo/simulate.ts)).
별도 구현이라 서버 로직이 바뀌어도(jobs 32–38) 데모가 자동으로 따라가지 못하고 **드리프트**한다. 실제 어긋난 사례:

- **야간 공고화 — 추상화 미작동(확인된 버그).** `toStar()`가 `abstractionStage`를 안 싣고(→ proto 기본 0), `demoConsolidate()`는 옛
  **`formSeedDelta`**(나이 트리거)만 올리고 `abstraction_stage`(반경 트리거, change 20)를 승급하지 않는다. 렌더(job 35)는 `aShape.w =
  abstractionStage`로 형태 단순화를 구동하므로, 데모에선 **한 달을 아무리 보내도 추상화 단계가 0** → 요지화가 영영 안 보인다.
- **링크.** 데모 전용 휴리스틱(같은 날·같은 mood 캡, `VALUES.demoLinking.*`)이라 서버의 의미 KNN + 시간 보너스 + **감정 유사도(job 37)** +
  흥분성 할당(22)과 다른 결과를 낸다.
- **조각화/감정.** 키워드 추정 + `DEMO_MAX_FRAGMENTS=3` 근사라 서버 AI 추출과 다르다(작성 흐름은 change 25가 다룸).
- **거리·밝기·재공고화·감쇠.** 일부는 공유 식(`entities/memory/model/activation.ts`)을 쓰지만, 데모 전용 바닥(`FLOOR`)·PE/gist 근사가 섞여
  서버 식과 부분만 일치한다.

즉 데모는 "검증 도구"를 표방하지만 실제 동작과 동치가 아니고, 유지보수 때 조용히 깨진다.

## 목표 (to-be)

**원칙: 데모는 실제 제품과 *동일한 로직*으로 시뮬레이션한다.** 백엔드는 DB를 바꿔야 해 비즈니스 로직이 Go에 살고, 데모는 FE에서만 도므로,
그 알고리즘을 **FE(TS)에 충실히 다시 구현(포팅)**해 둘이 *행동적으로 동치*가 되게 한다. 코드 자체는 Go↔TS라 공유 못 해도, **알고리즘과
정전 값은 단일 출처**이며 데모는 그 거울이다(중복 구현 비용은 감수 — 검증 가능성을 위해).

포팅 대상(서버 권위 로직의 FE 미러):

- **야간 공고화(change 20).** `demoConsolidate()`를 서버 패스의 충실한 포트로 교체 — 반경 스코프 ≤60·시간링크 약화/의미링크 강화·spread·
  **`abstraction_stage` 반경 트리거 단조 승급(GREATEST)**·마지막 연결 보호(degree≥1)·re-KNN. `toStar()`/생성 경로가 `abstractionStage`를
  항상 실어 보낸다. → 위 추상화 버그 해소(시간이 지나면 데모 별의 stage가 0→4로 올라 형태가 단계적으로 단순해진다).
- **링크 가중치.** 의미 유사도(클라 근사 임베딩/KNN) + 시간 보너스 + **감정 유사도(job 37)**, 후보 게이트 `cos≥0.75`, 캡 0.79 — 서버 식과 동일.
  흥분성 편향 할당·소프트 억제(22)도 같은 식.
- **거리=강함(change 18) · 밝기=반경(change 19) · 감쇠(12) · 간격효과 제거(38).** 이미 공유 가능한 순수 식은 데모·실렌더가 같은 함수를
  호출하고, 데모 전용 바닥/근사는 제거한다.
- **재공고화 PE 재성형(23).** PE·strength·magnitude·양방향·직접 이웃 한정·내용 변형(54, AI만 프리셋)을 서버 식과 동일하게.
- **단일 출처.** 데모 전용 `demo_*` 튜닝 노브(상한·바닥·링크 가중치·gist 근사)를 폐지하고 `spec/values.yaml` 하나로 수렴(프리셋 일기/사전분절
  *콘텐츠*만 데모 고유). 공유 가능한 공식은 `entities/**/model`의 **플랫폼 무관 순수 함수**로 추출해 데모·실렌더가 같은 함수를 import한다.
- **드리프트 방지 계약(parity contract).** 같은 입력 → 같은 결과를 고정한다: (a) 가능한 로직은 순수 TS로 추출해 양쪽이 공유, (b) Go↔TS로
  갈릴 수밖에 없는 부분은 **골든 픽스처 대조 테스트**로 동치를 못 박고, (c) policy/values 단일 출처를 강제. → jobs 32–38 같은 변경 때 데모가
  자동으로(또는 테스트가 실패해 명시적으로) 따라간다.

> 정직한 환원 불가 층: 임베딩/LLM은 데모에 없어 *진짜* 동치가 불가능하다 — 의미 임베딩·KNN은 **클라 근사**, 조각화/감정은 **프리셋**(change 25).
> 그 경계만 명시적 근사로 남기고, **그 외 결정론 비즈니스 로직은 서버와 동치**로 만든다.

## 범위 / 비목표 (회귀 경계)

**범위**: `shared/lib/demo/**` 시뮬레이션 엔진(공고화·재성형·링크·할당·감쇠·추상화)을 서버 비즈니스 로직의 충실한 포트로 재작성, 데모 전용
근사·`demo_*` 노브 제거, 공유 공식의 순수 함수 추출, parity 대조 테스트. `toStar()`가 `abstraction_stage`를 포함한 서버 동등 상태를 싣는다.

**비목표 (회귀 경계)**:
- 백엔드 동작·proto·RPC·sqlc·DB schema를 바꾸지 않는다(데모는 클라 미러일 뿐). 데모는 서버·AI·임베딩을 호출하지 않는다.
- **기능 자체**는 다른 change 소유 — [24](24.continuous-clock-speed-control.md)=연속 시계/배속, [25](25.demo-preset-diary-flow.md)=프리셋 작성
  흐름, [26](26.tutorial-validated-neuroscience.md)=튜토리얼. 27은 그 아래 **시뮬레이션 엔진의 로직 동치**만 책임진다(중복 금지). 의존: 24·25 위에 얹는다(24·25 → 27).
- 실렌더(인증 우주)의 동작을 바꾸지 않는다 — 공식 추출은 **행동 보존 리팩터**이고 실계정 경로 회귀를 만들지 않는다.
- 임베딩/LLM 동치는 비목표(환원 불가 — 클라 근사·프리셋으로 남긴다).
- 헌법 불변식(원본 불변·삭제 없음·서버 비좌표·`A_MIN=0.05` 단일 바닥·model 순수성)은 그대로.

## 수용 기준 (인수 조건)

1. 데모 야간 공고화가 서버 change 20 패스(반경 스코프 ≤60·시간링크 약화/의미링크 강화·spread·`abstraction_stage` 반경 승급·마지막 연결
   보호·re-KNN)를 충실히 수행하고, 시간이 지나면 데모 별의 `abstraction_stage`가 0→4로 올라 **형태가 단계적으로 단순(요지화)해진다**(현 버그 해소).
2. 데모 별 생성·링크가 서버 식과 동일 결과 — 의미 KNN 게이트(`cos≥0.75`)·시간 보너스·**감정 유사도(job 37)**·캡 0.79·흥분성 할당·소프트 억제 —
   를 내고, 데모 전용 링크 휴리스틱(`demoLinking` 등)이 제거된다.
3. 거리=강함·밝기=반경·감쇠·재공고화 PE 재성형이 실렌더와 **동일한 순수 함수/식**으로 계산되고, 데모 전용 바닥·근사가 제거된다.
4. 데모 전용 `demo_*` 튜닝 노브가 사라지고 데모·실로직이 `spec/values.yaml` 단일 출처를 공유한다(프리셋/콘텐츠만 데모 고유).
5. parity 보장 수단이 존재한다: 공유 가능한 공식은 순수 함수로 추출돼 데모·실렌더가 같은 함수를 쓰고, Go↔TS로 갈리는 부분은 **골든 대조
   테스트**로 동치가 고정된다(서버 로직이 바뀌면 데모 미러도 따라가거나 테스트가 실패한다).
6. 임베딩/LLM 의존부(의미 임베딩·KNN·조각화/감정)는 명시적 근사·프리셋으로만 남고, 그 경계가 문서/코드에 표기된다.
7. 백엔드·proto·RPC·sqlc·DB·실렌더(인증 우주)에 회귀·변경이 없다.
8. build·lint·typecheck·steiger 통과. 기존 데모 체험(회상·잠든 별·일기 목록·페르소나 전환·시간 흐름)·실계정 우주 회귀 없음.
