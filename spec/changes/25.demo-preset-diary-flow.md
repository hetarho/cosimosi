# 25. 변경: 데모 새 별 띄우기 = 프리셋 일기 작성 흐름 (production UI·로직 정렬)

> [plan/47.demo-free-mode-onboarding](../plan/47.demo-free-mode-onboarding.md)의 이미 구현된 동작을 바꾸는 **변경 제안**(WHAT 델타).
> 데모 `새 별 띄우기`를 *즉시 랜덤 별*에서 **production과 동일한 일기 작성 → 별 나누기(조각화) → fan-out** 체험으로 바꾸고,
> 데모 전용 강조/단축을 걷어내 **우리 프로덕트의 실제 로직**(생성·연결·밝기·드리프트)이 그대로 돌게 한다.

## 현재 (as-is)

데모 자유모드 하단 중앙 `새 별 띄우기`는 기록 폼을 열지 않고 **즉시 랜덤 별 1~5개**를 추가한다([use-demo-flow.ts](../../frontend/src/pages/home/model/use-demo-flow.ts)
`addDemoRandomStars` → [shared/lib/demo/data.ts](../../frontend/src/shared/lib/demo/data.ts) `demoAddRandomStars(count, demoToday())`).
production의 진짜 작성 흐름([features/record-memory/ui/MemoryForm.tsx](../../frontend/src/features/record-memory/ui/MemoryForm.tsx):
compose → `별로 분해`(SegmentMemory) → review(조각 감정/강도 편집) → `RecordMemory` 동기 fan-out)은 데모에서 쓰지 않는다.

데모 생성 경로엔 **데모 전용 강조/단축**이 박혀 있다:

- 조각화: `DEMO_MAX_FRAGMENTS=3` 상한 + 빈 줄/문장 분절 + 한국어 키워드 mood 추정([data.ts](../../frontend/src/shared/lib/demo/data.ts) `demoAddRecord`).
- 연결: 데모 전용 휴리스틱(같은 날·같은 mood·hot 별 캡)과 데모 링크 가중치(`VALUES.demoLinking.*`).
- 밝기/망각: 랜딩과 별개지만, 데모 카드/시연 흔적의 옛 바닥(`FLOOR`)·간이 감쇠가 production 거리=밝기 모델과 어긋날 수 있음.
- 사용자는 일기를 **쓰거나 별이 나뉘는 과정을 보지 못한 채** 별이 튀어나오는 것만 본다.

## 목표 (to-be)

데모 `새 별 띄우기`를 **production 작성 UI와 동일한 흐름**으로 바꾸고, 분절 결과만 프리셋으로 대체한다.

- **읽기전용 프리셋 일기**: `새 별 띄우기`를 누르면 production `MemoryForm`의 compose 화면과 **동일한 UI**가 열리되, 본문 input은
  **read-only**이고 **프리셋 일기 텍스트**(현재 페르소나·`demoToday()` 날짜에 맞춰 큐레이션된 한 편)가 미리 채워져 있다. 날짜도 프리셋.
- **별 나누기 = 동일 UI 조각화**: `별 나누기`(production의 `별로 분해`/SegmentMemory에 해당) 버튼을 누르면, **프리셋 사전분절**(미리
  작성된 조각 텍스트 + 조각별 mood/intensity/valence)이 production과 **동일한** segmenting → review → fan-out 애니메이션/시퀀스로 펼쳐진다.
  AI(SegmentMemory RPC)만 프리셋으로 대체하고, **UI·상태머신·표현·타이밍은 production 그대로** 쓴다(가능한 한 `MemoryForm`/`compose.machine` 재사용).
- **production 로직으로 우주 반영**: 펼쳐진 조각 별과 연결은 데모 전용 휴리스틱이 아니라 **production과 동일한 규칙·정전 값**으로 우주에
  들어간다 — 일내 결속·의미 KNN(클라 근사)·**감정 유사도 링크(job 37)**·거리=강함(연결·회상·감정이 반지름을 늦춤, change 18)·밝기=반지름(change 19).
- **데모 전용 강조/단축 제거**: `DEMO_MAX_FRAGMENTS=3` 같은 데모 상한, 데모 전용 mood 키워드 추정, 데모 밝기 바닥(`FLOOR`)·간이 감쇠,
  데모 전용 링크 가중치 등 "데모에서만 특별히 적용되던" 것을 걷어내고 production 정전 값(`spec/values.yaml`)을 따른다. 프리셋(콘텐츠)만
  데모 고유로 남고 *로직*은 production과 일치한다.
- **랜덤 별 생성 폐기**: 자유모드의 즉시 랜덤 별 1~5개 생성(`demoAddRandomStars` 진입점)은 제거한다(프리셋 작성 흐름이 대체).

> AI 부재의 정직한 경계: 데모엔 LLM/임베딩/서버가 없으므로 분절·감정·임베딩 KNN은 **프리셋/클라 근사**다. 이건 불가피한 데모
> 시뮬레이션 층이며, 그 위의 *UI·흐름·연결/밝기/드리프트 공식·정전 값*은 production과 동일하게 맞춘다(데모 전용 식 신설 금지).

## 범위 / 비목표 (회귀 경계)

**범위**: 데모 `새 별 띄우기` 진입점과 그 작성/조각화 흐름, 데모 생성·연결·밝기 로직의 production 정렬, 프리셋 일기/사전분절
코퍼스. production `MemoryForm`/`compose.machine`을 데모가 read-only·프리셋 모드로 재사용하도록 잇는다.

**비목표 (회귀 경계)**:
- 실서버 `RecordMemory`·`SegmentMemory`·AI 추출·임베딩·DB 쓰기·별가루 경제와 **연결하지 않는다**(프리셋이 AI 출력 대체). backend/proto/RPC/sqlc/DB 변경 없음.
- production `MemoryForm`의 **실계정 작성 동작을 바꾸지 않는다**(데모는 read-only·프리셋·서버 미호출 모드만 추가). 실계정 compose→review→submit 회귀 금지.
- 온보딩(페르소나 선택 → 모드 선택), 좌상단 페르소나/시간 버튼, 망원경, 회상, UI 숨김은 그대로(이 change는 `새 별 띄우기` 동작과 생성 로직만).
- 헌법: 원본 일기 불변, 별·시냅스 삭제 없음, 서버 비좌표, `A_MIN=0.05` 단일 바닥 유지. 데모 데이터는 실서버에 안 쓴다.
- 연속 시계/배속/야간 자동 발화는 [change 24](24.continuous-clock-speed-control.md) 소관(데모 야간 공고화 production 정렬은 두 change가 공유하는 경계).

## 수용 기준 (인수 조건)

1. 데모 자유모드에서 `새 별 띄우기`를 누르면 production `MemoryForm` compose와 **동일한 UI**가 열리고, 본문은 **read-only 프리셋 일기**
   (현재 페르소나·`demoToday()` 날짜)로 채워져 있다.
2. `별 나누기`를 누르면 **프리셋 사전분절**(조각 텍스트 + 조각별 mood/intensity/valence)이 production과 동일한 segmenting → review → fan-out
   시퀀스/애니메이션으로 펼쳐진다(AI만 프리셋 대체, UI·상태머신·표현은 production 그대로).
3. 펼쳐진 조각 별·연결이 production 규칙·정전 값으로 우주에 반영된다 — 일내 결속, 의미 KNN(클라 근사), **감정 유사도 링크**, 거리=강함, 밝기=반지름.
4. `DEMO_MAX_FRAGMENTS` 데모 상한, 데모 전용 mood 키워드 추정, 데모 밝기 바닥(`FLOOR`)·간이 감쇠, 데모 전용 링크 가중치가 제거되고
   production 정전 값을 따른다(데모 고유로 남는 건 프리셋 *콘텐츠*뿐).
5. 자유모드 즉시 랜덤 별 생성(`demoAddRandomStars` 진입점)이 더는 호출되지 않는다.
6. 실계정 `MemoryForm` 작성(compose→review→submit→fan-out)이 회귀 없이 동작하고, 데모는 서버를 호출하지 않는다.
7. 별·시냅스·원본 삭제·수정 없음. 데모 데이터는 실서버에 기록되지 않고 demo exit/페르소나 전환 리셋 경계에서 정리된다.
8. 백엔드·proto·RPC·sqlc 변경 없이 FE + values 생성물만으로 구현되고, build·lint·typecheck·steiger 통과. 온보딩·페르소나/시간 버튼·망원경·회상·UI 숨김 회귀 없음.
