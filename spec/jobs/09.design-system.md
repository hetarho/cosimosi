## Acceptance Criteria (from plan/09.design-system)
- [ ] A1 Web Tailwind/global CSS tokens and a DOM-free cross-platform token map (mobile/NativeWind) exist from one token source, imported only at each app's composition boundary.
- [ ] A2 Shared UI primitives are implemented under `shared/ui` with named exports and no imports from feature/domain slices; they render on web and mobile, with `*.native.tsx` siblings only where DOM and RN genuinely differ.
- [ ] A3 Primitive examples/tests cover default, disabled, loading/error where applicable, focus, and keyboard behavior.
- [ ] A4 UI copy in primitives is provided through props or i18n message calls; primitives do not embed product strings.
- [ ] A5 Token color pairs used for text meet documented contrast expectations.
- [ ] A6 Reduced-motion preferences are respected by shared transition styles.
- [ ] A7 Theme/background state is presentation-only and cannot mutate domain or cache data.
- [ ] A8 No universe renderer, mood-color product model, customization catalog, or final product screen is introduced.
