>
> **This is a design job — not the usual developer-verified one.** Completion is a **design-reviewer sign-off**
> against a rubric across **iterative review rounds** (Tier 2 below), not a single build/test pass. The
> Implementation Checklist builds the objective deliverables + the review substrate; the last phase runs the
> review loop until sign-off. Treat T014–T015 as a **loop**, not a one-shot step.

## Acceptance Criteria (from plan/56.2d-ui-design-language)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

**Tier 1 — objective deliverables (checkable)**

- [ ] A1 A living design-language guideline exists (`spec/tech/design-language.md`, 2D section) documenting:
      typographic system, color-role application (incl. 2D emotion accents), spacing/density/layout & hierarchy,
      elevation/depth, per-primitive visual style + composed chrome patterns, 2D motion, iconography, state treatments.
- [ ] A2 A design showcase surface renders the language across the canonical reference set (full primitive catalog in
      all states + the representative composed-screen set) on **web and mobile**.
- [ ] A3 All visual changes are expressed through `packages/ui` (`tokens.ts` + primitives + `base.css`);
      feature/widget/page slices remain free of hardcoded color/size escapes (the Epic-A zero-escape property holds).
- [ ] A4 The shipped Epic A DOM chrome (writing-flow screens) is retrofitted to the new language.
- [ ] A5 The plan-09 a11y baseline is preserved: token color pairs pass documented WCAG contrast; focus rings visible;
      reduced-motion honored; primitive keyboard/aria behavior unchanged.
- [ ] A6 Theme/background state stays presentation-only — no domain, cache, emotion, strength, or layout mutation.

**Tier 2 — iterative designer-review gate**

- [ ] A7 The 2D rubric (hierarchy & focus · typographic rhythm · spacing/density · color-role application & contrast ·
      component consistency · motion coherence · emotional/brand fit · web↔mobile parity) is scored by the design
      reviewer each round as Meets / Needs-work / Blocking.
- [ ] A8 A feedback ledger records every round's notes, each tracked to `resolved` or `deferred` (deferred listed).
- [ ] A9 **DoD:** the design reviewer signs off — every rubric dimension **Meets** and **zero open Blocking** —
      recorded as a dated sign-off. Completion is sign-off-gated and iterative; there is no single sequential pass.

## Implementation Checklist

<!-- How to build it, top to bottom. [P] = parallel (different files, no dependency). Flag (gen)/(migrate)/(gen:values). -->

**Substrate — review protocol + guideline draft**

- [ ] T001 Create the design-review protocol doc `spec/policy/ux/design-review.md` — the 2D rubric dimensions, the
      Meets/Needs-work/Blocking scale, the feedback-ledger format, and the sign-off gate. **Shared with job 39 (3D)** —
      author it generically so 39 reuses it with the 3D rubric set.
- [ ] T002 Start the working draft `spec/tech/design-language.md` with the 2D-section skeleton (typography · color
      application · spacing/layout/hierarchy · elevation · component style · motion · iconography · state treatments);
      fill it as decisions land, finalize at sign-off (T016).

**Review substrate — the showcase the designer inspects**

- [ ] T003 Build the web design-showcase surface by **extending `apps/web/src/pages/test/`** (a new design-showcase
      panel set, reusing its `lib/*-panel` pattern) — render the full `@cosimosi/ui` primitive catalog in every state
      (default / hover / focus / pressed / disabled / loading / error).
- [ ] T004 [P] Add the composed-screen reference set to the web showcase: the writing flow, a star-detail panel mock, a
      HUD, a modal/dialog, a list page, and empty/error states — enough real chrome to judge the language in context.
- [ ] T005 [P] Mirror the showcase on mobile (`apps/mobile/src/pages/test` or the RN test surface) — same catalog +
      composed set via the native primitive siblings.

**Define + apply the 2D language (the design work — iterated with Tier 2)**

- [ ] T006 Typographic system: define scale roles + line-height/measure rhythm; apply via `packages/ui/src/tokens.ts`
      (+ `base.css`); **(gen:tokens)**. Record the rules in the guideline.
- [ ] T007 Color-role application: define surface/text/border/state rules + the **2D emotion-accent rules that
      reference plan 57's palette values** (do not author palette values here); adjust `tokens.ts` color roles as
      needed; **(gen:tokens)**; keep contrast passing.
- [ ] T008 Spacing / density / layout & information-hierarchy principles; apply to the composed chrome patterns
      (panel / HUD / modal / list) in the showcase.
- [ ] T009 Restyle the `@cosimosi/ui` primitives to the component visual style **within plan 09's primitive API**
      (Button, IconButton, Dialog, Toast, Badge, TextField, TextArea, Switch, Checkbox, Tooltip, Skeleton …) — web
      `*.tsx` + `*.native.tsx` siblings, via `primitives/button-styles.ts` etc. No API/behavior changes.
- [ ] T010 2D motion / micro-interaction language on `tokens.duration`/`ease` (enter/exit, state feedback);
      **reduced-motion honored** via the existing `a11y/use-reduced-motion`.
- [ ] T011 Iconography conventions + state treatments (empty / loading / error / disabled) — documented + shown in the
      showcase.

**Retrofit — first application on live screens**

- [ ] T012 Retrofit the shipped Epic A DOM chrome to the language: `apps/web/src/widgets/writing-flow/**` +
      `apps/web/src/features/{write-diary,split-diary,launch-stars,revise-split}/ui/**` and the mobile equivalents
      (`apps/mobile/src/features/{…}`) — consuming **only** tokens/primitives.

**Guardrails — preserve what Epic A earned**

- [ ] T013 Enforce **zero hardcode escapes** (grep guard: no raw hex / `[Npx]` / arbitrary utilities in
      features/widgets/pages); verify the a11y baseline (`packages/ui` contrast test, focus, reduced-motion) and the
      theme-presentation-only invariant still hold.

**Iterative designer-review loop (Tier 2 — the real DoD)**

- [ ] T014 Design-review round: the designer scores the 2D rubric (T001) on the showcase + retrofit; log every note to
      the feedback ledger as `resolved` / `deferred`.
- [ ] T015 Resolve Blocking / Needs-work items → **loop back to T014**. Repeat until every rubric dimension is
      **Meets** and there are **zero open Blocking** items.
- [ ] T016 Finalize `spec/tech/design-language.md` (2D section) + record the **dated sign-off**; update
      `policy/ux/design-review.md`; list any `deferred` items explicitly.
