## 인수 조건 (acceptance — 기획 관점, changes/29에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [ ] A1 별 형태 선택이 **단일 축 3종 룩**(`polyhedron`·`liquid`·`spiky`)이다. form×surface 2축·25 조합·합성 id는 사라진다.
- [ ] A2 `abstraction_stage`가 오르면 형태가 **육안으로 또렷이** 단계 변형된다 — `polyhedron`은 면 수 20→12→8→4로 실제 지오메트리가 바뀌고, `liquid`는 점점 투명·구름화, `spiky`는 가시가 듬성→소멸(다각형). 단조(단계↑ = 더 추상).
- [ ] A3 같은 룩·같은 단계라도 **별마다 실루엣이 확연히 다르다**(랜덤성 확대 — 회전만 다른 게 아님).
- [ ] A4 같은 memory id·stage·seed면 항상 같은 모양(결정론, 세션·기기 무관, Math.random 비사용).
- [ ] A5 색(mood)·크기(intensity)·밝기(거리)는 이 변경으로 안 바뀐다(회귀 경계).
- [ ] A6 렌더가 (룩×단계) 버킷별 InstancedMesh로 분할되고 메시 개수가 상수(≤15)로 묶인다(별 수와 무관). 별이 단계를 넘으면 해당 버킷으로 이동한다. 수천 별에서 프레임 회귀가 없다.
- [ ] A7 미지/구 형태 id는 디폴트 룩(`polyhedron` 단계 0)으로 폴백하고 크래시하지 않는다.
- [ ] A8 FE `build`·`lint`·`steiger`·test 통과, 공유 우주(35/37)·근접 탐험(49)·데모(47/48) 회귀 없음.
