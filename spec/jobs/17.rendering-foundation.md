## Acceptance Criteria (from plan/14.rendering-foundation)
- [ ] A1 A single renderer boundary in `shared/lib` brings up three.js `WebGPURenderer` with automatic WebGL2 fallback; no slice imports `three` directly.
- [ ] A2 `packages/shaders` exists as a pure TSL toolkit (no DOM/native deps) and provides the shared shader/material/geometry helpers; shaders are authored in TSL (no raw-GLSL `onBeforeCompile`).
- [ ] A3 An asset-source port supplies visual bodies; the domain/mirror layer imports no concrete renderer.
- [ ] A4 A post-processing seam renders a TSL bloom/glow pass on the WebGPU path and degrades on WebGL2.
- [ ] A5 A skin is a named preset of non-domain background/environment params; presets are code content; the active skin is selected by one build-time constant (`values.yaml` → `gen:values`); changing it re-skins the universe background.
- [ ] A6 Skins are read through one seam designed so a runtime switch can replace the constant source without changing any consumer ([P4]-ready, not built).
- [ ] A7 The skin/presentation layer cannot mutate domain (emotion / position / strength) — presentation-only ([I3], [I11]).
- [ ] A8 A `/test` panel renders the skinned background + placeholder instanced meshes + the bloom pass + a preset switcher on web (and a mobile dev build), with graceful fallback; no domain data, no force-sim, no real stars/neurons.
- [ ] A9 The RN renderer host is built at MVP fidelity on the shared TSL source (web = open beta · mobile = MVP), with pinned versions and physical-device verification; `packages/shaders` is the single shared shader source for both.

## Implementation Checklist
- [ ] T001 **(library-setup)** Before adding deps, fetch CURRENT official docs + exact versions and **pin** them: `three` (`three/webgpu`, TSL), `@react-three/fiber` v9, three node post-processing, and `react-native-webgpu` (RN host + its three.js/R3F integration, New-Architecture + dev-client requirements).
- [ ] T002 [P] Scaffold `packages/shaders` — pure TS package (package.json, tsconfig, `src/`, single `index.ts`); add to the pnpm workspace and to the steiger / eslint-boundaries config so it stays **DOM/native-free** and importable by both apps.
- [ ] T003 In `packages/shaders`, build the **TSL toolkit**: shared shader/material/geometry node helpers authored in TSL (one source → WGSL + GLSL). No raw GLSL / no `onBeforeCompile`.
- [ ] T004 Add the **web renderer boundary** in `apps/web/src/shared/lib`: R3F v9 canvas host + three.js `WebGPURenderer` with automatic WebGL2 fallback; own `await init()`, frame `present`/submit, resize, color management, pixel-ratio cap. Slices mount the host; none import `three`.
- [ ] T005 Add the **asset-source port** (`VisualBodySource`: shader | glTF | primitive) consumed by the boundary; the domain/mirror layer imports the port, never `three` (ARCHITECTURE §3.4).
- [ ] T006 Add the **post-processing seam**: a TSL bloom/glow pass via three's node post pipeline, behind the boundary; verify on WebGPU and degrade on WebGL2.
- [ ] T007 [P] Build the **skin presets**: each skin = a named object of non-domain ambiance params (background/starfield, fog, exposure, bloom defaults, camera mood) + its TSL background/environment node graph; presets live as **code content** (a `packages/shaders` / render-config module), not in `values.yaml`.
- [ ] T008 Add the **skin seam** (`SkinProvider` + `useSkin()`) reading the active skin; design it so a future runtime switcher ([P4]) can swap the constant source with **no consumer change**. Guard: the skin layer performs **no domain writes** (emotion/position/strength).
- [ ] T009 **(gen:values)** Add `rendering.active_skin`, `rendering.max_pixel_ratio`, `rendering.instance_bucket_size` to `spec/values.yaml`; run `pnpm gen:values`; consumers import the generated constants from `packages/config` — **no hardcoded literals**. `active_skin` selects the active preset.
- [ ] T010 Add the **`/test` demo panel** (via `apps/web/src/pages/test` + `shared/test-panel`): skinned background + 2–3 placeholder `InstancedMesh` buckets + a `Line2` fat line + the bloom pass + a preset switcher + a demo orbit camera; on web; exercise the WebGL2 fallback. No domain data, no force-sim.
- [ ] T011 Add the **RN renderer host**: an `apps/mobile` `*.native` sibling of the boundary using `react-native-webgpu`, hosting the same `WebGPURenderer` + `packages/shaders` TSL source at **MVP fidelity** (lower instance counts / fewer post-FX acceptable). Metro resolves three's WebGPU build; New Architecture (RN ≥ 0.81); custom dev client (no Expo Go); pinned versions.
- [ ] T012 Verify the RN path on a **physical iOS + Android** dev build (skinned background + placeholder meshes render); record device caveats (emulator software-adapter fallback, `present()`/async `init()` differences).
- [ ] T013 Quality gates: FE + mobile typecheck/lint cover the new package + boundary; steiger / eslint-boundaries confirm `packages/shaders` stays DOM/native-free and that no slice imports `three` outside the boundary.
- [ ] T014 Scope guards (grep + review): no domain object rendering (stars `EpisodicMemory` / neurons / synapses / real `Emotion` nebula), no force-sim, no end-user runtime skin UI ([P4]); the skin layer makes no domain writes ([I3][I11]); positions are never stored ([I5]).
