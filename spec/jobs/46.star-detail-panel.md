## Progress (in flight)

**Decision (flagged at implement):** the 변천사 data source takes the **FE-port + stub** path — `features/star-provenance`
is built against a typed domain read port with a stub `api` adapter, deferring the real `GetProvenance` wire to plan 46
(Epic G). Job 46 stays FE-only (no proto/BE diff, per T010); A4 is verified by rendering a fixture response.

**Done — T001 (shared model):** `@cosimosi/universe` `star-detail.machine.ts` (`closed → meta → provenance`;
`RECALL`/`OPEN_DIARY` as no-op self-handlers so the emitted intents leave the phase intact) + the pure
`resolveSelection(selectedNodeId, stores)` selector (`episodic | neuron | gist | none`; gist recognized via an injectable
`gistMemoryId` recognizer so a gist body routes to the paid view (A7) without this resolver knowing the gist-body id
format) + machine tests. Selection stays owned by `universe-navigation.machine` (the canvas's actor).

**Done — the full WEB vertical (T002–T008, web):**

- `features/star-meta` — episodic meta (seed-driven glyph via `normalizeSeed`/`moodColor`, emotion, brightness via
  `effectiveBrightness` stub, written date, `effectiveStrength`) vs neuron info-only with **no emotion** ([I3]); every
  derived value read from the shared read-time functions, none re-derived (A2). _(The star "shape" is a flat seed-driven
  glyph — `@cosimosi/universe-render` exposes no small body-preview component and the panel must not import three (§3.4).)_
- `features/current-memory-text` — a pure `{ text }` presentational read (A3, no clock/spend/mutation). **`current_text`
  is not on the GetUniverse mirror and no single-memory text read exists**, so the widget passes `null` and the feature
  shows an unavailable note until a representation read lands (same deferral as provenance, option A).
- `features/star-provenance` — the 변천사 list ui + a typed `ProvenanceEntry` model + a **stubbed reader** (`api/get-provenance.ts`,
  returns `[]`) the provenance-export read rebinds; fetched via Query only on entering the provenance view.
- `widgets/star-detail` (ui + config) — the side-sheet over the canvas (no renderer remount, A1); subscribes to the
  selection via `useSelector` on the injected nav actor; composes the three features + the three hand-off buttons;
  imports no three/visual entity (§3.4, audited).
- **app-layer actor-lift (T006, web):** `UniverseCanvasWidget` now takes an optional `navigationActorRef` (owns its own
  when absent, for the test pages); `UniverseHomePage` creates the actor and passes it to both the canvas and the panel,
  and records the `RECALL`/`OPEN_DIARY`/gist intents for their downstream consumers.
- i18n keys (literary ko + en); `steiger.config.js` single-ref exemption for the star-detail vertical (matching the
  writing-flow/universe-time precedent); component tests (`StarMeta`/`CurrentMemoryText`/`StarProvenanceList`).
- Gates green: `pnpm --filter @cosimosi/web build` · `lint:web` · web tests (41) · `lint:raw-strings` · `check:gen`.

**Remaining — the MOBILE vertical (A11 parity) + mobile tests:** mirror the three features + widget for `apps/mobile`
(RN primitives + tokens-based StyleSheet, NOT Tailwind — NativeWind v5 is incompatible with bare-RN here) with a
bottom-sheet host, and lift the nav actor in the mobile `UniverseScreen`/canvas the same way; then the mobile component
tests and the mobile gates. The shared `model` (the machine + resolver) and the provenance `api`/`model` are reused verbatim.

## Acceptance Criteria (from plan/35.star-detail-panel)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [ ] A1 Clicking a node on the universe canvas (selection non-nil in the canvas machine, [23]) opens
      `widgets/star-detail` **over** the running canvas without remounting the renderer; deselecting / a close affordance
      dismisses it ([D1][U3]).
- [ ] A2 For an **episodic** (big) star the meta block shows **shape · emotion color · brightness · 작성일 · 강도 ·
      current forgetting state**; for a **neuron** (small) star it shows **info only, with no emotion** ([D1][I3]); all
      derived values come from `packages/memory-logic` read-time functions, not re-derived in the panel.
- [ ] A3 An episodic star's **forgotten current memory text** is shown (rendered faded per decay stage where Epic D
      provides it; full text until then), and viewing it is **free** — it **advances no clock**, **spends no 별가루**, and
      **restores no content** ([R1][G1][F1][T3][I8][I10]).
- [ ] A4 A **변천사 보기** view renders the star's shape + a **time-ordered stage-text list** with **kind**
      (생성/요지화/재공고화) and **source** (원본/시스템/사용자) labels, populated by the `GetProvenance` read (created
      baseline synthesized at read + `memory_provenance` entries, CC5); distortion is **not** separately flagged
      ([R8a][D1]).
- [ ] A5 **회고하기** (shown only for an episodic star) **opens the recall flow** (plan 36 / job 47) — this panel emits
      the request and does **not** itself recall, reconsolidate, price, or spend 별가루 ([R1][R6]).
- [ ] A6 **원본 일기 보기** emits the navigation intent to the diary reader at the star's origin diary (plan 47, [D5]);
      this panel does not build the reader.
- [ ] A7 A **gist (요지) star** selection routes to the paid gist-view surface (plan 34 / job 45, [R8]), not this
      panel's free-text block; a clicked **neuron** shows the info-only meta with no episodic buttons.
- [ ] A8 The panel exposes **no** override of a star's emotion / strength / position and **no** placement or
      meaning-layer control — it only reads ([I3][I11]); the provenance view shows representation history only and never a
      mutable original ([I2]).
- [ ] A9 The panel's **control-state** is one XState machine (`closed → meta → provenance`); the selected id stays
      owned by the **canvas** machine (single source) and the fetched provenance lives in Zustand/Query, not in machine
      context (§3.2).
- [ ] A10 **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in a
      literary / restrained Korean voice.
- [ ] A11 **web + mobile parity:** the same widget + three features run on both apps, sharing `model` / `api`
      verbatim; `*.native.tsx` siblings exist **only** for the panel / sheet primitives (§3.5).
