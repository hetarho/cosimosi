## Result (partial — 2026-07-05)

**Done & gate-green:**
- **R011 (A4) — optimistic launch no longer flickers.** `use-universe.ts` (both apps) now merges server truth with
  any optimistically-launched memory not yet visible server-side (server wins on id; optimistic-only rows survive an
  eager refetch / stale GET cache), and still clears all stores on a signed-out empty read (per-user isolation
  preserved). Fixed identically in web + mobile. `typecheck`/`lint`/`test` green.

**Deferred — the bulk promotion (R002, A1–A3) is a separate structural refactor, intentionally not rushed here:**
The write-vertical UI/api is coupled to the **app-local `episodic-memory` Zustand store** (`launch-stars.ts` writes
`useEpisodicMemoryStore`; the R3F layers read it), so a faithful promotion must *also* promote that store + its `@x`
graph into a shared package — a high-blast-radius move touching ~14 import sites across both app builds. Doing it in the
same pass as jobs 34/36/37 risked destabilizing both builds. It should be its own focused session: promote the pure
`model`/`api`/`lib` + the `writing-flow` machine (following the `packages/universe/universe-navigation.machine.ts`
precedent) and the app-local entity store, then repoint both apps and move the tests. The duplication is currently
**byte-identical** (0 drift verified), so nothing is broken today — the risk R002 addresses is *future* drift +
untested mobile copies.

- [ ] A3 **The write-vertical tests move with the code and run for both apps.** The Epic A logic tests
      (proposal/pick-awaken-seed/launch-stars/latent-field/contributors/channels/machine) live next to the promoted

- [ ] A5 **No behavior change / no regression.** The write flow (write → split → revise → launch → star appears +
      awaken) behaves identically on web and mobile; the app-shell/machine tests and `pnpm lint:fsd:layout` stay green.
