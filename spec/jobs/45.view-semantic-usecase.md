## Acceptance Criteria (from plan/34.view-semantic-usecase)
<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->
- [ ] A1 `ViewSemantic(userScope, episodicMemoryId, stage)` returns the **pregenerated `semantic_stages[stage]` text**
  read-only; it writes **nothing** — no `current_text`, `seed`, `recall_count`, brightness, gist timer,
  `memory_provenance`, or clock advance is touched. ([R8][I2][I8])
- [ ] A2 The gist text is **never generated** in this unit — it is read from `semantic_stages`, pregenerated at launch
  by the semanticize job; `ViewSemantic` calls **no** LLM. ([C7][R8a])
- [ ] A3 A view is **refused** (canonical error) when `semantic_stages` is NULL (not yet pregenerated) or when
  `stage > semantic_stage` (the memory has not *risen* to that stage) — the unit never fabricates a text for an
  unreached stage; the reached stage is **server-authoritative**. ([C6][V9], §2.9#8)
- [ ] A4 `ViewSemantic` derives a **gist-depth signal** (deeper stage ⇒ cheaper) and **spends via the consumer-owned
  `SpendGate`** carrying that signal — it **never** hardcodes or computes a Twinkle price; the spend is a
  **precondition** of the read (a gate refusal returns no text). ([R8][G4][G1], CC2/CC3)
- [ ] A5 In Epics C–E the bound `SpendGate` is the **no-op allow-all** default (viewing works with no economy, nothing
  charged); Epic F rebinds the real balance-check-and-deduct against the gist cost curve — with **no change** to
  `ViewSemantic`. ([G1][G2])
- [ ] A6 `ViewSemantic` performs **no reconsolidation and no episodic mutation** — it neither invokes the
  reconsolidation predicate (plan 32) nor resets brightness/decay/timer/LTP (plan 33); recall/reconsolidation are
  episodic-only and separate. ([R1][R8])
- [ ] A7 Reading a gist **does not advance the universe clock** — it is not one of the two advance triggers; the clock
  is unchanged. ([T3][T2][I10])
- [ ] A8 The `ViewSemantic` RPC is unary and **not** `NO_SIDE_EFFECTS` (it spends), the handler is **thin**
  (proto↔domain map + call), and no proto/sqlc type leaks into the use-case or pure domain; the `SpendGate` and the
  memory repository are consumer-owned interfaces. (§2.4/§2.7/§2.9#4/#6/#7)
- [ ] A9 The memory load and the spend are **scoped to the authenticated user**; a memory belonging to another user is
  not-found, and `pnpm lint:persistence` passes. (§4)
