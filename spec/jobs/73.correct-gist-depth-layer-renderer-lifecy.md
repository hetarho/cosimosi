## Acceptance Criteria (from code-review/05.plans-35-53-runtime-and-architecture-aud)

<!-- R013. Correct the renderer contract without changing product data or choreography. -->

- [ ] A1 **Band haze is visible from both camera directions.** Every horizontal `BandFog` slice
      renders from above and below (double-sided or equivalent paired geometry), while preserving
      additive blending, no depth write, negative render order, and raycast invisibility. (R013.)
- [ ] A2 **Frame and pick callbacks only observe committed instance data.** `GistStarLayer` performs
      no render-phase ref mutation; instance lookup, count, appearance channels, position mapping,
      and selection mapping publish as one commit-consistent snapshot. An aborted/concurrent render
      cannot expose a new ordering to the old committed mesh.
- [ ] A3 **Instance reorder/count regressions are covered.** Deterministic tests exercise add,
      remove, and reorder updates and prove that each rendered/picked index resolves the same
      `memoryId`, stage, position source, tint, softness, and size from one snapshot.
- [ ] A4 **Rise choreography and resource ownership remain unchanged.** Initial hydration settles
      silently; newly risen stages emit once and ease one-way from hippocampal z; removed stages
      drop state; geometries/materials are disposed once; no per-frame allocation or React-state
      update is introduced.
- [ ] A5 **No unrelated behavior change / no regression.** The two z bands, fog profile/tint/
      intensity, gist coordinates, picking contract, public component APIs, and plan 42 acceptance
      criteria remain intact on web and mobile. No backend, persistence, policy, or tuning value is
      changed.
