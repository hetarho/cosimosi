# Job 18: 데모 자유모드 온보딩 + change09 HUD 통합  (new)

> 구현 작업 문서. 출처 스펙: [plan/47.demo-free-mode-onboarding](../plan/47.demo-free-mode-onboarding.md).
> `/implement-job 18`이 아래 두 체크리스트로 구현한다. 선행 전제는 [changes/09.universe-mode-ux-rework](../changes/09.universe-mode-ux-rework.md) / [jobs/17.universe-mode-ux-rework](17.universe-mode-ux-rework.md)가 완료된 우주 HUD다.

## 인수 조건 (acceptance — 기획 관점, plan/47.demo-free-mode-onboarding에서)

- [ ] A1 랜딩에서 체험 우주를 시작해 `/`에 최초 진입하면 일반 우주 HUD나 기존 `DemoSimPanel`이 먼저 뜨지 않고, `누구의 우주를 탐험해볼까요?` 온보딩이 보인다.
- [ ] A2 온보딩 1단계에는 기존 세 페르소나(`student`, `worker`, `homemaker`)가 모두 표시되고, 선택한 페르소나가 데모 우주의 데이터 출처가 된다.
- [ ] A3 페르소나 선택 후에는 `기능 하나하나 알아보기`와 `자유롭게 탐험해보기` 두 선택지가 보인다.
- [ ] A4 `기능 하나하나 알아보기`는 이번 plan에서 튜토리얼을 시작하지 않는다. 준비 중/TBD 상태를 보여주고, 사용자가 자유모드 선택으로 돌아갈 수 있다.
- [ ] A5 `자유롭게 탐험해보기`를 누르면 데모 flow 상태가 `free`가 되고, change09 이후의 우주 셸 자유모드가 열린다.
- [ ] A6 자유모드에서는 기존 `SessionGate`의 체험 안내 문구와 우상단 별도 `체험 종료` pin이 보이지 않는다.
- [ ] A7 자유모드에서 체험 종료 동작은 햄버거 사이드바 안의 항목으로 가능하며, 실행 시 `exitDemoMode()`와 demo reset 경계를 거쳐 `/landing`으로 돌아간다.
- [ ] A8 자유모드에서는 좌하단 `기억 실험실` 칩, `뇌과학 이론` 칩, 기존 큰 컨트롤러 패널이 보이지 않는다.
- [ ] A9 좌상단 테마 버튼 아래에 페르소나 버튼이 있고, 클릭하면 세 페르소나를 고르는 팝오버가 열린다.
- [ ] A10 자유모드에서 페르소나를 바꾸면 선택된 페르소나의 초기 우주로 전환되고, 가상 시계와 체험 중 추가한 별은 초기화된다. 자유모드 상태는 유지된다.
- [ ] A11 페르소나 버튼 아래 시간 버튼을 클릭하면 `하루 후로 이동`, `한 달 후로 이동`, `처음으로` 팝오버가 열린다.
- [ ] A12 `하루 후로 이동`과 `한 달 후로 이동`은 기존 하루 단위 배치/조용한 재안정화 경로를 사용하고, 시간 이동 전후 별·시냅스 삭제를 만들지 않는다.
- [ ] A13 `처음으로`는 현재 선택된 페르소나를 유지한 채 가상 시계 0과 추가 별 0 상태로 돌아간다. 온보딩으로 돌아가지 않는다.
- [ ] A14 데모 자유모드에서 하단 중앙 `새 별 띄우기` 버튼을 누르면 기록 폼이나 날짜/감정 선택 UI 없이 즉시 별이 추가된다.
- [ ] A15 데모 `새 별 띄우기` 1회 클릭은 랜덤 개수 1~5개의 별을 추가하고, 각 별은 13종 canonical mood 중 하나를 랜덤으로 가진다.
- [ ] A16 데모 랜덤 별의 날짜는 모두 `demoToday()` 기준 오늘이며, 시간 이동 후에는 이동된 가상 날짜가 적용된다.
- [ ] A17 데모 랜덤 별은 기존 데모 연결 생성 규칙(같은 날, 같은 mood, 최근 회상 hot 별 등)을 재사용해 우주와 시냅스에 즉시 반영된다.
- [ ] A18 상단 중앙 `UI 숨기기`를 누르면 데모 페르소나 버튼, 시간 버튼, 하단 `새 별 띄우기`, 햄버거/망원경 등 우주 HUD가 숨고, `UI 보이기` 버튼만 남는다.
- [ ] A19 실계정 우주에서는 온보딩, 데모 페르소나/시간 버튼, 데모 랜덤 별 생성 동작이 나타나지 않는다.
- [ ] A20 데모 flow 상태와 추가 별은 실서버에 기록되지 않고, demo exit/출처 경계 리셋 후 실계정 우주에 섞이지 않는다.
- [ ] A21 백엔드·proto·RPC·sqlc 변경 없이 FE와 values 생성물만으로 구현된다.
- [ ] A22 build·lint·typecheck·steiger를 통과하고, 기존 데모 회상, 잠든 별, 일기 목록, 시간 이동, 페르소나 전환, 일반 모드 `새 별 띄우기` 작성 흐름에 회귀가 없다.

- [ ] T002 (gen:values) `spec/values.yaml`에 `demo_free_mode.random_star_min = 1`, `demo_free_mode.random_star_max = 5`를 추가하고 `pnpm gen:values`를 실행해 `frontend/src/shared/config/values.gen.ts`와 `backend/internal/values/values_gen.go`를 갱신한다. 랜덤 개수는 코드 리터럴이 아니라 생성 상수에서 읽는다.
- [ ] T003 `frontend/src/shared/lib/demo/flag.ts`에 데모 flow 상태(`not_started`/`persona_selected`/`free`/`tutorial_tbd`)와 sessionStorage helper를 추가한다. `exitDemoMode()`와 demo reset 경계에서 flow 상태가 함께 비워지게 하고, `index.ts`로 명시 export한다.
- [ ] T004 [P] `frontend/src/shared/lib/demo/*.test.ts`에 flow 상태 테스트를 추가한다: 신규 체험 진입 기본값, 페르소나 선택 후 상태 전환, `free` 유지 새로고침 시나리오, `exitDemoMode()` cleanup.
- [ ] T005 `frontend/src/shared/lib/demo/data.ts`에 데모 랜덤 별 helper를 추가한다. helper는 13종 canonical mood를 모두 처리하고, 기존 `demoAddRecord`/`demoAddStar`의 연결 생성 규칙을 재사용하며, `demoToday()` 날짜를 호출자가 주입할 수 있게 한다.
- [ ] T006 [P] 데모 랜덤 별 테스트를 추가한다. 랜덤 개수 범위, 날짜가 `demoToday()`로 들어가는지, 13종 mood fallback이 본문 없는 mood에서도 crash 없이 별을 만드는지, 생성 후 `demoStars()`/`demoSynapses()`에 반영되는지 검증한다.
- [ ] T007 랜딩의 체험 진입점(`HeroSection`, `CtaFooterSection`, `TryInUniverse`)을 정리한다. 체험 시작 시 demo 데이터와 flow를 초기화해 매 새 체험 세션이 온보딩부터 시작하고, 기존 `?sim=<id>` 진입이 자유모드에서 `DemoSimPanel`을 자동으로 열지 않게 한다.
- [ ] T008 데모 온보딩 UI를 추가한다. 위치는 `frontend/src/pages/home/ui/`의 새 컴포넌트(예: `DemoOnboarding.tsx`) 또는 change09 이후 우주 셸에 맞는 동등 위치로 두고, `demoPersonaList()` 기반 1단계와 튜토리얼 TBD/자유모드 2단계를 구현한다.
- [ ] T009 `HomePage` 또는 동등 우주 셸에 데모 flow gate를 연결한다. `isDemoMode()`이고 flow가 `free`가 아니면 우주 자유모드 HUD와 `DemoSimPanel` 대신 온보딩을 우선 렌더한다.
- [ ] T010 기존 `DemoSimPanel`을 자유모드 기본 화면에서 제거한다. 컴포넌트는 후속 튜토리얼용으로 남길 수 있지만, `free` 상태에서는 좌하단 `기억 실험실`/`뇌과학 이론` 칩과 큰 컨트롤러가 마운트되지 않게 한다.
- [ ] T011 `SessionGate`와 우주 사이드바 종료 동선을 정리한다. demo 상태에서는 `SessionGate`의 별도 안내/종료 chrome이 없고, 햄버거 사이드바의 `체험 종료`가 `exitDemoMode()` + `resetDemo()` + `/landing`을 수행한다.
- [ ] T012 데모 자유모드 좌상단 컨트롤을 추가한다. 테마 버튼 아래에 아이콘 버튼 2개를 세로 배치하고, 페르소나 팝오버는 세 페르소나 선택/현재 표시/전환 후 reset을 처리한다.
- [ ] T013 데모 시간 팝오버를 연결한다. `하루 후로 이동`/`한 달 후로 이동`은 `runTimeSkip(queryClient, 1/30)`와 `onQuietSettle`을 재사용하고, `처음으로`는 현재 페르소나·`free` 상태를 유지한 채 `resetDemo()`/캐시 무효화/가상 시계 0으로 돌아간다.
- [ ] T014 change09의 HUD 숨김 상태와 데모 컨트롤을 결합한다. UI 숨김에서는 페르소나/시간 버튼, 하단 데모 `새 별 띄우기`, 햄버거/망원경이 숨고 `UI 보이기`만 남는다.
- [ ] T015 하단 중앙 `새 별 띄우기` 버튼의 demo free 분기를 연결한다. 실계정은 기존 작성 폼을 열고, demo free는 values 범위에서 랜덤 count를 뽑아 `demoAddRandomStars(count, demoToday())`를 실행한 뒤 universe/dormant/records 관련 query와 렌더 스토어를 무효화한다.
