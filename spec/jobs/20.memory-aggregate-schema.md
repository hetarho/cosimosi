## Acceptance Criteria (from plan/16.memory-aggregate-schema)
<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->
- [ ] A1 A goose migration creates `diaries`, `episodic_memories`, `neurons`, `neuron_activations`, `synapses`, `embeddings`,
  and `jobs` with the exact columns, types, and keys in the schema above; `pnpm db:migrate` applies cleanly and
  `pnpm db:down` reverses cleanly (up/down round-trip leaves no residue).
- [ ] A2 Every product table has a `user_id TEXT NOT NULL` column, and `pnpm lint:persistence` passes (no table or query skips
  the per-user convention) [U1].
- [ ] A3 `synapses` has `UNIQUE (user_id, neuron_a_id, neuron_b_id)`, and stored rows satisfy the canonical
  `neuron_a_id < neuron_b_id` ordering, so an unordered neuron pair has at most one row per user [L8][I6].
- [ ] A4 `embeddings.vector` is `vector(N)` with `N` sourced from `ai.embedding_dim`, and an HNSW index exists on that column.
- [ ] A5 `episodic_memories` references `neurons` only through `neuron_activations` (no neuron columns on the root); `synapses`
  references `neurons` on both endpoints and no table carries a memory↔memory edge [I4][I6].
- [ ] A6 `episodic_memories.deleted_at` and `neurons.sealed_at` exist and default to NULL, and no migration or Epic A query
  performs a system-driven hard delete of any neuron, memory, or synapse [I1].
- [ ] A7 `diaries` has no `UPDATE` query and no `updated_at` column; representational change targets `episodic_memories`
  fields, never `diaries` [I2].
- [ ] A8 No table stores positions, coordinates, constellations, latent neurons, or nebula color [I5]; the read query returns
  only stored facts (emotion, base_strength, recall_count, universe times, seed, neuron_type, synapse strength +
  co-activation, activation edges, per-neuron connectivity).
- [ ] A9 `pnpm gen:sql` generates the memory queries into `dbgen`; only `internal/memory/pg` imports `dbgen`, and no sqlc row
  type appears in the domain model or context behavior [ARCHITECTURE §2.4 #4].
- [ ] A10 Repository interfaces are not declared in this unit; `memory/pg` returns concrete structs, and the queries needed by
  Epic A launch (insert diary/memory/neuron/activation/synapse upsert) and read (`GetUniverse` shape) exist and are
  each scoped by `user_id`.

- [ ] T002 (migrate) Write the goose migration `apps/api/db/migrations/00002_memory_aggregate_schema.sql`: one
  `-- +goose Up` creating the seven product tables **verbatim from plan 16 §"DB schema"** — `diaries` (append-only,
  no `updated_at`), `episodic_memories` (aggregate root: emotion columns `mood`/`valence`/`arousal`/`intensity`,
  nullable `seed`, reserved `semantic_stage`/`semanticize_timer_reset_at`/`semantic_stages`/`decay_stages`/
  `deleted_at`), `neurons` (`neuron_type`, reserved `sealed_at`), `neuron_activations` (PK
  `(episodic_memory_id, neuron_id)`, `weight`), `synapses` (`UNIQUE (user_id, neuron_a_id, neuron_b_id)`),
  `embeddings` (`vector(N)` with the literal `N` = the `ai.embedding_dim` value from T001 — one value, one source;
  HNSW index with `vector_cosine_ops`, operator class/params confirmed at implement per the plan), and `jobs`
  (`kind`/`payload`/`status`/`attempts`/`next_run_at`). Every table `user_id TEXT NOT NULL`; `id TEXT PRIMARY KEY`
  (backend-minted nanoid); times UTC. Matching `-- +goose Down` drops index + tables in reverse-dependency order.
  Apply with `pnpm db:migrate`.
- [ ] T003 [P] Create the `internal/memory` context as ONE package (no `domain/app/pg/rpc` split yet — ARCHITECTURE
  §2.3): `apps/api/internal/memory/doc.go` (context intent, mirroring `internal/persistenceprobe/doc.go`) +
  `apps/api/internal/memory/types.go` with the aggregate's domain types under canonical names — `Diary`,
  `EpisodicMemory` (root; references `Neuron` by id through activations), `Neuron` (its own shared root),
  `NeuronActivation`, `Synapse` (neuron↔neuron edge only), `Embedding`, `Job` — free of proto/sqlc/pgx/JSON/DB tags
  (§2.4). Emotion rides as `mood`/`valence`/`arousal`/`intensity` fields on the root (the `Emotion` VO is plan 17's).
  Declare NO repository interfaces (consumer-owned, plans 20–22 — §2.4 forbidden pattern #6) and NO types for
  positions/coordinates/constellations/latent neurons/nebula color [I5].
- [ ] T004 [P] Add the baseline sqlc query files under `apps/api/db/queries/memory/`: `launch.sql` — insert `Diary`;
  insert `EpisodicMemory`; insert/upsert `Neuron`; insert `NeuronActivation`; upsert `Synapse` via
  `INSERT … ON CONFLICT (user_id, neuron_a_id, neuron_b_id) DO UPDATE` (the atomic weighted upsert, §2.6); insert an
  `embeddings` row; enqueue a `jobs` row — and `universe.sql` — the `GetUniverse`-shape read of a user's
  `episodic_memories`, `neurons`, `synapses` (plus activation edges and per-neuron connectivity/degree), **stored
  facts only** (derived brightness/decay stay in shared pure functions, plan 18). Every query takes and filters by
  `user_id` (the 04/05 `UserScope` convention); NO `UPDATE` on `diaries`; NO `DELETE` anywhere.
  *Indexes for scoping (plan 16 §Design):* once `universe.sql` fixes the read shape, add the per-user `(user_id, …)`
  read indexes for its hot read paths into the T002 migration (same migration, before it ships) — or, if measurement
  is deferred, record the deferral here as an explicit tuning follow-up rather than leaving it silent.
- [ ] T005 (gen) Run `pnpm gen:sql` — regenerates `apps/api/db/gen` (`dbgen`, pgx/v5) with the new models + query

- [ ] T006 Build `apps/api/internal/memory/pg` — the persistence adapter and the ONLY package in this context that

  no SQL trigger in Epic A), and every method scoped through `platform.UserScope` (mirror

- [ ] T007 Tests: `apps/api/internal/memory/pg/store_test.go` — the synapse canonical-ordering invariant (both input

- [ ] T008 Migration round-trip (A1): `pnpm db:migrate` applies cleanly → `pnpm db:down` reverses cleanly with no
