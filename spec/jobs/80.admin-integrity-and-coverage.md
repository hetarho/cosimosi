## Acceptance Criteria (from code-review/06 · R002, R003, R004, R015)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [ ] A1 (R003) `GrantStardust` **rejects an empty grant id** with the already-wired `ErrGrantIDRequired`
      (→ `ADMIN_GRANT_ID_REQUIRED`, InvalidArgument) instead of auto-minting one — the money-path idempotency is
      enforced server-side, not by client convention. The web client already sends `crypto.randomUUID()`, so no FE
      regression; a direct-RPC retry that omits the id is refused rather than double-crediting.
- [ ] A2 (R004) `RevokeAdmin` **refuses a seed admin identified by email**, not only by id: a revoke targeting a user
      whose resolved email ∈ `seedEmails` returns `ErrSeedAdminUndemotable`, mirroring `isSeedIdentity`
      (`service.go:252`). The RPC no longer returns a misleading `isAdmin:false` for a still-admin seed.
- [ ] A3 (R002) `internal/admin/pg` has integration test coverage (DB-gated like the other `*_integration_test.go`)
      proving: the grant + audit rows are written in one transaction (atomic — a failed audit rolls back the grant
      record), the append-only logs are never UPDATE/DELETE, and an idempotent replay (same grant id) records exactly
      one row.
- [ ] A4 (R002) `internal/ai/runtime.go` has unit coverage: fingerprint change → rebuild, stable fingerprint → cached
      reuse, key rotation → rebuild, cleared-key resolution (shared with job 79's A2 — do not duplicate).
- [ ] A5 (R015) At least one service/rpc test asserts the provider-key hint **shape** (masked/short) using the real
      `secretbox` cipher, not the constant `fakeCipher.Hint` — so the masking assertion can actually fail if masking
      breaks.
- [ ] A6 (R002) The CI Postgres decision is made explicit: **either** `ci.yml` provisions a Postgres service so the
      DB integration tests run in CI, **or** `spec/tech/admin-console.md` states that DB integration coverage is
      local-only (so a green CI is not read as full coverage). (Coordinate with job 84 / R017 if touching CI.)
- [ ] A7 No behavior change beyond the stated fixes: capped/append-only/audited grant semantics, DB→env→mock
      resolution, and the write-only key boundary are all preserved.
