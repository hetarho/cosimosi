## 인수 조건 (acceptance — 기획 관점, changes/07.memory-weight-woven-background에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [ ] A1 **빈도 영속.** `memories.recall_count`가 생기고 회상(`RecallMemoryTouch`)마다 1 증가하며, `Star`(proto) payload로 클라에 전달된다. 기존 별은 합리적 기본값(예 1)으로 백필된다.
- [ ] A2 **R 공식.** 막 회상한 별은 R≈1(중앙 근처), 오래 휴면한 별은 R→낮음(외곽). **회상 횟수가 많은 별은 한 번만 회상한 별보다 같은 Δt에서 천천히 잊힌다**(τ가 저장강도 S로 길어짐). R·S는 순수 클라 모듈에서 계산된다(three/React/DOM 미의존).
- [ ] A3 **자기근접 = R.** `targetRadius`가 옛 `strength(0.7·act+0.3·int)` 대신 R을 쓴다. "거리=강함"(강할수록 중앙)이 보존되고, **별의 각도(방향)·force-sim 파라미터·표상 부동은 바뀌지 않는다**.
- [ ] A4 **배경이 감정색을 짜 넣는다.** 배경 스킨이 순위 상위 N개(스킨별 `emotionSlots`) 감정색을 **사용자 감정색(45)**·R-비중으로 텍스처에 합성한다. `emotionSlots:1` 스킨은 주요 감정 1색만, `13`(또는 전부) 스킨은 비중대로 다색을 녹인다. `0`이면 감정 무관 순수 텍스처.
- [ ] A5 **무드 오브 제거.** `AmbientNebula`(떠 있는 가산광 풀)가 제거되어 어떤 화면에도 투박한 구형 그라디언트가 없다. 요즘 감정은 배경 텍스처/톤으로만 드러난다.
- [ ] A6 **배경 차별화.** 배경 아이템들이 **색만이 아니라 받침색 + 무늬/질감**으로 갈린다(같은 fbm·색만 다른 상태가 아니다). 색감은 원색이 아닌 절제된 톤 위주다.
- [ ] A7 **arousal 재배선.** arousal이 `Σ R`에서 도출되어 배경 스킨의 전역 생동(밝기·움직임)을 정한다. 22 흥분성 게인 seam(`1 + 0.3·arousal`)은 헬퍼로 보존된다.
- [ ] A8 **계약 정리.** 서버는 `GetUniverse`에서 `AmbientMood`를 더는 싣지 않고(필드 reserved), 클라가 별(+`recall_count`)에서 순위·arousal을 파생한다. 서버 ambient 종합 코드가 제거되어도 코어 그래프 조회는 회귀 없이 동작한다.
- [ ] A9 **색·밝기 회귀 없음.** 별 색은 여전히 감정색(`resolveMoodRgb`)이고, 26 밝기 감쇠·self-glow·spec 03 반사 중립이 그대로다(이 변경이 별색/밝기를 안 건드림).
- [ ] A10 **데모·랜딩 재조정.** 데모 우주가 같은 R-순위·배경 짜임을 미러하고, 랜딩 `AmbientMoodCard`가 옛 오브가 아닌 새 배경-짜임 모델을 반영한다(또는 배경 카드로 흡수). 38 레이아웃 읽기·44/45 동작에 회귀가 없다.
- [ ] A11 **값 단일 출처.** 신규 가중치 계수(`tau0_days`·`tau_storage_gain`·`emo_consolidation` 등)와 arousal·배경 생동 계수가 `spec/values.yaml`에서 생성된 상수로 FE/BE 공유된다(하드코딩·이중 출처 없음). `emotionSlots`(스킨별 N)는 코드 카탈로그(시각 정의)에 둔다.
- [ ] A12 build·lint·vet·typecheck·steiger 통과, `prefers-reduced-motion`에서 배경 모션 정지(색 유지), 우주 셸·기존 화면 회귀 없음.
