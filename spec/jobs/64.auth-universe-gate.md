## Acceptance Criteria (from plan/53.auth-universe-gate)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [ ] A1 On **web**, the `/` route carries a `beforeLoad` **auth guard**: an `authenticated` session renders the
      universe; a settled `signedOut`/`expired`/`failed` session **redirects to the login entry**;
      `bootstrapping`/`refreshing` **holds on a neutral loading state without redirecting** (no signed-out flash)
      ([U1][U4]).
- [ ] A2 On **mobile**, the `app/navigation` root selects the stack from the **same** session snapshot:
      `authenticated` → `UniverseScreen`; settled signed-out → the login stack; `bootstrapping`/`refreshing` → a
      neutral splash — web↔mobile parity ([U1][§3.5]).
- [ ] A3 There is **no landing/marketing page** in v1 on either app: the unauthenticated default is **login**, the
      authenticated default is the **universe**, with no intermediate route ([U4]).
- [ ] A4 A successful sign-in ([04]'s action reaches `authenticated`) routes to the universe — on web to the carried
      `redirect`/`from` (or `/`), on mobile by swapping to the `UniverseScreen` stack.
- [ ] A5 **Sign-out** routes back to login on both apps and unmounts the universe view; it **never deletes** the
      user's data — a subsequent sign-in for the same user re-reads the same universe ([U1][I1]).
- [ ] A6 A **first-run** user (no memories) lands on the universe rendered as the **gray latent field**
      ([25.latent-star-field](../plan/25.latent-star-field.md), [V7]) with a single **"일기 쓰기"** affordance ([W1], [27]) —
      the empty universe is a welcome, not an error state, and is the _same_ `widgets/universe-canvas` scene with zero
      real stars ([U2]).
- [ ] A7 A directly-entered **web URL** while unauthenticated redirects to login and, after sign-in, returns to the
      originally requested route (`redirect`/`from` param); the router/nav library stays confined to `app/routes` /
      `app/navigation`, and the gate reads auth **through the [04] facade**, never Supabase directly ([§3.1]).
- [ ] A8 The gate never lets `GetUniverse` (or any product read) run without an authenticated session — the
      presentation-side complement of [04]'s server-side per-user isolation ([U1][§2.7]).
- [ ] A9 **All** user-visible copy (login entry, first-run welcome) resolves through the i18n seam (no raw strings;
      `lint:raw-strings` passes), in a literary / restrained Korean voice.
- [ ] A10 This unit adds **no** auth engine, renderer, read model, write flow, settings UI, RPC, or DB change — it
      composes [04]/[15]/[13]/[23]/[25]/[27]; `pnpm typecheck`, `pnpm lint` (incl. `lint:fsd:layout`,
      `lint:raw-strings`), `pnpm test:web`, and `pnpm build:web` pass with no BE changes.

## Grounding

- Constitution ([I1]–[I11]): [00.overview](../plan/00.overview.md) §3 (_The constitution_). This job is bound by
  **[I1]** — the system never deletes: logout only unmounts the view; the universe persists per-user for the next
  sign-in. It **honors [U1]** by never rendering a universe (or issuing a product read) without an authenticated
  session — the presentation-side complement of [04]'s server-side per-user isolation. All other domain invariants
  sit downstream of the read this unit merely mounts.
- Architecture (placement): [ARCHITECTURE.md](../ARCHITECTURE.md) FE **§3.1** (the gate is **app-layer only** — web
  `app/routes` (`guards/` beside the route tree), mobile `app/navigation`; `pages`/`features` never import the
  router/nav library and navigate through the seams), **§3.2** (auth session state is already [04]'s XState machine —
  the gate _reads_ its snapshot and adds no competing machine), **§3.5** (web↔mobile parity — the same rule expressed
  through each app's nav seam; disciplinary parity, not a shared route package), **§2.7** (`GetUniverse` is a unary
  user-scoped read — the gate guarantees it is only issued under an authenticated session). Invoke `/fe-architecture`
  - `/mobile-architecture` for every file placed (no BE surface).
- tech / policy / values touched: **values — none** (the status→decision mapping and route paths are code; the
  refresh timing observed is [04]'s existing `auth_session.access_token_refresh_skew_ms` /
  `supabase_auth.jwks_cache_ttl_ms`; the latent-field density is [job 30]'s `latent_star_*` keys — all reused, none
  new; UI copy is i18n content; a neutral-hold timeout key is added only if it ever proves cross-surface tunable —
  none expected, confirmed at implement). **tech/** — the as-built gate rule extends
  [tech/web-routing.md](../tech/web-routing.md) at completion (**not pre-created**): the `/` `beforeLoad` guard, the
  three-state mapping, the `redirect`/`from` return path, the mobile initial-stack mirror, and the "no landing in v1;
  login → universe direct" app-entry contract (no separate `policy/` doc — plan 53's call). **Depends on**
  [job 04](archive/04.auth-session.md) (the session snapshot + sign-in/out actions),
  [job 19](archive/19.web-router-foundation.md) (the `/` route + typed guard seam it fills),
  [job 13](archive/13.mobile-app-shell.md) (the nav registry + `UniverseScreen` + auth-callback links),
  [job 28](archive/28.universe-canvas.md) (the universe + `GetUniverse`),
  [job 30](archive/30.latent-star-field.md) (the empty-universe background),
  [job 32](archive/32.writing-flow-ui.md) (the 일기 쓰기 affordance). **Pairs with**
  [job 63](63.settings-page.md) (sign-out's long-term home — **this job builds first**);
