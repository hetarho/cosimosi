## Acceptance Criteria (from plan/20.encode-usecase)
<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->
- [ ] A1 `Encode` splits a diary body into **2–5 episodic memories** on event boundaries (place/person/activity/topic shifts),
  **not** emotion shifts; a result outside `[encode.min_memories, encode.max_memories]` is a retry/repair condition, not
  a silent clamp. ([E1][E2])
- [ ] A2 Each proposed memory carries an LLM-suggested **name** ([W2a]) and one **primary mood** from the 13-mood enum ([M1]);
  both are returned for the user to edit ([W2][W4]).
- [ ] A3 Each memory has **≥1 semantic neuron**, achieved as an extraction guideline (retry/repair when missing) — **never** by
  inserting a hollow placeholder neuron. ([E4])
- [ ] A4 Neurons are typed as exactly one of **semantic / spatial / entity**; spatial and entity are optional. ([E3][E5])
- [ ] A5 **Time is never emitted as a neuron** — the diary date informs temporal handling only and appears in no neuron. ([E6])
- [ ] A6 Neuron dedup is **conservative and type-differentiated**: entity = same referent; spatial = synonyms only, preserving
  granularity ("스타벅스" ≠ "스타벅스 강남점"); semantic = strictest. Merging is an identity judgement, distinct from
  similarity linking ([L3]). The embedding-assist similarity threshold is tunable (value set at implement). ([E10])
- [ ] A7 The encode output schema contains **only** `{memories:[{name, mood, neurons:[{name, type}]}]}` — no position, color,
  strength, seed, time, or delete field exists — so a prompt injection has no field in which to emit an
  invariant-violating value ([I1][I3][I5][I10]); this backs the strong prompt engineering as the second defense. ([W4a])
- [ ] A8 `ReviseSplit` re-runs the split from a natural-language instruction + the prior result against the same schema-forced
  output, bounded by `encode.max_revise_retries` and `encode.max_output_tokens`; exceeding a cap returns a canonical
  error, not an unbounded loop. ([W4a])
- [ ] A9 `SplitDiary` (and `ReviseSplit`) are **synchronous previews** that **persist nothing** — no Diary, EpisodicMemory,
  Neuron, or job is written. ([W2])
- [ ] A10 `PersistEncoded` (via `LaunchStars`) **atomically** persists Diary + EpisodicMemory + Neuron (deduped) +
  NeuronActivation in one transaction, sets `seed` and arousal-modulated `base_strength`, hands off to `Link` (this
  job only defines the in-transaction call seam; the actual `Link` invocation is wired by job 27 T006, which completes
  this clause), and
  **enqueues `embed` + `semanticize` jobs**; it returns `memory_ids` **and `new_neuron_ids`** (the genuinely-created
  neurons, for the awaken animation) optimistically (embeddings/gist fill on next read). ([E7][E7a][E8a][C7], §2.8)
- [ ] A11 The **Diary is immutable** — `PersistEncoded` inserts it once (append-only), and nothing in this unit updates it;
  `Encode`/`ReviseSplit` never write it. ([I2])
- [ ] A12 A **past-dated** diary (`diary_date < latest launched`) is **saved** but launches **no star**; the monotonic guard is
  enforced at launch. (Clock advance is Epic B; the writing flow surfaces a one-time confirmation notice — plan 27.)
  ([I10][T1])
- [ ] A13 RPC handlers are **thin** (proto↔domain map + call); split-count, semantic-neuron, dedup, cap, and monotonic policy
  live in the use-case/domain; no proto/sqlc type leaks into the use-case or pure domain; the `Extractor` and
  repositories are consumer-owned interfaces. (§2.4/§2.7/§2.9#4/#6/#7)
- [ ] A14 Every neuron fetch, dedup-candidate query, and write is **scoped to the authenticated user**. (§4)

- [ ] T004 **(gen)** Add this use-case's queries under `apps/api/db/queries/memory/` + their `memory/pg` row↔domain
  mappings: dedup-candidate fetch by exact/near `name` + `type` (per-user), the **narrow** embedding nearest-neighbour
  top-k over `embeddings` within `encode.dedup_similarity_threshold` (cosine over plan 16's HNSW index), and the latest
  launched memory's `created_universe_time` for the monotonic guard. Every query filters by `user_id`. Run
  `pnpm gen:sql`; `dbgen` rows never escape `memory/pg`. **(migrate) — none:** tables/migrations are plan 16's (job 20);
  this job adds no schema.
- [ ] T005 Implement `Encode` in `internal/memory`: assemble the per-user dedup-candidate set (T004) → call
  `Extractor.Split(ctx, body, diaryDate, existingNeurons)` → enforce the domain invariants on the returned structure —
  memory count within `[encode.min_memories, encode.max_memories]` handled as retry/repair (re-prompt within the cap,
  **no silent clamp** [E2]); ≥1 `semantic` neuron per memory as retry/repair, **never** a hollow placeholder ([E4]);
  every neuron `type` ∈ {`semantic`, `spatial`, `entity`} ([E3][E5]); time never a neuron ([E6]) — and return the
  proposed split. Persists nothing; computes no coordinate, color, strength, or seed.
- [ ] T006 Implement the `ReviseSplit` orchestration: same invariant enforcement through the `Extractor` revise variant
  (prior result + natural-language instruction, same schema-forced output [W4a]); bound retries/output by
  `encode.max_revise_retries` / `encode.max_output_tokens` (generated constants); on exceeding a cap return a canonical
  error, not an unbounded loop.
- [ ] T007 Implement `PersistEncoded` (invoked by `LaunchStars`) as **one transaction** (`platform` `InTx`): insert the
  Diary once — append-only, no UPDATE ([I2][W5], written at launch, not at split); apply the **monotonic launch guard**
  ([I10][T1]): if `diary_date <` the latest launched `created_universe_time`, the Diary is still saved but **no**
  EpisodicMemory launches (decided 2026-07 — the one-time confirmation notice UI is plan 27's; universe-clock advance is
  Epic B: reserved seam, **no task**).
- [ ] T008 In the same transaction, for each confirmed memory (when not past-dated): insert `EpisodicMemory` with `name`,
  the `Emotion` VO derived from `mood` (plan 17), a generated `seed` ([E7] — form/anchor hint, no stored coordinate
  [I5]), `created_universe_time = diary_date`, and `base_strength = arousalToInitialStrength(arousal)` (plan 17, [E8a]);
  persist neurons honoring the extractor's dedup — reference the existing neuron id when canonicalized onto one,
  otherwise create the typed `Neuron` ([E3][E10], never deleted [I1]) — tracking genuinely-created ids for
  `new_neuron_ids`; insert `NeuronActivation` rows with **uniform `weight = 1.0`** (decided 2026-07; the `NOT NULL`
  column always gets a value — differentiated [E8] weighting is a later-epic refinement, **no task**).
- [ ] T009 Complete the launch inside the same transaction: **define the in-transaction call seam for `Link`** — the
  invocation point inside `PersistEncoded` (plan 21; synapse math is `Link`'s) — **the actual `Link` invocation is
  wired by job 27 T006; A10's "hands off to `Link`" clause is completed by job 27**;
  enqueue `embed` + `semanticize` jobs through the `JobQueue` port (plan 16 `jobs` table, plan 22 worker); return
  `memory_ids` + `new_neuron_ids` optimistically (§2.8 — embeddings/gist fill on the next read, no polling). The async
  `link`/`extract`/`consolidate` job kinds stay reserved seams — **no task**.
- [ ] T010 Thin RPC handlers in `internal/memory/rpc`: map proto↔domain and call `Encode`/`PersistEncoded` — nothing
  else; split-count, semantic-neuron, dedup, cap, and monotonic policy stay in the use-case/domain (§2.9#7). Also add
  a thin **`GetUniverse`** handler over job 20's `universe.sql` read, deriving `universe_time` as the latest launched
  memory's `created_universe_time` (reusing the T004 latest-launched query; Epic A derivation — the `universe_state`
  clock stays Epic B), consumed by job 28's FE read. Resolve
  `platform.UserScope` from the auth context on every call (§4). Register `MemoryService` through the `NewServer(...)`
  constructor on the platform mux and inject the concrete `Extractor` adapter (plan 22's `internal/ai` real / keyless
  mock selection) at the composition root `cmd/api`.
- [ ] T011 [P] Unit tests for `Encode`/`ReviseSplit` with a fake `Extractor`: out-of-range memory count and
  missing-semantic-neuron results trigger retry/repair and yield a canonical error at `encode.max_revise_retries`
  (never a silent clamp, never a placeholder neuron); an invalid neuron type is rejected; no time neuron can appear
  (the domain shape has no field for it); previews persist **nothing** (no Diary/memory/neuron/job row written).
- [ ] T012 [P] Tests for `PersistEncoded`: single-transaction atomicity (a mid-step failure leaves no partial rows);
