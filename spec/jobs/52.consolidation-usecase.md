## Acceptance Criteria (from plan/41.consolidation-usecase)
<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->
- [ ] A1 `Consolidate(scope, from, to)` is bound as the concrete `AdvanceProgression.OnAdvance` handler and runs
  **inside the same advance transaction** as launch and sync-to-today; plan 30's port signature, call sites, and
  no-op-on-`from==to` guarantee are unchanged — only the composition-root binding changes from no-op to this handler.
  ([T4])
- [ ] A2 There is **no cron, scheduler, or background sweep** and **no new RPC** — the advance hook is the sole
  trigger, and consolidation is never a user action. ([T4])
- [ ] A3 For each memory whose semanticize timer crosses one or more stage boundaries in `[from, to]` (per plan 40's
  timer model anchored at `semanticize_timer_reset_at`), `semantic_stage` advances by the number of boundaries
  crossed, clamped to the 4-stage ceiling; a stage **never decrements** (z-axis one-way). No LLM call is made inline —
  the stage texts are already pregenerated ([C7]). ([C1][C6][C7])
- [ ] A4 Newly reached forgetting-stage texts are persisted during the same advance transaction: for every crossed
  stage whose `decay_stages[stage]` is missing, the handler writes plan 37's deterministic `decayStageText` output,
  while brightness and current stage remain read-time derived and no current brightness/text tick is stored.
  ([F1][R8a][T4])
- [ ] A5 Replay reorganization marks the **constellations touched in the interval** (stage-advanced memories + their
  neuron-sharing neighbors) for the force-sim's larger companion re-layout at read time; it stores **no coordinate**
  ([I5]) and is bounded to touched constellations, never the whole universe. ([C2])
- [ ] A6 `Downscale(strength, factor)` is a **new pure domain function** applied as a global, homeostatic,
  proportional renormalization of the user's synapses (weak edges lose proportionally more, strong edges preserved),
  returning a value in `0..1` with a **positive residual floor** — it **never removes** a synapse. ([C4][I1])
- [ ] A7 `Downscale` is **distinct from `Depress`/LTD** ([I9]): it is a separate pure function (global/homeostatic vs.
  associative/local), and this unit does not modify `Depress`. Both keep `strength` in `0..1`.
- [ ] A8 `Downscale` lives **once in Go and once in TS** (golden-parity, next to plan 18's synapse fns) and is pinned
  equal by golden-parity fixtures; the surrounding `Consolidate` orchestration is server-only and is not mirrored.
- [ ] A9 The reserved **`consolidate`** job kind is **activated**: interval-implied heavy or LLM-touching work
  (re-embedding after reorg; a `semanticize` re-enqueue when a *remaining* stage text is genuinely missing per [C7])
  is enqueued on the worker, not run inline; the advance transaction stays fast. ([C7], §2.8)
- [ ] A10 `Consolidate` is **idempotent / convergent over the interval**: re-running it for an already-consolidated
  interval advances no stage and downscales nothing further (timer anchor + stage ceiling), and `from == to` is a
  total no-op.
- [ ] A11 The Diary is never touched, no synapse/neuron/memory is deleted, and no position is stored — consolidation
  changes only representation (`semantic_stage`, `decay_stages`, synapse `strength`) plus a read-time re-layout
  marker. ([I1][I2][I5])
- [ ] A12 Every read, stage write, decay-stage text write, downscale batch, and enqueue is **scoped to the
  authenticated user**, and `pnpm lint:persistence` passes (§4).
