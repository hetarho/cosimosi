## Acceptance Criteria (from plan/38.forgetting-cost)
<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->
- [ ] A1 `AccessibilityCostWeight(decayDepth)` is a **pure, IO-free** function (deterministic; no clock, DB, transport,
  SDK, or randomness) in both `internal/memory` (Go) and `packages/memory-logic` (TS), reading only its argument and
  the generated `forgetting.*` constants. (ARCHITECTURE §2.5)
- [ ] A2 It is **monotone non-decreasing** in `decayDepth` over `[0, 1]` and strictly increasing across the interior —
  a deeper-decayed episodic star yields a weight ≥ (and, in the interior, >) that of a shallower one: deeper decay ⇒
  harder to reach ⇒ costlier. ([F4])
- [ ] A3 `AccessibilityCostWeight(0) = forgetting.cost_weight_floor` — a fully accessible (freshly recalled) star
  returns the **floor** (cheapest, but non-zero: recall is never free [G1]); this is the [F5] recovery endpoint
  expressed through the decay input. ([F5][G1])
- [ ] A4 The weight is **capped**: `AccessibilityCostWeight(1) = forgetting.cost_weight_cap` and every result stays
  within `[forgetting.cost_weight_floor, forgetting.cost_weight_cap]` — a maximally decayed (silent-engram) star is
  expensive but **bounded**, never unreachable/infinite. ([I1][F2])
- [ ] A5 **Recovery on recall is expressed through the input, not a separate function or a stored field**: because
  recall resets decay in plan 37 (`decayDepth → 0`), the weight returns to its floor; there is no `accessibility`
  column and no independent recovery path. ([F5][I5][I8])
- [ ] A6 The function's **input is 37's normalized decay depth** (`[0,1]`), not a raw stage index; this unit does
  **not** recompute decay, does not read `effectiveElapsed`/stages/arousal/connection directly, and adds no
  neighbor-offset math (the offset reaches it only via 37's depth). ([F1][F6][F7][R5])
- [ ] A7 The unit exposes a **weight, not a price**: it references no Twinkle constant, ledger, or balance; the recall
  Twinkle price is derived from this weight by plan 43's cost curve and spent by plan 44's gate (which **re-derives**
  the weight server-side, §2.9#8). D computes accessibility, F prices it, C spends it. ([G4][G1], CC2/CC3)
- [ ] A8 It is the **episodic-recall axis only** — it does **not** compute the gist-view cost (that is the gist-depth
  axis, `semantic_stage`, priced by F and read by plan 34). ([R8][G4])
- [ ] A9 The Go function and the TS mirror produce **identical outputs** for a shared golden-fixture table within a
  fixed tolerance (TS↔Go golden-parity), covering the floor at `0`, the cap at `1`, interior monotonicity, and
  out-of-range clamping; both read the same generated `forgetting.*` constants. (ARCHITECTURE §2.5)
- [ ] A10 No formula, curve shape, or clamp is stored in `spec/values.yaml`; only the floor, cap, and curvature
  coefficient are values (the convex shape stays code). (config rule)
