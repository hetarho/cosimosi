## Acceptance Criteria (from code-review/06 · R005, R006, R009, R012)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [ ] A1 (R006) Admin console errors route through the plan-59 shared seam — `useErrorToast` / `presentAppError` from
      `@cosimosi/errors` + the app-layer error-provider — **not** raw `cause.message`. No admin surface renders an
      untranslated ConnectError message. `AiConfigSection` no longer does
      `setError(cause instanceof Error ? cause.message : …)`.
- [ ] A2 (R005) Every admin dashboard section handles the **error** state, not only `isPending`: `JobsSection`,
      `UsageSection`, and the model-select surface render `m.admin_load_error()` (matching `ProviderKeysSection`) on a
      failed read — an outage never renders as an all-zero (`0n`) healthy queue.
- [ ] A3 (R009) Admin **mutations** (grant / promote / revoke in `UsersSection`) surface failures to the operator via
      the toast seam instead of `.catch(() => undefined)`.
- [ ] A4 (R012) The hardcoded operator strings in `AiConfigSection` are internationalized: `'unset'` becomes a message
      key and the server `source` enum (`db`/`env`/`empty`) is rendered through a small label helper mapped to i18n
      keys. No raw user-facing literal remains in the admin surface.
- [ ] A5 No behavior change to the happy path: successful reads/mutations render and refetch exactly as today; only the
      failure/degraded and copy paths change.
