## Acceptance Criteria (from plan/50.deletion-ui)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [ ] A1 A **full-delete** affordance (reached from the diary reader [47] or from the star-detail panel's source-diary action [35]) opens `widgets/deletion-flow`; when opened over the universe canvas, the renderer is not remounted ([23]). Its confirm states plainly that **all stars born from the diary** are affected, previews/lists them, and carries **both** the **30-day restore reassurance** and the **export-before/restore-window ([W6]) reassurance** before the action ([X1][X2][W6]).
- [ ] A2 Confirming full delete calls **`Release(diary_id)`**; on success all returned `episodic_memory_ids` are **optimistically removed** and the canvas stops rendering them with **no residual pull** ([X3][I5]); a failed call rolls back and the stars reappear (§2.8).
- [ ] A3 A soft-deleted diary release group inside its **30-day window** can be **restored** via `features/restore-memory` calling **`Restore(diary_id)`**, re-inserting the affected stars; the window length is read from generated config, not hardcoded ([X2]).
- [ ] A4 The **letting-go** flow runs in four steps — **say the words** → **`SuggestLetGo`** (this-memory-only semantic neuron candidates) → **user approves which to seal** → **`LetGo(approved_neuron_ids)`** — and **AI only suggests; the user decides which are sealed** ([X4][X6]).
- [ ] A5 Letting-go seals **semantic neurons only**; the flow states and the contract enforces that **emotion / time / space / entity / color / the star are kept** — the sealed memory **persists as a silent engram** (color/shape/position live on), and its content reads as thinned on the next read ([X4]).
- [ ] A6 Letting-go is **permanent with no restore timer** and the UI says so plainly at the seal step; it offers **no undo** after sealing, and reminds that the original words remain exportable ([X5][W6][I2]).
- [ ] A7 Letting-go copy frames the act as **symbolic release / blurring** and **claims no therapeutic efficacy**; **no screen implies the app substitutes for care** ([X7]).
- [ ] A8 When the backend sets the **heavy-state signal** on the `SuggestLetGo` response, the flow surfaces a **gentle, non-blocking professional-resource guidance** panel before the approve step (advisory, not gating); the **detection is the backend's**, this unit only renders the notice ([X7]).
- [ ] A9 The flow's request surface carries **only** `{ diary_id }` (full-delete / restore) and `{ episodic_memory_id, approved_neuron_ids }` (letting-go) — there is **no** `kind`, emotion, position, color, strength, or time field anywhere, and no way to command a hard delete or seal a shared/foreign neuron; the meaning layer cannot be overwritten ([I3][I11]).
- [ ] A10 Neither full delete nor letting-go mutates the `Diary` in place; the **export reassurance ([W6])** is shown in both flows and links [46.provenance-export](../plan/46.provenance-export.md). Full-delete copy says the objective record is preserved/exportable through the restore window and permanently removed after the sweep; letting-go copy says the original remains exportable because the diary itself is not deleted ([I2][D4][X2]).
- [ ] A11 The flow **never** triggers a system-driven delete and offers **no** "delete permanently now" for full-delete — removal is the user's explicit act, and the only hard delete (the 30-day sweep) is the backend's ([I1][X2]).
- [ ] A12 The flow's **control-state** is one XState machine (`idle → confirmingDelete → deleting → done` and `idle → phrasing → suggesting → approving → sealing → done` + error paths) with `suggesting`/`sealing` as loading states; **the typed phrase, candidate list, and selected ids live in Zustand/Query**, not machine context (§3.2).
- [ ] A13 **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in the most restrained / honest Korean voice, with **no therapeutic-efficacy claim** ([X7]).
- [ ] A14 **web + mobile parity:** the same widget + three features run on both apps, sharing `model` / `api` verbatim; `*.native.tsx` siblings exist **only** for the sheet / modal / input primitives (§3.5).

## Grounding

- Constitution ([I1]–[I11]): [00.overview](../plan/00.overview.md) §3 (_The constitution_). This job is bound by
  **[I1]** (deletion is the user's explicit act only — this UI is the sole place the exception is exercised; it
  triggers no system-driven delete and offers no immediate hard delete), **[I2]** (the `Diary` is immutable —
  neither flow mutates the record in place; the [W6] export reassurance makes that concrete), **[I3]/[I11]**
  (emotion never drives position/connection and the meaning layer is untouchable — the flow selects neurons to seal
  and sets nothing else; the request surface has no emotion/position/color/strength field), **[I5]** (positions are
  emergent — the flow removes/marks optimistically and never writes a position; the layout re-settles on the next
  read).

  three `features/*`, each one user action; hosts compose via public APIs), **§3.2** (one XState machine for
  control-state; phrase/candidates/ids in Zustand/Query; discrete scene events, never per-frame React state),
  **§3.4** (the domain→visual projection is one-way — the vanished star / silent engram is the read model's
  projection, not this widget's code), **§3.5** (web + mobile peers; `model`/`api` shared verbatim, `*.native.tsx`
