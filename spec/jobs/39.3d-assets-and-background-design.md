## Acceptance Criteria (from plan/57.3d-assets-and-background-design)

**Tier 1 — objective deliverables (checkable)**

- [ ] A1 A living guideline exists (`spec/tech/design-language.md`, 3D section) documenting: star form & material, the
      other bodies' aesthetics, background/skin/atmosphere & z-layer depth, the emotion palette values, the 3D state
      visual design (dim/gist/awaken), and the choreography feel.
- [ ] A2 A reference universe showcase scene renders every body type, every designed state, the skin(s)/background, and
      the full emotion-color spectrum on **web and mobile**.
- [ ] A3 All visual changes are expressed through `@cosimosi/3d-renderer` + `@cosimosi/universe-render` (+ the palette
      source); no per-scene hardcode, and node **positions remain emergent** from force-sim (never set here) ([I5]).
- [ ] A4 The emotion palette values are authored as one source, consumed by 3D (nebula/star color) and referenced by
      job 38 (plan 56) for 2D — the emotion aesthetic matches across 2D and 3D.
- [ ] A5 The shipped Epic A 3D bodies are retrofitted to the new identity.
- [ ] A6 Plan-14 performance floors are preserved (instancing, frame budget, TSL rendering); the web + React Native
      parity path stays intact.
- [ ] A7 Presentation-only — no domain, clock, layout, or meaning mutation; positions and the meaning layer untouched.

**Tier 2 — iterative designer-review gate**

- [ ] A8 The 3D rubric (star/asset form & silhouette · material/lighting · emotion-color legibility · depth & z-layer
      readability · state legibility · motion/choreography feel · atmosphere/background cohesion ·
      performance-within-budget) is scored by the design reviewer each round as Meets / Needs-work / Blocking.
- [ ] A9 A feedback ledger records every round's notes, each tracked to `resolved` or `deferred` (deferred listed).
- [ ] A10 **DoD:** the design reviewer signs off — every rubric dimension **Meets** and **zero open Blocking** —
      recorded as a dated sign-off. Completion is sign-off-gated and iterative; there is no single sequential pass.
