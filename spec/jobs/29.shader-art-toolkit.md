# Job 29: 셰이더 아트 툴킷 (shader-art-toolkit)  (new)

> 구현 작업 문서. 출처 스펙: [plan/50.shader-art-toolkit](../plan/50.shader-art-toolkit.md).

## 인수 조건 (acceptance — 기획 관점, plan/50.shader-art-toolkit에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [ ] A1 `shared/lib/r3f` 아래에 도메인 무관 셰이더 아트 툴킷이 있고, **이펙트 패밀리**(색/마스크 노드 생성)와 **오브젝트 패밀리**(지오메트리/SDF 생성)로 나뉜다. 공개 API는 `shared/lib/r3f`에서 노출된다.
- [ ] A2 툴킷의 모든 함수는 **순수**(노드/geometry in-out)하고, `MeshBasicNodeMaterial` 생성·dispose·uniform 변이·`useFrame`/clock·React hook·store·도메인 개념(스킨 id·가격·소유권·mood/emotionSlots)에 의존하지 않는다.
- [ ] A3 `entities/model`과 순수 `shared/lib`은 이 툴킷도 `three`도 import하지 않는다(불변원칙4). 툴킷은 `three`가 허용된 `shared/lib/r3f`에만 산다.
- [ ] A4 함수는 이름 있는 인자로 호출되고 계수는 호출부가 주입할 수 있다(매직넘버를 시그니처에 강제하지 않음). 긴 positional 체인이 없다.
- [ ] A5 시간/애니메이션이 필요한 기법은 시간을 파라미터(노드)로 받고, 내부에서 clock/`useFrame`을 굴리지 않는다(frozen-time idiom).
- [ ] A6 1차 primitive가 plan 51 배경이 요구하는 이펙트 기법(`worley`·`polar`/`logSpiral`·`kaleido`·`fresnel`·`fbm`·`domainWarp`)을 포함한다. 오브젝트 패밀리는 구조(경계·규약)와 1차 SDF/geometry 빌더를 포함하고 plan 52가 확장 가능하게 열려 있다.
- [ ] A7 노드 빌드 smoke 테스트와 geometry 생성 불변식 테스트가 있다. FE `build`·`lint`·typecheck·`steiger`가 통과한다.
- [ ] A8 기존 4축 렌더(배경·별·나·시냅스)는 동작이 바뀌지 않는다(이 job은 툴킷 *추가*만 — 소비처 전환은 30·31).

- [ ] T002 모듈 골격 — `shared/lib/r3f/shader-art/`에 `field.ts`·`noise.ts`·`pattern.ts`·`finish.ts`(이펙트) + `geometry.ts`·`sdf.ts`(오브젝트) + `index.ts`를 만들고, `shared/lib/r3f/index.ts`에서 re-export한다. 각 모듈 상단에 경계 규약 주석(순수·노드/geometry in-out·lifecycle/store/도메인 없음).
- [ ] T003 [P] noise — `fbm`(`mx_fractal_noise_float` 래퍼)·`worley`(`mx_worley_noise_*`, f1/f2)·`ridged`. 이름 있는 인자(`{ octaves, lacunarity, gain }`), 기본값 제공·호출부 주입 가능.
- [ ] T004 [P] field(공간 변환) — `domainWarp`·`toSpherical`(dir→lon/lat)·`polar`(dir→angle/radius)·`logSpiral`·`kaleido`·`rotate2`. 시간이 필요하면 노드 파라미터로 받는다(내부 clock 금지).
- [ ] T005 [P] pattern(→[0,1] 마스크) — `cellEdge`(f1,f2,sharpness)·`contourSteps`(f,steps)·`isoLine`·`metaball`/`smin`. 색은 입히지 않는다(소비처 몫).
- [ ] T006 [P] finish — `fresnel`(dir,power)·`iridescent`(angle,base)·톤 클램프 helper.
- [ ] T007 오브젝트 패밀리 1차 — `geometry.ts`/`sdf.ts`에 SDF primitive + `smin`/smooth union·subtract·displacement·lathe/extrude·instanced-attribute 빌더의 1차 집합과 규약. 순수(geometry/노드 in-out). plan 52가 확장할 수 있게 구조를 연다.
- [ ] T008 테스트 — `shared/lib/r3f/shader-art/*.test.ts`: 각 이펙트 함수가 유효 TSL 노드 그래프를 빌드하는 smoke, geometry 빌더의 정점/속성 불변식.
- [ ] T009 경계 검증 — `entities/model`·순수 `shared/lib`이 툴킷/`three`를 import하지 않고(lint/steiger), 툴킷이 material lifecycle·`useFrame`·store·도메인 모듈을 import하지 않음을 확인한다(A2·A3).
- [ ] T010 검증 실행 — `pnpm --filter ./frontend test`(툴킷 테스트)·`lint`·`build`·`steiger`. 기존 4축 렌더는 추가만이라 변화 없음을 확인(A8).
