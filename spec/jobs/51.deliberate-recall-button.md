## 인수 조건 (acceptance — 기획 관점, changes/35.deliberate-recall-button에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [ ] A1 별을 클릭하면 2초 dwell 자동 회상이 **더는 없다** — 클릭 즉시 그 별 내용(조각/흐려진/원본 토글)이 읽기전용으로 뜬다(3D 포트레이트·메타와 함께, change 32 유지).
- [ ] A2 **클릭만으로는 어떤 부작용도 없다**: 클릭 전후로 `last_recalled_at` 불변, `evolution_history`에 새 행 없음, AI 재작성 잡 미적재, 공동 회상 쌍 미누적, 별 레이어(밝기/형태/색) 무변(검증: 클릭 전후 GetUniverse·변천사 길이 동일).
- [ ] A3 패널에 **`이 별 자세히 보고 회상하기`** 버튼이 있고, 누르면 `RecallMemory`가 발화해 재점화·재공고화(PE 통과 시 재성형)·공동 회상 강화가 일어나며 패널이 회상 결과로 갱신된다.
- [ ] A4 회상 직후 **쿨다운**(values 신규, 기본 1시간) 안에는 같은 별 재회상이 막힌다 — 버튼 비활성 + 다음 가능 안내. 쿨다운 경과(또는 야간 공고화 1회) 후 다시 가능. 한 번도 회상 안 한 별의 첫 회상은 가능.
- [ ] A5 쿨다운·무부작용은 **BE가 권위로 강제**한다 — 쿨다운 중 회상 요청을 서버가 거부/무시하고(FE 비활성은 보조), 읽기전용 조회는 records/memories/evolution_history를 수정하지 않는다(진짜 NO_SIDE_EFFECTS). user_id 격리 유지.
- [ ] A6 쿨다운 임계는 `spec/values.yaml`(신규 `recall_cooldown`)에서 오며 코드 하드코딩이 아니다. 데모 가상 시계도 같은 값으로 판정한다(하룻밤 빨리감기로 쿨다운 해제 체험 가능).
- [ ] A7 회귀 없음: 원본 불변·"원본 일기 전체 보기" 토글·변천사(24)·별 3D 포트레이트/메타/발췌(change 32)·재공고화 물리(23/54)·공동 회상 강화 식·메인 우주 렌더가 그대로. FE build·lint·steiger·test / BE vet·build·test 통과.

### values + 계약 (먼저)
- [ ] T001 (gen:values) `spec/values.yaml` `recall:` 섹션에 `recall_cooldown_ms`(신규, 기본 `3600000` = 1시간) 추가. `recall.dwell_ms`는 회상 트리거에서 폐기되므로 사용처 제거 후 다른 용도가 없으면 정리(공동 회상은 버튼 트리거로 이동 — T006/T007). `pnpm gen:values`로 FE/BE 상수 생성.
- [ ] T002 (gen) `proto/cosimosi/v1/memory.proto` **append-only**(헌법5): ① 읽기전용 `rpc PeekMemory(PeekMemoryRequest) returns (PeekMemoryResponse)` + 메시지(`PeekMemoryRequest{memory_id}` / `PeekMemoryResponse{Record record; string fragment_text; string derived_text}`), `option idempotency_level = NO_SIDE_EFFECTS`. ② `RecallMemoryResponse`에 회상 성사/쿨다운 필드 추가(다음 빈 필드 번호) — 예 `bool recalled`·`int64 cooldown_remaining_ms`(쿨다운으로 막혔으면 `recalled=false`+잔여). `pnpm gen`(buf) → 호출부 수정.

### BE — 읽기전용 조회 + 회상 쿨다운 게이트
- [ ] T003 읽기전용 조회: `internal/memory/handler.go` `PeekMemory` + `service.go` `Peek(userID, memoryID)` — **부작용 없이** `repo.GetRecord`만 호출해 `Record`(+`FragmentText`·`DerivedText`)를 반환(touch·reconsolidate·EnqueueRewrite **안 함**). user_id 격리·읽기전용(GetRecord는 이미 read-only).
- [ ] T004 회상 쿨다운 게이트: `service.go` `RecallMemory`가 `TouchRecall` **전에** 그 별의 `last_recalled_at`을 읽어(repo에 read 추가 또는 `TouchRecall`이 쿨다운 게이트+성사여부 반환하도록 개정), `now - last < recall_cooldown_ms`면 **부작용 전부 스킵**(touch/reconsolidate/enqueue 안 함)하고 쿨다운 잔여를 응답으로 돌린다. `last_recalled_at`이 없거나(첫 회상) 쿨다운 경과면 기존대로 부작용 수행. handler가 `recalled`/`cooldown_remaining_ms`를 proto에 매핑.

### FE — 클릭=읽기, 버튼=회상, 쿨다운 UI
- [ ] T005 `features/recall/api/recall.ts`: `peekMemory(memoryId)` 신규(읽기전용 — `memoryClient.peekMemory`; 데모는 부작용 없는 읽기 `demoPeek`/내용만). `recallMemory`는 응답의 `recalled`/`cooldown_remaining_ms`를 `RecallResult`에 반영. 데모 `recallMemory`는 버튼에서만 `demoMarkRecalled`+`demoReshape`.
- [ ] T006 [MemoryPanel.tsx](../../frontend/src/features/recall/ui/MemoryPanel.tsx): `DWELL_MS` `setTimeout` **자동 회상 제거**. 패널이 열리면 `peekMemory`로 내용을 **즉시 읽기전용** 표시(조각/흐려진/원본 토글·포트레이트·메타는 change 32 그대로). **`이 별 자세히 보고 회상하기`** 버튼 추가 → `recallMemory`(부작용) 발화 후 회상 결과로 내용·별 갱신·우주 무효화. 쿨다운(서버 `cooldown_remaining_ms`>0)이면 버튼 비활성 + "다음 회상 가능까지 ~N분" 안내. 공동 회상 `recallFlushActor.send(RECORD_VIEW)`는 **버튼 누름**으로 이동.
- [ ] T007 [co-recall.ts](../../frontend/src/features/recall/model/co-recall.ts)·[recall-flush.machine.ts](../../frontend/src/features/recall/model/recall-flush.machine.ts): 공동 회상 누적 트리거를 dwell→버튼 회상으로. `DWELL_MS`/`dwell_ms` 의존 제거(공동 회상 강화 식·`co_recall_delta`·배치·멱등은 불변).
- [ ] T008 데모([shared/lib/demo](../../frontend/src/shared/lib/demo)): `peek`는 무부작용 내용 반환(`demoMarkRecalled` 안 함). 버튼 회상은 기존 `demoMarkRecalled`+`demoReshape`. 쿨다운은 `virtualNowMs` vs 데모 마지막 회상 시각으로 판정(하룻밤 빨리감기로 해제 — A6).

### 검증
