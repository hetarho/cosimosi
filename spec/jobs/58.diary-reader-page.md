## Acceptance Criteria (from plan/47.diary-reader-page)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [ ] A1 A **new route/screen** `pages/diary-reader`, **distinct** from the universe page, lists the user's `Diary`
      entries in reverse-chronological order (reader-app-style); the universe stays `/` and primary — the reader is a
      fully supporting archive ([D2]).
- [ ] A2 `GetDiaries` is a **unary, HTTP GET-eligible, `NO_SIDE_EFFECTS`, per-user-scoped, paginated** read returning
      each diary's `id`, immutable `body`, `diary_date`, `created_universe_time`, and its split membership
      (`DiarySplitRef` = memory id + name + mood); it carries no derived values and no positions ([I5]).
- [ ] A3 `entities/diary` is a **pure** mirror (no three) with a `model` (type + store) and an `api` mapping
      `GetDiaries` proto→domain; it is shared verbatim web + mobile.
- [ ] A4 Opening an entry shows the **full immutable body verbatim** ([D4][I2]) plus a **split view** listing the
      **2–5** episodic memories it split into ([E2][D3]), each chip showing the memory name + primary-emotion color;
      a diary whose memories are all soft-deleted still lists (record persists) but shows no chips ([I1]).
- [ ] A5 **Reading is free** ([G4]): listing, opening, reading the body, and viewing the split spend **no** 별가루
      and advance **no** clock; only the jump spends.
- [ ] A6 Each entry offers **"이 일기로 태어난 별 보기"** that displays [44]'s server quote
      `QuoteSpend(kind=DIARY_RECALL, diary_id)` = the sum of the diary's still-live memories' per-memory recall costs
      (decay-depth priced by [38]/[43]) and gates the action behind that spend ([D3][G4]); the widget hardcodes and
      computes no price.
- [ ] A7 Confirming the jump opens the **shared sync-consent modal** ([31]) first ([R1a]); **아니오** cancels with
      the clock **unmoved** and **nothing spent**; **예** proceeds to `RecallDiaryStars`.
- [ ] A8 On **예**, `RecallDiaryStars` ([33]) server-side syncs to today, spends, and reinforces the diary's
      still-live memory ids in one transaction — resetting each memory's `last_recalled_universe_time`, bumping
      `recall_count`/`EffectiveStrength` ([R3]), resetting the semanticize timer ([C6a]), and nudging neighbors
      ([R5]) — and it is **NOT** reconsolidation: **no** `current_text` and **no** `seed` change ([R6][I8][D3]). It
      is idempotent over the batch.
- [ ] A9 After a successful jump the clock is **synced to today** ([R1a]), the [31] **acceleration** plays over the
      sync interval, and the camera **flies** to the recalled cluster ([23]); the recovered
      brightness/decay/gist-timer ([F5]) surface on the **next** `GetUniverse` refetch (§2.8) — no polling, no
      streaming.
- [ ] A10 The reader is reachable from **two** entry points ([D5]): a universe-page affordance (lands at the list
      top) and the star-detail **원본 일기 보기** button ([35]) which deep-links to **that memory's** diary entry via
      `memory.diary_id`.
- [ ] A11 The `Diary` is **never** mutated, deleted, or re-split by this reader ([I2][D4]); the jump only ever
      **advances** the clock, never rewinds ([I10]).
- [ ] A12 The jump's **control-state** is one XState machine (`browsing → confirming → recalling → flying`); the
      list, selected entry, server quote, and target ids live in **Zustand/Query**, not machine context; a failed
      `RecallDiaryStars` returns to `browsing` with nothing spent and the clock unmoved (§3.2).
- [ ] A13 **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in a
      literary / restrained Korean voice.
- [ ] A14 **web + mobile parity:** the same page + widget + two features + entity mirror run on both apps, sharing
      `model`/`api` verbatim; `*.native.tsx` siblings exist **only** for the list/scroll/row primitives (§3.5).
