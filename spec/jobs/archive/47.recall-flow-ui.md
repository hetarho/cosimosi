## Acceptance Criteria (from plan/36.recall-flow-ui)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 A **회고하기** button on the star-detail panel opens `widgets/recall-flow` as a modal / bottom-sheet **over**
      the running universe canvas (the renderer is not remounted); the flow is scoped to an **episodic** star and is not
      offered for gist stars ([R1][D1][14]).
- [x] A2 When the universe clock ≠ today, the flow opens the **reusable sync-consent modal** built in plan 31 / job 42
      ([T2] case 2 / [R1a]); **예** ⇒ the later `Recall` call server-side syncs via `SyncToToday` and applies the recall
      atomically, then [31] plays the returned interval while the result is revealed; **아니오** ⇒ the whole flow is
      cancelled with the clock **unmoved**.
- [x] A3 When the clock is already today, the sync-consent modal is **skipped** and the flow goes straight to rewriting.
- [x] A4 `features/recall-star` shows the star's **faded current text** as the prompt-to-remember (the same free text
      the panel shows, [35]) and a **rewrite** text area; the original `Diary` is never shown or restored ([R1][I8]).
- [x] A5 On **회고 confirm** the flow calls the **`Recall`** RPC ([33]) with `{ episodic_memory_id, rewrite_text }` as
      a **synchronous** call (a `reconsolidating` loading state covers the LLM compare + recall); the request carries
      **no** position / color / strength / seed / time fields ([I3][I11]).
- [x] A6 On **prediction error** ([R6]) the result is **reconsolidated**: the star's **seed changes → shape changes**
      ([V5]) and its **current text** becomes the rewrite; brightness/decay/gist-timer/strength recover (anchors reset,
      the visible recovery lands with plan 39); the distortion is **not** announced ([R8a]).
- [x] A7 On **no prediction error** ([R4]) the result is **reinforced only**: the star's **shape and current text are
      unchanged**; only brightness/strength/gist-timer recovery occurs; the result view states this plainly.
- [x] A8 The FE **never** decides the branch — it reflects the server's `outcome` ([R6] compare is server-side); a
      failed `Recall` applies nothing and returns to a retriable `rewriting` with the rewrite text intact.
- [x] A9 The **`Diary` is never mutated** by recall or reconsolidation under any branch — only the representation
      (seed + current text) changes ([I2][R7]).
- [x] A10 The flow's **control-state** is one XState machine (`idle → confirmingSync → rewriting → reconsolidating →
result` + error/cancel paths) with `reconsolidating` as the loading state; the **loaded memory ref, rewrite text,
      and result live in Zustand/Query**, not machine context (§3.2).
- [x] A11 The flow **hardcodes no 별가루 price** and proceeds under the no-op allow-all spend gate; economy gating is
      deferred to Epic F (**CC2**), and it makes **no** direct clock/sync call (sync goes through [31]'s composition,
      [R1a]).
- [x] A12 **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in a
      literary / restrained Korean voice; the distortion is never announced ([R8a]).
- [x] A13 **web + mobile parity:** the same widget + feature run on both apps, sharing `model` / `api` verbatim;
      `*.native.tsx` siblings exist **only** for the sheet / rewrite-input primitives (§3.5).
