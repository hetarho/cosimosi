## 인수 조건 (acceptance — 기획 관점, changes/05.invite-link-onboarding에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [x] A1 `/admin?tab=invite`의 각 발행 코드 행에는 `https?://<현재 origin>/invite?code=<code>` 형식의 초대 URL을 복사하는 액션이 있다. 기존 코드 문자열 복사·revoke·목록 표시는 회귀하지 않는다.
- [x] A2 브라우저가 Web Share API를 지원하면 초대 URL 공유 액션이 OS/브라우저 공유 시트를 연다. 지원하지 않거나 실패하면 URL 복사로 폴백한다. 서버 발송 RPC나 이메일 전송은 생기지 않는다.
- [x] A3 미로그인 사용자가 `/invite?code=<code>`에 진입하면 `/sign-in`으로 즉시 리다이렉트되지 않고, 초대장 화면을 먼저 본다. 화면에는 "초대장을 받았어요. 나만의 우주를 만들어볼까요?" 계열의 절제된 카피와 `회원가입하기` CTA가 보인다.
- [x] A4 초대장 화면의 `회원가입하기`는 기존 `/sign-in`으로 이동하며, 인증 후 돌아올 redirect에 `/invite?code=<code>`를 보존한다. redirect는 내부 경로만 통과하고 `//`·`/\`·자기 재귀 루프를 허용하지 않는다.
- [x] A5 OTP 인증 또는 Google OAuth 후 사용자는 `/invite?code=<code>`로 돌아온다. 풀페이지 OAuth에서도 초대 코드가 소실되지 않는다. 인증된 비멤버라면 별도 코드 입력 없이 자동으로 `RedeemInviteCode(code)`가 호출된다.
- [x] A6 자동 redeem 성공 시 멤버십 캐시는 stale false가 남지 않도록 제거되고, 기존 환영 연출 후 redirect(없으면 `/`)로 이동한다.
- [x] A7 자동 redeem 실패 시 코드는 소비되지 않고 기존 사유별 오류 카피가 표시된다. 사용자는 같은 `/invite` 화면에서 코드를 수정하거나 다른 코드를 입력해 수동 redeem할 수 있다.
- [x] A8 이미 멤버인 사용자가 `/invite?code=<code>`를 열면 초대 코드는 추가 소비되지 않고 redirect(없으면 `/`)로 이동한다.
- [x] A9 `/invite`에 `code`가 없으면 기존 수동 초대 코드 입력 페이지가 그대로 동작한다. `MembershipGate`가 보내는 `/invite?redirect=<원래경로>` 흐름은 회귀하지 않는다.
- [x] A10 `/admin` 비관리자 NotFound 위장, `InviteAdminService` admin allowlist, `InviteService` auth-only, 코어 RPC membership 강제는 plan 41 수용 기준 그대로 유지된다.
- [x] A11 사용자-facing 카피는 `policy/ux/voice-tone.md`의 규칙을 따른다. 장식 이모지는 쓰지 않고, reduced-motion 환경에서도 초대장·redeem 상태의 의미가 전달된다.
- [x] A12 FE build/lint와 관련 라우트·초대 흐름 테스트가 통과한다.

- [x] T002 `/invite` 공개 진입 허용 — 현재 `/invite`는 `SessionGate` 안이라 미로그인이 곧장 `/sign-in`으로 튕긴다. `code`가 있으면 **미인증도 초대장 표면을 먼저** 보도록 라우트 배치/게이트를 조정(공개 진입 + 인증 후 자동 redeem 경로). 코어 우주 RPC membership 강제·`/admin` allowlist는 불변(회귀 경계). (`src/app/router.tsx`·`src/app/ui/SessionGate.tsx`)
- [x] T003 `InvitePage` `code` 분기 — (a) **미로그인+code**: 우주 백드롭 위 초대장 카피("초대장을 받았어요. 나만의 우주를 만들어볼까요?" 계열) + `회원가입하기` CTA → `/sign-in?redirect=/invite?code=<code>`. (b) **인증 비멤버+code**: 수동 입력 없이 `RedeemInviteCode(code)` 자동 호출. (c) **이미 멤버+code**: 무소비 → redirect(없으면 `/`). (d) **code 없음**: 기존 수동 입력 페이지 그대로. (`src/pages/invite/ui/InvitePage.tsx`·`api/invite-queries.ts`)
- [x] T004 자동 redeem 결과 처리 — 성공: 멤버십 쿼리 캐시 제거(stale false 방지·기존 흐름 재사용) → 환영 연출 → redirect(없으면 `/`). 실패(`NOT_FOUND`·`EXPIRED`·`EXHAUSTED`·`REVOKED`): 코드 무소비 + 기존 사유별 카피로 수동 입력 폴백(같은 화면에서 수정·재시도). (`src/pages/invite/ui/InvitePage.tsx`)
- [x] T005 로그인 redirect 보존 + OAuth 소실 방지 — 사인인 후 `/invite?code=<code>`로 복귀. **풀페이지 Google OAuth는 `/`로 복귀해 `?redirect` 소실**(plan 01)이므로, 초대 `code`(+redirect)를 **sessionStorage 등 클라 보존**으로 살려 복귀 시 `/invite?code=`로 복원. redirect는 내부 경로만(오픈 리다이렉트·자기 재귀 차단 — `safeRedirect` 재사용). (`src/app/ui/session-context.tsx`·`src/app/ui/SignInRoute.tsx`·`src/app/router.tsx`)
- [x] T006 [P] 사인인 초대 문맥(얕게) — redirect가 `/invite`면 "초대장으로 시작" 계열 절제 카피 보강(voice-tone). 새 auth API·가입 폼 없음. (`src/app/ui/SignInScreen.tsx`)
- [x] T007 [P] 관리자 초대 URL 복사/공유 — `/admin?tab=invite` 각 코드 행에 **초대 URL 복사** + **Web Share 공유**(미지원/실패 시 복사 폴백) 액션. URL = `` `${location.origin}/invite?code=<code>` ``. 기존 코드 문자열 복사·revoke·목록 회귀 없음. 서버 발송 RPC 신설 없음. (`src/pages/admin/ui/InviteCodesTab.tsx`·`api/invite-admin-queries.ts`)
- [x] T008 [P] 접근성·카피 — 초대장/redeem 상태가 reduced-motion에서도 의미 전달, 장식 이모지 금지(voice-tone). (`InvitePage`·`SignInScreen`)
- [x] T009 검증 — `pnpm --filter ./frontend build` + `lint` + 관련 라우트·초대 흐름 테스트. 회귀: `MembershipGate`의 `/invite?redirect=` 흐름·plan 41 게이트(A9·A10).
