# Job 19: 데모 튜토리얼모드 스포트라이트 투어  (new)

> 구현 작업 문서. 출처 스펙: [plan/48.demo-guided-tour](../plan/48.demo-guided-tour.md).
> `/implement-job 19`이 아래 두 체크리스트로 구현한다. 선행 전제는 [plan/47.demo-free-mode-onboarding](../plan/47.demo-free-mode-onboarding.md)이 완료된 데모 자유모드다.

## 인수 조건 (acceptance — 기획 관점, plan/48.demo-guided-tour에서)

- [x] A1 plan 47의 모드 선택에서 `기능 하나하나 알아보기`를 누르면 `tutorial` flow로 진입하고, 선택한 페르소나의 데모 우주가 뒤에 보인다.
- [x] A2 튜토리얼 시작 시 우주 캔버스는 언마운트되지 않고, 그 위에 dim + spotlight + coach card가 표시된다.
- [x] A3 튜토리얼은 `UI 숨김/보이기`, 테마, 페르소나, 시간, 햄버거, 시점 전환, 망원경, `일기` 탭, `별` 탭, `새 별 띄우기`, 별 클릭/회상을 순서대로 안내한다.
- [x] A4 각 단계는 현재 대상 버튼 또는 표면을 시각적으로 하이라이트한다. 대상 DOM을 찾지 못해도 튜토리얼은 crash하지 않고 중앙 안내 fallback을 보여준다.
- [x] A5 coach card에는 현재 단계 제목/짧은 설명/진행 표시/`이전`/`다음`/`건너뛰기`가 있다.
- [x] A6 사용자는 target을 직접 눌러 볼 수 있고, 직접 누르지 않아도 `다음`으로 진행할 수 있다.
- [x] A7 페르소나/시간/햄버거/망원경 단계에서 필요한 팝오버·사이드바·시트는 튜토리얼이 열거나 닫을 수 있으며, 다음 단계에 불필요한 표면은 정리된다.
- [x] A8 `UI 숨김/보이기` 단계에서 사용자가 UI를 숨겨도 튜토리얼 overlay는 남고, 다음 target 단계로 넘어가기 전 UI가 다시 보이게 복구된다.
- [x] A9 `새 별 띄우기` 단계에서 사용자가 버튼을 누르면 plan 47의 데모 랜덤 별 생성 경로가 그대로 실행되고, 서버 쓰기 없이 새 별이 데모 우주에 나타난다.
- [x] A10 별 클릭/회상 단계는 기존 회상 경로를 사용한다. 튜토리얼은 별/시냅스/원본 일기를 수정·삭제하지 않는다.
- [x] A11 `건너뛰기` 또는 마지막 단계의 `자유롭게 탐험하기`를 누르면 flow가 `free`가 되고 튜토리얼 overlay가 사라진다.
- [x] A12 튜토리얼 완료 뒤 같은 데모 세션에서 자유모드로 남으며, 새로고침해도 튜토리얼이 자동 재시작되지 않는다.
- [x] A13 `다시 보기` 진입점으로 사용자가 명시적으로 요청하면 첫 단계부터 다시 볼 수 있다.
- [x] A14 체험 종료(`exitDemoMode`)는 튜토리얼 flow와 step 상태를 함께 지운다.
- [x] A15 실계정 우주에서는 튜토리얼모드, 스포트라이트 overlay, demo tour 상태가 나타나지 않는다.
- [x] A16 튜토리얼 overlay는 캔버스 밖 DOM으로 구현되며, 3D 씬 안 `<Html>`을 추가하지 않는다.
- [x] A17 순수 step 정의/model은 React/DOM/three를 import하지 않는다.
- [x] A18 키보드 사용자는 `Tab`/`Enter`/`Escape`로 coach card와 target을 사용할 수 있고, screen reader에는 현재 단계와 target 이름이 전달된다.
- [x] A19 `prefers-reduced-motion`에서는 spotlight 이동/coach card 전환이 과한 motion 없이 즉시 또는 매우 절제된 전환으로 동작한다.
- [x] A20 백엔드·proto·RPC·sqlc 변경 없이 FE만으로 구현된다.
- [x] A21 build·lint·typecheck·steiger를 통과하고, plan 47 자유모드 진입/페르소나 전환/시간 이동/망원경/새 별 생성/일반 실계정 우주에 회귀가 없다.

## 구현 체크리스트 (implementation — 구현 관점)

- [x] T001 선행 게이트 확인 — plan 47/job 18의 데모 자유모드가 완료되어 있는지 확인한다. 모드 선택, `free` flow, 데모 랜덤 별 생성, 좌상단 페르소나/시간 버튼, 우상단 햄버거/시점/망원경, 하단 `새 별 띄우기`, UI 숨김이 아직 없으면 job 19를 시작하지 않고 job 18을 먼저 끝낸다.
- [x] T002 `frontend/src/shared/lib/demo/flag.ts`에 tutorial flow와 step 상태 helper를 추가한다. `enterTutorialMode`, `completeTutorial`, `skipTutorial`, `restartTutorial`, `get/setTutorialStep` 또는 동등 API를 두고, `exitDemoMode()`/reset 경계가 tutorial 상태를 함께 비우게 한다.
- [x] T003 [P] `frontend/src/shared/lib/demo/*.test.ts`에 tutorial flow 테스트를 추가한다: mode 선택에서 tutorial 진입, step 저장/복원, 완료·건너뛰기 후 `free`, `exitDemoMode()` cleanup, 실계정 모드에서 no-op.
- [x] T004 `frontend/src/widgets/demo-tour/model/steps.ts`를 만든다. step 정의는 target id, 제목, 짧은 설명, 필요 표면(open menu/telescope/none), 다음 전 cleanup만 담고 React/DOM/three를 import하지 않는다.
- [x] T005 `frontend/src/widgets/demo-tour/ui/DemoGuidedTour.tsx`를 만든다. dim, spotlight ring, coach card, progress dots, 이전/다음/건너뛰기/자유롭게 탐험하기 액션을 렌더한다.
- [x] T006 `frontend/src/widgets/demo-tour/model/use-tour-target.ts` 또는 동등 UI hook을 추가한다. `data-tour-id` target을 `ResizeObserver`/`getBoundingClientRect`로 측정하고 resize/scroll/표면 open 변화에서 rect를 갱신한다. target이 없으면 중앙 fallback rect 없이 coach card만 렌더한다.
- [x] T007 [P] reduced-motion 대응을 추가한다. `prefers-reduced-motion`이면 spotlight/coach 전환이 즉시 또는 최소 transition으로 동작하게 하고, CSS/animation 리터럴이 레이아웃을 흔들지 않게 한다.
- [x] T008 plan 47 HUD 요소에 `data-tour-id` 또는 ref registry를 부여한다: `ui-toggle`, `theme`, `persona`, `time`, `menu`, `view`, `telescope`, `new-star`.
- [x] T009 `UniverseExplorerSheet` 또는 동등 망원경 시트에 `explorer-diary-tab`, `explorer-star-tab` target id를 부여하고, 튜토리얼이 탭 전환을 요청할 수 있는 최소 제어 props를 추가한다.
- [x] T010 `HomePage` 또는 동등 우주 셸에 `DemoGuidedTour`를 마운트한다. `isDemoMode()` + flow `tutorial`에서 자유모드 HUD 위에 overlay를 올리고, 실계정/비demo에서는 절대 렌더하지 않는다.
- [x] T011 튜토리얼 step orchestration 콜백을 `HomePage`에 연결한다. step 진입 시 필요한 표면(페르소나 팝오버, 시간 팝오버, 햄버거 사이드바, 망원경 시트, 일기/별 탭)을 열고, 다음 단계에 불필요한 표면은 닫는다.
- [x] T012 UI 숨김 단계 복구를 구현한다. 사용자가 `UI 숨기기`를 눌러도 tour overlay는 남고, 다음 단계로 이동하기 전에 HUD가 자동으로 보이게 된다.
- [x] T013 `새 별 띄우기` 단계는 plan 47의 데모 랜덤 별 생성 경로를 그대로 호출하게 한다. 튜토리얼 전용 서버 쓰기/별 삭제/데이터 초기화는 만들지 않는다.
- [x] T014 별 클릭/회상 단계 fallback을 구현한다. 새로 만든 별 id를 추적할 수 있으면 해당 별을 안내하고, target 별을 특정할 수 없으면 canvas region/중앙 안내로 "별을 눌러 기억을 읽어요"를 보여준다. 기존 focus/recall 경로만 사용한다.
- [x] T015 coach card 접근성을 마감한다. 현재 단계는 `aria-live`로 전달하고, target name/step position을 screen reader에 제공하며, `Escape`는 건너뛰기 또는 닫기 확인 없이 자유모드로 수렴한다.
- [x] T016 키보드 흐름을 정리한다. coach card 버튼은 Tab 순서가 안정적이고, target을 직접 누르는 흐름과 `다음`으로 넘기는 흐름이 둘 다 가능해야 한다.
- [x] T017 [P] visual QA를 준비한다. desktop/mobile viewport에서 spotlight가 버튼을 가리지 않는지, coach card가 target·하단 버튼·시트 탭 위에 겹치지 않는지 수동 체크 목록을 job 구현 중 남긴다.
- [x] T018 [P] unit/component test를 추가한다. step 정의 순서, target fallback, 완료/건너뛰기 flow, demo-only 렌더 조건, `free` 전환을 검증한다.
- [x] T019 [P] FSD/레이어 확인 — `widgets/demo-tour/model/steps.ts`는 React/DOM/three 미의존, DOM 측정 hook은 ui 또는 widget 내부, `shared/lib/demo`는 app/page import 없음.
- [x] T020 구현 완료 후 SSOT 문서를 갱신한다: `plan/48.demo-guided-tour.md`는 as-built로, `plan/47.demo-free-mode-onboarding.md`의 `tutorial_tbd`는 plan 48 참조/완료 상태로, `plan/19.engram-experience.md`는 자유모드/튜토리얼모드 분기 기준으로 반영한다.
