## Acceptance Criteria (from plan/27.writing-flow-ui)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 A **"일기 쓰기"** affordance on the universe page opens the `widgets/writing-flow` modal / bottom-sheet **over** the
      running universe canvas (the renderer is not remounted); the entry point is distinct from the final "별 띄우기" confirm
      ([W1][W3]).
- [x] A2 `features/write-diary` provides a diary **body** input and a **diary-date** picker whose default value is **today**
      ([W5]); the date is user-editable.
- [x] A3 **별 쪼개기** calls **`SplitDiary`** as a **synchronous** preview and renders **2–5** proposed memories ([E2]), each
      showing its suggested **name** ([W2a]) and primary **emotion/mood** ([W2]); the split performs **no** persistence.
- [x] A4 Each proposed memory's **name** and **primary emotion** are **editable**, and memories can be **merged / split** and
      neuron **normalization** corrected — all within the session ([W2][W4][E10]).
- [x] A5 A **natural-language** instruction re-runs the split via **`ReviseSplit`** (sending the current proposal + the
      instruction) and replaces the proposal with the same schema-forced output ([W4a]); hand-edit and NL edit reach the
      same result.
- [x] A6 The editable surface exposes **only** name / emotion / neuron membership — there are **no** position, color,
      strength, or time controls anywhere in the flow ([W4a][I3]).
- [x] A7 Editing is **session-only**: an already-launched (past) star is **not** directly editable from this flow; the widget
      edits the pre-launch proposal, never a `GetUniverse` memory ([W4]).
- [x] A8 **별 띄우기** calls **`LaunchStars`**, which returns memory **ids**; the client **optimistically** inserts the new
      star(s) and the canvas renders them **immediately** with a visual effect ([W3]), while embeddings / gist texts fill on
      the **next** `GetUniverse` read (§2.8) — no polling, no streaming.
- [x] A9 A newly launched star **awakens** a latent star (the `features/awaken-neuron` animation is triggered, [25]); this
      flow emits the launch event and does not itself implement the animation or the star body.
- [x] A10 The `Diary` is persisted **immutable** and only at **launch** (atomically with its memories) — not at split; a failed
      launch rolls back the optimistic insert and returns to the editable proposal, leaving nothing persisted ([I2]).
- [x] A11 A **past-dated** diary (before the latest launched universe-time) **saves the diary but launches no star**, and the
      flow surfaces that outcome as a **one-time confirmation notice** before launch, in i18n copy ([W5][T1][I10]).
- [x] A12 The flow's **control-state** is one XState machine (`idle → writing → splitting → reviewing → revising → launching →
done` + error paths) with `splitting`/`revising` as loading states; **draft body, diary date, proposed memories, and
      edits live in Zustand/Query**, not in machine context (§3.2).
- [x] A13 **All** user-visible copy resolves through the i18n seam (no raw strings; `lint:raw-strings` passes), in a literary /
      restrained Korean voice.
- [x] A14 **web + mobile parity:** the same widget + four features run on both apps, sharing `model` / `api` verbatim;
      `*.native.tsx` siblings exist **only** for the sheet / input / date-picker primitives (§3.5).
