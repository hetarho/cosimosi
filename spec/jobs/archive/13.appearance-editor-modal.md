## 인수 조건 (acceptance — 기획 관점, changes/06.appearance-editor-modal에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [x] A1 좌상단 별가루·테마 알약을 누르면 **집중 모달**이 열리고, 상단 **고정 프리뷰**에 나 + 별 3개가 **시냅스로 연결(강한 연결·약한 연결 혼합)** 되어 현재 선택(4축)으로 렌더된다.
- [x] A2 모달에 `배경 / 별 / 광원 / 시냅스` 4개 탭이 있고, 탭 전환 시 해당 축의 항목 리스트가 **스크롤**된다. 광원 탭은 나(self) 축을 조정한다.
- [x] A3 항목 칩이 그라디언트 swatch가 아니라 **실제 모형 썸네일**(사전 렌더 이미지)로 표시된다 — 별/광원/시냅스는 형태, 배경은 백드롭 룩.
- [x] A4 항목을 고르면 고정 프리뷰가 즉시 그 선택을 반영하고(라이브 드래프트), 저장 시 우주에 반영된다. **드래프트→저장(미구매분 일괄 구매·잔액 부족 시 막힘)·되돌리기**가 기존대로 동작한다.
- [x] A5 잠긴 유료 항목은 자물쇠 + `저장 시 N 별가루`로 표시되고, 체험(데모)에선 전부 잠금 해제·라벨 `저장`(무상)이다.
- [x] A6 항목 썸네일은 **사전 렌더 1회 + 이미지 캐시**로 그려진다 — 칩마다 라이브 WebGPU 캔버스를 만들지 않는다(모달이 더하는 GPU 컨텍스트는 고정 프리뷰 + 썸네일 렌더로 제한).
- [x] A7 모달을 닫아도 드래프트·`AppearanceSaveBar` 상태가 기존 규칙대로 유지·동작한다(저장/되돌리기 회귀 없음).
