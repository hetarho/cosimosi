## Acceptance Criteria (from code-review/01.phase-1-critical-readiness-audit)

<!-- Testable end-state for each chosen finding. After building, verify each is true in the running code. -->

- [x] A1 (R006) A `signedOut`/`expired` adapter event delivered while `bootstrap()` is in flight is not lost: after
      bootstrap resolves, the session is `signedOut`, never re-authenticated. Covered by a regression test that fires
      `onChange(signedOut)` during a pending bootstrap.
- [x] A2 (R007) When `adapter.signOut()` rejects, the facade does not remain in a falsely-terminal state: the optimistic
      local flags (`locallySignedOut`/`suppressAdapterRefresh`/`epoch`) and machine state are restored before the error
      is surfaced. Covered by a test where `adapter.signOut()` rejects.
- [x] A3 (R008) `supabase-adapter.getAccessToken()` attempts a refresh when the cached session is expired/near-expiry
      and returns the refreshed token in the post-expiry / pre-auto-refresh window (honoring the `auth-adapter.ts`
      contract). Covered by a test with an injected clock.
- [x] A4 (R009) Booting the API with a _partial_ Supabase config (e.g. `SUPABASE_JWT_SECRET` set, URL unset; and the
      reverse) starts with auth disabled and logs "not configured" — `os.Exit`/`Fatalf` no longer fires. Covered by a
      Go test over the partial-config combinations.
- [x] A5 (R010) Mobile has a build-time/env feature-flag override seam (parity with web's `VITE_COSIMOSI_FLAG_`), so
      `platform.diagnosticsSurface` can be enabled on device without remote PostHog + granted consent. Covered by a test.
- [x] A6 (R011) `InMemoryTelemetryAdapter` set and get resolve the **same** key (both through the definition), so
      `setFeatureFlag` by the natural key is honored. The existing test sets by the natural key and still passes.
- [x] A7 (R012) Changing vendor props (`release`/`dsn`/`posthog`) closes/disposes the prior Sentry client before any
      re-init, so no second live client is created (or vendor config is made immutable for the provider's lifetime).
- [x] A8 (R014) The web i18n provider performs **no** external-store write during render; the override is applied only
      in an effect, and the in-code comment matches the implementation. Covered by a test asserting render notifies no
      locale subscribers.
- [x] A9 (R015) `kill-switch`/`operational` flag kinds are consulted regardless of consent; experiment/analytics flags
      remain consent-gated. Covered by a test: `consent='denied'` + remote kill-switch `true` → flag reflects remote.
- [x] A10 (R016) Registry definition fails fast (throws) when two flag keys derive the same env-override name. Covered by
      a test with two colliding keys.
- [x] A11 (no-regression) All existing suites stay green — `pnpm test:auth`, `test:observability`, `test:client-cache`,
      `test:state-machine`, `test:i18n`, `test:web`, `test:mobile`, `test:api` — and only the targeted broken behaviors
      change.

### Auth — packages/auth + apps/api (R006–R009)

- [x] T001 (R006) In [packages/auth/src/facade.ts](../../packages/auth/src/facade.ts) `onChange`, stop blanket-dropping
      terminal events during the bootstrap window: for `signedOut`/`expired`, advance `epoch` (and set
      `locallySignedOut`) so a resolving `bootstrap()` fails `shouldApply()`, or queue the terminal event and apply it
      after `BOOTSTRAP_DONE`. Add a regression test (onChange(signedOut) during a pending bootstrap → ends `signedOut`).
- [x] T002 (R007) In `facade.ts` `signOut()`, wrap `await adapter.signOut()` so a rejection restores the prior
      optimistic flags + machine state (then rethrow or surface the error). Add a rejection test.
- [x] T003 (R008) In [packages/auth/src/supabase-adapter.ts](../../packages/auth/src/supabase-adapter.ts)
      `getAccessToken()`, attempt `refreshSession()` when the session is expired/near-expiry before returning null. If a
      numeric near-expiry skew is needed, add it to `spec/values.yaml` and import the generated constant **(gen:values)**
      rather than hardcoding. Add a clock-injected test.
- [x] T004 (R009) Make partial Supabase config non-fatal: either in
      [apps/api/internal/platform/supabase_auth.go](../../apps/api/internal/platform/supabase_auth.go)
      `NewSupabaseJWTVerifierFromEnv` treat _any_ missing required field as unconfigured (`ok=false, nil err`), or in
      [apps/api/cmd/api/main.go](../../apps/api/cmd/api/main.go) `authHandlerOptions` stop `Fatalf`-ing on
      `ErrSupabaseAuthNotConfigured` and log+continue. Add Go tests for the partial combos. Run `go vet`/`go build`.
