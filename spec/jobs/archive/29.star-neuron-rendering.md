- [x] A1 Three visual (rendering) entities exist — `entities/star`, `entities/cell-star`, `entities/filament` — each
      projecting exactly one plan-23 domain-mirror entity (`episodic-memory` / `neuron` / `synapse`) imported **only via
      `@x`**, and each emitting its body through the plan-14 asset-source port (`VisualBodySource`); no slice imports
      `three` directly.
- [x] A2 The projection is **one-way**: no visual vocabulary (`star`/`cell-star`/`filament`/`radius`/`brightness`/…)
      appears as a domain symbol in the domain-mirror slices or their `api` mappers, and no visual entity writes back into
      the domain slice or the API mapper (the §1 cross-vocabulary lint passes).
- [x] A3 A `star` body's **size** is its `EffectiveStrength` (arousal-initial [E8a] + recall accumulation) read at
      render time from `packages/memory-logic` — larger strength renders a larger star ([V3]).
- [x] A4 A `star` body's **brightness** is its `EffectiveBrightness` read at render time from `packages/memory-logic`;
      in Epic A (no forgetting) it renders at full brightness and the brightness channel/uniform is present for Epic D to
      fill ([V2]) — the seam is noted, decay is not built.
- [x] A5 A `star` body's **color** is its primary emotion via the single `mood → color` palette seam owned by plan 17
      (`packages/emotion`); the star entity is palette-agnostic and color is bound to emotion **only** — no other channel
      sets color ([M3][V4][I3]).
- [x] A6 A `star` body's **shape** is a TSL shader form parameterized by the star's stored `seed`, giving a
      coherent-but-varied big-star form; two stars with different seeds render different forms ([V5]).
- [x] A7 The `seed` is treated as **immutable input**: this unit renders the current seed and never mutates or animates
      a seed change — seed change happens **only on reconsolidation** (Epic C `Reshape` [R6]); abstraction is z-axis gist,
      **not** star shape ([V5][C5]–[C7]).
- [x] A8 A `cell-star` (neuron) body is a **simple, seedless point** with **no emotion color** and no per-neuron
      seed-form, positioned at the force-sim coordinate the canvas feeds ([V5][I5]).
- [x] A9 A `filament` (synapse) body is a **batched camera-billboarded ribbon fat-line** whose **width and brightness
      track its effective synapse strength** (plan 18 `EffectiveSynapseStrength`, read-time); a filament connects **two
      neurons only** and is **never** drawn between two stars ([V6][I4][I6]).
- [x] A10 Emotion drives **only** color — it never sets a `cell-star`/`filament` geometry, a coordinate, or an edge
      ([I3]); positions/constellations stay emergent (drawn at fed coordinates, never stored or reverse-projected, [I5]).
- [x] A11 Star bodies are authored in **TSL** (compiling to WGSL + GLSL) and drawn **instanced**; cell-stars are
      instanced points and filaments are batched fat-lines; per-instance appearance rides instance attributes/uniforms and
      per-frame updates run in `useFrame`, never React state (§3.3/§3.2).
- [x] A12 The star / cell-star / filament bodies render on **web (open beta) from one TSL source shared with mobile
      (MVP)**; mobile is source-gate verified (typecheck · lint · test) with on-device render pending (see Decisions,
      `pnpm ios`). Mobile MVP may lower instance counts / drop post-FX without changing the shared body or projection (§3.5).
- [x] A13 This unit **refines** the plan-23 scene: the generic instanced nodes / fat-line edges are replaced by the
      specific star / cell-star / filament bodies inside the existing `widgets/universe-canvas`; the renderer, force-sim
      worker, camera rig, and `GetUniverse` read are consumed, not rebuilt.

## Review

- [x] `/code-review` applied (rejections noted with reason) · for non-trivial logic, `/codex:review --background`

### Review outcome (10 finders across `/code-review` + codex `gpt-5.5`; codex: no high-severity)

**Applied**

- `InstancedNodeLayer` per-instance-attribute upload leaked a GPU buffer on every channel change (new
  `InstancedBufferAttribute` each time) → reuse the buffer in place when the size is unchanged; skip upload at
  `count ≤ 0` (a zero-length attribute mismatches the min-1 capacity).
- Hoisted `makeScale` for the uniform-scale (cell-star) path — the per-instance recompose is only for stars.
- `filament-body` → `side = DoubleSide`: the camera-billboarded quad's facing flips with view/endpoint order, so a
  single-sided ribbon culled ~half the filaments as the camera orbits (regression vs the old 1px `LineSegments`).
- `FatLineLayer` grazing-angle perp fallback (world-axis) so an end-on filament keeps width instead of shimmering; a
  truly coincident edge (a == b) still collapses to invisible. Also dropped a set-during-render ref (read the geometry
  attribute in `useFrame`) and disposed the material in the non-adopt branch.
- NaN-safe channel mapping (`lerpClamp` floors a non-finite read-time value to the range min) so a skewed/corrupt DTO
  can't write a NaN scale / ribbon vertex — the star/filament path lacked the graph builder's finite-coercion.
- Deduped `elapsedUniverseDays` into `@cosimosi/memory-logic` (one universe clock, companion to the effective-* fns);
  star + filament import it. Mobile jest mock gained the new `@cosimosi/3d-renderer` symbols; dropped a dated code note.

**Rejected (with reason)**

- One-frame refetch transient (layer `count` from the deferred stores vs `firstNodeIndex`/`nodeIndex` from the sync
  graph): self-correcting next frame, no crash (buffer reads guarded), and inherent to the deferred sim-bridge (the
  coordinate buffer is itself repopulated in an effect, so a structural refetch already glitches one frame) — a
  read-model-flow refactor is out of this "refine the bodies" job's scope.
- Inline `ref` callback re-running `count=0`/`visible=false` per commit: pre-existing (not job 29), and R3F draws in
  `useFrame` (which restores them before paint), so no visible flicker.
- Dead `createPrimitiveBodySource` / `EdgeLineLayer` / `buildSynapseEndpointIndexPairs`: kept as valid generic package
  API (tested; `createPrimitiveBodySource` still cited in tech/rendering.md); deleting public API is out of scope.
- `normalizeSeed` mod-1e6 collision for congruent large seeds: negligible probability, seeds are meaningless visual
  values.

### Decisions

- **`Line2` → batched camera-billboarded ribbon** (A9/T004/T005): `Line2NodeMaterial`'s transparent path samples the
  opaque viewport texture the custom PostFX pipeline never exposes (WebGPU rejects the bind group — plan-23 as-built).
  The `FatLineLayer` ribbon is genuinely a batched fat-line (one mesh, variable width + additive glow, neuron-only
  endpoints) and survives the pipeline. **`VisualBodyKind` unchanged** — star/filament are `shader`, cell-star
  `primitive`; the ribbon body is carried by a `Mesh`, so no new kind was needed.
- **Boundary tooling extended** to admit the FSD `@x` cross-import (an `entities-x` element + `entities → entities-x`
  rule in each app's eslint boundaries) and to treat an `entities/<rendering-term>` slice as a visual path in the §1
  ubiquitous-language lint (rendering vocabulary native to the rendering layer, still forbidden in the mirrors).
- **On-device render pending** (like the rest of the RN scene): verify with `pnpm ios`; the mobile MVP instance-cap /
  post-FX reduction is confirmed via on-device profiling — the shared bodies and projection do not fork.
