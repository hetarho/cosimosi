## Acceptance Criteria (from plan/29.universe-clock)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 A goose migration creates `universe_state (user_id TEXT PRIMARY KEY, current_universe_time DATE NOT NULL, updated_at
TIMESTAMPTZ NOT NULL DEFAULT now())`; `pnpm db:migrate` applies cleanly and `pnpm db:down` reverses cleanly.
- [x] A2 `AdvanceClock(current, target)` returns `max(current, target)` and **never** returns a value earlier than `current`;
      there exists **no** operation (domain or SQL) that moves `current_universe_time` backward ([I10]).
- [x] A3 The diary-date monotonic constraint is a pure predicate: a diary dated **≥** the current clock launches (and the
      clock advances to that date); a diary dated **<** the clock is **saved but launches no star** ([T1]). (Enforcing it
      at launch is plan 30; this unit provides the rule.)
- [x] A4 Universe time is a **DATE** (day granularity) and is the model's single **read-time "now"** — a scalar elapsed-days
      reference for [F]/[C]/[L] derivations and **never** a position/coordinate ([T5][I7][I5]).
- [x] A5 A user with no launches has **no** `universe_state` row and reads a **nil** universe time; the row is created lazily
      on the first advance (upsert keyed by `user_id`), so the empty-universe read is unchanged from Epic A.
- [x] A6 The clock advance is a **single-row upsert** with `GREATEST(...)` (SQL-level monotonic guard mirroring `AdvanceClock`),
      so concurrent launches cannot rewind the clock or create duplicate rows.
- [x] A7 `pnpm gen:sql` generates the clock queries into `dbgen`; only `internal/memory/pg` imports `dbgen`, and no sqlc row
      type appears in the clock domain model or context behavior (ARCHITECTURE §2.4 #4).
- [x] A8 Every clock read/write is **scoped to the authenticated user** and `pnpm lint:persistence` passes ([U1], §4).
- [x] A9 Repository interfaces are **not** pre-declared here (they are consumer-owned by plan 30); `memory/pg` returns the
      concrete clock store + queries the use-case will bind to.

## Grounding

- Constitution ([I1]–[I11]): [00.overview](../plan/00.overview.md) §3 (_The constitution_). This job is bound by
  **[I10]** (time is monotonic / forward-only — `AdvanceClock = max` in the domain, `GREATEST` in SQL, no backward
  path by construction), **[I7]** (center = self; universe time is a scalar read-time "now", never a position axis —
  it has no path into layout), and it upholds **[I5]** by keeping _positions_ emergent while the clock is legitimately
  stored authoritative state (the invariant governs positions/constellations/nebula color, not the clock).
- Architecture (placement): [ARCHITECTURE.md](../ARCHITECTURE.md) BE §2.2 (the clock is core-domain behavior of the
  memory context — no new context), §2.3 (model in the context package; the `memory/pg` split already exists), §2.4
  (dependency rule — pure domain free of proto/sqlc/pgx; ports are consumer-owned, so plan 30 declares them), §2.6
  (`memory/pg` is the only sqlc/pgx seam), §4 (per-user isolation). BE-only scope — invoke `/be-architecture` for
  every file placed under `apps/api`; no FE/mobile surface.
- tech / policy / values touched: **values** — none: the plan introduces **no tuning scalars** (pure `max`/comparison
  logic; the forgetting/semanticize rates that read the clock own their values in Epics C–E; the temporal-link
  window/bonus stay in `synapse.*`, plan 18). **tech/** — on completion extend
  [tech/memory-persistence.md](../tech/memory-persistence.md) with the as-built clock rules (single per-user
  authoritative clock [T5]; stored state, not emergent [I5]; domain `max` mirrored by SQL `GREATEST` [I10]; universe
  time = the read-time "now" of all elapsed-universe-day derivations) — **not pre-written**. **policy/** — the
  domain rule lands in a new `policy/domain/universe-clock.md` at implement — **not pre-created**. **Depends on**

  seam), [05.data-persistence](../plan/05.data-persistence.md) (goose/sqlc conventions, `dbgen`, `lint:persistence`),
  [04.auth-session](../plan/04.auth-session.md) (`platform.UserScope`). **Consumed by**
  [30.time-advance-usecase](../plan/30.time-advance-usecase.md) / [job 41](41.time-advance-usecase.md) (calls
