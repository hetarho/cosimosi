- [x] A1 A star that has not been recalled in universe-time renders **dimmer**: the star-body brightness channel is fed
      the **real** read-time `EffectiveBrightness` (from `packages/memory-logic`, job 48), replacing Epic A's
      full-brightness placeholder, mapped within the existing `rendering.star_brightness_min`…`star_brightness_max` range
      ([V2][F1]).
- [x] A2 Brightness is **floored** — a fully-decayed star dims to `rendering.star_brightness_min`, **never to 0 /
      never removed** (the silent engram) ([F2][I1]); the star stays renderable at the floor.
- [x] A3 Filling the brightness channel requires **no new star body or shader graph** — the reserved uniform from
      plan 24 is bound to `EffectiveBrightness`; the change is at the projection layer, and the value recomputes on
      read-model / universe-time change, not per frame (§3.2/§3.3).
- [x] A4 A hovered/focused star's label shows its **current decay-stage text** (the current-memory text with words
      removed), not the whole text — the eroded memory reads as eroded before the panel opens ([F1][R8a]).
- [x] A5 The star-detail panel shows the **forgotten current-memory text for free** (the full current decay-stage
      text) plus a read-only **forgetting-degree** meta ("현재 망각 정도"); the panel shell, buttons, and 변천사/원본
      views remain job 46's ([D1][G1]).
- [x] A6 The decay-stage text is read from the `decay_stages`-backed **current-stage text** on the `GetUniverse` DTO
      (job 48's read extension); this unit **does not** compute the removal rule or index the stages — it renders the
      resolved current-stage text ([R8a][F9]).
- [x] A7 The word-loss is **not separately announced** — the eroded text is shown plainly, with no "this memory
      decayed" warning; the diarist discovers the loss by reading ([R8a]).
- [x] A8 After a recall (job 44) the star **brightens back to full** and its hover/panel text **fills back to whole**
      on the next `GetUniverse` read — a pure re-render of recovered read-time values; this unit issues **no** recall
      write and never resets brightness or mutates text itself ([F5][R2][I8]).
- [x] A9 Dimming and word-loss **move together** — a star reading a lower `EffectiveBrightness` also reads a deeper
      decay-stage text, because both derive from the same read-time decay in job 48 ([F1]).
- [x] A10 Forgetting (dimming + word-loss) is **independent of gist/semanticization** — a decay-stage text is never a
      gist-stage text, and this unit renders no z-rise / gist star ([F], §5.3.5); those are plans 40/42.
- [x] A11 Viewing the forgotten text (hover + panel) is **free** — no 별가루 gate; the recall-cost of a decayed star
      stays out of this unit ([G1]).
- [x] A12 This unit fills the **forgetting-dimming half** of the reserved acceleration slot ([V8], plan 31 / job 42):
      on a clock advance, stars that crossed a decay threshold dim as the acceleration plays; it does not re-own the
      choreography shell nor the gist/replay half.
- [x] A13 **All** user-visible copy (the forgetting-degree meta, any floor affordance) resolves through the i18n seam
      (no raw strings; `lint:raw-strings` passes); the decay-stage text is generated content, not an i18n message.
- [x] A14 **web + mobile parity:** the brightness channel is the shared TSL star body fed the same read-time value on
      both apps; the hover-label + panel-text sections share their read logic, with `*.native.tsx` siblings **only** for
      the label/panel primitives that differ (§3.5).
