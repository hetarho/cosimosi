## Acceptance Criteria (from plan/15.web-router-foundation)

- [x] A1 `apps/web` navigates via TanStack Router; `@tanstack/react-router` is imported **only** within
  `apps/web/src/app/routes/`, and `pnpm lint:fsd:layout` sees a segmented `app/routes` (no flat `app/` files).
- [x] A2 Routes form a **code-based, typed** route tree; the typed registry (`declare module … Register`) makes
  `Link` / `navigate` compile-time path-checked — an invalid `to=` **fails `pnpm typecheck`**.
- [x] A3 The hand-rolled path switch in `App.tsx` is removed; `/` renders `UniverseHomePage` and `/test` renders
  `TestPage` through the router, with `RouterProvider` composed **inside** the existing provider stack (providers
  outer, router inner) so every route sees observability / i18n / auth / cache.
- [x] A4 `/test` is registered only under the dev/diagnostics flag (`platform.diagnosticsSurface`, parity with mobile's
  `Diagnostics`) and is **not** reachable in a production build.
- [x] A5 An unknown path renders a **localized** not-found screen (i18n `m.*`, no raw strings), and route render errors
  surface through `ObservedErrorBoundary`.
- [x] A6 The app is mountable at an arbitrary route via injected **memory history** (no `window.location` coupling);
  existing `App` tests / storybook keep passing.
- [x] A7 `pages` / `features` navigate through a typed seam re-exported from `app/routes` — they do **not** import the
  router library name directly.
- [x] A8 `pnpm typecheck`, `pnpm lint` (incl. `lint:fsd:layout`, web boundary probe, `lint:raw-strings`),
  `pnpm test:web`, and `pnpm build:web` all pass; no mobile changes.
- [x] A9 No SSR / framework mode, no route loaders, and no product feature routes are added.
