# Job 18: background type registry  (change)

> Implementation work doc. Source spec: [changes/01.background-type-registry](../changes/archive/01.background-type-registry.md).
> /cosimosi:implement-job 18 built it via the two checklists below. Result reflected into the SSOT
> (plan 14 + tech/rendering.md); change doc archived.
>
> **Delta in one line:** stop the renderer hardcoding `scene.backgroundNode = nebulaBackgroundNode(skin)`. A background
> is now a **typed** thing (`{ type, props }`) resolved through a **registry**; the package is split into a **generic
> core** vs a concrete **`assets/`** module; a second type (`gradient`) proves it. All churn stayed inside
> `@cosimosi/3d-renderer` — the seam API and the consumers' skin fields (`bloom`/`camera`/`label`/`key`) were preserved,
> so the three consumers were zero-touch (verified by typecheck).
