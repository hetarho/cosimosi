## Acceptance Criteria (from plan/37.forgetting-decay)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 `effectiveElapsedDays(now, lastRecalled, created, offset)` returns universe-days from `lastRecalled` (or from
      `created` when `lastRecalled` is NULL), **plus** the signed `forgetting_offset_days`, **floored at `0`**; it is never
      negative and a never-recalled star still accrues elapsed days from creation ([F1][I10] + CC4).
- [x] A2 `EffectiveBrightness(effectiveElapsedDays, arousal, effectiveStrength)` satisfies **`= 1.0` at `elapsed = 0`**,
      is **monotone non-increasing** in `effectiveElapsedDays`, and stays within **`[forgetting.brightness_floor, 1]` тАФ
      never below the floor and never `0`** (silent engram; stored fact never mutated) ([F1][F2][I1][I5]).
- [x] A3 Higher `arousal` yields a **slower** brightness fade at the same elapsed days (`EffectiveBrightness`
      non-decreasing in `arousal`), governed by `forgetting.arousal_slow_coefficient`; **valence/mood are not inputs**
      ([F6][I3]).
- [x] A4 Higher `effectiveStrength` (connection/size) yields a **slower** fade at the same elapsed days, governed by
      `forgetting.connection_slow_coefficient` ([F7]).
- [x] A5 `DecayStage(effectiveElapsedDays, arousal, effectiveStrength)` is a **monotone-non-decreasing** step function
      of the same effective elapsed days (so stage and brightness move together), is `0` at `elapsed = 0`, and **floors at
      the maximum stage** = the derived length of `forgetting.stage_word_removal_ratios` тАФ there is **no stage past the
      last** ([F1][F2]).
- [x] A6 `decayStageText(currentText, stage, seed)` is **deterministic** given its inputs, removes the
      `forgetting.stage_word_removal_ratios[stage]` fraction of words (strictly increasing with stage) replacing each with
      the redaction token, and its removed-word set at stage _k+1_ is a **superset** of stage _k_'s ([F1]).
- [x] A7 `decayStageText` **preserves structure**: it never removes the first or last word of a sentence and prefers
      content words over function words, so even the **deepest stage is non-empty** and legible-as-fragments ([F2][F9]).
      (Exact ratios/POS rules are a `values.yaml`-tunable "to refine" per [F9].)
- [x] A8 Decay texts are produced by an **algorithm, not an LLM**, and are **independent of the semantic
      (`semantic_stages`) axis** тАФ forgetting removes words at random; semanticization compresses meaning ([F1], [C]
      independence).
- [x] A9 `ListUniverseEpisodicMemories` (and the `GetUniverse` DTO) **selects and returns `decay_stages`**, plus the
      anchor facts (`arousal`, `created_universe_time`, `last_recalled_universe_time`, and the fields for
      `EffectiveStrength`) so the client computes current stage/brightness from stored facts alone; the server does
      **not** pre-render a current decay text ([I5]). `forgetting_offset_days` (plan 32's column) is read once present,
      treated as `0` when absent.
- [x] A10 The Go (`internal/memory`) and TS (`packages/memory-logic`) implementations produce **identical outputs** for
      the shared golden-fixture table within a fixed tolerance, and both read the same generated `forgetting.*` constants
      (TSтЖФGo golden-parity, matching plan 18).
- [x] A11 All four functions are **pure**: deterministic, IO-free (no clock, DB, transport, SDK, or ambient
      randomness тАФ the word-removal PRNG is a seeded argument), operating only on their arguments and generated constants;
      the plan 18 `EffectiveBrightness` **stub is filled** here with its Epic-D dynamics, with no other read-time function
      changed.
- [x] A12 Forgetting is **read-time and cron-free** ([T4]): stage/brightness are pure functions of universe time;
      persisting a newly-reached stage's decay text is bound to plan 41's concrete advance hook, not a scheduler, and the
      system deletes nothing ([F3][I1]).

## Review

- [x] `/code-review` applied (no bugs) ┬╖ `/codex:review --background` (`-m gpt-5.5`) + a parallel Go/TS-parity finder
      agent. Findings merged (dedupe: both flagged the seed-parity bug):
  - **Applied тАФ seed int64 parity (High):** TS `decayStageText`/`seededRank` took `seed: number`, losing bits above
    `2^53` vs Go `uint32(seed)`. Widened to `number | bigint` normalized with `BigInt.asUintN(32, тАж)`; fixture now carries
    the seed as a string and pins a `9007199254740995` case.
  - **Applied тАФ sentence-anchor guard (Med):** `endsSentence` missed a terminator followed by a closing quote/bracket
    (`hello."`). Both runtimes now strip trailing closing punctuation before the terminator check; a fixture + unit case pin it.
  - **Applied тАФ elapsed-days date truncation (Med, latent):** Go `EffectiveElapsedDays` now truncates `now`/anchor to the
    UTC day (`utcDate`) so a caller passing a timestamp cannot desync from the date-only TS mirror.
  - **Rejected тАФ CJK-without-spaces never decays (Med):** the v1 word-removal is whitespace-tokenized by design ([F9]
    "to refine"); Korean (primary language) and Latin use spaces and decay correctly. Documented as a v1 limitation in
    `tech/forgetting-decay.md`.
  - **Rejected тАФ null-`now` asymmetry (Low):** the TS `now: string | null` guard (unborn universe тЖТ 0 elapsed) is an
    intentional client-side pre-launch behavior; Go is never called pre-launch (no clock row).
