## 인수 조건 (acceptance — 기획 관점, plan/51.background-skins에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [ ] A1 배경 조립이 `entities/appearance/ui`(별 `star-body`·나 `self-form`과 같은 층)에 있고, 위젯(`universe-canvas`)은 조립을 소유하지 않는다 — 선택 id로 entity 조립을 lookup해 렌더 lifecycle(머티리얼/uniform/`uTime`)만 운반한다.
- [ ] A2 위젯은 스킨 수와 무관하게 동작한다 — 효과별 `if/else`/`switch` 없이 registry/`for`로 N개를 처리하고, **새 배경 스킨 추가가 위젯 코드 수정 없이** 된다.
- [ ] A3 배경 스킨은 plan 50 툴킷의 이펙트 기법을 조립해 만들어진다(자체 매직넘버 재구현이 아님). 튜닝 수치는 entity 카탈로그의 **주석 달린 `params`** 에 있고, 한 스킨의 `params`를 바꿔도 다른 스킨에 영향이 없다.
- [ ] A4 배경 카탈로그는 `galaxy`(무료·`DEFAULT_BACKGROUND`)·`vortex`·`crystal`·`mandala`이고 확장 가능하다(추가 = 카탈로그 항목 + entity 조립 함수). 기존 7종·7effect는 제거됐다.
- [ ] A5 4종 모두 검은 우주 받침 위에 mood(`emotionSlots`·presence·감정색)를 입히고, 별의 mood 색·깊이를 침범하지 않으며(BackSide·depth off·`renderOrder=-11`·raycast none·draw call 1), presence=0이면 안전한 중립 딥스페이스로 보인다. reduced-motion 정지·arousal 게인이 유지된다.
- [ ] A6 미지/레거시 배경 id(기존 `vast` 등)는 `galaxy`로 폴백하고 렌더·편집 UI가 크래시하지 않는다.
- [ ] A7 유효 배경 id와 무료/가격이 카탈로그·values에서 파생돼 **새 배경 추가가 BE 코드 수정 없이** 된다(`theme` 문자열 id·proto·DB·item-id 소유권은 plan 44 그대로, 회귀 없음).
- [ ] A8 튜닝 `params`는 코드(entity)에 있고 `values.yaml`엔 셰이더 계수가 없다 — values는 `customization.free.background=galaxy` + 유료 3종 가격만(기존 배경 가격 제거).
- [ ] A9 FE `build`·`lint`·typecheck·`steiger` 통과, `AppearanceSwitcher`·랜딩/사인인/초대/방문·공유 우주 회귀 없음. 테스트(카탈로그/스키마 파생·미지 폴백·위젯 N-제네릭·스킨 조립).
