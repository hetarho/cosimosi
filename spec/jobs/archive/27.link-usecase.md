## Acceptance Criteria (from plan/21.link-usecase)
<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->
- [x] A1 After `PersistEncoded` launches a memory with *n* co-activated neurons, **all** `n·(n−1)/2` unordered neuron pairs of
  that memory have a synapse row (same-memory Hebbian wiring) [L1].
- [x] A2 When a launched memory reuses an existing neuron, **no extra edge** is created to represent the sharing; the two
  memories are connected purely through the shared neuron's `neuron_activations` membership, and `GetUniverse` returns
  that shared membership — the shared neuron *is* the link [L2].
- [x] A3 Two neurons co-activated across memories whose diary dates are within `synapse.temporal_window_days` receive the
  `synapse.temporal_bonus`; outside that window no temporal bonus is applied. The window is compared on the memories'
  dates, and no time value is written onto any neuron [L4][E6].
- [x] A4 `Link` creates or strengthens **no** synapse from embedding/semantic similarity, and reads no `embeddings` rows;
  the graph grows only from neuron reuse + Hebbian co-firing + the temporal bonus [L3].
- [x] A5 Every synapse `Link` writes is between two neurons; there is no memory↔memory edge, table, force, or attraction
  anywhere in the use-case, and inter-memory proximity remains emergent [I4][I6][L6][L7][E9].
- [x] A6 A first-time pair is initialised via plan 18's `InitialStrength` at the correct tier, with
  `synapse.initial_same_memory` > `synapse.initial_shared_neuron` > `synapse.initial_temporal` [L10]; the tier of a
  pair is determined by its strongest applicable signal.
- [x] A7 A repeat co-firing strengthens the stored base via plan 18's `Potentiate`, increasing in proportion to
  `(1 − current)` and never exceeding `synapse.strength_cap = 1.0`, converging smoothly toward the cap [L8][L9].
- [x] A8 Persistence uses `INSERT … ON CONFLICT (user_id, neuron_a_id, neuron_b_id) DO UPDATE`, storing the canonical
  `neuron_a_id < neuron_b_id`, incrementing `co_activation_count`, advancing `last_activated_universe_time`, and writing
  the stored **base** strength (not the read-time effective value); re-running `Link` for the same launch is idempotent
  (advances the same row, creates no duplicate) [L5][L8].
- [x] A9 Constellations are never stored: `Link` writes no coordinate, cluster, or memory↔memory relationship; clustering is
  computed by the force-sim (plan 19) from shared neurons + synapse strength at layout time [I5][L7] (ARCHITECTURE §2.9 #3).
- [x] A10 Every read and write is scoped to the authenticated `user_id`; `Link` never reads or writes another user's neurons,
  activations, or synapses [U1] (ARCHITECTURE §4).
- [x] A11 All strength values come from plan 18's pure functions (no strength arithmetic is re-implemented in `Link`), so the
  same inputs produce the same stored base under the client-side golden-parity mirror.
- [x] A12 `Link` performs no LLM/embedding call and no hard delete of any neuron or synapse; it only inserts or strengthens
  edges [I1].
