- [x] A1 Consolidation never publishes a risen `semantic_stage` or appends a `semanticized/system` provenance entry
      until that stage's generated text is non-empty. A missing ladder leaves the prior readable stage authoritative
      and records revision-bound pending work.
- [x] A2 Every semanticization job is tied to the memory representation revision it was created for. Delayed, retried,
      out-of-order, or superseded jobs cannot overwrite stages generated for newer `current_text`/seed state.
- [x] A3 Semanticization completion re-reads live source and atomically validates the current job lease plus
      representation revision, writes/merges the ladder, finalizes a pending stage/timer advance, appends exactly one
      non-empty event per newly materialized stage, and completes the job. Replaying completion is a no-op.
- [x] A4 A durable per-user processed-through marker (or equivalently strong interval ledger) makes synapse downscale
      exactly-once across duplicate or overlapping consolidation invocations. A rolled-back attempt remains retryable;
      `from == to` remains a total no-op.
- [x] A5 Future semanticized provenance is database-guarded against blank text and duplicate stage events. Legacy blank
      rows are handled without UPDATEing retained append-only history; all new revision/interval/event keys are
      authenticated-user scoped.
- [x] A6 Tests cover missing ladders, normal pregenerated ladders, delayed/reversed jobs, lease loss, duplicate
      completion, repeated/overlapping intervals, rollback, and cross-user negative controls.
- [x] A7 Existing semanticization, reconsolidation, forgetting-text, replay, and one-way gist-rise behavior remains
      intact. The Diary is never written, graph rows are never deleted, and no coordinate/read-time tick is persisted.
- [x] A8 Apart from the R005 correctness fixes, there is no unrelated behavior change and no regression of plans
      32, 40, or 41.

### As-built notes

- T001: the representation revision already existed (job 67's migration 00011); this job's migration is
  `00013_consolidation_convergence.sql` — pending-rise columns on `episodic_memories`
  (`pending_semantic_stage`, `pending_semantic_rise_at`), `universe_state.consolidated_through`,
  `memory_provenance.semantic_stage` + partial unique index + forward-only `NOT VALID` blank/stage guard.
- T002/T003: identity + expected-revision job rows also predate this job (job 67, `job_targets`); pending-rise
  metadata lives on the **memory row**, not the payload (payloads stay empty by the deletion-safe queue contract).
  The stale `RisenStage`/`CurrentStages` source fields were removed — the kept-stage merge moved into the
  completion transaction against LIVE row state.
- T004: `SaveJobSemanticStages` → `CompleteSemanticizeJob`: one pg transaction (graph advisory lock first →
  `LockSemanticizeCompletion` FOR UPDATE fence → live merge → `FinalizeSemanticizeCompletion` → provenance
  appends → fenced `CompleteJob`). The handler additionally refuses blank generated rungs (retryable).
- T005/T006: the timer anchor is consumed **eagerly** for deferred crossings (rise math counts from
  max(visible, pending) with the moved anchor); whitespace-only rungs count as missing; the watermark clamps
  duplicate/overlapping intervals to the unprocessed suffix and moves last in the transaction.

### Review record

- Inline adversarial review: applied mid-build (lock ordering advisory-first in the completion tx; repair
  horizon covers pending; replay set includes deferred rises).
- Codex (gpt-5.5, read-only, staged diff) — 3 findings:
  - **Applied:** whitespace-only pregenerated rung would abort the advance transaction against the blank-text
    guard → whitespace now counts as missing and defers (`stageText`/`missingStageText`/completion merge trim;
    regression test added).
  - **Rejected:** "reconsolidation should clear/fence the pending rise" — clearing would lose a crossing that
    already happened in universe-time forever (its units are consumed), violating one-way gist rise. Ladder
    texts have always derived from the live representation at generation time (plan 32's regen rule); the
    finalized rows are correctly dated at the crossing. Documented in plan 41.
  - **Rejected:** "duplicate same-revision jobs rewrite unrisen rungs by completion order" — same-revision
    generations are interchangeable by contract; visible stages are merge-protected, events are unique-indexed,
    and the cadence matches the pre-existing repair loop.
