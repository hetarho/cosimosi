## Acceptance Criteria (from plan/26.nebula-color-field)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 `entities/nebula` renders an ambient color field (TSL, WebGPU with automatic WebGL2 fallback) that **blends many
      per-star emotion colors at once** — colors coexist and bleed rather than collapsing to a single average color ([M4],
      [V4]).
- [x] A2 Each memory's local region reads **its own** mood color densely (strongest at the star, falling off with distance);
      away from any star the field is faint ([M4]).
- [x] A3 A memory's contribution **bleed radius / falloff extent is weighted by its `EffectiveStrength`** — a stronger star
      bleeds **wider** into the field; the mapping uses a `values.yaml` bleed-radius coefficient ([M4]).
- [x] A4 Every color the nebula draws comes **solely** from `moodColor(memory.mood)` via the single plan-17 palette seam; the
      nebula defines no color of its own, and there is no code path coloring the field from anything other than emotion
      ([M3], [M6]).
- [x] A5 The nebula reads emotion for **color only**; it reads no emotion axis into layout/connection and emits nothing into
      position, synapses, or strength ([I3], [M7]).
- [x] A6 The strength-weighting is a **visualization-output step**: it changes only rendered pixels and **never** affects the
      force-sim, clustering, synapse strength, or position ([M7]); disabling/altering it changes the picture, not the graph.
- [x] A7 The universe's global tone is **emergent** — there is no stored/modeled "universe color" value anywhere; the overall
      tone is only what the composited local contributions render as (mirroring how positions emerge from synapses) ([M4],
      [I5], §3.4).
- [x] A8 The weighting input is **`EffectiveStrength`** (base strength + recall accumulation), not raw `base_strength`, so the
      "emotions you re-read" mirror ([M5]) activates once recall accumulation (Epic C [R3]) lands **without rewiring** this
      unit; in Epic A the field blends by the arousal-set `base_strength` ([E8a]).
- [x] A9 An **honest notice** is shipped explaining that the universe color is _the emotions you re-read, not your emotional
      average_ ([M5], [PRD §1.4]); its copy is an `@cosimosi/i18n` message (`m.*`, `packages/i18n/messages`, `en` + `ko`) —
      **not** a hardcoded string in any component ([§3.1], ARCHITECTURE §5).
- [x] A10 Swapping the plan-17 palette re-colors the whole field with **no edit** to this unit, and a swapped palette cannot
      decouple color from emotion, override a memory's stored mood, or feed position/connection/strength ([I11], [P2]).
- [x] A11 The nebula (domain **emotion** color) coexists with the plan-14 skin **background** (non-domain ambiance) as a
      separate layer inside `widgets/universe-canvas`; the **skin never reads emotion** and the nebula never sets ambiance
      or overrides emotion — neither layer writes to the domain ([I3], [I11]).
- [x] A12 Mobile renders a **reduced-fidelity** field (fewer sampled contributors and/or lower-resolution field / cheaper
      falloff) from the **same** TSL source, preserving the meaning (many colors, strength-weighted bleed, emergent tone)
      (web = open beta · mobile = MVP).
- [x] A13 The field's shader graph is authored in **TSL** (no raw-GLSL fork) and mounts on the `@cosimosi/3d-renderer`
      substrate; per-frame contributor reads use `getSnapshot()`/coordinate buffers in `useFrame`, never React state
      (§3.3).

      domain import). Compositing is **additive/screen over the background, behind the bodies** — confirm the exact blend
      mode at implement and record it. Place it with the package's type-agnostic R3F layers (`src/layers/`, building on
      `shader-art/` blocks as needed) and export via `src/index.ts`. **Do not touch** the plan-14 _skin background_ type
      `assets/backgrounds/nebula.ts` — same word, different thing (non-domain ambiance).

      slice itself (§3.4):
  - `lib/contributors.ts` — pure contributor building: rendered memories → `(position, moodColor(memory.mood),

    read-time value the canvas provides — in Epic A it equals `base_strength`), never raw `base_strength` re-derived
    here — this is the Epic-C mirror seam.
  - `ui/NebulaField.tsx` — the R3F binding: mounts the T003 package layer, feeds the contributor buffer, reads per-frame
    positions from the canvas's coordinate buffers / `getSnapshot()` in `useFrame` — never React state.
  - `index.ts` — the slice's single public API.
    Every color goes through the plan-17 palette seam (`moodColor` via `packages/emotion`) — the slice computes **no**
