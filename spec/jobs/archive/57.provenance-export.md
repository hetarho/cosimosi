## Acceptance Criteria (from plan/46.provenance-export)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 `GetProvenance(episodic_memory_id)` returns the star's variant history **time-ordered by `universe_time`**
      with the synthesized **created/original baseline first**, each entry carrying `{kind, source, text,
universe_time}` where `kind ∈ {created, semanticized, reconsolidated}` and `source ∈ {original, system, user}`.
      ([R8a][D1])
- [x] A2 The **created/original baseline is synthesized at read** from the memory's creation facts — it is **not** a
      stored `memory_provenance` row and is **never** backfilled into Epic-A data; a star that has never been
      reconsolidated/semanticized still returns a one-entry history. ([CC5][R8a])
- [x] A3 `GetProvenance` emits **no separate distortion flag** — the history is a plain time-ordered list; distortion
      is found by reading it, not announced. ([R8a])
- [x] A4 `Export(format)` returns the user's **retained immutable original `Diary` records + their memories**
      serialized as **CSV or MD**, with the diary body **verbatim and un-mutated**; the mutable representation
      (`current_text`, stage texts) is not the export's authoritative content. User-hard-deleted release groups are
      absent because Epic H has removed them. ([W6][D4][I2])
- [x] A5 The **`Diary` never appears as a mutable entry in provenance** — the variant history records only the
      representation's evolution; the objective record is reached only via `Export` (and the reader [D2]). This unit
      issues **no `UPDATE`** to `diaries`. ([I2][R7])
- [x] A6 Both `GetProvenance` and `Export` are **read-only**: they advance no clock ([T3]), append no
      `memory_provenance` row, and spend **no stardust** (metadata/archive tier); they are GET-eligible /
      `NO_SIDE_EFFECTS`. ([T3][R1][G1])
- [x] A7 This unit **does not create, migrate, or write** `memory_provenance` — the table is owned by
      [32.reconsolidation-rules](../plan/32.reconsolidation-rules.md) and written by [33](../plan/33.recall-usecase.md)/[40](../plan/40.semanticization.md);
      this unit only reads and exports it. ([CC5])
- [x] A8 The export **honors the deletion/sealing exclusion** (soft-deleted memories `deleted_at IS NULL`, sealed
      neurons excluded) — respecting a "letting-go" in what is handed out — and performs no deletion itself; a
      saved-but-past-dated retained `Diary` (no star) is still exported. ([I1][X3][X4][T1])
- [x] A9 Handlers are **thin** (proto↔domain map + call); baseline synthesis, ordering, exclusion, and CSV/MD
      serialization live in the use-case/domain; no proto/sqlc type leaks into the use-case or pure domain; the
      `ProvenanceReader` and `ExportReader` are consumer-owned interfaces. (§2.4/§2.7/§2.9#4/#6/#7)
- [x] A10 Every provenance read and export query is **scoped to the authenticated user**; `GetProvenance` on another
      user's star returns `not found`, and `pnpm lint:persistence` passes. (§4/[U1])
