- [x] A6 `별 나누기`와 최종 `별 띄우기`는 행동 phase다. 각각 해당 버튼을 눌러야 진행되고 `다음`이 보이지 않는다.
- [x] A7 데모 튜토리얼에서 `별 띄우기` 완료 후 생성되는 별은 랜덤 위치/랜덤 별이 아니라 fixture에 정의된 고정 별이며, 카메라가 그 별을 안정적으로 프레이밍하고 spotlight가 그 별을 하이라이트한다.
- [x] A8 실계정 튜토리얼에서 `별 띄우기` 완료 후에는 실제 생성된 첫 별 ID를 추적해 그 별로 프레이밍/하이라이트한다. 서버 좌표를 새로 저장하지 않고 force-sim 결과 위에서 동작한다.
- [x] A9 튜토리얼 시작부터 첫 별 클릭/회상 설명 완료 전까지 마우스·터치·키보드 카메라 조작이 잠긴다. 작성 폼 버튼과 생성된 별 클릭은 잠기지 않는다.
- [x] A10 생성된 별 하이라이트 phase는 행동 phase이며, 사용자가 별을 눌러 회상 패널을 열어야 진행된다.
- [x] A11 회상 패널 설명은 job 47 완료 상태를 기준으로 상단 별 3D 포트레이트, 메타(`날짜 · 기분 · 강도 · 추상화`), 조각/원본 동선을 설명한다.
- [x] A12 회상 설명이 끝난 뒤 카메라 조작 잠금이 해제되고, 시점 전환/항해 실습 단계가 그 다음에 나온다.
- [x] A13 망원경 `일기`가 이미 선택된 상태에서는 `일기` 탭 버튼이 아니라 일기 패널 전체를 정보 하이라이트하고 `다음`으로 진행한다.
- [x] A14 `별` 탭은 행동 하이라이트이며 사용자가 눌러 별 탭으로 전환해야 진행된다. 전환 후 별 목록/잠든 별 안내는 정보 하이라이트로 설명한다.
- [x] A15 행동 phase에는 `다음` 버튼이 없고, 정보 phase에만 `다음` 버튼이 있다. 이 구분은 머신 상태(`snapshot.can('NEXT')` 또는 동등 구조)로 보장된다.
