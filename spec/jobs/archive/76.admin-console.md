## Acceptance Criteria (from plan/58.admin-console)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 A NEW `internal/admin` context exposes `admin.v1.AdminService`; it **imports no** `memory`/`twinkle`/`account`/`ai` internals — all cross-context access is through **consumer-owned ports** wired at `cmd/api`, and only `admin/pg` imports `dbgen` (§2.2/§2.4, CC8).
- [x] A2 An **admin-authorization interceptor** runs after plan 04's auth interceptor and rejects any non-admin caller of any `admin.v1` method with the canonical **permission-denied** error (distinct from _unauthenticated_); an anonymous caller is rejected as unauthenticated by plan 04.
- [x] A3 `IsAdmin` = the caller's user id/verified email ∈ `ADMIN_USER_IDS` (env seed, parsed at the composition root) **or** a row in `admin_users`. `GrantAdmin`/`RevokeAdmin` add/remove `admin_users` rows; a `RevokeAdmin` against a **seed** admin is **refused** (seed admins are undemotable). `GetAdminSelf` returns the caller's admin status.
- [x] A4 `SetAIConfig` writes an `ai_provider_config` row (per capability) with the API key **encrypted at rest**; it **rejects** an unknown or unimplemented provider slot (reusing plan-28 registry semantics) and an embedding model that cannot honor `ai.embedding_dim`. `GetAIConfig` returns provider/model/baseURL + `keySet` (+ masked hint) and **never** returns a plaintext key.
- [x] A5 The AI factory resolves **effective config = DB row → env → keyless mock**, and a `SetAIConfig` change takes effect **without a redeploy** (the vendor client rebuilds on config change); the metering, error taxonomy, port adapters, and mock fallback from [28](28.ai-provider-abstraction.md)/[22](22.ai-worker-pipeline.md) are unchanged.
- [x] A6 `ListUsers` returns **account metadata only** — id, email, signup date, admin status, stardust balance, and non-content counts (# diaries/# stars) — paginated (`admin.user_list_page_size`) and searchable; **no** admin RPC returns any diary text, emotion, position, connection, or memory content, and the `MemoryStatsPort` return type has no content field ([I2]).
- [x] A7 `GrantStardust(target, amount, note)` credits the target's **additional** balance via `twinkle.Earn` with the new `admin_grant` reason, is **capped** by `twinkle.admin_grant_max`, is **idempotent** (dedup by grant id), writes an `admin_stardust_grants` audit row + an `admin_audit_log` entry in the same transaction, and **deletes/prices/spends nothing** ([I1][I3]). `ListStardustGrants` reads the grants back.
- [x] A8 `GetAIUsage` returns today's per-capability call counts against `ai.daily_call_cap`/`ai.per_call_token_cap` from the plan-28 meter snapshot (read-only), with its in-process/in-memory scope honestly reflected; `GetJobHealth` returns aggregate `jobs`-table status/backoff/dead-letter counts (read-only, non-mutating).
- [x] A9 Every admin mutation (`GrantAdmin`/`RevokeAdmin`/`SetAIConfig`/`GrantStardust`) writes an append-only `admin_audit_log` row; the log never records a plaintext API key, and is never `UPDATE`d/`DELETE`d by the system ([I1]).
- [x] A10 A **web-only** `/admin` route exists (plan-15 registry) behind an **admin gate** (authenticated **and** admin); a non-admin cannot reach it; the affordance shows only for admins. **No mobile admin surface exists** and the parity rule is documented as waived.
- [x] A11 The `pages/admin` screen **composes** `features/admin-ai-config`/`admin-users`/`admin-usage`/`admin-jobs`, holds no domain logic, and all user-visible copy resolves through the i18n seam (`lint:raw-strings` passes) (§3.1).
- [x] A12 Goose migrations create `admin_users`, `ai_provider_config`, `admin_stardust_grants`, and `admin_audit_log`; `pnpm db:migrate` applies cleanly and `pnpm db:down` reverses cleanly; the service-global tables (`ai_provider_config`) are correctly excluded from the per-user `lint:persistence` gate (or scoped where they carry a user id), and it passes.
