## As-built notes (decided at implement)

- **Premise correction — `semantic_stage` joins the `GetUniverse` DTO (one field, no new RPC).**
  Plan 42 (and plan 40's read note) presumed "`semantic_stage` is already on the GetUniverse DTO";
  as-built it never was — the server facts carry it, but `EpisodicMemoryDto` dropped it at the
  wire. Rendering "N gist bodies per risen stage" is impossible without it, so the field is added
  (`semantic_stage = 13`), realizing the premise every Epic-E plan was written against rather than
  expanding the contract: still no new RPC, no coordinate field ([I5] — z stays client-derived via
  job 51's `gistCoordinate`). The A12 tick reads "adds no proto" in that corrected sense.
- **`rendering.gist_star_z_stage_step` is NOT added.** Job 51's golden-parity `gistCoordinate`
  already owns the stage→z map (a derived linear spread across the `force_sim.neocortex_z_*`
  band); a second step scalar would be config the coordinate fn never reads — drift by
  construction. Confirmed at implement as unnecessary.
- **The neutral rise is appearance-driven on the shared layer.** A stage can only newly appear
  after an advance (consolidation is the sole stage writer), so the rise plays exactly on the
  advance's read landing: new `(memory, stage)` instances ease one-way from the memory's
  hippocampal z into the band ([I10]); the first non-empty projection seeds silently (no
  page-load mass rise) and an empty interval adds no instance, so nothing plays (A8). The
  per-interval stage-rise events are emitted on `GistStarLayer.onStageRise` — the booked [V8]
  slot the later replay choreography consumes (A9). `GIST_RISE_DURATION_SECONDS` stays a
  code-level layer constant (T001's own condition).
- **Positions ride a derived-coordinate mapper.** `InstancedNodeLayer` gained an optional
  `getInstancePosition` (per-frame, allocation-free) so projection layers can copy a buffer
  slot's x, y with a derived z — the gist layer supplies it; the default contiguous-slot path is
  untouched. `COORDINATE_STRIDE` is now exported by the renderer as the buffer contract's owner.
- **The gap depth cue is `BandFog`** (`packages/3d-renderer/src/layers/BandFog.tsx`): an
  additive, depth-write-free, raycast-invisible haze slab across z 10–15, intensity from
  `rendering.gist_rise_layer_fog` — an affordance, never a wall, and never a click shield.
- **The gist selection seam carries the stage end-to-end.** `star-detail.machine`'s reserved
  recognizer became `resolveGist(nodeId) → {episodicMemoryId, stage}`; the panel injects the
  layer's `parseGistNodeId`, `onGistSelected(memoryId, stage)` flows to the page seam, and a
  gist pick sends SELECT only (no camera glide — a gist body is not a sim node).

**Applied (8):** the first-ever gist rise no longer pops in-band — the rise-seeding now gates on the
episodic store hydrating, not the gist projection being non-empty (a universe with memories but no
risen gists is loaded, so its first stage-1 rise animates); the rise captures its starting
hippocampal z at stamp time (a mid-rise force-sim re-layout can no longer drop the body — one-way
[I10]); the projection→rise diff + `onStageRise` moved from `useMemo` to a post-commit `useEffect`

top/bottom faces read invisible and whose sides read as radius-edge walls) to a stack of additive
horizontal glow discs across the gap — the haze reads from any view angle, never a wall; settled
bodies skip the per-frame ease math (a frozen `SETTLED` sentinel); the `DetailPanel` gist ternaries
collapsed to one `selection.kind === 'gist' ? selection : null` (dropping the fake stage-0
sentinel), web + mobile; the `InstancedNodeLayer` derived-vs-slot branches share one
scale+setPosition+setMatrixAt tail; comment narration cut (`built later`/`until then`/`plan-31
precedent; promote only if`/`plan 34/35`/`plan 40` → current-state).

**Rejected (with reason):** `key={instanceCount}` remount-on-count-change — real but **pre-existing
shared-infra behavior** (StarLayer already remounts when a launch grows the graph); a grow-only
capacity fix must pad every layer's channel arrays to capacity and get WebGPU instanced-buffer
sizing exactly right, so it belongs in its own rendering-perf refactor, not this feature job.
Re-select-same-gist is a no-op — the ViewSemantic viewer that consumes the recorded target is a
separate unit (plan 34/45); its re-trigger mechanism lands with that surface, not a speculative
counter here. GistStarLayer-vs-StarLayer tint-packing duplication — marginal at two call sites;
extracting a packer now would over-fit before a third instanced layer exists. Codex ran as a
background CLI review (gpt-5.5, 5 findings); three of the six `/code-review` finder angles died on a
transient TLS/proxy error mid-run — their partial output plus codex + the surviving
efficiency/conventions/reuse angles gave full coverage of the GistStarLayer hotspots.
