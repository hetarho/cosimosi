- [ ] A1 `frontend/src` non-generated 파일 기준 `as never`가 렌더/TSL 소비 파일에 직접 남지 않는다. 필요한 캐스트는 이름 있는 TSL helper 모듈 한 곳에 격리되고, 그 helper가 왜 필요한지 짧게 설명한다.

- [ ] A3 R3F/WebGPU renderer 관련 `as unknown as GLProps`/`WebGPURenderer`/controls shape 반복은 공용 helper 또는 hook으로 정리되어, `UniverseCanvas.tsx`, `UniverseOverlay.tsx`, `CosmosScene.tsx`, `BloomPass.tsx`, `controllers/navigation.tsx`, `controllers/flights.tsx`에서 같은 타입 assertion 문장이 반복되지 않는다.

- [ ] A5 `as const`와 명시된 테스트/mock 예외를 제외하고, 새로 추가되는 type assertion은 helper/guard 내부 또는 경계 모듈에만 존재한다. job은 `rg " as " frontend/src -g "*.ts" -g "*.tsx" -g "!*.gen.ts"` 결과를 검토해 예외 목록을 남긴다.
- [ ] A6 주석 정리는 최소한 다음 렌더링 hotspot을 수동 검토한다: `StarField.tsx`, `controllers/navigation.tsx`, `controllers/flights.tsx`, `SynapseFilaments.tsx`, `UniverseCanvas.tsx`, `UniverseNebula.tsx`, `star-body.ts`, `fluid-material.ts`.
- [ ] A7 주석 정리 후에도 회귀 방지에 필요한 설명은 유지된다: frozen-time `uTime`, BackSide/depth/renderOrder/raycast 없음, R3F/WebGPU renderer 제약, FSD import 경계, 보안/세션/쓰기 게이트, 도메인 수식의 출처.
- [ ] A8 change/spec 번호나 과거 수치 변경 내역을 설명하는 장문 주석은 코드에서 제거하거나 한 문장 이하의 현재형 의도 설명으로 축약한다. 필요한 이력은 spec/change/job 문서에 남긴다.
- [ ] A9 기능 동작과 시각 룩은 변경하지 않는다. 특히 우주 렌더링, 별/시냅스 모양, 항해/카메라, demo/tour, settings normalization의 사용자-facing 결과가 기존과 같다.
- [ ] A10 검증: `pnpm --filter ./frontend lint`, `pnpm --filter ./frontend build`, 관련 테스트(`appearance`, `synapse`, `demo flag`, `navigation`/`tour` 중 변경 파일에 닿는 것)를 실행한다. 타입 helper 변경으로 TSL 렌더가 깨지지 않는지 최소 smoke 확인을 남긴다.
