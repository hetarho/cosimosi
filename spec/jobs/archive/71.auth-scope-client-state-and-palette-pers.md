## Acceptance Criteria (from code-review/05.plans-35-53-runtime-and-architecture-aud)

<!-- R009 + R010 + R014. These are corrective refactors: isolate session state and preserve the
     intended auth/palette behavior while closing cross-account and first-frame gaps. -->

- [x] A1 **A scope change is an isolation boundary, not only a Query-cache event.** On
      `user A -> anonymous -> user B` (and direct A -> B in a test facade), every user-owned
      Zustand mirror, draft, target, deferred action channel, palette mirror, and Query entry is
      cleared before the new authenticated subtree can commit. No route (`/`, `/diary`, or
      settings) can render A's text, balance, release groups, selection, or pending action for B.
      A pending consent promise is settled/cancelled rather than orphaned. (R009.)
- [x] A2 **Palette writes are bound to the initiating session epoch.** Serialization is scoped to
      one authenticated user: a queued A write is cancelled before dispatch after the epoch
      changes, no A intent can be sent with B's live token, and a late A completion cannot mutate
      B's palette/store. Rapid writes within one epoch still converge on the last intent. (R010.)
- [x] A3 **Optimistic rollback returns to server-confirmed truth.** Palette persistence tracks the
      last canonical server-confirmed id separately from optimistic intent. Two or more failed
      rapid selections roll back to that confirmed id, not to an earlier unsaved selection;
      unknown ids resolve to and store the same canonical fallback id that is rendered. (R010.)
- [x] A4 **Palette-dependent children do not commit with the wrong palette.** An authenticated
      boot shows a neutral/loading shell until the stored preference (or the defined default on
      failure/unset/unknown) has been applied. The universe's first committed frame uses that
      palette, while a live successful change still recolors without `GetUniverse` refetch.
      (R010; plan 51 AC6-AC7.)
- [x] A5 **Desaturated colors have neutral hue evidence.** The axis validator returns zero warmth
      for grayscale and saturation-attenuated warmth for near-gray colors; grayscale/near-gray
      negative-valence entries do not produce a fabricated red-hue warning, saturated inversions
      remain warned, and every shipped palette remains warning-free. (R014.)
- [x] A6 **Both app shells prove account isolation.** Web and mobile integration tests exercise
      A -> sign-out -> B over the universe, diary, settings, balance/release mirrors, and pending
      target/draft channels, including a delayed palette request and the first palette commit.
- [x] A7 **No unrelated behavior change / no regression.** Auth bootstrap/gating and sign-out
      semantics, Query cache policy, plan 51 palette selection/fallback/live recolor, plan 52
      settings composition, and plan 53 route destinations remain intact. No RPC or persistence
      contract is changed by this client-lifecycle job.

- [x] T003 [P] Add web and mobile `app/model` reset aggregators for the still app-owned draft /
      interaction stores. Each aggregator calls the shared universe reset, clears its platform's
      local stores, cancels pending time-sync consent, and resets the active/persisted palette
      mirror to the canonical default. Keep the two inventories parity-tested until Job 72 moves
      their pure duplicates into packages.
- [x] T004 Add a session-scope boundary immediately below auth in each provider stack. When the
      stable scope key changes, withhold authenticated children, clear **all** queries (including
      an injected test QueryClient), run the reset aggregator, then mount the new scope. Preserve
      owned-client cleanup on provider unmount and avoid render-phase external-store mutations.
- [x] T005 Replace the module-global palette `persistQueue` with a session-scoped coordinator /
      operation epoch. Capture the initiating user/epoch, check it immediately before dispatch,
      invalidate it on scope reset, ignore stale responses, and keep serialization only among
      operations in the same epoch. Add controllable deferred-transport tests for queued and
      already-settling A operations during A -> B.
- [x] T006 Track canonical confirmed palette state independently from optimistic display state.
      Resolve `{ id, palette }` once through the registry for boot, server echoes, fallback, and
      rollback; update the confirmed id only after a same-epoch success. Cover two consecutive
      failures, superseded successes/failures, and unknown server ids.
- [x] T007 Rework both palette bootstrap hosts into a commit gate: apply the authenticated stored
      preference or deterministic fallback before releasing palette-dependent children. Provide
      the existing neutral loading treatment while pending and prove that the first universe

- [x] T008 [P] Change `checkPaletteAxisConsistency` to attenuate hue warmth by HSL saturation

- [x] T009 Add web and mobile shell regressions for A -> anonymous -> B across home/diary/settings
