## 인수 조건 (acceptance — 기획 관점, changes/13.demo-tour-state-machine에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [x] A1 투어 step·phase 전이가 `widgets/demo-tour/model/tour.machine.ts`의 XState v5 순수 머신으로 표현되고, 머신 정의는 React/DOM/three를 import하지 않는다(헌법4).
- [x] A2 행동 phase 상태는 `NEXT`를 처리하지 않아 `snapshot.can({type:'NEXT'})===false`이고 UI가 그 값으로 `다음`을 숨긴다. 정보 phase는 `can('NEXT')===true`라 `다음`이 보인다(point 1을 구조로 고정).
- [x] A3 DOM 관찰 await는 머신이 받는 이산 이벤트(`UI_HIDDEN`/`UI_SHOWN`·`PERSONA_OPENED`·`PERSONA_SELECTED`·`TIME_OPENED`·`TIME_MOVED`·`EXPLORER_OPENED` 등)로만 진행하고, 변화 기준(persona/clock)은 진입 baseline을 context에 담아 가드로 비교한다.
- [x] A4 시점 전환 항해 실습은 머신의 invoked `fromCallback` 액터가 rAF로 내부 샘플링 후 임계 도달 시 단 한 번의 이산 이벤트(`PRACTICE_MET`)를 보낸다 — 매 프레임 React state도, 매 프레임 머신 이벤트도 없다(헌법4/60fps). 항해 조작·물리·제스처는 불변(관찰만).
- [x] A5 항해 실습 phase는 상태 태그(`nav-practice`)를 노출하고, UI가 그 태그로 딤(거의 투명)·코치 카드(반투명·상단 중앙)를 적용한다(point 3을 머신 메타로 구동).
- [x] A6 메뉴(햄버거) 단계는 투어 step 목록·progress dots에 없다(point 2).
- [x] A7 카메라 모드 구동·표면 open/close·UI 숨김 복구가 머신의 액션/`emit`/노출 상태에서 파생돼 호출되며, 컴포넌트의 산발적 진행-동기화 `useEffect`가 제거된다.
- [x] A8 새로고침 시 같은 데모 세션이면 저장된 step에서 이어지고(머신이 `getTutorialStep()`로 hydrate), step 전환 시 `setTutorialStep()`로 저장된다. `다시 보기`=첫 step 재시작, `exitDemoMode()`=투어 flow·step·머신 상태 정리(plan 48 계약 불변).
- [x] A9 `DemoGuidedTour`는 스냅샷 파생값으로 렌더하고 입력을 이벤트로 보내기만 하며, "어느 phase로·언제 진행"하는 결정 로직을 갖지 않는다. 머신 context에 투어 데이터를 복제하지 않는다.
- [x] A10 `tour.machine.test.ts`가 `createActor`+`send`+`getSnapshot`만으로(React 없이) 진행 시퀀스와 게이팅을 단언한다: 행동 phase에서 `NEXT` 무시, 정보 phase에서 `NEXT` 진행, 각 step의 마지막 phase에서만 다음 step, 행동 이벤트 수신 시 진행, `EXIT`/마지막 단계 종료가 `free`로 수렴.
- [x] A11 plan 47 자유모드 진입·페르소나 전환·시간 이동·망원경·새 별 생성·일반 실계정 우주에 회귀가 없고, plan 48의 기존 수용 기준(안내 순서·하이라이트·crash-free 중앙 fallback·접근성 `Tab`/`Enter`/`Esc`·reduced-motion·FE-only·3D 씬 안 `<Html>` 금지)이 그대로 성립한다.
- [x] A12 build·lint·typecheck·steiger·vitest를 통과한다.

- [x] T001 `widgets/demo-tour/model/tour.machine.ts` 신규(XState v5, 순수 — React/DOM/three import 금지, 헌법4). `TOUR_STEPS`를 데이터로 유지하고 `(stepIndex, phaseIndex)` 커서를 context로 둔다. phase **종류별 상태**(`info` / `domAction` / `navPractice`)로 토폴로지를 잡고, 매 진행 후 `always` 전이가 새 phase의 종류로 라우팅한다(현재 phase 메타가 `await`/`mode`로 종류 결정). 이벤트: `NEXT`(가드 = 정보 phase일 때만 전이)·`PREV`·`EXIT` + 관찰 이벤트(`UI_HIDDEN`/`UI_SHOWN`/`PERSONA_OPENED`/`PERSONA_SELECTED`/`TIME_OPENED`/`TIME_MOVED`/`EXPLORER_OPENED`) + `PRACTICE_MET`. `xstate-guide` 템플릿·events-not-setters 준수.
- [x] T002 게이팅(point 1)을 구조로: 정보 phase 상태만 `NEXT` 전이를 가진다 → `snapshot.can({type:'NEXT'})`가 행동 phase에서 `false`. "각 step의 마지막 phase에서만 다음 step, 그 외에는 phase 진행"을 커서 전이로 표현(현 `goNext`/`goPrev` 분기 대체).
- [x] T003 변화 기준 await: `domAction` 진입 시 baseline(persona/clockDay)을 context에 담고, `PERSONA_SELECTED(id)`/`TIME_MOVED`가 baseline과 다를 때만 진행하는 가드. 그 외 관찰 이벤트(`*_OPENED`·`UI_*`)는 해당 phase의 `await`와 일치할 때만 진행.
- [x] T004 `navPractice` 상태에 `invoke` `fromCallback` 액터 `navSampler`(src·input만 선언, impl은 `provide()`로 UI가 주입). 액터는 rAF로 `navigation-input` 누적 카운터·항해 FSM heading 모드를 **내부** 샘플링, 진입 baseline 대비 임계(`VALUES.demoTour.*`) 도달 시 `PRACTICE_MET` **1회** sendBack. phase 재진입마다 invoke 재시작(새 baseline). 매 프레임 이벤트/React state 금지(헌법4/60fps).
- [x] T005 `navPractice` 상태에 태그 `nav-practice`(point 3). 머신 액션 선언(impl은 `provide()`): `driveMode`(현재 phase.mode가 heading과 다르면 `navigationActor.send(TOGGLE_MODE)`), `openSurface`(`telescope-star` step에서 시트 열기), `closeSurfaces`, `restoreUi`(uiHidden=false), step 진입 시 `view` 외 segment면 recall→nebula 정리 — 현 HomePage `useEffect`(544–563) + `handleTourPhaseMode`(487–492) 로직을 머신 액션으로 이전.
- [x] T006 지속성·생명주기: 모듈 싱글턴 `tourActor`(navigationActor·focusActor와 동형). 진입 시 `getTutorialStep()`로 시작 step hydrate, step 전환 구독에서 `setTutorialStep()` 저장(phase 미저장 — 현행 계약). `restartTutorial()`=첫 step, `exitDemoMode()`=stop/reset 연동.
- [x] T007 파생 selector(컴포넌트 밖): `selectStepIndex`·`selectTotal`·`selectPhaseView`(target/body/title 메타)·`selectCanNext`(=can NEXT)·`selectIsNavPractice`(=hasTag)·`selectIsLast`. `@xstate/react useSelector`로 구독.
