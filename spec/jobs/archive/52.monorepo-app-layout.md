### 구조 이동 (먼저 — 나머지 경로 수정의 기준)
- [x] T001 `apps/`·`packages/` 생성 후 tracked 앱을 `git mv`로 이동: `frontend → apps/web`, `backend → apps/api`, `blog → apps/blog`. `proto/`는 **루트 유지**(이동하지 않음). `node_modules`·`dist`·`tmp`·로컬 로그는 tracked가 아니므로 자동 제외 — 생성 산출물(`*/internal/gen`, `src/shared/api/gen`)은 tracked면 함께 이동하고, 아니어도 T005의 `pnpm gen`이 새 경로에 재생성한다. Go module path(`github.com/cosimosi/backend`)는 디렉터리와 무관하므로 import rewrite 불필요(A11).

### 워크스페이스 + 패키지 정체성
- [x] T002 `pnpm-workspace.yaml`: `frontend`·`blog` → `apps/web`·`apps/blog` + `packages/*` 추가. `apps/api`는 Go module이라 pnpm workspace 패키지로 넣지 않는다.
- [x] T003 [P] package name을 앱 정체성으로: `apps/web/package.json` `frontend`→`@cosimosi/web`, `apps/blog/package.json` `blog`→`@cosimosi/blog`.

### 루트 scripts (단일 진입점 유지)
- [x] T004 루트 `package.json` scripts를 새 selector로: `dev`·`dev:web`의 `--filter ./frontend`→`@cosimosi/web`, `dev:blog`의 `--filter ./blog`→`@cosimosi/blog`, `build:web`·`build:blog` 동일, `build:api`의 `cd backend`→`cd apps/api`. `dev:api`(docker compose) 커맨드는 그대로 — compose 쪽 경로는 T008에서 갱신.

### 루트 계약 + codegen 경로
- [x] T005 (gen) `git mv backend/buf.gen.yaml proto/buf.gen.yaml`. 출력 경로만 새 구조로: Go `backend/internal/gen`→`apps/api/internal/gen`, TS `frontend/src/shared/api/gen`→`apps/web/src/shared/api/gen`. `scripts/gen.mjs`의 template path `backend/buf.gen.yaml`→`proto/buf.gen.yaml`·sqlc mount `backend`→`apps/api`·skip 안내 경로, `scripts/lib.mjs`의 `hasBufConfig`(`backend/buf.gen.yaml`→`proto/buf.gen.yaml`)·`hasDbSchema`(`backend/...`→`apps/api/internal/db/schema.sql`), `buf.yaml` 주석의 template 경로를 갱신. `buf.yaml`의 `modules: [{path: proto}]`는 proto가 루트에 남으므로 불변. `pnpm gen`으로 동일 산출 재현 확인.
- [x] T006 [P] (gen:values) `scripts/gen-values.mjs`: tsOut `frontend/src/shared/config`→`apps/web/src/shared/config`, goOut `backend/internal/values`→`apps/api/internal/values`. `spec/values.yaml`은 루트 유지·수치 무변. `pnpm gen:values`로 새 경로 생성 확인.
- [x] T007 [P] `scripts/db.mjs` migrations mount `backend/internal/db/migrations`→`apps/api/internal/db/migrations`. `scripts/setup.mjs`의 잔여 `frontend`/`backend` 경로 점검·갱신.

### Docker / Compose
- [x] T008 `docker-compose.yml` backend 서비스: `context: ./backend`→`./apps/api`, source volume `./backend:/app`→`./apps/api:/app`. go-mod/tmp 볼륨(`backend_go_mod`·`backend_tmp`)은 의미 유지(이름 갱신은 선택). 외부 포트·서비스 의미 불변(A6). `docker-compose.prod.yml`은 ghcr 이미지 참조뿐이라 빌드 컨텍스트 없음 — 잔여 `backend` 경로 참조만 점검.
