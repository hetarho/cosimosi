## Review
- [x] `/code-review` applied — 3 finder angles + Codex (`gpt-5.5`, read-only). Findings triaged below.

**Applied (review-driven hardening, beyond the original finding set):**
- The persistence guard's SQL parsing was hardened past the R002/R012 fix: a single-pass `stripSqlNoise` lexer now
  masks comments, single-quoted string literals, and dollar-quoted bodies before paren-scanning/splitting — closing
  fail-open holes that a `(` / `)` / `;` / `--` / `$$` inside a literal would otherwise open (raised by both finders
  and Codex). The `CREATE TABLE` header tolerates table modifiers (`UNLOGGED`/`TEMP`/…), and an unbalanced body is
  flagged (fail-closed) rather than silently skipped. The guard self-test was broadened to cover these cases plus the
  `findPersistenceViolations` orchestrator contract.
- `mergeDefaultOptions` keeps merging `base.mutations` (faithful — no contract change) while still dropping the
  redundant `...overrides` spread, after three reviewers flagged the dropped-base-mutations divergence.
- `react.ts` lifecycle comment reworded to the timeless "why" (dropped the ambiguous "Matches ObservabilityProvider"
  cross-reference); `fake-adapter` comment trimmed of the "Tests cover both branches" tail.

**Decision (A6 / R001):** adopted the **create-once `useRef`** pattern (mirroring the package `ObservabilityProvider`)
rather than create+dispose-on-unmount. Disposing on unmount would diverge from the established app-root-singleton model
and reintroduce review 01's R013 StrictMode dead-facade hazard. The orphan — the actual defect — is eliminated because
a ref-guarded create runs exactly once.

**Rejected / deferred (with reason):**
- Codex: `user_id`-token presence ≠ a real column / WHERE-predicate (e.g. `REFERENCES users(user_id)`,
  `SELECT user_id …`). A pre-existing heuristic limit, not introduced here; tightening needs real SQL semantics and
  risks false-positives. The guard is a lint backstop; per-user isolation hardens further when the first product table
  lands (review 02 R016) and with RLS later. Recorded, not fixed.
- Codex: a `facade` prop toggling defined→undefined would now create a default facade instead of throwing. The prop is
  set once per provider instance (never toggles); the new behavior is more robust and matches `ObservabilityProvider`.
  Refuted.
- A native `Toast` render-test (T011) and a StrictMode render-test for `AuthProvider` (T009) are **verified by
  inspection + parity** with their tested web/observability counterparts: `packages/ui` vitest has no React-Native
  environment and `packages/auth` has no jsdom/@testing-library — adding that harness is out of this job's scope.
- `gen-values` hand-codes the single gofmt blank-line rule instead of piping through `gofmt`: deliberate, to keep
  `gen:values` runnable without Docker/Go (gofmt is Docker-only here). Verified gofmt-clean with the real tool.
