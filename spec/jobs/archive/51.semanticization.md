## Acceptance Criteria (from plan/40.semanticization)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 `Semanticize(currentStage, unitsElapsed)` is **monotone non-decreasing** (`≥ currentStage` for
      `unitsElapsed ≥ 0`) and **clamped** to `SEMANTIC_MAX_STAGE = 4` (a derived ladder length = the count of pregenerated
      gist texts, not a tuned value); it is deterministic and IO-free ([C7], z-axis one-way).
- [x] A2 A single advance may cross **multiple** stages (`unitsElapsed > 1`): `Semanticize` adds all whole units and
      clamps, and the consolidation use-case appends **one provenance row per crossed stage** so 변천사 stays continuous
      across a large clock jump (CC5, [R8a]).
- [x] A3 The gist-timer computes `unitsElapsed = floor(effectiveDays / semantic.gist_units_per_stage)` where
      `effectiveDays = max(0, now − semanticize_timer_reset_at) × timerModulation(arousal, connectionStrength)`, measured
      in **universe-days** ([C6a][I10]); at the reset anchor (`now = semanticize_timer_reset_at`) `unitsElapsed = 0` (no
      stage rises) ([F5]).
- [x] A4 Higher **arousal** and stronger **connection** _slow_ the gist-timer (`timerModulation ∈ (0,1]`), **reusing
      the forgetting modulation coefficients** (plan 37 / job 48, [F6][F7]) — no second modulation coefficient is
      introduced — and the modulation uses **arousal only, never valence** ([I3]).
- [x] A5 Recall/reconsolidation resetting `semanticize_timer_reset_at` to `now` **delays** the next stage
      (`unitsElapsed` recomputes to 0) but **never lowers** `semantic_stage` — already-risen gist stars are kept
      ([C6a][C7], z-axis one-way). (The reset _write_ is plan 33 / job 44; this unit defines the read semantics.)
- [x] A6 The gist axis is **independent** of forgetting: the gist-timer reads `semanticize_timer_reset_at`, while
      forgetting decay reads `last_recalled_universe_time`; a memory may be deeply decayed yet minimally gistified and
      vice versa ([F], [C]).
- [x] A7 `GistCoordinate(hippocampalX, hippocampalY, stage)` **copies x,y verbatim** from the hippocampal star and
      returns a `z` strictly inside the reserved neocortex band `[force_sim.neocortex_z_min, force_sim.neocortex_z_max]`
      (15..25), disjoint from the hippocampus band (0..10); the neocortex has **no force-sim** and no independent
      coordinate authority ([C5][C6][V9][I5]).
- [x] A8 Semanticization **never deletes** any memory, neuron, or synapse and never mutates the `Diary` — it only
      rises a derived stage and computes a coordinate; the concrete hippocampal star and its `current_text` remain
      ([I1][I2]).
- [x] A9 As stages rise, the model specifies a `memory_provenance` append with `kind = semanticized`,
      `source = system`, the risen stage's pregenerated text, and the advance `universe_time` (CC5, [R8a]); the table/port
      are owned by plan 32 / job 43 and the write executes in plan 41 / job 52 — this unit performs no IO.
- [x] A10 `semantic_stage` is exposed as the **gist-depth signal** for the view-cost curve ([R8][G4], CC3); this unit
      exposes the depth only and defines **no** Twinkle price (pricing is plan 43, Epic F).
- [x] A11 The Go functions (`internal/memory`) and the TS mirror (`packages/memory-logic`) produce **identical
      outputs** for a shared golden-fixture table within a fixed tolerance (TS↔Go golden-parity for the timer +
      z-coordinate), and both read the same generated `semantic.*` / `force_sim.*` constants; no formula, curve shape, or
      clamp is stored in `values.yaml`.
- [x] A12 This unit adds **no** DB column, query, RPC, or proto field — it reads columns that already exist on
      `episodic_memories` (plan 16) and are already returned by `GetUniverse`; it is pure domain math (§2.4/§2.5).
