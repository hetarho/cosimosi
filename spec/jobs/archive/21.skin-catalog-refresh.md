## 인수 조건 (acceptance — 기획 관점, changes/11.skin-catalog-refresh에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [x] A1 배경 카탈로그의 사용자-facing 색 정체성은 고정 `accent`·`bg`·`palette`가 아니라 현재 mood/감정색에서 파생된다. 같은 `vast`라도 주 감정이 다르면 배경 주 hue가 바뀐다.
- [x] A2 기존 4 배경(`vast`·`lively`·`calm`·`aurora-veil`)은 id 호환을 유지하되 레거시 고정 팔레트를 제거하고, 질감/패턴 프리셋으로 다시 정의된다. 네 항목은 색만 다른 같은 스킨처럼 보이지 않아야 한다.
- [x] A3 신규 배경 `signal-noise`·`abyssal-sea`·`cosmic-cliffs`가 카탈로그와 구매/소유권 검증에 포함된다. 세 항목은 각각 노이즈, 심해/바다, 성운 절벽의 질감 차이가 보인다.
- [x] A4 배경 swatch/tile은 고정 색 그라디언트가 아니라 현재 mood 기반 질감 preview 또는 중립 질감 token으로 표시된다. 배경 선택 UI가 "색 고르기"처럼 보이면 안 된다.
- [x] A5 빈 우주·미인증·로딩 상태에서도 배경은 중립 fallback으로 안전하게 렌더된다. 이 fallback은 별 mood 색이나 사용자 감정색을 오염시키지 않는다.
- [x] A6 `SELF_OBJECTS` 노출 카탈로그는 `mirrorball`·`prism-cube`·`neuron-bloom` 3종뿐이다. `nebula-heart`·`core`·`well`은 UI에서 사라지고, 남아 있는 레거시 선택값은 우주를 깨지 않고 `mirrorball`로 정규화된다.
- [x] A7 self 기본 무료 kind는 `mirrorball`이다. `spec/values.yaml`의 `customization.free.self`와 FE/BE 생성 상수가 이 값을 단일 출처로 공유한다.
- [x] A8 별 카탈로그는 기존 4종에 `pulsar` 1종을 추가한다. `pulsar`는 mood 색을 따르는 form 아이템이며, 별 색 규칙을 바꾸지 않는다.
- [x] A9 시냅스 카탈로그는 `filament`·`particle`·`dendrite` 3종만 노출한다. `beam`·`flow`는 UI에서 사라지고, 남아 있는 레거시 선택값은 `filament`로 정규화된다.
- [x] A10 `dendrite`는 작은 가지가 갈라지는 신경 돌기형 연결로 보이고, 색=양끝 별 mood 블렌드·weight 시각·삭제금지·Line2/TSL 제약을 유지한다.
- [x] A11 `spec/values.yaml`의 `customization.price`에서 제거된 아이템(`self:core`, `self:well`, `synapse:beam`, `synapse:flow`)은 더 이상 신규 구매 대상으로 검증되지 않는다. 신규 유료 아이템(`background:signal-noise`, `background:abyssal-sea`, `background:cosmic-cliffs`, `star:pulsar`, `self:prism-cube`, `self:neuron-bloom`, `synapse:dendrite`)은 가격이 있다.
- [x] A12 기존 사용자 settings/owned rows에 레거시 id가 남아 있어도 GetSettings/렌더/편집 UI가 크래시하지 않고, 저장 시 새 카탈로그 id만 영속된다. 데이터 행 삭제는 하지 않는다.
- [x] A13 별가루 구매·소유권·서버 선택 검증, 감정색 온보딩/편집, 공유 우주 owner settings 스냅샷, 랜딩/사인인/초대 플레이그라운드에 회귀가 없다.
- [x] A14 FE build/lint/typecheck/steiger와 관련 단위 테스트가 통과한다. values를 바꾸면 `pnpm gen:values` 생성물이 함께 갱신된다.

- FE+BE config 중심 변경. DB/proto 필드 추가는 없다. 단, proto 주석이 기본값을 언급하면 `nebula-heart` → `mirrorball`로 갱신하고 `(gen:proto)` 여부를 판단한다.
- change 10/job 20과 같은 plan 44·홈 편집 표면을 건드릴 수 있다. job 20이 먼저 구현되어 있으면 `AppearancePanel` 기준으로, 아직이면 현재 `AppearanceModal`/`AppearanceSwitcher` 기준으로 작업하고 충돌 없이 리베이스한다. 카탈로그·렌더·values는 job 20과 독립적으로 끝나야 한다.
- [x] T001 현 상태 감사 — `git status`와 `rg -n "BACKGROUNDS|BackgroundMeta|paletteForBackground|themeAccent|themeBg|UniverseNebula|SELF_OBJECTS|SelfObject|buildSelfForm|STAR_OBJECTS|buildStarBody|SYNAPSE_STYLES|SynapseStyle|nebula-heart|core|well|beam|flow|customization" frontend/src backend proto spec/values.yaml` 결과를 확인한다. job 20이 이미 적용됐는지 확인해 UI 호스트 경로(`AppearanceModal` vs `AppearancePanel`)를 결정한다.
- [x] T002 `(gen:values)` values 카탈로그 갱신 — `spec/values.yaml` `customization.free.self`를 `mirrorball`로 바꾸고, 가격표를 새 카탈로그에 맞춘다. 제거: `self:core`, `self:well`, `synapse:beam`, `synapse:flow`. 추가 권장값: `background:signal-noise=35`, `background:abyssal-sea=40`, `background:cosmic-cliffs=45`, `star:pulsar=45`, `self:prism-cube=35`, `self:neuron-bloom=45`, `synapse:dendrite=40`. 기존 유료 배경(`lively`·`calm`·`aurora-veil`), 기존 유료 별(`aurora`·`liquid`·`ember`), `synapse:particle` 가격은 유지한다. `pnpm gen:values` 실행 후 FE/BE 생성물 diff를 확인한다.
- [x] T003 타입·헬퍼 경계 정리 — `Background` union에 신규 3종을 추가하고, `SelfObject`/`SynapseStyle` union은 새 노출 카탈로그만 담게 한다. 레거시 문자열은 타입 union에 남기지 말고 `normalizeSelfObject`/`normalizeSynapseStyle` 같은 순수 헬퍼나 store merge 경계에서 처리한다. `itemId`/`isKnownItem`은 values 생성 상수만 보게 유지한다.
- [x] T004 배경 카탈로그 재정의 — `BACKGROUNDS`에서 사용자-facing fixed hue 정체성(`accent`·`bg`·`palette` 중심)을 제거/중립화하고, 각 항목을 `pattern`·`emotionSlots`·`texture/effect kind`·중립 token 중심으로 다시 쓴다. 기존 4종은 `vast`(저주파 깊이), `lively`(와류/입자), `calm`(느린 파동), `aurora-veil`(베일/줄무늬)로 재정의하고, 신규 `signal-noise`·`abyssal-sea`·`cosmic-cliffs`를 추가한다. `themes.ts` 호환 alias는 깨지지 않게 유지한다.
- [x] T005 mood-driven background 렌더 — `UniverseCanvas`/`UniverseNebula`가 스킨별 fixed palette hue를 주색으로 쓰지 않게 바꾼다. R-가중 `rankedEmotions` + 사용자 감정색(`resolveMoodRgb`)에서 dominant/secondary 색을 받아 tonal ramp를 만들고, 배경 스킨은 그 색을 어떻게 질감에 얹을지만 결정한다. empty/unauth/loading은 neutral deep-space fallback을 쓴다. `themeBg`는 clear color용 중립색 또는 mood-aware 경로로 정리하되, 별 mood 색과 자아 반사광 중립 원칙을 건드리지 않는다.
- [x] T006 비-우주 배경 소비처 정리 — `paletteForBackground(theme)`, `themeAccent(theme)`, `backgroundMeta(theme).palette`를 쓰는 사인인/랜딩/초대/CosmosScene 경로를 감사한다. 사용자 mood가 없는 화면은 중립 fallback 또는 해당 화면이 가진 샘플 mood만 사용하고, 기존 배경 id가 자홍/청록 같은 고정 hue를 몰래 주입하지 않게 한다. 랜딩/사인인 플레이그라운드가 4축 선택은 유지하되 "배경색 고르기"처럼 보이지 않게 한다.
- [x] T007 배경 UI token/swatch — `AppearanceSwitcher`와 홈 편집 표면(job 20 적용 전이면 `AppearanceModal`, 적용 후면 `AppearancePanel`/`AppearanceItemToken`)에서 배경 항목 swatch를 고정 색 그라디언트에서 질감 token으로 바꾼다. 가능하면 현재 mood 색을 주입한 preview를 쓰고, mood가 없으면 중립 질감 token을 쓴다. 항목 텍스트/라벨도 "색"보다 "결/질감"이 드러나게 조정한다.
- [x] T008 Self 카탈로그 교체 — `SELF_OBJECTS`를 `mirrorball`·`prism-cube`·`neuron-bloom` 3종으로 교체하고 `DEFAULT_SELF_OBJECT='mirrorball'`로 바꾼다. `buildSelfForm`을 새 3종 실루엣으로 재작성한다: mirrorball=faceted reflective/glint sphere, prism-cube=굴절/색분산 큐브, neuron-bloom=soma+dendrite branch 형태. 기존 `nebula-heart`·`core`·`well` 분기는 제거하되 레거시 선택값은 T011에서 기본값으로 정규화한다.
- [x] T009 Star `pulsar` 추가 — `StarObject` 타입·`STAR_OBJECTS` 메타·별 렌더(`buildStarBody`/form shader)·token/preview에 `pulsar`를 추가한다. mood 색 uniform/attribute 경로를 그대로 쓰고, `pulsar`가 별 색·밝기·기억 데이터 계약을 바꾸지 않게 한다.
- [x] T010 Synapse 카탈로그 정리 — `SYNAPSE_STYLES`에서 `beam`·`flow`를 제거하고 `dendrite`를 추가한다. `SynapseFilaments`의 beam/flow 분기와 주석을 정리하고, `dendrite`는 Line2/TSL 제약 안에서 작은 가지가 보이는 경로로 구현한다. `particle`은 유지한다. 레거시 `beam`·`flow` 선택값은 store/렌더에서 `filament`로 정규화한다.
- [x] T011 레거시 id 정규화/서버 응답 방어 — localStorage merge, `applyServerSettings`, visit/share owner settings, renderer fallback 경계에서 `nebula-heart`·`core`·`well`→`mirrorball`, `beam`·`flow`→`filament`로 안전 정규화한다. 저장/UpdateSettings로는 새 카탈로그 id만 나가야 한다. 기존 `user_owned_items` 행은 삭제하지 않고, removed price ids는 신규 구매만 막는다.
- [x] T012 BE whitelist/테스트 갱신 — BE는 generated `values.CustomizationFree`/`CustomizationPrice`로 whitelist를 파생하므로 values 생성물만으로 새 id 검증이 바뀌는지 확인한다. 설정 서비스 테스트가 있으면 unknown/free/paid/removed id 케이스를 갱신하고, 없으면 최소 단위 테스트를 추가해 `self:core`/`synapse:beam` 구매가 unknown으로 거부되고 `self:mirrorball` 선택은 free로 통과함을 확인한다.
- [x] T013 Proto/공유 타입 주석 갱신 — `proto/cosimosi/v1/memory.proto`와 generated docs/comments, `backend/internal/share/*`·`frontend/src/shared/api/gen/**` 주석 중 기본 self가 `nebula-heart`라고 쓰인 곳을 `mirrorball`로 갱신한다. proto comment를 바꾸면 `pnpm gen:proto` 또는 `pnpm gen`을 실행한다.
- [x] T014 [P] 테스트/토큰 회귀 방어 — `entities/appearance` store 테스트를 추가/수정해 persisted legacy ids와 server legacy ids가 새 기본값으로 정규화되는지 검증한다. `entities/memory/model/ambient.test.ts`는 mood 색 파생이 배경 주색으로 쓰이는 경로를 깨뜨리지 않는지 필요 시 보강한다.
- [x] T015 [P] 문서 반영 준비 — 구현 완료 시 `plan/44.customization-system.md`, `plan/25.ambient-mood.md`, `plan/38.self-anchored-layout.md`, `policy/domain/customization.md`, `policy/domain/universe.md`, `policy/domain/star.md`, `policy/domain/synapse.md`, 필요 시 `tech/architecture.md`를 새 truth로 갱신한다. "배경 색의 출처=배경 자체" 문구는 모두 "mood/감정색 파생, 스킨=질감"으로 바꾼다.
- [x] T016 검증 실행 — `pnpm gen:values`(필수), proto 주석 변경 시 `pnpm gen:proto` 또는 `pnpm gen`, FE build/lint/typecheck/steiger, 관련 unit test, BE `go test ./...` 또는 settings 패키지 테스트를 실행한다. `rg -n "nebula-heart|self:core|self:well|synapse:beam|synapse:flow|배경 자체의 색|paletteForBackground"`로 남은 레거시 참조가 의도된 archive/문서/정규화 맵뿐인지 확인한다.
