## 인수 조건 (acceptance — 기획 관점, plan/45.mood-color-onboarding에서)

- [x] A1 인증·멤버 사용자가 `/`에 진입할 때 `GetSettings` 응답에 13개 mood 전부의 유효한 `#RRGGBB` 감정색이 없으면 `/emotion-colors?redirect=<원래경로>`로 리다이렉트되고, `HomePage`와 `GetUniverse`는 아직 마운트되지 않는다.
- [x] A2 감정색 완료 여부는 최초 로그인 여부가 아니라 서버 설정 내용으로만 판정된다. 기존 사용자라도 13색 중 하나가 없거나 malformed이면 설정 페이지로 간다.
- [x] A3 `/emotion-colors`는 인증·멤버 사용자만 접근 가능하고, `EmotionColorGate` 밖에 있어 미완료 사용자가 루프 없이 저장을 끝낼 수 있다.
- [x] A4 설정 페이지는 13개 mood를 모두 보여주며, 초기 draft는 서버에 저장된 색이 있으면 그 값, 없으면 `MOOD_PALETTE` 기반 추천색이다.
- [x] A5 사용자는 각 mood에 대해 추천색과 완전히 다른 임의의 `#RRGGBB` 색을 선택할 수 있다. 너무 어둡거나 특이한 색도 저장을 막지 않는다.
- [x] A6 컬러 피커는 커스텀 스펙트럼 UI(saturation/value 면, hue rail, preview, hex 입력, 추천 복귀, 지원 시 EyeDropper)를 제공하고, 브라우저 기본 color input만 덩그러니 보이는 UI가 아니다.
- [x] A7 저장은 `UpdateSettings.emotion_colors`에 13개 색을 한 번에 보내며, 성공 후 DB의 `user_emotion_colors`에는 호출자 user_id 기준 mood당 한 행이 저장된다.
- [x] A8 서버가 invalid mood/color를 거부하면 어떤 감정색도 부분 저장되지 않고, 페이지는 draft를 유지하며 오류를 보여준다.
- [x] A9 저장 성공 직후 `applySettings`와 settings query cache 갱신이 일어나, redirect 후 `EmotionColorGate`가 stale empty settings 때문에 다시 설정 페이지로 튕기지 않는다.
- [x] A10 저장된 색은 `useAppearance.emotionColors`에 반영되고, 별(`StarField`)과 시냅스(`UniverseCanvas`)가 refetch 없이 해당 mood 색으로 재베이킹된다.
- [x] A11 이미 완료된 사용자가 `/emotion-colors`에 직접 들어가면 현재 서버 색으로 편집할 수 있고, 저장 전에는 DB가 바뀌지 않는다.
- [x] A12 데모 모드, 미인증 랜딩, 공개 방문 우주(`/u/$slug`)는 이 게이트를 거치지 않고 기존 기본/공유 appearance로 렌더된다.
- [x] A13 로그아웃·계정 전환 후 이전 사용자의 감정색이 다음 사용자에게 보이지 않는다. 다음 사용자 판정은 새 `GetSettings` 응답으로만 한다.
- [x] A14 FSD 단방향이 유지된다. 완료 판정/추천 변환은 React/DOM/three 미의존 순수 모듈이고, 라우트 게이트는 app, 화면은 pages, store/RPC helper는 entities/appearance에 둔다.
- [x] A15 build·lint·typecheck·steiger를 통과하고, 기존 30번 테마/오브제 설정 저장과 41번 초대 게이트 동작에 회귀가 없다.

- [x] T001 [P] `frontend/src/entities/appearance/model/emotion-colors.ts` 신규 — 순수 helper(React/DOM/three import 금지): canonical mood order(29 사분면 순), `recommendedEmotionColors`(`MOOD_PALETTE` → `#RRGGBB`), `rgbToHex`, `normalizeHex`, `isHexColor`, `isCompleteEmotionColors`, `mergeEmotionColorDraft(serverColors)`를 만든다. unknown mood·`MOOD_UNSPECIFIED`·malformed color는 완료 판정에서 무시하고, 저장값은 대문자 `#RRGGBB`로 정규화한다(A2/A4/A14).
- [x] T002 [P] `frontend/src/entities/appearance/model/emotion-colors.test.ts` 신규 — helper 단위 테스트: 13개 mood 전체성, 추천색이 `MOOD_PALETTE`에서 파생됨, 소문자 hex 정규화, 하나 누락/손상/unknown 포함 시 incomplete, server color가 있으면 추천값보다 우선한다. 기존 `mood.test.ts` 스타일(Vitest) 미러.
- [x] T003 `frontend/src/entities/appearance/api/settings-query.ts` — 감정색 full-set 저장 helper 추가(예: `saveEmotionColors(queryClient, draft)`): `Record<Mood,string>` draft를 proto `EmotionColor[]`로 매핑해 `UpdateSettings({ emotionColors })` 호출, 성공 응답으로 `applySettings(res)`를 즉시 호출하고 `settingsQueryOptions().queryKey` cache를 같은 응답으로 `setQueryData`한다(A7/A9). 기존 `pushSettings`의 theme/object/self/synapse 경로는 회귀 없이 유지한다.
- [x] T004 `frontend/src/entities/appearance/model/store.ts` — 필요 시 `setEmotionColors(colors)` full-set 액션 추가. per-user `emotionColors`는 계속 메모리 전용이고 `partialize`/localStorage에 들어가지 않아야 한다(A13). 44 작업이 이미 store를 확장한 상태면 그 구조 위에 최소 패치로 얹는다.
- [x] T005 `frontend/src/entities/appearance/index.ts` — T001/T003의 helper를 named export. 배럴 `export *`는 피하고 기존 공개 API를 유지한다.
- [x] T006 `frontend/src/app/ui/EmotionColorGate.tsx` 신규 — `SessionGate`/`MembershipGate` 안쪽에서 쓰는 FE 게이트. 데모 모드는 즉시 children, 그 외엔 `useQuery(settingsQueryOptions())`; pending은 스플래시, error는 재시도 UI, success는 `applySettings(data)` 후 `isCompleteEmotionColors(...)`가 true면 children, false면 `<Navigate to="/emotion-colors" search={{ redirect: location.href }} replace />`. children을 통과시키기 전까지 `HomePage`가 마운트되지 않게 한다(A1/A10/A12).
- [x] T007 `frontend/src/app/router.tsx` — `/emotion-colors` 라우트 추가. 라우트는 `<SessionGate><MembershipGate><EmotionColorPage /></MembershipGate></SessionGate>`로 감싸되 `EmotionColorGate` 밖에 둔다(A3). `/`와 `/gift/$token`은 `MembershipGate` 안쪽에 `EmotionColorGate`를 추가한다. `safeRedirect` reject 목록에 `/emotion-colors`·`/invite`·`/sign-in`을 포함해 자기 자신/게이트 루프를 막는다.
- [x] T008 `frontend/src/pages/emotion-colors/index.ts` + `ui/EmotionColorPage.tsx` 신규 — 페이지는 `settingsQueryOptions()`로 현재 settings를 읽고, helper로 draft를 초기화한다(서버 색 우선, 없으면 추천색). 저장 전까지 DB를 바꾸지 않고, 저장 성공 시 `saveEmotionColors` → redirect(없으면 `/`). invalid/서버 오류는 draft 유지 + error-feedback. 이미 완료된 사용자도 현재 서버 색으로 편집 가능(A4/A8/A11).
- [x] T009 `frontend/src/features/pick-emotion-colors/ui/EmotionColorPicker.tsx` 신규(또는 페이지 내부 컴포넌트로 시작) — custom picker 구현: saturation/value 면, hue rail, 현재/추천 preview, hex 입력, 추천 복귀, 전체 추천 초기화, 지원 시 EyeDropper 버튼. 브라우저 기본 `<input type="color">`는 visible primary UI로 노출하지 않는다. pointer/touch drag + keyboard arrow adjustment + roving/focus 상태를 제공하고, reduced-motion에서는 preview 전환/glow 모션을 줄인다(A5/A6).
- [x] T010 `EmotionColorPage` 레이아웃/미리보기 — 데스크톱은 mood swatch 목록 + 선택 mood 별 미리보기 + picker를 한 화면에서 조작 가능하게, 모바일은 compact grid/가로 목록 + 하단 저장 command bar(safe-area 보정). 감정 순서는 29 사분면 순서, mood label은 `MOOD_LABEL`/`moodLabel` 단일 출처. 긴 안내문/랜딩식 설명 대신 실제 설정 UI가 첫 화면이 되게 한다(A4/A6).
- [x] T011 저장·렌더 반영 연결 확인 — 저장 성공 후 `useAppearance.emotionColors` 참조가 갱신되고, `StarField`/`UniverseCanvas`가 기존 dependency로 색을 재베이킹하는지 확인한다. 필요 시 full-set setter가 새 object reference를 보장한다(A10). `StarField.tsx`/`UniverseCanvas.tsx` 자체 변경은 소비 경로가 이미 맞으면 하지 않는다.
- [x] T012 감정색 편집 진입점 추가 — 우주 메인 메뉴 Surface나 `AppearanceControls`에 "감정색" 명령을 추가해 완료 후에도 `/emotion-colors`로 재방문할 수 있게 한다. 44의 4축 인벤토리 작업과 충돌하면 최신 `AppearanceControls` 구조에 맞춰 작은 링크/버튼만 추가한다(A11).
- [x] T013 공개/체험/방문 경로 회귀 확인 — `/landing`, `/sign-in`, `/invite`, `/admin`, `/u/$slug`, 데모 모드는 `EmotionColorGate` 밖이거나 bypass라 기존 기본/공유 appearance로 렌더된다. `VisitPage` owner appearance 적용과 로그아웃/계정전환 `resetServerSettings()`가 감정색을 새지 않게 유지하는지 확인(A12/A13).
