- [x] `/code-review` applied (rejections noted with reason) · for non-trivial logic, `/codex:review --background`

Applied: `disabled || loading` (was `?? loading`, false slipped through); tooltip `aria-describedby` moved onto the
focusable trigger via `cloneElement`; focus-trap now excludes `tabIndex < 0`; Switch/Checkbox/Dialog gained an
`ariaLabel` fallback so a labelless control is never unnamed; native spinner color follows the variant foreground
(was hardcoded white); shared Spinner + button variant/focus-ring constants extracted to kill duplication.
Rejected: backdrop-click is pointer-only (keyboard close is covered by Escape + the focusable close button);
`FOCUSABLE_SELECTOR` export (internal detail); controlled↔uncontrolled mid-life switch warning (matches React's own
uncontrolled-input behavior — documented as "pick one"). codex ran via `-m gpt-5.5 -c service_tier=fast` (the
account rejects `priority`/`flex`).

Note on T004/native styling: NativeWind v5 was the initial choice but cannot run on the bare-RN community CLI (its
engine transforms through Expo's vendored Metro fork, which collides with the RN CLI's upstream Metro at
serialization). Native styling is therefore a React Native `StyleSheet` bridge from the same `tokens.ts`
(`native-styles.ts`) — one token source preserved, web Tailwind v4 unchanged. Mobile bundles on bare RN (verified via
`react-native bundle`); this also required enabling `@babel/plugin-transform-export-namespace-from` for
`@cosimosi/i18n`'s Paraglide output.

Note on T007: web render paths are tested in jsdom (testing-library) + a runnable web UI showcase. Native render-path
automated tests await a React Native test runner (not in the repo yet — quality-gates/test-harness territory); the
`*.native.tsx` siblings are covered by the package + mobile typecheck and a runnable native UI showcase (mobile bundle
verified).
