## Acceptance Criteria (from code-review/04 — R001, R003, R004)

<!-- Behavior preserved (jobs still drain, retry, back off), structure hardened. -->

- [x] A1 **A reclaimed job converges to `failed`.** A job whose handler crashes the worker (panic / process kill) or
      overruns its lease is eventually marked `failed` after the configured attempt budget — it is not retried forever.
      (R001 facet 1: attempts must advance on claim, not only on a gracefully-returned error.)
- [x] A2 **A stale lease holder cannot clobber a re-claimed job.** After a job's lease expires and another worker
      re-claims it, the original worker's `Complete`/`Retry`/`Fail` is a no-op (guarded by an ownership fence), so the
      re-claim's result wins. (R001 facet 2: no lost update.)
- [x] A3 **A job is never run concurrently by two workers via the lease-expiry path.** The fence makes a terminal
      transition succeed only for the current lease owner, so a lease-expired run cannot commit a duplicate metered
      call's result. (R001 facet 3.)
- [x] A4 **Lease duration and backoff are reconciled**, so "lease ≥ expected max handler runtime" is expressible from
      config rather than two independently-derived formulas.
- [x] A5 **A provider `Retry-After` hint is honored.** On a rate-limit error carrying a non-zero `RetryAfter`, the job
      is rescheduled no earlier than `now + RetryAfter` instead of the local exponential backoff. (R003.)
- [x] A6 **A named provider with an empty API key fails fast (or warns loudly) instead of silently mocking.** Setting a
      provider without its key no longer boots a keyless-mock seam under the guise of a real deployment. (R004.)
- [x] A7 **No behavior change to the happy path / no regression.** `FOR UPDATE SKIP LOCKED` claim, exponential backoff,
      cost-limit 24h reschedule, and idempotent terminal writes are unchanged; all existing `internal/memory`,
      `internal/platform/jobqueue`, and `internal/ai` tests stay green.
