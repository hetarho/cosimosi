- [x] A1 **The renderer package is typechecked by the gate.** `pnpm typecheck` (hence `pnpm check` / CI) runs
      `packages/3d-renderer`'s `tsc`. A deliberate type error in a `3d-renderer` file the apps don't import fails the
      gate. (R006.)
- [x] A2 **The comment-history lint covers Go and the forms actually used.** `pnpm lint:comments` scans
      `apps/api/**/*.go` and flags `plan N` / `job N` / `R0NN` / `Epic X` narration; `pnpm lint:comments:probe`
      proves it catches those and leaves the allowed anchors alone. **Policy decision:** `§N` architecture-section
      pointers and `[I2]`/`[E7a]` requirement-ID anchors are _allowed_ (design rationale that names a rule, not
      process history, and an established house convention tree-wide) — flagging `§N` would churn ~130 borderline
      comments across the whole codebase for no correctness gain. (R007.)
- [x] A3 **The existing process/plan comments are gone.** The plan/job/section-number narration flagged in review 04
      (Go ~100, packages, FE) is rewritten as timeless "why" or deleted; `pnpm lint:comments` is green afterward.
      Invariant tags that name a rule (`[I2]`, `[E10]`) may stay per the policy decision recorded in A2's rule set. (R007.)
- [x] A4 **Renderer default props source from generated config (or require the prop).** `ColorField`/`LatentField`
      no longer carry numeric defaults that contradict `spec/values.yaml` (`falloffExponent`, `baseIntensity`,
      latent `size`). One source of truth. (R008.)
- [x] A5 **Layout-shaping force-sim integration constants flow through `values.yaml`.** `velocity_damping` and
      `min_alpha` (the top-level integration tuning read in `simulation.ts`) are now generated config, seeded to the
      current literals so the golden fixture is byte-identical. **Scoped decision:** `BARNES_HUT_THETA`/`SOFTENING`
      (and `MIN_HALF_SIZE`, `FRAME_RATE_NORMALIZER`, `MAX_FRAME_STEP`) stay in code, documented as
      approximation/solver-internal (how the O(n log n) force is _computed_, not the emergent layout — `repulsion`
      already carries the layout knob through config); the `forces.ts`/`seed.ts` inline factors are formula
      coefficients (the config rule excludes formulas). (R009.)
- [x] A6 **The Node pin is coherent.** Either `check:node` enforces the `.node-version` pin (not just the engines
      floor) or the exact pin is dropped in favor of the `>=` floor — decided and documented. (R014.)
- [x] A7 **No behavior change / no regression.** These are gate + config-plumbing changes; the rendered output and
      force-sim golden output are unchanged (values.yaml keeps the same numbers), and all gates pass.
