## Review

- [x] Reviews applied — Codex (`gpt-5.5`, read-only) + an independent subagent pass over the diff.
  - **Applied**: mobile callback dedupe + exact-address matching (duplicate `getInitialURL`+`url` delivery raced the
    single-use code; `cosimosi://auth-callback.evil` matched the prefix); `cancelSignIn` gated on an
    external-OAuth-pending flag (a blanket AppState/pageshow cancel could abandon a slow password sign-in or race an
    in-flight code exchange); web client `flowType: 'pkce'` (tokens out of the return-URL fragment); per-method busy
    indicator + failure copy (incl. post-remount OAuth failures no longer showing password copy);
    `role="alert"`/`accessibilityLiveRegion` on the error text.
  - **Rejected with reason**: (a) cancel leaving `suppressAdapterRefresh=true` — identical to the existing `signOut`
    posture; self-heals on the next authenticated adapter event. (b) Android recents re-delivering a consumed
    callback intent can fail an in-flight bootstrap — indistinguishable from a legitimate cold-start completion
    without persisted dedupe state; self-heals via the next adapter event or app restart. (c) web consent-denied
    return shows no error (params dropped at `/`) — the user lands back on a retryable login form; surfacing it
    needs URL-error parsing at the route layer, deferred as polish. (d) web `signingIn` sticks if the OAuth
    navigation never commits (offline click) — needs a timeout knob; disproportionate for v1, recoverable by reload.
