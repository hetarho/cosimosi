## Acceptance Criteria (from plan/25.latent-star-field)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 **Rendering-only, no DB.** The latent-star field is generated and drawn entirely on the client; it produces **no**
      database rows, no RPC calls, and no server state. A `neurons` row exists only for an _activated_ neuron, and it is
      written by Encode ([20.encode-usecase](../plan/20.encode-usecase.md)), not by this unit ([E7a]).
      _Verified: the two slices carry no `api` segment, no transport/RPC import (grep); no `apps/api`/`proto`/`db` diff._
- [x] A2 **Field density & placement.** On scene init the universe shows roughly `rendering.latent_star_count` gray points,
      each with **no brightness, no color, and no identity**, positioned within the hippocampus z-band
      `[force_sim.hippocampus_z_min, force_sim.hippocampus_z_max]` (from [19.force-sim-layout](../plan/19.force-sim-layout.md))
      and filling the visible hippocampus volume in x,y.
      _Verified: `generateLatentField` + `latent-field.test.ts` (count, z ∈ band, x,y within radius, positions-only — no channels)._
- [x] A3 **Background layer.** The field renders as a background layer inside [23.universe-canvas](../plan/23.universe-canvas.md),
      _behind_ the real stars/neurons/synapses of [24.star-neuron-rendering](../plan/24.star-neuron-rendering.md); real bodies
      always render on top.
      _Verified: `LatentField` sets `depthTest=false`/`depthWrite=false`/`renderOrder=-1`; mounted before the real body layers._
- [x] A4 **Awaken on new-neuron creation.** When a launch creates a new neuron
      ([20.encode-usecase](../plan/20.encode-usecase.md)/[27.writing-flow-ui](../plan/27.writing-flow-ui.md)), exactly one gray
      latent star plays an **awaken** animation and hands off to the real `cell-star`. Multiple new neurons in one launch each
      awaken a distinct latent star.
      _Verified: `AwakenNeuron` reacts to `newNeuronIds`; `pickAwakenSeeds` distinctness tested (N births → N distinct)._
- [x] A5 **Awaken anchor = near a recent neuron, else random.** The gray star chosen to awaken is the one nearest a
      _recently-active_ neuron (the [L4] temporal window used conceptually); when there is no recent cue the pick is
      **random**. This selection is a client presentation decision — nothing about it is persisted or sent to the server.
      _Verified: `recentlyActiveNeuronIds` (window) + `pickAwakenSeeds` (nearest-to-anchor / random fallback) tested._
- [x] A6 **Final position emergent, not stored.** The awaken start point is only the animation's anchor; the real neuron's
      **final position is emergent** from the force-sim ([19.force-sim-layout](../plan/19.force-sim-layout.md)) and is **never
      stored** ([I5]). No "awaken position" is written anywhere.
      _Verified: the flare plays at the seed only; no coordinate is written anywhere (no DB, no store beyond consumed marks)._
- [x] A7 **Instanced + TSL.** The field is drawn with `InstancedMesh` bucket(s) and a material authored in **TSL** (compiles
      to WGSL + GLSL; no raw-GLSL `onBeforeCompile`), mounted through the [14.rendering-foundation](../plan/14.rendering-foundation.md)
      renderer boundary — no slice imports `three` directly ([ARCHITECTURE §3.3](../ARCHITECTURE.md)).
      _Verified: `LatentField` = `InstancedMesh` + `MeshBasicNodeMaterial` (TSL `positionNode` drift); no `three` import in any slice (grep + eslint)._
- [x] A8 **Count tunable; mobile lowers it.** `rendering.latent_star_count` is a single build-time generated scalar
      (`values.yaml` → `pnpm gen:values`); changing it changes the field density with no code edits, and the mobile MVP
      renders a reduced count ([ARCHITECTURE §3.5](../ARCHITECTURE.md)).
      _Verified: web uses `latentStarCount` (800), mobile uses `latentStarCountMobile` (300); both generated, no literals in code._
- [x] A9 **Emergent stays rendered-not-modeled.** The unit adds **no** `latent-star` type to the Go domain, DB, proto, or the
      FE domain-mirror; visual words do not travel upward into the domain slices ([ARCHITECTURE §3.4](../ARCHITECTURE.md)).
      _Verified: no `latent` in `packages/*` domain / `proto` / `apps/api` beyond the generated config scalars (grep)._

## Review

- [x] `/code-review` (3 parallel finder agents + verify) **and** Codex (`gpt-5.5`, background) applied. All findings
      fixed (none rejected):
  - **Burst overflow** (codex + review): a birth-burst larger than the flare pool consumed latent stars without
    flaring them and marked ids processed before a slot was assigned → now takes only `min(fresh, freeSlots)` per pass,
    consumes/claims only what actually flares; the rest retry.
  - **Awaken idempotency across remounts** (codex): `processed` lived in a component ref, so a remount holding the same
    `new_neuron_ids` could re-consume → moved to a module-level `awaken-registry` store (survives remounts).
  - **Unclamped frame delta** (review): a large delta after a tab resume could skip a whole flare → per-frame step
    clamped (`AWAKEN_MAX_STEP_S`).
  - **LatentField remount flash** (review): a fresh `InstancedMesh` starts full-count with zero matrices → stable ref
    now hides it (`count=0`/`visible=false`) until the matrix effect writes, mirroring `InstancedNodeLayer`.
  - **FrameTick ordering fragility** (review): correct by mount order — documented with a comment (no code change).
  - Verified-fine (no change): in-place `scales` mutation read by the layer, the depthTest/renderOrder background
    trick, TSL drift validity, disposal deps, Park-Miller determinism, web↔mobile parity.
  - Comment pass: swept new comments to the "why-only" bar; fixed one imprecise `drift` contract note.
