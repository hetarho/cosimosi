## As-built notes (decided at implement)

- **Replay marker representation (T003.4):** the preferred no-new-column option — the touched
  constellation's synapses get `last_activated_universe_time` refreshed to the advance time
  (`TouchConstellationSynapses`, GREATEST-guarded). That is the same trace a recall's reinforcement
  leaves, and the read already consumes it (filament fade / effective synapse strength), so the
  replay is visible with no schema change and no stored coordinate. The consolidate job row also
  carries the interval + stage-advanced ids as a secondary marker.
- **Convergence mechanics (A10):** `Semanticize(stage, units)` is incremental from the timer
  anchor, so materializing a rise also **consumes the crossed units from the anchor** —
  `semanticize_timer_reset_at` moves forward by exactly the days those units spanned (binary
  search over job 51's `GistUnitsElapsed`; residual sub-unit progress carries). Re-running an
  already-consolidated interval then implies zero units. Downscale-once-per-interval is
  guaranteed by the hook's fire-once-per-clock-move discipline (a held clock fires nothing —
  covered by the same-day re-sync tests), not by a stored interval marker.
- **`from == nil` (first-ever advance) is a no-op:** every row in the universe was born inside
  that same transaction at `to`; there is no prior interval to sleep over, and downscaling
  seconds-old synapses would be wrong.
- **Interval read reuses `ListUniverseEpisodicMemories`** (universe.sql) instead of a duplicate
  select in consolidate.sql — consolidation and the read path see one non-deleted per-user
  memory shape, so the anchors/constants can never drift between them.
- **Decay-stage fill fills every missing slot 1..target** (not only the newly crossed ones):
  the stored JSONB is a positional array, and an empty-string placeholder would render as an
  empty text on the client (`decayStages[stage-1] ?? currentText` misses `""`). Existing
  entries are never overwritten, which also keeps re-runs byte-identical after a
  reconsolidation changes `current_text`.
- **Missing-stage-text provenance:** a rise whose pregenerated text is absent still appends its
  semanticized/system row (empty text) — the row anchors the rise event; the text regeneration
  is re-enqueued (`semanticize`, keeping the leading non-empty texts per [C7]).
