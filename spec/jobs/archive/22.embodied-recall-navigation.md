## 인수 조건 (acceptance — 기획 관점, plan/49.embodied-recall-navigation에서)
<!-- 출처 스펙의 수용 기준. 구현 후 각 항목이 코드에서 참인지 검증한다. -->
- [x] A1 `recall`(별들 가까이서 탐험하기)로 들어가면 `SelfStar`(나 오브제)가 우주 원점을 떠나 카메라 어깨 너머(뒤+위) 앵커로 이동해 카메라와 함께 항해한다. `nebula`로 돌아오면 원점으로 복귀한다.
- [x] A2 정상 전진 비행(전방 시야)에서는 자아 오브제·광원이 화면에 보이지 않는다(뒤+위에 위치).
- [x] A3 `recall`에서 자아 광원(reflection 채널 위치)이 카메라 위치가 아니라 같은 어깨-너머 앵커다. 정면의 별이 뒤+위에서 오는 빛으로 정면광·터미네이터 음영(입체)으로 보이고, 머리 위 플래시처럼 평평하게 꽉 차지 않는다.
- [x] A4 멀리 있던 별에 다가가면 거리 falloff로 반사가 강해진다. 단 reflection은 recency 곱을 유지하므로 최근 회상한 별은 확 밝아지고, 잠든(오래된) 별은 다가가도 은은하게만 반응한다.
- [x] A5 채널 경계: 자아 광원·아바타 이동은 reflection 채널·렌더 위치만 바꾼다 — `selfGlow`·`activation`·`λ_eff`·별 색·force-sim 좌표·`A_MIN` 바닥은 불변.
- [x] A6 모드 전이(`recall`↔`nebula` 토글, fly-to) 동안 자아 오브제·광원이 원점 ↔ 어깨앵커 사이를 매끄럽게 잇고, 화면 중앙으로 튀어 들어와 bloom을 덮지 않는다.
- [x] A7 idle shake·벽 충격 shake가 자아 광원·아바타를 흔들지 않는다(앵커는 shake 적용 전 기준 위치에서 계산).
- [x] A8 `nebula`(멀리서 내 우주 보기)는 종전과 동일하다 — `SelfStar` 원점·정적 중심 광원, orbit+zoom-dolly 유지.
- [x] A9 헌법 준수: 별 삭제 없음·`A_MIN` 바닥 유지(헌법2); 별 force-sim 좌표 권위 불변, 중심 닻 원점 유지(헌법3); 새 `THREE.PointLight` 없음, 단일 InstancedMesh·TSL 반사 유지(헌법8).
- [x] A10 서버·proto·RPC·sqlc 변경 없음(FE 전용). 겹쳐보기(spec 37)는 기존 self-light 규칙 유지.

## 구현 체크리스트 (implementation — 구현 관점)
<!-- 어떻게 만드나. 위→아래 순서. [P]=서로 다른 파일·의존 없는 병렬. (gen)/(migrate)/(gen:values) 표시.
     config·튜닝 수치는 코드 하드코딩 금지 — spec/values.yaml에 추가 후 (gen:values)로 생성 상수 import. -->
- [x] T001 (gen:values) `spec/values.yaml` `star_lighting`에 `recall_light_back_offset: 6`(카메라 −forward 거리)·`recall_light_up_offset: 2`(카메라 up 거리) 추가 후 `pnpm gen:values` → `frontend/src/shared/config/values.gen.ts`에 `VALUES.starLighting.recallLightBackOffset`/`recallLightUpOffset` 생성. (코드는 생성 상수만 import — 하드코딩 금지.)
- [x] T002 `NavController`(`UniverseCanvas.tsx`) 자아 광원 블록 교체: 현재 `selfLightRef.current = camera.position`(shake 전 기준 위치) → **어깨-너머 앵커** 계산. `fwd = normalize(target − camPosBase)`, `up = normalize(camera.up)`, `anchor = camPosBase − fwd·BACK_OFFSET + up·UP_OFFSET`. 기존처럼 shake 적용 *전* `camPosBase`에서 계산하고 재사용 배열(`lightArr`)로 per-frame 할당 없이 채운다(A7). `recall` 밖에서는 기존대로 `selfLightRef.current = null`(A8).
- [x] T003 `SelfStar.tsx`: 원점 고정 `<mesh>`(position 0) → **위치 ref 매 프레임 적용**. `selfLightRef`(또는 동등 앵커 ref)를 prop으로 받아 `useFrame`에서 `ref.current`가 있으면 그 좌표를, null이면 원점을 메시 position에 쓴다. body color(요즘 감정)·form·`raycast` off·reduced-motion freeze 등 나머지 불변. (`recall`에서 광원과 아바타가 같은 앵커를 공유 — A1·A2·A3.)
- [x] T004 `UniverseCanvas.tsx` 마운트부: `<SelfStar>`에 앵커 ref 전달(`<NavController selfLightRef={selfLightRef}>`와 동일 ref 공유). `selfObject` prop 유지.
- [x] T005 전이 보간(A6): `modeTransition`/`flyingToStar`(tag `transitioning`)에선 ref가 null이라 아바타가 원점 폴백에 머무름 → `recall` 도착(`ARRIVED`)·이탈 시 원점↔앵커 튐 방지. `SelfStar`에서 위치를 직전값→목표로 ease(lerp)하거나 도착 시점에만 앵커로 붙여, 전이 중 아바타가 화면 중앙으로 들어와 bloom을 덮지 않게 한다.
- [x] T006 검증: reflection 셰이더(`buildStarBody`)·`StarField`의 `selfLightRef` 소비 경로는 **수식·구조 무변**임을 확인(광원 위치 uniform 하나만 바뀜). 필요 시 `self_distance`/`self_decay`만 눈으로 미세 재튜닝(값 변경은 T001 경로로, 수식 불변). recency 곱이 유지돼 잠든 별이 접근에 은은하게만 반응하는지 확인(A4).
- [x] T007 `nebula`·겹쳐보기 회귀 확인: `nebula`는 `SelfStar` 원점·정적 중심 광원·orbit+zoom 그대로(A8), `UniverseOverlay`는 기존 self-light 규칙 유지(A10).
