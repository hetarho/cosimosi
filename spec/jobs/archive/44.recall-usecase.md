## Acceptance Criteria (from plan/33.recall-usecase)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [ ] A1 `Recall` and `RecallDiaryStars` **compose `SyncToToday`** ([30]) so recall happens on a clock synced to today,
      and return the advance interval `{previous_universe_time, universe_time}`; the **sync-consent gate is the UI's**
      ([36]/[47]) — this use-case exposes the capability, not the modal. ([R1a][T2] case 2)
- [ ] A2 `Recall` and `RecallDiaryStars` **spend Twinkle** through the consumer-owned `SpendGate` port and **never
      hardcode a price**; Epic C binds an **allow-all no-op** (the loop works without cost), Epic F binds the real gate; a
      denied spend aborts with a canonical error and resets nothing. ([G1][D3], §CC2)
- [ ] A3 On **every** recall (either branch), `Reinforce` resets the anchors — `last_recalled_universe_time =
universe_time`, `recall_count += 1`, `semanticize_timer_reset_at = universe_time` — batch-LTPs the co-activated
      synapses, **bumps `EffectiveStrength`** (size), and applies the neighbor forgetting ± ([R5], neighbors only).
      ([R2][R3][R5][C6a] reset)
- [ ] A4 The **visible** brightness/decay/gist recovery is **not** computed here — `Recall` writes anchors, and Epics
      D/E's read-time functions render the effect ([F5][R2]) against those anchors. ([R2][F5], §CC1)
- [ ] A5 `Reinforce` is an **idempotent batch LTP** over the memory's co-activated synapses (calls `Potentiate` once per
      edge, saturating toward the cap), **distinct** from the `Potentiate` primitive, and touches **no** memory↔memory
      edge. ([R3][I4][I6])
- [ ] A6 **Prediction error is an LLM semantic compare** of the rewrite vs. `current_text` (content change, not
      wording/order), held behind the consumer-owned `PredictionError` port. ([R6])
- [ ] A7 **No prediction error → reinforce only:** `current_text` and `seed` are **unchanged**; brightness/strength/timer
      still recover (via A3). ([R4][I8])
- [ ] A8 **Prediction error → `Reconsolidate`:** rewrite `current_text`, **change `seed`** (new shape [V5]), regenerate
      only the **remaining, not-yet-created** stage texts on the new basis while **keeping** already-risen gist
      stars/stages, and append a `reconsolidated/user` `memory_provenance` row (the neighbor ± + anchors + strength already
      ran in `Reinforce`). ([R6][C7][R8a])
- [ ] A9 The **`Diary` is never written** by `Recall`/`Reinforce`/`Reconsolidate`; only representation (`current_text`,
      `seed`, stage texts, anchors, synapse strengths, provenance) changes. ([I2][R7])
- [ ] A10 The neighbor forgetting ± is applied on **every** recall (both branches, via `Reinforce`) to **neighbor**
      episodic memories only (by shared _semantic_-neuron count), never to the recalled star (which recovers wholly [F5]);
      the ± math/threshold/column are plan 32's, invoked here — so the whole-diary jump ([D3]/[47]) gets [R5] without
      reconsolidation. ([R5])
- [ ] A11 `RecallDiaryStars` applies `Reinforce` to every still-live memory born from the diary in one transaction with
      sync + spend; it does **not** call `PredictionError` or `Reconsolidate`, writes no `current_text`, and changes no
      `seed`. ([D3][R4][R6][I8])
- [ ] A12 Reconsolidation is on **episodic** stars only; gist-star viewing ([R8]) is plan 34 and is not a recall path.
      ([R1])
- [ ] A13 The `Recall` / `RecallDiaryStars` RPCs are **not** `NO_SIDE_EFFECTS`; `Recall` sends only `memory_id` +
      `rewrite_text` (no seed, strength, decay, price, or diary field), `RecallDiaryStars` sends only `diary_id`, so
      reshape/bump/anchors/price/affected memory ids are all server-derived. (§2.7/§2.9#8)
- [ ] A14 Handlers are **thin** (proto↔domain map + call); sync/spend/compare/branch/anchor/reinforce/reconsolidate/
      provenance policy lives in the use-case/domain; no proto/sqlc leaks into the use-case or pure domain; `SpendGate`,
      `PredictionError`, `MemoryProvenanceStore`, and the repositories are consumer-owned interfaces.
      (§2.4/§2.7/§2.9#4/#6/#7)
- [ ] A15 Every read, spend, compare, write, and provenance append is **scoped to the authenticated user** and
      `pnpm lint:persistence` passes (§4).

- [ ] T004 Build **`Reinforce`** in `internal/memory` (e.g. `recall.go`) — the idempotent recall-effects bundle applied
      on every recall (A3/A5/A10): (1) anchor resets (`last_recalled_universe_time = universe_time`, `recall_count += 1`,
      `semanticize_timer_reset_at = universe_time`); (2) **batch LTP** — apply `Potentiate` once per edge over every
      synapse among the recalled star's member neurons, incrementing `co_activation_count`/`last_activated_universe_time`,
      saturating toward `values.SynapseStrengthCap` (idempotent per recall event; **no** memory↔memory edge [I4][I6]);
      (3) the `EffectiveStrength` bump rides the incremented `recall_count` (the plan-18/32 golden-parity read — no
      separate write); (4) **neighbor forgetting ±** — compute each neighbor's shared **semantic**-neuron count
      (spatial/entity excluded, emotion never counted [I3]), map through job 43's `NeighborForgettingDelta`, and apply the
      additive offsets to **neighbors only** (the recalled star gets no self-offset [F5]). Distinct from the `Potentiate`
      primitive it calls (plan 18's seam).
- [ ] T005 Build **`Reconsolidate`** (error branch only, A8): `current_text ← rewriteText`; `seed ← Reshape(seed,
newSeed)` with the fresh entropy minted **here** (the use-case owns randomness; the domain stays pure — plan 32);
      enqueue the **remaining-stage regeneration** job — only not-yet-created `semantic_stages`/`decay_stages` texts are
      regenerated from the new `current_text` via the plan-22 jobs queue (async LLM pass; optimistic write §2.8 — texts
      fill on the next read), **keeping** already-created gist stages ([C7]); append the `reconsolidated/user`
      `memory_provenance` row via `MemoryProvenanceStore` (`text = rewriteText`, post-sync `universe_time`). The `Diary`
      is never written (A9).
- [ ] T006 Build **`Recall`** — the 회고하기 orchestration in one `InTx` (plan 33 §Design steps 1–7): compose
