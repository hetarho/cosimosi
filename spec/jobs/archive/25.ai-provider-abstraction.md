## Acceptance Criteria (from plan/28.ai-provider-abstraction)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 `LLMClient` and `EmbeddingClient` capability interfaces exist in `internal/ai`, **declared next to their consumers**
      (the plan-22 port adapters); the port adapters import **no vendor SDK type**, and no provider client imports a
      memory domain type, a prompt string, or a port DTO [§2.2/§2.4].
- [x] A2 The `LLMClient` request carries an **output JSON schema** and every provider adapter enforces it via its native
      structured-output mechanism — the schema-forced Extractor shape ([W4a]) survives a provider swap with **zero**
      change to port adapters or `internal/memory`.
- [x] A3 **Exactly one** concrete provider per capability is implemented in Epic A — Anthropic (`LLMClient`) and Voyage AI
      (`EmbeddingClient`), exact model ids confirmed at implement — and the remaining slots (OpenAI, DeepSeek, Z.ai,
      Gemini; OpenAI/Gemini embeddings) require only a new adapter + factory case, no consumer change.
- [x] A4 Provider selection is composition-root wiring from runtime env, **independently per capability**: key present →
      that provider (metered); key absent → the keyless deterministic mock; unknown provider name → **startup error**,
      never a silent default.
- [x] A5 Vendor errors normalize to the shared typed set — rate-limited · auth-failed · cost-capped ·
      malformed-structured-output — and plan 22's job backoff / RPC error mapping consume only the typed set (no vendor
      error type escapes `internal/ai`).
- [x] A6 Cost metering (`ai.per_call_token_cap`, `ai.daily_call_cap`, identical-input caching) wraps the capability
      interfaces and applies **uniformly to every provider**; the mock is unmetered.
- [x] A7 Every `EmbeddingClient` yields vectors of dimension `ai.embedding_dim`; a provider/model that cannot honor it
      fails at startup wiring, not at insert time.
- [x] A8 No `spec/values.yaml` key is introduced: provider names, model ids, API keys, and base URLs are runtime
      env/secrets only.

## Review

- [x] `/code-review` (3 finder angles) + `codex -m gpt-5.5` merged.
  - **Applied:** (1) metering seam cached the raw response _before_ the port adapter parsed it — a schema-conforming but
    semantically-invalid response (bad mood enum, empty stage) poisoned the identical-input cache and broke retry
    recovery (job-24 parity regression); fixed with an optional `Validate` hook the port adapter passes on the request,
    which the seam runs before caching. (2) Voyage assigned vectors by loop index without checking the returned indices
    are contiguous `0..n-1` — a duplicated/gapped index would silently misalign embeddings; now rejected as
    malformed-structured-output. (3) Anthropic `mapError` mapped transient `408`/`425` to terminal auth-failed instead
    of retryable rate-limited. (4) `Adapters.Mode` mislabeled an injected client with the env provider name; now reports
    `real`.
  - **Rejected:** (a) "auth-failed is documented terminal but the jobqueue retries it" — real, but `internal/platform/jobqueue`
    is outside this job's blast radius and the typed error is correctly produced/consumed; a terminal fast-path is a
    plan-22 backoff change. (b) "client only checks `json.Valid`, not full schema" — Anthropic structured outputs enforce
    the schema server-side, so `json.Valid` is a sufficient defensive check; deeper semantic validation is deliberately
    the port adapter's job. (c) "charge happens before the provider call, billing failures" — preexisting job-24
    behavior (limit attempts, not successes), not a regression. (d) "no single-flight for concurrent identical calls" —
    the caching guarantee is per-completed-call; the worker serializes jobs and the spec does not require single-flight.
