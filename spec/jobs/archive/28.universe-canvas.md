## Acceptance Criteria (from plan/23.universe-canvas)

<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->

- [x] A1 `widgets/universe-canvas` **mounts** `@cosimosi/3d-renderer`'s canvas host + skin/post pipeline (plan 14) and
      composes its scene inside it; it does **not** re-bootstrap the renderer (no `init`/`present`/resize/color/pixel-ratio,
      no skin system, no post pipeline) and imports `three` only through the package's seams — never directly.
- [x] A2 The three domain-mirror entities `entities/{episodic-memory,neuron,synapse}` exist, are **pure (no three import)**,
      each with a `model` (FE domain type + Zustand store) and an `api` segment that maps the `GetUniverse` proto DTOs
      (`EpisodicMemoryDto`/`NeuronDto`/`SynapseDto`) → FE domain types.
- [x] A3 `GetUniverse` is a **unary, HTTP GET-eligible, `NO_SIDE_EFFECTS`, per-user-scoped** read returning the stored facts
      (emotion, `base_strength`, `recall_count`, created/last-recalled universe time, seed, `neuron_type`, per-neuron
      connectivity/degree, synapse strength + co-activation, activation edges) plus the current `universe_time`; it carries
      **no** positions/coordinates [I5] and no read-time-derived values.
- [x] A4 The projected graph renders as **`InstancedMesh` nodes + GPU line-segment edges** (as-built: one
      `InstancedMesh` per node kind + plain `THREE.LineSegments`; the `Line2` fat-line + `instance_bucket_size` bucketing
      are deferred to plan 24); edges connect **neurons only** (never memory↔memory) [I4][I6].
- [x] A5 `packages/force-sim` runs in a **Web Worker**; the renderer reads the coordinate `Float32Array` into
      `InstancedMesh.matrix` / uniforms in `useFrame`, and the scene is **not** driven by 60 fps React state (no per-frame
      React re-render, no per-frame store read; per-frame control reads use `getSnapshot()`).
- [x] A6 A product **camera/navigation rig** provides zoom · rotate · pan · focus · fly and **replaces** plan 14's demo orbit
      camera for the real universe; camera + selection modes are an XState machine in the widget's `model/` segment.
- [x] A7 Layout **centers on self/identity** with **radius = connectivity** [V1][I7]; positions are **emergent and never
      stored/persisted** by this unit [I5]; the z-layer 2-tier band is shown as **3D depth**, with Epic A populating the
      hippocampus layer [V9].
- [x] A8 The universe is the app's **main page**: web `/` → `UniverseHomePage` (through plan 15's typed route registry, replacing
      plan 14's placeholder home) and mobile `UniverseScreen` (through plan 13's typed navigation registry), rendering the
      real graph.
- [x] A9 Web + mobile share the widget, machine, graph builder, and entity `model`/`api` **verbatim**; `*.native.tsx` siblings
      exist **only** for the canvas host + gesture/pointer input; mobile MVP may run lower instance counts / fewer post-FX.
- [x] A10 The star/neuron/synapse shader bodies, latent field, nebula, writing flow, and recall/detail panels are **not**
      built here (nodes are generic instances, edges generic filaments); the visual bodies arrive through plan 14's
      asset-source port so plan 24 refines them without changing this widget.
