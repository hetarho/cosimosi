## Acceptance Criteria (from plan/19.force-sim-layout)
<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->
- [x] A1 `packages/force-sim` is a pure module with **no** DOM, `three`, React, or native imports, and is consumed
  identically by `apps/web` and `apps/mobile`.
- [x] A2 It exposes a `tick(dt)` that takes node + edge arrays (neurons, synapses with strength, episodic memories,
  neuron activations with weights, per-neuron connectivity) and returns a `Float32Array` of coordinates; the renderer
  reads that buffer into `InstancedMesh.matrix` / uniforms and does not drive the tick from React state.
- [x] A3 Positions are **never** persisted, never sent to the server, and never a proto field; the server owns
  neurons / synapses / activations / centrality facts only ([I5], [ARCHITECTURE §2.9](../ARCHITECTURE.md#29-forbidden-patterns) #8,
  [§4](../ARCHITECTURE.md#4-cross-cutting-patterns)).
- [x] A4 A node's **radius from center** is a function of its connectivity / self-relevance (well-connected → near
  center); the **center is self/identity, not time** ([V1], [I7]).
- [x] A5 An episodic-memory node settles at the **neuron-weighted centroid** of its neurons — stronger (higher
  activation weight) neurons pull harder ([E8]) — and has no independent position force.
- [x] A6 There is **no** force, edge, or spring between two episodic-memory nodes; memory proximity is emergent from
  shared neurons + synapse strength ([I4], [I6], [L6], [L7]).
- [x] A7 **Emotion never affects layout**: valence / arousal / mood / intensity are not accepted as inputs and have no
  path into position or connection ([I3], [M7]).
- [x] A8 New nodes start from a **seed**: co-created neurons seed near each other, a reused existing neuron seeds near
  its existing position, and the final position emerges from the forces ([E7], [I5]); baseline **repulsion** prevents
  nodes from collapsing into one mass ([E11]).
- [x] A9 All computed layout is confined to the **hippocampus z-band** (`force_sim.hippocampus_z_min`..`hippocampus_z_max` =
  0..10); the **neocortex z-band** (15..25) is reserved as bounds + the "copy x,y, raise z" coordinate contract with
  no force-sim, and Epic A places nothing there ([C5], [C6], [V9]).
- [x] A10 Given the same input graph + `force_sim.seed` + tick sequence, the module produces **identical** coordinates
  on web and mobile, verified by **golden fixtures** — a client-only determinism guarantee, not a server position
  authority ([I5], brief §7).
- [x] A11 The simulation is designed to run in a **Web Worker** off the render thread with Barnes-Hut O(N log N)
  neighbor forces (its worker-friendly API and buffer contract are honored by the worker host in
  [23.universe-canvas](../plan/23.universe-canvas.md)).

- [x] T003 Define the graph **input types + output contract** (`src/graph.ts`): neurons (with a per-neuron
  connectivity/degree fact), synapses (undirected neuron↔neuron edges with strength 0..1), episodic memories, neuron
  activations (memory→neuron with a co-membership weight), optional per-node **seed hint**. **No emotion, color, or
  time-derived fields exist on the types** — layout is structurally incapable of consuming them ([I3]). Output = one
  interleaved `Float32Array` (`[x, y, z, …]`) plus the **node-index map owned and returned by this module** (covering
  neuron nodes *and* memory-star nodes); **decide + record the exact packing now** (plan defers it to implement) with
  the contract: renderer copies straight into `InstancedMesh.matrix` / uniforms, zero per-frame allocation. Inputs
  structured-cloneable, output transferable (worker-friendly). Types are this module's own — the domain-mirror →
  graph mapping is the consumer's job (plan 23), never proto directly.
- [x] T004 [P] Implement the **seeded PRNG** (`src/rng.ts`) keyed by `force_sim.seed` — a small deterministic
  generator (e.g. mulberry32/sfc32) driving **every** stochastic choice; no `Math.random` anywhere in the package.
  Same seed + same call sequence → identical doubles on any JS runtime (golden-parity substrate).
- [x] T005 Implement **seed placement** (`src/seed.ts`): neurons co-created in one episodic memory seed **near each
  other**; a reused existing neuron seeds **at/near its current position** (stability across launches); an external
  seed hint (e.g. the latent-star "awaken" anchor from plan 25) is honored as **a starting point only**; otherwise
  draw from the seeded PRNG. Seeds are initial conditions — final positions emerge from the forces ([E7], [I5]).
- [x] T006 Implement the **force set** (`src/forces.ts` + `src/barnes-hut.ts`): `center_strength` pull toward the
  origin modulated by synapse-weighted connectivity (radius = self-relevance; center = self/identity, not time
  [V1], [I7]); `link_distance` springs along synapse edges + `charge` interaction (angle/cluster from shared neurons);
  baseline `repulsion` via a **Barnes-Hut O(N log N)** approximation (pattern separation [E11]); episodic-memory
  nodes pulled to the **activation-weighted centroid** of their neurons — stronger neurons pull harder, no
  independent memory force, and **no memory↔memory force/edge/spring exists in the code** ([E8], [I4], [L7]). All
  coefficients read from `VALUES.forceSim`.
- [x] T007 Implement the simulation factory + `tick(dt)` (`src/simulation.ts`): create the stateful sim from
  (graph, `VALUES.forceSim`); one deterministic integration step per `tick(dt)` with `tick_alpha_decay` cooling to a
  settled layout; **z is a settled force output clamped/biased into the hippocampus band**
  (`hippocampus_z_min`..`hippocampus_z_max`) — genuinely 3D from Epic A. Export the **neocortex band bounds as the
  reserved coordinate contract only** (`neocortex_z_min`..`neocortex_z_max`, "copy x,y, raise z" — Epic E); **no
  simulation runs there — no task, reserved**. Active pattern-separation of similar constellations: **no task —
  reserved slot** ([E11]); worker plumbing (message protocol, transferables, lifecycle): **no task — owned by plan 23**.
- [x] T008 Assemble the public API (`src/index.ts`): the factory, `tick(dt)`, graph/buffer/node-index types, and the
  exported z-band bounds. Worker-friendly by construction: structured-cloneable in, transferable `Float32Array` out;
  nothing imports DOM, `three`, React, or native modules.
- [x] T009 **Golden fixtures** (`src/fixtures/*.json` + `src/determinism.test.ts`): commit canonical
  (graph, seed) → coordinate-buffer pairs; assert byte-identical output for same graph + `force_sim.seed` + tick

  re-implementation) determinism pin — client-only, **not** a server position authority.
- [x] T010 Layout-rule tests (`src/simulation.test.ts`): higher connectivity → smaller settled radius (monotonic);
  an episodic memory settles at its neurons' activation-weighted centroid, and heavier weights pull harder; two
  memories sharing no neurons feel no mutual attraction; co-created neurons start clustered / a reused neuron starts
  at its prior position; repulsion keeps distinct nodes from collapsing to one point; every output z lies within the
  hippocampus band and nothing is placed in the neocortex band.
- [x] T011 Purity / worker-safety verification (`src/purity.test.ts` + grep): no `document`/`window`/`three`/
  `react`/`react-native` import anywhere in `packages/force-sim` (its `package.json` depends only on

## Grounding
- Constitution ([I1]–[I11]): [00.overview](../plan/00.overview.md) §3 (*The constitution*). This unit is bound by
  **[I5]** (positions emergent, never stored), **[I7]** (center = self; radius = self-relevance, not time),
  **[I3]** (emotion never drives position/connection), **[I4]** (no memory↔memory attraction/edge), and
  **[I6]** (synapses are neuron↔neuron only).
- Architecture (placement): [ARCHITECTURE.md](../ARCHITECTURE.md) §3.5 — `packages/force-sim` is a **pure cross-app
  package** (no Vite/Metro/DOM/native deps; both apps consume it from the start). §3.3 — sim off the render thread:
  a pure `tick(dt)` in a Web Worker, Barnes-Hut O(N log N), renderer only *reads* coords. §2.5 — a domain service is
  a pure function over the aggregates. §2.9 #8 + §4 — no server-authoritative client-derived state; derived, not
  stored. Invoke `/fe-architecture` for the surfaces this job touches.
- tech / policy / values touched: **values — the new `force_sim.*` group** (`charge`, `link_distance`,
  `center_strength`, `repulsion`, `tick_alpha_decay`, `hippocampus_z_min`, `hippocampus_z_max`, `neocortex_z_min`,
  `neocortex_z_max`, `seed`) generated via `pnpm gen:values` (T001). **tech/** — the as-built layout rules
  ("positions emergent, client-derived, never stored; pure `tick(dt)` off the render thread; determinism via seeded
  PRNG + golden fixtures") land on completion — extend `tech/rendering.md` or add `tech/force-sim.md`, decided at
  implement, **not pre-written**. **policy/ux** — "layout is emergent from relational structure only — emotion never
  touches position or connection" lands on completion. Depends on
  [16.memory-aggregate-schema](../plan/16.memory-aggregate-schema.md) alone (owns the neuron / synapse / activation /

  [24.star-neuron-rendering](../plan/24.star-neuron-rendering.md) (positions bodies from the buffer).
