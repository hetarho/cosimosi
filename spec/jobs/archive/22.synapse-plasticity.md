## Acceptance Criteria (from plan/18.synapse-plasticity)
<!-- The source spec's acceptance criteria. After building, verify each is true in the running code. -->
- [x] A1 `Potentiate(strength, rate)` is **monotone non-decreasing** in `strength` for `rate ∈ [0, 1]`, `strength ∈ [0, 1]`:
  `Potentiate(s, r) ≥ s`.
- [x] A2 `Potentiate` exhibits **diminishing returns**: for a fixed `rate`, the increment `Potentiate(s, r) − s` is
  non-increasing in `s` (the step shrinks as strength grows), and repeated application **asymptotes to** `synapse.strength_cap`.
- [x] A3 `Potentiate` **never exceeds the cap**: `Potentiate(s, r) ≤ synapse.strength_cap` for all valid inputs, and
  `synapse.strength_cap = 1.0`; potentiating a synapse already at `1.0` returns `1.0`.
- [x] A4 The strength cap is a **single value** (`synapse.strength_cap = 1.0`) shared by all signal kinds; no per-signal cap
  exists — per-signal differences appear only in initials (and, if used, the rate).
- [x] A5 `InitialStrength` returns a value for each of the three `SignalKind`s, strictly ordered
  `InitialStrength(same-memory) > InitialStrength(shared-neuron) > InitialStrength(temporal)`, and all three are
  `> 0` and `< synapse.strength_cap` (a low start).
- [x] A6 `Depress(strength, amount)` weakens a single synapse (result `≤ strength`), **clamped at the floor `0.0`** (never
  negative), and is defined and named **distinctly** from `Downscale` — this plan implements no `Downscale`; `Downscale`
  (SHY, global/sleep) is Epic E ([I9]). The `Depress` seams for recall neighbour-inhibition ([R5]) and deletion
  contribution-weakening ([X]) are documented but their policy is not implemented here.
- [x] A7 `EffectiveSynapseStrength(base, elapsedUniverseDays)` satisfies `EffectiveSynapseStrength(base, 0) = base`, is **non-increasing**
  in `elapsedUniverseDays`, and stays within `[0, base]` (decays toward but not below `0`, never mutating the stored
  base). It is governed by `synapse.strength_decay_per_day`.
- [x] A8 The Go functions (`internal/memory`) and the TS mirror (`packages/memory-logic`) produce **identical outputs** for a
  shared golden-fixture table within a fixed tolerance (TS ↔ Go golden-parity), and both read the same generated
  `synapse.*` constants.
- [x] A9 All four functions are **pure**: deterministic, IO-free (no clock, DB, transport, SDK, or randomness), and operate
  only on their arguments and generated constants — verified by their placement in the pure domain layer and by the
  parity tests being runnable with no infrastructure.
- [x] A10 No formula, curve shape, or clamp value is stored in `spec/values.yaml`; only the coefficients, cap, and initials
  listed in the plan's Policy / Values Impact are values (formulas stay in code).
- [x] A11 The **memory-level** `EffectiveStrength(base_strength, recall_count)` and `EffectiveBrightness(…)` stubs are defined
  here, **distinct** from the synapse `EffectiveSynapseStrength`; in Epic A `EffectiveStrength(base, 0) = base` and
  `EffectiveBrightness` is full, with recall-accumulation (Epic C) and forgetting-decay (Epic D) reserved as seams and
  introducing no new `values.yaml` key.
