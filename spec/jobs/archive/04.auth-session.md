- [x] A1 Web and mobile have an auth provider/facade that exposes session status, user id, token access for transport, and auth actions without leaking Supabase objects into feature slices.
- [x] A2 The session lifecycle is represented by an XState v5 machine whose context contains only serializable control data and ids.
- [x] A3 RPC clients attach bearer tokens for protected calls and can be tested with an in-memory auth adapter.
